# 如何正确地显示随机消息?

## 核心问题场景

英语学习App首页需要随机显示单词,随着单词表变大,`ORDER BY RAND()`性能急剧下降,影响首页打开速度。



### 2. 三种随机排序优化方案(实践核心)

#### 方案1:主键范围随机(不严格随机)
```sql
SELECT MAX(id), MIN(id) INTO @M, @N FROM t;
SET @X = FLOOR((@M - @N + 1) * RAND() + @N);
SELECT * FROM t WHERE id >= @X LIMIT 1;
```

**扫描行数**: 3行
**缺陷**: ID有空洞时概率不均匀(如ID=4的概率是其他行的2倍)

#### 方案2:行数偏移随机(严格随机)
```sql
SELECT COUNT(*) INTO @C FROM t;
SET @Y = FLOOR(@C * RAND());
SET @sql = CONCAT("SELECT * FROM t LIMIT ", @Y, ",1");
PREPARE stmt FROM @sql;
EXECUTE stmt;
DEALLOCATE PREPARE stmt;
```

**扫描行数**: C + Y + 1行
**优势**: 概率严格均匀
**代价**: 比方案1高,但远低于ORDER BY RAND()

#### 方案3:多次偏移随机(取多个值)
```sql
SELECT COUNT(*) INTO @C FROM t;
SET @Y1 = FLOOR(@C * RAND());
SET @Y2 = FLOOR(@C * RAND());
SET @Y3 = FLOOR(@C * RAND());
-- 执行3次LIMIT Y,1
```

**扫描行数**: C + (Y1+1) + (Y2+1) + (Y3+1)
**优化思路**: 按Y值排序后,利用WHERE id > 上一个id减少扫描

### 配置参数
- `tmp_table_size`: 内存临时表大小限制(默认16M)
- `sort_buffer_size`: 排序缓冲区大小
- `max_length_for_sort_data`: 控制排序模式选择
- `internal_tmp_disk_storage_engine`: 磁盘临时表引擎(默认InnoDB)

### 执行计划标识
- `Using temporary`: 需要临时表
- `Using filesort`: 需要排序操作

### 验证方法
- 慢查询日志查看`Rows_examined`验证扫描行数
- `OPTIMIZER_TRACE`查看排序算法选择



## 思考问答

1. **为什么内存临时表选择rowid排序而非全字段排序?**
   - 内存表回表仅访问内存,无磁盘开销,优化器优先减少排序行数

2. **优先队列排序何时使用?有何优势?**
   - LIMIT N且N较小时使用,避免归并排序的全表排序浪费,仅维护N行数据的堆

3. **如何优化方案3减少扫描行数?**
   - 将Y1、Y2、Y3排序后,利用WHERE id > 上一个id LIMIT (Y2-Y1),1减少重复扫描
