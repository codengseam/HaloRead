# 开篇词：这一次，让我们一起来搞懂MySQL

## 核心知识清单（20%重点）

### 🔹 MySQL学习三大主线（核心框架）

**事务、索引、锁**是MySQL知识体系的三条核心主线，贯穿整个专栏。

- **事务**：保证数据一致性的基础机制
  - 应用场景：转账操作、订单处理等需要原子性的业务
  - 常见错误：误以为事务自动提交，未显式控制事务边界
  - 思考：为什么转账操作必须用事务？如果不用事务会出现什么问题？

- **索引**：提升查询性能的关键结构
  - 应用场景：高频查询字段、WHERE条件字段、JOIN关联字段
  - 常见错误：过度索引导致写入性能下降，或索引字段选择不当
  - 思考：主键选择递增ID和随机ID，对索引性能有什么不同影响？

- **锁**：并发控制的核心机制
  - 应用场景：高并发场景下的数据一致性保障
  - 常见错误：锁粒度选择不当导致性能问题或死锁
  - 思考：读多写少的场景应该选择什么锁策略？

**实操练习**（30秒落地）：列出你最近项目中用到的3个数据库表，分析每个表是否需要索引、需要什么类型的事务控制、可能存在什么锁竞争。



### 🔹 原理驱动的实践思维

**核心价值**：理解原理后，能做出更优的技术决策

- **SQL编写**：知道语句在数据库端如何执行，性能如何优化
- **架构设计**：明确哪些数据处理让数据库做更好，哪些在缓存层处理更优
- **表结构设计**：建表和建索引时为查询优化做综合考虑
  - 递增主键 vs 随机主键的选择
  - 主键列的选择策略
  - 索引字段的选择与组合

**实践导向**：每个原理说明后配套实践问题，实现"理论→实战"的闭环
