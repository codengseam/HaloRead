# 核心观点

**双指针是处理数组/字符串问题的核心技巧**，通过维护两个（或多个）指针的移动策略，将 O(n²) 的暴力解法优化到 O(n)，是「空间换时间」思想的典型应用。

# 解题思维框架（通用套路）

## 核心模式识别

**什么特征说明应该用双指针？**

✅ **适用场景**：
1. 题目要求时间复杂度优于 O(n²)
2. 涉及**有序数组/字符串**的查找、配对
3. 需要**原地修改**数组（不能使用额外空间）
4. 问题涉及**子数组/子串**（连续区间）
5. 有**回文**、**反转**等对称性操作
6. 需要找到满足某种条件的**分割位置**
7. 链表相关操作（环检测、中点查找等）

❌ **不适用场景**：
- 需要随机访问任意位置的数据
- 数据无序且无法预处理

## 双指针的四大类型

### 类型 1：左右指针（相向双指针）

```
场景：有序数组的两端向中间收敛
特点：一个从左往右，一个从右往左
典型问题：两数之和、三数之和、容器盛水
```

**通用模板**：

```java
// Java 版
int left = 0, right = nums.length - 1;
while (left < right) {
    if (/* 满足条件 */) {
        // 处理结果
        left++;  // 或 right--;
    } else if (/* 条件1 */) {
        left++;
    } else {
        right--;
    }
}
```

```python
# Python 版
left, right = 0, len(nums) - 1
while left < right:
    if # 满足条件:
        # 处理结果
        left += 1  # 或 right -= 1
    elif # 条件1:
        left += 1
    else:
        right -= 1
```

### 类型 2：快慢指针（同向双指针）

```
场景：链表操作或数组去重
特点：快指针探索，慢指针标记有效位置
典型问题：删除重复项、环检测、移除元素
```

**通用模板**：

```java
// Java 版
int slow = 0;
for (int fast = 0; fast < nums.length; fast++) {
    if (/* fast 指向的元素满足条件 */) {
        // 将 fast 的值赋给 slow
        nums[slow] = nums[fast];
        slow++;
    }
}
// slow 就是处理后数组的长度
```

```python
# Python 版
slow = 0
for fast in range(len(nums)):
    if # fast 指向的元素满足条件:
        nums[slow] = nums[fast]
        slow += 1
# slow 就是处理后数组的长度
```

### 类型 3：滑动窗口（可变窗口）

```
场景：寻找满足条件的最长/最短子数组
特点：右指针扩展窗口，左指针收缩窗口
典型问题：无重复字符子串、最小覆盖子串
```

**通用模板**：

```java
// Java 版
int left = 0, right = 0;
while (right < s.length()) {
    // 扩大窗口：将右边元素加入窗口
    char c = s.charAt(right);
    window.put(c, window.getOrDefault(c, 0) + 1);
    right++;

    // 当窗口不满足条件时，收缩窗口
    while (/* 窗口需要收缩的条件 */) {
        char d = s.charAt(left);
        window.put(d, window.get(d) - 1);
        left++;
    }

    // 更新答案（在收缩完成后）
}
```

```python
# Python 版
left = right = 0
while right < len(s):
    # 扩大窗口
    c = s[right]
    window[c] = window.get(c, 0) + 1
    right += 1
    
    # 收缩窗口
    while # 窗口需要收缩的条件:
        d = s[left]
        window[d] -= 1
        left += 1
    
    # 更新答案
```

### 类型 4：背向双指针

```
场景：回文检测、中心扩散

```

}

# 经典例题解析

## 例题 1: [LeetCode 15] 三数之和

- **难度**：Medium
- **题意简述**：给定包含 n 个整数的数组 `nums`，判断 `nums` 中是否存在三个元素 a，b，c ，使得 a + b + c = 0？找出所有和为 0 且**不重复**的三元组。
- **思路分析**：
  1. 先排序（关键！排序后才能使用双指针去重）
  2. 固定第一个数 `nums[i]`，然后用左右指针在 `[i+1, n-1]` 范围内找两数之和等于 `-nums[i]`
  3. 去重策略：
     - 第一个数去重：如果 `nums[i] == nums[i-1]` 则跳过
     - 第二、三个数去重：找到一组解后，跳过相同的左右指针值

- **代码实现**：

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);  // 排序是前提
        
        for (int i = 0; i < nums.length - 2; i++) {
            // 第一个数去重
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            int left = i + 1, right = nums.length - 1;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    
                    // 第二、三个数去重
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    left++;
                    right--;
                } else if (sum < 0) {
                    left++;  // 和太小，增大左指针
                } else {
                    right--;  // 和太大，减小右指针
                }
            }
        }
        return result;
    }
}
```

```python
def threeSum(nums):
    result = []
    nums.sort()  # 排序是前提
    
    for i in range(len(nums) - 2):
        # 第一个数去重
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        
        left, right = i + 1, len(nums) - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total == 0:
                result.append([nums[i], nums[left], nums[right]])
                
                # 第二、三个数去重
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1
                
    return result
```

- **复杂度分析**：
  - 时间复杂度：O(n²)，排序 O(n log n) + 双重循环 O(n²)
  - 空间复杂度：O(1)，不考虑输出结果的空间

- **变体与扩展**：
  - [LeetCode 16](https://leetcode.cn/problems/3sum-closest/) 最接近的三数之和
  - [LeetCode 18](https://leetcode.cn/problems/4sum/) 四数之和（再加一层循环）
  - [LeetCode 259](https://leetcode.cn/problems/3sum-smaller/) 较小的三数之和



## 例题 3: [LeetCode 11] 盛最多水的容器

- **难度**：Medium
- **题意简述**：给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。
- **思路分析**：
  1. 容积 = `min(height[left], height[right]) * (right - left)`
  2. 关键洞察：移动较短的边才可能使容积变大
  3. 证明：如果移动较长的那边，宽度减小，高度不会增加（因为受限于较短边），容积一定变小

- **代码实现**：

```java
class Solution {
    public int maxArea(int[] height) {
        int left = 0, right = height.length - 1;
        int maxWater = 0;
        
        while (left < right) {
            // 计算当前容积
            int currentWater = Math.min(height[left], height[right]) * (right - left);
            maxWater = Math.max(maxWater, currentWater);
            
            // 移动较短的那一边
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        return maxWater;
    }
}
```

```python
def maxArea(height):
    left, right = 0, len(height) - 1
    max_water = 0
    
    while left < right:
        # 计算当前容积
        current_water = min(height[left], height[right]) * (right - left)
        max_water = max(max_water, current_water)
        
        # 移动较短的那一边
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
            
    return max_water
```

- **复杂度分析**：
  - 时间复杂度：O(n)
  - 空间复杂度：O(1)

- **变体与扩展**：

  - 如何推广到三维情况？



## 例题 5: [LeetCode 3] 无重复字符的最长子串

- **难度**：Medium
- **题意简述**：给定一个字符串 `s`，请你找出其中不含有重复字符的**最长子串**的长度。
- **思路分析**（滑动窗口）：
  1. 使用滑动窗口 `[left, right)` 维护当前无重复字符的子串
  2. 用 HashMap 记录每个字符最后出现的位置
  3. 当遇到重复字符时，将 `left` 移动到该字符上一次出现位置的下一个位置

- **代码实现**：

```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        Map<Character, Integer> window = new HashMap<>();
        int left = 0, maxLength = 0;
        
        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            
            // 如果字符已在窗口中，更新 left
            if (window.containsKey(c)) {
                // 注意取 max，防止 left 回退
                left = Math.max(left, window.get(c) + 1);
            }
            
            // 更新字符位置
            window.put(c, right);
            
            // 更新答案
            maxLength = Math.max(maxLength, right - left + 1);
        }
        return maxLength;
    }
}
```

```python
def lengthOfLongestSubstring(s):
    window = {}  # 字符 -> 最后出现的位置
    left = max_length = 0
    
    for right, char in enumerate(s):
        if char in window:
            # 注意取 max，防止 left 回退
            left = max(left, window[char] + 1)
        
        window[char] = right
        max_length = max(max_length, right - left + 1)
    
    return max_length
```

- **复杂度分析**：
  - 时间复杂度：O(n)，每个字符最多被访问两次
  - 空间复杂度：O(min(m, n))，m 是字符集大小

- **变体与扩展**：
  - [LeetCode 438](https://leetcode.cn/problems/find-all-anagrams-in-a-string/) 找到字符串中所有字母异位词

# 常见陷阱与优化

## ❌ 易错点 1：忘记排序（针对相向双指针）

- **问题描述**：在使用左右指针解决「两数之和」类问题时，忘记先对数组排序
- **后果**：无法保证双指针的正确性，可能漏掉解或产生错误解
- **正确做法**：
  ```java
  Arrays.sort(nums);  // 必须先排序！
  ```
- **示例**：LeetCode 15 三数之和必须先排序

## ❌ 易错点 2：去重逻辑遗漏

- **问题描述**：在找所有解的问题中（如三数之和），没有处理重复元素导致结果中出现重复三元组
- **正确做法**：
  ```java
  // 第一层循环的去重
  if (i > 0 && nums[i] == nums[i - 1]) continue;
  
  // 找到解后的去重
  while (left < right && nums[left] == nums[left + 1]) left++;
  while (left < right && nums[right] == nums[right - 1]) right--;
  ```

## ❌ 易错点 3：滑动窗口的更新时机

- **问题描述**：在滑动窗口算法中，不确定应该在扩大窗口时还是收缩窗口后更新答案
- **正确做法**：
  - ✅ **通常在收缩窗口完成后**更新答案（此时窗口满足所有条件）
  - ❌ 不要在扩大窗口时立即更新（此时窗口可能还不满足条件）
  
  ```java
  // 正确模式
  while (right < n) {
      // 1. 扩大窗口
      window.add(s[right]);
      right++;
      
      // 2. 收缩窗口（直到满足条件）
      while (!isValid(window)) {
          window.remove(s[left]);
          left++;
      }
      
      // 3. 更新答案（此时窗口一定合法）
      updateAnswer();
  }
  ```

## ❌ 易错点 4：指针越界

- **问题描述**：没有处理好空数组、单元素等边界情况
- **正确做法**：
  ```java
  // 始终检查边界
  if (nums == null || nums.length == 0) return result;
  // 循环条件要严谨
  while (left < right) { ... }  // 不是 <=
  ```

## ✅ 优化技巧 1：提前终止

- **技巧描述**：在某些问题中可以提前剪枝，减少不必要的计算
- **应用场景**：
  - 三数之和：如果 `nums[i] > 0` 且数组已排序，可以直接 break（后面的数都 > 0，不可能和为 0）
  - 两数之和：如果当前和已经超过目标值且数组有序，可以提前停止
  
  ```java
  // 三数之和的剪枝
  if (nums[i] > 0) break;  // 后续都不可能和为 0
  ```

## ✅ 优化技巧 2：减少 HashMap 操作

- **技巧描述**：在滑动窗口中，可以用数组代替 HashMap（当字符集有限时）
- **应用场景**：
  - 只包含小写字母：`int[] count = new int[26]`
  - ASCII 字符：`int[] count = new int[128]`
  
  ```java
  // 比 HashMap 快
  int[] count = new int[128];
  count[s.charAt(right)]++;
  ```

## ✅ 优化技巧 3：双向遍历

- **技巧描述**：某些问题可以从左到右和从右到左各扫描一次，合并结果
- **应用场景**：
  - 接雨水：分别计算左右两边的贡献
  - 买卖股票：正向和反向各 DP 一次
