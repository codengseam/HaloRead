# 核心观点

**二分搜索的本质是「在有序信息中通过排除法快速缩小搜索范围」**，其应用远超「在有序数组中找目标值」，核心在于识别问题中的**单调性**，将 O(n) 的线性搜索优化到 O(log n)。

# 解题思维框架（通用套路）

## 二分搜索的 3 大应用场景

### 场景 1：在有序数据结构中查找目标值

**典型问题**：
- 在有序数组中查找某个元素是否存在
- 查找元素的第一个/最后一个出现位置
- 查找插入位置（元素不存在时应该插入哪里）

**关键特征**：
- ✅ 数据具有**有序性**（升序或降序）
- ✅ 可以通过比较中间元素与目标值的关系缩小范围
- ✅ 时间复杂度要求优于 O(n)

### 场景 2：查找分割位置（满足条件的边界）

**典型问题**：
- 找到第一个满足/不满足某个条件的位置
- 找到最后一个满足/不满足某个条件的位置
- 寻找峰值、旋转点等特殊位置

**关键特征**：
- ✅ 存在一个**临界点**，一侧满足条件，另一侧不满足
- ✅ 条件具有**单调性**（如果 mid 满足，则某一侧的所有元素都满足/不满足）
- ✅ 本质是在答案空间上二分

### 场景 3：在答案集上二分（最小化最大值/最大化最小值）

**典型问题**：
- 分配问题：将 n 个任务分配给 m 个人，使最大工作时间最小化
- 吃香蕉问题：Koko 以速度 K 吃完所有香蕉的最小时间
- 船运问题：D 天内运送完所有包裹的最小运载能力

**关键特征**：
- ✅ 答案是一个**数值范围** [low, high]
- ✅ 存在**判定函数** check(mid)：判断 mid 是否可行
- ✅ check 函数具有**单调性**：如果 mid 可行，则更大/更小的某些值也可行

## 代码模板（Java 版）

### 模板 1：基础二分查找（查找目标值是否存在）

```java
int binarySearch(int[] nums, int target) {
    // Corner case 处理
    if (nums == null || nums.length == 0) {
        return -1;
    }
    
    int start = 0, end = nums.length - 1;
    
    // 要点1: 使用 start + 1 < end 避免 死循环
    while (start + 1 < end) {
        // 要点2: 防止整数溢出
        int mid = start + (end - start) / 2;
        
        // 要点3: 分情况讨论，mid 不 +1 也不 -1
        if (nums[mid] == target) {
            return mid;  // 找到目标
        } else if (nums[mid] < target) {
            start = mid;  // 目标在右半部分
        } else {
            end = mid;   // 目标在左半部分
        }
    }
    
    // 要点4: 循环结束后单独处理 start 和 end
    if (nums[start] == target) {
        return start;
    }
    if (nums[end] == target) {
        return end;
    }
    return -1;  // 未找到
}
```

## 代码模板（Python 版）

### 模板 1：基础二分查找

```python
def binary_search(nums, target):
    """
    基础二分查找：在有序数组中查找目标值
    
    Args:
        nums: 有序数组（升序）
        target: 目标值
    
    Returns:
        目标值的索引，不存在返回 -1
    """
    # Corner case 处理
    if not nums:
        return -1
    
    start, end = 0, len(nums) - 1
    
    # 使用 start + 1 < end 避免死循环
    while start + 1 < end:
        # Python 不存在整数溢出问题
        mid = (start + end) // 2
        
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            start = mid
        else:
            end = mid
    
    # 循环结束后单独处理 start 和 end
    if nums[start] == target:
        return start
    if nums[end] == target:
        return end
    return -1
```

## 例题 1: [LeetCode 704] 二分查找

- **难度**：Easy
- **题意简述**：给定一个 `n` 个元素**升序排列**的整数数组 `nums` 和一个目标值 `target`。写一个函数搜索 `nums` 中的 `target`，如果目标值存在返回下标，否则返回 `-1`。
- **思路分析**：
  - 这是最基础的二分查找，直接套用模板即可
  - 注意处理 corner case（空数组）
  
- **代码实现**：

```java
class Solution {
    public int search(int[] nums, int target) {
        if (nums == null || nums.length == 0) {
            return -1;
        }
        
        int left = 0, right = nums.length - 1;
        
        while (left <= right) {  // 这里可以用 <= 因为是简单查找
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;  // 可以 +1 因为不会错过
            } else {
                right = mid - 1;  // 可以 -1 因为不会错过
            }
        }
        return -1;
    }
}
```

- **复杂度分析**：
  - 时间复杂度：O(log n)
  - 空间复杂度：O(1)

- **变体与扩展**：
  - 这是所有二分搜索的基础，务必熟练掌握



## 例题 3: [LeetCode 35] 搜索插入位置

- **难度**：Easy
- **题意简述**：给定一个排序数组和一个目标值，如果在数组中找到该目标值则返回索引。如果没有，返回它将会被按顺序插入的位置。要求算法时间复杂度为 O(log n)。
- **思路分析**：
  - 本质就是找第一个 >= target 的位置
  - 如果 target 比所有元素都大，返回 nums.length
  - 可以复用「找第一个位置」的逻辑

- **代码实现**：

```java
class Solution {
    public int searchInsert(int[] nums, int target) {
        if (nums == null || nums.length == 0) return 0;
        
        int start = 0, end = nums.length - 1;
        
        // 找第一个 >= target 的位置
        while (start + 1 < end) {
            int mid = start + (end - start) / 2;
            if (nums[mid] >= target) {
                end = mid;
            } else {
                start = mid;
            }
        }
        
        // 检查 start 和 end
        if (nums[start] >= target) return start;
        if (nums[end] >= target) return end;
        
        // target 比所有元素都大
        return nums.length;
    }
}
```

## 例题 5: [LeetCode 33] 搜索旋转排序数组

- **难度**：Medium
- **题意简述**：整数数组 `nums` 按升序排列，数组中的值**互不相同**。在传递给函数之前，`nums` 在预先未知的某个下标 `k`（0 <= k < nums.length）上进行了**旋转**。例如，`[0,1,2,4,5,6,7]` 在下标 3 处经旋转后可能变为 `[4,5,6,7,0,1,2]`。给你旋转后的数组 `nums` 和一个整数 `target`，如果 `nums` 中存在这个目标值，返回它的下标，否则返回 `-1`。要求算法时间复杂度必须是 O(log n) 级别。
- **思路分析**：
  1. 虽然数组被旋转了，但**一半是有序的**
  2. 通过比较 `nums[mid]` 和 `nums[left]` 判断哪半边有序
  3. 判断 target 是否在有序的那半边，从而决定搜索方向
  
- **代码实现**：

```java
class Solution {
    public int search(int[] nums, int target) {
        if (nums == null || nums.length == 0) return -1;
        
        int left = 0, right = nums.length - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                return mid;
            }
            
            // 判断哪半边是有序的
            if (nums[left] <= nums[mid]) {
                // 左半边有序
                if (nums[left] <= target && target < nums[mid]) {
                    // target 在左半边的有序区间内
                    right = mid - 1;
                } else {
                    // target 在右半边
                    left = mid + 1;
                }
            } else {
                // 右半边有序
                if (nums[mid] < target && target <= nums[right]) {
                    // target 在右半边的有序区间内
                    left = mid + 1;
                } else {
                    // target 在左半边
                    right = mid - 1;
                }
            }
        }
        return -1;
    }
}
```

# 常见陷阱与优化

## ❌ 易错点 1：死循环

- **问题描述**：当 `start` 和 `end` 相邻时，`mid = (start + end) / 2 = start`，如果更新 `start = mid`，就会陷入死循环
- **示例**：
  ```java
  // 错误写法：可能死循环
  while (start < end) {
      int mid = start + (end - start) / 2;
      if (nums[mid] < target) {
          start = mid;  // 当 start + 1 == end 时,mid=start,死循环!
      } else {
          end = mid;
      }
  }
  ```
- **正确做法**：
  ```java
  // 正确写法：使用 start + 1 < end
  while (start + 1 < end) {
      int mid = start + (end - start) / 2;
      if (nums[mid] < target) {
          start = mid;
      } else {
          end = mid;
      }
  }
  // 循环结束后单独处理 start 和 end
  ```

## ❌ 易错点 2：边界处理错误

- **问题描述**：不确定循环结束后应该返回 `start` 还是 `end`，或者不知道何时 `+1/-1`
- **规则总结**：
  ```
  情况1: 简单查找（不涉及重复元素）
  → 可以用 while (left <= right)，更新时 left = mid + 1, right = mid - 1
  
  情况2: 查找边界（涉及重复元素）
  → 用 while (start + 1 < end)，更新时不 +/- 1
  → 最后单独判断 start 和 end
  
  情况3: 答案集二分
  → 用 while (low < high)，通常需要根据具体情况决定是否 +/- 1
  ```

## ❌ 易错点 3：整数溢出

- **问题描述**：计算 `mid = (start + end) / 2` 时，如果 `start + end` 超过 `Integer.MAX_VALUE` 会溢出
- **影响语言**：Java、C++（Python 无此问题）
- **正确做法**：
  ```java
  // 错误：可能溢出
  int mid = (start + end) / 2;
  
  // 正确：防止溢出
  int mid = start + (end - start) / 2;
  
  // 或者使用无符号右移（效率更高）
  int mid = (start + end) >>> 1;
  ```

## ❌ 易错点 4：重复元素的处理

- **问题描述**：数组中有重复元素时，普通的二分查找无法确定返回哪个位置
- **解决方案**：
  - 找第一个位置：使用 `>= target` 作为条件，优先向左收缩
  - 找最后一个位置：使用 `<= target` 作为条件，优先向右收缩
  - 关键区别只在最后判断顺序不同（先判断 start 还是先判断 end）

## ✅ 优化技巧 1：统一模板减少记忆负担

- **技巧描述**：记住一套通用模板，通过微调适应不同场景
- **推荐模板**（适用于90%的情况）：

  ```java
      } else {

      }
  }
  // 后处理
  ```

## ✅ 优化技巧 2：判定函数的设计

- **技巧描述**：对于答案集二分，设计一个好的 `check()` 函数是关键
- **设计原则**：
  1. check(mid) 应该能在合理时间内完成（通常是 O(n) 或 O(n log n)）
  2. check 函数必须具有**单调性**
  3. 边界值的选取要确保答案一定在 [low, high] 范围内

- **示例**（吃香蕉问题 LeetCode 875）：
  ```java
  boolean check(int K) {
      // 判断以速度 K 能否在 H 小时内吃完
      int hours = 0;

          hours += Math.ceil((double) pile / K);
      }
  ```

## ✅ 优化技巧 3：二分答案 vs 二分下标

- **技巧描述**：明确区分两种不同的二分应用方式
- **区别**：
  | 类型 | 搜索对象 | 应用场景 | 复杂度 |
  |------|---------|---------|--------|
  | 二分下标 | 数组索引 | 有序数组查找 | O(log n) |
  | 二分答案 | 数值范围 | 优化问题 | O(log range × check_cost) |
