# 核心观点

栈和队列是**操作受限的线性表**——栈遵循"后进先出(LIFO)"，队列遵循"先进先出(FIFO)"。这种"限制"恰恰是它们的价值所在：**用约束换取语义清晰和正确性保证**。理解栈在函数调用、表达式求值中的应用，以及队列在 BFS、线程池中的使用，能帮你识别何时该用这些结构，并掌握单调栈/单调队列等高级变体。

## 核心概念解析（20% 重点深度解析）

### 1. 栈（Stack）：LIFO 的威力

- **定义**：一种只允许在**一端**（栈顶）进行插入和删除操作的线性数据结构，遵循**后进先出（LIFO, Last In First Out）**原则
- **为什么重要**：
  - ⭐ **函数调用栈的基础**：理解递归、调试堆栈溢出的前提
  - 浏览器前进后退、括号匹配、撤销操作的天然数据结构
  - 后续学习DFS深度优先搜索、回溯算法的必备基础

- **关键机制**：

  **基本操作**（时间复杂度均为O(1)）：
  ```
  push(item): 入栈，将元素压入栈顶
  pop():      出栈，弹出并返回栈顶元素
  peek():     查看栈顶元素但不弹出
  isEmpty():  判断栈是否为空
  ```

  **两种实现方式对比**：

| 特性 | 顺序栈（数组实现） | 链式栈（链表实现） |
|------|-------------------|-------------------|
| 底层结构 | 固定/动态数组 | 单链表 |
| 入栈/出栈 | O(1) | O(1) |
| 扩容问题 | 需要动态扩容 | 天然支持动态大小 |
| 内存连续性 | 是（缓存友好） | 否 |
| 空间开销 | 可能有浪费 | 每节点多一个指针 |

  **支持动态扩容的顺序栈——均摊复杂度分析**：
  ```java
  // 扩容策略：当栈满时申请2倍空间并搬移数据
  push(item):
    if (count == capacity):
        newData = new array[capacity * 2]  // 申请2倍空间
        copy(oldData → newData)            // O(n) 数据搬移
        capacity *= 2
    items[count++] = item                  // O(1) 正常入栈

  // 均摊分析：
  // 假设当前容量K已满，第K+1次push触发扩容（成本K）
  // 接下来K-1次push都是O(1)
  // 总成本 = K + (K-1) ≈ 2K
  // 均摊每次 = 2K/K = O(1) ✅
  ```

- **时间/空间复杂度**：
  - 时间：push/pop/peek 都是 **O(1)**（均摊）
  - 空间：O(n)，其中n为栈中元素个数
- **实际应用场景**：

  **① 函数调用栈**（最核心的应用）
  ```
  操作系统为每个线程分配独立的调用栈存储函数临时变量
  调用流程：
  main() {
      int a = 1;
      ret = add(3, 5);   // 将a, ret压入main的栈帧
                          // 创建add的栈帧（x=3, y=5, sum）
      res = a + ret;     // add返回，add栈帧出栈
  }

  栈帧内容：局部变量、返回地址、参数等
  ⚠️ 递归太深会导致 StackOverflowError（栈空间有限）
  ```

  **② 浏览器的前进后退功能**
  ```
  使用两个栈实现：
  - 栈X：存放可以后退的页面（后退时从X弹出压入Y）
  - 栈Y：存放可以前进的页面（前进时从Y弹出压入X）

  示例流程：
  访问 a→b→c：X=[a,b,c], Y=[]
  后退一次：X=[a,b], Y=[c]
  再后退：X=[a], Y=[c,b]
  前进：X=[a,c], Y=[b]
  访问新页面d：X=[a,c,d], Y=[] （清空Y！）
  ```

  **③ 括号匹配检查**
  ```
  输入："{[()()]}"  → 合法 ✅
  输入："{[}]"      → 非法 ❌

  算法：
  遇到左括号 → 压入栈
  遇到右括号 → 弹出栈顶比对是否匹配
  结束后栈为空 → 合法；否则非法

  应用：代码编辑器语法检查、JSON/XML格式验证
  ```

  **④ 表达式求值**（编译器原理）
  ```
  表达式：3 + 5 * 8 - 6

  使用两个栈：
  - 操作数栈：[3, 5, 8, 6]
  - 运算符栈：[+, *, -]

  规则：
  遇到数字 → 压入操作数栈
  遇到运算符 → 与栈顶比较优先级
    - 当前 > 栈顶 → 压入运算符栈
    - 当前 ≤ 栈顶 → 弹出运算符+2个操作数计算，结果压入操作数栈
  ```
- **关联知识**：[[T04_递归与分治]]（函数调用栈）, [[T13_回溯算法]]（显式栈替代递归）

### 2. 队列（Queue）：FIFO 的秩序

- **定义**：一种只允许在**一端（队尾）插入**、**另一端（队头）删除**的线性数据结构，遵循**先进先出（FIFO, First In First Out）**原则
- **为什么重要**：
  - **BFS广度优先搜索**的核心数据结构
  - 线程池、消息队列、任务调度系统的底层模型
  - 现实生活中的排队场景（买票、打印机任务）的直接抽象

- **关键机制**：

  **基本操作**（时间复杂度O(1)）：
  ```
  enqueue(item): 入队，添加到队尾
  dequeue():     出队，移除并返回队头元素
  peek():        查看队头元素
  isEmpty():     判断队列是否为空
  ```

  **实现方式**：

  **① 顺序队列（数组实现）的问题**
  ```
  普通顺序队列的问题：假溢出
  - 数组前面有空位但无法使用（因为dequeue后head前移了）
  解决方案：循环队列（取模运算复用空间）
  ```

  **② 循环队列**（推荐）
  ```
  关键公式：
  - 入队：(tail + 1) % capacity
  - 出队：(head + 1) % capacity
  - 判空：head == tail
  - 判满：(tail + 1) % capacity == head  （牺牲一个空间）

  优点：空间利用率高，不会出现假溢出
  ```

  **③ 链式队列**
  ```
  使用链表实现，天然支持动态扩容
  维护head和tail指针，入队操作O(1)，出队操作O(1)

  应用：Java LinkedList（实现了Deque接口）
  ```

- **时间/空间复杂度**：
  - 时间：enqueue/dequeue 都是 **O(1)**
  - 空间：O(n)
- **实际应用场景**：

  **① 线程池中的任务队列**
  ```
  Java ThreadPoolExecutor：
  - corePoolSize：核心线程数
  - workQueue：任务队列（BlockingQueue）

  提交流程：
  1. 当前线程 < corePoolSize → 创建新线程执行
  2. 当前线程 ≥ corePoolSize → 任务放入workQueue排队
  3. 队列满且线程 < maxPoolSize → 创建非核心线程
  4. 都满 → 执行拒绝策略（AbortPolicy/DiscardPolicy等）

  常见队列类型：
  - ArrayBlockingQueue：有界数组循环队列
  - LinkedBlockingQueue：无界链式队列（默认）
  - SynchronousQueue：不存储元素，直接传递
  ```

  **② BFS广度优先搜索**
  ```
  典型应用：二叉树层序遍历、迷宫最短路径、社交网络好友推荐

  算法模板：
  def bfs(root):
      queue = [root]
      visited = set([root])
      while queue:
          node = queue.dequeue()
          process(node)
          for neighbor in node.neighbors:
              if neighbor not in visited:
                  queue.enqueue(neighbor)
                  visited.add(neighbor)
  ```

  **③ 消息队列中间件**
  ```
  RabbitMQ/Kafka/RocketMQ：
  - Producer → Queue → Consumer
  - 解耦生产者和消费者
  - 削峰填谷（缓冲突发流量）
  - 异步处理提高响应速度
  ```
- **关联知识**：[[T10_图论基础]]（BFS）, [[T08_二叉树]]（层序遍历）

### 3. 单调栈与单调队列（高级应用）

- **定义**：
  - **单调栈**：栈内元素保持**单调递增或递减**的栈
  - **单调队列**：队列内元素保持**单调递增或递减**的队列
- **为什么重要**：
  - ⭐ **面试高频考点**（LeetCode 739.每日温度、239.滑动窗口最大值）
  - 能将某些O(n²)的暴力解法优化到**O(n)**
  - 解决"下一个更大/更小元素"、"滑动窗口极值"等问题

- **关键机制**：

  **① 单调栈——找下一个更大元素**
  ```
  问题：给定数组[2,1,2,4,3]，找每个元素的下一个更大元素
  输出：[4,2,4,-1,-1]

  算法（单调递减栈）：
  遍历数组，维护一个单调递减的栈：
  - 当前元素 > 栈顶 → 栈顶的下一个更大元素就是当前元素，弹出栈顶
  - 当前元素 ≤ 栈顶 → 压入栈

  过程模拟：
  i=0(val=2): stack=[2]
  i=1(val=1): 1≤2 → stack=[2,1]
  i=2(val=2): 2>1 → 弹出1, result[1]=2; 2≤2 → stack=[2,2]
  i=3(val=4): 4>2 → result[2]=4; 4>2 → result[0]=4; stack=[4]
  i=4(val=3): 3≤4 → stack=[4,3]
  结束：剩余元素的result=-1

  时间复杂度：O(n)（每个元素最多进出栈一次）
  空间复杂度：O(n)
  ```

  **② 单调队列——滑动窗口最大值**
  ```
  问题：数组[1,3,-1,-3,5,3,6,7]，窗口大小k=3，求每个窗口的最大值
  输出：[3,3,5,5,6,7]

  算法（单调递减队列）：
  维护一个单调递减的双端队列（存下标而非值）：
  1. 新元素从队尾进入，删除所有比它小的元素（保证单调性）
  2. 如果队首下标超出窗口范围，从队头弹出
  3. 队首元素即为当前窗口最大值

  时间复杂度：O(n)（每个元素最多进出队列一次）
  对比暴力法：O(n×k)
  ```
- **时间/空间复杂度**：
  - 时间：**O(n)**（看似多层循环，实际每个元素只操作常数次）
  - 空间：**O(n)**（最坏情况下栈/队列存n个元素）
- **实际应用**：
  - **股票交易**：找到未来价格更高的那天卖出（Next Greater Element）
  - **直方图最大矩形面积**（LeetCode 84）：单调栈经典难题
  - **网络流量监控**：滑动窗口统计QPS/TPS峰值
  - **编译器语法分析**：匹配嵌套结构的层级关系
- **关联知识**：[[T01_数组]], [[T05_排序算法]]

## 次要信息速览（80% 快速带过）

- 【双端队列（Deque）】：
  - 两端都可以进出的队列（ArrayDeque / LinkedList in Java）
  - 可以当作栈、队列、双端队列使用
  - 应用：实现滑动窗口、回文检查

- 【优先级队列（Priority Queue）】：
  - 不是FIFO，而是按**优先级**出队（通常用堆实现）
  - 应用：任务调度（操作系统进程调度）、Dijkstra最短路径算法、Huffman编码
  - Java中：`PriorityQueue`（默认最小堆）

- 【栈 vs 队列选择指南】：
  - 用栈：需要反转顺序（撤销操作）、配对匹配（括号）、DFS
  - 用队列：保持原有顺序（BFS、任务调度）、缓冲区（生产者-消费者）
  - 用双端队列：两端都需要操作（滑动窗口、回文串）

- 【JVM内存中的栈 vs 堆】：
  - **栈内存**：存储局部变量、方法调用（我们讲的这种栈）
  - **堆内存**：存储对象实例（通过new创建）
  - 栈空间小但速度快，堆空间大但有GC开销

# 思考问答

1. **如何用两个栈实现一个队列？要求enqueue和dequeue都是O(1)均摊复杂度？**
   - 提示：一个栈负责入队，另一个栈负责出队，当出队栈为空时批量转移

2. **如何用两个队列实现一个栈？push和pop的时间复杂度是多少？**
   - 提示：每次pop时将一个队列的元素转移到另一个队列，留下最后一个

3. **以下场景应该用栈还是队列？为什么？**
   - 浏览器的Undo/Redo功能
   - 打印机的打印任务管理
   - 函数调用的返回地址记录
   - 操作系统的进程调度

4. **单调栈解决"直方图中最大的矩形"问题的思路是什么？**
   - 提示：对于每个柱子，向左右扩展找到第一个比它矮的柱子作为边界

5. **循环队列中为什么要牺牲一个空间来判断队列是否已满？有没有其他方法？**
   - 提示：考虑增加size变量或使用sentinel标记

# 实践要点

- ✅ **明确使用场景**：需要LIFO选栈，FIFO选队列，不要混用
- ✅ **注意边界条件**：栈空不能pop，队列空不能dequeue，提前判断isEmpty()
- ✅ **掌握循环队列**：面试常考实现，牢记`(tail+1)%capacity`取模技巧
- ✅ **练习单调栈/队列**：这是区分初级和中级的分水岭（LeetCode 739/239/84）
- ❌ **用数组实现队列不考虑假溢出**：必须使用循环队列或链式队列
- ❌ **忽略栈空间限制**：递归过深会导致StackOverflowError，改用迭代或增大栈大小
- ❌ **线程安全**：多线程环境下使用ConcurrentLinkedQueue/BlockingQueue而非普通实现
