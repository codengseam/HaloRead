# 核心观点

动态规划（DP）是**"将问题分解为重叠子问题，通过记录子问题的解避免重复计算"**的优化方法。从双十一凑单的"暴力枚举→递归→记忆化搜索→DP表格"的演进路径，到DP四步法（定义状态、状态转移方程、初始化、遍历顺序）的标准化流程，体现了**"用空间换时间，自底向上构建解"**的核心思想。理解最优子结构（全局最优包含子问题最优）、无后效性（当前状态不受未来决策影响）、重复子问题（相同子问题被多次计算）这三大特性，能帮你区分DP与贪心（只看局部）、回溯（穷举+剪枝）的边界。掌握编辑距离、LCS、0-1背包、LIS等经典案例和空间压缩技巧，是解决复杂优化问题的核心能力。

## 核心概念解析（20% 重点深度解析）

### 1. DP 的核心思想与三大特性

- **定义**：Dynamic Programming 通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法
- **为什么重要**：
  - ⭐ **面试中的绝对重灾区**：DP题目占比极高，且区分度强
  - **从暴力到高效的思维跃迁**：理解算法优化的本质

- **关键机制——从递归到 DP 的完整演进**：

  **① 案例引入：斐波那契数列**
  ```
  问题：求第 n 个斐波那契数
  定义：F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)

  【版本1：纯递归】（指数级）
  int fib(int n) {
      if (n <= 1) return n;
      return fib(n-1) + fib(n-2);
  }

  调用树（n=5）：
                  fib(5)
                /        \
          fib(4)          fib(3)
         /      \        /      \
     fib(3)  fib(2)   fib(2)  fib(1)
    /    \    /    \   /    \
  fib(2)fib(1)fib(1)fib(0)...

  问题：大量重复计算！
  fib(3) 计算了2次，fib(2)计算了3次...
  时间复杂度：O(2^n)


  【版本2：记忆化搜索】（线性）⭐ Top-down DP
  int[] memo = new int[n+1];
  Arrays.fill(memo, -1);              // -1 表示未计算

  int fibMemo(int n) {
      if (n <= 1) return n;
      if (memo[n] != -1) return memo[n]; // 缓存命中！直接返回

      memo[n] = fibMemo(n-1) + fibMemo(n-2);
      return memo[n];
  }

  时间复杂度：O(n)（每个值只算一次）
  空间复杂度：O(n)（memo数组 + 递归栈）

  关键改进：计算过一次就记住，下次直接查表


  【版本3：迭代 DP】（线性）⭐ Bottom-up DP
  int fibDP(int n) {
      if (n <= 1) return n;

      int[] dp = new int[n+1];
      dp[0] = 0;                         // base case
      dp[1] = 1;                         // base case

      for (int i = 2; i <= n; i++) {
          dp[i] = dp[i-1] + dp[i-2];     // 状态转移方程
      }
      return dp[n];
  }

  时间复杂度：O(n)
  空间复杂度：O(n)

  改进：消除了递归的开销


  【版本4：空间压缩】（线性+常数空间）
  int fibOptimized(int n) {
      if (n <= 1) return n;

      int prev2 = 0;                     // 相当于 dp[i-2]
      int prev1 = 1;                     // 相当于 dp[i-1]

      for (int i = 2; i <= n; i++) {
          int curr = prev1 + prev2;      // dp[i]
          prev2 = prev1;                 // 滚动更新
          prev1 = curr;
      }
      return prev1;
  }

  时间复杂度：O(n)
  空间复杂度：O(1) ✅ 最优！

  总结演进路径：
  暴力递归 O(2^n)
    ↓ 加缓存
  记忆化搜索 O(n) [Top-down]
    ↓ 去递归化
  迭代DP O(n) [Bottom-up]
    ↓ 观察依赖关系
  空间压缩 O(1) [滚动数组]
  ```

  **② DP 三大特性详解**
  ```
  特性1：最优子结构（Optimal Substructure）
  ────────────────────────────────────────
  定义：原问题的最优解包含其子问题的最优解

  直观解释：
  如果要从北京到上海的最短路线经过济南，
  那么"北京到济南"这一段也必须是最短的。
  否则可以用更短的那段替换，得到更短的全程路线。

  验证方法（Cut-and-Paste 证明）：
  假设最优解 A 包含子问题 S 的解 S'
  如果 S' 不是 S 的最优解，设 S* 是 S 的最优解
  用 S* 替换 S'，得到新解 A'
  则 A' 比 A 更优，矛盾！
  所以 S' 必须是最优的 ✅

  反例（不满足最优子结构）：
  长钢条切割问题——如果每段切割有固定成本，
  则最优策略可能不是由子问题最优策略组合而成。


  特性2：无后效性（No Aftereffect）
  ────────────────────────────────────
  定义：一旦某个阶段的状态确定，此后过程的演变
        不受之前各种状态和决策的影响

  也称为"马尔可夫性"（Markov Property）

  直观解释：
  "过去怎么样不重要，重要的是现在在哪里"
  在迷宫中，你只需要知道当前位置，不需要知道怎么来的

  示例（满足无后效性）：
  最短路径问题：dp[i][j] = 从起点到(i,j)的最短距离
  一旦到达(i,j)，之前的路径不影响后续选择

  反例（不满足无后效性）：
  某些博弈问题：当前收益可能取决于之前的选择序列


  特性3：重复子问题（Overlapping Subproblems）
  ─────────────────────────────────────────
  定义：在求解过程中，相同的子问题会被反复计算

  这是使用DP的根本原因！
  如果没有重复子问题，分治就够了（如归并排序）

  判断方法：
  画出递归调用树 → 观察是否有重复节点

  斐波那契的递归树中，fib(3)出现2次，fib(2)出现3次...
  这就是典型的重复子问题！
  ```

### 2. DP 四步法：标准解题框架

- **定义**：一套系统化的 DP 解题方法论，适用于绝大多数 DP 问题
- **为什么重要**：
  - ⭐ **面试答题的标准流程**：让思路清晰、表达有序
  - **避免遗漏边界条件和初始化错误**

- **关键机制——四步详解与案例演示**：

  **① Step 1：定义状态（最关键的一步！）**
  ```
  核心问题：dp 数组（或dp表）的含义是什么？

  常见状态定义模式：

  模式A：一维 dp[i]
  含义：前 i 个元素/到第 i 个位置 ... 的最优值
  例：爬楼梯 dp[i] = 到第 i 阶的方法数
      最大子数组和 dp[i] = 以第 i 个元素结尾的最大和

  模式B：二维 dp[i][j]
  含义：前 i 个... 前 j 个... 的最优值
  例：LCS dp[i][j] = text1前i个与text2前j个的LCS长度
      编辑距离 dp[i][j] = word1前i个转word2前j个的最少操作

  模式C：高维 dp[i][j][k]...
  含义：多个维度的约束
  例：0-1背包 dp[i][w] = 前i个物品容量w时的最大价值
      买卖股票III dp[i][k][0/1] = 第天k次交易持有/不持有的最大利润

  💡 技巧：问自己 "我想知道什么信息才能做出决策？"
  这个信息就是状态的含义
  ```

  **② Step 2：推导状态转移方程**
  ```
  核心问题：如何从已知的子状态推导出当前状态？

  思考方式：
  "最后一步做了什么选择？"

  以爬楼梯为例：
  问题：每次可以走1阶或2阶，走到第n阶有多少种方法？

  状态：dp[i] = 走到第 i 阶的方法数

  最后一步分析：
  要到达第 i 阶，最后一步只有两种可能：
  a) 从第 i-1 阶走 1 步上来 → 有 dp[i-1] 种方法
  b) 从第 i-2 阶走 2 步上来 → 有 dp[i-2] 种方法

  所以：dp[i] = dp[i-1] + dp[i-2]

  Base Case：
  dp[0] = 1 （有一种方法：原地不动）
  dp[1] = 1 （只有一种方法：走1步）


  以0-1背包为例：
  问题：n个物品，重量w[]，价值v[]，容量W，最大价值？

  状态：dp[i][j] = 用前 i 个物品，容量为 j 时的最大价值

  最后一步分析（对物品 i）：
  a) 不选第 i 个物品：dp[i][j] = dp[i-1][j]
  b) 选第 i 个物品（如果放得下）：dp[i][j] = dp[i-1][j-w[i]] + v[i]

  取两者最大值：
  dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

  Base Case：
  dp[0][j] = 0 （0个物品，价值为0）
  dp[i][0] = 0 （容量为0，什么都装不下）
  ```

  **③ Step 3：初始化（Base Case）**
  ```
  为什么重要？
  - 状态转移需要"起点"
  - 错误的初始化会导致整个 DP 表错误

  常见初始化模式：

  模式1：dp[0] = 0 或 1
  例：最大子数组和 dp[0] 可以定义为 nums[0] 或 0
       取决于是否允许空子数组

  模式2：dp 全部初始化为无穷大（求最小值时）
  例：编辑距离 dp[i][0] = i（i次删除操作）
       dp[0][j] = j（j次插入操作）

  模式3：dp 全部初始化为 0（求最大值时）
  例：LCS dp[i][0] = 0, dp[0][j] = 0

  模式4：dp 全部初始化为 -1 或 false
  例：某些可行性判断问题

  ⚠️ 注意：初始化要保证状态转移的正确执行
  ```

  **④ Step 4：确定遍历顺序**
  ```
  核心原则：当计算 dp[i] 时，它所依赖的子问题必须已经被计算过！

  一维 DP 的遍历：
  - 通常从左到右：for i = 1 to n
  - 特殊情况：从右到左（如某些博弈问题）

  二维 DP 的遍历：

  情况A：dp[i][j] 依赖 dp[i-1][j] 和 dp[i][j-1]
  → 正序遍历即可（先行后列或先列后行都可以）

  情况B：dp[i][j] 只依赖上一行
  → 可以做空间压缩（滚动数组）

  情况C：dp[i][j] 依赖 dp[i+1][j]（后续状态）
  → 需要逆序遍历

  检查方法：
  写出状态转移方程 → 圈出右边的所有依赖项
  → 确保这些依赖项在当前项之前被计算
  ```

### 3. 经典 DP 案例：编辑距离

- **定义**：Levenshtein Distance，衡量两个字符串相似度的指标，定义为将一个字符串转换为另一个字符串所需的最少单字符操作次数
- **为什么重要**：
  - ⭐ **DP 二维表的经典范例**：状态定义、转移方程、初始化都非常典型
  - **实际应用广泛**：拼写检查、DNA序列比对、 diff 工具

- **关键机制——编辑距离的完整推导**：

  **① 问题定义**
  ```
  给定两个字符串 word1 和 word2
  允许三种操作：
  - 插入（Insert）一个字符
  - 删除（Delete）一个字符
  - 替换（Replace）一个字符

  每种操作的代价都是 1
  求：word1 → word2 的最小操作次数

  示例：
  word1 = "horse", word2 = "ros"

  horse → rorse (替换 h → r)
  rorse → rose  (删除 r)
  rose  → ros   (删除 e)
  操作次数 = 3
  ```

  **② 状态定义与转移方程**
  ```
  状态定义：
  dp[i][j] = word1 的前 i 个字符转换为 word2 的前 j 个字符
             所需的最小操作次数

  其中：
  - i 的范围：0 到 m（m = word1.length()）
  - j 的范围：0 到 n（n = word2.length()）

  状态转移方程（核心！）：

  分析 word1[i-1] 和 word2[j-1] 的关系：

  情况1：word1[i-1] == word2[j-1]（最后一个字符相同）
  ─────────────────────────────────────────────────
  不需要任何操作！直接继承子问题的结果
  dp[i][j] = dp[i-1][j-1]

  图示：
  word1: ... X
  word2: ... X
           ↑ 相同，无需操作


  情况2：word1[i-1] ≠ word2[j-1]（最后一个字符不同）
  ───────────────────────────────────────────────────
  三种操作取最小值：

  a) 删除 word1[i-1]：
     先把 word1[0..i-2] 变成 word2[0..j-1]，然后删除最后一个字符
     dp[i][j] = dp[i-1][j] + 1

  b) 插入 word2[j-1]：
     先把 word1[0..i-1] 变成 word2[0..j-2]，然后插入 word2[j-1]
     dp[i][j] = dp[i][j-1] + 1

  c) 替换 word1[i-1] 为 word2[j-1]：
     先把 word1[0..i-2] 变成 word2[0..j-2]，然后替换
     dp[i][j] = dp[i-1][j-1] + 1

  综合起来：
  dp[i][j] = min(
      dp[i-1][j] + 1,      // 删除
      dp[i][j-1] + 1,      // 插入
      dp[i-1][j-1] + 1     // 替换
  )

  最终方程：
  if (word1[i-1] == word2[j-1]):
      dp[i][j] = dp[i-1][j-1]
  else:
      dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1
  ```

  **③ 完整代码实现**
  ```
  int minDistance(String word1, String word2) {
      int m = word1.length(), n = word2.length();
      int[][] dp = new int[m + 1][n + 1];

      // Base Case 初始化
      for (int i = 0; i <= m; i++) {
          dp[i][0] = i;                   // word1前i个 → 空，需i次删除
      }
      for (int j = 0; j <= n; j++) {
          dp[0][j] = j;                   // 空 → word2前j个，需j次插入
      }

      // 填表
      for (int i = 1; i <= m; i++) {
          for (int j = 1; j <= n; j++) {
              if (word1.charAt(i-1) == word2.charAt(j-1)) {
                  dp[i][j] = dp[i-1][j-1]; // 字符相同，无操作
              } else {
                  dp[i][j] = Math.min(
                      Math.min(dp[i-1][j], dp[i][j-1]),
                      dp[i-1][j-1]
                  ) + 1;
              }
          }
      }

      return dp[m][n];
  }

  时间复杂度：O(m × n)
  空间复杂度：O(m × n)

  DP 表可视化（word1="horse", word2="ros"）：

         ""  r   o   s
      "" [0,  1,  2,  3]
      h  [1,  1,  2,  3]
      o  [2,  2,  1,  2]
      r  [3,  2,  2,  2]
      s  [4,  3,  3,  2]
      e  [5,  4,  4,  3]

  结果：dp[5][3] = 3 ✅
  ```

### 4. 其他经典 DP 问题速览

- **定义**：一系列具有代表性的 DP 经典问题，每种代表一类 DP 模式
- **为什么重要**：
  - ⭐ **面试题库的主要来源**：掌握经典就能应对变种
  - **训练 DP 思维的最佳素材**

- **关键机制——四大经典问题**：

  **① 最长公共子序列（LCS - Longest Common Subsequence）**
  ```
  问题描述：
  找出两个字符串的最长公共子序列（不要求连续）

  示例：
  text1 = "abcde", text2 = "ace"
  LCS = "ace"，长度 = 3

  状态：dp[i][j] = text1[0..i-1] 与 text2[0..j-1] 的 LCS 长度

  转移方程：
  if (text1[i-1] == text2[j-1]):
      dp[i][j] = dp[i-1][j-1] + 1       // 匹配，LCS长度+1
  else:
      dp[i][j] = max(dp[i-1][j], dp[i][j-1])  // 不匹配，取较大者

  初始化：dp[i][0] = 0, dp[0][j] = 0

  应用：Git diff（文件差异比较）、生物信息学（基因序列相似度）

  时间复杂度：O(m × n)
  空间复杂度：O(m × n)，可压缩至 O(min(m,n))
  ```

  **② 0-1 背包问题**
  ```
  问题描述（前面已提过，这里给完整代码）：

  int knapsack01(int W, int[] wt, int[] val, int n) {
      int[][] dp = new int[n + 1][W + 1];

      for (int i = 1; i <= n; i++) {          // 遍历每个物品
          for (int w = 0; w <= W; w++) {       // 遍历每个容量
              if (wt[i-1] <= w) {              // 能装下
                  dp[i][w] = Math.max(
                      dp[i-1][w],              // 不选
                      dp[i-1][w-wt[i-1]] + val[i-1]  // 选
                  );
              } else {
                  dp[i][w] = dp[i-1][w];        // 装不下，只能不选
              }
          }
      }
      return dp[n][W];
  }

  空间压缩（一维数组）：
  int knapsackCompressed(int W, int[] wt, int[] val, int n) {
      int[] dp = new int[W + 1];

      for (int i = 0; i < n; i++) {
          for (int w = W; w >= wt[i]; w--) {   // ⚠️ 必须逆序！
              dp[w] = Math.max(dp[w], dp[w-wt[i]] + val[i]);
          }
      }
      return dp[W];
  }

  为什么必须逆序？
  - 正序会使用本层已经更新的值（相当于每个物品可用多次→完全背包）
  - 逆序确保使用的是上一层（i-1）的值（每个物品只用一次→0-1背包）
  ```

  **③ 最长递增子序列（LIS - Longest Increasing Subsequence）**
  ```
  问题描述：
  找出最长严格递增子序列的长度

  示例：nums = [10,9,2,5,3,7,101,18]
  LIS = [2,3,7,101]，长度 = 4

  解法1：经典 O(n²) DP
  状态：dp[i] = 以 nums[i] 结尾的最长递增子序列长度

  转移：
  dp[i] = 1  （至少包含自身）
  for j in 0..i-1:
      if nums[j] < nums[i]:
          dp[i] = max(dp[i], dp[j] + 1)

  result = max(dp[i]) for all i

  解法2：优化的 O(n log n) + patience sorting
  维护一个 tails 数组：tails[x] = 长度为 x+1 的递增子序列的最小末尾值

  for num in nums:
      在 tails 中找到第一个 >= num 的位置（二分查找）
      如果找到了，替换该位置的值为 num
      如果没找到（num比所有值都大），追加到tails末尾

  LIS长度 = tails.length

  示例追踪：nums = [10,9,2,5,3,7,101,18]

  10 → tails = [10]
  9  → tails = [9]       （替换）
  2  → tails = [2]       （替换）
  5  → tails = [2, 5]    （追加）
  3  → tails = [2, 3]    （替换）
  7  → tails = [2, 3, 7] （追加）
  101→ tails = [2, 3, 7, 101] （追加）
  18 → tails = [2, 3, 7, 18] （替换）

  LIS长度 = 4 ✅
  ```

## 次要信息速览（80% 快速带过）

- **【DP 分类体系】**：
  ```
  按状态维度分类：
  - 一维 DP：爬楼梯、打家劫舍、最大子数组和
  - 二维 DP：编辑距离、LCS、不同路径、正则表达式匹配
  - 高维 DP：买卖股票系列（天数×交易次数×持仓状态）

  按问题类型分类：
  - 最值型：0-1背包、最大子数组和、LIS
  - 存在型：跳跃游戏、分割等长子串
  - 计数型：不同路径、目标和方法数

  按状态转移方向分类：
  - Forward DP：从前向后推导（大多数情况）
  - Backward DP：从后向前推导（部分博弈问题）
  - 双向 DP：前后同时推导（较少见）
  ```

- **【常见 DP 优化技巧】**：
  | 技巧 | 适用场景 | 效果 |
  |------|---------|------|
  | 滚动数组/空间压缩 | dp[i]只依赖前几行 | O(n²)→O(n)空间 |
  | 单调队列优化 | 滑动窗口最值 | O(n²)→O(n)时间 |
  | 斜率优化/凸优化 | 特定形式的DP | O(n²)→O(n)或O(n log n) |
  | 四边形不等式 | 区间DP | O(n³)→O(n²) |
  | 状态压缩（ bitmask）| 小规模集合DP | 用整数表示集合状态 |

- **【DP 解题 Checklist】**：
  ```
  □ 明确问题要求：最值？存在性？计数？方案数？
  □ 定义 dp 数组的含义（最重要！）
  □ 推导状态转移方程（考虑最后一步的所有选择）
  □ 确定 base case 和初始化值
  □ 确定遍历顺序（确保依赖项先被计算）
  □ 考虑是否可以做空间优化
  □ 用简单例子手动验证 DP 表
  □ 分析时间和空间复杂度
  ```

# 思考问答

1. **编辑距离问题中，如果增加一种"交换相邻两字符"的操作（代价为1），状态转移方程应该如何修改？**
   - 提示：新增一种转移 `dp[i][j] = dp[i-2][j-2] + 1`（当 word1[i-1]==word2[j-2] 且 word1[i-2]==word2[j-1] 时）

2. **0-1背包的空间压缩为什么必须逆序遍历？如果正序遍历会发生什么？这对应了什么其他问题？**
   - 提示：正序=完全背包（每个物品可无限使用），因为会用到当前轮已更新的值

3. **LIS 的 O(n log n) 解法中，tails 数组存储的为什么是"最小末尾值"？请证明这个贪心选择的正确性。**
   - 提示：对于同样长度的递增子序列，末尾值越小，越容易扩展得更长

4. **如何判断一个问题是否适合用 DP 解决？请给出你的判断流程图。**
   - 提示：是否有最优子结构？是否有重叠子问题？是否无后效性？三个条件缺一不可

5. **区间 DP（如矩阵链乘法、回文分割）的状态定义和普通 DP 有何不同？它的遍历顺序有什么特点？**
   - 提示：状态通常是 dp[i][j] 表示区间 [i,j] 的答案，遍历时先枚举区间长度再枚举起点

# 实践要点

- ✅ **熟练掌握 DP 四步法**：状态定义→转移方程→初始化→遍历顺序
- ✅ **能够手推小型 DP 表**：用 3×4 或 4×4 的例子验证思路正确性
- ✅ **掌握经典问题的模板代码**：LIS、LCS、编辑距离、0-1背包、最大子数组和
- ✅ **学会空间压缩的基本技巧**：一维滚动数组、注意遍历顺序
- ✅ **能区分不同类型的 DP 问题**：最值型 vs 存在型 vs 计数型
- ❌ **在没有明确定义状态的情况下就开始写转移方程**：这是最常见的错误来源
- ❌ **忽视 base case 的初始化**：特别是涉及无穷大或特殊值的场景
- ❌ **混淆 0-1 背包和完全背包的遍历顺序**：前者逆序，后者正序
