# 核心观点

图是**"描述实体间多对多关系的数据结构"**，核心价值在于用顶点和边的模型表达社交网络、地图导航、依赖关系等复杂系统。从邻接矩阵的O(V²)空间但O(1)查询到邻接表的O(V+E)空间但需遍历邻接链，体现了**"空间与时间的权衡是数据结构设计的永恒主题"**。DFS的递归深度探索适合路径查找和连通性分析，BFS的层级遍历天然支持无权图最短路径，两者共同构成了图算法的基础工具集。理解图的分类（有向/无向、有权/无权、连通/非连通、DAG）能帮你在实际场景中正确选择表示方式和遍历策略。

## 核心概念解析（20% 重点深度解析）

### 1. 图的存储方式：邻接矩阵 vs 邻接表

- **定义**：图由顶点（Vertex）集合和边（Edge）组成，存储方式决定了空间效率和操作性能
- **为什么重要**：
  - ⭐ **图算法的基础**：所有图算法都建立在存储结构之上
  - **工程选型的关键**：社交网络、地图、编译器依赖分析等场景需要不同的存储方案

- **关键机制——两种存储方式的对比**：

  **① 邻接矩阵（Adjacency Matrix）**
  ```
  结构：二维数组 matrix[V][V]

  无权图示例（4个顶点的无向图）：
       A   B   C   D
    A [ 0   1   1   0 ]
    B [ 1   0   1   1 ]
    C [ 1   1   0   0 ]
    D [ 0   1   0   0 ]

  含义：matrix[i][j] = 1 表示 i 和 j 之间有边
        matrix[i][j] = 0 表示没有边

  有权图：matrix[i][j] 存储权重（如距离、成本）
         没有边时存 ∞ 或特殊值

  操作复杂度：
  - 判断两顶点是否有边：O(1) ⭐
  - 获取某个顶点的所有邻居：O(V)
  - 添加/删除边：O(1)
  - 空间复杂度：O(V²)

  适用场景：
  ✅ 稠密图（E ≈ V²，边很多）
  ✅ 需要频繁判断两点是否相连
  ✅ 顶点数量较少（< 10000）

  实际案例：
  - 微信好友关系（大部分人是好友，稠密）
  - 小型地图的邻接关系
  ```

  **② 邻接表（Adjacency List）**
  ```
  结构：数组 + 链表（或动态数组）

  示例（同上4个顶点的无向图）：

  顶点A → [B, C]
  顶点B → [A, C, D]
  顶点C → [A, B]
  顶点D → [B]

  代码实现（简化版）：
  class Graph {
      int V;                          // 顶点数
      LinkedList<Integer> adj[];      // 邻接表数组

      Graph(int v) {
          V = v;
          adj = new LinkedList[v];
          for (int i = 0; i < v; i++)
              adj[i] = new LinkedList<>();
      }

      void addEdge(int v, int w) {
          adj[v].add(w);              // 无向图需要双向添加
          adj[w].add(v);
      }
  }

  操作复杂度：
  - 判断两顶点是否有边：O(degree(v)) 或 O(V)
  - 获取某个顶点的所有邻居：O(degree(v)) ⭐
  - 添加边：O(1)（链表尾部插入）
  - 空间复杂度：O(V + E)

  适用场景：
  ✅ 稀疏图（E << V²，边很少）
  ✅ 需要频繁遍历邻居
  ✅ 顶点数量很大（百万级）

  实际案例：
  - 微博关注关系（每个人关注的人很少，稀疏）
  - 网页链接关系（每个网页链接数有限）
  - 社交网络的推荐系统
  ```

  **③ 工程选择决策树**
  ```
  选择邻接矩阵的条件：
  1. 图是稠密的（E > V² / 10）
  2. 需要快速判断任意两点是否相连
  3. 图的结构相对稳定（不频繁增删边）
  4. 内存充足

  选择邻接表的条件：
  1. 图是稀疏的（E < V² / 10）
  2. 主要操作是遍历某个顶点的邻居
  3. 需要频繁增删边
  4. 内存受限或顶点数巨大

  混合方案（十字链表/邻接多重表）：
  - 同时支持高效的有向图操作
  - 用于特定领域（如编译器的流图分析）
  ```

### 2. 深度优先搜索（DFS）：递归的艺术

- **定义**：从起点出发，沿着一条路走到底，遇到死胡同再回溯换路
- **为什么重要**：
  - ⭐ **图算法的核心基础**：拓扑排序、环检测、连通分量、路径查找
  - **理解递归思维的最佳实践**：栈的隐式调用 vs 显式模拟

- **关键机制——DFS 的两种实现方式**：

  **① 递归实现（简洁优雅）**
  ```
  void dfsRecursive(Graph graph, int v, boolean[] visited) {
      visited[v] = true;
      System.out.print(v + " ");           // 处理当前节点

      for (int neighbor : graph.adj[v]) {  // 遍历所有邻居
          if (!visited[neighbor]) {
              dfsRecursive(graph, neighbor, visited);  // 递归访问
          }
      }
  }

  时间复杂度：O(V + E)（每个顶点和边各访问一次）
  空间复杂度：O(V)（递归栈深度，最坏情况是一条长链）

  执行过程示例（从A开始）：

  图结构：A-B, A-C, B-D, B-C

  DFS(A):
  ├── 访问 A，标记 visited[A]=true
  ├── 遍历 A 的邻居 [B, C]
  │   ├── B 未访问 → DFS(B)
  │   │   ├── 访问 B，标记 visited[B]=true
  │   │   ├── 遍历 B 的邻居 [A, D, C]
  │   │   │   ├── A 已访问，跳过
  │   │   │   ├── D 未访问 → DFS(D)
  │   │   │   │   ├── 访问 D，标记 visited[D]=true
  │   │   │   │   └── D 的邻居 [B] 都已访问，返回
  │   │   │   └── C 未访问 → DFS(C)
  │   │   │       ├── 访问 C，标记 visited[C]=true
  │   │   │       └── C 的邻居都已访问，返回
  │   │   └── 返回
  │   └── C 已访问，跳过
  └── 完成

  输出顺序：A → B → D → C
  ```

  **② 栈实现（显式控制）**
  ```
  void dfsIterative(Graph graph, int start) {
      boolean[] visited = new boolean[graph.V];
      Stack<Integer> stack = new Stack<>();

      stack.push(start);
      visited[start] = true;

      while (!stack.isEmpty()) {
          int v = stack.pop();
          System.out.print(v + " ");

          for (int neighbor : graph.adj[v]) {
              if (!visited[neighbor]) {
                  stack.push(neighbor);
                  visited[neighbor] = true;  // 入栈时就标记！
              }
          }
      }
  }

  注意事项：
  ⚠️ 入栈时必须立即标记 visited，否则同一节点可能被多次入栈
  ⚠️ 邻居的入栈顺序会影响遍历结果（但不影响正确性）
  ⚠️ 递归实现是「先处理当前节点」，栈实现也可以改为「后处理」
  ```

  **③ DFS 的经典应用**
  ```
  应用1：拓扑排序（Topological Sort）
  场景：确定代码文件的编译依赖顺序、课程学习顺序

  Kahn算法（基于BFS）：
  1. 统计每个节点的入度
  2. 将入度为0的节点加入队列
  3. 取出队列头部节点，输出；其邻居入度减1
  4. 若邻居入度变为0，加入队列
  5. 重复直到队列为空

  DFS算法（后序遍历 + 逆序）：
  1. 对每个未访问节点执行DFS
  2. 节点所有邻居访问完毕后，将该节点加入结果列表
  3. 最终结果列表的逆序即为拓扑排序

  时间复杂度：O(V + E)

  应用2：检测有向环（Cycle Detection）
  方法：在DFS中使用三个状态
  - WHITE(0): 未访问
  - GRAY(1): 正在访问（在递归栈中）
  - BLACK(2): 已完成访问

  如果遇到 GRAY 状态的节点，说明存在环！

  应用3：求连通分量（Connected Components）
  方法：对每个未访问节点启动DFS，每次DFS覆盖的所有节点构成一个连通分量

  应用4：路径查找（Path Finding）
  方法：DFS + 回溯，记录当前路径，找到目标时返回
  ```

### 3. 广度优先搜索（BFS）：层级遍历的威力

- **定义**：从起点出发，逐层向外扩展，先访问距离近的节点再访问远的
- **为什么重要**：
  - ⭐ **无权图最短路径的标准解法**：BFS的层级特性天然保证找到最短路径
  - **社交网络分析的基石**：六度分隔理论、好友推荐

- **关键机制——BFS 的完整流程**：

  **① 基础实现**
  ```
  void bfs(Graph graph, int start) {
      boolean[] visited = new boolean[graph.V];
      Queue<Integer> queue = new LinkedList<>();

      queue.offer(start);
      visited[start] = true;

      while (!queue.isEmpty()) {
          int v = queue.poll();
          System.out.print(v + " ");

          for (int neighbor : graph.adj[v]) {
              if (!visited[neighbor]) {
                  queue.offer(neighbor);
                  visited[neighbor] = true;  // 入队时标记！
              }
          }
      }
  }

  时间复杂度：O(V + E)
  空间复杂度：O(V)（队列最多存储一层节点）

  执行过程示例（从A开始的无权图）：

  图结构及边权重（假设都是1）：
  A --1-- B --1-- D
  |      /
  1    /
  |  /
  C

  BFS 层级展开：
  Level 0: [A]                    ← 距离为0
  Level 1: [B, C]                 ← 距离为1（A的直接邻居）
  Level 2: [D]                    ← 距离为2（B的邻居，C无新邻居）

  输出顺序：A → B → C → D
  最短路径：A→D 的最短距离 = 2（A→B→D）
  ```

  **② BFS 求无权图最短路径**
  ```
  扩展：记录距离和前驱节点

  int[] distance = new int[graph.V];     // 到起点的距离
  int[] predecessor = new int[graph.V];  // 前驱节点（用于重建路径）
  Arrays.fill(distance, -1);
  Arrays.fill(predecessor, -1);

  void bfsShortestPath(Graph graph, int start) {
      Queue<Integer> queue = new LinkedList<>();
      queue.offer(start);
      distance[start] = 0;

      while (!queue.isEmpty()) {
          int v = queue.poll();
          for (int neighbor : graph.adj[v]) {
              if (distance[neighbor] == -1) {  // 未访问
                  distance[neighbor] = distance[v] + 1;
                  predecessor[neighbor] = v;
                  queue.offer(neighbor);
              }
          }
      }
  }

  重建路径（从 end 到 start）：
  List<Integer> path = new ArrayList<>();
  for (int cur = end; cur != -1; cur = predecessor[cur]) {
      path.add(cur);
  }
  Collections.reverse(path);  // 反转得到 start → end 的路径

  示例结果（上图）：
  distance[A]=0, distance[B]=1, distance[C]=1, distance[D]=2
  predecessor[B]=A, predecessor[C]=A, predecessor[D]=B
  A→D 的路径：[A, B, D]
  ```

  **③ BFS vs DFS 对比总结**

  | 维度 | DFS | BFS |
  |------|-----|-----|
  **数据结构** | 栈（递归/显式） | 队列 |
  **遍历顺序** | 深度优先（一条路走到底） | 广度优先（逐层扩展） |
  **空间复杂度** | O(h)，h=树的高度（最好O(log n)，最坏O(n)） | O(w)，w=最大层宽度 |
  **最短路径** | ❌ 不保证（除非用其他方法） | ✅ 保证（无权图） |
  **适用场景** | 路径查找、拓扑排序、环检测 | 最短路径、层级遍历、社交网络分析 |
  **实现难度** | 递归版简单，迭代版稍复杂 | 相对直观 |

### 4. 图的分类体系

- **定义**：根据边的性质不同，图可以分为多种类型，每种类型对应不同的算法和应用
- **为什么重要**：
  - ⭐ **正确选择算法的前提**：DAG才能拓扑排序，有权图才用Dijkstra
  - **建模现实世界的基础**：不同场景需要不同的图模型

- **分类体系一览**：
  ```
  ┌─────────────────────────────────────────────────────┐
  │                      图的分类                       │
  ├─────────────┬─────────────┬─────────────────────────┤
  │  方向性     │   权重性    │     连通性              │
  ├─────────────┼─────────────┼─────────────────────────┤
  │ • 无向图    │ • 无权图    │ • 连通图（无向）        │
  │   (Undirected)│   (Unweighted)│ • 强连通图（有向）     │
  │             │             │ • 弱连通图（有向）       │
  │ • 有向图    │ • 有权图    │ • 非连通图              │
  │   (Directed) │   (Weighted)│                         │
  │             │             │                         │
  │ 特例：      │ 特例：      │ 特例：                  │
  │ • DAG       │ • 完全图    │ • 二分图                │
  │ (有向无环图) │             │ • 欧拉图                │
  │             │             │ • 哈密顿图              │
  └─────────────┴─────────────┴─────────────────────────┘

  详细说明：

  【方向性】
  无向图：边没有方向，A-B 等价于 B-A
  - 例：Facebook 好友关系（双向关注）
  - 存储：邻接表双向添加边

  有向图：边有方向，A→B 不等价于 B→A
  - 例：Twitter 关注关系（单向）、网页链接、依赖关系
  - 存储：邻接表只按方向添加边
  - 重要概念：
    * 入度(In-degree)：指向该节点的边数
    * 出度(Out-degree)：从该节点出发的边数

  DAG（有向无环图）：
  - 定义：有向图且不存在环
  - 重要性：可以拓扑排序！
  - 应用：任务调度、版本控制(Git)、包依赖管理

  【权重性】
  无权图：边只有"有/无"的区别
  - 例：网页链接（存在或不存在）
  - 最短路径：BFS（边数最少）

  有权图：边带有数值属性（距离、成本、容量等）
  - 例：地图导航（道路长度）、网络路由（延迟）
  - 最短路径：Dijkstra（非负权）、Bellman-Ford（含负权）

  【连通性】
  连通图（无向图中）：
  - 任意两个顶点之间都存在路径
  - 非连通图可以有多个连通分量

  强连通图（有向图中）：
  - 任意两个顶点 u,v 之间存在 u→v 和 v→u 的路径
  - 例：环形交通网络

  特殊图类型：
  二分图：顶点可分为两组，组内无边，边只在两组之间
  - 例：用户-商品关系、男女配对问题
  - 判断方法：BFS染色法

  完全图：每对顶点之间都有边
  - n个顶点的完全图有 n(n-1)/2 条无向边
  - n个顶点的完全有向图有 n(n-1) 条有向边
  ```

## 次要信息速览（80% 快速带过）

- 【图的常用术语】：
  - **顶点（Vertex/Node）**：图中的基本单元，也称为节点
  - **边（Edge/Arc）**：连接两个顶点的线段
  - **度（Degree）**：与顶点相连的边数（无向图）；入度+出度（有向图）
  - **路径（Path）**：顶点序列，相邻顶点间有边
  - **环（Cycle）**：起点和终点相同的路径
  - **简单路径**：不重复经过顶点和边的路径
  - **连通分量（Connected Component）**：极大连通子图

- 【图的遍历注意事项】：
  ```
  1. 必须使用 visited 数组避免重复访问和无限循环
  2. 对于非连通图，需要对每个未访问节点启动遍历
  3. DFS 可能导致栈溢出（递归太深），改用显式栈或增加系统栈大小
  4. BFS 的队列可能很大（如星形图），注意内存限制
  ```

- 【常见图算法时间复杂度速查】：
  | 算法 | 时间复杂度 | 备注 |
  |------|-----------|------|
  | BFS/DFS 遍历 | O(V+E) | 基础遍历 |
  | 拓扑排序（Kahn） | O(V+E) | DAG专用 |
  | Dijkstra（二叉堆） | O((V+E)logV) | 非负权最短路 |
  | Bellman-Ford | O(VE) | 支持负权 |
  | Floyd-Warshall | O(V³) | 全源最短路 |
  | Prim（二叉堆） | O((V+E)logV) | 最小生成树 |
  | Kruskal | O(E log E) | 最小生成树 |
  | A* | O(E log V) | 启发式搜索 |

- 【社交网络应用案例】：
  ```
  三度好友关系（BFS应用）：
  - 以自己为起点进行BFS
  - Layer 1：直接好友（一度）
  - Layer 2：好友的好友（二度）
  - Layer 3：三度好友
  - 统计Layer 3的节点数即为三度好友数量

  微博/微信存储选择：
  - 微博（关注关系，稀疏）：邻接表
  - 微信（好友关系，较稠密）：可考虑邻接矩阵或混合方案
  ```
