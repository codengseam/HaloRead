## 题目信息

**题目信息**：LeetCode 232. Implement Queue using Stacks / 剑指 Offer 09

*   **原题链接**：[Implement Queue using Stacks](https://leetcode.cn/problems/implement-queue-using-stacks/)
*   **难度**：Easy
*   **描述**：请你仅使用两个栈实现先入先出队列。队列应当支持一般队列支持的所有操作（push、pop、peek、empty）。

### 🛒 题目 "人话" 翻译

这道题就像是 **"倒腾水杯"** 的游戏。

*   **栈（Stack）**：就像一个 **"羽毛球筒"**，只能从上面塞进去，也只能从上面拿出来（后进先出，LIFO）。
*   **队列（Queue）**：就像一个 **"排队通道"**，先来的人先办业务（先进先出，FIFO）。

**你的任务**：给你两个羽毛球筒（栈），让你模拟出一个排队通道（队列）的效果。也就是说，你塞进去的顺序是 1, 2, 3，拿出来的顺序也得是 1, 2, 3。



### 💡 思路 "推导" 过程

我们来模拟一下小白的思考过程：

**第一阶段：简单粗暴** 🐢
*   **想法**：我有两个栈，那我每次 `push` 的时候，就把所有元素倒腾一遍，保证新来的沉底？
*   **结局**：可以是可以，但是太累了！每次进一个人都要把所有人挪一遍位置，效率太低。

**第二阶段：懒人智慧（双栈分工）** 🐇
*   **想法**：平时我不倒腾，只有在 **"要取东西"**（Pop）的时候，如果出货区（栈 B）是空的，我才把进货区（栈 A）的东西一股脑倒过去。
*   **规则**：
    1.  **入队**：直接扔进 **栈 A**。
    2.  **出队**：
        *   如果 **栈 B** 里有货，直接拿。
        *   如果 **栈 B** 空了，把 **栈 A** 里的所有货一次性"倒"进 **栈 B**，然后再拿。
*   **结局**：非常高效！每个元素只会被"倒腾"一次（从 A 进，从 A 出来进 B，从 B 出），平均时间复杂度是 O(1)。



### 🚗 代码 "带逛"

```python
class MyQueue:
    """
    用两个栈模拟队列
    """
    def __init__(self):
        # 准备两个羽毛球筒
        # stack_in 只负责接收新来的（进货区）
        self.stack_in = []
        # stack_out 只负责往外拿（出货区）
        self.stack_out = []

    def push(self, x: int) -> None:
        """
        将元素 x 推到队列的末尾
        """
        # 有新元素来了？直接扔进进货区，不管别的
        self.stack_in.append(x)

    def pop(self) -> int:
        """
        从队列的开头移除并返回元素
        """
        # 1. 如果出货区还有货，直接拿走，不用麻烦进货区
        if self.stack_out:
            return self.stack_out.pop()
        
        # 2. 如果出货区空了，看看进货区有没有货
        # 如果进货区也空了，那整个队列就真没货了
        if not self.stack_in:
            return -1 # 或者抛出异常
            
        # 3. 进货区有货，出货区没货 -> 执行"倒腾"操作
        # 把 stack_in 的东西全部倒进 stack_out
        # 就像把一个杯子里的水倒进另一个杯子
        while self.stack_in:
            val = self.stack_in.pop()
            self.stack_out.append(val)
            
        # 4. 倒腾完了，stack_out 的最上面就是最老的元素，拿走它！
        return self.stack_out.pop()

    def peek(self) -> int:
        """
        返回队列开头的元素（只看不拿）
        """
        # 逻辑和 pop 一样，只是最后一步不删除元素
        # 这里复用一下代码逻辑，先拿到队头
        ans = self.pop()
        # 因为只是看一眼，所以拿出来还得放回去
        # 注意：放回哪里？放回 stack_out！因为它本来就在出货区待着
        self.stack_out.append(ans)
        return ans

    def empty(self) -> bool:
        """
        只要两个区有一个有货，队列就不为空
        """
        return not self.stack_in and not self.stack_out
```

> 💡 **教练加餐：面试常考点 & 刷题技巧**
> *   **均摊复杂度分析**：面试官可能会问"`pop` 操作里有个 `while` 循环，为什么说它是 O(1) 的？"。
>     *   **答案**：虽然偶尔一次 `pop` 会很慢（需要倒腾），但对于每个元素来说，它一生只会被 `push` 进 A 一次，从 A `pop` 出来一次，`push` 进 B 一次，从 B `pop` 出来一次。平均下来，每个元素的操作次数是常数，所以叫 **均摊 O(1)**。
> *   **工程应用**：这其实就是 **双缓冲队列 (Double Buffering)** 的思想，在很多异步系统、图形渲染中都有应用。一个负责写，一个负责读，互不干扰，效率极高。
