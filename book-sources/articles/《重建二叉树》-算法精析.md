## 题目信息

**题目信息**：LeetCode 105. Construct Binary Tree from Preorder and Inorder Traversal / 剑指 Offer 07

*   **原题链接**：[Construct Binary Tree from Preorder and Inorder Traversal](https://leetcode.cn/problems/construct-binary-tree-from-preorder-and-inorder-traversal/)
*   **难度**：Medium
*   **描述**：给定一棵树的前序遍历 `preorder` 与中序遍历 `inorder`。请构造二叉树并返回其根节点。

### 🛒 题目 "人话" 翻译

这道题就像是 **"乐高积木复原"** 游戏。

*   你把一个拼好的乐高模型（二叉树）摔散了。
*   但是幸运的是，你保留了两份说明书：
    1.  **前序遍历（Preorder）**：就像是 **"零件清单"**，按"头 -> 左手 -> 右手"的顺序记录了所有零件。**最重要的信息是：清单的第一个零件，绝对是整个模型的"头"（根节点）！**
    2.  **中序遍历（Inorder）**：就像是 **"合影照片"**，按"左手 -> 头 -> 右手"的顺序拍了一张照。**最重要的信息是：如果你找到了"头"在哪里，那么照片里"头"左边的就是左手，"头"右边的就是右手。**

**你的任务**：利用这两份说明书，把原来的乐高模型一模一样地拼回去。



### 💡 思路 "推导" 过程

我们来模拟一下小白的思考过程：

**第一阶段：手足无措** 🐢
*   **想法**：给我两个数组 `[3,9,20,15,7]` 和 `[9,3,15,20,7]`，我怎么知道谁连着谁啊？
*   **观察**：前序数组第一个是 `3`。哦！那 `3` 肯定是根节点。
*   **行动**：先把 `3` 造出来。

**第二阶段：顺藤摸瓜** 🐇
*   **想法**：既然 `3` 是根，那我们在中序数组里找找 `3` 在哪。
*   **发现**：中序数组是 `[9, 3, 15, 20, 7]`。`3` 在中间！
*   **推导**：
    *   `3` 左边的 `[9]` 肯定属于左子树。
    *   `3` 右边的 `[15, 20, 7]` 肯定属于右子树。
*   **分配任务**：
    *   左子树只有一个 `9`，简单，直接连上。
    *   右子树有三个 `15, 20, 7`，稍微复杂点，但我们手里也有它们的前序 `[20, 15, 7]`（从前序数组里对应位置截取）。
    *   对右子树重复刚才的过程：`20` 是头，`15` 在左，`7` 在右...

**第三阶段：总结规律（递归）** ✨
*   **核心逻辑**：
    1.  `preorder[0]` 是 **根**。
    2.  在 `inorder` 里找到 **根** 的位置 `idx`。
    3.  `inorder[0:idx]` 是 **左子树**。
    4.  `inorder[idx+1:]` 是 **右子树**。
    5.  递归去构建左右子树。



### 🚗 代码 "带逛"

这里展示最直观的 **"切片递归法"**。虽然切片会增加一些空间消耗，但对于理解算法逻辑是最好的。

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

def buildTree(preorder, inorder):
    """
    根据前序和中序遍历构造二叉树
    :param preorder: List[int] 前序遍历数组
    :param inorder: List[int] 中序遍历数组
    :return: TreeNode 根节点
    """
    # 1. 递归的尽头：如果零件清单空了，说明没有节点要拼了
    if not preorder or not inorder:
        return None
    
    # 2. 找到老大：前序遍历的第一个元素永远是根节点
    root_val = preorder[0]
    root = TreeNode(root_val)
    
    # 3. 在合影（中序）中找到老大的位置，以此划分左右地盘
    # index() 方法可以帮我们找到元素的下标
    mid_index = inorder.index(root_val)
    
    # 4. 拆分中序遍历（合影）
    # 左边的部分是左子树的合影
    inorder_left = inorder[:mid_index]
    # 右边的部分是右子树的合影
    inorder_right = inorder[mid_index+1:]
    
    # 5. 拆分前序遍历（零件单）
    # 左子树的零件数量 = len(inorder_left)
    # 所以在前序数组中，根节点后面紧接着的就是左子树的零件
    preorder_left = preorder[1 : 1+len(inorder_left)]
    # 剩下的就是右子树的零件
    preorder_right = preorder[1+len(inorder_left) :]
    
    # 6. 递归构建左右子树，并接回到根节点上
    root.left = buildTree(preorder_left, inorder_left)
    root.right = buildTree(preorder_right, inorder_right)
    
    return root
```

> 💡 **教练加餐：面试常考点 & 刷题技巧**
> *   **面试进阶（优化版）**：上面的代码虽然好懂，但 `inorder.index()` 和切片操作 `[:]` 会比较慢。面试官可能会问"如何优化？"。
>     *   **答案**：用 **哈希表 (HashMap)** 预先存好 `inorder` 中每个值对应的索引 `val -> index`。这样查找根节点位置就从 O(N) 变成了 O(1)。
>     *   同时，不传切片（拷贝数组），而是传 **下标范围**（比如 `pre_start, pre_end, in_start, in_end`），这样空间复杂度就能降到 O(N)。
> *   **刷题技巧**：只要涉及到"树的还原"问题，**找根节点** 永远是第一步。前序/后序定根，中序定左右。
