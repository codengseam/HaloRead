# 产品本质课 · 写作提示词存档

> 本文件为专栏写作过程使用的提示词与执行方案存档，不进入专栏展示（下划线前缀文件被 check_book_structure.py 豁免、被 build_site.py 排除出站点）。保留用于复盘与迭代。

## 一、专栏定位

- 主题：从第一性原理讲透"产品的本质是什么、如何做好一款产品"
- 读者：产品经理、创业者、想做产品的人、想理解产品思维的非产品人
- 单轴设计：第一性原理 → 用户价值 → 用户任务 → 思维 → 哲学 → 发现 → 设计 → 增长 → 组织 → 避坑 → 结语
  - 先讲"产品是什么"（第一性原理）
  - 再讲"为谁做"（用户价值 + 用户任务）
  - 再讲"怎么想"（思维 + 哲学，认知框架先于方法论）
  - 再讲"怎么做"（发现 + 设计）
  - 再讲"怎么长大"（增长）
  - 再讲"谁来做"（组织）
  - 最后"别踩坑"（避坑）和"收尾"（结语）
- 输出：44 篇正文 + 3 个辅助文件（_meta.yaml / _参考资料.md / _提示词.md）+ 4 个调研存档（_research_*.md）

## 二、执行流程（已落地）

1. **多专家团评估方案**：用 Task 工具启动 3 个 subagent（架构师/内容专家/规则）并行评审大纲，识别 P0×3 + P1×6 + 33 条引用真实性风险 + 16 条数字事实风险，全部吸收到修订计划。
2. **并行调研核验**：用 Task 工具启动 4 个 subagent 按主题簇并行核验误传，产出 `_research_*.md` 4 份（哲学章 / 用户价值+任务论 / 设计+发现 / 增长+组织+思维），含名家原话核实 + 数字事实锁定 + 概念归属易错清单。
3. **整合调研报告后，并行写作**：用 Task 工具启动 6 个 subagent 按章节归属分片并行写作，每片 ≤ 8 篇。
4. **严格质检**：用 Task 工具启动 4 个 subagent 按维度横切 44 篇（真实性/可读性/顺序/引用+灵魂），跑 `check_book_structure.py --output output --strict` 清零 P0/P1/P2。

## 三、写作硬约束（写入每个撰写 subagent 的 query）

### 3.1 路径与命名

1. 目录：`/workspace/output/产品本质课/`（中文名）
2. 文件命名：两级路径 `{chapter}_{event}.md`，中文，下划线 `_` 分隔，禁止用 `-`，禁止子目录，禁止「模块N」前缀
3. chapter 名全部中文（用户任务论 / 第一性原理 / 用户价值论 / 思维 / 哲学 / 发现 / 设计 / 增长 / 组织 / 避坑 / 开篇 / 结语）

### 3.2 frontmatter 八字段（缺一不可）

```yaml
---
title: <情境化大标题，不与 book title 重复，如"第 1 章　为什么重新理解产品">
book: 产品本质课
chapter: <与文件名下划线前部分严格一致>
event: <与文件名下划线后部分严格一致>
sort: <章内事件序号，1-based>
chapter_sort: <章号，1-12>
created_at: 2026-07-07
source_agents:
- product-expert
---
```

- `check_book_structure.py` 硬阻断 6 字段（title/book/chapter/event/sort/chapter_sort），`created_at`/`source_agents` 不阻断但项目惯例要求八字段完整。
- `frontmatter.book` 必须等于目录名 `产品本质课`。
- `frontmatter.chapter` 必须等于文件名 chapter 部分。
- `frontmatter.event` 必须等于文件名 event 部分。
- 同一章所有文件的 `chapter_sort` 必须一致。
- 章内 `sort` 必须从 1 开始连续递增。单事件章节（结语）sort=1。

### 3.3 archetype: modern

- 五段式结构，二级标题固定用 `## 一、入戏` / `## 二、破题` / `## 三、方法论` / `## 四、避坑` / `## 五、践行`（带中文序号，参考形象管理课先例）。
- 正文首个 `#` 为情境化大标题（不与 frontmatter title 重复）。

### 3.4 正文规范

1. 正文中文，每篇 2500-4000 字。
2. 现代白话为主，自然口语化，像樊登/脱不花讲书有节奏、有温度。
3. 场景化叙事：用"会议室、地铁、家庭账本、产品评审会"等读者能代入的画面。
4. 语气随段落变化：入戏生动、破题冷静、方法论清晰、避坑犀利、践行坚定。
5. 每篇开头一句边界声明"本篇覆盖 X，不覆盖 Y（见 XX 篇）"——切割与相邻模块/现有专栏的重叠。

### 3.5 文末规范（字面匹配，不可变体）

每篇文末固定：

```markdown
## 参考来源

- 《书名/篇名》：对应内容
- ...

> 信息截止：2026-07
```

- 文末标题必须字面含「参考来源」四字（`## 参考来源`）。**禁止**用「参考资料」「本章参考资料」「引用」「参考」等变体（content_quality.py `check_sources_section` 正则 `##?\s*参考来源` 只认这四字）。
- 每条来源注明书名/篇名及对应内容，不单列书名。
- 大意转述标注「大意据《XXX》」。

### 3.6 引用规则

1. **古今名家库**（按主题选，每篇至少 2 位）：
   - 产品方法论：俞军、梁宁、Marty Cagan、克里斯坦森、Eric Ries
   - 用户体验：Don Norman、Jesse James Garrett、Steve Krug、Alan Cooper、深泽直人
   - 产品哲学：乔布斯、张小龙、迪特·拉姆斯、贝索斯
   - 增长：Marc Andreessen、Geoffrey Moore、Nir Eyal、Sean Ellis、Fred Reichheld、Everett Rogers
   - 心理/决策：卡尼曼、西奥迪尼、BJ·福格
2. **名家原话三种处理方式**：
   - 方式 A（原文直引）：有明确出处的英文原话 + 中文翻译，标注具体出处（年份+场合）。英文原话单段 ≤1 句话（≤30 单词），中文转述单段 ≤50 字。
   - 方式 B（大意转述）：记不清原话但记得观点，标注「大意据 XXX」。
   - 方式 C（事件转述）：不引原话只述事件，标注「据 XXX 报道」。
3. **禁止行为**：
   - 禁止把转述伪装成原文加引号。
   - 禁止笼统说"贝索斯在致股东信中说"——必须标注具体年份。
   - 禁止把产品 slogan 当成创始人语录（如"再小的个体，也有自己的品牌"是公众号 slogan，不是张小龙个人语录）。
   - 禁止跨语言直译失真。
4. **行内引用密度**：每千字行内引用 ≤ 3 处，同一出处连续出现只保留一处。普通叙述句不挂行内出处。
5. **跨篇重复概念治理**（首篇定义+挂出处，后续篇一笔带过，禁重复挂）：
   - 俞军公式（用户价值=新体验-旧体验-替换成本）：用户价值论首篇挂出处，后续篇用「前面讲过的俞军公式」一笔带过。
   - JTBD/用户任务：用户任务论首篇定义+挂出处，后续篇不重复挂。
   - 牛奶之问案例：只在"焦糖布丁与牛奶之问"篇详述，其余一笔带过。
   - MVP：只在"发现_MVP与精益创业"篇详述定义。
   - 北极星指标：只在"增长_北极星指标"篇定义+挂出处。
   - 用户体验五要素：只在"设计_用户体验五要素"篇详述。

### 3.7 概念归属易错清单（必须在文中显式交代，不可写错）

- "少即是多"（Less is more）= 密斯·凡德罗（**非**乔布斯/拉姆斯）
- "少但更好"（Less but better）= 迪特拉姆斯
- "用户体验五要素"= Jesse James Garrett（**非** Norman）
- "无意识设计"= 深泽直人（**非** Norman）
- "PMF"（产品市场契合）= Marc Andreessen 2007 博客首提（**非** Ries/Thiel）
- "NPS"（净推荐值）= Fred Reichheld 2003 HBR 论文（**非** Bain 公司发明）
- "北极星指标"= Sean Ellis 2011 博客首提（**非** Ries/Thiel）
- "点线面体"= 曾鸣《智能商业》首提（梁宁借用，**非**梁宁原创）
- 跨越鸿沟五类用户比例（2.5%/13.5%/34%/34%/16%）= Everett Rogers 扩散理论（Moore 引用，**非** Moore 原创）
- "Stay hungry, stay foolish"原出自 Stewart Brand《Whole Earth Catalog》1974 停刊号（乔布斯 2005 斯坦福演讲**引用**而非原创）
- "用户价值=新体验-旧体验-替换成本"= 俞军《产品方法论》2019（用词是"替换成本"不是"转换成本"）

### 3.8 数字事实硬约束（必须按核验版本写）

- NPS 公式 = Promoters% - Detractors%（百分比相减，**不是数量相减**）；评分 0-10 分，9-10 为 Promoters，7-8 为 Passives（不计入），0-6 为 Detractors。
- PMF 40% 阈值 = Sean Ellis 2009 调查问卷，问"如果以后不能用这个产品你会非常失望吗"，≥40% 选"非常失望"视为达到 PMF。
- 跨越鸿沟五类用户比例 = 2.5%（创新者）/ 13.5%（早期采用者）/ 34%（早期大众）/ 34%（晚期大众）/ 16%（落后者），合计 100%。
- 上瘾模型四阶段 = Trigger（触发）/ Action（行动）/ Variable Reward（多变酬赏）/ Investment（投入）；Reward 三类型 = Tribe（部落）/ Hunt（猎物）/ Self（自我）。
- 设计冲刺五天 = Monday Map / Tuesday Sketch / Wednesday Decide / Thursday Prototype / Friday Test。
- 用户体验五要素 = 5 层：战略层/范围层/结构层/框架层/表现层。
- 情感化设计三层次 = 本能层/行为层/反思层（Norman《Emotional Design》2003）。
- 设计十诫 = 10 条（不多不少），原文德语，英文标准译本来自 Vitsoe 官网。
- 俞军公式 = 3 项：新体验、旧体验、替换成本。
- JTBD 三维度 = 功能/情感/社会（**非**四维度，警惕有人加 aspirational）。

### 3.9 英文术语全角括号隔断法（16 词不在白名单）

`MODERN_ENGLISH_WHITELIST` 仅含 KPI/OKR/CEO/HR/PR/CFO/CTO/COO/offer/bug/BATNA/CRIB/PPT/DNA/ID/APP/API/PDF/MBA/EMBA/VIP/360度 共 22 词，可裸用。

以下 16 词**不在白名单**，必须采用「全角括号隔断法」：
- 首次出现：中文全称（英文缩写）—— 如「产品市场契合（PMF）」
- 后续出现：一律改用中文全称 —— 如「产品市场契合」
- 禁止：正文中裸写英文缩写（如「PMF 达成后」「DAU 增长」）

16 词清单：PMF / JTBD / NPS / ROI / DAU / MAU / MVP / UX / UI / ARPU / GMV / SEO / ASO / PRD / MRD / BRD。

参考来源区域（`## 参考来源` 段内）可保留英文原书名/原术语，不受此约束。

### 3.10 modern 桶质检红线

1. **「不是X而是Y」句式**：每篇 ≤ 3 处。多余改写为「与其说 X，不如说 Y」「关键不在 X，而在 Y」「X 是表象，Y 是内核」。
2. **大词限用**：「底层逻辑」「底层操作系统」每篇 ≤ 1 次。建议替换为「根本」「本质」「关键」「底子」「根基」。
3. **AI 套话黑名单**（命中 ≥ 3 次扣 5 分）：综上所述、历史的车轮、不仅...更、让我们看到、以史为鉴、在历史的长河中、不禁让人深思、宛如一颗璀璨的明珠、时代的缩影、深刻地揭示了。全部回避。
4. **错别字清单**（28 词）：做为→作为 / 按耐→按捺 / 交待→交代 / 既使→即使 / 那怕→哪怕 / 必竞→毕竟 / 凑和→凑合 / 甘败下风→甘拜下风 / 一愁莫展→一筹莫展 / 美仑美奂→美轮美奂 / 不径而走→不胫而走 / 黄梁美梦→黄粱美梦 / 竭泽而鱼→竭泽而渔 / 棉薄之力→绵薄之力 / 墨守陈规→墨守成规 / 磐竹难书→罄竹难书 / 趋之若骛→趋之若鹜 / 声名雀起→声名鹊起 / 谈笑风声→谈笑风生 / 委屈求全→委曲求全 / 不能自己→不能自已 / 一如继往→一如既往 / 仗义直言→仗义执言 / 走头无路→走投无路 / 饮鸠止渴→饮鸩止渴 / 顶力相助→鼎力相助 / 不加思索→不假思索 / 按步就班→按部就班。
