# 第 25 章　全栈段位：从码农到智能体编排者

> **边界声明**：本篇为"全栈章"第四篇，也是本章收尾篇。承接前三维——第 22 篇讲现状（Agentic 拐点）、第 23 篇讲学习（技能优先级重塑）、第 24 篇讲进阶（超级个体四大角色）——本篇给出从 L1 到 L5 的完整段位跃迁地图，每一段位配"判定标志 + 跃迁动作"，让前面的理论落到可操作的自评与进阶路径上。本专栏与《AI 大模型学习》中"程序员转型"系列有明确分工：后者讲具体工具操作，本专栏讲段位体系与能力跃迁。

## 一、为什么需要一套段位体系

前面三篇讲了现状、学习、进阶，但有一个问题始终没回答：**我现在在哪？我下一步该去哪？**

这个问题不解决，所有理论都是悬空的。一个人可能读完前三篇，觉得"Agentic Engineering 很好、超级个体很酷"，但回过头不知道自己处于什么位置、该做什么动作。这就是段位体系的价值——它给你一张地图，让你能定位自己、看清下一步。

本篇的段位体系，是作者整理的合成框架，依据两个来源：

- CSDN hiwangwenbing《AI 编程等级》2026-06-13（blog.csdn.net/hiwangwenbing/article/details/161960582），给出了 L0-L5 的完整梳理
- 51CTO 三层分化分析 2026（blog.51cto.com/u_15239532/14533050），给出了从执行到编排的能力分层

**（作者整理，依据 CSDN hiwangwenbing《AI 编程等级》+ 51CTO 三层分化分析）**——这是本篇段位体系的标注方式。它不是某个权威机构的官方标准，而是基于公开资料的整理与综合，目的是给在岗进阶者一个可操作的自评框架。

## 二、hiwangwenbing 的 L0-L5 完整梳理

在展开本专栏的五段位之前，先把 hiwangwenbing 的原始 L0-L5 梳理列清楚，作为参照系。来源：CSDN hiwangwenbing《AI 编程等级》2026-06-13 blog.csdn.net/hiwangwenbing/article/details/161960582。

hiwangwenbing 的梳理把 AI 编程能力分成六级：

- **L0 纯手动**：完全不用 AI，传统 IDE 全流程手写。
- **L1 代码补全**：AI 做局部补全，开发者仍是唯一决策者。
- **L2 Vibe Coding**：用自然语言描述意图，AI 生成可运行代码。
- **L3 Spec Coding**：写规格，AI 按规格生成代码，强调规格驱动。
- **L4 Agentic Coding**：Agent 自主完成 macro action，人审查决策。
- **L5 智能体编排**：写规格让 Agent 团队协作交付，人定义目标与约束。

这个六级梳理的价值，在于它把"AI 参与度"和"人的角色"两个维度同时呈现——从 L0 到 L5，AI 参与度递增，人的角色从"执行者"递变为"编排者"。

本专栏在 hiwangwenbing L0-L5 基础上做整合：L0+L1 合并为本专栏 L1，hiwangwenbing 的 L2 并入本专栏 L3，L3+L4 合并为本专栏 L4，L5 保留。因此本专栏段位为 L1/L3/L4/L5 四档（L2 并入 L3）。

本专栏在 hiwangwenbing 的基础上做了一个简化整合：把 L0 与 L1 合并为"L1 码农"（因为两者都是"AI 不参与决策"的范畴），把 L2 单列为"L3 Vibe Coder"（对应自然语言驱动），把 hiwangwenbing 的 L3 Spec Coding 和 L4 Agentic Coding 合并为"L4 Agentic Engineer"（因为 Spec Coding 是 Agentic Engineering 的前置技能），把 L5 保留为"L5 智能体编排者"。这样得到本专栏的五段位。

## 三、AI 全栈五段位（作者整理，依据 CSDN hiwangwenbing + 51CTO）

下面展开本专栏的五段位。每一段位都配"判定标志"（你怎么知道自己在这级）和"跃迁动作"（你怎么升到下一级）。

### L1 码农（纯手动）

**定义**：无 AI 参与，传统 IDE 全流程手写。

**判定标志**：从未用过 AI 工具，或者只用过一两次就放弃了。日常工作中，写代码靠手敲、查资料靠搜索、调 bug 靠打断点。AI 工具对你来说是一个"听说很火但我没试过"的东西。

**这一级的人在哪**：很可能是资历较深的传统工程师，对 AI 工具有本能的抵触——"AI 写的代码我看不上""我自己写更快更稳"。也可能是刚入行的新人，还没接触到 AI 工具。

**风险**：这是风险最高的一级。不是因为会被 AI 直接替代，而是因为同行正在用新范式把产能拉到一个你追不上的量级。Karpathy 在第 22 篇里说的"I have never felt more behind as a programmer"，对应的正是这一级的人——当他们意识到差距时，差距已经很大了。

**跃迁动作**：接入 Copilot。不需要一上来就学 Claude Code 或 Cursor，先从最轻量的 Copilot 补全开始，让 AI 进入你的工作流。这一步的目标不是"提效"，而是"破除心理抵触"——亲身体验 AI 能做什么、不能做什么。

### L2 Copilot 用户（代码补全）

**定义**：AI 做局部补全，开发者仍是唯一决策者。

**判定标志**：日常用 Copilot 或类似工具做代码补全，写一个函数时 AI 会建议下几行，你接受或拒绝。但所有决策都是你做的——写什么、怎么写、什么时候停，AI 只是"打字加速器"。

**这一级的人在哪**：这是 2026 年大多数工程师的常态。他们已经接受了 AI 工具，但工作流本质上还是传统模式——AI 是辅助，人是主体。

**局限**：这一级的天花板很低。AI 在这里只发挥了"补全"能力，远未触及 macro action。Karpathy 说的"the default workflow changed"，对这一级的人还没发生。

**跃迁动作**：尝试 Vibe Coding。挑一个周末项目，强迫自己全程用自然语言驱动 AI 生成代码，不手写。这一步的目标是体验"AI 主导、人审查"的工作流，打破"AI 只是补全"的认知。

### L3 Vibe Coder（自然语言驱动）

**定义**：用自然语言描述意图，AI 生成可运行代码，抬高下限。

**判定标志**：能用自然语言生成完整功能。给你一个需求，你能用 Prompt 驱动 AI 产出一个能跑的实现，不需要手写大量代码。你的产出速度明显高于 L2，能快速做原型、验证想法。

**这一级的人在哪**：这是 2025 年大部分时间里的"前沿"。很多人在 2025 年尝到了 Vibe Coding 的甜头，觉得"AI 编程就是这样了"。

**局限**：正如第 24 篇讲的，Vibe Coding 有三大失败模式——安全漏洞、不可维护、上下文崩溃。一旦任务复杂度上来，需要跨模块重构、需要长程一致性、需要工程化部署，L3 就开始失速。你的下限被 Vibe Coding 抬高了，但天花板还没被撑起来。

**跃迁动作**：学 Agentic Engineering。从"一次对话生成一个功能"升级到"让 Agent 自主完成一个 macro action"。具体来说，开始用 Claude Code 或 Cursor 的 Agent 模式，让 Agent 跨文件完成任务，你做审查和决策。这一步的目标是体验 macro action，理解"AI 主导长程任务"的工作流。

### L4 Agentic Engineer（Agent 自主 macro action）

**定义**：Agent 实现功能 / 重构 / 调研 / 部署 / 测试，人审查决策，抬高上限。

**判定标志**：能让 Agent 自主完成跨文件任务。给你一个中等复杂度的任务（比如"为这个模块补齐测试套件""把这个子系统从 REST 迁移到 GraphQL"），你能让 Agent 自主完成，你做关键审查和决策。你的产出不再受限于"敲键盘的速度"，而受限于"判断力和编排能力"。

**这一级的人在哪**：这是 2026 年中比较前沿的位置。能做到这一级的人，已经摸到了 Karpathy 所说的 Agentic Engineering 门槛，对应第 22 篇里的"处境三"。

**能力要求**：这一级要求同时具备第 23 篇讲的前四项能力——系统设计（拆解任务）、AI 工具使用（编排 Agent）、代码审查（评估产出）、沟通（与 Agent 协作）。编码能力的权重降到最低，但不是零。

**局限**：这一级的人依然是"单 Agent 思维"——一次让一个 Agent 做一件事。当任务复杂到需要多个 Agent 协作时，单 Agent 模式会力不从心。

**跃迁动作**：编排多 Agent。从"指挥一个 Agent"升级到"指挥一个 Agent 团队"。具体来说，开始用 LangGraph、CrewAI 或官方 Agent SDK，设计多 Agent 协作流程——一个 Agent 调研、一个 Agent 实现、一个 Agent 审查。这一步的目标是体验"编排者"角色，对应第 24 篇讲的超级个体四大角色。

### L5 智能体编排者（Spec Coding 多 Agent 协作）

**定义**：写规格让 Agent 团队协作交付，人定义目标与约束。

**判定标志**：能编排 Agent 团队完成系统级任务。给你一个复杂任务（比如"做一个新功能从设计到上线"），你能写出规格、设计 Agent 编排、让 Agent 团队协作交付，你只在关键节点做判断。你的产出价值不再受限于"个人执行带宽"，而受限于"编排带宽"——你能编排多复杂的任务、协调多少个 Agent。

**这一级的人在哪**：这是 2026 年中最稀缺的位置。能做到这一级的人，就是第 24 篇讲的"超级个体"。他们不是更大的执行者，而是更小的指挥官。

**能力要求**：这一级要求四大角色（AI 训练师、质量守门人、架构决策者、工作流设计师）同时到位。任何一项短板，都会在多 Agent 协作中暴露——AI 训练不到位则 Agent 不稳定，质量守门不到位则产出不可信，架构决策不到位则切分错误，工作流设计不到位则协作混乱。

**这一级之上还有什么**：本专栏不再展开更高段位。L5 已经是当下可观察到的最高段位，再往上属于"定义新范式"的范畴，不在本专栏的覆盖范围内。

## 四、跃迁共性：判断力权重上升，编码权重下降

把五段位横向对比，能看到一条清晰的跃迁共性：

**从手写 → 补全 → 生成 → 自主 → 编排，每一步判断力权重上升，编码权重下降。**

| 段位 | 编码权重 | 判断力权重 | 人的核心动作 |
|---|---|---|---|
| L1 码农 | 极高 | 低 | 全流程手写 |
| L2 Copilot 用户 | 高 | 中 | 手写 + 接受补全 |
| L3 Vibe Coder | 中 | 中高 | 描述意图 + 接受生成 |
| L4 Agentic Engineer | 低 | 高 | 审查 + 决策 |
| L5 智能体编排者 | 极低 | 极高 | 定义目标 + 编排 |

这张表揭示了一个反直觉的趋势：**段位越高，编码权重越低，判断力权重越高**。L1 的核心动作是"写代码"，L5 的核心动作是"定目标"。中间的跃迁，本质上是把"执行"逐步外包给 AI，把"判断"逐步集中到人。

这也是为什么第 23 篇把"系统设计"列为 2026 年权重第一的技能——系统设计的本质就是判断力。它能让你在 L4、L5 段位上站稳。

## 五、"3% 判断力"：AI 包办 97% 编码

跃迁共性的极端表述，是来自牛客的一个判断（来源：牛客 m.nowcoder.com/discuss/885940402564370432）：

> AI 包办 97% 编码，考剩下 3% 判断力。

这个数字不需要当作精确统计，但它传递的信号是清晰的：在 L5 段位，编码工作几乎完全外包给 AI，人的核心价值集中在那 3% 的判断上——决定让 AI 做什么、怎么做、什么时候介入、什么时候接管。

这 3% 的判断力，包含几层：

**第一层：任务判断。** 这个任务该不该让 AI 做？哪些部分 AI 做、哪些人做？这是架构决策者的工作。

**第二层：规格判断。** 任务规格写得对不对？边界清不清？验收标准明不明？这是工作流设计师的工作。

**第三层：产出判断。** AI 的产出能不能合并？哪里有问题？哪里要打回重做？这是质量守门人的工作。

**第四层：调优判断。** Agent 表现不达标时，是 Prompt 问题、上下文问题、还是模型能力问题？这是 AI 训练师的工作。

这四层判断，就是第 24 篇讲的超级个体四大角色的具象化。它们加起来，就是那 3% 的判断力。

## 六、段位跃迁的常见误区

讲完段位地图，讲几个跃迁过程中常见的误区。

**误区一：跳级跃迁。**

有人想从 L2 直接跳到 L4，跳过 Vibe Coding 阶段。这通常行不通。Vibe Coding 阶段训练的是"用自然语言精确描述意图"的能力，这是 Agentic Engineering 的前置技能。跳过这一级，到了 L4 你会发现自己写不出有效的 Prompt，Agent 产出质量上不去。段位跃迁是阶梯式的，每一级都有不可替代的能力训练。

**误区二：段位焦虑。**

有人读完段位体系，发现自己还在 L2，立刻焦虑起来，觉得自己"落后了"。这种焦虑没必要。段位是工具，不是评判。L2 的人在 2026 年依然是大多数，重要的不是你现在在哪级，而是你是否有清晰的跃迁路径。本篇给出的"判定标志 + 跃迁动作"，就是为每个人提供可操作的下一步。

**误区三：段位优越感。**

反过来，有人到了 L4 或 L5，就开始看不上 L2、L3 的人。这也没必要。不同段位适合不同任务——L3 的 Vibe Coding 在原型阶段依然是最快的选择，L4 的单 Agent 模式在中等复杂度任务上依然高效。段位高不等于"什么任务都要用高段位方式做"，匹配任务复杂度才是关键。

**误区四：把段位当线性进度条。**

段位不是线性进度条——不是"今天 L2，努力三个月到 L3，再努力半年到 L4"。段位跃迁有阶跃性，需要特定时机的触发。比如从 L3 跃迁到 L4，往往需要遇到一个"Vibe Coding 解决不了"的真实任务，才会被迫学 Agentic Engineering。强行跃迁不如等机会触发。

## 七、自评与下一步

读完本篇，建议做一次自评：

1. 对照五段位的"判定标志"，确定自己当前在哪一级。如果跨在两级之间，取较低的那级——保守自评比乐观自评更有利于进步。
2. 找到当前段位的"跃迁动作"，把它作为下一步的具体目标。
3. 不要同时追多个跃迁——一次只追一个，跑通再追下一个。

段位体系的价值，不在于"分出三六九等"，而在于"让每个人看清自己的位置和下一步"。本专栏"全栈章"四篇到此收尾——现状给了判断，学习给了路径，进阶给了角色，段位给了地图。接下来第 26 篇结语，会把整个专栏"知彼解己"的闭环收口。

## 八、本篇小结

把段位体系一句话压缩：**AI 全栈五段位——L1 码农、L2 Copilot 用户、L3 Vibe Coder、L4 Agentic Engineer、L5 智能体编排者——每一段位配判定标志与跃迁动作；跃迁共性是判断力权重上升、编码权重下降；L5 段位的核心是那 3% 的判断力，对应超级个体四大角色。**
