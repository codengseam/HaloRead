# RAG、Agent、微调：AI 开发岗的必问三件套

AI 应用开发岗——也叫 Agent 工程师、RAG 工程师、LLM 应用工程师——是 2026 年招得最猛、淘汰率也最高的一类。岗位新、要求杂、面试官自己也半路出家，于是题型高度集中在"必问三件套 + 项目深挖 + 系统设计"三块。本文按这三块拆解社区面经里的真实题型，并给出能落地的应对策略。

## 一、必问三件套：RAG / Agent / 微调

以下题目整理自社区面经（来源：InfoQ Top20 xie.infoq.cn/article/8960744994b927abcaff938fa；51CTO 30 题 blog.51cto.com/u_16213657/14637080）。

### 1.1 RAG：不是讲流程，是讲优化

**完整流程题**：索引 → 检索 → 生成。这是入门题，几乎所有人都会背。面试官真正想听的是"你在哪一步做了什么优化"。

**RAG vs 微调**：知识不足用 RAG（外部知识注入），能力不足用微调（模型本身行为改变）。这是分水岭题。陷阱是"什么时候两者都做"——答案：当模型既缺知识又缺能力时，先微调补能力、再 RAG 补知识，顺序不能反。

**长文档切片策略**：错误答法"按固定长度切"。正确答法"语义感知切片"——按段落/标题/语义边界切，重叠窗口防信息断裂，长文档可分层（先摘要检索、再细节检索）。

**混合检索 + Rerank**：向量检索召回语义相近、BM25 召回关键词命中，二者融合后用 Cross-Encoder 做 Rerank，把真正相关的排到前面。能讲清"为什么不能只用向量"（向量对专有名词、数字、代码不敏感）就过关。

### 1.2 Agent：ReAct vs Plan-Execute 是核心判断题

**ReAct vs Plan-Execute**：这是 2026 年 Agent 面试的招牌题。标准答法：5 步以内的简单任务用 ReAct（思考—行动—观察循环），复杂多步任务用 Plan-Execute（先规划全流程、再分步执行）。能讲清"为什么 ReAct 在长任务上会崩"（中间步骤错误累积、上下文爆炸）才算真懂。

**Function Calling 原理**：LLM 只输出 JSON（函数名 + 参数），应用层执行实际函数调用，再把结果回填给 LLM。安全设计要点：参数校验、权限隔离、危险操作二次确认。误区："LLM 自己调用函数"——错，LLM 永远不直接执行，只产出意图。

**Agent vs Workflow**：控制权在 LLM 就是 Agent（动态决策），控制权在代码就是 Workflow（固定流程）。社区面经里的共识是：**别什么都上 Agent**。强一致性场景（支付、合规）用 Workflow，灵活对话场景用 Agent，混合场景用 Workflow 兜底 + Agent 局部决策。

**Multi-Agent 四种架构**：主从（一个 master 调度多个 worker）、流水线（A→B→C 串行）、对等（agent 间平等协商）、混合。能讲清每种适合什么场景、各自的失败模式，比背架构图值钱。

**Agent 四要素**：感知（输入理解）+ 规划（任务拆解）+ 行动（工具调用）+ 记忆（上下文管理）。

**工具调用失败三层处理**：重试（网络抖动）→ 参数修正（让 LLM 修正参数）→ 降级（换工具或转人工）。能讲清"什么时候不重试直接降级"（参数语义错误、权限不足）是加分项。

**Memory 三层**：Working Memory（当前对话上下文）、Short-term（本次会话历史）、Long-term（跨会话持久化，常向量库实现）。

### 1.3 前沿概念：MCP / A2A / SDD / Harness

- **MCP 协议**：类比"AI 工具调用的 USB-C"，标准化 LLM 与外部工具的连接。能讲清"为什么需要 MCP"（每个工具单独写 adapter 太碎、MCP 统一接口）就够。
- **A2A / AG-UI 协议**：Agent 间通信 / Agent 与 UI 交互的协议层。
- **SDD / OpenSpec**：Spec-driven development，先写规格、再让 Agent 实现。
- **Harness Engineering 五模块**：上下文管理、工具管理、记忆管理、规划管理、安全管理的工程化框架。

前沿概念不必全会，但要知道"它解决什么问题"，否则面试官追问就露馅。

## 二、项目深挖：六道连珠追问

项目深挖题整理自社区面经（来源：阿里 Agent 一面 InfoQ xie.infoq.cn/article/731413cf711c678136541e260），典型六连问：

1. **异构数据（图片）怎么处理？** OCR + 图文对齐 + 多模态 embedding，要讲清"图片里的表格怎么进检索"。
2. **人工打标可行吗？** 成本、规模、一致性三个维度评估，能讲"打标采样而非全量"是工程意识。
3. **回答正确性怎么保证？** RAG 引用 + 置信度过滤 + 人工抽检，三件套。
4. **评测模型怎么构造问题？** 别只说"找一批问题"，要讲"从真实日志采样 + 边界 case 人工补充 + 难例集"。
5. **幻觉怎么识别？** 自一致性（多次采样一致性）、引用回溯（答案能否在原文找到）、外部事实库校验。
6. **用户问题不在文档里怎么处理？** 拒答 + 引导，而不是硬编。能讲"怎么定义'不在文档里'"（检索分数低于阈值）是加分项。

还有几道经典追问：

- **LoRA 除了减参数还有什么优点？** 训练快、可插拔（一个 base 多个 LoRA adapter）、显存友好、易于版本管理。
- **检索召回率怎么定义？** 召回的相关文档数 ÷ 总相关文档数。难点在"相关"怎么定义——人工标注 gold set 是标准做法。
- **忠实度（faithfulness）怎么测？** 答案中每个 claim 是否能在检索到的原文中找到支撑，常用 LLM-as-judge 自动评估。

## 三、高分话术：把流程说成优化

社区面经里流传一段高分 RAG 话术，值得参考：

> "我做了一个高召回率的商品问答 RAG。分块用 bge-large-zh 做向量化，检索用混合检索（向量 + BM25），向量库用 Milvus 的 IVF_FLAT，单次检索 200ms 内返回；初版准确率 70%，加 Cohere rerank 后提升到 90%。后续优化方向是 query 改写和长尾实体消歧。"

这段话好在哪？每一步都有具体技术选型 + 量化效果 + 优化方向。反例是"我用 LangChain 搭了个 RAG，效果不错"——这种回答等于没说。

## 四、系统设计：两道经典题

### 4.1 千万级文档、TTFT < 500ms 的 RAG 系统

这是 2026 年高频系统设计题。答题框架：

- **索引层**：分层索引（语义聚类 + 簇内向量），减小检索空间；热数据常驻内存、冷数据落盘。
- **检索层**：两阶段检索——粗排（ANN，IVF/HNSW）召回 Top-100，精排（Cross-Encoder rerank）取 Top-5。
- **生成层**：流式输出降首 Token 时延；prompt 压缩（只塞相关性最高的 chunk）；KV Cache 复用。
- **工程层**：异步索引更新、读写分离、多级缓存（query cache + embedding cache）。

能讲清"500ms 怎么分配"（检索 150ms、rerank 100ms、生成首 Token 200ms、网络 50ms）就有架构感。

### 4.2 智能客服 Agent

标准架构：意图识别 → 状态机管理对话 → 安全确认（高风险操作二次确认）→ 转人工兜底 → 四层评估（响应率、解决率、满意度、转人工率）。

加分项：讲清"什么时候不进 Agent 直接走规则"（订单查询、物流查询这种结构化场景，规则比 Agent 快且准），"什么时候必须转人工"（情绪检测触发、连续两轮未解决）。

## 五、应对策略：判断力 + 量化 + 工程取舍

### 5.1 讲优化策略，不背流程

RAG 五步、Agent 四要素背得再熟也没用。面试官想听的是"你在哪一步做了什么优化、为什么这么选、效果多少"。准备项目时，每个项目至少讲三个优化点和它们的量化效果。

### 5.2 "何时用 Agent 何时用 Workflow"的工程判断力

这是 2026 年 AI 开发岗的灵魂题。错误答法："都用 Agent，更智能。"正确答法：按场景分——强一致性、强可控、强合规用 Workflow；需要灵活决策、多工具编排用 Agent；混合场景用 Workflow 做骨架、Agent 做局部决策。能举一个"我之前用过 Agent 但后来改成 Workflow"的案例，判断力立现。

### 5.3 量化效果："检索率优化 20%"怎么算

别只说"检索率优化 20%"，要讲清：

- 优化前的检索率是多少、怎么测的（gold set 召回率）
- 优化做了什么（加 BM25、调 chunk size、换 embedding 模型）
- 优化后的检索率是多少
- 用了多长时间、踩了什么坑

四个数字 + 一个坑，比"提升了 20%"有说服力一百倍。

### 5.4 工程取舍：别假装全知

AI 开发岗最容易翻车的瞬间是"什么都答会"。遇到没做过的，老实说"我没做过 X，但我做过类似的 Y，如果让我做 X 我会先 Z"。面试官更欣赏"有边界感 + 有迁移能力"的候选人，而不是"什么都懂一点什么都不深"的万金油。
