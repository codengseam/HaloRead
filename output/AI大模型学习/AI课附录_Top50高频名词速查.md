# 附录：Top50 高频名词速查

这份速查表收录 AI 大模型领域最常出现的 50 个名词，按主题分成八组，每个词条给一句话解释，方便随时翻查；所有定义基于公认的技术理解，不涉及编造。

## 一、基础概念

**人工智能（Artificial Intelligence, AI）**：让机器具备类似人类感知、推理、学习和决策能力的技术总称，是机器学习、深度学习、大模型等所有子领域的上位概念。

**机器学习（Machine Learning, ML）**：让机器从数据中自动学习规律、而非靠人工编写规则的方法，分为监督学习、无监督学习和强化学习三大类。

**深度学习（Deep Learning, DL）**：基于多层神经网络的机器学习方法，通过层层抽象自动提取特征，是图像识别、语音识别和大语言模型背后的核心技术。

**大语言模型（Large Language Model, LLM）**：用海量文本训练、参数规模达数十亿到数千亿的神经网络模型，能理解和生成自然语言，如 GPT、Claude、GLM 系列。

**生成式 AI（Generative AI）**：能生成新内容（文本、图像、音频、代码）而非仅做分类预测的 AI 系统，大语言模型和扩散模型都属于这一类。

**词元（Token）**：模型处理文本的最小单位，可以是一个字、一个词或一个词的一部分，Token 数量直接影响费用、速度和上下文容量。

**上下文窗口（Context Window）**：模型一次推理能读取和记住的最大 Token 数量，决定了单次对话能装入多少信息，常见从 8K 到 200K 不等。

**幻觉（Hallucination）**：模型生成看似合理但实际错误或编造内容的现象，是大模型落地最核心的风险来源，需通过 RAG、约束、护栏等手段缓解。

## 二、模型架构

**Transformer**：2017 年提出的神经网络架构，靠注意力机制并行处理序列，是几乎所有现代大语言模型（GPT、Claude、GLM）的共同底座。

**编码器（Encoder）**：Transformer 的一部分，负责把输入文本压缩成富含语义的向量表示，BERT 是典型编码器模型，擅长理解类任务。

**解码器（Decoder）**：Transformer 的一部分，负责根据已有表示逐个生成下一个 Token，GPT 系列是典型解码器模型，擅长生成类任务。

**注意力机制（Attention）**：让模型在处理当前词时能"关注"到序列中其他相关词并加权聚合信息的机制，是 Transformer 区别于 RNN 的关键。

**自注意力（Self-Attention）**：注意力机制的一种，让序列中每个词都和序列内所有词（包括自己）计算相关性，从而捕捉长距离依赖关系。

**多头注意力（Multi-Head Attention）**：并行运行多组自注意力，每组关注不同维度的关系（如语法、语义、指代），再拼接结果，让模型同时捕捉多种模式。

**混合专家模型（Mixture of Experts, MoE）**：把模型拆成多个"专家"子网络，每次推理只激活其中一部分，用更少算力实现更大参数规模，如 GLM-4.5、DeepSeek 系列。

## 三、训练与对齐

**预训练（Pre-training）**：用海量无标注文本让模型学习语言规律的第一个训练阶段，目标是掌握通用知识，产出"基座模型"。

**监督微调（Supervised Fine-Tuning, SFT）**：用人工编写的高质量"指令-回答"对训练模型，让它学会按指令格式回答，从"会说话"变成"会听话"。

**基于人类反馈的强化学习（Reinforcement Learning from Human Feedback, RLHF）**：用人类对模型回答的偏好打分训练奖励模型，再用强化学习优化主模型，让输出更符合人类期望。

**对齐（Alignment）**：让模型行为符合人类价值观、安全要求和用户意图的过程，RLHF、DPO 都是对齐的常见手段，核心是"听话且不闯祸"。

**微调（Fine-tuning）**：在基座模型上用特定领域数据继续训练，让模型适配某类任务或行业，比从头训练便宜得多，但不是万能药。

**直接偏好优化（Direct Preference Optimization, DPO）**：一种比 RLHF 更简单的对齐方法，跳过奖励模型直接用偏好数据优化主模型，训练更稳定。

## 四、提示词与上下文

**提示词（Prompt）**：用户给模型的任务说明，写得越清楚模型越容易稳定输出，是使用大模型最核心的技能。

**系统提示词（System Prompt）**：定义模型角色、边界、输出格式和禁止事项的高优先级指令，优先级高于普通用户输入，决定模型的整体行为基调。

**少样本（Few-shot）**：在提示词里给模型几个示例，让它照着示例的模式回答，比纯指令更直观，适合格式固定或任务复杂的场景。

**思维链（Chain-of-Thought, CoT）**：让模型把推理过程一步步写出来再给答案的技巧，能显著提升数学、逻辑等复杂任务的准确率。

**零样本（Zero-shot）**：不给任何示例，只靠指令让模型直接回答，依赖模型自身的通用能力，简单任务可用，复杂任务效果不如 Few-shot。

**上下文工程（Context Engineering）**：在有限窗口里组织和注入最有价值信息的系统工程，涵盖系统提示词、记忆、检索结果、工具结果等组成。

**温度（Temperature）**：控制模型输出随机性的参数，值越低越确定保守、值越高越发散有创意，事实问答用低温度、创意写作用高温度。

## 五、RAG 与检索

**检索增强生成（Retrieval-Augmented Generation, RAG）**：先从知识库检索相关资料，再把资料塞进上下文让模型基于资料回答，是缓解幻觉、接入私有知识的主流方案。

**嵌入（Embedding）**：把文本转成一组数字向量，让语义相近的文本向量距离也近，是向量检索和相似度计算的基础。

**向量数据库（Vector Database）**：专门存储和查询向量的数据库，能快速找到与查询向量最相似的若干条记录，如 Milvus、Pinecone、Chroma。

**分块（Chunking）**：把长文档切成 200-500 字的小片段再向量化入库，切分策略直接影响检索质量，常见按段落、标题或语义切分。

**重排序（Reranking）**：用更精细的交叉编码器对检索召回的候选片段重新打分排序，把最相关的排到前面，是 RAG 性价比最高的优化点。

## 六、Agent 与工具

**智能体（Agent）**：以大模型为大脑、能自主规划并调用工具完成多步任务的系统，可记为 Agent = LLM + 上下文 + 工具 + 记忆 + 规划 + 护栏。

**ReAct**：一种 Agent 范式，让模型交替进行"思考（Reasoning）"和"行动（Acting）"，边推理边调用工具，是大多数 Agent 框架的基础循环。

**工具调用（Tool Calling）**：模型根据需要选择并调用外部函数、API 或数据库的能力，让模型从"只会说"变成"能做事"，是 Agent 的手脚。

**模型上下文协议（Model Context Protocol, MCP）**：Anthropic 提出的连接 AI 应用与工具、数据、工作流的开放标准接口，让工具即插即用，类似"AI 的 USB-C"。

**子智能体（Sub-agent）**：主 Agent 把复杂任务拆解后委派给的专业化 Agent，各司其职、主 Agent 负责编排，提升复杂任务的可靠性和并行度。

**多智能体（Multi-Agent）**：多个 Agent 协作完成同一目标的架构，可分工、可讨论、可相互审查，适合需要多视角或长流程的复杂任务。

## 七、工程化

**测试框架（Harness）**：为 Agent 搭建的标准化运行与评测框架，负责调度模型、工具、护栏和评估，把"能跑的 demo"变成"可上线的产品"。

**循环（Loop）**：Agent 反复执行"感知-规划-行动-观察"的迭代过程，通过多轮循环逼近目标，是 Agent 区别于一次性问答的核心机制。

**评估（Eval）**：用测试集自动衡量模型或 Agent 输出质量的方法，覆盖单步、轨迹和最终结果三层，上线前和变更后都要跑，防止悄悄退化。

**护栏（Guardrails）**：在输入、输出和工具行为上自动执行业务规则的安全机制，包括输入校验、权限控制、输出过滤、敏感信息保护，让 Agent 不闯祸。

**RCR（检索-压缩-重排，Retrieval-Compression-Reranking）**：上下文工程的标准漏斗框架——先检索召回候选、再压缩瘦身、最后重排精选，把海量信息筛成"刚好够用"的精华注入上下文。

**追踪（Trace）**：记录 Agent 一次执行中所有模型调用、工具调用、路由和检索的完整链路，没有 Trace 就无法回放、调试和审计 Agent 的决策过程。

## 八、应用与产品

**最小可行产品（Minimum Viable Product, MVP）**：用最少功能验证核心假设的产品形态，AI 项目尤其要先做 MVP 跑通闭环、再迭代增强，避免一上来就堆功能。

**Copilot**：嵌入在工作流中辅助人类完成任务的 AI 助手模式，人主导、AI 辅助，与 Agent 模式（AI 主导、人监督）形成对照，代表如 GitHub Copilot。

**智能体框架（Agent Framework）**：用于快速搭建 Agent 的开发框架，封装了模型调用、工具管理、记忆、规划等通用能力，如 LangChain、LlamaIndex、Coze、AutoGen。

**中间丢失（Lost in the Middle）**：模型对放在上下文中间位置的信息注意力弱、容易忽略的现象，提示长上下文里关键信息要放在开头或结尾。

**事实锚定（Grounding）**：让模型回答时依赖可验证的外部资料而非凭参数记忆生成，RAG 是最常见的落地方式，是提升可信度、降低幻觉的关键。
