# 第 05 章：Transformer 三大件：编码器、解码器、注意力

> 前置知识：学完第 04 章（Token）即可
> 学完你能：①说清 Transformer 的三大组成部分 ②理解为什么是这三大件改变了 AI ③不再对"大模型"感到神秘

## 一、讲清楚：这一章要拆什么

如果你只能记住一个关于大模型的事实，请记住这一条：

GPT、Claude、Gemini、通义千问、豆包、文心一言、Kimi、DeepSeek……这些你叫得出名字的现代大模型，骨架都是同一个——Transformer。

Transformer（一种基于注意力机制的神经网络架构）不是某个公司的专利，它是 2017 年 Google 团队发表的一篇论文里提出的架构。这篇论文叫《Attention Is All You Need》，发表在当年的 NeurIPS 会议上，作者署名是 Vaswani 等人：「Vaswani et al., Attention Is All You Need, NeurIPS, 2017」。

理解 Transformer，你就理解了大模型的"骨头"。剩下的参数规模、训练数据、对齐方法，都是往这副骨头上挂肉。

这一章要拆三件事：

1. **编码器（Encoder）**：把输入读进去，变成一组"含义向量"。
2. **解码器（Decoder）**：根据这组向量，一个字一个字往外蹦。
3. **注意力（Attention）**：决定哪些词更值得关注，是 Transformer 的心脏。

注意力这一节只点到为止，第 06 章会专门展开。

## 二、看案例：2017 年那场机器翻译的"质量飞跃"

要理解 Transformer 为什么重要，得先看它替代了什么。

2017 年之前，做机器翻译的主流技术是 RNN（循环神经网络，Recurrent Neural Network）和它的进阶版 LSTM。RNN 的工作方式像人逐字读书：读一个词，更新一下记忆，再读下一个词。

这种方式有个先天缺陷：**它只能逐字读，不能并行**。

一句话 30 个词，RNN 必须按顺序处理 30 次。一篇 500 词的文章就要 500 次。训练时 GPU 再强，也只能干等着前一步算完。

更致命的是"遗忘"。读到句尾时，句首的信息已经被覆盖得差不多了。所以早期的机器翻译，长句子一塌糊涂——前半句翻得还行，后半句就开始胡说。

2017 年，Vaswani 团队提出了 Transformer。它在 WMT 2014 英德翻译任务上，把 BLEU 分数（机器翻译质量的标准指标）从当时最好的 25.2 一下子拉到 28.4，同时训练时间大幅缩短。

更关键的是，Transformer 把"逐字读"换成了"同时看"。一篇 500 词的文章，它一次性就能全部吃进去，每个词都能直接看到其他所有词。

这个改动听起来不起眼，但它打开了一扇门：模型可以无限变大，训练可以无限堆 GPU，而不会因为"顺序依赖"卡死。

这扇门的另一头，就是今天的大模型。

## 三、拆原理：三大件各干什么

### 1. 编码器：像"读完一本书写出读书笔记"

编码器干的活，可以用一个动作概括：**把一段输入变成一组含义向量**。

什么叫"含义向量"？回忆一下第 04 章讲的 Token。Token 只是把文字切成了小块，每个小块本身没有"意思"。编码器要做的事，就是给每个 Token 算出一个高维向量（通常几千维），让这个向量能表达"这个词在这个句子里到底是什么意思"。

类比一下：你读完一本《三国演义》，合上书，脑子里留下的不是逐字逐句的原文，而是一组"印象"——曹操是奸雄、刘备是仁君、诸葛亮是智者、关羽是义士。这组印象就是"含义向量"。

编码器就是干这个的。它读完整段输入，吐出一组向量，每个向量对应一个 Token，但每个向量都已经"吸收"了上下文。

注意这句话里的"吸收了上下文"——这是 Transformer 比前一代技术强的地方。在 RNN 里，一个词的向量只受它前面词的影响；在 Transformer 里，一个词的向量受整段输入所有词的影响。

### 2. 解码器：像"看着提纲写作文"

解码器干的活，可以用一个动作概括：**根据编码器给的那组向量，一个字一个字往外蹦**。

继续用读书的类比：编码器写出了"读书笔记"，解码器就拿着这份笔记，开始"续写"。

它不是一次性把整段答案吐出来，而是真的一个 Token 一个 Token 地蹦：

- 第一步：看一眼编码器给的笔记，蹦出第一个 Token，比如"今"。
- 第二步：看一眼笔记，再看一眼自己刚蹦出来的"今"，蹦出第二个 Token，比如"天"。
- 第三步：看一眼笔记，再看一眼"今天"，蹦出第三个 Token，比如"天"……
- 如此循环，直到蹦出一个特殊的"结束符"。

这就是为什么你在用 ChatGPT 时，看到的是答案一个字一个字往外冒——这不是前端在做动画，是模型真的在这么工作。

每蹦一个字，模型都要重新算一遍"接下来最可能是什么"。这种"看着自己已经写出来的字，决定下一个字"的机制，专业上叫"自回归"（Autoregressive，模型用自己的历史输出作为下一步输入的生成方式）。

### 3. 注意力：像"划重点"

注意力机制是 Transformer 的心脏，也是它名字里"Attention"的来源。

它解决的问题是：当模型在处理某个词时，应该"关注"哪些其他词？

举个例子，句子"小明去银行取钱，他很高兴"。"他"这个字指代谁？是人都能看出来是"小明"。但模型怎么知道？靠的就是注意力——"他"这个 Token 会去和句子里的每个 Token 算一个"相关度分数"，分数最高的那个就是"小明"。

注意力机制让模型不再机械地逐字处理，而是能"划重点"——哪些词和当前这个词关系大，就多看几眼。

这一节先点到此。Q、K、V 三个矩阵、多头注意力这些细节，第 06 章会展开。

### 4. 三大件怎么拼起来

把三大件拼起来，Transformer 的完整流程是这样的：

1. 输入文本被切成 Token（第 04 章讲过）。
2. Token 被转成初始向量。
3. **编码器**把初始向量加工成"含义向量"。
4. **解码器**根据含义向量，结合自己已经蹦出来的内容，预测下一个 Token。
5. 在编码器和解码器内部，**注意力机制**负责"划重点"。

这就是 Transformer 的全貌。GPT、Claude、Gemini 这些模型，无论参数从几十亿涨到几千亿，骨架都是这一套。

### 5. 一个关键区别：GPT 是"只用解码器"

Transformer 原始论文里，编码器和解码器是成对出现的——编码器读源语言，解码器生成目标语言，这是为机器翻译设计的。

但后来人们发现，这个架构可以拆开用：

- **只用编码器**（Encoder-only）：代表是 BERT。擅长"理解"任务，比如分类、问答、情感分析。
- **只用解码器**（Decoder-only）：代表是 GPT 系列、Claude、Gemini。擅长"生成"任务，比如续写、对话、写代码。
- **编码器+解码器**（Encoder-Decoder）：代表是 T5、原始 Transformer。适合翻译、摘要这类"输入一段、输出一段"的任务。

GPT 的全称是 Generative Pre-trained Transformer，"Generative"（生成式）这个词，就是因为它只用解码器、靠"自回归"一个字一个字蹦出来的。

今天主流的对话大模型（GPT、Claude、Gemini、通义、豆包、Kimi、DeepSeek）几乎都是 Decoder-only 架构。原因之一是：解码器天生适合"接着写"，而接着写这件事，恰好能统一几乎所有语言任务——问答就是"接着问题写答案"，翻译就是"接着原文写译文"，摘要就是"接着长文写短文"。

所以你常听人说"GPT 是生成式模型"，本质就是它用了 Transformer 的解码器这一半。

## 四、避坑指南：三个常见误区

### 误区一：纠结公式

网上讲 Transformer 的文章，十篇有八篇一上来就贴公式：Softmax、点积、归一化、LayerNorm……普通人看两眼就劝退了。

公式是给实现者看的。你要建立的是直觉，不是会推导。记住三个类比就够了：

- 编码器 = 写读书笔记
- 解码器 = 看着提纲写作文
- 注意力 = 划重点

等真要自己训练模型或者读源码时，再回头补公式不迟。

### 误区二：把 Transformer 等同于 GPT

经常有人混用这两个词，说"GPT 就是 Transformer"。

更准确的说法是：**GPT 是基于 Transformer 解码器架构的一类模型**。Transformer 是骨架，GPT 是这副骨架上长出来的一种具体模型。

类比一下：Transformer 是"内燃机原理"，GPT 是"装了内燃机的某款车"。Claude、Gemini、豆包是另外几款车，原理都是内燃机，但车型、调校、油耗各不相同。

### 误区三：以为模型"理解"了你的话

这是最深的误区。

Transformer 做的事情，本质是**根据上下文预测下一个 Token**。它不会"理解"你的意图，它只是在算概率——"在这个上下文里，下一个最可能出现的字是什么"。

之所以看起来像理解了，是因为它见过海量文本，把"问题→答案"的模式学得足够好。但这和人类意义上的"理解"是两回事。

打个比方：一个外语很好但不懂中文的人，拿着一本厚厚的"中英对照问答手册"，你问什么他都能翻到对应页抄答案。看起来他"懂中文"，其实他只是在查表。

这个区别很重要，因为它直接决定你怎么用大模型：

- 不要指望它"真正懂你"，要把背景、约束、目标说清楚。
- 不要被它流畅的表达迷惑，关键事实要自己核对。
- 不要把它当人，要把它当一个"超级会接话的统计机器"。

## 五、悟本质：并行处理 + 全局视野

如果非要一句话说清 Transformer 为什么改变了 AI，那就是：

**它让 AI 能同时看全文，而不是逐字读。**

这句话有两层意思。

**第一层：并行处理。**

RNN 必须按顺序处理，第 N 个词必须等前 N-1 个词算完。Transformer 没有这个限制，所有词可以同时算。这意味着训练时间可以从"年"压缩到"周"，模型规模才能从"百万参数"涨到"千亿参数"。

没有并行处理，就没有今天的大模型。算力再强，顺序依赖卡死你。

**第二层：全局视野。**

RNN 里，一个词只能"看到"它前面的词（双向 RNN 能看到后面，但仍然受距离限制）。Transformer 里，每个词都能直接看到所有其他词，距离不再是问题。

这意味着模型处理长文本时，不会"读到尾忘了头"。一句话的第 1 个词和第 100 个词，在 Transformer 眼里是"邻居"。

这两层合起来，就是 Transformer 的本质：**并行处理 + 全局视野**。

理解了这一点，你就能解释很多现象：

- 为什么大模型能写长文章不跑题？因为全局视野。
- 为什么大模型训练要烧那么多 GPU？因为并行处理需要堆算力。
- 为什么 2017 年之后 AI 突然加速？因为 Transformer 把"顺序瓶颈"打破了。

再往深想一层：Transformer 的成功，不在于它"更聪明"，而在于它"更适合现代硬件"。GPU 天生擅长并行计算，RNN 的顺序依赖让 GPU 大量核心闲置。Transformer 把顺序依赖去掉，让 GPU 能满负荷运转——技术路线和硬件趋势对上了，爆发就是迟早的事。

这也是为什么后来 GPU 厂商（英伟达）成了大模型时代最大的赢家之一：Transformer 这种架构，把对并行算力的需求推到了极致。

## 六、结语

Transformer 不是魔法，它是一副骨架——编码器读、解码器写、注意力划重点。所有现代大模型，都是在这副骨架上做文章。

## 七、自测三问

**问题一：Transformer 的三大件分别对应什么动作？**

参考答案：编码器对应"读"（把输入变成含义向量，像写读书笔记）；解码器对应"写"（根据向量一个字一个字蹦，像看着提纲写作文）；注意力对应"划重点"（决定哪些词更值得关注）。

**问题二：GPT 为什么叫"生成式"模型？**

参考答案：因为 GPT 只用了 Transformer 的解码器部分，靠"自回归"机制一个 Token 一个 Token 地往外蹦。这种"接着写"的方式天然适合生成任务，所以叫生成式。

**问题三：Transformer 相比 RNN 的两个本质优势是什么？**

参考答案：一是并行处理——所有词可以同时计算，不再受顺序依赖限制，训练时间大幅缩短，模型才能越做越大；二是全局视野——每个词都能直接看到所有其他词，不受距离限制，长文本不会"读到尾忘了头"。
