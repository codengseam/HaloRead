# 第 07 章 训练三阶段：预训练、SFT、RLHF

> 前置知识：学完第 05、06 章（Transformer、注意力）即可
> 学完你能：①说清大模型训练的三个阶段各干什么 ②理解为什么模型需要"对齐" ③看懂技术新闻里的训练术语

## 讲清楚

本章解决一个问题：一个会聊天的 AI，到底是怎么"练"出来的？

很多人第一次用 ChatGPT 时都会产生一种错觉——以为它天生就会对话，天生就懂礼貌，天生就知道拒绝有害问题。其实不是。一个能当助手用的大模型，背后要经历三段"教育"，每一段解决不同的问题，缺一不可。

这三段分别叫：

- **预训练**（Pre-training）：让模型"博览群书"，学会语言和世界知识
- **监督微调**（SFT, Supervised Fine-Tuning）：让模型"上学做题"，学会按问题给答案
- **人类反馈强化学习**（RLHF, Reinforcement Learning from Human Feedback）：让模型"接受纠偏"，学会什么回答更讨人喜欢、更安全

这一章写给两类人看：一是想看懂技术新闻的普通人，二是想自己微调或选模型的程序员。为什么重要？因为现在所有大模型——GPT-4、Claude、Gemini、DeepSeek、Qwen——无一例外都走这条路。看懂这三阶段，就拿到了读懂整个行业技术新闻的钥匙。

## 看案例

把这三阶段想成一个孩子的成长。

**预训练像"博览群书"**。想象一个孩子，从三岁起被关进一个超大图书馆，里面有几千亿字的网页、书籍、论文、代码。没人教他什么，只让他玩一个游戏：随便翻到一页，遮住下半句，让他猜下一句是什么。一开始他乱猜，后来越猜越准。几年下来，他没上过一天学，但已经"读"过了人类有史以来几乎所有的公开文字。他能续写任何一段话，能模仿任何人的口吻，能背出唐诗三百首，能写一段 Python——但他不会"对话"。你问他"你好"，他可能接着写"吗？我是张三"，因为他学到的是"你好吗？我是张三"这种续写模式。

**SFT 像"上学做题"**。这个孩子被送去上学。老师不再让他瞎续写，而是给他一对对的"问题-答案"：问"中国的首都是哪里"，答"北京"；问"写一首关于春天的诗"，答"春风又绿江南岸……"。做了几万道这样的题之后，孩子学会了一件事——别人问问题，我要给答案，而不是接着往下写。这就是从"会写字"到"会写作文"的跨越。

**RLHF 像"家长纠偏"**。孩子会写作文了，但写得不一定好。同样一个问题"怎么减肥"，他可能写"少吃多动"四个字，也可能写一篇三千字的科普。家长（人类标注员）出场，给他几个版本的回答打分：这个详细的有用，那个敷衍的没用；这个安全的可以发，那个教人催吐的不能发。孩子根据打分反复调整，慢慢学会"什么回答更好"。这一步教的不只是知识，是"品格"——什么该说、什么不该说、怎么说更得体。

这三步走完，才是一个能用的 ChatGPT。

## 拆原理

### 一、预训练：用"下一个词预测"学会整个世界

预训练的核心机制只有一个：**下一个词预测**（Next Token Prediction）。给模型一段文字的前 N 个词，让它预测第 N+1 个词是什么。预测对了就奖励，错了就调整参数。

听起来简单到可笑，但量变引起质变。当训练数据达到几千亿 token、模型参数达到几百亿、训练算力达到几千张 GPU 跑几个月——奇迹就出现了。模型不仅学会了语法，还学会了事实知识、推理能力、代码风格、多语言切换。

吴恩达在 *Generative AI for Everyone* 课程里讲过一个被广泛引用的比喻：大模型的预训练就像"教一个学生读遍互联网"。它不是在记忆某一段话，而是在海量阅读中"涌现"出对语言规律和世界知识的理解。「Andrew Ng, Generative AI for Everyone · DeepLearning.AI，2023」

预训练有几个关键特征：

- **数据是"生"的**：网页、书籍、代码，什么都有，没有标注，模型自己找规律
- **目标是"续写"**：本质是给前文，接后文
- **算力极贵**：一次预训练动辄几百万到上千万美元，是只有大厂才玩得起的游戏
- **产出叫"基座模型"**（Base Model）：会续写，但不会对话

基座模型直接拿来用会出问题。你问它"今天天气怎么样"，它可能不回答，而是接着写"今天天气怎么样？这是一个常见问题……"——因为它学到的就是续写，不是回答。

### 二、SFT：从"会写字"到"会写作文"

监督微调（SFT，Supervised Fine-Tuning）做的事很直接：用一批人工写好的"问题-答案"对，再训练一遍基座模型，让它学会"对话格式"。

数据长这样：

```
用户：中国的首都是哪里？
助手：中国的首都是北京。
```

```
用户：帮我写一首关于春天的诗。
助手：春风又绿江南岸，明月何时照我还……
```

几万到几十万条这样的对话喂进去，模型就学会了一个关键格式：看到"用户：xxx"就接"助手：xxx"，看到问号就给答案而不是续写问题。

SFT 的本质是**教格式和任务模式**，不是教知识。知识在预训练阶段已经学完了，SFT 只是告诉模型"你已经会的东西，要这么用"。

类比一下：一个大学生读了四年书，知识都装在脑子里，但他没参加过面试。SFT 就是给他做几套模拟题，告诉他"面试官问什么，你要这么答"。他的知识没变，但答题的"姿势"对了。

SFT 之后产出的模型叫"指令微调模型"（Instruction-Tuned Model），已经能当助手用了。早期的一些开源模型（比如 LLaMA-1 之后的 Alpaca、Vicuna）就是只做到这一步，效果已经不错。

那为什么还要 RLHF？

### 三、RLHF：用人类偏好教模型"什么回答更好"

SFT 教会了模型"怎么回答"，但没教它"什么回答更好"。同样问"怎么减肥"，SFT 之后的模型可能给出三种回答：

- A："少吃多动。"（敷衍）
- B："建议控制饮食、每周运动三次，具体可以……"（有用）
- C："你可以试试催吐，效果很快。"（有害）

SFT 阶段数据里写的是 B，模型大概率会答 B。但如果用户换个问法，或者问到训练数据没覆盖的角落，模型可能滑到 A 或 C。

RLHF（Reinforcement Learning from Human Feedback，人类反馈强化学习）就是来解决这个问题的。它的核心思路：**让人类来当裁判，给模型的回答打分，模型根据分数学习"什么样的回答更受欢迎"**。

具体分三步：

**第一步：训练一个"打分模型"**（Reward Model）。让人类标注员看同一问题的多个回答，按"哪个更好"排序。比如对上面三个回答，排序是 B > A > C。用这些排序数据训练一个小模型，让它学会"给任意一个回答打分"。

**第二步：用打分模型给基座模型的回答打分**。让 SFT 之后的模型对一堆问题各生成几个回答，用打分模型给每个回答打分。

**第三步：用强化学习算法（通常是 PPO）让模型朝高分方向调整**。模型生成回答 → 打分 → 调整参数 → 再生成 → 再打分……循环往复，模型慢慢学会"生成高分回答"。

RLHF 教的不是知识，是**偏好和价值观**：详细比敷衍好、安全比有害好、诚实比胡说好、有用比啰嗦好。

这一步最早由 OpenAI 在 2022 年的 InstructGPT 论文里系统化提出。「Ouyang et al., Training language models to follow instructions with human feedback, NeurIPS, 2022」ChatGPT 横空出世的关键，不是 GPT-3.5 的基座有多强，而是 RLHF 把它对齐成了一个能用的助手。

### 四、为什么必须 RLHF：模型需要"对齐"

技术圈有个词叫**对齐**（Alignment），意思是让模型的行为和人类的期望、价值观对上号。

为什么需要对齐？因为预训练模型本质是个"续写机器"，它的目标函数是"下一个词预测得准不准"，不是"回答得有没有用、安不安全"。这两个目标不是一回事。

举个极端例子：训练数据里有大量论坛喷子言论、阴谋论、暴力描写。模型学得很认真，因为它不知道这些"不该学"。你问它"怎么制造危险品"，它可能从化学论坛续写出来一段——续写得很准，但很危险。

对齐就是给模型装上"价值观过滤器"：知道什么该说、什么不该说、什么场景下要谨慎。没有对齐，模型再聪明也不能放出来用。

所以三阶段的本质是：

- 预训练给模型**能力**
- SFT 给模型**格式**
- RLHF 给模型**品格**

能力、格式、品格，缺一个都不能当助手。

## 避坑指南

**坑一：以为预训练模型能直接用。**

很多人下载了 LLaMA、Qwen 的基座版本（base），直接拿来对话，发现答非所问、胡言乱语，以为模型不行。其实是用错了。基座模型只会续写，要对话得用 SFT 之后的版本（通常带 -chat、-instruct 后缀）。下载模型时一定看清是 base 还是 chat。

**坑二：混淆 SFT 和 RLHF。**

技术新闻里经常混着说"模型经过微调"，但 SFT 和 RLHF 是两回事。SFT 是用标准答案教，RLHF 是用偏好排序教。前者教"怎么答"，后者教"答得好不好"。看到"基于人类反馈的强化学习"这种说法，特指 RLHF；只说"微调"，多半是 SFT。

**坑三：忽视对齐的成本。**

RLHF 不是免费的。它需要大量人工标注（标注员时薪不低）、需要训练打分模型、需要多轮强化学习迭代。开源社区里很多模型只做到 SFT，就是因为 RLHF 太贵。自己微调模型时，如果只做 SFT 不做 RLHF，模型能用但"品格"不够，容易在边界问题上翻车。

**坑四：以为 RLHF 万能。**

RLHF 能让模型更讨人喜欢，但不能让模型变聪明。模型的能力上限在预训练阶段就定了，RLHF 只是"激发"已有能力。一个 70 亿参数的模型再怎么 RLHF，也打不过 7000 亿参数的模型。RLHF 是"调教"不是"开光"。

**坑五：把"对齐税"当不存在。**

RLHF 之后模型在某些任务上会变弱——比如代码生成、数学推理——因为模型学会了"说人话"，但牺牲了一点"硬核能力"。这叫"对齐税"（Alignment Tax）。工业界现在用 RLHF 的变体（如 DPO、RLAIF）来减轻这个副作用，但完全消除做不到。

## 悟本质

把三阶段抽象一下，会发现它对应人类教育的三个层次：

- **预训练 = 知识**：博览群书，建立世界模型
- **SFT = 技能**：学会把知识用在具体任务上
- **RLHF = 品格**：学会什么该做、什么不该做、怎么做更好

这个"知识 → 技能 → 品格"的递进，不只适用于 AI，也适用于人。

一个读过一万本书的人，不一定是个好老师（缺 SFT）；一个好老师，不一定是个有师德的人（缺 RLHF）。完整的人，三样都得有。

更深的洞察是：**预训练学的是"是什么"，SFT 学的是"怎么做"，RLHF 学的是"该不该"**。

"是什么"靠数据规模，"怎么做"靠任务示范，"该不该"靠价值反馈。这三层加起来，才是一个能用的智能体。

理解了这一点，再去看任何大模型的技术新闻，都能一眼看懂：这家公司在哪一层发力。Meta 开源 LLaMA 是在预训练层卷；OpenAI 的 GPT-4 是三层都强；Anthropic 的 Claude 强在 RLHF（他们叫 Constitutional AI，是 RLHF 的变体）；DeepSeek 走的是"预训练 + SFT + RLHF"全栈自研。看懂三阶段，就看懂了行业地图。

## 结语

大模型不是天生会聊天，它是被三段教育"教"出来的——预训练给知识，SFT 给技能，RLHF 给品格。

## 自测三问

**问题一：为什么基座模型（Base Model）不能直接当助手用？**

参考答案：基座模型只学了"下一个词预测"，本质是续写机器，不会"对话"。你问它问题，它会接着写问题，而不是给答案。必须经过 SFT 学会对话格式，再经过 RLHF 学会"什么回答更好"，才能当助手。

**问题二：SFT 和 RLHF 教的东西有什么本质区别？**

参考答案：SFT 教的是"怎么答"——用标准答案教模型按问题给回答、按格式输出；RLHF 教的是"答得好不好"——用人类偏好排序教模型生成更详细、更安全、更有用的回答。前者教格式和任务模式，后者教偏好和价值观。

**问题三：如果只能选一个阶段做，哪个最关键？**

参考答案：预训练最关键。模型的能力上限在预训练阶段就定了，SFT 和 RLHF 只是"激发"和"对齐"已有能力。一个预训练没做好的模型，再怎么微调也救不回来。但如果预训练已经够强，只做 SFT 不做 RLHF，模型能用但"品格"不够，容易在安全、有用性上翻车。三层是递进关系，不是替代关系。
