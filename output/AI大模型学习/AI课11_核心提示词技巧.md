# 第 11 章：核心技巧：角色、Few-shot、思维链

> 前置知识：学完第 10 章（消息结构）即可
> 学完你能：①掌握三大核心提示词技巧 ②知道什么场景用什么技巧 ③写出比 80% 新手更好的提示词

## 一、讲清楚：提示词不是"写得越长越好"

新手写提示词（Prompt，给 AI 的输入指令），最容易犯一个错：以为写得越长越详细，效果就越好。于是把一段话写到 800 字，把所有要求、所有背景、所有注意事项全堆进去，结果 AI 反而抓不住重点，回答东拉西扯。

提示词的优劣，不取决于长度，而取决于"是否降低了 AI 理解你意图的不确定性"。降低不确定性的手段有很多，但真正高频、通用、能覆盖大部分场景的，就三个：

- **角色扮演**（Role-Playing）：给 AI 设定一个身份，让它"以某个角色的视角"回答。
- **少样本提示**（Few-shot Prompting）：给 AI 看几个输入输出的例子，让它"照着学"。
- **思维链**（Chain-of-Thought，CoT）：让 AI"一步步想"，把推理过程写出来再给结论。

这三个技巧不是互相替代，而是互相补充。一个成熟的提示词，往往同时用上两到三个。本章会用同一个需求——给一款智能水杯写产品文案——分别演示三种技巧的效果差异，再讲清楚每种技巧的适用场景和操作方法。

## 二、看案例：同一个需求，三种技巧三种效果

需求是这样的：你有一款智能水杯，能记录饮水量、提醒喝水、水温显示，售价 199 元，目标用户是都市白领。你要让 AI 写一段电商详情页的产品文案。

### 写法 A：什么都不用，直接问

```
帮我写一段智能水杯的产品文案。
```

AI 给的文案大概是这样："这款智能水杯，让您的生活更健康。它能记录饮水量，提醒您喝水，还有水温显示功能。价格实惠，仅需 199 元，快来购买吧！"

能看，但平庸。像所有电商页面里那段没人看的介绍。原因很简单：AI 不知道你是写给谁看的、要什么风格、突出什么卖点，它只能用"最安全、最通用"的方式回答。

### 写法 B：加角色扮演

```
你是一位有 10 年经验的电商文案，擅长把普通产品写出高级感。
帮我写一段智能水杯的产品文案，售价 199 元，目标用户是都市白领。
```

文案立刻变了味：会从"凌晨两点，你还在改最后一版方案"这种场景切入，把"提醒喝水"包装成"别让忙碌透支身体"，把"水温显示"包装成"每一口都是刚好的温度"。有了角色的视角，文案有了情绪和场景。

### 写法 C：加 Few-shot

```
帮我写智能水杯的产品文案。参考下面两个例子的风格：

例子1（降噪耳机）：城市很吵，但你只需要安静。戴上它，世界为你按下静音键。
例子2（护眼台灯）：熬夜加班的眼睛，值得被温柔对待。无频闪，光线如晨曦。

现在写智能水杯的文案。
```

AI 会照着例子的句式——"短场景 + 一句情绪化总结"——写出："开会开到忘记喝水？它记得你每一次缺席的饮水。199 元，替你守住每天的八杯水。"

### 写法 D：加思维链

```
你是一位资深电商文案。帮我写智能水杯的产品文案。
请先一步步分析：1)目标用户的核心痛点是什么 2)产品的哪个功能最能击中这个痛点 3)用什么场景和情绪包装 4)最后再写文案。
```

AI 会先输出一段分析："都市白领的痛点是工作忙、忘记喝水、亚健康；最能击中的功能是饮水提醒；用'忙碌中被照顾'的情绪包装……"分析完，再给出文案。这种写法的好处是：你能看到 AI 的"思路"，发现它哪里想偏了可以及时纠正；而且经过推理的文案，逻辑通常更扎实。

四个写法，同一个需求，效果从平庸到精准。差别不在字数，在于你有没有用对技巧。

## 三、上手步骤

### 1. 角色扮演（Role-Playing）：给 AI 一个身份

角色扮演是最简单、性价比最高的技巧。一句话就能用上："你是一位……"

**怎么写好一个角色？** 三个要素：

- **身份**：你是谁。"你是一位有 10 年经验的电商文案"比"你是文案"好——经验年限会让 AI 调用更专业的内容。
- **擅长什么**：你擅长什么。"擅长把普通产品写出高级感"——给 AI 一个明确的方向。
- **风格约束**：你怎么说话。"用短句，每段不超过两行，结尾留一个钩子"——风格越具体，输出越可控。

一个完整的角色设定：

```
你是一位有 10 年经验的电商文案，擅长把普通产品写出高级感。
你的风格：用短句，每段不超过两行，先讲场景再讲功能，结尾留一个让人想点进去的钩子。
```

**在工具里怎么设置：**

- **Claude 网页版**：直接在对话框第一句写"你是一位……"，Claude 会在本轮对话里采纳。要长期生效，用 API 时把这段放进 system 参数（见第 10 章）。
- **Coze**：搭 Bot 时在"人设与回复逻辑"输入框里写角色设定，这个框底层就是 system 消息，全局生效。
- **ChatGPT**：普通对话里写"你是一位……"即可；要长期生效，在"自定义 GPT"的 Instructions 里写。

**适用场景**：需要特定风格、专业视角、行业 know-how 的任务。写文案、写代码、做咨询、做翻译，都适合先设角色。

**不适用场景**：纯事实查询（"中国的首都是哪"）、简单计算。这些任务加角色反而拖慢速度、增加 Token 消耗。

### 2. 少样本提示（Few-shot Prompting）：给 AI 看例子

少样本提示的核心思想是：与其费劲描述"我要什么样的输出"，不如直接给 AI 看几个例子，让它照着学。这比用文字描述风格高效得多——因为风格本身就很难用语言精确描述。

**最简单的形式**（文本风格模仿）：

```
帮我给产品写一句话文案。参考：

产品：降噪耳机
文案：城市很吵，但你只需要安静。

产品：护眼台灯
文案：熬夜的眼睛，值得被温柔对待。

产品：智能水杯
文案：
```

AI 会自动补上"文案："后面的内容，句式和情绪都跟前两个例子对齐。

**结构化输出的 Few-shot**（用 JSON 示例，这是工程里最常用的形式）：

```
把用户的评论分类为"正面""负面""中性"。输出 JSON。

评论：这个水杯颜值很高，但提醒功能经常失灵。
输出：{"sentiment": "中性", "reason": "既有赞美也有抱怨"}

评论：太好用了，每天喝水都变多了！
输出：{"sentiment": "正面", "reason": "纯赞美"}

评论：水温显示不准。
输出：
```

AI 会输出：`{"sentiment": "负面", "reason": "纯抱怨"}`

这种写法的好处是：你不用费劲解释"JSON 字段叫什么、reason 写什么、sentiment 取哪几个值"——例子已经把格式和规则都"演示"清楚了。模型从例子里归纳规则，比从文字描述里理解规则更准。

**给几个例子合适？** 通常 2-3 个就够。少了模型抓不住规律，多了浪费 Token 还可能过拟合（只模仿例子、不会泛化）。如果任务复杂、输出格式严格，可以加到 5 个，但一般不超过 5 个。

**适用场景**：输出格式固定（JSON、表格、特定模板）、风格需要模仿、分类任务、信息抽取任务。

**不适用场景**：开放式创作（写小说、写长文）、任务本身没有"规律"可学。这种场景给例子反而限制 AI 的发挥。

### 3. 思维链（Chain-of-Thought, CoT）：让 AI 一步步想

思维链是三个技巧里"含金量"最高的一个。它的发现来自一篇重要论文：「Wei et al., Chain-of-Thought Prompting Elicits Reasoning in Large Language Models, NeurIPS, 2022」。这篇论文证明了一个反直觉的结论：对于需要多步推理的任务（数学题、逻辑题、复杂规划），让大模型"把推理过程写出来再给答案"，比直接让它给答案，正确率显著提升。

**最简单的触发方式**：在提示词末尾加一句"让我们一步一步思考"（Let's think step by step）。就这一句，往往能让模型从"直接猜答案"切换到"先推理再回答"。

**对比一下。** 简单题看不出差别，换一道复杂的：

```
一个车间有 3 条产线，A 产线每小时产 120 件，B 产线是 A 的 1.5 倍但良品率只有 80%，C 产线每小时产 150 件良品率 90%。问 8 小时后三条产线共产出多少良品？
```

不加 CoT，模型容易在中间算错；加了 CoT：

```
一个车间有 3 条产线……（同上）
请一步一步计算每条产线的良品数，最后汇总。
```

模型会输出："A 产线：120 × 8 = 960 件，良品率 100%，良品 960；B 产线：120 × 1.5 = 180 件/小时，8 小时 1440 件，良品率 80%，良品 1152；C 产线：150 × 8 = 1200 件，良品率 90%，良品 1080；合计 960 + 1152 + 1080 = 3192 件。"——每一步都摆出来，你能一眼看出哪步算错了。

**CoT 的两种用法：**

- **Zero-shot CoT**：直接加"让我们一步一步思考"。最简单，适合大多数场景。
- **Few-shot CoT**：给几个"带推理过程的例子"，让模型学着把推理写出来。适合任务特别复杂、推理路径有固定模式的场景。

**适用场景**：数学计算、逻辑推理、多步规划、复杂决策、代码调试分析。凡是"中间过程错了、结论一定错"的任务，都该用 CoT。

**不适用场景**：简单事实查询、创意写作、翻译。这些任务不需要推理，加 CoT 反而让输出变得啰嗦。

### 4. 在实际工具中操作：Claude 与 Coze

讲了三个技巧的原理，下面看在真实工具里怎么用。

**Claude（以网页版为例）：**

打开 Claude 对话框，把角色、Few-shot 例子、CoT 指令写在同一条消息里。一个组合写法：

```
你是一位资深数据分析师，擅长把杂乱的数据整理成清晰的结论。

参考这个分析风格：
问题：上个月销售额为什么下降？
分析：1)先看总销售额同比环比 2)拆分到各品类找异常 3)拆分到各渠道找异常 4)给出最可能的原因。

现在分析这个问题：本周用户活跃度下降 15%，请一步步分析可能的原因，最后给出结论。
```

Claude 会先输出分析步骤，再给结论。你可以在它的分析过程中追问"第 2 步的品类数据具体是多少"，引导它深入。

**Coze（以搭一个文案 Bot 为例）：**

在 Coze 里搭 Bot，三个技巧可以"固化"进 Bot 配置，不用每次重写：

- **角色**：填在"人设与回复逻辑"里——"你是一位有 10 年经验的电商文案……"。
- **Few-shot**：在"人设"里附上 2-3 个文案范例，或在"知识库"里上传一批历史优秀文案，Bot 会参考。
- **CoT**：在"人设"里加一条规则——"写文案前，先分析目标用户痛点、产品核心卖点、适用场景，再写文案"。

配置完，用户每次只需输入"产品：XX，售价：XX，目标用户：XX"，Bot 就会按"先分析后写文案"的流程输出。这就是把技巧"产品化"——一次配置，反复使用。

## 四、验收标准

学完本章，用下面三个任务自测：

1. **角色扮演任务**：选一个你熟悉的职业（律师、健身教练、理财顾问都行），写一段角色设定，让 AI 用这个角色的视角解释"什么是复利"。验收：AI 的回答能体现该职业的视角和术语，不是泛泛而谈。
2. **Few-shot 任务**：给 AI 三个"用户评论 → 情感标签"的例子，让它给第四条评论打标签，输出 JSON。验收：AI 输出的 JSON 格式和字段跟例子完全一致，标签合理。
3. **CoT 任务**：找一道需要三步以上计算的应用题，分别用"直接问"和"加 CoT"两种方式问 AI，对比答案。验收：加了 CoT 的版本，推理过程清晰可见，且答案正确率不低于直接问。

**场景判断自测**：遇到下面三个任务，你该用哪个技巧？

- 把会议纪要整理成固定格式的表格 → Few-shot（格式固定，给例子最有效）
- 帮你算一笔房贷提前还款是否划算 → CoT（多步计算，要看到推理）
- 给一篇技术博客写个有网感的标题 → 角色扮演（需要特定风格）

三个都能答对，说明你已经能根据场景选技巧了。

## 五、悟本质：提示词的本质是"降低不确定性"

三个技巧看起来各不相同，但本质是同一件事：**降低 AI 理解你意图的不确定性**。

- 角色扮演，降低的是"视角和风格"的不确定性——AI 不用猜你要什么调性，角色已经定了。
- Few-shot，降低的是"输出格式和模式"的不确定性——AI 不用猜你要什么结构，例子已经演示了。
- CoT，降低的是"推理路径"的不确定性——AI 不用一次性从问题跳到答案，可以把中间步骤铺开，每一步都有据可依。

为什么"降低不确定性"这么重要？因为大模型的生成本质是概率采样（见第 04、09 章）。同一个问题，AI 内部的概率分布是分散的——它"可能"往十个方向回答。你的提示词每加一个约束，就砍掉几个不想要的方向，让概率集中到你想要的那条路上。

从这个角度看，写提示词和"指挥一个聪明但没经验的实习生"是一回事：你不需要手把手教他每一步，但你需要把"我要什么、不要什么、参考什么"讲清楚。讲得越清楚，他越不容易跑偏。

还有一个推论：**当 AI 能力变强，提示词的"约束"可以变少**。早期的模型需要你写很长的 prompt 才能给出好结果；现在的强模型，很多时候一句话就够了。但这不意味着技巧没用——能力越强，对"模糊指令"的容忍度越高，但遇到复杂任务、严格格式、需要推理的场景，三个技巧依然立竿见影。技巧不会过时，过时的只是"对弱模型才需要的那些繁琐写法"。

## 六、结语

提示词不是写得越长越好，而是越能降低 AI 的不确定性越好——角色定视角，Few-shot 定格式，CoT 定推理，三招够用大半辈子。
