# 第 08 章 多模态入门：让 AI 看图、听声、读表

> 前置知识：学完第 04 章（Token）即可
> 学完你能：①理解多模态 AI 的基本原理 ②知道什么时候该用图像/语音输入 ③看懂 GPT-4o、Claude、Gemini 的多模态能力差异

## 讲清楚

本章解决一个问题：AI 不只能读文字，它还能看图、听语音、看表格。这是怎么做到的？

2024 年之后，"多模态"（Multimodal）已经成了大模型的标配。GPT-4o、Claude 3.5、Gemini 1.5、Qwen-VL、DeepSeek-VL——你能叫得出名字的新模型，几乎都支持图像输入，部分还支持语音和视频。如果你还在用"只能输入文字"的方式和 AI 对话，等于只用了一半的能力。

这一章写给三类人看：一是想用 AI 处理图片/语音的普通人，二是想给应用加多模态能力的程序员，三是想看懂各家模型差异的人。为什么重要？因为很多任务用文字描述又慢又差，扔张图过去三秒搞定——多模态不是炫技，是效率工具。

## 看案例

三个真实场景。

**场景一：给 AI 一张冰箱照片让它写菜谱。** 你打开冰箱，拍一张照片扔给 GPT-4o："冰箱里有这些食材，帮我设计三天的晚餐。"模型识别出鸡蛋、西红柿、青菜、剩饭、半块豆腐，给你列出三天的菜单和做法。整个过程十秒。如果用纯文字，你得一个一个打字报菜名，光列清单就五分钟。

**场景二：给 AI 一张报错截图让它修 bug。** 程序员在 Cursor 里贴一张终端报错截图，问"这个错怎么修"。模型识别出错误信息、定位到代码行、给出修复方案。比复制粘贴文字快，而且截图保留了完整上下文（包括终端颜色、堆栈层级），信息更全。

**场景三：给 AI 一段语音让它总结会议。** 录了半小时会议，扔给 Claude，让它总结要点、提取待办、列出争议。模型直接听语音输出文字，不用先转录再总结。GPT-4o 甚至能实时语音对话，你说话它秒回，像打电话一样。

这三个场景有个共同点：**信息从"文字"扩展到了"图像"和"声音"**。AI 不再是个只会读字的瞎子，它长了眼睛和耳朵。

## 拆原理

### 一、图像 Token 化：把照片切成马赛克拼图

第 04 章讲过，文字被切成 Token 喂给模型。图像怎么办？答案是：**也切成 Token**。

具体做法叫 **Patch 化**（Patch Tokenization）。把一张图片切成固定大小的小方块（比如 16×16 像素一块），每个方块当成一个"图像 Token"。一张 224×224 的图片，切成 16×16 的方块，就是 14×14=196 个 Patch，也就是 196 个 Token。

每个 Patch 经过一个视觉编码器（通常是 Vision Transformer，简称 ViT），变成一个向量。这个向量和文字 Token 的向量在同一个维度空间里，可以拼在一起喂给大模型。

类比一下：把一张照片切成马赛克拼图，每一块马赛克是一个"词"。模型读图就像读一段文字——只不过这段"文字"是由马赛克拼图组成的。

这个思路最早由 Google 在 2020 年的 ViT 论文里提出。「Dosovitskiy et al., An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale, ICLR, 2021」后来 OpenAI 的 CLIP 模型把它和文字对齐起来，奠定了多模态的基础。「Radford et al., Learning Transferable Visual Models From Natural Language Supervision, ICML, 2021」

**关键洞察**：图像和文字在底层都是"Token 序列"。模型不关心 Token 来自文字还是图像，它只关心 Token 之间的关系。这就是多模态能跑通的根基。

### 二、跨模态对齐：让"猫"这个词和猫的图对上号

光把图像切成 Token 还不够。模型还得知道"猫"这个词和一张猫的图片说的是同一个东西。这就需要**跨模态对齐**（Cross-modal Alignment）。

类比一下：你有一本中英词典，左边是中文词，右边是英文词。"猫"对应"cat"，"狗"对应"dog"。词典的作用是把两种语言"对齐"到同一个意思上。

跨模态对齐做的事类似：把"猫"这个词的向量和猫的图像的向量，映射到同一个向量空间里，让它们距离很近。训练方法是对比学习——给模型看几百万对"图片+描述"（比如一张猫的图配文字"一只橘猫趴在窗台上"），让它学会把相关的图文拉近、不相关的拉远。

对齐之后，模型就"懂"了：用户说"猫"，和用户传一张猫的图，指的是同一个东西。它可以跨模态推理——你给它一张图问"这是什么"，它能用文字回答"这是一只猫"；你给它一段文字"画一只猫"，它能生成猫的图（图像生成模型干的事）。

**关键洞察**：多模态不是"图像模型 + 文字模型"简单拼接，而是把两种模态映射到同一个语义空间。对齐做得好不好，直接决定多模态能力强弱。

### 三、语音多模态：让 AI "听"和"说"

语音比图像更复杂，因为它有时间维度——一段语音是连续的音频流，不是静态的图。

早期做法是"拼接三个模型"：语音转文字（ASR）+ 文字大模型 + 文字转语音（TTS）。你说话 → ASR 转成文字 → 大模型回答 → TTS 转成语音。能用，但慢，而且丢失了语气、情感、停顿这些信息。

新一代做法是**原生语音多模态**：直接把音频切成 Token（音频 Token），和文字、图像 Token 一起喂给模型。模型直接"听"音频，直接"说"音频，不用中间转文字。

GPT-4o 走的就是这条路。OpenAI 在发布会上演示过：用户对着手机说话，GPT-4o 秒回，还能模仿语气、唱歌、学口音。这种实时感和情感表达，是"拼接三个模型"做不到的。

**关键洞察**：原生多模态的优势是"信息无损"。语音里的语气、停顿、情感，文字转写会丢失，但音频 Token 保留。这就是为什么 GPT-4o 听起来比传统语音助手"有感情"。

### 四、原生多模态 vs 拼接多模态：为什么"原生"更强

现在市面上的多模态模型分两类：

**拼接派**：文字模型 + 视觉模型 + 语音模型，三个模型用胶水粘起来。早期 GPT-4（带 Vision 的版本）就是这种——视觉模型识别图像，转成文字描述，再喂给文字模型。优点是工程简单，缺点是信息损失大、延迟高。

**原生派**：一个模型从头训练就同时吃文字、图像、音频 Token，三种模态在同一个 Transformer 里处理。GPT-4o、Gemini 1.5 是这种。优点是信息无损、延迟低、能跨模态推理（比如看视频同时听声音），缺点是训练难度大、数据要求高。

为什么原生更强？因为**模态之间会互相补充**。看一段视频，光看画面你不知道背景音乐是欢快还是悲伤；光听声音你不知道画面里是谁在说话。原生多模态能同时处理两种信号，理解更完整。拼接派做不到——视觉模型和音频模型各干各的，最后在文字层面汇合，丢掉了模态间的细粒度关联。

各家模型的多模态能力差异，主要就体现在"原生程度"上：

- **GPT-4o**：原生多模态，文字+图像+音频统一处理，实时语音对话是招牌
- **Claude 3.5**：图像输入强（读图表、识别 UI 截图准），但语音能力相对弱
- **Gemini 1.5**：原生多模态，强在长上下文（可处理超长视频和音频）
- **Qwen-VL / DeepSeek-VL**：开源阵营的视觉模型，图像理解能力接近闭源第一梯队

选模型时看你的场景：要实时语音选 GPT-4o；要读复杂图表选 Claude；要处理长视频选 Gemini；要本地部署选 Qwen-VL。

**关键洞察**：原生多模态的本质不是"加更多模态"，而是"让模态之间能对话"。未来的方向是视频、3D、触觉都进来，模型变成一个全感官的智能体。

## 避坑指南

**坑一：以为 AI 真的"看懂"了图。**

模型识别图像，本质是在向量空间里找模式匹配。它"认出"一只猫，不是像人那样理解猫是什么，而是因为训练数据里有几百万张猫的图，它学会了猫的视觉特征。所以模型会被对抗样本骗——稍微改几个像素，模型把熊猫认成长臂猿。涉及安全、医疗、法律的图像判断，不能完全信模型，必须有人工复核。

**坑二：用多模态做需要像素级精度的任务。**

多模态模型擅长"看个大概"——识别物体、理解场景、读文字。但不擅长像素级精度任务，比如测量零件尺寸、识别微小瑕疵、读密密麻麻的表格数字。这类任务该用专门的计算机视觉模型，不是大模型。把大模型当万能锤子，会把所有问题都砸歪。

**坑三：忽视图像 Token 的成本。**

图像不是免费的。一张高清图扔给 GPT-4o，可能消耗几百到几千个 Token，比一段长文字还贵。批量处理图片时账单会爆炸。优化方法：上传前压缩图片、裁掉无关区域、降低分辨率。API 调用时关注图像 Token 计费规则，别等账单来了才心疼。

**坑四：以为多模态模型什么图都能处理。**

模型对图像的理解受训练数据限制。常见物体（猫狗、汽车、食物）识别很准，但小众领域（医学影像、工业图纸、卫星图）容易翻车。专业领域要么微调，要么用专门模型，别指望通用多模态模型包打天下。

**坑五：把语音多模态当 ASR 用。**

原生语音多模态（GPT-4o）能听懂语气、情感、停顿，但很多应用还停留在"语音转文字再处理"的老路。如果你的任务需要理解语气（客服情绪分析、会议氛围判断），用原生语音多模态效果远好于 ASR + 文字模型。反之，如果只是要文字记录，ASR 更便宜。

## 悟本质

多模态的本质是**统一表征**——把不同形式的信息（文字、图像、音频）翻译成同一种"语言"（Token 序列），让一个模型能同时处理。

这个洞察可以迁移到很多地方：

- **跨语言**：中文、英文、法文，底层都是 Token，模型能跨语言迁移知识
- **跨模态**：文字、图像、音频，底层也都是 Token，模型能跨模态推理
- **跨领域**：代码、数学、音乐，只要能 Token 化，就能塞进大模型

更深一层：**任何能被"离散化 + 序列化"的信息，都能被大模型处理**。这就是为什么大模型的边界一直在扩展——从文字到图像，从图像到音频，从音频到视频，从视频到 3D，从 3D 到机器人动作。每多一种模态，模型就多一个"感官"。

理解了这一点，就能预测未来：多模态不会停在图像和语音。下一步是视频（已经出现，如 Sora、Gemini 1.5 视频）、3D（机器人、AR/VR）、甚至脑电信号。每一种新模态的加入，都会打开一批新应用。

对普通人来说，多模态意味着**用最自然的方式和 AI 交互**——不用打字，拍照就行；不用敲键盘，说话就行。对程序员来说，多模态意味着**重新设计交互界面**——从"输入框 + 按钮"变成"摄像头 + 麦克风 + 屏幕"。这是下一代人机交互的范式转移。

## 结语

多模态的本质，是把文字、图像、声音翻译成同一种语言，让 AI 长出眼睛和耳朵。

## 自测三问

**问题一：为什么图像能和文字一起喂给同一个大模型？**

参考答案：图像被切成 Patch（小方块），每个 Patch 经过视觉编码器变成一个向量，这个向量和文字 Token 的向量在同一个维度空间里。模型不关心 Token 来自文字还是图像，只处理 Token 序列。所以图像和文字可以拼在一起喂给同一个模型。

**问题二：原生多模态（如 GPT-4o）比"拼接多个模型"强在哪里？**

参考答案：原生多模态用一个模型同时处理文字、图像、音频 Token，三种模态在同一个 Transformer 里交互，信息无损、延迟低、能跨模态推理（比如看视频同时听声音）。拼接派是三个模型各干各的，最后在文字层面汇合，丢失了模态间的细粒度关联，延迟也高。

**问题三：什么任务不适合用多模态大模型？**

参考答案：需要像素级精度的任务（测量尺寸、识别微小瑕疵、读密集表格数字）、专业领域图像（医学影像、工业图纸、卫星图）、以及纯文字记录任务（用 ASR 更便宜）。多模态大模型擅长"看个大概"和跨模态推理，不擅长精度任务和专业领域。
