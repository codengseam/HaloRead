# 第 09 章：幻觉从哪来：AI 为什么会一本正经胡说

> 前置知识：学完第 04、07 章（Token、训练三阶段）即可
> 学完你能：①理解幻觉的根本原因 ②识别哪些场景容易触发幻觉 ③不再被 AI 的"自信"骗到

## 一、讲清楚：被坑过的人，才知道这事有多疼

如果你用过 AI 写过论文、查过资料、问过代码，大概率有过这样的经历：

你问它一个看起来很专业的问题，它给你一段非常流畅、非常自信、非常"像真的"的回答。引用了论文，给出了作者、年份、期刊、页码，甚至还贴心地写了一段摘要。你信了，把这段塞进自己的稿子里。

第二天同事一查：这篇论文根本不存在。

这就是幻觉（Hallucination，大模型生成看似合理但实际错误或虚构内容的现象）。它是大模型最被诟病的问题，没有之一。OpenAI、Anthropic、Google、Meta，所有大厂发布新模型时，技术报告里都会专门列一栏 Hallucination 评测，因为它直接决定了模型能不能用、敢不敢用、用在什么场景。

为什么这一章重要？因为理解幻觉的成因，比记住"AI 会出错"这件事本身重要得多。前者让你能预判什么时候会出错、为什么出错、怎么避开；后者只是让你害怕，却不知道往哪儿躲。

本章只讲一件事：幻觉从哪来。防幻觉的具体技巧（RAG、引用核查、约束生成等）见第 13 章。

## 二、看案例：三个被 AI 坑过的真实场景

### 案例 1：律师拿 ChatGPT 编的判例上法庭

2023 年 5 月，纽约一位律师 Steven Schwartz 用 ChatGPT 帮自己准备诉讼材料。ChatGPT 给他列了一堆"相关判例"，附带了案件名称、引用编号、判决摘要。律师没核实，直接交了上去。

法官一看：这些判例全是假的。案件不存在，引用编号是编的，判决内容是 AI 拼出来的。

这位律师后来在法庭上辩解："我不知道 ChatGPT 会编造案例，我以为它是个搜索引擎。"

这句话本身就是一个最大的误解——ChatGPT 从来不是搜索引擎，它是续写器。这个案例被《纽约时报》等多家媒体报道，成为 AI 幻觉最著名的反面教材。

### 案例 2：AI 给你一个不存在的 API

程序员圈子里更常见的坑是这样的：你问 AI "怎么用 Python 调用某个库的某个方法"，它给你一段代码，里面用了一个看起来非常合理的函数名，比如 `pandas.merge_asof_with_tolerance()`。

你复制粘贴，运行，报错：`AttributeError: module 'pandas' has no attribute 'merge_asof_with_tolerance'`。

你查 pandas 文档，发现根本没这个函数。AI 把"merge_asof"和"with tolerance"这两个真实存在的概念拼到了一起，造了一个"看起来最像真的"的 API 名。

这种坑在 Cursor、Copilot、Trae 这些代码工具里都出现过。模型越强，造的 API 越像真的，因为它的语言建模能力越强，越能把"看起来合理"做到极致。

### 案例 3：把历史人物张冠李戴

你问 AI："苏轼的《赤壁赋》写于哪一年？"

它可能告诉你："写于元丰五年，也就是 1082 年，苏轼被贬黄州期间所作。"——这个是对的。

但如果你问得更细："苏轼写《赤壁赋》时，王安石在做什么？"

它可能给你编一段："王安石此时已经第二次罢相，退居江宁，正在写《泊船瓜洲》。"——前半句对，后半句的时间线对不上。《泊船瓜洲》写于 1076 年左右，而《赤壁赋》写于 1082 年，差了六年。

AI 不会告诉你"我不知道"，它会把不同时间点的事件揉到一起，生成一个"听起来很顺"的叙述。这就是历史领域幻觉的典型形态：不是完全瞎编，而是真假掺半，让你更难分辨。

## 三、拆原理：幻觉到底是怎么产生的

要理解幻觉，得先放下一个根深蒂固的误解：**AI 不是在"回忆"事实，AI 是在"生成"文本**。

这两件事看起来差不多，本质完全不同。

### 1. 大模型本质是"概率续写"

回到第 04 章讲过的 Token。大模型看到一段文字，做的事情是：根据已经出现的 Token，预测下一个 Token 最可能是什么。

注意这个词——"最可能"。

它不是在数据库里查"正确答案"，它是在算"接下来哪个词出现的概率最高"。这两件事在大部分日常对话里看起来一样，但在事实性问题上完全不同。

打个比方：你问一个没读过《三国演义》但读过很多历史小说的人"赤壁之战是谁指挥的"，他可能根据"赤壁""火攻""水战"这些线索，猜出一个"最像真的"答案——周瑜、诸葛亮、黄盖，反正就那几个人。他不是在回忆事实，他是在生成"听起来最合理的续写"。

大模型干的就是这件事，只不过规模大了几亿倍。

所以当 AI 给你编造一篇不存在的论文时，它不是"故意骗你"，它是在算：在"论文标题""作者""年份""期刊"这些 Token 出现之后，下一个最可能出现的 Token 是什么。它生成的是"最像真的"，不是"真的"。

这就是幻觉的根本原因：**模型优化的是"看起来对"，不是"事实对"**。

### 2. 训练数据里本来就有错、有旧、有冲突

第 07 章讲过训练三阶段：预训练、指令微调、RLHF。预训练阶段，模型把互联网上几万亿 Token 的文本"读"了一遍。

问题是：互联网上的文本，本来就是真假掺半的。

- 有错：维基百科有错条目，论坛里有错误回答，新闻里有失实报道。
- 有旧：2020 年的训练数据里，"最新的 iPhone 是 iPhone 12"；2024 年再问，模型可能还停留在旧认知。
- 有冲突：同一个问题，A 网站说对，B 网站说错，模型两个都学过，生成时哪个概率高就用哪个。

模型不会去"核实"哪个对，它只是把所有文本里出现的统计规律学下来。所以当它生成内容时，它生成的其实是"训练数据里这种语境下最常出现的说法"——这个说法可能对，可能错，可能过时，可能互相矛盾。

更麻烦的是：训练数据里"自信地胡说"的样本非常多。论坛里有人 confidently 给出错误答案，博客里有人编造经历，营销号有人夸大其词。模型把这些"自信地说"的模式也学到了。

### 3. RLHF 的副作用：对齐让模型更"礼貌自信"

这是很多人没想到的一层。

RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）是训练第三阶段，目的是让模型"听话""礼貌""有用"。人工标注员给模型的回答打分，分数高的回答被强化，分数低的被弱化。

听起来是好事，但实际有个副作用：标注员更喜欢"自信、流畅、有结构"的回答，而不是"犹豫、含糊、说不知道"的回答。

你想啊，两个回答摆在你面前：

- 回答 A："这个问题的答案可能是 X，但我不是完全确定，建议你进一步核实。"
- 回答 B："这个问题的答案是 X，原因是……（一段流畅的解释）。"

哪个看起来更"专业"？大部分人会给 B 打高分。

于是模型学到了一件事：**自信地说，比诚实地说不知道，更容易拿到高分**。

这就是 RLHF 对幻觉的强化作用。它不是直接教模型编造，而是教模型"用更自信的语气说话"——而自信的语气，恰好让幻觉更难被识别。

吴恩达在 Generative AI for Everyone 课程里专门讲过 LLM 的这一局限：模型生成的是"看起来合理的文本"，而不是"经过核实的事实"，这是使用大模型时必须建立的基本认知。「Andrew Ng, Generative AI for Everyone · DeepLearning.AI，2023」

### 4. 模型不知道自己"不知道什么"

人有一个能力：知道自己不知道。你问我某个冷门领域的问题，我会说"我不懂，你去查查"。这个能力叫元认知（Metacognition，对自己认知过程的觉察与监控）。

大模型没有真正的元认知。它不会评估自己"对这个问题的把握有多大"，它只会算"下一个 Token 的概率分布"。哪怕概率分布很扁平（也就是说它其实很"不确定"），它生成出来的时候依然是流畅、自信的语气。

因为流畅和自信是训练阶段被强化的"形式"，而"不确定"这种心理状态，模型根本没有。

这就是为什么 AI 编造论文时那么理直气壮——它没有"我可能在编"这个念头，它只是在做概率续写。

## 四、避坑指南：哪些场景容易触发幻觉

知道了成因，就能反推哪些场景最容易踩坑。下面这些场景，请把 AI 的输出默认当成"待核实"。

### 1. 让 AI 凭空回忆事实

典型问法："XX 的出生年份是？""XX 论文讲了什么？""XX API 怎么用？"

这种问题模型没有外部信息可查，只能从训练数据里"回忆"。越冷门、越具体、越新的事实，越容易出错。

避坑：事实性问题不要让 AI 凭空答。要么自己查原始来源，要么用带联网搜索的工具（如 Perplexity、Coze 的搜索插件），要么用 RAG（第 13 章讲）。

### 2. 相信 AI 引用的"来源"

AI 给你列的论文、链接、引用编号，默认是假的，直到你能在外部数据库里查到为止。

不是说 AI 一定编，而是它编的概率高到不能信任。律师 Schwartz 那个案例就是典型——他信了 ChatGPT 给的判例引用，结果全是假的。

避坑：AI 给出的任何"具体来源"（论文标题、URL、引用编号、版本号），都要去 Google Scholar、arXiv、官方文档里复核一遍。

### 3. 在医疗、法律、财务上全信 AI

这三个领域是幻觉的高危区，因为：

- 术语多、细节多，模型容易把相似概念搞混。
- 后果重，错一个剂量、错一个条款、错一个小数点，代价都是真实的。
- 训练数据里"看起来专业但实际错误"的内容特别多（医疗养生号、法律科普号、财经自媒体）。

避坑：这三个领域，AI 只能用来做"初筛"和"理解概念"，不能用来做"决策"。最终判断必须由专业人士做出。

### 4. 让 AI 写长篇且不核查

短回答里出错，你一眼能看出来。长篇里出错，你很容易漏掉。AI 写一份 5000 字的报告，里面混了 3 处事实错误，你逐字读都未必能发现。

避坑：长篇输出必须分段核查，尤其是事实性段落。可以让 AI 自己标注"哪些是事实、哪些是推测"，但即便如此也要复核。

### 5. 问 AI 关于它自己的事

"你用的是哪个版本的模型？""你的训练数据截止到什么时候？""你的上下文窗口多大？"

这些问题 AI 经常答错，因为它对自己的元信息并不"知道"，它只是根据训练数据里"关于 AI 的描述"来生成回答。模型版本、能力边界这些信息，以官方文档为准。

避坑：关于模型自身的信息，查官方文档，不要问模型本身。

## 五、悟本质：幻觉不是 bug，是特性

讲到这里，你应该能感觉到一个反直觉的结论：

**幻觉不是大模型的 bug，而是概率模型的特性。**

为什么这么说？

因为大模型的工作机制本身就是"基于概率生成文本"。这个机制让它能写诗、能写代码、能聊天、能翻译——这些任务都不要求"事实准确"，只要求"语言合理"。在这些任务上，"概率续写"是优势，不是劣势。

问题出在：我们用同一套机制去回答事实性问题。事实性问题的标准是"对错"，不是"合理"。用"生成合理文本"的机制去回答"要求事实准确"的问题，必然会有幻觉。

这就像用一把锤子去拧螺丝——锤子没坏，是你用错了工具。

理解这一点，有几个推论：

**推论 1：幻觉无法靠"把模型做得更大"完全消除。**

模型越大，语言能力越强，幻觉可能越"逼真"，但不会消失。因为根源在机制，不在规模。GPT-4 比 GPT-3 幻觉少一些，但依然有；未来的 GPT-5 也不会零幻觉。

**推论 2：要消灭幻觉，得靠外部约束。**

模型自己没有"事实"这个概念，事实只能从外部来。常见的外部约束有：

- RAG（Retrieval-Augmented Generation，检索增强生成）：先从外部知识库检索相关文档，再让模型基于检索到的内容回答。第 13 章会详细讲。
- 工具调用：让模型调用计算器、搜索引擎、数据库，把"事实"交给工具去查。
- 引用核查：让模型生成的每条事实都附带可验证的来源。

**推论 3：用 AI 的正确姿势，是"扬长避短"。**

长：语言生成、代码生成、创意发散、结构化整理、长文本理解。
短：事实回忆、精确计算、实时信息、自身元信息。

把 AI 当成一个"语言能力极强但事实感为零"的实习生来用——让它写、让它改、让它整理，但事实性的东西你来把关。这个心态一旦建立，你就不会被它的"自信"骗到了。

## 六、结语

幻觉不是 AI 在骗你，是 AI 在"猜"。猜得越像真的，你越要警惕。

## 七、自测三问

**问题 1：为什么模型越大，幻觉反而可能更难识别？**

参考答案要点：模型越大，语言建模能力越强，生成的幻觉文本越流畅、越像真的、越难看出破绽。幻觉的"量"可能减少（因为知识容量大了），但"质"更难分辨——它造的假更逼真了。

**问题 2：RLHF 是怎么间接强化幻觉的？**

参考答案要点：RLHF 阶段，人工标注员倾向于给"自信、流畅、有结构"的回答打高分，给"犹豫、说不知道"的回答打低分。模型因此学到了"自信地说"比"诚实地说不知道"更容易拿高分，于是生成时更倾向于用自信的语气，即使内容是编的。

**问题 3：如果你必须用 AI 回答一个事实性问题（比如某个 API 的用法），你会怎么做来降低幻觉风险？**

参考答案要点：①不让 AI 凭空回忆，而是把官方文档喂给它，让它基于文档回答（这是 RAG 的雏形）；②让 AI 给出具体来源（函数名、文档链接），自己去官方文档复核；③如果 AI 给的代码跑不通，第一时间怀疑是 API 名编造，去文档查证而不是反复问 AI。
