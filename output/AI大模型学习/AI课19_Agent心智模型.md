# 第 19 章：Agent 心智模型：从聊天到干活

> 前置知识：学完第 10 章（消息结构）即可
> 学完你能：①理解 Agent 和普通聊天的本质区别 ②掌握 Agent 的核心公式 ③看懂 Agent 应用的架构

## 一、讲清楚：从"回答问题"到"完成任务"的跃迁

前面十几章，我们讲的都是"怎么和 AI 聊天"——怎么写 prompt、怎么设 system、怎么构造消息。这些都是在解决一个问题：让 AI 把话答好。

但 2023 年之后，AI 圈最热的词不再是"聊天"，而是"Agent"（智能体，能自主感知环境、做出决策、调用工具完成任务的 AI 系统）。原因很简单：只会聊天的 AI，价值有限。

举个最直观的对比：

- 你问 ChatGPT："帮我订一张明天去北京的机票。"它会回答："很抱歉，我无法直接订机票，建议您访问携程或 12306……"——它给你建议，但不替你做。
- 你对一个 Agent 说同样的话，它会：查你明天的日程 → 搜航班 → 比价 → 选一个合适的 → 调用订票接口 → 把订单号返回给你。

前者是"嘴"，后者是"手"。这个从"会说"到"会做"的跃迁，就是 Agent。

本章解决三个问题：Agent 到底是什么、它和普通聊天有什么本质区别、一个 Agent 应用长什么样。学完之后，你看 LangChain、Coze 的 Bot、AutoGPT 这些东西，都能一眼看懂它们的骨架。

## 二、看案例：同一个任务，聊天机器人 vs Agent

### 案例：规划三天北京旅行

我们拿一个具体任务来对比——"帮我规划一个三天的北京旅行"。

**聊天机器人的回答**（典型 ChatGPT 风格）：

> 第一天：上午故宫，下午天坛，晚上王府井；第二天：长城一日游；第三天：颐和园 + 圆明园。建议提前订酒店，住东直门附近交通方便。

回答得很完整，但有几个问题：

- 它不知道你哪天去，所以"明天""下周"它都没法对齐。
- 它不知道机票酒店的真实价格，"建议提前订"是空话。
- 它不知道你预算多少、有没有老人小孩、爱不爱爬山。
- 最关键：它说完就完了，你还得自己打开携程订机票、订酒店、查天气。

**Agent 的执行过程**（典型 Agent 框架的行为）：

1. 先问你几个澄清问题："哪天出发？几个人？预算多少？有没有特殊需求？"
2. 拿到答案后，调用天气 API 查北京那几天的天气。
3. 调用航班搜索 API，查出发地到北京的机票。
4. 调用酒店搜索 API，按你的预算和位置偏好筛酒店。
5. 综合天气、交通、景点开放时间，排出一个行程。
6. 问你："机票选 9 点那班可以吗？酒店选了东直门的一家，350 一晚，行吗？"
7. 你确认后，调用订票接口、订酒店接口，把订单号发给你。
8. 最后生成一份行程单 PDF，发到你的邮箱。

差别在哪？聊天机器人给你"建议"，Agent 替你"把事办了"。

这就是 Agent 的核心特征：**它不只是回答，它会行动**。

## 三、拆原理：Agent 的核心公式

### 1. Agent = LLM + Planning + Tool use + Memory

业界对 Agent 有一个被广泛引用的公式（参见「菜鸟教程 · AI Agent，什么是 Agent」）：

> **Agent = LLM（大脑）+ Planning（规划）+ Tool use（执行）+ Memory（记忆）**

逐个拆开看：

**LLM（大脑）**：Agent 的核心推理引擎。它负责理解任务、做决策、生成自然语言。没有 LLM，就没有 Agent——前面那些"会查 API、会订票"的能力，本质都是 LLM 在指挥。LLM 在 Agent 里的角色，就像人的大脑：接收信息、思考、下指令。

**Planning（规划）**：把一个大目标拆成可执行的子步骤。比如"规划北京旅行"这个目标，会被拆成"查天气 → 查机票 → 查酒店 → 排行程 → 确认 → 下单"。规划能力是 Agent 区别于聊天机器人的关键——聊天机器人是"问一句答一句"，Agent 是"接到目标后自己拆解、自己排步骤"。

**Tool use（执行）**：调用外部工具完成具体动作。LLM 本身只会"说话"，不会"做事"。要查天气，得调天气 API；要订票，得调订票接口；要算数学，得调计算器。Tool use 就是给 LLM 装上"手"和"脚"，让它能真正影响外部世界。

**Memory（记忆）**：记住中间状态和历史信息。Agent 执行一个任务可能要十几步，每一步的结果都要存下来，下一步才能接着用。比如查完机票，要把航班号、价格、时间记下来；查完酒店，要把酒店信息记下来；最后排行程时，这些信息都要用上。Memory 分短期记忆（当前任务的上下文）和长期记忆（跨任务的经验）。

这四个组件缺一不可。少了 LLM，Agent 没法推理；少了 Planning，Agent 只会一问一答；少了 Tool use，Agent 只会说话不会做事；少了 Memory，Agent 做完一步就忘了上一步。

### 2. 聊天 vs Agent：两种完全不同的工作模式

很多人把 Agent 理解成"更聪明的聊天机器人"，这是根本性的误解。两者的工作模式完全不同。

**聊天机器人的模式：输入 → 输出**

```
用户输入 → LLM 生成回复 → 输出
```

一条直线。用户问什么，模型答什么，结束。下一轮对话是新一轮的"输入→输出"。

**Agent 的模式：目标 → 规划 → 执行 → 反馈 → 迭代**

```
用户给目标 → Agent 拆解规划 → 调用工具执行 → 观察结果 →
  → 如果没完成，回到规划 → 如果完成，返回最终结果
```

一个循环。Agent 接到目标后，不是直接给答案，而是先想"这事分几步做"，然后一步步执行，每步执行完看结果对不对，不对就调整，对了就继续，直到任务完成。

这个差别决定了两者适用的场景完全不同：

- **聊天机器人适合**：问答、写作、翻译、解释——一次性、不需要外部动作的任务。
- **Agent 适合**：订票、写代码并运行、数据分析并出报告、自动化办公流程——需要多步骤、需要调用外部工具、需要根据中间结果调整的任务。

### 3. Agent 的自主性：自己决定下一步做什么

Agent 最让人震撼的特点是"自主性"——它能自己决定下一步做什么。

普通聊天机器人是被动的：你问一句，它答一句。你不问，它不动。

Agent 是主动的：你给它一个目标，它自己决定"先做什么、再做什么、什么时候调什么工具、什么时候该问用户、什么时候算完成"。

举一个写代码的例子。你对一个编程 Agent 说："帮我写一个网页爬虫，爬豆瓣电影 Top 250。"

聊天机器人会直接给你一段代码，能不能跑、爬不爬得到，它不管。

Agent 的行为是这样的：

1. 先想：这事分几步？——写爬虫 → 测试 → 修 bug → 输出结果。
2. 写第一版代码。
3. 自己运行代码（调用执行工具）。
4. 发现报错了（比如被反爬）。
5. 自己分析报错，改代码：加 User-Agent、加延时。
6. 再运行，发现爬到一半被封 IP。
7. 改用代理池，再运行。
8. 爬完了，把结果整理成 CSV，输出给你。

整个过程没有人干预，Agent 自己发现问题、自己调整、自己迭代。这就是"自主性"。

自主性的来源还是那个公式：LLM 负责思考和决策，Planning 负责拆步骤，Tool use 负责执行，Memory 负责记住中间结果。四个组件配合，Agent 就能"自己跑起来"。

### 4. 从"对话框问答"到"目标驱动"的思维转变

理解 Agent，最大的门槛不是技术，而是思维方式的转变。

**聊天思维**：我把问题问清楚，AI 把答案给清楚。重点是"问得准"和"答得对"。

**Agent 思维**：我把目标描述清楚，Agent 自己想办法完成。重点是"目标定得对"和"让 Agent 有能力执行"。

这个转变体现在三个层面：

**第一，从"问句"到"目标"。**

聊天时你说："Python 怎么读 CSV 文件？"——这是一个问句。
用 Agent 时你说："帮我把这个 CSV 文件里的数据按销售额排序，输出一份报告。"——这是一个目标。

问句期待答案，目标期待结果。

**第二，从"一轮"到"多轮自主"。**

聊天时每一轮都是你主导：你问，它答；你再问，它再答。
用 Agent 时你只发起一次，剩下的轮次 Agent 自己跑：它自己问自己、自己答自己、自己调工具、自己看结果。

**第三，从"语言输出"到"动作输出"。**

聊天机器人的输出是文字。Agent 的输出可以是文字，也可以是"一个动作"——发了一封邮件、下了一个订单、改了一行代码、生成了一个文件。

这个思维转变为什么重要？因为很多人用 Agent 框架（比如 Coze 搭 Bot、用 LangChain 写应用），还是按聊天机器人的思路来用——一问一答、每步都干预。结果 Agent 的能力完全没发挥出来，还觉得"Agent 也就这样"。

正确的用法是：把目标给清楚，把工具配齐，然后放手让 Agent 自己跑。你只在关键节点确认或纠偏。

## 四、避坑指南：Agent 的三个常见误用

### 1. 把 Agent 当聊天机器人用

典型表现：搭了一个 Agent，但每次只问一句话、等一句话回答，不让它调工具、不让它多步执行。

这就好比买了一辆车，却只用它当椅子坐。Agent 的价值在于"自主完成多步任务"，你把它当聊天框用，它就退化成一个又慢又贵的聊天机器人（因为 Agent 框架的 overhead 比直接调 LLM 高）。

正确用法：给 Agent 一个"需要多步完成"的目标，让它自己拆解、自己执行。如果任务一步就能答完，直接用聊天机器人就行，不必上 Agent。

### 2. 给 Agent 太模糊的目标

典型表现：

```
用户：帮我搞个营销方案。
Agent：（懵）……搞什么的？给谁看？预算多少？什么渠道？
```

Agent 的自主性建立在"目标清晰"之上。目标模糊，Agent 要么反复问你（体验差），要么自己瞎猜（结果差）。

给 Agent 目标，要像给实习生派活一样具体：

- **做什么**：写一份小红书种草文案。
- **给谁看**：25-35 岁女性，关注护肤。
- **要什么风格**：口语化，有真实体验感，不要硬广。
- **交付什么**：3 篇文案，每篇 300 字以内。
- **约束**：不要出现"全网最低价"这类违规词。

目标越具体，Agent 的自主性越能发挥；目标越模糊，Agent 越容易跑偏。

### 3. 忽视 Agent 的"跑偏"风险

Agent 有自主性，自主性意味着它可能"自己跑偏"。

真实案例：有人让 Agent "帮我研究一下某只股票，写个分析报告"。Agent 上网搜资料、查财报、写报告，一切正常。但中途它搜到一篇负面新闻，顺着链接一路点进去，最后花了一小时研究这家公司的竞争对手，报告写成了"为什么不建议买这只股票"——和用户初衷完全相反。

这种跑偏在 Agent 应用里很常见。原因有几个：

- **LLM 的注意力会被新信息带跑**：搜到一个有意思的链接，就顺着往下走，忘了原始目标。
- **工具调用没有边界**：Agent 能调的工具越多，跑偏的路径越多。
- **没有检查点**：Agent 一路狂奔，中间没人拦。

避坑方法：

- **目标里写清楚边界**："只研究这只股票本身，不要扩展到竞争对手。"
- **限制工具数量**：只给 Agent 完成任务必需的工具，不要一股脑全给它。
- **设置检查点**：每完成几步，让 Agent 汇报进度，用户确认后再继续。Coze、LangGraph 这些框架都支持"人工审批"节点。
- **设最大步数**：防止 Agent 陷入无限循环（下一章讲 ReAct 时会重点讲这个）。

## 五、悟本质：Agent 是把 LLM 从"嘴"变成"手"

讲了这么多，Agent 的本质可以用一句话概括：**把 LLM 从嘴变成手**。

LLM 本身是一个语言模型——它会说话，会推理，会生成文本。但它没有"手"：它不能查数据库、不能调 API、不能写文件、不能发邮件。它只能"说"。

Agent 给 LLM 装上了手：

- **Planning 是手的大脑指挥**：决定先伸哪只手、再伸哪只手。
- **Tool use 是手本身**：每只手对应一个工具，能干一类活。
- **Memory 是手的记忆**：记住刚才抓过什么、放到哪了。

装上手之后，LLM 就从"会说话的脑子"变成"会做事的人"。

这个本质有几个推论：

**推论 1：Agent 的能力上限，由工具决定，不由 LLM 决定。**

LLM 再聪明，没有工具就是光杆司令。一个接了 100 个高质量工具的普通 LLM，能做的事远多于一个没接工具的最强 LLM。所以搭 Agent 应用，关键不是挑最贵的 LLM，而是配最合适的工具。

**推论 2：Agent 的可靠性，由最弱的环节决定。**

Agent = LLM + Planning + Tool use + Memory。这四个环节任何一个拉胯，整个 Agent 就拉胯。LLM 推理弱，决策就错；Planning 拆得乱，步骤就乱；工具质量差，执行就失败；Memory 不靠谱，上下文就丢。搭 Agent 要均衡提升，不能只优化一个点。

**推论 3：Agent 不是"更强的 LLM"，而是"LLM + 系统"。**

很多人以为 Agent 是 GPT-5、Claude 4 这种"更强的模型"带来的。其实不是。Agent 是一个系统——LLM 只是其中一个组件，外面还包着规划器、工具集、记忆模块、调度逻辑。同样的 LLM，套不同的系统，能力天差地别。

理解这一点，你就明白为什么 OpenAI、Anthropic 都在推"Agent 框架"而不是只推"更强的模型"——因为 Agent 时代的竞争，是系统层面的竞争，不是模型参数的竞争。

## 六、结语

Agent 不是更聪明的聊天机器人，而是给 LLM 装上了规划、工具和记忆，让它从"会说"变成"会做"。

## 七、自测三问

**问题 1：用一句话讲清 Agent 和普通聊天机器人的本质区别。**

参考答案要点：聊天机器人是"输入→输出"的一问一答，只产生语言；Agent 是"目标→规划→执行→反馈→迭代"的循环，能调用工具、产生动作、自主完成多步任务。本质区别是：聊天机器人是"嘴"，Agent 是"手"。

**问题 2：Agent 核心公式里四个组件各自的作用是什么？为什么缺一不可？**

参考答案要点：LLM 是大脑，负责推理决策；Planning 是规划，把大目标拆成子步骤；Tool use 是执行，调用外部工具完成具体动作；Memory 是记忆，记住中间状态和历史信息。缺 LLM 没法推理，缺 Planning 退化成一问一答，缺 Tool use 只会说话不会做事，缺 Memory 做完一步忘上一步。四个组件协同，Agent 才能自主跑起来。

**问题 3：你要搭一个"自动写周报"的 Agent，会怎么配目标、工具和检查点？**

参考答案要点：①目标要具体——"每周五下午 4 点，从我的日历和任务系统拉本周完成的事项，按工作、问题、下周计划三段式写一份 500 字以内的周报，发到我邮箱"；②工具——日历读取 API、任务系统 API、邮件发送 API、文本生成（LLM 本身）；③检查点——生成初稿后先发给我确认，我改完再发送，避免 Agent 自作主张发出去；④设最大步数和工具白名单，防止 Agent 跑偏去查无关信息。
