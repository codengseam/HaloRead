## 一、讲清楚

很多人对"AI 编程"的理解还停留在"打开 ChatGPT，把需求粘进去，把代码复制回来"。这其实是 2023 年的玩法。

到了 2026 年，AI 编程早就不是这么用的。它已经长成了一批专门的工具——AI 编程工具（AI Coding Tools，专门为写代码而设计的 AI 辅助软件）。这些工具不是聊天框，而是直接嵌进你的编辑器、终端，甚至自己就是一个 IDE。

为什么必须用专门工具？因为 ChatGPT 这种聊天框有三个硬伤：

第一，它看不到你的代码库。你问"我这个项目里 UserService 怎么改"，它根本不知道你的 UserService 长什么样，你得把整个文件粘过去。

第二，它改不了你的文件。它给你一段代码，你得手动复制、粘贴、对齐缩进、保存。改一个文件还好，改十个文件就累死。

第三，它没有上下文记忆。你刚问完 A 函数，转头问 B 函数，它已经忘了 A 是什么。

专门的 AI 编程工具就是为了解决这三件事：能读你的代码库、能直接改你的文件、能记住你的项目上下文。

2026 年主流的 AI 编程工具有四大选手：GitHub Copilot、Cursor、Trae、Claude Code。这一章就把这四个工具的定位、强项、价格、适合人群讲清楚，让你能选出最适合自己的那一个。

## 二、看案例

为了让你直观感受四个工具的差异，我们用同一个需求来跑一遍：

> 写一个待办应用（Todo App），要求：能添加任务、标记完成、删除任务、本地存储。

### 用 GitHub Copilot 怎么做

打开 VS Code，新建一个 todo.js 文件，敲下：

```javascript
// 待办应用：支持添加、完成、删除、本地存储
class TodoApp {

```

Copilot 立刻在光标后面灰灰地补全出整个类的骨架：构造函数初始化数组、addItem、toggleComplete、deleteItem、saveToLocalStorage、loadFromLocalStorage。你按 Tab 接受，再敲几个函数名，它继续补全实现。

如果你想问问题，按 Ctrl+I 唤出 Copilot Chat，问"怎么用 localStorage 持久化"，它给你一段代码，你复制粘贴。

特点：补全快、流畅，但需要你主导节奏，文件得自己建、自己组织。

### 用 Cursor 怎么做

打开 Cursor，按 Ctrl+I 唤出 Composer（多文件编辑器），输入：

> 在当前目录建一个待办应用，前端用 HTML+JS，要支持添加、完成、删除、本地存储。生成 index.html、style.css、todo.js 三个文件。

Cursor 的 Composer 会一次性生成三个文件的完整内容，并在右侧显示 diff（差异对比）。你点 Accept All，三个文件就建好了。如果某个函数写得不对，你选中那段代码按 Cmd+K，输入"改成用 Map 存储"，它就地改。

特点：多文件协同、能读整个代码库、Composer 一次出活多。

### 用 Trae 怎么做

打开 Trae，切到 Agent 模式，输入：

> 帮我做一个待办应用，要支持添加、完成、删除、本地存储。

Trae 的 Agent 会自己规划：先建项目结构，再写 HTML，再写 JS，再写 CSS，最后跑一遍看效果。中间它会问你"用 React 还是原生 JS"，你回答"原生"，它继续。整个过程像在跟一个初级程序员交代任务。

特点：Agent 模式自主性强、中文交互流畅、能切换不同模型（Claude、GPT、Gemini）。

### 用 Claude Code 怎么做

打开终端，进入一个空目录，敲：

```bash
claude
```

进入 Claude Code 交互界面，输入：

> 在当前目录建一个待办应用，原生 HTML+JS，支持添加、完成、删除、本地存储。

Claude Code 会自己读目录、建文件、写代码、跑测试。它甚至会自己运行 `node -v` 检查环境、运行 `ls` 看目录结构。整个过程在终端里完成，不打开任何 GUI（图形界面）。

特点：终端原生、长任务自主、适合服务器端和无 GUI 场景。

四个工具，四种风格。下面挨个讲怎么上手。

## 三、上手步骤

### 1. GitHub Copilot：行内补全的祖师爷

GitHub Copilot 是最早成气候的 AI 编程工具，2021 年由 GitHub 和 OpenAI 联合推出，底层模型从早期的 Codex 换到了现在的 GPT-4o、Claude 等。

**定位**：编辑器插件，嵌入 VS Code、JetBrains、Visual Studio、Neovim 等主流编辑器。

**强项**：行内补全（Inline Completion，你写到一半它接着写，按 Tab 接受）。这种"边写边补"的体验最接近"AI 默默帮你打字"。

**安装步骤**（以 VS Code 为例）：

1. 打开 VS Code，左侧扩展面板搜索 "GitHub Copilot"。
2. 点 Install。
3. 重启 VS Code，右下角会提示登录 GitHub 账号。
4. 浏览器跳转 GitHub 授权页，授权完成。
5. 回到 VS Code，新建一个文件开始写代码，灰色的补全建议就会出现，按 Tab 接受。

**价格**（截至 2026 年，据 GitHub 官方定价页）：

- Free 版：每月有限补全次数和 Chat 次数。
- Pro 版：$10/月，无限补全。
- Business 版：$19/月/人，团队管理。
- Enterprise 版：$39/月/人。

**适合人群**：刚接触 AI 编程的新手、已经在用 VS Code/JetBrains 的人、不想换编辑器的人。

### 2. Cursor：AI 原生编辑器

Cursor 是 Anysphere 公司 2023 年推出的 AI 编辑器，基于 VS Code 的开源版本 fork（分叉复制）而来。这意味着 VS Code 的插件几乎都能在 Cursor 里用，迁移成本极低。

**定位**：独立的 AI 编辑器，不是插件。

**强项**：

- **Composer（多文件编辑）**：一次描述需求，它同时改多个文件，给出 diff 让你审。
- **Cmd+K（行内编辑）**：选中代码，按 Cmd+K，输入指令，就地修改。
- **Chat（侧边栏对话）**：能引用整个代码库、特定文件、特定函数作为上下文。
- **Codebase Indexing（代码库索引）**：把你的整个项目向量化，问问题时它能找到相关文件。

**安装步骤**：

1. 访问 cursor.com，下载对应操作系统的安装包。
2. 安装并打开，首次启动会让你登录。
3. 登录后，它会自动检测你的 VS Code 配置和插件，问你要不要导入。
4. 导入完成后，打开一个项目，按 Ctrl+I 试一下 Composer。

**价格**（截至 2026 年，据 Cursor 官方定价页）：

- Hobby（免费）：有限 Composer 次数。
- Pro：$20/月，500 次高级请求/月。
- Business：$40/月/人。

**适合人群**：每天写代码 4 小时以上的重度用户、需要跨多文件改动的开发者、愿意为效率付费的人。

### 3. Trae：字节出品的中文友好 AI IDE

Trae 是字节跳动 2025 年初推出的 AI IDE，同样基于 VS Code fork。它和 Cursor 的形态很像，但有几个明显差异。

**定位**：独立的 AI IDE，国内版叫 Trae（基于豆包模型），国际版叫 Trae International（基于 Claude、GPT 等海外模型）。

**强项**：

- **Agent 模式**：你给一个目标，它自己拆解步骤、建文件、写代码、跑测试。比 Cursor 的 Composer 更"自主"。
- **多模型切换**：在界面里直接切 Claude 3.5 Sonnet、GPT-4o、Gemini 等，不用换工具。
- **中文友好**：界面、文档、社区都是中文，对国内开发者友好。
- **免费**：截至 2026 年中，Trae 国内版完全免费。

**安装步骤**：

1. 访问 trae.cn，下载安装包。
2. 安装并打开，用手机号或字节账号登录。
3. 首次启动会引导你选择模型和主题。
4. 打开一个项目，点右上角的 Agent 模式开关，输入需求试试。

**价格**（截至 2026 年）：

- 国内版：免费。
- 国际版：免费版 + Pro 订阅。

**适合人群**：国内开发者、喜欢中文交互的人、想免费体验 Agent 模式的人、新手。

### 4. Claude Code：终端里的 AI 程序员

Claude Code 是 Anthropic 2025 年推出的命令行 AI 编程工具。它不是 IDE，甚至不是编辑器插件，而是一个跑在终端里的命令行程序。

**定位**：命令行 AI 编程工具，Agentic（自主代理，能自己规划并执行多步任务）。

**强项**：

- **终端原生**：直接在终端里跑，能执行任何 shell 命令、读写任何文件、跑任何脚本。
- **长任务自主**：给它一个大任务（比如"重构这个项目的测试"），它能自己干半小时，中间不需要你盯着。
- **Git 集成**：能读 git log、git diff，能自己提交 commit。
- **可脚本化**：能嵌入 CI/CD 流水线，做自动化。

**安装步骤**：

1. 确保已安装 Node.js 18+。
2. 终端执行：

```bash
npm install -g @anthropic-ai/claude-code
```

3. 安装完成后，进入任意项目目录，执行：

```bash
claude
```

4. 首次运行会引导你登录 Anthropic 账号并授权。
5. 进入交互界面后，直接用自然语言下指令。

**价格**（截至 2026 年，据 Anthropic 官方文档）：

- 按 Claude API 用量计费，没有固定月费。
- 重度使用一天可能几十美元，轻度使用几美元。
- 适合有 API 预算的团队和重度用户。

**适合人群**：命令行党、服务器端开发者、需要长任务自主执行的人、DevOps 和 SRE。

### 5. 四大工具对比表

| 工具 | 形态 | 强项 | 价格（2026） | 适合人群 |
|---|---|---|---|---|
| GitHub Copilot | 编辑器插件 | 行内补全、生态广 | Free / $10/月 | 新手、VS Code 用户 |
| Cursor | 独立 IDE | Composer 多文件、代码库理解 | Free / $20/月 | 重度用户、专业开发者 |
| Trae | 独立 IDE | Agent 模式、多模型、中文 | 免费 | 国内开发者、新手 |
| Claude Code | 命令行 | 终端原生、长任务自主 | 按 API 用量 | 命令行党、DevOps |

### 6. 选型建议

**如果你是新手**：先用 GitHub Copilot 或 Trae。Copilot 装在 VS Code 里，零迁移成本；Trae 免费、中文、Agent 模式上手快。两个都试一周，看哪个顺手。

**如果你是重度用户**（每天写代码 4 小时以上）：上 Cursor。Composer 和代码库索引能省下大量"找文件、改文件"的时间，$20/月对专业开发者是划算的。

**如果你是命令行党**：上 Claude Code。终端原生、长任务自主，特别适合服务器端开发、DevOps、SRE。

**如果你在国内、想要中文体验**：上 Trae。界面、文档、社区都是中文，模型切换方便。

**如果你是团队**：GitHub Copilot Business 或 Cursor Business，有团队管理、用量统计、合规审计。

## 四、验收标准

学完这一章，你应该能做到：

1. **能安装并使用至少一个工具完成"写一个函数"的任务**。比如用 Copilot 在 VS Code 里写一个排序函数，或用 Cursor 的 Composer 写一个工具函数。
2. **能说出四个工具的定位差异**。比如 Copilot 是插件、Cursor 是 IDE、Trae 是 IDE、Claude Code 是命令行。
3. **能根据自己的情况选出合适的工具**。比如新手选 Copilot/Trae，重度用户选 Cursor，命令行党选 Claude Code。

具体的验收动作：

- 打开任一工具，新建一个文件，写一个函数：输入一个数组，返回去重后的数组。
- 用工具的 AI 能力完成（补全或对话都行）。
- 跑通这个函数，确认结果正确。

跑通了，这一章就算过关。

## 五、悟本质

聊完四个工具，往深里想一层：AI 编程工具到底改变了什么？

表面上看，它让你写代码更快。但本质的变化是——**编程这件事，从"写代码"变成了"描述需求 + 审代码"**。

以前的程序员，70% 的时间在敲键盘写代码，30% 的时间在想需求。现在的程序员，这个比例正在反过来：70% 的时间在想需求、描述需求、审 AI 写的代码，30% 的时间在亲手敲关键代码。

这意味着什么？

意味着程序员的核心能力在迁移。**会敲代码**这件事的价值在下降，因为 AI 敲得比你快、比你准、还不累。**会描述需求**（把模糊的想法变成清晰的指令）和**会审代码**（一眼看出 AI 写的代码哪里有问题）的价值在上升。

这跟摄影行业的变迁很像。数码相机和手机摄像头普及后，"会调光圈快门"不再是摄影师的核心能力，"会构图、会讲故事、会选片"才是。AI 编程工具就是程序员行业的"手机摄像头"——它把"写代码"这件事民主化了，但"判断什么代码是好代码"这件事，反而更稀缺了。

所以选哪个工具不是最关键的。最关键的是：**不管用哪个工具，都要把"描述需求"和"审代码"这两件事练熟**。这两件事，是下一章的主题。

## 六、结语

工具会迭代，模型会升级，但"会要、会审、会改"这三件事，是 AI 时代程序员的护城河。
