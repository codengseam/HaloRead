# 算法课·12｜动态规划：从记忆化到状态定义

> 前置知识：学完第 04 章（递归），尤其要懂"递归是函数调用自己、每层有独立栈帧"
> 学完你能：①给纯小白讲清动态规划为什么能把 $O(2^n)$ 优化到 $O(n)$ ②默写 0-1 背包、LCS、股票买卖三个模板并讲透每一行 ③理解状态定义的"维度升降"——什么时候该加维度、什么时候能降维度 ④用对数器对比暴力递归与 DP 的正确性 + 性能 ⑤识别区间 DP、树形 DP、状态机 DP 三类高级形态

## 一、问题与思路：从"重复算"到"记住答案"

### 1. 一个生活场景：爬楼梯有多少种走法

假设你住 10 楼，每次只能走 1 级或 2 级台阶，问从 1 楼到 10 楼有多少种走法？

你可能会硬想：第一步走 1 级，剩下 9 级再想；第一步走 2 级，剩下 8 级再想。所以"到 10 楼的走法数 = 到 9 楼的走法数 + 到 8 楼的走法数"。

这就是斐波那契！设 $f(n)$ 是到 $n$ 楼的走法数，则 $f(n) = f(n-1) + f(n-2)$，$f(1)=1, f(2)=2$。

**但这里藏着一个致命问题**：如果你直接照着这个递推式写递归——`f(10) = f(9) + f(8)`，然后 `f(9) = f(8) + f(7)`，`f(8)` 被算了两次！再往下，`f(7)` 被算三次，`f(6)` 被算五次……整个计算树呈指数级膨胀。$f(50)$ 在普通机器上要算好几万年。

**人类会怎么做？** 拿张纸，从 $f(1)=1, f(2)=2$ 开始，一格一格往下填：$f(3)=3, f(4)=5, f(5)=8 \dots$ 填到 $f(10)$ 只要 10 步。**每一步都用前面已经算过的答案，绝不重复算第二遍**。

这就是动态规划（Dynamic Programming，简称 DP）的核心思想：**发现重复子问题，把子问题的答案记下来，避免重复计算**。

### 2. 再看一个场景：背包装东西最多值多少

你有个容量 4kg 的背包，面前三件商品：A（1kg，价值 15）、B（3kg，价值 20）、C（2kg，价值 10）。怎么选让总价值最大？

穷举所有子集当然能做——但 $n$ 件商品有 $2^n$ 个子集，20 件就 100 万，30 件就 10 亿。

仔细想："考虑前 $i$ 件商品、背包剩余容量 $j$ 时能拿的最大价值"——这个状态只依赖"考虑前 $i-1$ 件商品"的更小问题。如果第 $i$ 件不拿，答案等于"前 $i-1$ 件、容量 $j$"的答案；如果拿，答案等于"前 $i-1$ 件、容量 $j - w_i$"的答案加上 $v_i$。两者取较大值。

**这又是"大问题拆成子问题、子问题被反复问到"的模式**。DP 不是新东西，它是"递归 + 记忆"的工程化版本。

### 3. DP 成立的两个判据

不是所有问题都能用 DP。Bellman 在 1950 年代提出这个方法时，明确给出了两个必要条件：

**判据一：最优子结构（Optimal Substructure）。** 问题的最优解，包含子问题的最优解。背包问题里"前 $i$ 件、容量 $j$"的最优解，要么来自"前 $i-1$ 件、容量 $j$"的最优解，要么来自"前 $i-1$ 件、容量 $j-w_i$"的最优解加上 $v_i$——两个子问题的解都是最优的，才能拼出全局最优。

反例：求"最长简单路径"（不重复经过顶点的最长路）就没有最优子结构。从 A 到 C 的最长路如果经过 B，A 到 B 的子段未必是 A 到 B 的最长路（因为最长路可能用了 B 后面的点，破坏"不重复"约束）。这种问题 DP 不直接适用。

**判据二：重叠子问题（Overlapping Subproblems）。** 子问题会被反复问到。斐波那契里 $f(8)$ 被 $f(10)$ 和 $f(9)$ 都问了一次——重叠了。背包问题里"前 $i-1$ 件、容量 $j'$"会被多个状态问。

反例：归并排序的子问题不重叠——左半边排完就丢、右半边排完就丢，每个子区间只处理一次。归并是分治，不是 DP。

**核心思想**：DP 的本质突破在于——**把"递归发现的重叠子问题"从"重复算"变成"算一次记一辈子"**。这不是发明新算法，是把递归的"自顶向下"翻转为"自底向上"，把指数级的重复计算压成多项式。理解这一点，你就理解了 DP 为什么"快"——不是因为它更聪明，是因为它拒绝做重复劳动。

### 4. 历史地位：Bellman 与"动态规划"这个名字

1954 年，Richard Bellman 在 *Bulletin of the American Mathematical Society* 发表《The Theory of Dynamic Programming》，系统提出这个方法。他在自传《Eye of the Hurricane》里有个著名吐槽：当年他在兰德公司做研究，老板讨厌数学，"research"这个词都犯忌讳。他需要一个听起来不像研究的名字糊弄审批，于是挑了 "dynamic"（动态、有冲击力）+ "programming"（规划，线性规划里已用），组合成 "dynamic programming"——既好听又查不出是数学研究。

名字是糊弄人的，方法是真东西。Bellman 当时研究的是多阶段决策优化（导弹拦截、库存管理），发现这些问题都有"阶段 + 状态 + 决策"的结构。他把"状态"抽象出来，提出"最优性原理"：**一个最优策略的子策略，对子问题而言也必然最优**——这就是最优子结构的最初表述。

DP 后来成为算法领域最庞大的家族之一：背包、最长子序列、编辑距离、股票买卖、区间合并、树形依赖……几乎所有"求最优/计数/存在性"的题目，先想 DP。CLRS [1] 用整整 4 章（15-18 章）讲 DP，比排序还多。

## 二、原理与实现：从暴力递归到状态转移方程

### 1. 三阶段演进：用斐波那契看透 DP 的诞生

这是 DP 入门最关键的一节。**不理解这三阶段的演进，背再多模板都是空中楼阁**。我们用 $f(n) = f(n-1) + f(n-2)$，$f(0)=0, f(1)=1$ 这个最简单的递推，看 DP 是怎么一步步被"逼"出来的。

**阶段一：暴力递归（自顶向下，$O(2^n)$）**

```python
def fib_violent(n):
    if n == 0:                    # base case 1
        return 0
    if n == 1:                    # base case 2
        return 1
    return fib_violent(n-1) + fib_violent(n-2)   # 大问题拆成两个子问题
```

**逐行讲透**：

- **`if n == 0: return 0`**：最小子问题，直接给答案。递归必须有 base case，否则无限调用栈溢出。
- **`if n == 1: return 1`**：第二个 base case。斐波那契需要两个初始值，因为递推依赖前两项。
- **`return fib(n-1) + fib(n-2)`**：大问题拆成两个子问题相加。这正是递推定义的直接翻译。

**复杂度**：$O(2^n)$。画调用树：$f(5)$ 调 $f(4)$ 和 $f(3)$，$f(4)$ 调 $f(3)$ 和 $f(2)$，$f(3)$ 被调了两次……节点数随 $n$ 指数膨胀。$f(50)$ 大约 $2^{50} \approx 10^{15}$ 次调用，普通机器跑几万年。

**根因**：$f(3)$ 这种小问题被反复算了。算 $f(5)$ 时算了一次 $f(3)$，算 $f(4)$ 时又算了一次 $f(3)$，而两次算出来的答案完全一样——重复劳动。

**阶段二：记忆化递归（自顶向下 + 缓存，$O(n)$）**

既然 $f(3)$ 被反复算，且答案不变，那就第一次算完存起来，下次直接取。

```python
memo = {}                          # 全局缓存

def fib_memo(n):
    if n == 0:
        return 0
    if n == 1:
        return 1
    if n in memo:                  # 算过就直接返回，不再递归
        return memo[n]
    memo[n] = fib_memo(n-1) + fib_memo(n-2)   # 算完存进缓存
    return memo[n]
```

**逐行讲透**：

- **`memo = {}`**：缓存字典，键是 $n$，值是 $f(n)$。
- **`if n in memo: return memo[n]`**：核心一行。算过的子问题直接取，不再递归。这一行把指数级变成线性。
- **`memo[n] = fib(n-1) + fib(n-2)`**：算完立刻存。注意先算右边的递归、再赋值——存的是已经算出的结果。

**复杂度**：$O(n)$。每个 $f(k)$ 只算一次（第一次问的时候算，之后都从缓存取），共 $n$ 个不同状态。空间 $O(n)$（缓存 + 递归栈）。

**这是 DP 的雏形**：思路是"自顶向下"——从 $f(n)$ 出发往下问，问到 $f(0)$ 才有答案，但每个问题只算一次。记忆化递归等价于 DP，只是写法不同。很多人（包括左神）面试时优先写记忆化，因为它和暴力递归只差一个缓存，不容易写错。

**阶段三：动态规划（自底向上，$O(n)$，可压到 $O(1)$）**

记忆化是"从大问小"，那能不能反过来"从小推大"？能。既然 $f(0), f(1)$ 已知，$f(2) = f(1)+f(0)$，$f(3) = f(2)+f(1)$……一路推到 $f(n)$。

```python
def fib_dp(n):
    if n == 0:
        return 0
    if n == 1:
        return 1
    dp = [0] * (n + 1)             # dp[k] 存 f(k)
    dp[0] = 0                      # 填 base case
    dp[1] = 1
    for k in range(2, n + 1):      # 从小到大填表
        dp[k] = dp[k-1] + dp[k-2]  # 状态转移方程
    return dp[n]
```

**逐行讲透**：

- **`dp = [0] * (n + 1)`**：建一张表，下标 $k$ 对应状态 $f(k)$。DP 的核心动作就是"填表"。
- **`dp[0] = 0; dp[1] = 1`**：填 base case。表的"起点"必须有值，后面的格子才能算。
- **`for k in range(2, n + 1)`**：从小到大遍历。**遍历顺序极其重要**——必须保证算 $dp[k]$ 时它依赖的 $dp[k-1], dp[k-2]$ 已经算好。这是 DP 易错点之一。
- **`dp[k] = dp[k-1] + dp[k-2]`**：状态转移方程。这是 DP 的"灵魂一行"——把"大问题怎么由子问题组成"用一行写出来。

**复杂度**：$O(n)$ 时间，$O(n)$ 空间。比记忆化省了递归栈开销（不会栈溢出），常数因子更小。

**还能优化吗？能——状态压缩**。注意 $dp[k]$ 只依赖 $dp[k-1]$ 和 $dp[k-2]$，更早的格子（$dp[k-3], dp[k-4] \dots$）再也不会被用到。那何必存整张表？两个变量滚动就够了：

```python
def fib_dp_opt(n):
    if n == 0:
        return 0
    prev2, prev1 = 0, 1            # prev2 = f(k-2), prev1 = f(k-1)
    for k in range(2, n + 1):
        cur = prev1 + prev2        # f(k) = f(k-1) + f(k-2)
        prev2, prev1 = prev1, cur  # 滚动：旧 f(k-1) 变新 f(k-2)，新 f(k) 变新 f(k-1)
    return prev1
```

**复杂度**：$O(n)$ 时间，$O(1)$ 空间。这就是"状态压缩"——当 $dp[k]$ 只依赖有限几个前驱状态时，用滚动变量代替整张表。**这是 DP 高手和菜鸟的分水岭**：菜鸟会填表，高手会压缩。

**三阶段总结**：

| 阶段 | 写法 | 时间 | 空间 | 思路方向 |
|---|---|---|---|---|
| 暴力递归 | 直接翻译递推 | $O(2^n)$ | $O(n)$ 栈 | 自顶向下 |
| 记忆化递归 | 递归 + 缓存 | $O(n)$ | $O(n)$ | 自顶向下 |
| 动态规划 | 自底向上填表 | $O(n)$ | $O(n)$→$O(1)$ | 自底向上 |

**DP 的诞生路径就是这条线**：暴力递归发现重复 → 加缓存记忆化 → 翻转为自底向上填表 → 发现依赖有限前驱就压缩空间。理解这条线，你拿到任何 DP 题，都能从"写暴力递归"起步，一步步优化到 DP。

### 2. 状态定义与状态转移方程：用 0-1 背包讲透

DP 题最难的不是写代码，是**定义状态**和**写转移方程**。我们用 0-1 背包（每件商品要么拿、要么不拿，最多拿一次）讲透。

**问题**：$n$ 件商品，第 $i$ 件重量 $w[i]$、价值 $v[i]$，背包容量 $W$。求能拿的最大总价值。

**第一步：定义状态**（这是 DP 最考验功力的一步）。

设 $dp[i][j]$ = "考虑前 $i$ 件商品（下标 $0..i-1$），背包剩余容量为 $j$ 时，能拿的最大价值"。

**为什么这样定义？** 因为"前 $i$ 件、容量 $j$"这个状态，恰好能拆成"前 $i-1$ 件"的更小状态——这是最优子结构的体现。两个维度分别刻画"商品进度"和"剩余容量"，缺一不可。

**第二步：写状态转移方程**。

对第 $i$ 件商品（重量 $w[i-1]$，价值 $v[i-1]$，注意下标偏移），只有两种选择：

- **不拿**：最大价值 = $dp[i-1][j]$（前 $i-1$ 件、容量 $j$，第 $i$ 件没影响）
- **拿**（前提 $j \geq w[i-1]$）：最大价值 = $dp[i-1][j - w[i-1]] + v[i-1]$（前 $i-1$ 件、容量减去这件的重量，加上这件的价值）

两者取较大：

$$dp[i][j] = \max\bigl(dp[i-1][j],\;\; dp[i-1][j - w[i-1]] + v[i-1]\bigr) \quad (j \geq w[i-1])$$

当 $j < w[i-1]$ 时只能不拿：$dp[i][j] = dp[i-1][j]$。

**第三步：确定 base case 和遍历顺序**。

- base case：$dp[0][j] = 0$（前 0 件商品，价值当然是 0）；$dp[i][0] = 0$（容量 0，什么都拿不了）
- 遍历顺序：$i$ 从 1 到 $n$，$j$ 从 0 到 $W$。算 $dp[i][j]$ 时它依赖 $dp[i-1][\dots]$，所以 $i$ 必须从小到大。

**第四步：写代码**。

```python
def knapsack01(w, v, W):
    n = len(w)
    dp = [[0] * (W + 1) for _ in range(n + 1)]   # (n+1) x (W+1) 的表
    for i in range(1, n + 1):                     # 遍历每件商品
        for j in range(W + 1):                    # 遍历每个容量
            dp[i][j] = dp[i-1][j]                 # 不拿第 i 件
            if j >= w[i-1]:                       # 容量够才考虑拿
                dp[i][j] = max(dp[i][j], dp[i-1][j - w[i-1]] + v[i-1])
    return dp[n][W]
```

**逐行讲透**：

- **`dp = [[0] * (W + 1) for _ in range(n + 1)]`**：建 $(n+1) \times (W+1)$ 二维表。`dp[0][j]` 表示"前 0 件"，全 0（base case 自动满足）。
- **`for i in range(1, n + 1)`**：从第 1 件开始（第 0 件是 base case）。注意 `w[i-1]` 是第 $i$ 件商品的重量——这种下标偏移是写 DP 最容易错的地方。
- **`for j in range(W + 1)`**：遍历所有容量。$j$ 从 0 到 $W$ 都要算，因为后面 $dp[i][j']$（$j' > j$）可能依赖 $dp[i-1][j]$。
- **`dp[i][j] = dp[i-1][j]`**：先默认"不拿"，这是合法的基础选择。
- **`if j >= w[i-1]`**：容量够才考虑"拿"。容量不够时跳过，保持"不拿"的值。
- **`dp[i][j] = max(dp[i][j], dp[i-1][j - w[i-1]] + v[i-1])`**：状态转移方程的直接翻译。

**复杂度**：$O(nW)$ 时间，$O(nW)$ 空间。

**状态压缩**：注意 $dp[i][\dots]$ 只依赖 $dp[i-1][\dots]$——前一行。那只要一维数组，从后往前更新就行：

```python
def knapsack01_opt(w, v, W):
    n = len(w)
    dp = [0] * (W + 1)             # 一维滚动数组
    for i in range(n):
        for j in range(W, w[i] - 1, -1):   # 从大到小遍历！
            dp[j] = max(dp[j], dp[j - w[i]] + v[i])
    return dp[W]
```

**关键陷阱：内层 $j$ 必须从大到小！** 为什么？因为"拿第 $i$ 件"要用 $dp[i-1][j-w[i]]$——即"上一轮"的值。如果 $j$ 从小到大更新，算 $dp[j]$ 时 $dp[j-w[i]]$ 已经被这一轮更新过了，相当于"第 $i$ 件被拿了多次"，这其实是完全背包（每件可拿无限次）的方程。**0-1 背包要降维，必须倒序遍历**——这是面试高频踩坑点。

### 3. 状态定义的"维度升降"：高手的核心技巧

DP 题做到一定量，你会发现真正的难点不在写方程，而在"状态怎么定义"。定义得好，方程自然就出来了；定义得不好，怎么写都不对。这里讲两个高阶技巧：**加维度**和**降维度**。

**何时加维度**：当"当前状态"不足以推出下一步时，加维度补足信息。

经典例子：股票买卖。问题："每天可买可卖，最多交易 $k$ 次，求最大利润"。如果你只定义 $dp[i]$ = "第 $i$ 天的最大利润"，根本写不出方程——因为你不知道第 $i$ 天结束时手里有没有股票，没股票才能买、有股票才能卖。所以必须加一个维度记录"是否持股"：

$$dp[i][j][s], \quad s \in \{0=\text{不持}, 1=\text{持有}\}$$

其中 $j$ 是"已交易次数"。三个维度分别刻画"第几天""交易了几次""当前是否持股"，缺一个都无法描述状态。

**何时降维度**：当某一维只依赖有限前驱时，用滚动数组压掉。

斐波那契里 $dp[k]$ 只依赖 $dp[k-1], dp[k-2]$——三个变量滚动，$O(n)$ 空间压到 $O(1)$。背包里 $dp[i][j]$ 只依赖 $dp[i-1][\dots]$——压掉 $i$ 维，一维数组滚动。

**降维的本质**：观察到"未来不依赖很久以前的过去"。$f(k)$ 不依赖 $f(k-3)$，那 $f(k-3)$ 就可以扔掉。这是 DP 空间优化的统一思想。

**一个反例：什么时候不能降维**。区间 DP（如石子合并）的 $dp[i][j]$ 表示"区间 $[i,j]$ 的最优解"，它依赖 $dp[i][k]$ 和 $dp[k+1][j]$（所有 $k$）——依赖的是"中间任意位置"，不是固定前驱。这种就很难压缩，老老实实留 $O(n^2)$ 表。

### 4. DP 五大模型概览

DP 题目千变万化，但 90% 可归入五类模型。理解这五个，面试大部分 DP 题都有思路。

| 模型 | 代表题 | 状态定义 | 复杂度 | 特征 |
|---|---|---|---|---|
| 背包 | 0-1/完全/分组背包 | $dp[i][j]$=前 $i$ 件容量 $j$ 的最优 | $O(nW)$ | "选/不选" + 容量约束 |
| 线性 DP | 打家劫舍、爬楼梯 | $dp[i]$=到第 $i$ 个位置的最优 | $O(n)$ | 一维顺序推进 |
| 双串 DP | LCS、编辑距离 | $dp[i][j]$=两串前 $i$、前 $j$ 的最优 | $O(nm)$ | 两个序列配对 |
| 状态机 DP | 股票买卖 | $dp[i][s]$=第 $i$ 天状态 $s$ 的最优 | $O(n \cdot \|S\|)$ | 离散状态间转移 |
| 区间 DP | 石子合并、最长回文 | $dp[i][j]$=区间 $[i,j]$ 的最优 | $O(n^3)$ | 区间分裂 |

后面三段会逐一展开背包、LCS、股票买卖的模板，区间 DP 和树形 DP 在高手向小节讲。

## 三、实践与面试：手撕模板、对数器、面试题

### 1. 面试手撕模板一：0-1 背包（按"技术的水有多深"八层讲）

```python
def knapsack01(w, v, W):
    """0-1 背包：每件商品最多拿一次，求容量 W 下的最大价值。"""
    n = len(w)
    dp = [0] * (W + 1)                          # 一维滚动数组
    for i in range(n):                          # 遍历每件商品
        for j in range(W, w[i] - 1, -1):        # 容量从大到小（关键！）
            dp[j] = max(dp[j], dp[j - w[i]] + v[i])
    return dp[W]
```

**面试时怎么讲**（按"技术的水有多深"八层，挑重点讲）：

1. **问题层**（核心思想）：每件商品"拿或不拿"，容量有限，求最大价值。本质是"在约束下做最优决策"。
2. **原理层**：定义 $dp[i][j]$=前 $i$ 件、容量 $j$ 的最大价值。方程：$dp[i][j] = \max(dp[i-1][j],\; dp[i-1][j-w_i] + v_i)$。"不拿"和"拿"取较大。
3. **优劣层**：$O(nW)$ 时间、$O(W)$ 空间（压缩后）。$W$ 大时退化——这是"伪多项式"复杂度。
4. **演进层**：暴力穷举 $O(2^n)$ → 二维 DP $O(nW)$ 空间 $O(nW)$ → 一维滚动 $O(nW)$ 时间 $O(W)$ 空间。压缩靠"倒序遍历"保住"上一轮的值"。
5. **陷阱层**：①倒序遍历（否则变完全背包）②下标偏移 `w[i]` vs `w[i-1]` ③容量为 0 的边界。
6. **变体层**：完全背包（每件无限拿）正序遍历；分组背包（每组选一件）多一层组循环；求"恰好装满"改 base case。
7. **工程层**：$W$ 上百万时 $O(nW)$ 不可行，要换思路（贪心 + 逼近、分支限界）。背包是 NP-hard，DP 只是伪多项式解法。
8. **迁移层**：所有"选/不选 + 容量/预算约束"题都是背包变体——分割等和子集、零钱兑换、单词拆分。

**面试加分点**：主动提"倒序遍历是为了保住上一轮值、正序就变完全背包"——这一句话能区分你是否真懂。

### 2. 面试手撕模板二：最长公共子序列 LCS

```python
def lcs(s1, s2):
    """最长公共子序列长度。dp[i][j] = s1 前 i 个字符与 s2 前 j 个字符的 LCS 长度。"""
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:              # 两字符相等：都纳入 LCS
                dp[i][j] = dp[i-1][j-1] + 1
            else:                                # 不等：退一格取较大
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

**逐行讲透**：

- **`dp = [[0] * (n + 1) for _ in range(m + 1)]`**：$(m+1)\times(n+1)$ 表。`dp[0][j]` 和 `dp[i][0]` 全 0（空串与任何串的 LCS 为 0），自动满足 base case。
- **`if s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] + 1`**：两字符相等，最优策略是"都纳入 LCS"，长度 = 两边都退一格的 LCS + 1。这一行是 LCS 的"灵魂"——为什么不是 `max(dp[i-1][j], dp[i][j-1]) + 1`？因为相等字符配对一定不亏（不配对的话这个字符两边都用不上），配对后剩下的就是 $dp[i-1][j-1]$。
- **`else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])`**：两字符不等，s1 退一格或 s2 退一格，取较大。

**复杂度**：$O(mn)$ 时间和空间。空间可压缩到 $O(\min(m,n))$（因为 $dp[i][j]$ 只依赖 $dp[i-1][j-1], dp[i-1][j], dp[i][j-1]$ 三个，用两行滚动 + 一个变量记对角线）。

**面试官追问"怎么求出 LCS 字符串本身"**：从 $dp[m][n]$ 反向回溯。如果 $s1[i-1]==s2[j-1]$，这个字符是 LCS 的一员，加入结果，$i,j$ 都退一格；否则往 $dp$ 值较大的方向退。

### 3. 面试手撕模板三：股票买卖（状态机 DP）

**问题**：股票价格数组 `prices`，每天可买可卖（持有期间不能再买），求最大利润。

```python
def max_profit(prices):
    """买卖一次的最大利润。状态机：每天有"不持股"或"持股"两种状态。"""
    if not prices:
        return 0
    # dp_hold = 今天结束时持股的最大利润；dp_cash = 今天结束时不持股的最大利润
    dp_cash = 0                    # 第 0 天不持股，利润 0
    dp_hold = -prices[0]           # 第 0 天持股 = 当天买入，利润 -prices[0]
    for i in range(1, len(prices)):
        # 今天不持股 = 昨天就不持股 / 今天卖出（昨天持股 + 今天价格）
        dp_cash = max(dp_cash, dp_hold + prices[i])
        # 今天持股 = 昨天就持股 / 今天买入（昨天不持股 - 今天价格）
        dp_hold = max(dp_hold, dp_cash - prices[i])
    return dp_cash                 # 最后一天必然不持股（持股不如卖掉）
```

**逐行讲透**：

- **`dp_cash = 0`**：第 0 天不持股，没买也没卖，利润 0。
- **`dp_hold = -prices[0]`**：第 0 天持股，意味着第 0 天买入，利润是负的买入价。
- **`dp_cash = max(dp_cash, dp_hold + prices[i])`**：今天不持股的两种来源——昨天就不持股（啥也不干）、或昨天持股今天卖（持股利润 + 卖出价）。
- **`dp_hold = max(dp_hold, dp_cash - prices[i])`**：今天持股的两种来源——昨天就持股（啥也不干）、或昨天不持股今天买（不持股利润 - 买入价）。

**这里有个隐藏陷阱**：上面这版 `dp_cash` 和 `dp_hold` 在同一天内更新，`dp_hold` 用了更新后的 `dp_cash`——这其实允许"同一天买入又卖出"。对于"买卖一次"无影响（同天买卖利润 0），但对于"含冷静期""含手续费"的变体就要用临时变量保存上一轮值。

**为什么这是"状态机 DP"**：把每天的"持股状态"建模成状态机的节点，"买/卖/不动"是状态间的转移边。每天就是状态机走一步。状态机 DP 的通用套路：**把离散状态做成维度，转移就是状态机走边**。

**变体扩展**（同框架改状态）：

- 最多交易 $k$ 次：加维度 $j$=已交易次数，$dp[i][j][s]$
- 含冷静期（卖后第二天才能买）：加状态"冷冻期"
- 含手续费：卖出时减手续费
- 无限次交易：去掉交易次数维度，就是上面的模板

### 4. 对数器：暴力递归 vs DP（左神风格）

DP 写完怎么验证对？**对数器**——写一个绝对正确的暴力解（穷举 / 朴素递归），和 DP 解对比，跑 5000 次随机数据，有一次不一致就说明 DP 有 bug。

下面用 0-1 背包演示：

```python
import random

def knapsack01_violent(w, v, W, i=0):
    """暴力递归：第 i 件选或不选，穷举所有可能。绝对正确但 O(2^n)。"""
    if i == len(w):
        return 0
    # 不拿第 i 件
    best = knapsack01_violent(w, v, W, i + 1)
    # 拿第 i 件（容量够才拿）
    if W >= w[i]:
        best = max(best, v[i] + knapsack01_violent(w, v, W - w[i], i + 1))
    return best

def knapsack01_dp(w, v, W):
    """DP 解：待验证。"""
    n = len(w)
    dp = [0] * (W + 1)
    for i in range(n):
        for j in range(W, w[i] - 1, -1):
            dp[j] = max(dp[j], dp[j - w[i]] + v[i])
    return dp[W]

def checker(test_times=5000, max_n=8, max_w=10, max_v=20):
    """对数器：随机数据对比暴力解和 DP 解。"""
    for _ in range(test_times):
        n = random.randint(0, max_n)
        w = [random.randint(1, max_w) for _ in range(n)]
        v = [random.randint(1, max_v) for _ in range(n)]
        W = random.randint(0, max_w * 2)
        ans_v = knapsack01_violent(w, v, W)
        ans_d = knapsack01_dp(w, v, W)
        if ans_v != ans_d:
            print(f"出错！w={w}, v={v}, W={W}, 暴力={ans_v}, DP={ans_d}")
            return False
    print("对数器验证通过！DP 解与暴力解 5000 次随机数据全部一致。")
    return True

checker()
```

**为什么 `max_n=8`**：暴力解 $O(2^n)$，$n=8$ 时 $2^8=256$ 次调用，跑 5000 轮也就百万级，几秒能完。如果 $n=20$，$2^{20} \approx 10^6$，5000 轮要 $5 \times 10^9$，太慢。**对数器的暴力解要够小够快**，否则验证本身就成了瓶颈。

**对数器的价值**：

- **覆盖边界**：随机 $n$ 包含 0（空背包）、$W$ 包含 0（容量为 0）、$w[i]$ 包含 1 和上限——覆盖手写时容易遗漏的边界。
- **不依赖人工**：跑 5000 次比手动构造测试全面得多，能抓出"特定下标偏移错""倒序遍历写成正序""base case 漏写"等隐蔽 bug。
- **左神原话**："DP 写完先写暴力解对数器，别直接交。暴力解是最朴素的穷举，最不容易错；DP 是优化版，最容易在状态转移、下标、遍历顺序上出错。"

**对高手的启发**：对数器思想不止验证 DP——任何"有朴素正确解可对照"的优化都能用。写完记忆化？对数器对比暴力递归。写完状态压缩？对数器对比未压缩版。这是"工程化验证"思维，比"理论上推导正确"更可靠。

### 5. 教科书做法 vs 生产做法

| 场景 | 教科书做法 | 生产做法 | 原因 |
|---|---|---|---|
| 小规模背包（$nW < 10^7$） | 二维 DP | 一维滚动 DP | 空间优化，常数因子小 |
| 大规模背包（$W > 10^6$） | DP | 贪心 + 分支限界 / 启发式 | DP 是 $O(nW)$ 伪多项式，$W$ 大就崩 |
| 实时决策（库存调度） | 离线 DP | 在线强化学习 | 环境动态变化，DP 需要已知模型 |
| 状态空间爆炸 | 普通 DP | 记忆化 + 剪枝 / 值函数逼近 | 状态太多存不下 |

**生产里 DP 的真实场景**：

1. **推荐系统的序列决策**：用户浏览序列建模，状态=用户历史，DP 求最优推荐策略。
2. **自然语言处理的分词/对齐**：维特比算法本质是 DP（隐马尔可夫模型的最可能状态序列）。
3. **生物信息学的序列比对**：编辑距离（下文讲）的 DP 是 BLAST 等工具的基础。
4. **运筹优化的路径规划**：TSP 的位掩码 DP（下文讲）是小规模 TSP 的精确解法。

### 6. 三个真实工程坑

**坑一：把 0-1 背包的倒序写成正序，变成完全背包。**

```python
# 错误：正序遍历，第 i 件可能被"拿多次"
for i in range(n):
    for j in range(w[i], W + 1):          # 正序！
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])
```

症状：算出的价值比正确值大。根因：正序遍历时算 $dp[j]$ 用到的 $dp[j-w[i]]$ 已是这一轮更新过的值，相当于"第 $i$ 件被拿了多次"。修复：0-1 背包倒序，完全背包正序——记住"0-1 倒序、完全正序"。

**坑二：遍历顺序错了，依赖了还没算的状态。**

```python
# 错误：LCS 的 j 从 0 开始，dp[i][j-1] 还没算
for i in range(1, m + 1):
    for j in range(0, n + 1):              # j 从 0 开始！
        ...
```

症状：数组越界或答案错。根因：算 $dp[i][j]$ 依赖 $dp[i][j-1]$，$j$ 从 0 开始时 $dp[i][-1]$ 越界。修复：遍历顺序必须保证"依赖的状态先算"。一维 DP 一般从左到右；二维 DP 一般两层都从小到大；状态压缩时要分析依赖方向。

**坑三：状态定义不够，写不出正确方程。**

经典反例：股票买卖含冷静期（卖后第二天才能买）。如果只定义 $dp[i]$=第 $i$ 天最大利润，根本写不出——因为"今天能不能买"取决于"昨天有没有卖"，而"昨天有没有卖"信息丢失了。修复：加状态维度，$dp[i][s]$，$s \in \{\text{持股}, \text{不持股可买}, \text{不持股冷冻}\}$。**状态定义不足是 DP 题最常见的卡壳点**——遇到写不出方程时，第一反应应该是"是不是漏了状态维度"。

| 题目 | 模型 | 状态定义 | 复杂度 | 难度 |

| 打家劫舍 | 线性 DP | $dp[i]$=前 $i$ 家的最大收益 | $O(n)$ | 中等 |
| 0-1 背包 | 背包 | $dp[i][j]$=前 $i$ 件容量 $j$ 的最大价值 | $O(nW)$ | 中等 |
| 零钱兑换 | 完全背包 | $dp[j]$=凑出金额 $j$ 的最少硬币数 | $O(nW)$ | 中等 |
| 最长公共子序列 | 双串 DP | $dp[i][j]$=两串前 $i$、前 $j$ 的 LCS | $O(mn)$ | 中等 |
| 编辑距离 | 双串 DP | $dp[i][j]$=两串前 $i$、前 $j$ 的最少操作 | $O(mn)$ | 中等 |
| 最长递增子序列 | 线性 DP | $dp[i]$=以 $i$ 结尾的 LIS 长度 | $O(n^2)$/ $O(n\log n)$ | 中等 |
| 买卖股票最佳时机 | 状态机 DP | $dp[i][s]$=第 $i$ 天状态 $s$ 的最大利润 | $O(n\cdot\|S\|)$ | 中等 |
| 打家劫舍 III | 树形 DP | 后序遍历返回(偷/不偷)两值 | $O(n)$ | 中等 |
| 最长回文子串 | 区间 DP | $dp[i][j]$=区间 $[i,j]$ 是否回文 | $O(n^2)$ | 中等 |
| 戳气球 | 区间 DP | $dp[i][j]$=区间 $(i,j)$ 内戳完的最大收益 | $O(n^3)$ | 困难 |

**面试题**："手写 0-1 背包，说明复杂度，并讲解如何压缩空间。"

**满分回答模板**（边写边讲，要点）：

1. "我定义 $dp[i][j]$=前 $i$ 件商品、容量 $j$ 的最大价值。方程：$dp[i][j] = \max(dp[i-1][j], dp[i-1][j-w_i] + v_i)$，前者是不拿，后者是拿（容量够时）。"
2. "二维版 $O(nW)$ 时间、$O(nW)$ 空间。观察到 $dp[i]$ 只依赖 $dp[i-1]$，压成一维滚动数组，$O(nW)$ 时间、$O(W)$ 空间。"
3. "压缩的关键：内层 $j$ 必须从大到小遍历，因为'拿第 $i$ 件'要用上一轮的 $dp[j-w_i]$。正序遍历会把 $dp[j-w_i]$ 更新成这一轮的值，相当于第 $i$ 件被拿了多次——那就变成完全背包了。"
4. "$O(nW)$ 是伪多项式复杂度（$W$ 是数值不是规模），背包问题是 NP-hard，DP 只在 $W$ 不大时可行。"

**面试官追问"完全背包怎么改"**：把内层倒序改正序——正序遍历允许"同一件被拿多次"，正好是完全背包的语义。

## 四、速查与自测

### 速查表：DP 三阶段与五大模型

**三阶段演进**（用斐波那契演示）：

| 阶段 | 写法 | 时间 | 空间 | 关键动作 |
|---|---|---|---|---|
| 暴力递归 | 直接翻译递推 | $O(2^n)$ | $O(n)$ 栈 | 大问题拆子问题 |
| 记忆化递归 | 递归 + 缓存 | $O(n)$ | $O(n)$ | 子问题算一次记一辈子 |
| 动态规划 | 自底向上填表 | $O(n)$ | $O(n)$→$O(1)$ | 按依赖顺序填表 + 滚动压缩 |

**DP 成立两判据**：

| 判据 | 含义 | 反例 |
|---|---|---|
| 最优子结构 | 问题最优解含子问题最优解 | 最长简单路径（约束破坏子结构） |
| 重叠子问题 | 子问题被反复问到 | 归并排序（子问题不重叠） |

**五大模型速查**：

| 模型 | 状态定义 | 复杂度 | 代表题 |
|---|---|---|---|
| 背包 | $dp[i][j]$=前 $i$ 件容量 $j$ | $O(nW)$ | 0-1 背包、零钱兑换 |
| 线性 DP | $dp[i]$=到第 $i$ 的最优 | $O(n)$ | 爬楼梯、打家劫舍、LIS |
| 双串 DP | $dp[i][j]$=两串前 $i$、前 $j$ | $O(mn)$ | LCS、编辑距离 |
| 状态机 DP | $dp[i][s]$=第 $i$ 阶段状态 $s$ | $O(n\cdot\|S\|)$ | 股票买卖 |
| 区间 DP | $dp[i][j]$=区间 $[i,j]$ | $O(n^3)$ | 石子合并、最长回文 |

**高手向形态**：

| 形态 | 特征 | 代表题 |
|---|---|---|
| 区间 DP | 区间分裂，依赖中间任意位置 | 石子合并、戳气球 |
| 树形 DP | 树上后序遍历，子树返回状态 | 打家劫舍 III、树的直径 |
| 状态机 DP | 离散状态间转移 | 股票系列 |
| 状态压缩（位掩码） | 用二进制位编码状态集合 | TSP、小集合覆盖 |

### 高手向：四种高级 DP 形态

**1. 区间 DP**：状态是"一段区间"，转移是"在区间内找个分裂点 $k$"。

代表：石子合并。$n$ 堆石子排成一排，每次合并相邻两堆代价为两堆之和，求最小总代价。

$$dp[i][j] = \min_{i \leq k < j}\bigl(dp[i][k] + dp[k+1][j] + \text{sum}(i,j)\bigr)$$

其中 $\text{sum}(i,j)$ 是区间 $[i,j]$ 的石子总和（前缀和预处理 $O(1)$）。遍历顺序不是按 $i,j$ 大小，而是**按区间长度**从小到大——因为大区间依赖小区间。复杂度 $O(n^3)$。

**2. 树形 DP**：状态在树上，转移是"子树状态合并"。

代表：打家劫舍 III（树形街道，相邻不能同时偷）。对每个节点返回两个值：`(偷该节点的最大收益, 不偷该节点的最大收益)`，后序遍历合并子树。

```python
def rob_tree(root):
    def dfs(node):
        if not node:
            return (0, 0)               # (偷当前, 不偷当前)
        left = dfs(node.left)
        right = dfs(node.right)
        # 偷当前：左右子节点都不能偷
        rob = node.val + left[1] + right[1]
        # 不偷当前：左右子节点各自"偷或不偷取较大"
        not_rob = max(left) + max(right)
        return (rob, not_rob)
    return max(dfs(root))
```

**关键**：树形 DP 的"状态"是子树根的离散选择，"转移"是后序遍历时合并子树返回值。一次遍历 $O(n)$。

**3. 状态机 DP**：把离散状态做成维度，转移是状态机走边（前文股票买卖已展开）。

**4. 位掩码 DP（状态压缩 DP）**：用二进制位的 0/1 编码"集合是否被选"，把指数级的状态空间压成一个整数。

代表：旅行商问题 TSP。$n$ 个城市，求从 0 出发访问所有城市再回 0 的最短路径。

$$dp[S][v] = \min_{u \in S}\bigl(dp[S \setminus \{v\}][u] + \text{dist}(u, v)\bigr)$$

其中 $S$ 是已访问城市集合（用 $n$ 位二进制编码，共 $2^n$ 种），$v$ 是当前所在城市。复杂度 $O(2^n \cdot n^2)$——指数级，但 $n \leq 20$ 可行。这是"用位运算压缩集合状态"的典范。

### 自测三问

**问题一：** 动态规划为什么能把斐波那契从 $O(2^n)$ 优化到 $O(n)$？请用"重复子问题"解释，并说明暴力递归和记忆化递归的本质区别。

**参考答案：** 暴力递归 $f(n) = f(n-1) + f(n-2)$ 在调用树里，$f(8)$ 被 $f(10)$ 和 $f(9)$ 各算一次，$f(7)$ 被算三次……子问题 $f(k)$ 被反复计算，调用树节点数 $O(2^n)$。记忆化递归在暴力递归基础上加缓存——每个 $f(k)$ 第一次算完存起来，之后直接取。$n$ 个不同子问题各算一次，总复杂度 $O(n)$。**本质区别不是算法更聪明，是拒绝做重复劳动**——发现子问题重叠，算一次记一辈子。DP 是把记忆化翻转为自底向上填表，省了递归栈开销，还能做状态压缩。

**问题二：** 0-1 背包做状态压缩时，内层 $j$ 为什么必须从大到小遍历？正序遍历会变成什么？

**参考答案：** 0-1 背包方程 $dp[i][j] = \max(dp[i-1][j], dp[i-1][j-w_i] + v_i)$，"拿第 $i$ 件"要用**上一轮**（第 $i-1$ 件）的 $dp[j-w_i]$。压成一维数组时，如果 $j$ 从小到大遍历，算 $dp[j]$ 之前 $dp[j-w_i]$ 已经被这一轮更新成"第 $i$ 件也拿了"的值——相当于"第 $i$ 件被拿了多次"，变成完全背包（每件可拿无限次）的方程。倒序遍历保证算 $dp[j]$ 时 $dp[j-w_i]$ 还是上一轮的值，第 $i$ 件最多被拿一次。**记住口诀：0-1 背包倒序、完全背包正序**。

**问题三：** 股票买卖为什么必须用"状态机 DP"，不能用普通一维 DP？请从"状态定义不足"的角度解释，并说明何时该加维度。

**参考答案：** 普通一维 $dp[i]$=第 $i$ 天最大利润，写不出方程——因为"今天能做什么"取决于"昨天结束时是否持股"：没股票才能买、有股票才能卖。$dp[i]$ 丢失了"是否持股"这个关键信息，所以无法判断今天能否买/卖。必须加状态维度 $s \in \{\text{不持股}, \text{持股}\}$，定义 $dp[i][s]$=第 $i$ 天结束状态为 $s$ 的最大利润。**何时加维度的判据：当前状态不足以推出下一步决策时加维度**。股票买卖的"是否持股"、含冷静期的"是否冷冻"、含交易次数限制的"已交易几次"——都是当前状态不够、必须加维度的例子。**遇到 DP 写不出方程，第一反应应该是"是不是漏了状态维度"**。

### 算法思想 × 生活迁移

DP 的核心思想是"**把大问题拆成子问题，记住子问题答案避免重复计算**"。这个思想在生活中比比皆是。

**迁移一：背单词用记忆曲线而不是反复从头背。** 1000 个单词，每天从头背一遍——那是暴力递归，$O(n^2)$ 还记不住。正确做法是 DP 式：每个单词记一次"上次背的日期 + 记忆强度"，下次只复习"强度衰减到阈值"的单词。每个单词的状态被记住，不重复劳动。这正是艾宾浩斯遗忘曲线的本质——把"复习决策"建模成状态转移。

**迁移二：做饭先备料再下锅，而不是边切边炒。** 一道四个菜的晚饭，如果边切边炒（暴力递归式），切完第一个炒第一个，炒的时候切第二个被锅打断——反复切换上下文，效率极低。正确做法是 DP 式：先把所有菜洗切配好（自底向上填表，把所有"子问题"答案备齐），再依次下锅（用子问题答案组装最终解）。备料就是"算子问题存起来"，下锅就是"查表组合"。

**迁移三：项目复盘写"经验沉淀文档"。** 团队遇到一个坑，解决了。如果不记录，下次别人踩同一个坑——这是暴力递归，子问题被反复解决。正确做法是 DP 式：踩坑后立刻写进经验文档（缓存子问题答案），下次有人遇到类似问题先查文档（取缓存）。这就是为什么大公司都搞 Wiki + Postmortem——本质是"组织级 DP"，把个人经验沉淀成组织记忆，避免重复踩坑。

**迁移四：相亲先列硬性条件再细聊。** 100 个候选人，每个都见一面深聊——$O(n^2)$ 做法。DP 式：先用硬性条件（年龄/地域/学历）筛到 20 个（粗筛子问题），再用软性条件（兴趣/价值观）筛到 5 个（精筛子问题），最后深聊。每一层都用上一层的结果，不重复筛选。和希尔的"分层精化"异曲同工，但 DP 更强调"子问题答案被记住、被复用"。

**为什么这些迁移成立：** DP 成立的前提是"问题可拆成子问题 + 子问题会被反复问到"。生活中很多决策都满足这个前提——学习、做饭、团队协作、择偶，都有"子问题反复出现"的特征。迁移成立的关键是"识别出哪些子问题会被反复问到，提前把答案备好"——这是 DP 思想最普适的内核。**拒绝重复劳动**，无论在算法里还是在生活里，都是高效的源头。
