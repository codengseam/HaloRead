# 算法课·18｜回溯算法：选与不选之间的撤销艺术

> 前置知识：学完第 17 章（递归），尤其要懂"递归是函数调用自己"和"递归栈自动管理状态"
> 学完你能：①给纯小白讲清回溯和 DFS 的关系——为什么说回溯是"可撤销的 DFS" ②默写回溯三件套模板（路径 + 选择列表 + 撤销）并讲透每一行 ③手撕 N 皇后、全排列、子集、组合四大经典题 ④理解剪枝的本质——"提前终止无效分支"和"不写剪枝也能对，只是慢" ⑤用对数器对比暴力枚举验证回溯正确性 ⑥分清回溯和 DP 的边界——重叠子问题是分水岭

## 一、问题与思路：选与不选之间的搜索困境

### 1. 一个生活场景：走迷宫为什么要回头

假设你走进一座迷宫，规则只有一条——遇到岔路就先走左边，走不通就退回来走右边。你身上带了一支笔，每走一格就在地上画个记号；走到死胡同，就擦掉最后一个记号退回上一格，换条路再试。整个过程中，你不会"瞬移"，也不会"忘记自己走过哪里"——你靠的就是"边走边记、走错就擦"。

这个"画记号 + 擦记号"的动作，就是回溯算法（Backtracking）的现实原型。它解决的核心问题是一类搜索问题：**当答案藏在一棵巨大的"决策树"里，你不知道哪条路径通向终点，只能系统性枚举所有可能，遇阻即退**。

为什么需要回溯？因为这类问题有一个反直觉的特征：**很多时候你无法在岔路口判断哪条路是对的，只能先走下去试试**。比如 N 皇后——你在第一行放皇后时，根本无法预知这个选择会不会让第 N 行无解；只有往下放，放不下时退回来，才知道"刚才那步错了"。这种"必须试了才知道"的特性，决定了回溯无法用贪心或 DP 一遍算完——它必须保留"撤销"的能力。

### 2. 回溯是什么：DFS 的"可撤销"变体

先给一个准确动作定义：

> 回溯算法 = 深度优先搜索（DFS）+ 在递归返回时撤销当前层所做的状态修改。

这个定义里有三个关键词，缺一不可：

- **DFS**：一条路走到底，不走完不回头。和 BFS（广度优先）相对——BFS 用队列一层层扩，回溯用递归栈一条条钻。
- **状态修改**：在递归进入下一层前，把"当前选择"记下来（加入路径、标记已用）。
- **撤销**：在递归返回后，把刚才记的状态抹掉（移出路径、取消标记），让"兄弟分支"看到的是干净状态。

很多人把回溯和 DFS 混为一谈，这里要划清边界：**DFS 是遍历图/树的通用方法，回溯是 DFS 的一种用法——专用于"在决策树上搜索所有解"**。DFS 不一定修改状态（比如遍历一棵树只读节点值），但回溯一定修改+撤销状态（因为要枚举所有候选解）。可以说：回溯是 DFS 的特例，DFS 是回溯的载体。

为什么撤销这么重要？因为不撤销会"污染兄弟分支"。看全排列 `[1,2,3]` 的例子：第一层选 1，第二层选 2，第三层选 3，得到 `[1,2,3]`。返回到第二层时，必须把"3 已用"这个标记撤销，才能去选第三层另一个候选；如果忘撤销，第三层就永远没得选，整个搜索在第一个解后就死了。**撤销是回溯能"系统枚举所有解"的机制保障**。

### 3. 历史地位：经典搜索范式的根基

回溯的思想可以追溯到 1950 年代。Dijkstra 在 1972 年的论文中明确把回溯作为"系统化搜索"（systematic search）的一种范式。它最经典的应用场景 N 皇后问题，则源自 1848 年国际象棋棋手 Max Bezzel 提出的 8 皇后谜题——在 8×8 棋盘上放 8 个皇后，使其互不攻击。

回溯在算法谱系中的位置：它处在"暴力枚举"和"启发式搜索"之间。比纯暴力聪明的地方在于"剪枝"——能提前判断某些分支不可能产生解，直接跳过；比启发式搜索（如 A*）保守的地方在于不依赖估值函数，保证枚举完整。当问题需要"找所有解"或"找最优解且无法用 DP"时，回溯几乎是默认选择。

理解回溯，也是理解后续更高级搜索算法（分支定界、约束传播、SAT 求解器）的根基——它们都是"回溯 + 更强的剪枝/约束推理"的扩展。这也是为什么算法专家评审强烈建议在专栏里补回溯这一章——少了它，DP、贪心、图搜索之间会缺一根串起来的线。

## 二、原理与实现：回溯三件套与经典问题

### 1. 回溯三件套模板：路径 + 选择列表 + 撤销

所有回溯题，骨架都是同一套——三件套：

```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        结果.append(路径[:])   # 为什么用 [:]：存快照，避免后续撤销改到结果里的同一对象
        return
    for 选择 in 选择列表:
        做选择                  # 把选择加入路径，标记已用
        backtrack(路径, 选择列表)
        撤销选择                 # 把选择移出路径，取消标记——这一步是回溯的灵魂
```

三件套对应三件事：

- **路径（path）**：当前已经做出的选择序列。比如全排列里 `[1,2]` 表示前两步选了 1 和 2。
- **选择列表（choices）**：当前层还能做哪些选择。全排列里就是"还没用过的数字"。
- **撤销（undo）**：递归返回后，把刚才"做选择"的副作用全部抹掉。

**逐行讲透**（左神风格，每一行都不能含糊）：

- **`if 满足结束条件`**：递归终止条件。通常是"路径长度等于目标长度"（全排列、组合）或"路径铺满棋盘"（N 皇后、数独）。
- **`结果.append(路径[:])`**：为什么写 `路径[:]` 而不是 `路径`？因为 `路径` 是同一个 list 对象，后面撤销会改它，最后结果里全是空 list。`[:]` 是浅拷贝，存的是当前快照。
- **`for 选择 in 选择列表`**：枚举当前层所有可能选择。这是回溯"系统枚举"的体现——不漏掉任何一个分支。
- **`做选择`**：通常两步——`path.append(x)` 和 `used[x] = True`。前者记路径，后者记"已用"。
- **`backtrack(...)`**：进入下一层。递归栈会自动管理状态。
- **`撤销选择`**：和"做选择"对称——`path.pop()` 和 `used[x] = False`。**这一步是回溯区别于普通 DFS 的关键**，也是新手最容易漏的一步。

**关键观察**：三件套的对称性——做选择和撤销选择必须严格对称。`append` 配 `pop`，`used[x] = True` 配 `used[x] = False`，`board[i][j] = 'Q'` 配 `board[i][j] = '.'`。漏掉任何一个撤销，整个搜索就会"串味"——兄弟分支看到的是被污染的状态。

### 2. 全排列：最经典的入门题

**题目**：给定一个不含重复数字的数组 `nums`，返回所有全排列。

```python
def permute(nums):
    def backtrack(path, used):
        # 终止：路径长度等于原数组长度，说明所有数都选完了
        if len(path) == len(nums):
            res.append(path[:])   # 为什么 [:]：存快照，否则后续 pop 会改到结果
            return
        for i in range(len(nums)):
            if used[i]:
                continue            # 为什么跳过：已选过的数不能再选，保证排列不重复
            path.append(nums[i])
            used[i] = True
            backtrack(path, used)
            path.pop()              # 撤销路径
            used[i] = False         # 撤销标记——和上面 True 严格对称

    res = []
    backtrack([], [False] * len(nums))
    return res
```

**走一遍 `nums = [1,2,3]`**：

```
第一层选 1: path=[1], used=[T,F,F]
  第二层选 2: path=[1,2], used=[T,T,F]
    第三层选 3: path=[1,2,3], used=[T,T,T] → 满足终止，存 [1,2,3]
    撤销 3: path=[1,2], used=[T,T,F]
  撤销 2: path=[1], used=[T,F,F]
  第二层选 3: path=[1,3], used=[T,F,T]
    第三层选 2: path=[1,3,2] → 存 [1,3,2]
    撤销 2
  撤销 3
撤销 1
第一层选 2: ... (得到 [2,1,3] 和 [2,3,1])
第一层选 3: ... (得到 [3,1,2] 和 [3,2,1])
```

**复杂度**：n 个数全排列共 $n!$ 个解，每个解需要 $O(n)$ 拷贝到结果，总时间 $O(n \cdot n!)$，空间 $O(n)$ 递归栈深度（不含结果存储）。

**为什么是 $n!$ 而不是 $n^n$**：因为有 `used[i]` 剪掉了"重复选同一元素"的分支。如果不剪枝（允许重复选），就是 $n^n$ 个解，对应"可重复排列"问题。剪枝的强度直接决定搜索空间大小。

### 3. N 皇后：回溯的标杆题

**题目**：在 $n \times n$ 棋盘上放 $n$ 个皇后，使其互不攻击（同行、同列、同两条对角线只能有一个皇后）。返回所有摆法。

```python
def solve_n_queens(n):
    def backtrack(row, cols, diag1, diag2, path):
        # 终止：所有行都放完
        if row == n:
            res.append(path[:])
            return
        for col in range(n):
            # 为什么用位运算/集合判冲突：O(1) 查询，比遍历棋盘快
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue    # 冲突分支直接跳过——这就是剪枝
            path.append(col)
            cols.add(col); diag1.add(row - col); diag2.add(row + col)
            backtrack(row + 1, cols, diag1, diag2, path)
            path.pop()
            cols.remove(col); diag1.remove(row - col); diag2.remove(row + col)

    res = []
    backtrack(0, set(), set(), set(), [])
    return res
```

**关键技巧：用集合代替二维棋盘判冲突。** 三个集合分别记录：

- `cols`：哪些列已被占用
- `diag1`：主对角线方向（左上到右下），同一条线上 `row - col` 相同
- `diag2`：副对角线方向（右上到左下），同一条线上 `row + col` 相同

为什么能用 `row - col` 和 `row + col` 标识对角线？因为在同一条主对角线上，所有格子的 `row - col` 是常数（往右下走 row 和 col 同步加 1，差不变）；同一条副对角线上 `row + col` 是常数（往左下走 row 加 1、col 减 1，和不变）。**这是把二维冲突检测压成 $O(1)$ 的标准技巧**。

**复杂度**：最坏枚举 $n^n$ 种摆法（每行 n 个位置都试），剪枝后实际探索的节点远小于此。8 皇后的解只有 92 个，但回溯探索的节点数约 2000，剪枝把 $8^8 \approx 1677$ 万压到几千。这正是剪枝的价值——**不是改变最坏复杂度上界，而是把实际搜索空间砍掉几个数量级**。

### 4. 子集与组合：选与不选两种写法

回溯题里有一类"每个元素选或不选"的问题，模板有两种等价写法。理解它们的等价性，能帮你彻底吃透回溯。

**题目**：给定不含重复元素的数组 `nums`，返回所有子集。

**写法一：选/不选（每个元素做决策）**

```python
def subsets_v1(nums):
    def backtrack(i, path):
        # 每个节点都是一个合法子集，都存
        res.append(path[:])
        for j in range(i, len(nums)):
            path.append(nums[j])         # 选 nums[j]
            backtrack(j + 1, path)
            path.pop()                   # 不选 nums[j]（撤销）
    res = []
    backtrack(0, [])
    return res
```

**写法二：枚举起点（在循环里选）**

```python
def subsets_v2(nums):
    def backtrack(start, path):
        res.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)        # 为什么 i+1 而不是 i：避免重复选自己
            path.pop()
    res = []
    backtrack(0, [])
    return res
```

两种写法生成的子集顺序不同，但集合相同。**为什么 `i + 1` 而不是 `i`**：因为 `i` 会允许重复选同一元素，导致无限递归；`i + 1` 强制"下一个选的位置严格靠后"，保证每个元素只被选一次。

**组合问题**（从 `n` 个数里选 `k` 个）和子集几乎一样，区别只在终止条件：子集每个节点都存，组合只在 `len(path) == k` 时存。

### 5. 剪枝优化：无效分支提前终止

剪枝（Pruning）是回溯从"能跑通"到"能跑快"的关键。它的思路是：**在递归进入下一层之前，判断这条分支是否可能产生解，不可能就直接 `continue` 跳过**。

剪枝分两类：

**剪枝一：可行性剪枝**——分支不可能产生合法解，跳过。N 皇后的 `if col in cols: continue` 就是这一类：放下去就冲突，后面再怎么放都救不回来。

**剪枝二：最优性剪枝**——分支可能产生解但不可能比当前最优更好，跳过。这类剪枝在"求最优解"的回溯题里常见，比如旅行商问题（TSP）——当前路径长度已经超过已知最短回路，剪掉。

**剪枝正确性原则**：剪枝只能剪"一定不会产生解/一定不是最优解"的分支，绝不能剪"可能产生解"的分支。剪错了，就漏解。所以验证剪枝正确性的标准是：**剪枝前后跑对数器，结果集必须完全一致**。

**剪枝的代价**：剪枝判断本身有开销。如果判断成本高于剪掉的搜索量，剪枝反而拖慢。所以"剪枝越早越好"是有条件的——判断必须 $O(1)$ 或接近 $O(1)$。N 皇后用集合判冲突就是 $O(1)$，如果改成遍历棋盘判冲突，剪枝成本 $O(n)$，反而可能更慢。

### 6. 五指标评价

| 指标 | 回溯（含剪枝） | 说明 |
|---|---|---|
| 最好 | $O(\text{解数} \times n)$ | 第一个分支就找到所有解，无回溯 |
| 平均 | 依赖剪枝强度 | N 皇后 8×8 约 2000 节点；无剪枝 $n^n$ |
| 最坏 | $O(n^n)$ 或 $O(n!)$ | 取决于问题；全排列 $O(n \cdot n!)$ |
| 空间 | $O(n)$ 递归栈 + $O(\text{解数} \times n)$ 结果 | 不计结果时为 $O(n)$ |
| 是否稳定 | 否 | 输出顺序依赖枚举顺序，但不影响解集正确性 |
| 是否自适应 | 否 | 不像排序那样识别"已有序"提前退出 |

回溯的复杂度有两个特点值得记住：①**最坏复杂度通常很糟**（指数级或阶乘级），但实际剪枝后往往能跑；②**解的数量决定下界**——如果问题本身有 $n!$ 个解，任何算法至少要 $O(n!)$ 时间把它们列出来，回溯已经是渐近最优。

## 三、实践与面试：对数器、工程坑、面试高频题

### 1. 对数器：对比暴力枚举验证

回溯题的对数器比排序更难写——因为"标准答案"通常是另一个暴力实现。思路是：写一个绝对正确但很慢的暴力枚举（比如 `itertools.permutations`），和你的回溯对比，跑随机数据看结果集是否一致。

```python
import itertools
import random

def permute(nums):
    """你的回溯实现。"""
    def backtrack(path, used):
        if len(path) == len(nums):
            res.append(path[:])
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            path.append(nums[i])
            used[i] = True
            backtrack(path, used)
            path.pop()
            used[i] = False
    res = []
    backtrack([], [False] * len(nums))
    return res

def checker(test_times=500, max_n=8, max_val=20):
    """对数器：用 itertools.permutations 作为标准答案。"""
    for _ in range(test_times):
        n = random.randint(0, max_n)
        nums = [random.randint(0, max_val) for _ in range(n)]
        # 标准答案：itertools 全排列，再转 list
        expected = [list(p) for p in itertools.permutations(nums)]
        # 你的回溯
        got = permute(nums)
        # 比较时要去重——因为有重复数字时 itertools 也会产生重复排列
        if sorted(map(tuple, expected)) != sorted(map(tuple, got)):
            print(f"出错！输入: {nums}, 期望: {expected}, 你的结果: {got}")
            return False
    print("对数器验证通过！")
    return True

checker()
```

**对数器写法的两个坑**：

- **结果顺序不一致**：回溯和 `itertools` 生成的排列顺序可能不同，不能直接 `==` 比较。用 `sorted(...)` 把两边都排序后再比。
- **重复元素**：`nums` 有重复数字时，`itertools.permutations` 会产生重复排列（把两个相同数字当成不同），回溯也会。比较时要么先去重，要么允许重复。

**对高手的启发**：回溯对数器的关键不是"找一个标准库"，而是"找一个绝对正确的对照实现"。没有标准库的题（比如 N 皇后），可以写一个 $O(n^n)$ 纯暴力枚举每个格子放/不放，再过滤合法解——慢但绝对对，拿来对照回溯。

### 2. 回溯 vs DP：重叠子问题的分水岭

回溯和动态规划（DP）经常被搞混，因为它们都解决"多阶段决策"问题。区分两者的关键是一个问题特征——**重叠子问题**。

**回溯的特征**：子问题不重叠，每条路径独立。比如全排列——选了 `[1,2]` 后再选 3，和选了 `[2,1]` 后再选 3，是两个不同的子问题（剩余可用数字不同）。所以无法用"记忆化"加速，只能老老实实枚举所有路径。

**DP 的特征**：子问题重叠，相同子问题被多次求解。比如斐波那契——`fib(5)` 需要 `fib(4)` 和 `fib(3)`，`fib(4)` 又需要 `fib(3)` 和 `fib(2)`，`fib(3)` 被算了两次。用记忆化（memo）记录已算过的子问题，就能从指数级降到多项式级。

**判定方法**：写一个朴素回溯，如果发现"同样的状态被重复进入"，就说明有重叠子问题，可以加 memo 升级成记忆化搜索（等价于 DP）。如果每个状态只进一次，就是纯回溯，加 memo 没用。

```python
# 例子：爬楼梯——重叠子问题，回溯会超时，加 memo 变 DP
def climb_stairs_backtrack(n):
    """纯回溯：指数级，n=40 就跑不动。"""
    if n <= 1:
        return 1
    return climb_stairs_backtrack(n - 1) + climb_stairs_backtrack(n - 2)

def climb_stairs_dp(n, memo=None):
    """加 memo：O(n)，等价于 DP。"""
    if memo is None:
        memo = {}
    if n <= 1:
        return 1
    if n in memo:
        return memo[n]
    memo[n] = climb_stairs_dp(n - 1, memo) + climb_stairs_dp(n - 2, memo)
    return memo[n]
```

**对高手的启发**：回溯 + memo = 记忆化搜索 = 自顶向下 DP。三者是同一件事的不同视角。理解这一点，就不会把回溯和 DP 当成两个割裂的东西——它们是"枚举所有可能"这条思路在不同问题特征下的两种实现策略。

### 3. 三个真实工程坑

**坑一：忘记撤销，结果只有一个解。**

```python
def permute_bug(nums):
    def backtrack(path, used):
        if len(path) == len(nums):
            res.append(path[:])
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            path.append(nums[i])
            used[i] = True
            backtrack(path, used)
            path.pop()
            # used[i] = False   ← 漏了这一行！
    res = []
    backtrack([], [False] * len(nums))
    return res

print(permute_bug([1,2,3]))   # 只输出 [[1,2,3]]，其他解全丢了
```

症状：只输出一个解，其他解全丢失。根因：`used[i]` 没撤销，第二层选完 2 返回后，`used[1]` 还是 `True`，导致后续无法再选 2。修复：补上 `used[i] = False`，和 `used[i] = True` 严格对称。

**坑二：用 `res.append(path)` 而不是 `res.append(path[:])`，结果全是空 list。**

```python
def subsets_bug(nums):
    def backtrack(start, path):
        res.append(path)         # ← 没用 [:]，存的是 path 的引用
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    res = []
    backtrack(0, [])
    return res

print(subsets_bug([1,2,3]))
# 输出: [[], [], [], [], [], [], [], []]  ← 全是空 list
```

症状：结果里全是空 list。根因：`res.append(path)` 存的是 `path` 这个 list 对象的引用，所有结果指向同一个对象；递归返回时 `path.pop()` 把元素全弹出，最后所有引用都指向空的 `path`。修复：`res.append(path[:])` 存快照。

**坑三：剪枝写错位置，漏掉合法解。**

```python
def combination_sum_bug(candidates, target):
    """从 candidates 中选数（可重复选），和为 target 的所有组合。"""
    def backtrack(start, path, remain):
        if remain == 0:
            res.append(path[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remain:
                break    # ← 这里用 break，前提是 candidates 已排序
            path.append(candidates[i])
            backtrack(i, path, remain - candidates[i])
            path.pop()
    res = []
    backtrack(0, [], target)   # ← 没排序就用了 break
    return res
```

症状：某些输入漏掉合法解。根因：剪枝用 `break`（跳过当前及之后所有）而非 `continue`（只跳过当前），前提是数组已排序——只有排序后"当前太大"才意味着"后面更大也太大"。如果没排序就 `break`，会把后面的合法小数字也跳过。修复：要么先 `candidates.sort()`，要么把 `break` 改成 `continue`（更慢但安全）。

### 4. 面试高频题：括号生成、分割回文串、组合总和

这三题是回溯面试的"高频三件套"，覆盖了回溯的三种典型变形。

**题一：括号生成**（生成 $n$ 对括号的所有合法组合）

```python
def generate_parenthesis(n):
    def backtrack(path, left, right):
        # left/right：已用的左右括号数
        if len(path) == 2 * n:
            res.append(path[:])
            return
        if left < n:        # 为什么 left < n：左括号最多用 n 个
            path.append('(')
            backtrack(path, left + 1, right)
            path.pop()
        if right < left:    # 为什么 right < left：右括号不能比左括号多，否则不合法
            path.append(')')
            backtrack(path, left, right + 1)
            path.pop()
    res = []
    backtrack([], 0, 0)
    return res
```

讲解要点：这题的关键不是"选/不选"，而是"在每一步用剩余配额做决策"。剪枝条件 `right < left` 直接保证了任意前缀里左括号数 ≥ 右括号数——这是合法括号串的定义。

**题二：分割回文串**（把字符串切成若干子串，每个子串都是回文）

```python
def partition(s):
    def is_palindrome(sub):
        return sub == sub[::-1]   # 为什么用切片反转：O(n) 判断，简洁
    def backtrack(start, path):
        if start == len(s):
            res.append(path[:])
            return
        for end in range(start + 1, len(s) + 1):
            sub = s[start:end]
            if not is_palindrome(sub):
                continue    # 不是回文的分支直接剪掉
            path.append(sub)
            backtrack(end, path)
            path.pop()
    res = []
    backtrack(0, [])
    return res
```

讲解要点：这题的"选择"是切点位置，剪枝是"当前子串不是回文就跳过"。优化方向：用 DP 预处理所有子串是否回文，把 `is_palindrome` 从 $O(n)$ 降到 $O(1)$。

**题三：组合总和**（从 candidates 中选数，可重复选，和为 target）

```python
def combination_sum(candidates, target):
    candidates.sort()   # 为什么先排序：让剪枝能用 break 而非 continue
    def backtrack(start, path, remain):
        if remain == 0:
            res.append(path[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remain:
                break    # 排序后，当前太大后面更大，直接 break
            path.append(candidates[i])
            backtrack(i, path, remain - candidates[i])   # 为什么传 i 不传 i+1：可重复选
            path.pop()
    res = []
    backtrack(0, [], target)
    return res
```

讲解要点：①排序让剪枝从 `continue`（$O(n)$）升级到 `break`（更早终止）；②`backtrack(i, ...)` 传 `i` 而非 `i+1`，允许重复选同一元素——这是"组合总和"和"子集"的关键区别。

**面试时怎么讲**（按"技术的水有多深"八层面，挑重点讲）：

1. **问题层**：回溯解决"在决策树上搜索所有解"的问题，特点是"必须试了才知道哪条路对"。
2. **原理层**：DFS + 撤销。三件套——路径、选择列表、撤销。撤销是和普通 DFS 的区别。
3. **优劣层**：能找所有解（贪心做不到）、能处理无最优子结构的问题（DP 做不到）；但最坏指数级，依赖剪枝。
4. **演进层**：朴素回溯 → 加剪枝 → 加 memo 变记忆化搜索（等价 DP）→ 加下界估计变分支定界。回溯是搜索算法的根基。

**面试加分点**：主动提"回溯 + memo = 记忆化搜索 = 自顶向下 DP"，能立刻把回溯和 DP 串起来——这是很多候选人讲不清的关系。

## 四、速查与自测

### 速查表：回溯核心要素与典型题

| 要素 | 说明 | 常见写法 |
|---|---|---|
| 路径 | 已做的选择序列 | `path` (list) |
| 选择列表 | 当前层可选的候选 | `for i in range(...)` 或 `for choice in candidates` |
| 撤销 | 抹掉当前层的选择 | `path.pop()` + 取消标记 |
| 终止条件 | 路径达到目标长度/铺满棋盘 | `if len(path) == n` 或 `if row == n` |
| 剪枝 | 跳过无效分支 | `if 冲突: continue` 或 `if 超限: break` |

**典型回溯题分类**：

| 题型 | 代表题 | 关键剪枝 |
|---|---|---|
| 排列问题 | 全排列 | `used[i]` 跳过已选 |
| 子集/组合 | 子集、组合总和 | `start` 参数防重复选 |
| 棋盘问题 | N 皇后、数独 | 集合 $O(1)$ 判冲突 |
| 分割问题 | 分割回文串 | 当前段非回文则跳过 |
| 生成问题 | 括号生成 | `right < left` 保证合法 |

### 自测三问

**问题一：** 回溯和 DFS 的关系是什么？为什么说"回溯是 DFS 的可撤销变体"？

**参考答案：** DFS 是遍历图/树的通用方法，回溯是 DFS 的一种用法——专用于"在决策树上搜索所有解"。两者关系是：回溯一定包含 DFS（用递归栈一条路走到底），但 DFS 不一定是回溯（遍历一棵树只读节点值就不需要撤销状态）。回溯区别于普通 DFS 的关键在于"撤销"——在递归返回时把当前层做的状态修改抹掉，让兄弟分支看到干净状态。这个撤销动作是回溯能"系统枚举所有解"的机制保障，也是新手最容易漏的一步。

**问题二：** 回溯三件套是哪三件？为什么"做选择"和"撤销选择"必须严格对称？

**参考答案：** 三件套是：路径（已做的选择序列）、选择列表（当前层可选候选）、撤销（递归返回时抹掉当前层状态）。对称性的原因：做选择时改了两个状态——`path.append(x)` 和 `used[x] = True`，撤销时必须两个都还原——`path.pop()` 和 `used[x] = False`。漏掉任何一个，兄弟分支就会看到被污染的状态。比如全排列里只 `pop()` 忘了 `used[x] = False`，第二层选完 2 返回后 `used[1]` 仍是 `True`，后续永远无法再选 2，整个搜索在第一个解后就死了。**对称性是回溯正确性的硬约束**。

**问题三：** 回溯和 DP 的分水岭是什么？什么样的回溯题可以加 memo 升级成 DP？

**参考答案：** 分水岭是"重叠子问题"。回溯适用于子问题不重叠、每条路径独立的场景（如全排列——选 `[1,2]` 后再选 3，和选 `[2,1]` 后再选 3 是不同子问题，剩余可用数字不同）。DP 适用于子问题重叠、相同子问题被多次求解的场景（如斐波那契——`fib(3)` 被算两次）。判定方法：写朴素回溯，如果发现"同样的状态被重复进入"，就说明有重叠子问题，可以加 memo 记录已算过的子问题，从指数级降到多项式级——这就是记忆化搜索，等价于自顶向下的 DP。**回溯 + memo = 记忆化搜索 = 自顶向下 DP，三者是同一件事的不同视角**。

### 算法思想 × 生活迁移

回溯的核心思想是"**系统枚举 + 遇阻即退**"——不预判哪条路对，先走下去试，走不通就撤销最后一步换条路再试。

**迁移一：试错式学习新技能。** 学一门新技术（比如一门框架），不要等"看完全部文档再动手"——那是贪心思路，要求每一步都选最优。实际学习中你无法预知哪个教程最适合自己，只能回溯式：先看 A 教程试做一个小项目，做不通就退回来换 B 教程（撤销 A 的部分认知），再试。每次"撤销"不是白费——你知道了"A 教程在某个点讲不清楚"，这个信息留下来指导后续选择。

**迁移二：相亲/招聘的"先试后退"。** 找对象或招人，无法在第一眼判断是否合适（决策树太深，无法一眼看穿）。回溯式做法是：先深入了解一个候选人（一条路走到底），发现关键不匹配就退回（撤销），换下一个。关键是要"撤销干净"——不要让上一个候选人的印象污染对下一个的判断，每次都从干净状态开始评估。这就是为什么面试官培训里强调"避免锚定效应"——锚定就是不撤销的状态污染。

**迁移三：方案设计的"先实现再回退"。** 写一个复杂功能时，不要等想清楚所有细节再动手。可以先按一个可行方案写到一半，发现某处走不通（比如选的库不支持某个需求），就退回那个分叉点换方案。git 的分支和 revert 命令，本质上就是给代码改动提供"撤销"能力——让你敢"先试错再回退"，而不是在动手前纠结半年。

**为什么这些迁移成立：** 回溯成立的前提是"决策可以撤销"且"撤销成本可接受"。这个前提在试错学习、人际选择、方案设计里都成立——试错有成本但不大，撤销能让"试错"成为可行策略。迁移不成立的场景是"撤销成本极高"——比如手术、跳楼、签合同后违约，这些场景必须用贪心或 DP 式的"一步到位"，因为回溯式试错的代价无法承受。
