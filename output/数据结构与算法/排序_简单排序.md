# 算法课·04｜简单排序三兄弟：冒泡、选择、插入的直觉与边界

> 前置知识：学完第 03 章（排序总览与五指标评价框架）
> 学完你能：①给纯小白讲清冒泡/选择/插入三种排序的直觉思路 ②默写三种排序并讲透每一行 ③用五指标评价它们的优劣，讲清"为什么都是 $O(n^2)$ 但插入实际最快" ④用对数器验证排序正确性 ⑤知道简单排序在 Timsort、小数据、面试中的真实用武之地

## 一、问题与思路：人类最直觉的三种排序方式

### 1. 一个生活场景：打扑克牌整理手牌

想象你在打扑克，每摸一张牌都要插到手牌里让手牌保持有序。你会怎么做？

最自然的动作是：**拿新牌，从右往左和已有的牌比，找到位置插进去**。这就是插入排序（Insertion Sort）——人脑整理手牌的直觉算法。

但如果你手里有一把乱序的牌要全部排好，可能换一种方式：**扫一遍找最小的，放最前；再扫一遍找次小的，放第二**。这就是选择排序（Selection Sort）——"每次选最小"的直觉算法。

还有一种：**两两比较，大的往后换，让大牌"冒"到右边**。这就是冒泡排序（Bubble Sort）——"相邻交换"的直觉算法。

**这三种是人类最自然的排序方式**，也是计算机科学最早的排序算法。它们都是 $O(n^2)$，看起来"低级"，但各有各的用武之地。理解它们的差异，比记住"它们都是 $O(n^2)$"重要得多。

### 2. 核心思想：三种"消除逆序对"的策略

第 03 章讲过，排序的本质是"消除逆序对"。简单排序三兄弟代表了三种不同的消除策略：

- **冒泡**：相邻交换消除逆序对。每消除一个逆序对需要一次交换。
- **选择**：找全局最小消除。每轮把最小放最前，相当于一次消除它和前面所有元素的逆序对（但代价是先全扫描找最小）。
- **插入**：把当前元素插到正确位置。一次比较+移动消除一个逆序对，但移动是连续的，缓存友好。

**记住这三种"消除逆序对"的策略**。后面讲快排、归并时，你会发现它们的本质是"用分治批量消除逆序对"——把每次消一个升级成每次消一批，从 $O(n^2)$ 突破到 $O(n\log n)$。简单排序是理解这条主线的起点。

### 3. 历史背景：为什么这些算法在 50-60 年代集中出现

简单排序的诞生和计算机早期硬件有关。1950-60 年代，内存小、CPU 慢，排序是稀缺操作。冒泡排序的最早描述可追溯到 1956 年（FORTRAN 早期论文），选择和插入更早（手工计算时代就在用）。

这些算法的核心思想是**人类对"有序化"的直觉反应**——遇到乱序，最直觉的就是"两两比较交换"（冒泡）、"找最小放最前"（选择）、"逐个插入"（插入）。计算机科学家只是把这些直觉形式化成代码。

## 二、原理与实现：三个算法的机制、代码与五指标评价

### 1. 冒泡排序（Bubble Sort）

**机制**：从头到尾两两比较相邻元素，顺序错就交换。一轮下来，最大元素"冒泡"到末尾。重复 n-1 轮。

```python
def bubble_sort(nums):
    n = len(nums)
    for i in range(n - 1):
        swapped = False   # 自适应优化：本轮无交换说明已有序
        for j in range(n - 1 - i):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]
                swapped = True
        if not swapped:   # 本轮没交换，数组已有序，提前退出
            break
    return nums
```

**逐行讲透**（左神风格）：

- **`for i in range(n - 1)`**：外层 n-1 轮。为什么 n-1 不是 n？因为 n 个元素，n-1 轮后最后剩下的那个自然就位了，不用再排。
- **`swapped = False`**：本轮开始前重置标志。这是冒泡能"自适应"的关键——如果某一轮全程没交换，说明数组已经有序。
- **`for j in range(n - 1 - i)`**：内层比较范围。为什么是 `n-1-i`？因为前 i 轮已经把 i 个最大元素冒到末尾了，末尾 i 个位置已就位，不用再比较。
- **`if nums[j] > nums[j + 1]`**：用严格大于。为什么不用 `>=`？因为相等时不交换，保证稳定性（相等元素的相对顺序不变）。
- **`if not swapped: break`**：本轮无交换就提前退出。这让冒泡在"基本有序数据"上变成 $O(n)$。

**五指标评价**：

| 指标 | 冒泡 | 说明 |
|---|---|---|
| 最好 | $O(n)$ | 加 `swapped` 优化后，基本有序数据一轮就退出 |
| 平均 | $O(n^2)$ | 总比较 $\approx n^2/2$ |
| 最坏 | $O(n^2)$ | 逆序数组，每次都要交换 |
| 空间 | $O(1)$ | 原地 |
| 稳定 | 是 | 相等不交换 |
| 自适应 | 是（加优化） | `swapped` 标志让它能识别有序 |
| 缓存 | 好 | 顺序访问相邻元素 |

### 2. 选择排序（Selection Sort）

**机制**：每轮从未排序区间找最小值，与未排序区间首位交换。n-1 轮后完成。hello-algo 的[选择排序章节](https://www.hello-algo.com/chapter_sorting/selection_sort/)讲得很直观。

```python
def selection_sort(nums):
    n = len(nums)
    for i in range(n - 1):
        k = i   # 记录最小元素下标
        for j in range(i + 1, n):
            if nums[j] < nums[k]:
                k = j
        nums[i], nums[k] = nums[k], nums[i]   # 把最小换到位置 i
    return nums
```

**逐行讲透**：

- **`for i in range(n - 1)`**：外层 n-1 轮，和冒泡一样。
- **`k = i`**：k 记录"当前未排序区间的最小元素下标"，初始假设是未排序区间首位 i。
- **`for j in range(i + 1, n)`**：内层扫描未排序区间 `[i+1, n)`，找真正的最小。
- **`if nums[j] < nums[k]: k = j`**：发现更小的就更新 k。注意只更新下标，不立即交换——这样每轮只有一次交换（最后才换）。
- **`nums[i], nums[k] = nums[k], nums[i]`**：把最小元素换到位置 i。每轮只交换这一次。

**关键洞察——交换次数最少**：选择排序每轮只交换一次（冒泡可能交换多次）。如果交换代价高（大对象、磁盘上的记录），选择比冒泡好。这是选择排序的核心优势。

**关键洞察——不稳定**：跨距离交换破坏稳定性。比如 `[5a, 5b, 3]`，第一轮把 3 换到位置 0，原 `5a` 被换到位置 2——`5a` 和 `5b` 的相对顺序变了。

**五指标评价**：

| 指标 | 选择 | 说明 |
|---|---|---|
| 最好 | $O(n^2)$ | 不自适应，已有序也要全扫 |
| 平均 | $O(n^2)$ | 总比较 $\approx n^2/2$ |
| 最坏 | $O(n^2)$ | 同上 |
| 空间 | $O(1)$ | 原地 |
| 稳定 | 否 | 跨距离交换 |
| 自适应 | 否 | 必须扫描完才知道最小 |
| 缓存 | 好 | 顺序访问 |

### 3. 插入排序（Insertion Sort）

**机制**：把数组分成"已排序"和"未排序"两部分。每次从未排序取一个，在已排序里从右往左找位置插入。就像打扑克时整理手牌。

```python
def insertion_sort(nums):
    for i in range(1, len(nums)):
        key = nums[i]
        j = i - 1
        while j >= 0 and nums[j] > key:   # 大于 key 的元素后移
            nums[j + 1] = nums[j]
            j -= 1
        nums[j + 1] = key   # 插入到正确位置
    return nums
```

**逐行讲透**：

- **`for i in range(1, len(nums))`**：从第 2 个元素开始（第 1 个天然有序）。为什么从 1 不从 0？因为位置 0 的元素自成"已排序部分"，插入从第 2 个开始。
- **`key = nums[i]`**：保存当前要插入的元素。为什么先存？因为后面 `nums[j+1] = nums[j]` 会覆盖位置 i 的值，不存就丢了。
- **`j = i - 1`**：j 指向已排序部分的最后一个元素。
- **`while j >= 0 and nums[j] > key`**：从右往左扫描已排序部分，找第一个 ≤ key 的位置。用 `>` 不用 `>=`——相等时不后移，保证稳定。
- **`nums[j + 1] = nums[j]`**：大于 key 的元素后移一位。注意是"后移"不是"交换"——一次赋值，比冒泡的交换（3 次赋值）轻量。
- **`nums[j + 1] = key`**：把 key 插入到正确位置（j+1）。

**关键洞察——自适应极强**：对基本有序数据，内层 while 几乎不执行，接近 $O(n)$。这是简单排序里自适应性最强的。

**关键洞察——缓存友好**：内层 while 是连续访问相邻元素，CPU 缓存命中率极高。

**五指标评价**：

| 指标 | 插入 | 说明 |
|---|---|---|
| 最好 | $O(n)$ | 已有序数据，内层 while 不执行 |
| 平均 | $O(n^2)$ | 总比较+移动 $\approx n^2/4$ |
| 最坏 | $O(n^2)$ | 逆序数据，每个都要移到底 |
| 空间 | $O(1)$ | 原地 |
| 稳定 | 是 | 严格大于，相等不后移 |
| 自适应 | 是 | 极强，基本有序数据接近 $O(n)$ |
| 缓存 | 极好 | 连续访问，缓存命中率最高 |

### 4. 三者横向对比：为什么都是 O(n²)，但实际差异很大

| 维度 | 冒泡 | 选择 | 插入 |
|---|---|---|---|
| 比较 | 相邻比较 | 全扫描找最小 | 与已排序部分比较 |
| 交换/移动 | 多次交换（$O(n^2)$ 次） | 最少交换（$O(n)$ 次） | 连续移动（$O(n^2)$ 次但快） |
| 稳定 | 是 | 否 | 是 |
| 自适应 | 是（加优化） | 否 | 是（最强） |
| 缓存 | 好 | 好 | 极好 |
| 实际速度 | 最慢 | 中 | 最快 |

**为什么插入最快**？两个原因：①比较+移动操作比交换轻量（交换是 3 次赋值，移动是 1 次）②缓存友好（连续访问）。这就是为什么 Timsort 内部对小数组（n < 64）用插入排序——它在小数据上常数最小。

**什么时候选择排序更好**？当交换代价远大于比较代价时。比如排序的是磁盘上的大记录，每次交换要读写磁盘，比较只是读关键字。这时选择排序的"最少交换"特性让它更快。

**什么时候冒泡更好**？几乎没有。冒泡在所有维度上都不优于插入。它的价值主要是教学——演示"相邻交换消除逆序对"这个最朴素的思路。

### 5. 为什么它们都是 O(n²)：逆序对视角的本质

n 个元素的数组最多有 $\binom{n}{2} = n(n-1)/2$ 个逆序对。简单排序每次操作最多消除一个逆序对：

- 冒泡：每次相邻交换消除 1 个逆序对
- 选择：每轮把最小元素换到最前，消除它和前面所有元素的逆序对（看似消除多个，但代价是先全扫描找最小，扫描本身是 $O(n)$，所以总成本仍 $O(n^2)$）
- 插入：每次插入消除 1 个逆序对（移动一次消除一个）

总逆序对最多 $O(n^2)$，每次操作消除 1 个，所以总成本 $O(n^2)$。**这就是简单排序无法突破 $O(n^2)$ 的根本原因——它们每次操作只消除一个逆序对**。

后面讲快排、归并时，你会发现它们的本质是"一次操作消除多个逆序对"——这就是分治能突破 $O(n^2)$ 的关键。记住这条主线，所有排序算法都能串起来。

## 三、实践与面试：手撕模板、对数器、面试题

### 1. 面试手撕模板（三个都要会）

面试中如果被要求手写简单排序，**插入排序是最高频的**（因为它是 Timsort 的底层，面试官爱考），冒泡和选择次之。三个都要会默写。

**插入排序（最常用）**：

```python
def insertion_sort(nums):
    for i in range(1, len(nums)):
        key = nums[i]
        j = i - 1
        while j >= 0 and nums[j] > key:
            nums[j + 1] = nums[j]
            j -= 1
        nums[j + 1] = key
    return nums
```

**面试时怎么讲插入排序**（按"技术的水有多深"八层面）：

1. **问题层**：为什么需要插入排序——它是 Timsort 的底层，对小子数组排序时用。
2. **原理层**：把数组分已排序和未排序两部分，每次从未排序取一个，在已排序里从右往左找位置插入。
3. **优劣层**：稳定、自适应（基本有序 $O(n)$）、缓存极好；但 $O(n^2)$ 大数据不行。
4. **演进层**：插入排序 → 希尔排序（跨步长插入）→ Timsort（归并 + 插入混合）。

**面试加分点**：主动提"插入排序是 Timsort 对小数据切排的底层"，能立刻展示你懂生产排序的原理，而不只是会背插入排序代码。

### 2. 对数器：左神风格的验证方法

写完排序不要靠肉眼检查，用对数器验证：

```python
import random

def insertion_sort(nums):
    for i in range(1, len(nums)):
        key = nums[i]
        j = i - 1
        while j >= 0 and nums[j] > key:
            nums[j + 1] = nums[j]
            j -= 1
        nums[j + 1] = key
    return nums

def checker(test_times=5000, max_n=100, max_val=1000):
    """对数器：跑 test_times 次，每次随机数据对比。"""
    for _ in range(test_times):
        n = random.randint(0, max_n)         # 随机长度（含 0 和 1）
        arr1 = [random.randint(0, max_val) for _ in range(n)]
        arr2 = arr1[:]
        insertion_sort(arr1)
        arr2.sort()
        if arr1 != arr2:
            print(f"出错！输入: {arr2}, 你的结果: {arr1}")
            return False
    print("对数器验证通过！")
    return True

checker()
```

**对数器的价值**：覆盖边界（空数组、单元素、全重复值）、不依赖人工、跑 5000 次只要几秒。左神原话："对数器是算法工程师的基本功。"

### 3. 教科书做法 vs 生产做法

| 场景 | 教科书做法 | 生产做法 | 原因 |
|---|---|---|---|
| 通用排序 | 手写冒泡/选择/插入 | `sorted()`（Timsort） | Timsort 内部对小数组用插入，对大数组用归并 |
| 小数据（n<64） | 手写插入 | `sorted()`（自动切到插入） | 插入在小数据上常数最小 |
| 基本有序数据 | 手写插入 | `sorted()`（Timsort 自适应） | Timsort 识别有序 run，接近 $O(n)$ |
| 大记录少交换 | 手写选择 | 视场景定制 | 选择排序交换最少 |

**生产里几乎不手写简单排序**，但理解它们的价值在于：

1. **理解 Timsort 的底层**：Timsort 是"归并 + 插入"的混合，对小子数组用插入。不懂插入就不懂 Timsort 为什么快
2. **小数据场景**：n<50 时插入排序比快排快（快排的递归开销在小数据上不划算）。很多库的 `sorted()` 内部对小数组切到插入
3. **嵌入式/资源受限场景**：插入排序代码极简、原地、稳定，适合资源受限环境

### 4. 三个真实工程坑

**坑一：在已有序数据上跑冒泡，没加 `swapped` 优化。**

```python
# 没优化的冒泡，对已有序数据仍是 O(n^2)
def bubble_bad(nums):
    n = len(nums)
    for i in range(n - 1):
        for j in range(n - 1 - i):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]
    return nums

# 优化版：加了 swapped，对有序数据 O(n)
def bubble_good(nums):
    n = len(nums)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]
                swapped = True
        if not swapped:
            break
    return nums
```

症状：在"近有序数据"上跑没优化的冒泡，耗时是优化版的几十倍。修复：永远加 `swapped` 优化。

**坑二：用选择排序排对象，丢稳定性。**

```python
students = [("Alice", 90), ("Bob", 90), ("Carol", 85)]
# 用选择排序按分数排
# 第一轮：找到最小 Carol(85)，与 Alice(90) 交换
# 结果：[("Carol", 85), ("Bob", 90), ("Alice", 90)]
# Alice 和 Bob 的相对顺序被破坏了！
```

症状：同分学生的顺序乱了。修复：用稳定排序（插入或 Timsort），或把姓名加进排序键 `key=lambda x: (x[1], x[0])`。

**坑三：用插入排序排大数据，慢到不可接受。**

```python
# 在 10 万元素上用插入排序
import random
data = list(range(100000))
random.shuffle(data)
insertion_sort(data)   # 10^10 次操作，按 10^8/s 算要 100 秒
```

症状：数据量上来后卡死。修复：用 `sorted()`（$O(n\log n)$，10 万数据约 $1.7 \times 10^6$ 次操作，毫秒级）。**简单排序只适合小数据（n<50）或基本有序数据。**

### 5. 面试高频题

**面试题一**："手写插入排序，说明时间复杂度。"

**满分回答**：默写上面的模板，边写边讲：①从第 2 个元素开始（第 1 个天然有序）②key 保存当前元素（防止被覆盖）③从右往左找位置（用 `>` 保证稳定）④后移不是交换（1 次赋值比 3 次快）。复杂度：最好 $O(n)$（已有序），平均/最坏 $O(n^2)$，空间 $O(1)$，稳定。

**面试题二**："插入排序和冒泡排序都是 $O(n^2)$，为什么插入实际更快？"

**满分回答**：两个原因。①操作轻量：插入是"后移"（1 次赋值），冒泡是"交换"（3 次赋值），插入每次操作代价更低。②缓存友好：插入排序内层 while 连续访问相邻元素，CPU 缓存命中率高。加上插入的自适应性更强（基本有序数据内层 while 几乎不执行，接近 $O(n)$），实际比冒泡快 2-3 倍。

**面试题三**："Timsort 为什么对小数据用插入排序？"

**满分回答**：插入排序在小数据（n<64）上常数最小——没有递归开销（快排的递归在小数据上不划算），没有辅助数组操作（归并要 $O(n)$ 辅助数组），且缓存友好。Timsort 对大数组用归并保证 $O(n\log n)$，对子数组切到插入保证小数据的低常数。这是"按数据规模选算法"的工程范例。

## 四、速查与自测

### 速查表：简单排序三兄弟五指标对比

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 | 自适应 | 交换次数 |
|---|---|---|---|---|---|---|---|
| 冒泡 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 是 | 是（加优化） | $O(n^2)$ |
| 选择 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 否 | 否 | $O(n)$ |
| 插入 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 是 | 是（最强） | $O(n^2)$（移动） |

### 自测三问

**问题一：** 冒泡、选择、插入都是 $O(n^2)$，但实际跑起来插入最快，为什么？

**参考答案：** 三个原因。①操作轻量：插入是"移动"（1 次赋值），冒泡是"交换"（3 次赋值），插入每次操作代价更低。②缓存友好：插入排序内层 while 连续访问相邻元素，CPU 缓存命中率高；冒泡也连续但交换多。③自适应性最强：对基本有序数据，插入内层 while 几乎不执行，接近 $O(n)$；冒泡加 `swapped` 优化也能自适应，但常数比插入大。选择排序不自适应，无论数据多有序都要全扫描找最小，所以最慢。

**问题二：** 选择排序不稳定，为什么？请用一个具体例子说明。能不能改成稳定的？

**参考答案：** 不稳定的原因是跨距离交换。例：`[5a, 5b, 3]` 按升序排，第一轮找最小是 3（下标 2），与位置 0 的 5a 交换，得到 `[3, 5b, 5a]`——5a 和 5b 的相对顺序变了。改成稳定的方法：不交换，而是"把最小元素插到前面"（类似插入排序的移动），但这样就不是选择排序了，变成插入排序。选择排序的不稳定是它的本质特性，无法在不改变算法本质的情况下修复。

**问题三：** 生产里几乎不手写简单排序，那为什么还要学？请用"Timsort 的底层"来回答。

**参考答案：** Timsort（Python/Java 默认排序）是"归并 + 插入"的混合算法。它对大数组用归并保证 $O(n\log n)$，对子数组（n<64）切到插入排序——因为插入在小数据上常数最小。不懂插入排序的"小数据优势"和"缓存友好"，就理解不了 Timsort 为什么这样设计。学简单排序不是为了生产手写，是为了理解现代混合排序的底层逻辑。这是"第二层懂原理"的典型例子——表面看生产只用 `sorted()`，但不懂底层就调不好参数、排不了疑难。

### 算法思想 × 生活迁移

简单排序三兄弟的核心思想是"**不同策略消除逆序对**"——同一目标，不同路径，各有适用场景。

**迁移一：整理房间有三种方式，对应三种排序。**

- 冒泡式：每次看到相邻两件东西顺序不对就调一下。效率低，但持续进行。
- 选择式：先找出"最该放最前面"的一件（比如最重要的），放好；再找次重要的。适合"按重要性整理"。
- 插入式：拿一件新东西，找到它在现有顺序里的位置插进去。适合"增量维护有序状态"——比如书架已经排好，新买一本书找位置插入，比推倒重来快得多。

**迁移二：增量维护 vs 推倒重来。** 插入排序的智慧在于"利用已有顺序"。生活中很多场景是"已有基础，做增量"：团队已有规范，新人加入是"插入"；文档已有结构，新增章节是"插入"。这种场景下，插入排序的 $O(n)$（增量插入到已有序部分）远比推倒重排 $O(n\log n)$ 高效。**判断场景是"批量重建"还是"增量维护"，决定该用归并还是插入——这个判断力，是算法思维向项目管理的迁移。**

**迁移三：选策略看"操作代价"。** 选择排序的"最少交换"特性，在"交换代价高"的场景反而最优。生活里：搬家时（交换代价高——搬家具很累），先选最重要的搬（选择式），比两两比较着搬（冒泡式）高效。判断"哪种操作代价高"，决定用哪种策略——这是"按操作代价选算法"的迁移。

**为什么这些迁移成立：** 排序的本质是"把无序变有序"，这个目标在生活中无处不在（整理、规划、归档）。三种简单排序对应三种"从无序到有序"的策略，每种策略的适用条件在生活里同样成立。迁移成立的关键是"识别场景特征"——是增量还是批量、是数据小还是大、是否需要稳定——识别对了，自然选对策略。
