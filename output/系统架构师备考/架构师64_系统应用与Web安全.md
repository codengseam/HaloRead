# 第64章：系统应用与Web安全

系统加固是中级信安工程师案例题的高频考点——给你一台 Windows/Linux 服务器或一个 Web 应用，让你列出加固清单。这章在信安工程师里占 8-12 分，考察操作系统、数据库、Web 应用、终端、邮件五块的安全配置。对架构师来说，这章的价值是把"安全配置项"变成"系统设计规范"：知道 Linux 用 shadow 存密码、知道 Cookie 三属性怎么配、知道零信任终端准入怎么搭，方案才经得起攻防演练。这章把加固清单一次给齐，配 OWASP Top 10 详解和零信任终端架构实战。

## 原理

### 【中级重点】操作系统安全加固

**Windows 加固**（本地安全策略五项）：

| 策略 | 内容 |
|---|---|
| **账号策略** | 密码复杂度、长度、历史、最长期限、账户锁定阈值 |
| **审核策略** | 登录/对象访问/特权使用/账户管理审核 |
| **用户权限分配** | 谁能本地登录、网络登录、关机、改系统时间 |
| **安全选项** | 限制匿名访问、不显示上次用户名、禁止 SMB v1 |
| **事件日志** | 日志大小、保留期、权限 |

**Linux 加固**：

- **密码存储**：`/etc/passwd` 存用户信息（公开可读），密码哈希存 `/etc/shadow`（仅 root 可读），这是 Linux 安全设计的关键。
- **PAM**（Pluggable Authentication Modules，可插拔认证模块）：统一认证框架，配置密码复杂度、登录失败锁定等。
- **MAC 强制访问控制**：SELinux（基于类型强制 Type Enforcement）/AppArmor（基于路径），对进程权限细粒度控制，默认拒绝。
- **文件权限**：rwx 三组（属主/组/其他），用 `chmod`/`chown` 管理；特殊权限位 SUID/SGID 慎用。
- **sudo**：最小授权，配置 `/etc/sudoers` 限定可执行命令，避免直接用 root。
- **chroot**：改变进程根目录，隔离文件系统访问。

口诀：Linux 加固 = **"shadow 存密码、PAM 管认证、SELinux 管权限、sudo 限提权、日志留痕"**。

### 【中级重点】数据库安全

| 措施 | 说明 |
|---|---|
| **账号管理** | 删默认账号（sa/root）、最小权限、强密码 |
| **权限授权** | GRANT/REVOKE 精细授权，避免 ALL PRIVILEGES |
| **视图隔离** | 用视图限制可见列/行，行级安全 |
| **审计日志** | 记录敏感操作（DDL/DML/登录） |
| **透明数据加密 TDE** | 存储层加密，对应用透明 |
| **备份恢复** | 定期全量+增量，异地备份 |

不同数据库特点：SQL Server 用 TDE 和审计功能强；Oracle 有 VPD（虚拟私有数据库）行级安全；MySQL 开源轻量，需注意默认配置（root 远程登录、test 库）。TDE（Transparent Data Encryption）加密数据文件和日志，密钥独立管理，防物理盘丢失泄露。

### 【中级重点】Web 安全与 OWASP Top 10

**OWASP Top 10（2021 版）**：

| 排名 | 风险 | 应对 |
|---|---|---|
| A01 | 失效访问控制 | 服务端鉴权、最小权限、默认拒绝 |
| A02 | 加密失败 | 强算法、传输加密、密钥管理 |
| A03 | 注入 | 参数化查询、输入校验 |
| A04 | 不安全设计 | 威胁建模、安全设计模式 |
| A05 | 安全配置错误 | 最小配置、默认安全、补丁 |
| A06 | 易受攻击组件 | 依赖扫描、版本管理 |
| A07 | 认证失败 | 强密码、MFA、会话管理 |
| A08 | 软件数据完整性失败 | 签名验证、CI/CD 完整性 |
| A09 | 日志监控不足 | 记录关键事件、告警、SIEM |
| A10 | SSRF | URL 白名单、禁内网访问 |

**同源策略（Same-Origin Policy）**：浏览器只允许同协议+同域名+同端口的脚本互相访问，是 Web 安全基石。跨域用 CORS（跨域资源共享）、CSP（内容安全策略）控制。

**Cookie 安全三属性**：

| 属性 | 作用 |
|---|---|
| **HttpOnly** | 禁止 JS 读取 Cookie，防 XSS 窃取 |
| **Secure** | 仅 HTTPS 传输，防明文嗅探 |
| **SameSite** | 限制跨站发送（Strict/Lax/None），防 CSRF |

**CSRF Token**：服务端生成一次性令牌，表单提交时校验，攻击者无法预测 token 即无法伪造请求。

**CSP（Content Security Policy）**：通过 HTTP 头声明允许加载的资源来源，限制内联脚本执行，是 XSS 纵深防御手段。

### 【中级重点】终端安全

| 方案 | 全称 | 特点 |
|---|---|---|
| **杀毒软件** | — | 特征码+启发式+行为分析，依赖病毒库 |
| **HIDS** | 主机入侵检测 | 监控主机行为，告警为主 |
| **EDR** | 端点检测与响应 | 行为分析+响应+溯源，比杀毒增强 |

EDR（Endpoint Detection and Response）相比传统杀毒：从"查杀已知病毒"升级到"行为分析+威胁狩猎+自动响应+攻击溯源"，能发现未知威胁（如文件less攻击、内存攻击）。现代终端安全首选 EDR。

### 【中级重点】邮件安全三件套

| 机制 | 全称 | 作用 |
|---|---|---|
| **SPF** | 发件人策略框架 | DNS 声明哪些 IP 可发本域邮件，防发件人伪造 |
| **DKIM** | 域名密钥识别邮件 | 邮件签名，收件方验签，防篡改 |
| **DMARC** | 基于域的消息认证报告与一致性 | 结合 SPF+DKIM，定义失败处理策略+报告 |

三者配合：SPF 防伪造发件域名，DKIM 防邮件篡改，DMARC 统一策略和报告。企业邮箱防钓鱼必配。

## 实践

### 【高级衔接】零信任终端准入架构

传统安全假设"内网可信"，终端一旦接入内网就放行——这就是 2017 年 WannaCry 在内网横扫的根因。零信任反过来：**任何终端接入都要验证设备合规性+身份认证+最小权限**。

**零信任终端准入架构四要素**：

1. **设备合规性检查**：接入前检查是否装 EDR/补丁是否最新/是否越狱/磁盘是否加密，不合规进隔离修复区。
2. **身份认证**：用户身份（MFA 多因素）+设备身份（设备证书），双因子校验。
3. **最小权限授权**：基于身份+设备状态+上下文动态授权，按应用粒度而非网络粒度开放。
4. **持续认证**：不一次性信任，会话期间持续监测行为，异常即降权或断开。

**架构对比**：

| 维度 | 传统 VPN | 零信任 |
|---|---|---|
| 信任模型 | 接入即信任 | 持续验证 |
| 访问粒度 | 网络层（整网） | 应用层（单应用） |
| 隐患 | 横向移动风险 | 最小权限收敛 |
| 终端要求 | 弱 | 强（合规检查） |

落地建议：终端装 EDR+合规检查代理，身份用 SSO+MFA，访问控制用 SDP（软件定义边界）或 ZTNA（零信任网络访问），后端微服务间用 mTLS。设备不合规一律隔离，不进生产网。

### 真题示例

**2024 年·案例分析·系统加固(考生回忆)**：某 Linux Web 服务器被发现 `/etc/passwd` 中多个用户密码字段非空，且 root 允许 SSH 远程登录。给出至少 5 条加固建议。

**参考答案**：①密码哈希迁移到 `/etc/shadow`，确保 passwd 中密码字段为 `x`；②禁止 root 直接 SSH 登录（`PermitRootLogin no`），用普通用户+sudo；③配置 PAM 强制密码复杂度和登录失败锁定；④启用 SELinux 强制访问控制；⑤关闭无用服务、最小化安装、定期补丁；⑥配置审计日志（auditd）记录特权操作；⑦SSH 改用密钥登录禁用密码。

## 速查/自测

### 30 秒速查卡

- **Linux 密码**：passwd 存信息、shadow 存哈希（关键设计）
- **Linux MAC**：SELinux（类型强制）/AppArmor（路径）
- **PAM**：统一认证模块，管密码复杂度+锁定
- **数据库**：GRANT/REVOKE 精细授权、TDE 透明加密、视图隔离
- **OWASP 2021 前 3**：失效访问控制/加密失败/注入
- **同源策略**：协议+域名+端口三者全同
- **Cookie 三属性**：HttpOnly（防 XSS 窃取）/Secure（仅 HTTPS）/SameSite（防 CSRF）
- **CSRF 防御**：Token + SameSite Cookie
- **CSP**：内容安全策略，限制资源来源，防 XSS
- **终端**：EDR > HIDS > 杀毒（行为分析+响应+溯源）
- **邮件**：SPF（防伪造域名）+DKIM（防篡改）+DMARC（统一策略）

### Windows/Linux 加固清单

| 类别 | Windows | Linux |
|---|---|---|
| 账号 | 删默认账号、改管理员名 | 删多余账号、禁 root 远程登录 |
| 密码 | 复杂度+锁定策略 | PAM 配置复杂度+锁定 |
| 权限 | 用户权限分配最小化 | sudo 最小授权、SELinux/AppArmor |
| 审计 | 审核策略+事件日志 | auditd+syslog |
| 服务 | 关无用服务、禁 SMBv1 | 最小化安装、关无用端口 |
