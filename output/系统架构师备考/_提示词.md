## 三、写作硬约束（写入每个写作 subagent 的 query）

1. 目录：/workspace/output/系统架构师备考/
2. 文件命名：架构师XX_章节名.md（XX 两位数前导零，章节名 4-12 字中文）
3. frontmatter 七字段：title / book / chapter / event / sort / chapter_sort / created_at / source_agents
4. **chapter_sort 用纯整数（禁止前导零！YAML 八进制陷阱：`010`→8、`017`→15）**——前导零只允许出现在文件名/章节字符串 `架构师01`

6. 单章 4 段结构：`## 概念` / `## 原理` / `## 实践` / `## 速查/自测`（自定义内容嵌入 4 段内，避免标题命中灵魂扣分）

8. 真题题号沉文末「## 参考来源」，正文行内出处 ≤3/千字

## 八、关键教训

1. **YAML 八进制陷阱**：frontmatter 的 `chapter_sort` 若写 `010` 会被 PyYAML 解析为八进制 = 8，导致站点章节顺序静默错乱（check_book_structure 不报错，因为都是合法 int）。必须用纯整数 1..55。前导零只允许出现在文件名/章节字符串 `架构师01`（字符串排序安全）。
2. **「参考来源」标题字面匹配**：check_sources_section 用 `##?\s*参考来源` 正则匹配，`## 参考` 不匹配会报「缺参考来源章节」并触发灵魂维度扣分。必须用 `## 参考来源`。
3. **knowledge 桶 4 段结构**：自定义单章结构（这章考什么 / 对比辨析 / 真题示例 / 速查卡）若直接用为 `##` 标题，会命中灵魂维度扣分（如「30 秒速查卡」含数字+量词坏模式）。必须对齐 `## 概念 / ## 原理 / ## 实践 / ## 速查自测` 4 段，自定义内容嵌入段内。
4. **2024 机考后真题题号失真**：2024 起软考机考题库随机抽题，不同考生题号不同，禁用「第 32 题」精确题号，改「2024 年·综合知识·XX 相关(考生回忆)」。
