# 测试用例设计，会画流程图就能拿分

软件测试是综合知识稳定出 2-3 题的考点，案例偶尔考用例设计。把白盒覆盖层级、黑盒等价类边界值答题步骤背熟，几乎不用动脑就能拿分。

## 概念

这章考什么：
- **科目**：综合知识（2-3 题），案例分析（用例设计可能考）
- **分值**：综合约 3 分，案例可能 8-10 分
- **考频**：中高频

核心类比：测试像验房。白盒测试是"打开墙看管线走对没"（看代码内部逻辑）；黑盒测试是"不管墙里啥样，只看水龙头出水对不对"（只看输入输出）；灰盒是"既看龙头又看水表"（介于两者）。

## 原理

### 一、测试分类（两大维度）

**按阶段分**：
- **单元测试**：测函数/模块，开发自测，白盒为主，用驱动模块和桩模块
- **集成测试**：测模块间接口，分自顶向下（用桩）/自底向上（用驱动）/大爆炸（一次性集成）/三明治
- **系统测试**：测整个系统，黑盒为主，含功能/性能/安全/恢复
- **验收测试**：用户参与，含 α 测试（开发者环境）/β 测试（用户环境）

**按可见性分**：
- **白盒测试**：基于代码结构，关注覆盖
- **黑盒测试**：基于功能规格，关注输入输出
- **灰盒测试**：兼顾两者

### 二、白盒测试覆盖层级（必背，由弱到强）

| 层级 | 含义 | 强度 |
|---|---|---|
| 语句覆盖 | 每条语句执行一次 | 最弱 |
| 判定覆盖（分支覆盖） | 每个判定真/假各取一次 | 弱 |
| 条件覆盖 | 每个条件真/假各取一次 | 弱 |
| 判定/条件覆盖 | 同时满足判定 + 条件 | 中 |
| 条件组合覆盖 | 每个判定中条件所有组合 | 强 |
| 路径覆盖 | 所有执行路径 | 最强 |
| **MC/DC 修正条件判定覆盖** | 每个条件独立影响判定结果 | 航空级标准 |

强度排序：路径 > 条件组合 > MC/DC > 判定/条件 > 判定 ≈ 条件 > 语句。

**MC/DC 难点**：要求每个条件都能独立改变判定结果。例 `A and (B or C)`，固定其他条件，让 A 单独翻转判定结果。这是航空电子 DO-178C 强制要求。

### 三、黑盒测试方法

- **等价类划分**：把输入域分为有效等价类（合理输入）和无效等价类（非法输入），每类取一个代表。先划分再取值。
- **边界值分析**：取边界及附近值。规则：min、min+1、nominal、max-1、max。错误往往在边界。
- **因果图法**：复杂条件组合，画因果图→判定表→生成用例。适合多输入多输出。
- **错误推测法**：凭经验猜易错点。
- **场景法**：用基本流 + 备选流构造场景。

**答题顺序**：等价类划分→边界值补充→因果图覆盖组合。

### 四、自动化测试与 TDD

- **自动化测试**：用 Selenium / JUnit / PyTest 等工具执行脚本，适合回归测试。
- **测试驱动开发 TDD（Test-Driven Development）**：红-绿-重构循环——先写失败的测试→写最少代码让测试通过→重构。XP 的核心实践之一。

易混点：
- **α 测试 vs β 测试**：α 在开发者环境、开发者陪同；β 在用户实际环境、开发者不在场。
- **桩模块 vs 驱动模块**：桩模拟被调用的下层模块（自顶向下）；驱动模拟调用上层模块（自底向上）。
- **回归测试 vs 验收测试**：回归是改完代码重测旧功能；验收是交付前用户确认。

## 实践

**真题示例（2024 年·综合知识·白盒测试相关·考生回忆）**：给一段含 if-else 嵌套的代码，问要达到某覆盖层级至少需要多少测试用例。答案要数判定数、条件数、路径数。

**白盒测试用例设计答题步骤**：
1. **画控制流图**：每个判定节点一个圆圈，编号
2. **圈复杂度 V(G)**：V(G) = 边数 - 节点数 + 2，或 V(G) = 判定节点数 + 1。V(G) 决定独立路径数
3. **列独立路径**：从入口到出口的路径，每条至少含一条新边
4. **设计用例**：每条路径一个用例，输入数据让代码走该路径
5. **验证覆盖层级**：检查是否覆盖到目标层级

**黑盒答题步骤**：
1. 找输入变量及其取值范围
2. 划分等价类（有效 + 无效）
3. 每个等价类取一个代表值
4. 边界值补充（min、min+1、max-1、max）
5. 列出测试用例表

**论文应用**：论文"质量保证"段写"本项目采用 V 模型开发，单元测试覆盖率达 85%，关键模块采用 MC/DC 覆盖，集成测试自底向上，系统测试含压力测试和安全性测试，验收测试由用户在准生产环境执行"，体现测试体系完整性。

## 速查/自测

**30 秒速查卡**：

| 维度 | 关键词 |
|---|---|
| 单元测试 | 函数级、白盒、桩/驱动 |
| 集成测试 | 接口、自顶向下用桩 |
| 系统测试 | 整体、黑盒、压力 |
| 验收测试 | 用户、α/β |
| 语句覆盖 | 最弱 |
| 路径覆盖 | 最强 |
| MC/DC | 航空级 |
| 等价类 | 有效+无效 |
| 边界值 | min/min+1/max-1/max |

**白盒覆盖口诀**：语判条组路，由弱到强数；MC/DC 单独记，航空级标准。

**自测题**：
1. 圈复杂度 V(G) = 4，至少需要多少独立路径测试用例？→ 4 个
2. 自顶向下集成测试用什么模拟下层模块？→ 桩模块
3. 边界值"min, min+1, nominal, max-1, max"中 nominal 是什么？→ 正常中间值
4. TDD 三步循环是什么？→ 红（写失败测试）→绿（写最少代码通过）→重构

**一句话小结**：白盒覆盖层级 + 黑盒等价类边界值——两个答题步骤背熟，案例综合都不丢分。
