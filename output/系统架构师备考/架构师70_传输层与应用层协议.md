## 概念

### 这章考什么

- **科目**：中级网络工程师综合知识、案例分析
- **考频**：核心高频，TCP 握手和 DNS/HTTP 几乎每年必考
- **分值**：中级约 10-15 分
- **难度**：中等，TCP 状态机和拥塞控制是难点

传输层把端到端通信封装成可编程的端口和连接，应用层协议是业务落地的载体。HTTP 和 DNS 是互联网的两大基石，TCP 三次握手四次挥手是面试和考试的双高考点。

## 原理

### 【中级重点】传输层功能

传输层在端到端基础上做四件事：复用分用（多应用共享一条连接）、差错控制、流量控制（接收方限速）、拥塞控制（网络拥塞时降速）。TCP 提供可靠面向连接，UDP 提供不可靠无连接。

### 【中级重点】端口

| 范围 | 类型 | 示例 |
|---|---|---|
| 0-1023 | 知名端口 | HTTP 80、HTTPS 443 |
| 1024-49151 | 注册端口 | MySQL 3306 |
| 49152-65535 | 动态端口 | 客户端临时端口 |

知名端口速记：FTP 数据 20 控制 21、SSH 22、Telnet 23、SMTP 25、DNS 53、DHCP 67/68、TFTP 69、HTTP 80、POP3 110、NNTP 119、NTP 123、IMAP 143、SNMP 161、HTTPS 443、RDP 3389。

### 【中级重点】TCP 传输控制协议

#### 三次握手

建立连接的三步：

1. 客户端发 SYN，序号 seq=x，状态 CLOSED → SYN_SENT
2. 服务端回 SYN+ACK，seq=y、ack=x+1，状态 LISTEN → SYN_RCVD
3. 客户端发 ACK，ack=y+1，状态 SYN_SENT → ESTABLISHED，服务端收到后 SYN_RCVD → ESTABLISHED

为什么三次不是两次？防止历史连接请求被服务端接受导致资源浪费。SYN_RCVD 状态等的就是这第三个 ACK 的确认。

#### 四次挥手

1. 主动方发 FIN，seq=u，状态 ESTABLISHED → FIN_WAIT_1
2. 被动方回 ACK，ack=u+1，状态 ESTABLISHED → CLOSE_WAIT，主动方 FIN_WAIT_1 → FIN_WAIT_2
3. 被动方发 FIN，seq=v，状态 CLOSE_WAIT → LAST_ACK
4. 主动方回 ACK，ack=v+1，状态 FIN_WAIT_2 → TIME_WAIT，被动方 LAST_ACK → CLOSED
5. 主动方等 2MSL 后 TIME_WAIT → CLOSED

TIME_WAIT 等 2MSL（最大段寿命，默认 2 分钟）原因：保证最后一个 ACK 到达对方，防止旧连接报文干扰新连接。服务端大量 CLOSE_WAIT 通常是应用没主动 close，是排查重点。

#### 流量控制

滑动窗口：接收方在 ACK 里带窗口大小 WIN，告诉发送方还能收多少字节。窗口为 0 时发送方暂停，靠窗口探测恢复。

#### 拥塞控制

四个算法：

- **慢启动 Slow Start**：cwnd 从 1 指数增长（1→2→4→8…），到 ssthresh 转线性
- **拥塞避免 Congestion Avoidance**：cwnd 线性增长，每 RTT 加 1
- **快重传 Fast Retransmit**：连续收到 3 个重复 ACK 立即重传，不等超时
- **快恢复 Fast Recovery**：ssthresh=cwnd/2，cwnd=ssthresh，直接进入拥塞避免

流量控制是接收方限制发送方，拥塞控制是发送方感知网络限制自己，两者方向不同。

#### 可靠传输

序号、确认、超时重传三件套。超时时间 RTO 根据 RTT 动态计算。

### 【中级重点】UDP

UDP 用户数据报协议 User Datagram Protocol：无连接、不可靠、首部仅 8 字节（源端口、目的端口、长度、校验和）、不拥塞控制、适合实时音视频、DNS、DHCP、SNMP。

### 【中级重点】DNS 域名系统

层次结构：根域名（.）→ 顶级域 TLD（com、cn、org）→ 二级域（example）→ 子域。根服务器全球 13 组 IP，靠 anycast 扩展。

解析流程：

- **递归查询**：客户端问本地 DNS，本地 DNS 一路问到底返回结果
- **迭代查询**：本地 DNS 问根，根返回 TLD 地址，本地 DNS 再问 TLD，依次下去

实际中客户端到本地 DNS 是递归，本地 DNS 到各级是迭代。

记录类型：

| 类型 | 含义 |
|---|---|
| A | 域名→IPv4 |
| AAAA | 域名→IPv6 |
| CNAME | 别名 |
| MX | 邮件交换 |
| NS | 域名服务器 |
| TXT | 文本记录 |
| PTR | IP→域名 反向 |
| SOA | 起始授权 |
| SRV | 服务记录 |

DNS 同时用 UDP 53 和 TCP 53：默认 UDP，响应超 512 字节或区域传送用 TCP。

### 【中级重点】DHCP

DORA 四步：

1. **Discover**：客户端广播找服务器
2. **Offer**：服务器广播回应分配的 IP
3. **Request**：客户端广播确认接受（让其他服务器收回 Offer）
4. **Acknowledge**：服务器确认，含租约、网关、DNS

租约默认 8 天，到 50% 续租，到 87.5% 广播找其他服务器。跨网段用 DHCP 中继代理（路由器配 ip helper-address）。

### 【中级重点】HTTP

请求方法：GET 取、POST 提交、PUT 全量更新、PATCH 部分更新、DELETE 删、HEAD 取头、OPTIONS 探测支持的方法。

状态码：

| 类别 | 含义 | 示例 |
|---|---|---|
| 1xx | 信息 | 100 Continue |
| 2xx | 成功 | 200 OK |
| 3xx | 重定向 | 301 永久、302 临时、304 缓存 |
| 4xx | 客户端错 | 400 参数、401 未认证、403 禁止、404 找不到 |
| 5xx | 服务端错 | 500 内部错、502 网关、503 不可用、504 网关超时 |

版本演进：

| 版本 | 关键特性 |
|---|---|
| 1.0 | 短连接，每请求一连接 |
| 1.1 | 长连接 Keep-Alive、管道化、Host 头 |
| 2.0 | 二进制分帧、多路复用、HPACK 头压缩、服务器推送 |
| 3.0 | 基于 QUIC（UDP），0-RTT 握手，抗丢包 |

### 【中级重点】HTTPS 与 TLS

HTTPS = HTTP over TLS。TLS 握手简化版：

1. ClientHello：客户端发支持的密码套件、随机数
2. ServerHello：服务端选定套件、随机数、证书
3. 客户端验证证书链（CA → 中间 CA → 服务器证书）
4. 密钥交换生成 pre-master，导出会话密钥
5. 双方用对称密钥加密会话

TLS 1.3 把握手压到 1-RTT，0-RTT 模式更快但有重放风险。

### 【中级重点】邮件协议

- SMTP 发邮件 25/465（SSL）/587（STARTTLS）
- POP3 下载到本地 110/995（SSL），默认下载后删服务器
- IMAP 在线浏览 143/993（SSL），服务器保留

### 【中级重点】文件传输协议

- FTP 控制 21、数据 20（主动）、被动模式数据端口随机
- TFTP UDP 69，简单无认证，用于固件升级
- SSH 22，SFTP/SCP 基于 SSH
- Telnet 23，明文不安全，已淘汰

### 【中级重点】SNMP

Manager/Agent 架构，MIB 管理信息库存被管对象，OID 树状定位。Get 取、Set 设、Trap 主动告警。

## 实践

### 【高级衔接】高并发服务架构

TCP 连接数优化在高并发场景关键。每条连接占内存（内核 socket buffer 几 KB 到几十 KB），百万并发需调内核参数：fs.file-max 增文件句柄、net.ipv4.ip_local_port_range 扩客户端端口、net.core.somaxconn 增监听队列、tcp_tw_reuse 复用 TIME_WAIT。

长连接管理是 RPC 框架核心。短连接频繁握手浪费 RTT，长连接复用降低延迟，但需心跳保活和连接池管理。gRPC 基于 HTTP/2 多路复用，一条连接跑多请求，对架构影响是连接数不再等于并发数，单连接吞吐大幅提升。

**真题示例（2023年·综合知识·TCP握手相关·考生回忆）**：客户端发 SYN 后等待什么状态？答：SYN_SENT。服务端收到 SYN 后回什么、进入什么状态？答：回 SYN+ACK，进入 SYN_RCVD。

### 自测题

1. TCP 四次挥手后主动方进入什么状态等多久？答：TIME_WAIT 等 2MSL。
2. DNS 区域传送用哪个端口协议？答：TCP 53。
3. HTTP 2.0 用什么协议头压缩？答：HPACK。

一句话小结：TCP 三次握手四次挥手状态机必背，DNS 递归迭代要分清，HTTP 版本演进看多路复用。
