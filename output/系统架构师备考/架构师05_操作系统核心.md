# 第5章：进程、内存与死锁，操作系统的三块硬骨头

操作系统是架构师的"内功"，进程怎么调度、内存怎么分页、死锁怎么破，这些直接决定你设计的系统能不能扛住并发。

## 概念

综合知识里 OS 是大头，**3-4 题/年**，单题 1 分。更关键的是 **案例分析常考 PV 操作和银行家算法**，论文里讲并发/性能也绕不开死锁和内存管理。

核心类比：进程是工厂里的一个"工单"，线程是工单里干活的"工人"。一个工单可以有多个工人并行干活（多线程），但工人共享工单的工位（共享内存）。死锁就是几个工单互相等对方手里的工具，谁都不撒手，整个车间卡死。

## 原理

### 进程管理

进程三态：就绪（等 CPU）→ 运行（占 CPU）→ 阻塞（等资源）。三态转换：
- 就绪 → 运行：调度选中
- 运行 → 就绪：时间片到
- 运行 → 阻塞：等 I/O 或资源
- 阻塞 → 就绪：资源到位

注意：**没有"阻塞→运行"直接转换**，这是高频陷阱题。

线程 vs 进程：进程是资源分配单位，线程是 CPU 调度单位。线程共享进程的内存空间，切换开销小。

进程间通信（IPC，Inter-Process Communication）：管道、消息队列、共享内存、信号量、socket。

### PV 操作（重点中的重点）

P 操作（wait）：申请资源，S = S - 1，若 S < 0 则阻塞。
V 操作（signal）：释放资源，S = S + 1，若 S ≤ 0 则唤醒一个等待进程。

S 的含义：S > 0 表示可用资源数；S < 0 表示等待队列中的进程数。

经典模型：生产者-消费者。设 empty = N（空位）、full = 0（产品）、mutex = 1（互斥）。
- 生产者：P(empty) → P(mutex) → 放产品 → V(mutex) → V(full)
- 消费者：P(full) → P(mutex) → 取产品 → V(mutex) → V(empty)

易错：**P(empty) 必须在 P(mutex) 之前**，否则死锁。mutex 永远在最内层。

### 死锁

四条件（缺一不可）：互斥、请求保持、不剥夺、循环等待。

四种对策：
- **预防**：破坏某一条件。如一次性申请所有资源（破坏请求保持）、资源有序分配（破坏循环等待）。
- **避免**：银行家算法，分配前先算会不会进不安全状态。
- **检测**：资源分配图，找环。
- **解除**：剥夺资源或终止进程。

**银行家算法**：先算"需求矩阵 Need = Max - Allocation"，再试分配，然后算安全性序列（找一个进程其 Need ≤ Available，回收它的 Allocation，重复）。能找到完整安全序列就安全，否则拒绝分配。

### 存储管理

分页：物理地址 = 块号 × 块大小 + 偏移；逻辑地址 = 页号 × 页大小 + 偏移。页表做映射。优点无外碎片，有内碎片（最后一页填不满）。

分段：按程序逻辑分（代码段、数据段、栈段），便于共享保护，有外碎片。

段页式：先分段再分页，结合两者优点，最常用。

页面置换（必考）：
- **OPT（最佳置换）**：淘汰未来最久不用的，理论最优不可实现，作基准。
- **FIFO（先进先出）**：淘汰最先进来的，有 Belady 异常（页框增多缺页反增）。
- **LRU（最近最少用）**：淘汰最久没访问的，性能接近 OPT，最常用。

| 算法 | 实现 | Belady 异常 |
|---|---|---|
| OPT | 不可实现 | 无 |
| FIFO | 队列 | 有 |
| LRU | 时间栈/计数器 | 无 |

### 文件管理

目录结构：单级、两级、树形、无环图（共享）。FAT（File Allocation Table，文件分配表）用链式存储，inode（索引节点）用多级索引。

inode 存文件元数据（大小、权限、数据块指针），不存文件名。文件名在目录项里。

## 实践

**真题示例（2024年·综合知识·PV操作相关，考生回忆）**：某系统有 3 个生产者、2 个消费者，缓冲区容量 5，写出 PV 操作。

答题公式：先定义信号量（empty=5, full=0, mutex=1），再套生产者-消费者模板，强调 P(empty) 在 P(mutex) 前。踩分点：信号量初值、P/V 顺序、是否互斥保护。

**银行家算法案例**：给 Max、Allocation、Available 三张表，问能否分配。三步：①算 Need；②试分配后算 Available；③求安全序列，写出进程执行顺序即得分。

**论文应用**：讲"高并发系统设计"时，用 PV 操作思想说明资源池限流；讲"分布式锁"时类比互斥信号量。提到 LRU 替换能体现缓存设计功底。

**踩分词**：PV 操作、信号量初值、安全序列、Belady 异常、Need=Max-Allocation、局部性。

## 速查/自测

**30 秒速查卡**

| 考点 | 一句话 |
|---|---|
| 进程三态 | 就绪/运行/阻塞，无阻塞→运行 |
| P/V 操作 | P 减 1 阻塞，V 加 1 唤醒 |
| 死锁四条件 | 互斥/请求保持/不剥夺/循环等待 |
| 银行家 | Need=Max-Alloc，找安全序列 |
| FIFO | 有 Belady 异常 |
| LRU | 最常用，无 Belady |

**口诀**：P 减 V 加记牢，先申请资源再上锁，否则死锁跑不了；银行家先算 Need，安全序列找到了才放行。

**自测题**：
1. 进程从"阻塞"能否直接到"运行"？答：不能，必须先回"就绪"。
2. empty=5, mutex=1 的生产者，若写成 P(mutex)→P(empty) 会怎样？答：缓冲区满时生产者占着 mutex 等 empty，消费者拿不到 mutex 取不出产品，**死锁**。
3. FIFO 的 Belady 异常指什么？答：增加页框数，缺页率反而上升。

一句话小结：OS 章节案例题拿分靠"PV 顺序 + 银行家安全序列 + LRU 替换"，把生产者-消费者模板背熟就稳了。
