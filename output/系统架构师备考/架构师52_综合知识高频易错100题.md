# 第52章：综合知识高频易错100题，考前错题本最后一刷

综合知识75题单选，看似一题1分不起眼，但它是三科里最容易"差一两分过线"的科目。这一章把全科目按高频考点分成9大类、100个易错知识点钉死，每个易错点配正确答案和"为什么错"的解析。把它当成错题本来用——考前一周每天过一遍，错的标红，再错的标星。下面每类都按"易错点 → 正解 → 解析"三段式，专治你"看了会、做就错"的毛病。

## 概念

### 这章怎么用

- **不是新教材**：假设你已完成第4到第40章的系统学习，这章是"错题集"性质，只点易错、只讲陷阱，不重复基础知识。
- **建议节奏**：D-7到D-5每天3类，每类30分钟；D-4做整套真题时回头查这100点；D-1把打星的重点再过一遍。
- **标注约定**：🔥=高频陷阱（几乎年年考）、⚠️=易混点（模棱两可）、⭐=送分题（不能丢）。
- **正确做法**：盖住"正解"先自己答，错了再读解析；只看解析等于白看。

### 命题人最爱挖的坑

1. **概念混淆**：流水线周期取最长段，不是平均；FIFO 有 Belady 异常，LRU 没有。
2. **范围边界**：子网划分后全 0 全 1 子网算不算；BCNF 比 3NF 多要求什么。
3. **方法归属**：SAAM、ATAM、CBAM 都来自 SEI，不要记串；螺旋模型是 Boehm 提出、风险驱动。
4. **箭头方向**：UML 类图六种箭头粗细虚实全不同，案例和综合都爱考。
5. **数值计算**：Cache 平均访问时间、McCabe 复杂度、流水线加速比，公式记错直接丢分。

一句话：综合知识 60% 是"背准了就有分"，40% 是"算对了才有分"，先把 60% 的背准分钉死。

## 原理

按 9 大类组织，共 100 个易错知识点。每条只给"易错点—正解—解析"，不展开。

### 一、计算机基础（15个）

**1.🔥 流水线周期**：取最长段，不是平均。流水线周期 = max(取指、分析、执行)。解析：流水线节拍由最慢的那一段决定，木桶效应。

**2.🔥 流水线加速比**：S = 不用流水线时间 / 用流水线时间。不要把分子分母记反。一条指令 k 段各 Δt，n 条指令：顺序时间 = nkΔt，流水时间 = (k+n-1)Δt。

**3. 流水线吞吐率**：TP = n / (k+n-1)Δt，当 n→∞ 时趋近 1/Δt。⚠️ 不要写成 n/(nkΔt)。

**4.🔥 中断向量**：中断服务程序入口地址。中断向量表存的是地址，不是指令。区分"向量地址"和"中断向量"——前者是表项位置，后者是服务程序入口。

**5. 中断响应优先级**：硬件固定，不可改；中断嵌套靠"屏蔽字"软件调整。多重中断必须"开中断"才能嵌套。

**6.🔥 Cache 平均访问时间**：t = h·t1 + (1−h)·t2，h 是命中率。⚠️ 不要漏掉 (1−h) 那一项，也别把 t1（Cache 速度）和 t2（主存速度）搞反。

**7. Cache 映射方式**：直接映射（快、冲突多）、全相联（慢、无冲突）、组相联（折中）。组相联 = 组间直接、组内全相联。

**8. 虚拟存储器**：页式（页固定，无外碎片有内碎片）、段式（段不定，有外碎片）、段页式（先分段再分页，三者优点合并）。

**9. TLB（快表）**：页表的 Cache，存最近访问的页表项。命中率高的 TLB 是"Cache 的 Cache"。

**10.⭐ 进程 vs 线程**：进程是资源分配的基本单位，线程是 CPU 调度的基本单位。线程切换不换地址空间，进程切换要换。

**11. 进程三态**：就绪→运行（调度）、运行→就绪（时间片到）、运行→阻塞（等资源）、阻塞→就绪（资源到位）。⚠️ 没有"阻塞→运行"直接转换。

**12. 进程通信方式**：管道、信号、消息队列、共享内存、信号量、socket。共享内存**最快**（少一次拷贝）。

**13.🔥 死锁四必要条件**：互斥、请求保持、不剥夺、环路等待。四个全满足才死锁，破一个即可预防。

**14. CRC 校验**：用生成多项式做模 2 除法，能检错不能纠错。⚠️ 海明码能纠 1 位错，CRC 只检错。

**15. 浮点数**：阶码决定范围，尾数决定精度。阶码位数越多范围越大，尾数位数越多精度越高。

### 二、操作系统（10个）

**16.🔥 PV 操作**：P 是 wait（P=proberen 荷兰语"测试"，semWait，减 1），V 是 signal（V=verhogen"增加"，semSignal，加 1）。⚠️ 不要按字母含义记，记"减一加一"。

**17. 前驱图转 PV**：每条前驱边对应一个信号量初值 0，前驱做完 V，后继做 P。多对多用多个信号量。

**18.🔥 生产者-消费者**：empty=缓冲容量、full=0、mutex=1。生产者 P(empty) P(mutex)…V(mutex) V(full)；消费者 P(full) P(mutex)…V(mutex) V(empty)。⚠️ P(empty) 和 P(mutex) 顺序不能反，反了死锁。

**19. 死锁处理三策略**：预防（破坏四条件）、避免（银行家，运行时判断）、检测与解除（允许死锁发生再处理）。

**20.🔥 银行家算法**：属于"避免"，不是预防。判断安全状态：找"剩余需求 ≤ 可用资源"的进程，释放它的资源，循环能否全部完成。

**21.🔥 页面置换**：OPT（最优，未来不用的先换出，理论用）、FIFO（先进先出）、LRU（最近最少使用）。性能 OPT > LRU > FIFO。

**22.🔥 Belady 异常**：只有 FIFO 会出现——分配页框增加，缺页率反而上升。LRU 和 OPT 不会。

**23. 磁盘调度算法**：FCFS（先来先服务）、SSTF（最短寻道，可能"饥饿"）、SCAN（电梯算法，到头返回）、C-SCAN（单向循环）。⚠️ LOOK 是 SCAN 不走到头的变种。

**24. 文件系统**：FAT 用链接结构（隐式链接表）、inode 用多级索引（含直接/一级/二级/三级指针）、UFS/NTFS 用索引。

**25. 作业调度**：短作业优先 SJF 平均等待时间最短，但可能"长作业饥饿"；响应比高者优先可解决饥饿。

### 三、数据库（12个）

**26.🔥 范式递进**：1NF（属性原子）→ 2NF（消除部分函数依赖，非主属性不部分依赖候选键）→ 3NF（消除传递依赖）→ BCNF（每个决定因素都是候选键）。

**27.🔥 BCNF vs 3NF**：BCNF 要求"所有决定因素都必须是候选键"，3NF 允许决定因素是候选键或被候选键包含。BCNF 比 3NF 更严。

**28. 函数依赖**：X→Y 表示 X 决定 Y。部分依赖（X 的真子集决定 Y）违反 2NF；传递依赖（X→Y→Z，Y 不决定 X）违反 3NF。

**29.⭐ 事务 ACID**：原子性 Atomicity、一致性 Consistency、隔离性 Isolation、持久性 Durability。⚠️ 一致性靠应用保证，不是 DBMS 直接管。

**30.🔥 隔离级别**：读未提交（脏读）、读已提交（不可重复读）、可重复读（幻读，MySQL 默认）、串行化（无并发问题）。隔离越严并发越低。

**31.🔥 三种读异常**：脏读（读未提交）、不可重复读（同一行两次读不同）、幻读（同一查询两次结果集行数不同）。⚠️ 不可重复读是"行内容变"，幻读是"行数变"。

**32. 锁**：共享锁 S（读锁，可共存）、排他锁 X（写锁，独占）。S+X、X+X 都冲突。两阶段锁 2PL 是可串行化充分条件。

**33.🔥 关系代数符号**：选择 σ（行筛选）、投影 π（列筛选）、连接 ⋈（笛卡尔积 + 条件）、除 ÷（"全包含"查询）。

**34.🔥 SQL：HAVING vs WHERE**：WHERE 在分组前过滤行，HAVING 在分组后过滤组，HAVING 可用聚合函数。

**35. SQL：外连接**：左外连接保留左表全部行、右外保留右表、全外保留两边。匹配不上的填 NULL。

**36. E-R 图三要素**：实体（矩形）、属性（椭圆）、联系（菱形）。联系两端标 1:1 / 1:N / M:N。

**37.🔥 反规范化**：为提高查询性能主动降低范式（加冗余、加派生列、表合并）。代价是更新成本和一致性维护。

### 四、计算机网络（8个）

**38.🔥 TCP 三次握手**：① 客户端 SYN→服务端 ② 服务端 SYN+ACK→客户端 ③ 客户端 ACK→服务端。⚠️ 为什么不是两次？防止已失效的连接请求到达，导致服务端资源浪费。

**39. TCP 四次挥手**：FIN→ACK→FIN→ACK。TIME_WAIT 状态等 2MSL（最长报文寿命），防止最后 ACK 丢失。

**40.🔥 HTTP 状态码**：2xx 成功、3xx 重定向（301 永久 / 302 临时 / 304 缓存未改）、4xx 客户端错（400 参数错 / 403 拒绝 / 404 不存在）、5xx 服务端错（500 内部 / 503 不可用）。

**41.⭐ HTTPS**：HTTP + SSL/TLS，端口 **443**（HTTP 是 80）。HTTPS = HTTP 在传输层加密，靠非对称加密交换对称密钥，对称加密传数据。

**42. DNS 查询**：递归查询（主机问本地 DNS，本地 DNS 全程负责到底）、迭代查询（本地 DNS 依次问根→顶级→权威）。主机到本地是递归，本地 DNS 之后是迭代。

**43.🔥 子网划分**：IP = 网络号 + 子网号 + 主机号。主机号全 0 是网络地址、全 1 是广播地址，可用主机数 = 2^主机位 − 2。⚠️ CIDR 不分 A/B/C 类，全 0 全 1 子网现代可用。

**44. OSI 七层 vs TCP/IP 四层**：OSI = 物理链路网络传输会话表示应用；TCP/IP = 网络接口网络层传输层应用层。HTTP 在应用层，TCP 在传输层，IP 在网络层。

**45. TCP vs UDP**：TCP 面向连接、可靠、有序、全双工、字节流；UDP 无连接、不可靠、报文、开销小。DNS 用 UDP（响应小），HTTP 用 TCP。

### 五、软件工程（10个）

**46.🔥 过程模型**：瀑布（线性、文档驱动）、原型（需求不清先做原型）、增量（分块交付）、螺旋（**Boehm 提出、风险驱动**）、迭代（反复精化）。

**47.🔥 螺旋模型**：每圈四象限——制定计划、风险分析、实施工程、客户评估。最大特点是**风险分析**，适合大型复杂项目。

**48. RUP**：统一过程，用例驱动、以架构为中心、迭代增量。四阶段：初始、细化、构造、移交。⚠️ 细化阶段最关键，确定架构基线。

**49.🔥 测试覆盖**：语句覆盖（最弱）< 判定覆盖 < 条件覆盖 < 判定-条件覆盖 < 条件组合覆盖 < 路径覆盖（最强）。白盒测试用。

**50. 黑盒测试**：等价类划分、边界值分析、因果图、判定表、错误推测。⚠️ 边界值是黑盒最常用，专门盯边界。

**51.🔥 McCabe 复杂度**：V(G) = E − N + 2（边数 − 节点数 + 2），或 V(G) = 判定节点数 + 1。⚠️ 用第二个公式更快。

**52. CMMI 五级**：1 初始（混乱）/ 2 已管理 / 3 已定义 / 4 量化管理 / 5 持续优化。⚠️ 级别名易混，记"混管理定义量化优化"。

**53. 软件度量**：LOC（代码行，简单但语言相关）、FP（功能点，语言无关）。FP = UFC × TCF，更客观。

**54.🔥 软件维护四类**：纠错性（改 bug，占比最大约 20%）、适应性（环境变化）、完善性（新增功能，占比最大约 50%~60%）、预防性（防患未然）。⚠️ 完善性维护占比最高，不是纠错性。

**55. 软件测试 V 模型**：需求分析↔验收测试、概要设计↔系统测试、详细设计↔集成测试、编码↔单元测试。左开发右测试一一对应。

### 六、面向对象与 UML（10个）

**56.🔥 多态**：重载 overload（同名不同参，编译时绑定，静态多态）vs 重写 override（子类覆盖父类同名同参方法，运行时绑定，动态多态）。⚠️ 重载看参数列表，重写看继承关系。

**57. 静态多态 vs 动态多态**：静态 = 重载、模板（编译期决定）、动态 = 虚函数（运行期决定）。多态的核心是"同一消息不同行为"。

**58.🔥 UML 图分类**：静态图（类图、对象图、构件图、部署图、包图）、行为图（用例图、活动图、状态机图）、交互图（序列图、通信图、定时图）。⚠️ 用例图属行为图。

**59.🔥 类图六种关系**：泛化（实线空心三角，继承）、实现（虚线空心三角，接口）、关联（实线，有引用）、依赖（虚线箭头，临时使用）、聚合（实线空心菱形，整体与部分可分离）、组合（实线实心菱形，整体与部分同生命周期）。

**60. 关系强弱**：组合 > 聚合 > 关联 > 依赖。组合最强，部分随整体生灭。

**61.🔥 用例图三关系**：include（包含，A 必做 B）、extend（扩展，A 可选做 B）、泛化（子用例继承父用例）。⚠️ include 是"必选"，extend 是"可选"。

**62. 序列图 vs 通信图**：序列图强调时间顺序（纵轴时间）、通信图强调对象结构关系。两者等价可互转。

**63. 状态图 vs 活动图**：状态图描述对象状态转换（事件驱动）、活动图描述工作流（活动→活动）。活动图类似流程图。

**64. 接口 vs 抽象类**：接口只定义行为契约（Java8 前无实现）、抽象类可有部分实现可含状态。一个类可实现多接口、只能继承一个抽象类。

**65.🔥 SOLID 原则**：S 单一职责、O 开闭（对扩展开放对修改关闭）、L 里氏替换（子类能替父类）、I 接口隔离（接口小而专）、D 依赖倒置（依赖抽象不依赖具体）。

### 七、设计模式（15个）

**66.🔥 23 模式分类**：创建型 5（工厂方法、抽象工厂、建造者、原型、单例）、结构型 7（适配器、桥接、组合、装饰器、外观、享元、代理）、行为型 11（责任链、命令、解释器、迭代器、中介者、备忘录、观察者、状态、策略、模板方法、访问者）。口诀"创5结7行十一"。

**67.🔥 工厂方法 vs 抽象工厂**：工厂方法造"一种"产品（一个工厂一个产品）、抽象工厂造"一族"产品（一个工厂多个相关产品）。⚠️ 抽象工厂用于产品族，不是产品等级。

**68. 单例要点**：私有构造、静态实例、全局访问点。⚠️ 多线程下要双重检查锁 DCL 或静态内部类。

**69. 建造者 vs 工厂**：建造者关注"分步构造复杂对象"（同样部件不同顺序）、工厂关注"造哪种产品"。建造者分离构造与表示。

**70. 原型模式**：用 clone 复制已有对象，避免重复初始化。⚠️ 深拷贝 vs 浅拷贝，深拷贝要复制引用对象。

**71.🔥 适配器 vs 装饰器 vs 外观**：适配器转换接口让不兼容的能协作、装饰器增强不改接口、外观提供简化入口。三者都"包一层"目的不同。

**72.🔥 桥接模式**：分离抽象与实现，两者独立扩展。例"形状×颜色"两个维度用桥接避免子类爆炸。

**73. 组合模式**：树形结构，叶子和容器统一接口。文件系统是典型例子。

**74. 代理模式种类**：远程代理、虚拟代理（延迟加载）、保护代理（权限）、智能引用代理。⚠️ 代理控制访问，装饰器增强行为。

**75. 享元 vs 单例**：享元共享多个细粒度对象（如棋子）、单例保证全局唯一。享元用工厂管理池。

**76.🔥 观察者 vs 发布订阅**：观察者模式主题直接通知观察者（耦合）、发布订阅通过 broker 解耦（完全不知道彼此）。⚠️ 软考答案里"观察者"常指前者。

**77.🔥 策略 vs 状态**：策略模式客户端选算法（主动）、状态模式状态切换自动驱动行为（被动）。两者结构几乎一样，区别在"谁触发切换"。

**78. 模板方法 vs 策略**：模板方法用继承（子类重写步骤）、策略用组合（运行时换算法）。模板方法骨架不变部分可变。

**79. 责任链 vs 命令**：责任链请求沿链传递直到处理、命令把请求封装成对象可排队可撤销。⚠️ 责任链"一个请求一个处理者"，命令"请求当对象"。

**80. 访问者双分派**：访问者用两次分派实现"根据元素类型 + 访问者类型"分发。⚠️ 双分派是访问者核心机制。

### 八、架构风格（10个）

**81.🔥 五大类架构风格**：数据流（批处理、管道-过滤器）、调用返回（主程序-子程序、面向对象、层次）、独立构件（进程通信、事件驱动）、虚拟机（解释器、基于规则的系统）、仓库（数据库、超文本、黑板）。

**82.🔥 管道-过滤器**：数据流风格，过滤器独立、管道单向流动。Unix 命令 `ls | grep | sort` 是典型。⚠️ 增量计算、流式处理选它。

**83. 黑板架构三要素**：知识源（独立模块）、黑板数据（共享数据结构）、控制（调度知识源）。⚠️ 黑板适合"无确定算法"的问题，如语音识别、专家系统。

**84. 解释器风格**：虚拟机类，包含解释器引擎 + 被解释程序 + 状态。⚠️ 解释器适合"频繁变化的语言/规则"。

**85. 事件驱动 / 隐式调用**：独立构件类，构件通过事件订阅而非直接调用。⚠️ 控制反转，发送者不知道谁接收。

**86.🔥 MVC 架构**：Model（数据+业务）、View（展示）、Controller（输入转发）。⚠️ MVC 是组合模式，目的解耦数据、展示、控制。

**87. B/S vs C/S**：C/S 客户端要装、性能好、跨平台差；B/S 浏览器即用、易维护、性能弱。⚠️ 三层 B/S = 表示层 + 功能层 + 数据层。

**88. 富互联网应用 RIA**：在 B/S 上加富客户端能力（如 AJAX、Flex），兼具桌面体验和 Web 部署。

**89.🔥 SOA vs 微服务**：SOA 用 ESB 企业服务总线集中编排、服务粒度大、技术栈统一；微服务去中心化、服务粒度小、技术栈自由、独立部署。⚠️ 微服务是 SOA 的轻量演进。

**90. RESTful API**：资源用 URI、操作用 HTTP 动词（GET/POST/PUT/DELETE）、无状态、可缓存。⚠️ REST 是架构风格不是协议。

### 九、质量属性与评估（10个）

**91.🔥 六大质量属性**：性能、可用性、可靠性、安全性、可修改性、可测试性。常加易用性、功能性凑成"质量属性场景"。

**92.🔥 可靠性 vs 可用性**：可靠性是"多长时间不出错"（MTBF）、可用性是"多长时间能用" = MTBF/(MTBF+MTTR)。⚠️ 可靠性看故障频率，可用性还看修复速度。

**93. 性能战术三类**：资源管理（增加资源、引入并发）、资源仲裁（调度策略）、请求管理（限流、队列）。⚠️ 缓存属于资源管理。

**94.🔥 可用性战术**：错误检测（ping/echo、心跳、异常检测）、错误恢复（表决、冗余、回滚）、错误预防（从服务中删除、事务、进程监视器）。

**95. 安全性战术**：抵抗攻击（身份认证、授权、数据加密、限制暴露）、检测攻击（入侵检测 IDS）、从攻击中恢复（审计跟踪、恢复）。

**96.🔥 ATAM 九步**：① ATAM 方法呈现 ② 商业驱动呈现 ③ 架构呈现 ④ 架构方法识别 ⑤ 效用树生成 ⑥ 基于效用树分析架构方法 ⑦ 头脑风暴场景 ⑧ 基于场景分析架构方法 ⑨ 结果呈现。⚠️ 第 6 步基于效用树、第 8 步基于场景，最易混。

**97.🔥 ATAM 三阶段**：第 1~4 步呈现与调查（评估团队视角）、第 5~9 步分析与评估（含两次场景分析）、报告阶段。⚠️ 不是"三阶段"严格对应步数，注意官方分法。

**98. 效用树**：根→质量属性→属性细分→场景→响应度量，叶子标"重要度 + 难度"优先级（H/M/L）。⚠️ 效用树是 ATAM 第 5 步产物。

**99.🔥 SAAM 五步**：场景开发、架构描述、单个场景评估、场景交互评估、总体评估。⚠️ SAAM 是 ATAM 前身，五步、关注可修改性、不做权衡分析。

**100.⭐ 场景六要素**：刺激源、刺激、环境、制品、响应、响应度量。⚠️ 这是质量属性场景的标准描述，案例题常用。

## 速查/自测

### 速查卡：九类高频陷阱一览

| 类别 | 最易错陷阱 | 正解 |
|---|---|---|
| 计算机基础 | 流水线周期取平均 | 取最长段 |
| 操作系统 | Belady 异常所有算法都有 | 仅 FIFO |
| 数据库 | BCNF = 3NF | BCNF 更严，决定因素必须候选键 |
| 网络 | 三次握手为什么不是两次 | 防失效连接请求 |
| 软件工程 | 维护占比最高是纠错性 | 完善性维护 |
| UML | 组合与聚合箭头一样 | 组合实心菱、聚合空心菱 |
| 设计模式 | 策略 vs 状态一样 | 谁触发切换不同 |
| 架构风格 | SOA 用微服务总线 | SOA 用 ESB |
| 质量评估 | ATAM 第6步基于场景 | 基于效用树 |

### 口诀串记

- **流水线**："周期取长，加速比反着算，吞吐率 n 除以总时间"。
- **死锁**："互请不环四条件，破一即可预防"。
- **范式**："一原二部三传递，BC 决定因素候选键"。
- **隔离**："读未脏、读已不可重、可重幻、串行无"。
- **UML 箭头**："实线三角是泛化，虚线三角是实现，实心菱是组合，空心菱是聚合"。
- **ATAM**："九步三阶段，六效用树八场景，最后呈现结果"。
- **设计模式**："创5结7行十一，工厂造族不造种，策略主动状态被动"。
