# 第4章：从晶体管到流水线，看穿CPU如何跑代码

CPU不是黑盒，搞清楚它怎么取指令、怎么走流水线、怎么响应中断，你才能在架构选型时知道瓶颈在哪。

## 概念

这章属于综合知识科目的"基础硬骨头"，考频稳定在 **2-3 题/年**，单题 1 分，零碎但必须拿下。考查重点集中在 CPU 组成、CISC/RISC 对比、存储层次、流水线加速比计算、中断响应过程，外加数据结构与算法的复杂度辨析。

核心类比：CPU 像一个厨房。ALU 是灶台（负责算），CU 是菜谱调度员（指挥先切后炒），寄存器是手边调料架（最快），缓存是冰箱（次快），内存是楼下仓库（慢），外存是郊外冷库（最慢）。流水线就是把切菜、炒菜、装盘拆成多道工序并行流转，让吞吐量翻倍。

## 原理

### CPU 组成三件套

CPU = 运算器（ALU）+ 控制器（CU）+ 寄存器组。

- **ALU（Arithmetic Logic Unit，算术逻辑单元）**：干加减乘除和与或非。
- **CU（Control Unit，控制单元）**：取指令、译码、发控制信号，是 CPU 的总指挥。
- **寄存器**：通用寄存器存中间结果；PC（Program Counter，程序计数器）存下条指令地址；IR（Instruction Register，指令寄存器）存当前指令；PSW/标志寄存器存状态位。

易混点：PC 指向"下一条要取的指令"，IR 存"正在执行的那条"。两者一个看未来一个看现在，别颠倒。

### CISC vs RISC

| 维度 | CISC（复杂指令集） | RISC（精简指令集） |
|---|---|---|
| 指令长度 | 可变 | 固定 |
| 指令数 | 多（上百条） | 少（基本几十条） |
| 寻址方式 | 丰富 | 少（寄存器-寄存器） |
| 执行时间 | 不固定 | 多数单周期 |
| 流水线 | 难优化 | 友好 |
| 代表 | x86 | ARM、MIPS |

记法：CISC 像"多功能瑞士军刀"，啥都能干但每件都重；RISC 像"专用厨刀"，少而精，流水线最顺。

### 存储层次（速度从快到慢）

寄存器 → 缓存（L1/L2/L3）→ 内存（主存）→ 外存（硬盘/SSD）。

缓存命中用**局部性原理**：时间局部性（刚用过的还会用）+ 空间局部性（用了这个，旁边的也会用）。命中率 = 命中次数 / 总访问次数。平均访问时间 = 命中率 × 缓存时间 + (1-命中率) × 内存时间。

### 总线三类

- **数据总线**：传数据，宽度决定一次传多少位。
- **地址总线**：传地址，宽度决定寻址空间（n 位 → 2^n）。
- **控制总线**：传控制信号（读/写/中断响应）。

易错：32 位地址总线最大寻址 = 2^32 = **4GB**（按字节编址时），不是看数据总线宽度。

### 流水线与加速比

经典五级流水线：取指（IF）→ 译码（ID）→ 执行（EX）→ 访存（MEM）→ 写回（WB）。

流水线周期 = 最慢那一级的耗时。n 条指令、k 级流水线：

- 不用流水线：k × Δt × n
- 用流水线：(k + n - 1) × Δt
- 加速比 S = (k × n) / (k + n - 1)，n 很大时趋近 k

流水线冲突三类：结构冲突（硬件争用）、数据冲突（数据依赖）、控制冲突（跳转指令）。

### 中断

中断向量 = 中断服务程序入口地址。中断响应过程：关中断 → 保存断点（PC 和 PSW）→ 找中断向量 → 跳转服务程序 → 开中断 → 执行 → 返回断点。注意"保存现场"在服务程序内部做，不是响应阶段。

### 数据结构与算法基础（高频小考点）

| 算法 | 平均时间 | 最坏时间 | 备注 |
|---|---|---|---|
| 二分查找 | O(log n) | O(log n) | 需有序 |
| 快速排序 | O(n log n) | O(n²) | 最坏已排序 |
| 归并排序 | O(n log n) | O(n log n) | 稳定 |
| 堆排序 | O(n log n) | O(n log n) | 不稳定 |

## 实践

**真题示例（2024年·综合知识·流水线相关，考生回忆）**：5 级流水线，每级 2ns，执行 100 条指令，求加速比。

答题公式：直接套 S = (k × n)/(k + n - 1) = (5×100)/(5+100-1) = 500/104 ≈ **4.81**。踩分点：分子用 k×n（不是 k×n×Δt，Δt 会约掉），分母是 k+n-1，结果保留两位小数。

**案例/论文应用**：论文里讲"性能优化"时，可拿流水线思想类比——把请求处理拆成多级异步队列，吞吐量提升类比流水线加速比。讲缓存设计时引用局部性原理是加分项。

**踩分词清单**：局部性原理、命中率、加速比、中断向量、CISC/RISC、寻址空间。

## 速查/自测

**30 秒速查卡**

| 考点 | 一句话 |
|---|---|
| CPU 三件套 | ALU 算、CU 控、寄存器存 |
| PC vs IR | PC 指下一条，IR 存当前 |
| CISC vs RISC | CISC 复杂可变长，RISC 精简单周期 |
| 流水线加速比 | S = kn/(k+n-1) |
| 中断向量 | 服务程序入口地址 |
| 地址总线 n 位 | 寻址 2^n |

**口诀**：取译执访写，五级流水跑；最快寄存器，最慢是外存；中断先关再存点，向量一跳就执行。

**自测题**：
1. 16 位地址总线，按字节编址，最大寻址空间是多少？答：2^16 = **64KB**。
2. 流水线 4 级，每级 1ns，10 条指令，加速比？答：(4×10)/(4+10-1) = 40/13 ≈ **3.08**。
3. 快排最坏时间复杂度？答：O(n²)，出现在已有序输入。

一句话小结：CPU 章节拿分靠"流水线公式 + 局部性 + 中断流程"三件套，记牢加速比 S=kn/(k+n-1) 就赢一半。
