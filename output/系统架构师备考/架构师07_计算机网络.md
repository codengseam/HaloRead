# 第7章：七层模型与四次挥手，网络协议的握手哲学

网络是分布式系统的命脉，搞不懂 TCP 三次握手和 HTTPS 握手，你设计的接口加签、长连接、证书全靠猜。

## 概念

综合知识里网络 **3-4 题/年**，单题 1 分。重点在 OSI/TCP-IP 分层、TCP/UDP 区别、握手挥手、HTTPS 原理、加密体系。

核心类比：网络分层像寄快递。应用层是你写的内容，传输层是快递公司（TCP 签收确认、UDP 扔了不管），网络层是分拣中心（IP 选路），数据链路层是运输车，物理层是公路。每一层只跟上下层打交道，不越级。

## 原理

### OSI 七层 vs TCP/IP 四层

| OSI 七层 | TCP/IP 四层 | 典型协议 |
|---|---|---|
| 应用层 | 应用层 | HTTP/FTP/DNS/SMTP |
| 表示层 | 应用层 | SSL/TLS、JPEG |
| 会话层 | 应用层 | RPC、Socket |
| 传输层 | 传输层 | TCP/UDP |
| 网络层 | 网际层 | IP/ICMP |
| 数据链路层 | 网络接口层 | ARP、以太网 |
| 物理层 | 网络接口层 | 双绞线、光纤 |

记忆口诀：物数网传会表应（从下往上）。TCP/IP 把上三层合并成应用层、下两层合并成网络接口层。

### TCP vs UDP

| 维度 | TCP | UDP |
|---|---|---|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠（确认重传） | 不可靠 |
| 顺序 | 有序 | 无序 |
| 速度 | 慢 | 快 |
| 头部 | 20 字节 | 8 字节 |
| 场景 | HTTP/SSH/邮件 | DNS/视频/直播 |

### 三次握手四次挥手

**三次握手**（建连）：
1. 客户端 → SYN, seq=x → 服务器
2. 服务器 → SYN+ACK, seq=y, ack=x+1 → 客户端
3. 客户端 → ACK, ack=y+1 → 服务器

为什么三次不是两次？防止历史失效连接误建，避免服务端白等。

**四次挥手**（断连）：
1. 客户端 → FIN → 服务器
2. 服务器 → ACK → 客户端（服务器进入 CLOSE_WAIT）
3. 服务器 → FIN → 客户端
4. 客户端 → ACK → 服务器（客户端 TIME_WAIT 2MSL 后关闭）

为什么四次？TCP 全双工，每方都要单独关。TIME_WAIT 等 2MSL（最大报文生存时间）防止最后 ACK 丢失。

### HTTP 与 HTTPS

HTTP 默认 80 端口，明文传输；HTTPS 默认 443，HTTP + SSL/TLS（Secure Sockets Layer / Transport Layer Security）。

HTTPS 握手简化版：客户端发支持的加密套件 → 服务器返回证书和公钥 → 客户端验证证书（CA 签名）→ 客户端生成会话密钥用公钥加密发回 → 双方用对称密钥通信。

要点：**非对称加密（RSA）只用于交换会话密钥，之后用对称加密（AES）传数据**——兼顾安全与性能。

DNS（Domain Name System）解析顺序：浏览器缓存 → 系统缓存 → hosts → 本地 DNS → 根 → 顶级 → 权威。

### 网络安全

对称加密（AES、DES）：加解密同密钥，快，但密钥分发难。
非对称加密（RSA、ECC）：公钥加密私钥解密，慢，解决密钥分发。

数字签名：私钥签名，公钥验签——保证不可抵赖和完整性。
数字证书：CA 用自己私钥给服务器公钥签名，客户端用 CA 公钥验证。

### CDN 原理

CDN（Content Delivery Network，内容分发网络）把静态资源缓存到离用户近的边缘节点。用户请求 → DNS 智能解析到最近 CDN 节点 → 命中直接返回，未命中回源拉取并缓存。核心价值：**降低延迟、减轻源站压力**。

## 实践

**真题示例（2024年·综合知识·TCP握手相关，考生回忆）**：TCP 建立连接为什么用三次握手而非两次？

答题公式：①防止已失效的连接请求突然到达服务器，造成资源浪费；②双方都要确认对方的收发能力正常。两次握手无法确认客户端的接收能力。踩分点："防止失效连接""双向确认""避免服务端资源浪费"。

**HTTPS 握手题**：问对称还是非对称。答：握手用非对称（RSA/ECDHE）协商会话密钥，传输用对称（AES）。两者分工：非对称解决密钥分发，对称解决性能。

**论文应用**：讲"接口安全设计"引 HTTPS + 双向证书；讲"高并发架构"引 CDN 边缘缓存降低源站压力、长连接复用减少握手开销。提到 TIME_WAIT 调优（缩短 2MSL、复用连接）是加分项。

**踩分词**：三次握手、四次挥手、TIME_WAIT、2MSL、CA、对称/非对称、CDN。

## 速查/自测

**30 秒速查卡**

| 考点 | 一句话 |
|---|---|
| OSI 七层 | 物数网传会表应 |
| TCP vs UDP | TCP 可靠有序，UDP 快无连接 |
| 三次握手 | SYN→SYN+ACK→ACK |
| 四次挥手 | FIN→ACK→FIN→ACK |
| TIME_WAIT | 等 2MSL，防最后 ACK 丢失 |
| HTTPS | 握手非对称，传输对称 |
| CDN | 边缘节点缓存，DNS 智能调度 |

**口诀**：三次建连四次挥，TIME_WAIT 等 2MSL；HTTPS 握手用非对称，传数据还得对称来；签名私签公验，证书 CA 来背书。

**自测题**：
1. UDP 头部多少字节？答：**8 字节**。
2. HTTPS 握手阶段和传输阶段分别用什么加密？答：握手非对称（RSA/ECDHE），传输对称（AES）。
3. 四次挥手中服务器收到客户端 FIN 后进入什么状态？答：**CLOSE_WAIT**，发完自己的 FIN 后进入 LAST_ACK。

一句话小结：网络拿分靠"七层分层 + 三握四挥 + HTTPS 双加密 + CDN 调度"，把三次握手为什么不能两次讲清楚就过关。
