# 第14章：用十四种图把系统画明白

UML（Unified Modeling Language，统一建模语言）是架构师和需求方、开发方沟通的"工程图纸"。综合知识考2题，但真正的大头在案例分析——设计模式题几乎必带一张 UML 类图让你读，读不懂箭头就没法动笔。

## 概念

### 这章考什么

- **科目**：综合知识2题 + 案例分析（设计模式题标配类图）
- **考频**：综合稳定 **2题**；案例设计模式题**必考**类图阅读
- **分值**：综合2分；案例可能占5-15分
- **难度**：★★★☆☆（图多、箭头易混）

UML 2.x 共 **14种图**，分两大类：

- **结构图（7种）**：描述系统静态结构——类图、对象图、构件图、部署图、组合结构图、包图、轮廓图
- **行为图（7种）**：描述系统动态行为——用例图、活动图、序列图、通信图、定时图、状态图、交互概览图

记忆口诀：**结七行七，结构看静态，行为看动态**。

其中高频考点是：用例图、类图（箭头是命门）、序列图、活动图、状态图。其余图知道归属即可。

## 原理

### 用例图（Use Case Diagram）

描述系统功能与外部参与者的关系。三要素：

- **参与者（Actor）**：人或其他系统，画成小人
- **用例（Use Case）**：系统功能，画成椭圆
- **关系**：参与者与用例连线；用例间三种关系

用例间关系（高频考点）：

| 关系 | 符号 | 含义 | 例子 |
|---|---|---|---|
| 包含 include | `<<include>>` 虚线箭头 | A 必然调用 B | "下单"必包含"登录" |
| 扩展 extend | `<<extend>>` 虚线箭头 | A 在特定条件下调用 B | "支付"可选扩展"用优惠券" |
| 泛化 generalization | 空心三角实线 | 用例继承 | "在线支付"泛化"支付宝支付" |

辨析：**include 是必须，extend 是可选**。箭头方向都是从"基用例"指向"被包含/被扩展用例"。

### 类图（Class Diagram）——案例必考

类图是最核心的图。一个类用三栏矩形表示：

```
┌─────────────┐
│   类名       │
├─────────────┤
│ - 私有属性   │  ← 属性栏
│ + 公有属性   │
├─────────────┤
│ + 方法()    │  ← 方法栏
└─────────────┘
```

可见性符号：`+`公有、`-`私有、`#`保护、`~`包级。

**类间关系（箭头是命门）**——必背速查：

| 关系 | 符号 | 记忆 | 强弱 |
|---|---|---|---|
| 继承/泛化 | 实线 + 空心三角箭头 | "is-a" | 最强 |
| 实现 | 虚线 + 空心三角箭头 | "implements" | 强 |
| 组合 | 实线 + 实心菱形 | "强拥有，同生共死" | 强 |
| 聚合 | 实线 + 空心菱形 | "弱拥有，可分离" | 中 |
| 关联 | 实线（可带箭头） | "长期持有" | 中 |
| 依赖 | 虚线箭头 | "临时使用" | 最弱 |

记忆口诀：**空三角是继承/实现（实虚看线），菱形是拥有（实心组合空心聚合），虚箭头是依赖**。

**组合 vs 聚合**（高频易混）：
- **组合**：整体与部分**同生命周期**，部分不能脱离整体存在。如鸟和翅膀
- **聚合**：整体与部分**可独立存在**。如球队和球员

```java
// 组合：翅膀随鸟生灭
class Bird {
    private Wing wing = new Wing();  // 内部创建，同生共死
}
// 聚合：球员可离开球队
class Team {
    private List<Player> players;
    public void addPlayer(Player p) { players.add(p); }  // 外部传入
}
```

### 序列图（Sequence Diagram）

按时间顺序展示对象间消息交互。关键元素：

- 生命线（Lifeline）：对象下方虚线
- 激活条（Activation）：生命线上的矩形，表示对象执行中
- 消息类型：
  - **同步消息**：实心三角箭头，调用方等待返回
  - **异步消息**：开放箭头（线段+开口箭头），调用方不等返回
  - **返回消息**：虚线箭头

辨析：同步是"打电话等对方说话"，异步是"发短信不等回复"。

### 活动图（Activity Diagram）

描述工作流或业务流程。关键元素：

- **动作**（圆角矩形）
- **开始节点**（实心圆）、**结束节点**（牛眼）
- **决策节点**（菱形，带条件）
- **分叉/汇合**（粗横线）：表示**并发**，分叉拆出多个并行流，汇合等所有流到齐
- **泳道**（Swimlane）：按角色或部门分区，标明动作归属

注意：分叉汇合管并发，决策节点管选择，两者别混。

### 状态图（State Diagram）

描述对象生命周期内的状态变化。要素：

- 状态（圆角矩形）
- 转移（带箭头实线，标注触发事件[条件]/动作）
- 初始状态（实心圆）、终止状态（牛眼）

例子：订单状态——待支付→已支付→已发货→已完成；已支付→已取消。

### 其余结构图（了解归属）

- **对象图**：类图的实例快照
- **构件图**：展示组件及依赖
- **部署图**：展示硬件节点及部署的组件
- **组合结构图**：类的内部结构
- **包图**：包的组织
- **轮廓图**：UML 扩展机制

### 其余行为图（了解归属）

- **通信图**：序列图的"对象关系版"，强调结构而非时间
- **定时图**：强调时间约束
- **交互概览图**：活动图与交互图的混合

## 实践

### 真题示例

**2024年·综合知识·UML相关(考生回忆)**：以下 UML 关系中，表示整体与部分同生命周期的是？
A. 聚合
B. 组合
C. 依赖
D. 关联

**答案**：B。组合是同生共死，聚合可分离。

### 案例设计模式题应对

案例设计模式题几乎必给一张类图，要求：
1. 读出模式名称
2. 解释角色
3. 补全代码或填空

读图三步法：
1. **先找箭头**：空心三角→继承/实现；菱形→组合/聚合；这决定模式骨架
2. **数类数量**：一个抽象+多个具体 → 多半是工厂/策略；层层包装 → 装饰器；链式调用 → 责任链
3. **看方法签名**：方法返回自己类型 → 建造者/装饰器；方法接受自己类型 → 访问者/组合

### 答题公式

回答"该图是什么模式"时：

> 该类图体现了 XX 模式。其中 AbstractX 是抽象主题，ConcreteX 是具体实现，它们通过【继承/组合/聚合】关系组织。客户端依赖抽象，符合开闭原则，可在不修改原代码的前提下新增 XX。

### 论文应用

写"软件架构设计/设计模式应用"论文时，UML 类图是论证利器。可在设计说明段写：

> 系统核心模块采用策略模式，类图中 PaymentStrategy 为抽象策略，AlipayStrategy、WechatStrategy 为具体策略，Context 类持有策略引用并通过聚合关系组合，运行时动态切换。

### 口诀

**结七行七十四图，类图箭头是命门；空三角看继承实现（实虚看线），菱形分聚合组合（实心强空心弱），虚线箭头是依赖。**

### 自测题

1. 序列图中同步消息和异步消息的箭头有何区别？
2. 活动图的"分叉汇合"和"决策节点"分别表示什么？能否互换？
3. 一个类图中，A 类实心菱形指向 B 类，B 类生命周期是否独立于 A？

### 一句话小结

UML 十四图分结构行为各七，类图箭头是案例设计模式题的命门——空三角继承实现、实心菱形组合、空心菱形聚合、虚线依赖，背熟就能读图得分。
