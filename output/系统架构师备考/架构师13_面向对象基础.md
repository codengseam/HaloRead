# 第13章：用四大特性驯服变化的代码

面向对象（Object-Oriented, OO）的本质不是写 class，而是用封装、继承、多态、抽象这四把刀，把变化关进笼子里。这一章在综合知识里年年考2-3题，看似送分，实则坑在重载与重写、静态与动态绑定的辨析。

## 概念

### 这章考什么

- **科目**：综合知识（上午题）
- **考频**：稳定 **2-3题**，每题1分
- **题型**：概念辨析为主，少量代码片段判断输出
- **难度**：★★☆☆☆（基础送分，但易混点能丢分）

类比一下：面向过程是"厨师一个人从头炒到尾"，数据和函数散落各处；面向对象是"后厨分工"，切菜的、掌勺的、摆盘的各管各的数据（封装），新人能跟老师傅学手艺（继承），同一个"炒菜"指令在不同厨师手里出不同菜（多态），先定菜谱再让具体厨师实现（抽象）。

四大特性是骨架，类与对象是血肉，消息传递是神经，绑定机制决定多态何时生效。这套机制是后面 UML、设计原则、设计模式的共同语言，必须吃透。

## 原理

### 封装（Encapsulation）

把数据和操作数据的方法绑在一起，并隐藏内部实现，只暴露必要接口。

```java
public class Account {
    private double balance;  // 数据藏起来

    public void deposit(double amount) {  // 只开存款口
        if (amount > 0) balance += amount;
    }
    public double getBalance() { return balance; }
}
```

类比：ATM机，你只能按屏幕按钮（公开方法），不能拆开机箱改余额（私有数据）。封装的真正价值不是"藏起来"，而是**降低耦合**——内部实现变了，外部调用者无感。

### 继承（Inheritance）

子类获得父类的属性和方法，并可扩展或修改。

```java
class Animal {
    void breathe() { System.out.println("呼吸"); }
}
class Dog extends Animal {  // 自动拥有 breathe()
    void bark() { System.out.println("汪汪"); }
}
```

类比：徒弟继承师傅的手艺，还能自创招式。继承有单继承和多继承之分，Java 类单继承、接口多继承，C++ 支持多继承（带来菱形问题）。

### 多态（Polymorphism）

同一消息发给不同对象，产生不同行为。分两种：

- **编译时多态**：重载（Overload），方法名相同参数不同，编译器根据参数类型决定调谁
- **运行时多态**：重写（Override），子类覆盖父类方法，运行时根据实际对象类型决定调谁

```java
class Shape {
    void draw() { System.out.println("画图形"); }
    void draw(int color) { }  // 重载 Overload
}
class Circle extends Shape {
    @Override
    void draw() { System.out.println("画圆"); }  // 重写 Override
}

Shape s = new Circle();
s.draw();  // 输出"画圆"——运行时多态
```

类比：开会喊"开始"，程序员开始写代码，设计师开始画图，PM 开始催进度——同一个指令，不同角色不同响应。

### 抽象（Abstraction）

抽取事物本质特征，忽略非本质细节。通过抽象类和接口实现。

```java
abstract class Vehicle {
    abstract void move();  // 只声明，不实现
}
interface Flyable {
    void fly();
}
```

类比：画地图时只标主干道，不画每条小巷。抽象是设计层面的能力，决定系统的分层和扩展性。

### 类与对象、消息传递

- **类**是模板/蓝图，**对象**是类的实例
- **消息传递**：对象 A 调用对象 B 的方法，就是发消息。`obj.method(args)` 即向 obj 发送 method 消息

### 重载 vs 重写（Override）——高频易混点

| 对比项 | 重载 Overload | 重写 Override |
|---|---|---|
| 发生位置 | 同一个类内 | 父子类之间 |
| 方法名 | 相同 | 相同 |
| 参数列表 | **必须不同** | **必须相同** |
| 返回类型 | 可不同 | 相同或子类型（协变返回） |
| 访问修饰符 | 无限制 | 不能比父类更严 |
| 绑定时机 | **静态绑定**（编译期） | **动态绑定**（运行期） |
| 多态类型 | 编译时多态 | 运行时多态 |

口诀：**重载看参数（编译定），重写看对象（运行定）**。

### 静态绑定 vs 动态绑定

- **静态绑定**（Static Binding）：编译时确定调用哪个方法。重载、private 方法、static 方法、final 方法都是静态绑定
- **动态绑定**（Dynamic Binding）：运行时根据对象实际类型决定。重写的实例方法走动态绑定

经典坑题：

```java
class Father {
    void say() { System.out.println("父"); }
}
class Son extends Father {
    void say() { System.out.println("子"); }
}
Father f = new Son();
f.say();  // 输出"子"——动态绑定
```

变量声明类型是 Father，但实际对象是 Son，动态绑定看实际类型，所以调 Son 的 say()。

## 实践

### 真题示例

**2024年·综合知识·面向对象相关(考生回忆)**：以下关于重载和重写的描述，正确的是？
A. 重载发生在父子类之间，重写发生在同一类内
B. 重载要求参数列表不同，重写要求参数列表相同
C. 重载是动态绑定，重写是静态绑定
D. 重载和重写都体现运行时多态

**答案**：B。A 反了；C 反了，重载静态、重写动态；D 错，重载是编译时多态。

### 答题公式

遇到"以下属于 XX"题，先定位考的是哪个特性，再用三步判断：
1. **看位置**：同类内还是父子类？→ 区分重载/重写
2. **看参数**：参数变不变？→ 重载变、重写不变
3. **看绑定**：编译期还是运行期？→ 重载静态、重写动态

### 案例论文应用

写"面向对象分析与设计"论文时，可在方法描述段落引用四大特性作为设计依据：

> 本系统采用面向对象方法，通过封装将订单状态与操作收敛在 Order 类内，避免散落式修改；通过继承复用支付策略基类；借助多态在运行期切换具体支付实现，满足开闭原则。

这样写比空谈"使用了面向对象"有说服力，体现你理解特性背后的设计意图。

## 速查/自测

### 30秒速查卡

| 特性 | 一句话 | 关键词 |
|---|---|---|
| 封装 | 藏数据、露接口 | private + public |
| 继承 | 子承父业 | extends |
| 多态 | 同消息不同响应 | 重载 + 重写 |
| 抽象 | 抽本质、留接口 | abstract + interface |

| 绑定 | 时机 | 对应 |
|---|---|---|
| 静态 | 编译期 | 重载 / private / static / final |
| 动态 | 运行期 | 重写的实例方法 |

### 口诀

**封继多抽四把刀，藏数据传手艺变响应留接口；重载看参数编译定，重写看对象运行定。**

### 自测题

1. 一个类中定义了三个 method：`method(int)`、`method(double)`、`method(int,int)`，这属于什么？体现哪种多态？
2. `Father f = new Son(); f.say();` 调用的是谁的 say()？为什么？
3. private 方法能被重写吗？属于静态绑定还是动态绑定？

### 一句话小结

四大特性是 OO 的骨架，重载重写与绑定时机是综合知识丢分高发区——记住"重载看参数编译定，重写看对象运行定"就不会错。
