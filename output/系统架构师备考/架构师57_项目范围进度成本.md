# 第57章：范围进度成本三巨头，挣值公式与关键路径是计算命门

范围、进度、成本三大管理是中级《系统集成项目管理工程师》的"三巨头"，合计每年 10-15 分，是综合知识计算题的重灾区。WBS 分解、PERT 三点估算、CPM 关键路径、挣值技术 EVT 这四套公式记牢，案例题至少拿一半分。

## 概念

### 这章考什么
- **科目**：中级综合知识（每年 10-15 题）+ 案例分析计算题
- **分值**：中级约 10-15 分；案例 5-15 分
- **考频**：高频，计算题重灾区
- **难度**：中等偏上，公式多、计算多

核心类比：**范围管理**画的是"做什么不做什么"的圈——圈外的事坚决不干，否则范围蔓延；**进度管理**排的是"先做什么后做什么"的时间表——找出哪条路径最关键，关键路径延误一天，项目就晚交付一天；**成本管理**管的是"花多少钱、花得值不值"——挣值技术 EVT 把进度和成本揉到一起算，用 CPI/SPI 一眼看出项目是赚是亏。

## 原理

### 【中级重点】范围管理：WBS 是命门

范围管理五过程：**收集需求 → 定义范围 → 创建 WBS → 确认范围 → 控制范围**。

**收集需求**的工具：访谈、焦点小组、引导式研讨会、头脑风暴、问卷、原型法。

**定义范围**的产出是**项目范围说明书**，明确项目边界、可交付成果、验收标准、排除项、制约因素、假设条件。

**创建 WBS（Work Breakdown Structure，工作分解结构）** 是范围管理命门。把项目可交付成果逐层分解成可管理的工作包。关键规则：

| 规则 | 内容 |
|---|---|
| **8/80 规则** | 工作包工作量在 8 小时到 80 小时之间（1 天到 10 天），太小管理成本高，太大难控 |
| **100% 原则** | 所有子项加起来必须等于父项的 100%，不重不漏 |
| **分解到工作包** | 最底层叫工作包（Work Package），是估算成本和进度的基础 |
| **逐步细化** | 远期工作可粗分解，近期工作细分解（滚动式规划） |

**WBS 词典**：对每个工作包的详细描述，包括工作内容、责任人、进度里程碑、成本预算、质量要求、合同信息。

**确认范围 vs 控制质量**（高频易错）：
- **确认范围**：客户/发起人正式验收可交付成果，关注"做没做客户要的"。
- **控制质量**：检查可交付成果是否符合质量标准，关注"做得对不对"。
- 时序：先控制质量（内部查），再确认范围（客户验收）。

**控制范围**：监控范围状态，管理范围变更。范围蔓延（Scope Creep）是无控制的范围扩大，是范围管理最大敌人。

### 【中级重点】进度管理：PERT 与 CPM 双柱

进度管理六过程：**定义活动 → 排序活动 → 估算活动资源 → 估算活动历时 → 制定进度计划 → 控制进度**。

**活动定义**：把工作包进一步分解成可执行的活动。**活动排序**用紧前关系绘图法（PDM），四种逻辑关系：

| 关系 | 含义 |
|---|---|
| **FS 完成-开始** | 紧前完成，紧后才能开始（最常见） |
| **FF 完成-完成** | 紧前完成，紧后才能完成 |
| **SS 开始-开始** | 紧前开始，紧后才能开始 |
| **SF 开始-完成** | 紧前开始，紧后才能完成（最少见） |

**历时估算**之三点估算 PERT（Program Evaluation and Review Technique）公式：

$$TE = \frac{OT + 4 \times MT + PT}{6}$$

- OT 乐观时间（Optimistic Time）
- MT 最可能时间（Most Likely Time）
- PT 悲观时间（Pessimistic Time）

**标准差** σ = (PT - OT) / 6，**方差** σ² = [(PT - OT) / 6]²。

**关键路径法 CPM（Critical Path Method）**：

- 关键路径是项目网络中**历时最长**的路径，决定项目最短工期。
- 关键路径上的活动**总浮动时间为零**，任何延误都直接延误总工期。
- 一个项目可能有多条关键路径。

**浮动时间两兄弟**（高频计算）：

| 类型 | 定义 | 关键路径上 |
|---|---|---|
| **总浮动时间 TF（Total Float）** | 不影响总工期的可延误量 = 最晚开始 - 最早开始 | 0 |
| **自由浮动时间 FF（Free Float）** | 不影响紧后活动最早开始的可延误量 = 紧后最早开始 - 本活动最早完成 | 通常 ≥ 0 |

计算公式：
- **TF = LS - ES = LF - EF**（LS 最晚开始、ES 最早开始、LF 最晚完成、EF 最早完成）
- **FF = min(紧后 ES) - EF**

**甘特图（Gantt Chart）**：横条图显示活动起止，直观但**不显示依赖关系**。
**PERT 网络图**：节点+箭线，显示依赖，能算关键路径。

**进度压缩两招**：
- **赶工（Crashing）**：加资源（加人、加班）缩短工期，**增加成本**。
- **快速跟进（Fast Tracking）**：原本串行的活动并行做，**增加返工和协调风险**，一般不增成本。

口诀：**赶工加钱、快速跟进加风险**。

### 【中级重点】成本管理：挣值技术 EVT 是绝对命门

成本管理四过程：**估算成本 → 制定预算 → 控制成本**（规划组）+ 控制成本（监控组）。

**成本估算四方法**：

| 方法 | 特点 | 适用 |
|---|---|---|
| **类比估算** | 用历史相似项目估算，省时但不准 | 早期信息少 |
| **参数估算** | 用历史数据参数×数量 | 有可靠参数 |
| **自下而上估算** | 各工作包估算后汇总，最准但费时 | 详细信息齐 |
| **三点估算** | 乐观/最可能/悲观加权 | 不确定性高 |

**制定预算——储备分析**：

| 储备类型 | 含义 | 谁管 | 是否纳入成本基准 |
|---|---|---|---|
| **应急储备** | 应对已识别风险 | 项目经理 | **纳入**成本基准 |
| **管理储备** | 应对未识别风险 | 高层/发起人 | **不纳入**成本基准，动用需审批 |

**成本基准** = 工作包估算 + 应急储备。**项目预算** = 成本基准 + 管理储备。

**挣值技术 EVT（Earned Value Technique）三大基础值**：

| 缩写 | 全称 | 含义 |
|---|---|---|
| **PV** | Planned Value 计划价值 | 计划要干的活的预算成本 |
| **EV** | Earned Value 挣值 | 实际干完的活的预算成本 |
| **AC** | Actual Cost 实际成本 | 实际干完的活实际花的钱 |
| **BAC** | Budget at Completion 完工预算 | 项目总预算 |

**偏差与指数公式**（必背命门）：

| 公式 | 含义 | 判读 |
|---|---|---|
| **CV = EV - AC** | 成本偏差 | >0 省钱，<0 超支 |
| **SV = EV - PV** | 进度偏差 | >0 提前，<0 拖后 |
| **CPI = EV / AC** | 成本绩效指数 | >1 省钱，<1 超支，=1 持平 |
| **SPI = EV / PV** | 进度绩效指数 | >1 提前，<1 拖后，=1 持平 |

**预测公式**（高频计算）：

| 公式 | 含义 |
|---|---|
| **EAC = BAC / CPI** | 完工估算（按当前效率推算） |
| **EAC = AC + (BAC - EV)** | 完工估算（剩余按原计划干，偏差是一次性的） |
| **EAC = AC + (BAC - EV) / (CPI × SPI)** | 完工估算（剩余受成本和进度双重影响） |
| **ETC = EAC - AC** | 完工尚需估算 |
| **VAC = BAC - EAC** | 完工偏差（>0 省钱，<0 超支） |
| **TCPI = (BAC - EV) / (BAC - AC)** | 完工尚需绩效指数，必须达到才能按 BAC 完成 |

**口诀**：**CV EV 减 AC，SV EV 减 PV，CPI EV 比 AC，SPI EV 比 PV；EAC 最常用 BAC 除 CPI**。

判读铁律：**EV 永远是被减数/分子**，所以 CV、SV、CPI、SPI 的符号和 1 的关系都看 EV 与谁比。EV 比 AC/PV 大就越好。

## 实践

### 【高级衔接】挣值技术在架构项目中的实战

架构师用 CPI/SPI 判断项目健康度，比单纯看进度百分比准得多。一个项目"进度完成 80%"可能掩盖了 SPI=0.7 的真相——看着快，实际慢。

健康度判读四象限：

| SPI \ CPI | CPI ≥ 1（省钱） | CPI < 1（超支） |
|---|---|---|
| **SPI ≥ 1（提前）** | 健康，可适当加挑战 | 花钱买速度，查成本 |
| **SPI < 1（拖后）** | 慢但省钱，查进度 | 双红，需紧急干预 |

架构师看到 CPI<1 且 SPI<1 的"双红"项目，第一反应不是加人（布鲁克斯定律：加人让延期的项目更延期），而是回到架构本身：是不是过度设计导致成本超支？是不是技术选型不稳导致返工拖慢进度？挣值数据是架构健康度的"体温计"。

### 网络图计算真题示例

**真题示例**（2023年·中级案例分析·进度管理相关，考生回忆）：

> 某项目有 5 个活动，逻辑关系与历时如下：
> - A：5 天，开始活动
> - B：4 天，A 完成后开始
> - C：6 天，A 完成后开始
> - D：3 天，B 完成后开始
> - E：5 天，C、D 完成后开始
>
> 求：(1) 关键路径；(2) 总工期；(3) 活动 B 的总浮动时间。

**解题步骤**：

**第一步：列路径**
- 路径 1：A → B → D → E = 5+4+3+5 = **17 天**
- 路径 2：A → C → E = 5+6+5 = **16 天**

**第二步：找关键路径** = 最长路径 = **A→B→D→E，17 天**。

**第三步：算 B 的总浮动**
- 路径 A→B→D→E 是关键路径，B 在关键路径上，**TF = 0**。

**延伸**：若问活动 C 的总浮动时间：
- C 在路径 A→C→E（16 天）上，比关键路径少 1 天，**TF(C) = 17 - 16 = 1 天**。

### 挣值计算真题示例

**真题示例**（2024年·中级综合知识·成本管理相关，考生回忆）：

> 某项目 BAC=100 万，到第 6 月末，PV=40 万、EV=30 万、AC=50 万。求 CV、SV、CPI、SPI、EAC（按当前效率）、VAC。

**计算**：
- CV = EV - AC = 30 - 50 = **-20 万**（超支 20 万）
- SV = EV - PV = 30 - 40 = **-10 万**（拖后 10 万的工作量）
- CPI = EV / AC = 30 / 50 = **0.6**（每花 1 元只挣 0.6 元价值）
- SPI = EV / PV = 30 / 40 = **0.75**（进度效率 75%）
- EAC = BAC / CPI = 100 / 0.6 ≈ **166.67 万**（按当前效率，最终要花 166.67 万）
- VAC = BAC - EAC = 100 - 166.67 = **-66.67 万**（超支 66.67 万）

判读：CPI=0.6、SPI=0.75，**双红**，项目严重超支且拖后，必须紧急干预。

### 案例分析答题公式

进度/成本案例题答题套路："**算什么 + 公式 + 代入 + 判读 + 建议**"五段式。

1. **算什么**：明确题目要求算哪个指标（CV/SV/CPI/SPI/EAC 等）。
2. **公式**：写出对应公式。
3. **代入**：把题目数据代入。
4. **判读**：>0 或 <0、>1 或 <1 的含义。
5. **建议**：基于判读给出纠偏建议（赶工、快速跟进、范围调整、风险应对）。

## 速查/自测

### 30 秒速查卡

| 主题 | 关键点 | 公式/口诀 |
|---|---|---|
| WBS 8/80 规则 | 工作包 8-80 小时 | 1-10 天 |
| 确认范围 vs 控制质量 | 客户验收 vs 内部查 | 先质后范围 |
| PERT 三点估算 | 乐观加四倍最可能加悲观除以六 | TE=(OT+4MT+PT)/6 |
| PERT 标准差 | (PT-OT)/6 | σ |
| 关键路径 | 最长路径，总浮动为零 | 决定总工期 |
| 总浮动 TF | LS-ES=LF-EF | 关键路径上=0 |
| 自由浮动 FF | min(紧后ES)-EF | 通常≥0 |
| 赶工 vs 快速跟进 | 加钱 vs 加风险 | — |
| CV/SV | EV-AC / EV-PV | EV 永远被减 |
| CPI/SPI | EV/AC / EV/PV | EV 永远分子 |
| EAC（常用） | BAC/CPI | 按当前效率 |
| ETC | EAC-AC | 尚需成本 |
| VAC | BAC-EAC | 完工偏差 |
| 应急储备 | 已识别风险，PM 管 | 纳入基准 |
| 管理储备 | 未识别风险，高层管 | 不纳入基准 |

### CPM 计算口诀

```
最长路径是关键，关键路径时差零；
ES EF 正向推，LS LF 逆向推；
TF 等于 LS 减 ES，FF 看紧后最早始。
```

### 自测题

**自测 1**（PERT 计算）：某活动乐观 4 天、最可能 6 天、悲观 16 天，求期望工期和标准差。

> 参考答案：TE = (4 + 4×6 + 16) / 6 = (4+24+16)/6 = 44/6 ≈ **7.33 天**。σ = (PT - OT)/6 = (16-4)/6 = **2 天**。

**自测 2**（挣值计算）：某项目 BAC=200 万，月末数据 PV=80 万、EV=60 万、AC=100 万。求 CV、CPI、SPI、EAC、VAC，并判读项目状态。

> 参考答案：
> - CV = 60-100 = **-40 万**（超支）
> - CPI = 60/100 = **0.6**（超支）
> - SPI = 60/80 = **0.75**（拖后）
> - EAC = 200/0.6 ≈ **333.33 万**
> - VAC = 200-333.33 = **-133.33 万**（超支）
> - 判读：CPI<1 且 SPI<1，双红，项目严重超支且进度拖后，需立即干预。

**自测 3**（CPM 计算）：某项目活动 A(3天)→B(4天)→D(2天)，A(3天)→C(5天)→D(2天)。求关键路径、总工期、活动 C 的总浮动时间。

> 参考答案：
> - 路径 1：A→B→D = 3+4+2 = 9 天
> - 路径 2：A→C→D = 3+5+2 = 10 天
> - 关键路径 = A→C→D，**10 天**
> - C 在关键路径上，TF(C) = **0**

### 一句话小结

WBS 用 8/80 分解不重不漏，PERT 三点估工期、CPM 找最长路径定关键，挣值 EVT 用 EV 永远当被减数/分子——**CV/SV 看正负、CPI/SPI 看是否过 1、EAC 用 BAC 除 CPI**，三大公式记牢，中级计算题拿满分。
