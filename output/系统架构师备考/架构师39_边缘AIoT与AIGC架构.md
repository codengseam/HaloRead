# 第39章：把算力推到数据产生的地方，把智能塞进每一层架构

云计算把算力集中到了数据中心，但物联网和实时 AI 的爆发又把算力往回推——数据产生在边缘、在终端，全送到云端既慢又贵还不安全。于是架构从"云为中心"演进成"云-边-端协同"，AI 也不再只在云端训练推理，而是渗透进感知、网络、平台每一层。本章覆盖边缘计算、AIoT 和 AIGC 系统架构三个新增考点。

## 概念

**这章考什么**：综合知识中边缘计算与 AIoT 合计约 1-2 题，AIGC 部分为**拓展了解**（综合可能 1 题常识题，不要投入过多）。考频整体不高但属于 2026 新增前沿，案例分析可能结合物联网场景出题。

**核心类比**：云计算像"中央厨房"——所有食材运到一起加工再分发，规模大但路途远；边缘计算像"社区厨房"——在你家楼下就近做饭，快、省运费、食材也不用大老远送。AIoT 则是给每个厨房配上智能厨师，AIGC 是这个厨师能即兴创作新菜谱。

三个概念的层次关系：**边缘计算是算力部署形态**，**AIoT 是物联网 + AI 的融合架构**，**AIGC 是 AI 生成内容的能力体系**。它们可以叠加——边缘节点跑 AIoT 平台，平台里调 AIGC 模型。

## 原理

### 一、边缘计算：算力下沉到数据旁边

**云-边-端协同三层架构**：

| 层 | 位置 | 职责 | 典型设备 |
|---|---|---|---|
| **云（Cloud）** | 中心数据中心 | 大模型训练、全局调度、长期存储 | GPU 集群、对象存储 |
| **边（Edge）** | 靠近数据源的边缘节点 | 实时推理、本地聚合、协议转换 | 边缘服务器、5G MEC |
| **端（Device）** | 终端设备 | 数据采集、轻量推理、执行控制 | 摄像头、传感器、PLC |

**边缘计算 vs 云计算**（高频对比）：

| 对比维度 | 云计算 | 边缘计算 |
|---|---|---|
| 延迟 | 高（数据往返中心） | 低（就近处理） |
| 带宽 | 占用大（全量上传） | 省带宽（本地处理只传结果） |
| 数据隐私 | 数据出本地，隐私风险高 | 敏感数据可不出域 |
| 算力 | 强、集中 | 弱、分散 |
| 适合场景 | 大模型训练、离线分析 | 实时控制、视频分析、低延迟推理 |

记忆要点：边缘的三张王牌是**低延迟、省带宽、保隐私**；代价是算力分散、运维复杂。考题常给一个场景（自动驾驶、工业质检、智能安防）问"为什么用边缘"，答这三点。

### 二、AIoT：物联网装上 AI 的大脑

AIoT = AI + IoT，不是简单叠加，而是让物联网从"连起来"升级到"懂起来"。

**四层架构**（务必背熟，案例论文万能框架）：

| 层 | 作用 | 关键技术 | 一句话理解 |
|---|---|---|---|
| **感知层** | 采集物理世界数据 | 传感器、RFID、摄像头 | 物联网的眼睛和耳朵 |
| **网络层** | 数据传输 | 5G、NB-IoT、LoRa、Wi-Fi | 物联网的神经 |
| **平台层** | 数据处理与设备管理 | 物联网平台、边缘 AI、规则引擎 | 物联网的大脑 |
| **应用层** | 业务呈现 | 智慧城市、工业互联网、智能家居 | 物联网的手和嘴 |

**易混点**：物联网平台层 vs 边缘计算节点——平台层是逻辑分层概念，边缘节点是物理部署位置，两者正交。边缘节点可以承担平台层的部分职责（本地推理、协议转换）。

**AIoT 与纯 IoT 的区别**：纯 IoT 只负责"连接和上报"，AIoT 在平台层和端侧加了 AI 推理，能本地决策、预测性维护、异常检测，不只是把数据传到云端让人看。

### 三、AIGC 系统架构（拓展了解，别过度投入）

**重要提示**：AIGC 在综合知识中最多 1 题常识题，培训机构估算非官方。掌握下面三个架构名词的"是什么 + 解决什么问题"即可，不必深挖数学原理。

#### 1. 大模型推理资源调度

大模型推理的瓶颈不是算力总量，而是**显存**和**调度效率**：

- **算力调度**：多请求多模型共享 GPU 集群，按优先级和 SLA 分配，类似操作系统的进程调度但粒度更细。
- **批处理（Batch）**：把多个推理请求合并成一批一起算，提高 GPU 利用率。代价是批内最长请求决定整体延迟，要在吞吐和延迟间权衡。
- **显存管理 KV Cache**：Transformer 推理时每生成一个 token 都要存历史 token 的 Key/Value，叫 KV Cache，随上下文长度线性增长，是显存大头。管理不好就 OOM。

**类比**：算力调度像餐厅排座，批处理像拼桌（一桌坐满才上菜效率高但等位久），KV Cache 像每个客人占的桌面面积（聊得越久占得越大）。

#### 2. RAG 检索增强生成

大模型两个老毛病：**知识截止**（训练完之后的事不知道）和**幻觉**（一本正经胡说）。RAG（Retrieval-Augmented Generation）治这俩病。

**RAG 三段流程**：
1. **检索（Retrieve）**：用户问题先去**向量库**（Vector Database，把文档转成向量存起来）里找最相关的片段。代表向量库 Milvus、Pinecone。
2. **增强（Augment）**：把检索到的片段拼进 Prompt，给模型当参考资料。
3. **生成（Generate）**：大模型基于"问题 + 参考资料"生成答案。

**类比**：RAG 像开卷考试——不让模型凭记忆答（容易记错），而是先翻书（检索）再照着书答（生成）。向量库就是那本"按语义找内容"的索引手册。

**记忆要点**：RAG = 向量库 + 检索 + 生成，解决知识时效性和幻觉。

#### 3. Agent 智能体架构

Agent（智能体）= 大模型 + 工具调用 + 自主规划，能完成多步任务，不只是回答问题。

**四要素架构**：
- **感知（Perception）**：接收用户指令或环境信号
- **规划（Planning）**：把大目标拆成子任务（如 ReAct、思维链 CoT）
- **行动（Action）**：调用工具（搜索、代码执行、API）
- **记忆（Memory）**：短期对话上下文 + 长期向量记忆

**类比**：普通大模型像只会背书的实习生，你问一句答一句；Agent 像能上网、会查资料、会拆任务的助理，你说"帮我做个调研报告"，它自己规划步骤、调工具、整合输出。

**记忆口诀**：Agent 四件套——**感知收指令、规划拆任务、行动调工具、记忆存上下文**。

## 实践

### 真题示例

**2024 年·综合知识·边缘计算相关（考生回忆）**：考察边缘计算相比云计算的优势，正确项指向低延迟、节省带宽、数据隐私。

**2025 年·综合知识·AIoT 相关（考生回忆）**：考察物联网四层架构归属，传感器、RFID 属于感知层。

### 答题公式

**边缘计算场景题踩分词**（写到就有分）：
1. **云-边-端协同**三层
2. **低延迟 / 省带宽 / 保隐私**三大优势
3. **数据本地处理**，敏感数据不出域
4. 边缘做**实时推理**，云端做**模型训练和全局调度**

**AIoT 案例论文应用模板**（智慧工厂场景）：
- 感知层：产线传感器 + 工业相机采集振动、温度、图像
- 网络层：5G + 工业以太网回传
- 平台层：边缘节点做实时质检推理（缺陷检测），云端做模型训练和全局看板
- 应用层：MES 对接、预测性维护告警
- 效果：质检从云端回传秒级延迟降到边缘毫秒级，带宽节省 70%

**AIGC 常识题答题要点**：
- RAG 解决什么：知识时效 + 幻觉
- RAG 三段：检索 → 增强 → 生成
- Agent 四要素：感知、规划、行动、记忆
- 大模型推理瓶颈：显存（KV Cache）+ 调度效率（Batch）

### 易错点

1. **边缘计算不是替代云计算**——是协同，边缘管实时、云端管训练和全局，别答成"边缘取代云"。
2. **AIoT 四层不要漏层或错位**——感知/网络/平台/应用，传感器在感知层，5G 在网络层，规则引擎在平台层。
3. **RAG 不是微调（Fine-tuning）**——RAG 是推理时检索外部知识，不改模型权重；微调是改权重。两者解决相似问题但机制不同。
4. **Agent ≠ 大模型**——大模型是 Agent 的"大脑"，Agent 还要有工具和规划，能多步行动。

## 速查/自测

### 30 秒速查卡

| 考点 | 一句话 |
|---|---|
| 边缘 vs 云 | 边缘低延迟 / 省带宽 / 保隐私，算力弱 |
| 云-边-端 | 云训练调度、边实时推理、端采集执行 |
| AIoT 四层 | 感知 / 网络 / 平台 / 应用 |
| 感知层设备 | 传感器、RFID、摄像头 |
| RAG 三段 | 检索 → 增强 → 生成 |
| RAG 解决 | 知识时效 + 幻觉 |
| Agent 四要素 | 感知、规划、行动、记忆 |
| 大模型推理瓶颈 | 显存（KV Cache）+ 调度（Batch） |

**口诀**：边缘三宝低省保（低延迟、省带宽、保隐私），物联四层感网平用，RAG 检增生治幻觉，Agent 感规行记四件套。

### 自测题

1. 自动驾驶为什么必须用边缘计算而不能全靠云计算？
   > 车辆控制要求毫秒级响应，数据往返云端延迟不可接受；且每天产生海量数据全上传带宽吃不消；部分行车数据涉及隐私不宜出车。

2. 物联网四层架构中，5G 和规则引擎分别属于哪层？
   > 5G 属网络层（负责传输），规则引擎属平台层（负责数据处理和设备管理）。

3. RAG 和模型微调都能让大模型用上新知识，核心区别是什么？
   > RAG 是推理时检索外部知识拼进 Prompt，不改模型权重，知识可实时更新；微调是改模型权重，更新成本高、周期长，但运行时不依赖检索。

**一句话小结**：边缘计算把算力推到数据旁边换低延迟，AIoT 给物联网四层装上 AI 大脑，AIGC 三架构（推理调度 / RAG / Agent）是 2026 拓展常识——记住名词和它解决什么问题就够，别陷进数学。
