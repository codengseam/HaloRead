# 决策之道 · 写作提示词与执行方案存档

> 本文件为专栏写作过程使用的提示词、执行方案与验收记录存档，不进入专栏展示（下划线前缀文件被 check_book_structure.py 豫免、被 build_site.py 排除出站点）。保留用于复盘与迭代。所有资料基于 2026-07 WebSearch 核验与既有出版物。

## 一、专栏定位

- 主题：重大人生决策方法论——道的层面
- 读者：一个快三十岁、过去因情绪冲动和盲从做出多次后悔决策的人（花钱找婚介当托、买房买车亏几十万、报名包过考研班被骗、借钱给朋友不还、买基金亏了卖飞了、现在面临换工作和结婚）
- 核心目标：建立一套"道"层面的决策方法论，让读者在重大决策时做出当下正确且事后不后悔的选择。讲层次要深，是道的层面，不是术的罗列
- archetype：modern（现代方法论桶）
- 输出目录：`/workspace/output/决策之道/`
- 输出：25 篇正文 + 3 个辅助文件（_meta.yaml / _参考资料.md / _提示词.md）

## 二、执行流程（已落地）

1. **多专家团评估方案**：用 Task 工具启动 3 个 subagent（架构师/内容专家/规则专家）并行评审大纲+调研补全资料，整合意见后定稿。关键修订：22→25篇（新增复盘篇、修心之道扩为6篇、认知之镜新增情绪劫持篇、决策之框合并为6篇）
2. **并行调研补全资料**：3 个 subagent 共核验 71 条资料（东方修心24+决策框架23+认知防骗投资24），发现 6 类伪语录/误引并建立规避清单
3. **写资产文件**：创建 _meta.yaml（6 chapters 映射）+ _参考资料.md（资料存档）+ _提示词.md（执行方案存档）
4. **并行写作**：用 Task 工具启动多波 subagent 并行写作 25 篇
5. **严格质检**：跑 check_book_structure.py --strict + 内容质检 + 一致性检测
6. **三方评审 + 修复**：3 个评审 subagent 评审，修复硬伤后复跑质检

## 四、写作硬约束（写入每个写作 subagent 的 query）

1. 目录：`/workspace/output/决策之道/`
2. 文件命名：两级路径 `{chapter}_{event}.md`，中文，下划线 `_` 分隔，**禁止用 `-`**，禁止子目录。event 内不得再出现 `_`（第一个下划线后全是 event）
3. 每篇必须有完整 frontmatter 八字段：
   ```
   ---
   title: xxx（含章节·副标题格式，如"开篇·为什么总在后悔"）
   book: 决策之道
   chapter: 开篇
   event: 为什么总在后悔
   sort: 1
   chapter_sort: 1
   created_at: 2026-07-08
   source_agents:
     - decision-writer
   ---
   ```
4. archetype: modern（现代方法论桶），写入 _meta.yaml 即可，不写入每篇 frontmatter
5. 正文中文，每篇 2500-4000 字
6. **五段固定结构** + 中文序号 + 冒号 + 灵魂短语副标题：
   - `## 一、入戏：{灵魂短语}` —— 用读者一个具体失败决策场景开场（300-500字）
   - `## 二、破题：{灵魂短语}` —— 拆解问题本质，引出本篇核心思想（500-700字）
   - `## 三、方法论：{灵魂短语}` —— 核心方法/模型，配真实案例（700-1000字）
   - `## 四、避坑：{灵魂短语}` —— 本篇方法最常见的3个误用（400-600字）
   - `## 五、践行：{灵魂短语}` —— 本周可做的1-2个最小动作（300-500字）
   - 五段内分小节一律 `###`，禁止再起 `##`
7. 文末必须有 `## 参考来源`（**精确这四字**，不是"参考资料""参考文献""本章参考资料"，content_quality.py 只认"参考来源"）
8. **禁内联跳转**：禁用一切括号跳转，包括"（见XX篇）""（详见XX章）""（见上文/前文）""（XX篇展开）"等所有带括号的篇/章提示；提及他篇必须融入自然语句，如"后面XX篇会专门讲""XX篇展开过这个机制"，不得用括号包裹
9. **禁用"底层逻辑""底层操作系统"**（0 容忍，出现即报）；"底层"二字单独可用但建议替换为"机制/原理/根本/底子/根基"
10. 术语词序铁律："英文+中文"词序（如「WRAP 决策流程」），首次出现用全角括号隔断「中文全称（缩写）」，之后用中文
11. 子标题/小节用"名词：洞察短语"格式，禁裸2字词
12. 每篇开头一句边界声明（blockquote）：`> 本篇讲 X，不讲 Y`（不附任何括号跳转）
13. 文末参考来源后加 `> 信息截止：2026-07`
14. **引号强制用「」**，禁用 ASCII 双引号包中文
15. **「不是X而是Y」句式每篇 ≤3 处**，改写多样化（与其说X不如说Y / 关键不在X而在Y / X是表象Y是内核 / 真正的问题不是X，是Y）
16. **行内引用每千字 ≤3 处**，同一出处连续出现只保留一处，普通叙述句不挂行内出处
17. **引用真实性**：只引用本文件第六节核验过的资料，不杜撰书籍版本/作者/出版社/年份/数据；名人语录存疑不引；古文必须标注准确出处（章/篇）
18. **跨篇协调**：金句/案例/理论只在归属篇完整展开，其余篇一笔带过（见第五节跨篇协调表）
19. 现代专栏放宽：行业通用词白名单（KPI/HR/offer/bug/OKR/CEO 等不算中英文混杂）；"容易被忽略/可见/这说明"等常见中文不计 AI 味
20. 禁出现"我们可以看到""这告诉我们""综上所述""总而言之""众所周知"等 AI 套路句式
21. 每批次撰写后必须 Glob/LS 核对实际产出文件数 = 预期（subagent 输出丢失先例）
22. 数据/年份/人名必须与 _参考资料.md 核验一致，禁止自行编造

### D. 骗局话术→心理机制映射（实战篇写作素材）

- 红娘：「你都这么大年纪了」→损失厌恶+稀缺；「80%成功率」→虚构社会认同；密闭格子间→系统2耗竭；「看人下菜碟」→锚定；「结婚返利」→沉没成本绑架
- 考研班：「交5万保过985不过全退」→双重锚点+沉没成本；「通过率95%」→虚构社会认同+权威；23项条款→退费门槛；「退费60工作日」实6-12月→拖延耗损
- 杀猪盘：「茫茫人海相遇命中注定」→喜好+联盟；「内部漏洞稳赚不赔」→稀缺+权威；小额试水伪造盈利→间歇性强化；「这点钱都不敢投」→激将；提现受阻「充值刷流水」→沉没成本深陷
- 借钱：「就周转几天发工资马上还」→互惠+喜好；借钱低声下气要钱趾高气昂→认知失调；「都是朋友这点钱还催」→道德绑架框架效应；小借款按时还→大借款→破窗效应
