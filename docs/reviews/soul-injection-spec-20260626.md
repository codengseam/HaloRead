## 一、核心目标

把 HaloRead 内容从"显性合规、隐性失准的及格 AI 产物"提升到"有灵魂、有史观穿透、能让人记住"的顶流水准。

**判别标准**：盲测时，读者能感受到新版有当年明月那种"把历史人物当活人写、残酷是内核、史观穿透而非史料堆砌"的质感，而旧版没有。

**反标准（什么不算成功）**：
- 仅仅删了 AI 套话、降了引用密度 → 这是合规，不是灵魂。
- 仅仅换了风格词、更口语化 → 这是皮相，不是灵魂。
- 洞察仍然套在任何"刚直悲剧"人物上都成立 → 这是正确废话，不是独家。

## 二、当年明月三约束（最高优先级，违反即重写）

### 约束 1：活人测试

**禁止**：使用"奸臣/忠臣/伟人/小人/刚峰/仁弱书生"等脸谱化标签定义人物。

**必须**：写出人物在特定局势下的"两难、挣扎、恐惧与无奈"。

**海瑞上疏的具体要求**：
- 旧版写法："海瑞的刚，在于他不怕得罪人"——这是标签。
- 新版要求：写出海瑞备棺那夜的犹豫。他不是不怕死，他是怕死但不能不死——因为他是海瑞，他这一辈子就是靠"刚"立住的，他一旦软了，他自己就塌了。这份"不能不刚"比"不怕死"更残酷。
- 旧版写法："嘉靖帝的复杂，在于他聪明，可聪明用错了地方"——这是正确废话。
- 新版要求：写出嘉靖看完疏后的那个"默然"。他怒，要抓海瑞；黄锦一句话，他默然。这个默然里有什么？有他被戳穿的羞耻，有他知道海瑞是对的却不能认的窝囊，有他想杀海瑞又怕成纣王的算计。一个皇帝被一个六品官骂到"默然"，这份默然比任何暴怒都重。

### 约束 2：残酷底色

**禁止**：涉及生死、党争、国运节点时使用轻佻网络用语和上帝视角戏谑。

**必须**：体现命运的不可抗力与历史的残酷性。

**海瑞上疏的具体要求**：
- 海瑞上疏不是爽文。一个六品官骂皇帝，备了棺材，妻儿散了，僮仆跑了——这不是英雄时刻，这是孤独到极致的时刻。
- 狱中闻嘉靖驾崩，海瑞放声大哭，把吃下去的都吐出来——这个"吐"字是残酷的核心。他骂了嘉靖一辈子，嘉靖死了他却哭到吐。这份哭不是忠，是他突然发现：他这一辈子的敌人没了，他这一辈子的意义也悬空了。这才是残酷。
- 海瑞死于万历十五年，家无余财，连丧葬费都是同僚凑的——这个结尾不能写成"清官的悲壮"，要写成"道德在制度面前的彻底失效"。他守了一辈子道德，道德救不了他，也救不了明朝。

### 约束 3：史观穿透

**禁止**：只讲"发生了什么"，提炼"放之四海皆准"的正确废话（如"道德是底线不是工具""极言能戳心"）。

**必须**：讲透"为什么只能这样发生"，提炼"具体的、独家的、反直觉的"洞察。

**海瑞上疏的具体要求（这是旧版最弱的一环）**：
- 旧版问道悟道四条："极言之功在于备死""道德是底线不是工具""权势怕极言""清官是标杆不是解药"——这四条**套在任何刚直悲剧人物上都成立**（比干、杨椒山、杨涟），判 0 分。
- 新版必须提炼**只能从海瑞身上得出的洞察**。候选方向（不是定论，是 ToneSetter 要深挖的）：
  - **海瑞的不可能性**：海瑞不是"想用道德解决制度问题"（黄仁宇的论，旧版照搬），而是他**除了道德没有别的武器**。举人出身、没有师生网络、没有家族背景，他唯一能调动的资源就是"我比你干净"。这份"只有道德"不是选择，是处境。这个洞察是海瑞独有的。
  - **备棺的双重意义**：备棺不是"以死相谏"那么简单。备棺是海瑞把自己**从活人变成象征**的仪式。活人可以谈判，象征不能。海瑞备棺的那一刻，他就把自己逼到了"不能回头"的位置——这跟项羽破釜沉舟是同构的，但项羽是为了赢，海瑞是为了**输得有意义**。
  - **嘉靖不杀海瑞的真正原因**：不是黄锦那句话，不是嘉靖有良知。是嘉靖**不能杀**。杀了海瑞，海瑞成烈士，嘉靖成纣王；不杀，海瑞坐牢，嘉靖还是被骂的皇帝。嘉靖选了后者，是两害相权取其轻。这个算计，比"良知"更真实，也更残酷。

## 二（补）·约束 4：网文叙事技巧（起承转合 + 埋钩子）

当年明月之所以火，除了史观，还有"读起来停不下来"的节奏。v1.0 盲测暴露的问题：灵魂注入了但事件稀薄、节奏平。必须用网文技巧补丰满度和节奏感。

### 4.1 起（开篇钩子）

用反常细节或悬念开场，3 句话内勾住读者。

- 反例（旧版）："海瑞是海南琼山人，回族"——平铺直叙，没有钩子。
- 正例："嘉靖四十五年二月，北京城里还有寒气。户部主事海瑞在灯下坐了很久，他做了一个决定——买棺材。"

### 4.2 承（铺垫期待）

每段结尾埋下一个"为什么"或"接下来怎样"的钩子，让读者想翻下一段。

- 例："胡宗宪咽下了。可胡宗宪咽得下，严嵩的党羽咽不下。"——埋下"严嵩党羽会怎样"的钩子。
- 例："海瑞被下了狱。他等着死。"——短句断行，制造"等死"的悬念。

### 4.3 转（反转与反差）

在读者以为要往 A 走时，往 B 走。用"可""然而""偏偏"制造转折。

- 例：海瑞备棺等死→嘉靖死了他却哭到吐（读者预期他高兴，实际他崩溃）。
- 例：海瑞清廉一辈子→死后丧葬费都是同僚凑的（反差）。

### 4.4 合（收束留味）

结尾不直接说教，留一个意象或反问让读者自己想。

- 例："他是海瑞。他要是变了，他就不是海瑞了，他什么都不是了。"

### 4.5 章回体悬念

允许在段落间用过渡句制造翻页感，模仿当年明月、蔡东藩的章回体节奏。

- 例："可海瑞是怎么走到这一步的？得从淳安说起。"
- 例："这一默然，比任何暴怒都重。"
- 例："海瑞出狱了。可故事没完。"

### 4.6 反差对比

用身份/境遇/结果的强烈反差制造戏剧张力。

- 例：六品官 vs 皇帝（身份反差）。
- 例：二斤肉 vs 惊动总督（结果反差）。
- 例：备棺等死 vs 出狱起复（境遇反差）。
- 例：道德守一生 vs 丧葬费靠凑（理想 vs 现实反差）。

### 4.7 适用边界

网文技巧服务于"读起来停不下来"，**不能滑向爽文化**。约束 2（残酷底色）仍为最高优先级——不能用网文的"爽"消解历史的"残酷"。两者的平衡点：节奏快、钩子多、转折狠，但底色克制、不戏谑、不爽化。

## 三、打破六段式 → 逻辑流

### 现状问题

`rules.md` 第 7 行："每篇笔记固定五段正文加一段结语，顺序不可调换"。

这条规则导致：
- 讲事情/讲人物/讲背景被物理割裂，同一事件要重复讲三遍。
- 讲道理/问道悟道被逼成独立模块，硬凑正确废话。
- 当年明月笔法是夹叙夹议，议论散在叙事里，不是单独成段。

### 新规则：逻辑流引导

**保留**：frontmatter 八字段、文末参考来源、结语一句话——这些是工程规范，不动。

**打破**：物理六段式。改为"逻辑流引导"：
1. 允许夹叙夹议，人物与事件揉碎在时间线中交织。
2. 议论化为散落在文中的"太史公曰"式金句点评，不单独成段。
3. 仅在结尾做自然收束（结语保留）。
4. **篇幅不设上限**：以"读起来停不下来且人物立体"为唯一标准。v1.0 设的 1500-2500 字上限导致事件被砍、人物单薄，已废弃。
5. **事件丰满度硬约束**：旧版的关键事件一个不能少（教谕不跪/严嵩背景/鄢懋卿巡盐/胡公子/二斤肉/备棺/上疏/黄锦/默然/下狱/闻崩吐/应天退田/万历十五年之死），用逻辑流融入而非单独成段。
6. **允许扩充新史料**：不只限于旧版内容，可引入《明实录》《万历起居注》《国榷》等史料中的新素材，让人物和事件更立体。

### 海瑞上疏的逻辑流设计

不再按"讲事情→讲人物→讲背景→讲道理→问道悟道→结语"线性推进，而是按**海瑞的命运弧线**推进：

1. **开篇**：备棺那夜（场景切入，不是编年切入）——海瑞买棺材、诀妻、遣仆，写出"不能不刚"的处境。
2. **上疏**：上疏内容 + 嘉靖的默然（事件 + 人物心理交织）——写出"戳心"和"不能杀"的算计。
3. **前史**：淳安知县惩胡公子、买二斤肉（插叙，解释海瑞为什么是海瑞）——写出"只有道德"的处境根源。
4. **狱中**：闻嘉靖驾崩大哭到吐（高潮）——写出残酷底色。
5. **结局**：万历十五年海瑞之死（收束）——写出道德在制度前的失效。
6. **结语**：一句话。

议论（讲道理）散在 2、3、5 节里，不单独成段。名家点评（黄仁宇、谷应泰、《明史》赞语）化为行文中的引用，不堆成"讲道理"段。

## 四、ToneSetter 定调节点设计

### 架构位置

```
Orchestrator → ToneSetter(串行) → 5 Specialist(并行) → Editor → 合规质检 → 总编 → Save
```

ToneSetter 在 Orchestrator 之后、5 Specialist 之前，串行执行。

### ToneSetter 职责

阅读原始史料（用户输入的书名/章节/事件 + Agent 检索的史料），输出 **300-500 字的《本篇核心史观与情感基调大纲》**，包含：

1. **核心史观**（1 句话）：本篇要讲透的、只能从这个事件得出的独家洞察。
2. **情感基调**（1 句话）：冰冷/悲悯/激昂/讽刺……（允许逐篇换调，不强制统一）。
3. **核心冲突**（1-2 句）：本篇的人物两难是什么。
4. **灵魂锚点**（2-3 个）：本篇必须写到位的具体场景/细节（如海瑞的"备棺那夜""狱中吐""万历十五年之死"）。
5. **风格锚点**：对标当年明月哪一篇的笔法（如"对标《海瑞的孤独》的悲悯+冷峻"）。

### ToneSetter 输出注入方式

将"基调大纲"作为 System Context 强制注入后续 5 个 Specialist，确保所有人"带着同一个灵魂写作"。

### 海瑞上疏的 ToneSetter 预期输出（示例）

```
核心史观：海瑞的不可能性——他不是选择道德，他是除了道德没有别的武器。
情感基调：悲悯 + 冷峻。悲悯是对海瑞的孤独，冷峻是对制度的不留情。
核心冲突：海瑞想用道德撬动皇权，可道德撬不动皇权，他只能用命去撬。
灵魂锚点：①备棺那夜的"不能不刚"；②嘉靖默然里的"不能杀"；③狱中闻崩大哭到吐；④万历十五年家无余财之死。
风格锚点：对标当年明月《海瑞的孤独》，冷峻叙述 + 悲悯收束。
```

## 五、总编 Agent（Chief Editor）终审设计

### 架构位置

合规质检（现有 content_reviewer_sub + quality.py）通过后，总编 Agent 做灵魂终审。

### 总编职责（区别于现有 content_reviewer_sub）

**不做**：不查错别字、不查引用密度、不查 frontmatter（那是合规质检的活）。

**只做**：回答三个问题（灵魂三问），输出 GO / REWORK（附具体重做方向）。

#### 灵魂三问

1. **活人测试**：本篇的核心人物，读者读完能不能用一句话说出他的"两难"？
   - 能 → 通过。
   - 不能（人物仍是标签）→ REWORK，方向："重写 [人物] 段，写出他在 [具体场景] 的 [两难/恐惧/无奈]"。

2. **洞察独家性**：本篇提炼的核心洞察，套在别的刚直悲剧人物（比干/杨椒山/杨涟）上成不成立？
   - 套不上，只能从本事件得出 → 通过。
   - 套得上 → REWORK，方向："洞察 [XXX] 是正确废话，重挖只能从 [本事件] 得出的独家规律"。

3. **底色敬畏感**：面对生死与历史悲剧，叙述语气是否保持了克制与悲悯？有没有不合时宜的戏谑、轻浮或爽文化？
   - 克制 → 通过。
   - 有戏谑/爽文化 → REWORK，方向："[具体段落] 语气轻浮，重写为克制叙述"。

### 总编输出格式

```json
{
  "verdict": "GO" | "REWORK",
  "soul_questions": {
    "live_human_test": {"pass": true, "reason": "..."},
    "insight_exclusivity": {"pass": false, "reason": "..."},
    "tone_reverence": {"pass": true, "reason": "..."}
  },
  "rework_direction": "若 REWORK，给出具体重做方向（null 若 GO）"
}
```

### 阈值校准（试点期）

试点期（海瑞上疏首篇）总编**只打标记不强制打回**，跑 5 篇后校准阈值。校准通过后再改为"任一问 fail 即 REWORK"。

## 七、盲测标准

### 盲测方式

1. 旧版 B（现有海瑞上疏）和新版 A 隐去版本信息。
2. 同时呈现给人类评审（你）。
3. 对照当年明月《明朝那些事儿》海瑞章节，回答以下问题：

### 盲测问题清单

1. **活人感**：哪一版的海瑞更像"活人"，哪一版更像"标签"？
2. **残酷底色**：哪一版让你感受到海瑞的孤独和历史的残酷，哪一版只是在叙述事件？
3. **洞察独家性**：哪一版的洞察是"只能从海瑞得出"的，哪一版是"套在哪都成立"的？
4. **可截图句**：哪一版有让你想截图传播的句子，哪一版没有？
5. **整体体感**：哪一版更像当年明月，哪一版更像 AI 流水线？

### 通过标准

- 问题 1-4 中新版胜出 ≥3 项 → 灵魂注入成功，固化方案。
- 新版胜出 <3 项 → 灵魂注入失败，复盘迭代 spec。
- 新版胜出 = 3 项但有 1 项旧版明显更优 → 部分固化 + 针对性修复。

## 九、风险与回滚

### 风险

1. **打破六段式破坏存量一致性**：686 篇存量仍是六段式，新版不是。→ 缓解：只在 feature 分支试，不动 master 存量；盲测通过后再决定是否存量清洗。
2. **总编一票否决误杀**：试点期只打标记不强制打回，跑 5 篇校准阈值。
3. **ToneSetter 串行拖慢量产**：试点期不计较速度，固化后再优化（可考虑缓存）。
4. **灵魂注入后反而不"合规"**：新版可能引用密度变化、段落结构变化，触发现有质检。→ 缓解：合规质检规则同步更新（引用密度上限放宽、六段式不再强制）。

### 回滚

- feature 分支不合并即回滚，master 零影响。
- 已删的 7 个明纪重复文件可从 git 历史恢复。

## 十、superpowers 适配复盘（LoopAgent 沉淀素材）

本次执行 superpowers 的适配结论：

### 可直接用（5 个，纯方法论）

- `test-driven-development`：先写失败测试再改。→ 应用：灵魂三问可写成 pytest 断言。
- `verification-before-completion`：证据先于断言。→ 应用：盲测必须有你的人类判断证据，不能我自说自话。
- `systematic-debugging`：无根因不修。→ 应用：盲测失败时先走根因调查，不盲目迭代。
- `receiving-code-review`：拒绝表演式赞同。→ 应用：你对 spec 的反馈我如实接受，不拍马屁。
- `writing-plans`：bite-sized 任务、无占位符。→ 应用：本 spec 的执行步骤要细到可验证。

### 可移植（2 个，改写为 Trae 原生）

- `writing-skills`：SDO（description 只写 when 不写 what）。→ 应用：审视 HaloRead 现有 7 个 .trae/skills/ 的 description 质量。
- `brainstorming`：分节设计审批门。→ 应用：dev-workflow 第一阶段补"需求细化→分节确认"。

### 仅参考（7 个，依赖 Trae 不支持的能力）

> **2026-06-27 复核更新（BUG-031）**：`dispatching-parallel-agents` 已原生化为 `.trae/skills/dispatching-parallel-agents/SKILL.md`，适配 Trae `Task` 工具（`subagent_type` 参数）。原判定"依赖原生 subagent 调度，Trae 做不到"已过时——Task 工具支持会话内启动 subagent。下方 6 个技能的"仅参考"判定不变。

- ~~`dispatching-parallel-agents`~~：**已原生化**（见 `.trae/skills/dispatching-parallel-agents/SKILL.md`），作为 `plan-review` skill 并行评审主路径的调度底座。
- `requesting-code-review` / `subagent-driven-development`：仍依赖 Trae 不支持的原生 subagent 调度形态，HaloRead 走 LangGraph 路径 B 或 Task 工具路径 C 替代。
- `using-superpowers`：依赖 session-start hook，Trae 做不到。
- `using-git-worktrees`：HaloRead 单仓库，引入 worktree 属过度工程化。
- `finishing-a-development-branch` / `executing-plans`：与 HaloRead 已有 git-merge-guardian + dev-workflow 重叠。

### push 到 GitHub 的筛选原则

只有"用过且有正向提升"的 superpowers 改造才 push：
- 5 个可直接用的技能 → 如果在灵魂注入中实际遵循并有效，将其核心纪律写入 `.trae/rules/` 后 push。
- 2 个可移植的技能 → 如果改写成 Trae 原生 skill 后有效，push 新 skill。
- 7 个仅参考的 → 不 push（vendor/superpowers/ 保留但不进 .trae/skills/）。
