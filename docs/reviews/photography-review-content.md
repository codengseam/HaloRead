### 2. 镜头型号/卡口/等效焦距（镜头选购篇）：✅ 准确

### 3. ISO/快门/对焦参数：❌ 发现 4 处参数错误

**错误 1（严重·感光度篇第36行）**：动态范围档位张冠李戴。
- 原文：「ISO 100 时 90D 动态范围约 13.5 档（DxOMark），**ISO 1600 时降到约 10.2 档**，到 ISO 6400 时只剩约 8 档。」
- 正确值：约 10.2 档对应 **ISO 3200**，不是 ISO 1600。
- 证据：① 参数基石第41行「ISO 3200 | 噪点开始明显…动态范围降至约 10.2 档」；② 同篇第50行自相矛盾「ISO 3200：噪点开始明显，动态范围降至约 10.2 档」；③ `人像实拍_夜景人像.md` 第106行「ISO 1600 仍保持良好动态范围」佐证 ISO 1600 并非 10.2 档。
- 影响：ISO 1600 是 90D「可用档位上限」，把它的动态范围说成降到 10.2 档会误导读者以为 ISO 1600 已严重劣化，与同篇「ISO 1600 可用上限」结论自相矛盾。

**错误 2（严重·感光度篇第95行）**：虚构 ISO 50 扩展档。
- 原文：「ISO 50、H:51200 是扩展档位，画质反而比相邻档差。」
- 正确值：90D **没有 ISO 50 扩展档**。90D 原生 ISO 100-25600，仅可向上扩展 H:51200，不能向下扩展 ISO 50。
- 证据：① `风光旅游_水景长曝光.md` 第117行明确写「ISO 100 是 90D 的最低原生 ISO（不可扩展到 ISO 50）」——本专栏内部已自证 90D 无 ISO 50；② Canon 官方规格 90D ISO 范围仅标注 H（51200），无 L/50 扩展。
- 影响：把不存在的档位列为「扩展档」，新手会去菜单里找 ISO 50 找不到，且误导「ISO 50 画质更差」的伪结论。

**错误 3（概念性·感光度篇第89-97行）**：把其他机型特性套用到 90D。
- 原文：「实际最干净的 ISO 档位是 160…ISO 160 时相机从原生 ISO 200 往下拉一档…这种『原生 ISO 的整数倍』现象在很多相机上都存在。」
- 问题：90D 的**基准（原生）ISO 是 100，不是 200**。90D 不存在「从原生 ISO 200 向下扩展到 160」的机制。「ISO 160 比 ISO 100 更干净」是早期部分 Canon 机型（如 5D Mark II 时代，原生 ISO 160）的传说法门，对 90D（DIGIC 8 时代基准 ISO 100）不成立，且缺乏 90D 实测数据支撑。
- 影响：鼓励新手用 ISO 160 拍风光精修，违背 90D「ISO 100 画质最优」的事实，与参数基石「ISO 100-400 画质优秀，噪点极低」的结论冲突。

**错误 4（数据·黄金时刻与蓝调篇第60行）**：Live View DPAF 对焦亮度范围写错。
- 原文：「90D 的 Live View 全像素双核 CMOS AF 在弱光下表现优秀，**最大 EV -3 仍可对焦**。」
- 正确值：Live View DPAF 对焦亮度范围是 **EV -5 至 18**，即最暗可到 EV -5，不是 EV -3。
- 证据：① 参数基石第21行；② `器材操作_90D机身熟悉.md` 第35/65/71 行均明确「Live View 亮度范围 EV -5 至 18」「EV -5 比 OVF 的 EV -3 低两档」；③ `对焦与测光_对焦模式与双核AF.md` 第50行「对焦亮度范围 EV -5 至 18」。
- 影响：EV -3 是 OVF 相位对焦的下限，本篇把 OVF 的下限误植到 Live View DPAF 上，恰恰抹平了 90D「Live View 比 OVF 更暗还能对焦两档」的核心优势，与该篇「Live View 在弱光下表现优秀」的论点自相矛盾。

- 问题清单：
  1. `曝光三要素_感光度.md` 第36行：ISO 1600 动态范围 10.2 档 → 应为 ISO 3200（篇内自相矛盾）
  2. `曝光三要素_感光度.md` 第95行：ISO 50 扩展档不存在 → 90D 无 ISO 50，应删除（与水景篇矛盾）
  3. `曝光三要素_感光度.md` 第89-97行：ISO 160 最干净 / 原生 ISO 200 → 90D 基准 ISO 是 100，此说法不适用
  4. `光影运用_黄金时刻与蓝调.md` 第60行：Live View DPAF EV -3 → 应为 EV -5（与机身/对焦篇矛盾）

## 四、无虚构数据（7分）

### 1. 定价区间：✅ 合理

接单定价篇给出的品类定价区间符合 2026 年上海主流市场行情：
- 个人写真 800-2,500 元/次（含精修 8-15 张）—— ✅ 合理。
- 亲子/孕妇/百天 1,500-4,000 元/次—— ✅ 合理。
- 旅行跟拍 城市内 800-2,000 元/小时或 1,500-3,500 元/次—— ✅ 合理。
- 轻婚纱 4,000-8,000 元/次—— ✅ 合理。
- 婚纱三档（5,699-7,999 / 8,000-10,999 / 11,000-12,999 元）—— ✅ 区间结构合理。
- 「精修单价反推法」公式与「单张隐含成本 80-150 元正常区间、200 元以上陷阱区间」—— ✅ 逻辑自洽，有方法论价值。
- 城市差异折算系数（一线 800-2,500 / 二线 600-1,800 / 三四线 400-1,200）—— ✅ 合理。

### 2. 客单价曲线：❌ 数据严重失真，疑似虚构或误引

**问题（接单定价篇第37行）**：
- 原文：「2023 年约 9,800 元 → 2025 年约 6,800 元（价格战回落后） → **2026 年第一季度回升至约 18,500 元**，中高端占比突破 60%。」
- 问题一（内部矛盾）：同篇三档结构最高档为 11,000-12,999 元，**三档均价不可能达 18,500 元**——若半定制档封顶 12,999 元，平均客单价却 18,500 元，数学上不成立（除非存在未列出的第四档高端定制，但文中未交代）。
- 问题二（涨幅失真）：2025 年 6,800 元 → 2026 年第一季度 18,500 元，**一个季度客单价上涨约 2.7 倍**，在任何正常市场都不可能出现。即使考虑「中高端占比突破 60%」的结构性上移，加权平均也不可能在 3 个月内从 6,800 跃升到 18,500。
- 问题三（来源未闭合）：参考来源「上海婚嫁行业协会 2025-2026 消费报告（待核实具体 URL）」明确标注待核实，该 18,500 元数字的出处未闭合核验。
- 结论：该数据疑似虚构或对源数据误读（例如把「高端档位客单价」误写成「平均客单价」，或把「全年累计」误写成「季度均值」）。属严重的「无来源支撑的虚构数据」问题。

### 3. 引流价陷阱警示：✅ 合理

「3,000-5,000 元婚纱全包在 2026 年上海已基本退出主流」「AI 旧数据残留的『3000 元拍婚纱』在 2026 年上海已不是行情，是营销话术」—— ✅ 判断方向合理，且明确区分了「行情」与「营销话术」，未把营销话术当数据。

### 4. 参考来源标注：⚠️ 多处「待核实」

接单定价篇参考来源中：
- 「上海婚嫁行业协会 2025-2026 消费报告（待核实具体 URL）」
- 星空摄影篇：「Sequator 堆栈降噪软件 http://www.starrover.net/sequator/ （待核实当前 URL）」——该 URL 指向 starrover.net，疑为另一款软件（Star Rover）域名，Sequator 官方发布页实际在 Google Sites，此处 URL 存疑。

虽然「待核实」标注体现了作者的诚实，但「待核实」项被作为正文数据的支撑来源仍属未闭合核验。

- 问题清单：
  1. `商业变现_接单定价.md` 第37行：2026 年第一季度客单价 18,500 元——与同篇三档结构（最高 12,999 元）矛盾，涨幅（6,800→18,500）不合理，来源未闭合。建议改为「2026 年中高端档位客单价约 18,500 元」或删除该具体数字，改用区间表述。
  2. `商业变现_接单定价.md` 参考来源：上海婚嫁行业协会消费报告 URL 待核实——请补充可访问 URL 或改为「作者整理自上海婚嫁行业协会公开数据」。
