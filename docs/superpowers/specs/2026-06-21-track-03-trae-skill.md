# 子计划 3：Trae Skill 交互入口

## 上下文

本项目是个人 AI 深度阅读助手，核心引擎已经能通过命令行接收输入并生成 Markdown。本轨道的目标是把核心引擎包装成 Trae Skill，让用户在 Trae 对话框中直接触发。

详见：
- [总计划](./2026-06-21-deep-reading-master-plan.md)
- [README.md](../../../../README.md)

## 目标

封装一个 Trae Skill，让用户用自然语言在 Trae 中触发深度阅读工作流。

## 范围

1. 创建 Skill 文件
   - `.trae/skills/deep-reading/SKILL.md`
   - 按 Trae Skill 规范编写元数据和 Prompt

2. 设计 Skill 行为
   - 识别用户意图：
     - "我读完了《资治通鉴·周纪二》商鞅变法"
     - "生成资治通鉴 周纪二 商鞅变法的笔记"
     - "资治通鉴"（只有书名）
     - "商鞅变法"（只有事件）
   - 加载 `.trae/rules/rules.md` 规范，作为生成要求的一部分
   - 当信息不足时，向用户确认书名/章节/事件
   - 调用本地 Python 脚本执行生成
   - 返回生成结果摘要、文件路径、内容预览

3. 错误处理
   - 核心引擎未安装/未配置时给出提示
   - 生成失败时返回错误信息
   - API Key 缺失时提示配置 `.env`

4. 示例用法
   - 在 Skill 中提供 3-5 个示例对话

## 依赖

- 核心引擎可运行（`src/main.py`）
- `.trae/rules/rules.md` 存在

## 输入输出

- 输入：用户在 Trae 中的自然语言
- 输出：
  - 调用 `python src/main.py ...`
  - 返回文件路径、摘要、内容预览

## 关键设计决策

- Skill 本身不执行代码，只负责意图识别和触发本地脚本。
- Skill Prompt 要明确说明：用户输入可能是句子、书名、章节名，需要先解析再调用。
- Skill 调用脚本时，优先使用自然语言输入方式 `--input`，让 Orchestrator 去解析。
- 返回信息要简洁，避免一次性输出整篇 Markdown。

## 验收标准

- [x] Skill 能识别用户意图并提取书名/章节/事件
- [x] 信息不足时会询问用户
- [x] 能正确调用本地 Python 引擎
- [x] 返回生成的文件路径和摘要
- [x] 有示例用法

## 建议执行方式

本地 Trae 环境执行。测试时可在 Trae 对话框中直接调用 Skill。
