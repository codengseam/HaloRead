# 子计划 4：Web 管理界面

## 上下文

本项目是个人 AI 深度阅读助手，核心引擎生成 Markdown 笔记并保存到 `output/` 目录。本轨道的目标是提供一个本地 Web 管理界面，方便用户浏览、搜索、阅读和生成笔记。

详见：
- [总计划](./2026-06-21-deep-reading-master-plan.md)
- [README.md](../../../../README.md)

## 目标

提供一个本地 HTML 页面，用于管理、分类和阅读生成的笔记。

## 范围

1. 后端服务
   - `src/web/app.py`：使用 Flask 或 FastAPI
   - 扫描 `output/` 目录，按书名/章节构建索引
   - 提供 API：
     - `GET /api/notes`：列出所有笔记
     - `GET /api/notes/{path}`：读取单篇笔记
     - `POST /api/generate`：触发新笔记生成

2. 前端页面
   - 左侧导航：书籍 → 章节 → 事件 树形结构
   - 右侧阅读区：Markdown 渲染
   - 顶部工具栏：生成新笔记、搜索
   - 简单响应式布局

3. 生成新笔记入口
   - 表单输入：书名、章节、事件
   - 支持自然语言输入
   - 调用核心引擎生成，生成后刷新列表

4. 可选增强（第一阶段不做）
   - 阅读进度
   - 标签云
   - 双向链接图谱

## 依赖

- 核心引擎的输出 Markdown 格式
- `output/` 目录结构

## 输入输出

- 输入：`output/` 目录结构
- 输出：本地 Web 服务页面

## 关键设计决策

- 后端只读 `output/` 目录，不直接修改笔记内容。
- 生成新笔记时调用核心引擎的 CLI/API。
- Markdown 渲染在前端完成，后端只返回原始文本。
- 页面风格简洁、中文为主，不追求复杂 UI。
- 默认监听本地端口，不暴露到公网。

## 验收标准

- [x] 启动 Web 服务后能看到所有已生成笔记
- [x] 点击笔记标题能在右侧渲染 Markdown
- [x] 能通过界面触发新笔记生成
- [x] 页面风格简洁、中文为主
- [x] 搜索功能可用

## 建议执行方式

本地执行，因为读取本地文件系统。前端部分可在云端 AI 辅助生成，但最终在本地整合测试。
