# 评论系统设计方向 · 3 选 1

> 面向"深度阅读助手"静态站点的原文批注式评论系统。本文给出 3 个差异化视觉/交互方向，每个方向均落地到 CSS 变量与组件结构，供后续实现 agent 直接取用。
>
> 现有站点设计 token（三方向共用基线）：
>
> | Token | 值 | 说明 |
> |---|---|---|
> | `--bg-paper` | `#faf8f5` | 暖纸色正文背景 |
> | `--bg-sidebar` | `#f5f2ed` | 侧栏背景 |
> | `--bg-hover` / `--bg-active` | `#ede9e2` / `#e8e3da` | 悬浮/选中态 |
> | `--ink-primary` | `#2c2c2c` | 正文墨色 |
> | `--ink-secondary` / `--ink-muted` | `#5a5651` / `#8a8580` | 次级/弱化文字 |
> | `--accent` / `--accent-hover` | `#8b5a2b` / `#6d4520` | 棕色主色 |
> | `--border` | `#e0dcd4` | 分隔线 |
> | `--shadow` | `rgba(0,0,0,0.06)` | 基础阴影 |
> | `--radius` | `6px` | 全站圆角 |
> | `--font-sans` | `-apple-system, "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif` | 无衬线 |
> | `--font-serif` | `"Songti SC", "Source Han Serif SC", "Noto Serif SC", serif` | 宋体正文 |
>
> 阅读区结构：顶部 toolbar 60px + 左侧 sidebar 280px（可拖拽 240–360）+ 右侧 reader；`.markdown-body` 最大宽 820px 居中，`padding: 2.5rem 3rem`，正文 `font-size: 1.05rem / line-height: 1.85`。
>
> 评论类型固定 5 种：**错误 / 夸奖 / 讨论 / 补充 / 感想**。三方向共用同一数据模型与 JS 接口，仅视觉/交互层不同。

---

## 方向 A · 飞书云文档风

### 一句话定位

最贴近飞书/Notion 的现代协作体验——右侧悬浮评论面板 + 气泡式选区工具条 + 蓝色品牌色点缀，用"冷色工具层"嵌在"暖纸内容层"上，形成有意图的现代/古典对比。

### 视觉语言

**配色策略**：阅读区保留暖纸底 `#faf8f5` 不动，评论系统作为"工具层"引入一套独立的冷色系，与纸色形成"工具 vs 内容"的视觉分工。

```css
:root {
  /* —— A 方向：评论系统专属变量（前缀 --cmtA-）—— */
  --cmtA-brand:        #3370ff;   /* 飞书蓝，主交互色 */
  --cmtA-brand-soft:   #e8f0ff;   /* 品牌色 8% 底 */
  --cmtA-brand-hover:  #245bdb;
  --cmtA-panel-bg:     #ffffff;   /* 评论面板纯白，与纸色分层 */
  --cmtA-panel-border: #ebebeb;

  /* 5 种评论类型色（标签 + 高亮复用）*/
  --cmtA-type-error:    #f54a45;  /* 错误·红 */
  --cmtA-type-praise:   #00b42a;  /* 夸奖·绿 */
  --cmtA-type-discuss:  #3370ff;  /* 讨论·蓝 */
  --cmtA-type-supply:   #ff7d00;  /* 补充·橙 */
  --cmtA-type-feeling:  #722ed1;  /* 感想·紫 */

  /* 高亮底色（默认 + 各类型）*/
  --cmtA-hl-default:    rgba(255, 213, 79, 0.42);  /* 未分类·暖黄 */
  --cmtA-hl-error:      rgba(245, 74, 69, 0.16);
  --cmtA-hl-praise:     rgba(0, 180, 42, 0.16);
  --cmtA-hl-discuss:    rgba(51, 112, 255, 0.16);
  --cmtA-hl-supply:     rgba(255, 125, 0, 0.16);
  --cmtA-hl-feeling:    rgba(114, 46, 209, 0.16);

  --cmtA-radius-card:   8px;
  --cmtA-radius-bubble: 12px;
  --cmtA-radius-pill:   999px;
  --cmtA-shadow-float:  0 4px 16px rgba(0, 0, 0, 0.08);
  --cmtA-shadow-panel:  -4px 0 24px rgba(0, 0, 0, 0.06);

  /* 字体分工：评论用 sans（工具感），原文保持 serif（内容感）*/
  --cmtA-font-ui:       var(--font-sans);
}
```

**字体与材质**：评论卡片、面板、按钮一律用 `--font-sans`（PingFang），原文高亮段保持 `--font-serif`（宋体）。这种"正文宋体 + 工具无衬线"的双轨制，是飞书风的核心——让评论明显是"贴上去的工具"，不与原文混为一谈。面板纯白 `#fff`，比纸色 `#faf8f5` 亮一档，自然浮起。

**氛围**：清爽、高效、可协作。圆角比站点基线略大（8–12px vs 6px），阴影更"浮起"（`0 4px 16px`），整体偏 SaaS 工具感。

### 核心交互模式

| 环节 | 形态 |
|---|---|
| **圈选触发** | 鼠标松开选区后，选区正上方 8px 处弹出**横向气泡工具条**（评论 ✎ / 复制 ⧉ / 引用 " ），宽约 120px，带小三角指向选区 |
| **写评论** | 点工具条"评论"→ 右侧面板**新建评论卡片**并自动聚焦 textarea；选区同步高亮 |
| **高亮样式** | 选区加**类型色底色**（16% 透明度），hover 时底色加深至 28% 并在右上角浮出评论数角标 `③` |
| **点击高亮** | 点击文中高亮 → 右侧面板滚动定位到对应卡片，卡片左侧亮起 3px 品牌色条 |
| **评论区位置** | reader 右侧**固定悬浮面板**，宽 340px，默认折叠为 44px 窄条（仅图标 + 计数），展开后覆盖在 reader 右侧但不挤压正文（正文 max-width 仍 820px，面板 absolute 浮于其右） |
| **评论卡片** | 头像（首字母圆形）+ 昵称 + 类型 pill 标签 + 引用原文片段（灰底 2 行截断）+ 正文 + 时间 + [回复] [解决] |
| **回复** | 卡片内联展开子输入框，回复以缩进树呈现（每层缩进 20px，最多 3 层） |
| **解决** | 点"解决"→ 卡片整体降至 50% 透明度 + 划线 + 移入面板底部"已解决"折叠区；原文高亮变为灰色虚线底 |
| **导出/专家团** | 面板顶部工具行：[导出 JSON] [启用专家团] 两个 ghost 按钮 |

**关键尺寸**：

- 气泡工具条：高 32px，padding `0 4px`，按钮 24×24，三角 6px
- 评论面板：宽 340px，折叠 44px，距 reader 右边缘 0，距顶 60px（toolbar 下），高 `calc(100vh - 60px)`
- 评论卡片：padding `14px 16px`，卡片间距 8px，头像 28×28
- 类型 pill：高 20px，padding `0 8px`，字号 12px，`border-radius: 999px`，底色 = 类型色 12%，字色 = 类型色
- 引用原文块：`bg #f7f7f7`，`radius 4px`，padding `6px 10px`，`-webkit-line-clamp: 2`

### 动效设计

| 元素 | 动效 | 时长/缓动 |
|---|---|---|
| 气泡工具条出现 | `scale(0.96)→1` + `opacity 0→1`，transform-origin 底部三角 | `150ms ease-out` |
| 评论面板展开 | `translateX(20px)→0` + `opacity 0→1` | `200ms ease-out` |
| 高亮 hover 加深 | `background-color` 过渡 | `200ms ease` |
| 卡片定位高亮条 | 左侧 3px 条 `scaleY(0)→1` 从顶部展开 | `180ms ease-out` |
| 解决态过渡 | 整卡 `opacity 1→0.5` + 原文高亮 `background→灰色虚线` | `250ms ease` |
| 新回复出现 | `translateY(-6px)→0` + `opacity` | `160ms ease-out` |

整体节奏快（120–250ms），偏"工具响应感"，不拖沓。

### 适合场景与取舍

**适合**：团队多人审稿、追求效率、用户已熟悉飞书/Notion 操作心智、需要频繁回复与解决流程。

**取舍**：
- ✅ 学习成本最低，协作功能最完整，回复/解决流程最顺
- ✅ 冷色工具层与暖纸内容层对比清晰，评论不会"糊"在原文里
- ⚠️ 蓝色品牌色与站点棕色 accent 是两套色系，视觉上"现代工具感"会盖过"古籍气质"，气质割裂感最强
- ⚠️ 右侧 340px 面板在窄屏（<1100px）会与 820px 正文打架，需做响应式（<1024px 面板转底部抽屉）

## 方向 B · 古籍批注风

### 一句话定位

与现有暖纸宋体深度统一，仿古人眉批夹注——朱笔赭石下划线标记原文，右侧留白做批注栏，浮层如手贴便笺，让评论系统本身就是古籍阅读体验的一部分。

### 视觉语言

**配色策略**：完全不引入冷色，全部在暖色谱内做文章。引入"朱砂红"作朱批色、"赭石"作批注框色，呼应古人"朱墨双色"的批注传统。

```css
:root {
  /* —— B 方向：评论系统专属变量（前缀 --cmtB-）—— */
  --cmtB-vermilion:    #c0392b;   /* 朱砂红，朱笔下划线/重点 */
  --cmtB-vermilion-soft: rgba(192, 57, 43, 0.12);
  --cmtB-ochre:        #a0522d;   /* 赭石，批注框边 */
  --cmtB-ochre-soft:   rgba(160, 82, 45, 0.10);
  --cmtB-ink:          var(--ink-primary);   /* 批注墨色，沿用 */
  --cmtB-paper-note:   #fdf6e3;   /* 便笺宣纸色，比正文底略黄 */
  --cmtB-paper-note-2: #f5efdc;   /* 便笺次级 */

  /* 5 种类型 → 印章色（克制，均在暖色谱）*/
  --cmtB-seal-error:    #b03a2e;  /* 错误·深朱 */
  --cmtB-seal-praise:   #b8860b;  /* 夸奖·金 */
  --cmtB-seal-discuss:  #8b5a2b;  /* 讨论·棕（=站点 accent）*/
  --cmtB-seal-supply:   #6e4b2a;  /* 补充·深褐 */
  --cmtB-seal-feeling:  #9c6b3c;  /* 感想·浅褐 */

  /* 高亮：朱笔下划线为主，淡赭底为辅 */
  --cmtB-hl-underline:  var(--cmtB-vermilion);
  --cmtB-hl-bg:         rgba(192, 57, 43, 0.08);  /* 极淡朱底 */
  --cmtB-hl-bg-hover:   rgba(192, 57, 43, 0.16);

  --cmtB-radius-note:   3px;      /* 便笺方正，仿古籍 */
  --cmtB-radius-seal:   2px;
  --cmtB-shadow-note:   0 1px 3px rgba(120, 80, 40, 0.12), 0 0 0 1px rgba(160,82,45,0.08);
  --cmtB-shadow-float:  0 6px 18px rgba(120, 80, 40, 0.14);

  /* 字体：批注也用 serif，但用楷体感（KaiTi 作为批注字）*/
  --cmtB-font-note:     "Kaiti SC", "STKaiti", "KaiTi", var(--font-serif);
}
```

**字体与材质**：批注正文用 `--cmtB-font-note`（楷体优先，回退宋体），与原文宋体同属衬线家族但字态更"手写"，呼应古人批注手书传统。便笺底色 `#fdf6e3` 比正文 `#faf8f5` 略黄一档，模拟"另贴一张纸"。可选叠加极轻纸纹噪点（`background-image` SVG noise，opacity 0.03）增强材质感。

**氛围**：沉静、古雅、沉浸。圆角刻意收小到 2–3px（方正，仿古籍版框），阴影偏暖（`rgba(120,80,40,...)` 而非纯黑），整体像在宣纸上批注。

### 核心交互模式

| 环节 | 形态 |
|---|---|
| **圈选触发** | 选区松开后，选区**右端**浮现一枚"朱笔圈点"图标按钮（●，朱砂色），点击即在该行右侧批注栏新建便笺 |
| **写评论** | 便笺以"手贴"姿态出现在右侧批注栏对应行高位置，自动聚焦；选区同步加朱笔下划线 |
| **高亮样式** | 选区加**朱砂波浪下划线**（`text-decoration: underline wavy var(--cmtB-vermilion); text-underline-offset: 3px; text-decoration-thickness: 1.5px`）+ 极淡朱底；hover 时朱底加深 |
| **点击高亮** | 点击文中朱批处 → 右侧对应便笺轻微"弹跳"高亮（边框变朱色 + 轻微放大） |
| **评论区位置** | reader 右侧**批注栏（margin notes）**，宽 240px，便笺按对应原文行高纵向排布；正文 `.markdown-body` 让出右侧 margin（`margin-right: 260px`），便笺 absolute 定位到对应行 |
| **便笺形态** | 像贴纸：`bg #fdf6e3`，左侧 2px 赭石边，右上角微折角（CSS triangle），顶部一枚**印章式类型标签**（朱底白字/阴文，2×2 字符如"误""赞""议""补""感"） |
| **回复** | 便笺内以缩进小字追加，仿"再批"（前缀"·"），无头像，仅署名 |
| **解决** | 点"圈结"→ 便笺盖一枚"阅"字朱印（半透明覆盖），原文下划线变虚线 |
| **汇总区** | 文末（markdown-body 末尾）追加一节"批注汇总"，竖列所有便笺，按原文先后排序，每条带"跳转原文"锚点 |
| **导出/专家团** | toolbar 右侧加一枚"朱印"风格按钮 [批注 · 导出] / [延请名家]，点击弹出（复用现有 modal 样式） |

- 批注栏：宽 240px，距 reader 右边缘 0，`padding-top` 与正文对齐
- 正文让位：`.markdown-body { margin-right: 260px; }`（仅 B 方向，240 栏宽 + 20 间距）
- 便笺：宽 220px，`padding: 10px 12px`，`radius: 3px`，便笺间距 10px
- 朱笔下划线：`text-decoration-thickness: 1.5px`，`text-underline-offset: 3px`
- 印章标签：22×22px，字号 13px，`font-weight: 700`
- 折角：右上 8×8 CSS triangle，`bg #f5efdc`
- 圈点按钮：12×12 圆，朱砂色，`box-shadow: 0 0 0 3px rgba(192,57,43,0.15)`

### 动效设计

| 元素 | 动效 | 时长/缓动 |
|---|---|---|
| 朱笔圈点出现 | `opacity 0→1` + `scale(0.6)→1`，像"点上去" | `180ms ease-out` |
| 便笺贴入 | `translateY(-8px) rotate(-1deg)→0` + `opacity`，轻微倾斜像手贴 | `260ms cubic-bezier(.2,.8,.3,1)` |
| 朱笔下划线绘制 | `background-size: 0→100%` 横向展开（伪元素模拟"运笔"） | `320ms ease-out` |
| 便笺定位高亮 | 边框 `ochre→vermilion` + `scale(1)→1.02→1` 轻弹 | `300ms ease-out` |
| 圈结（解决） | "阅"字朱印 `scale(1.4)→1` + `opacity 0→0.7` 盖下 | `280ms ease-out` |
| 汇总区跳转 | 平滑滚动 + 目标便笺高亮一次 | scroll `400ms` |

整体节奏偏慢（180–320ms），带轻微"手作"倾斜与"运笔"感，克制不弹跳过度。

### 适合场景与取舍

**适合**：个人沉浸式阅读、文史爱好者、追求"古人读书"仪式感、站点气质统一性优先。

**取舍**：
- ✅ 与站点暖纸宋体气质**完全统一**，评论系统本身就是阅读体验的一部分，无割裂
- ✅ 朱墨批注有强烈文化辨识度，差异化最强，最"像这个站点该有的样子"
- ✅ margin notes 模式让批注与原文位置对应直观，符合古籍夹注阅读习惯
- ⚠️ margin notes 对**响应式极不友好**：窄屏无法保留右侧批注栏，需降级为文末汇总（移动端体验弱）
- ⚠️ 楷体（Kaiti SC）在非 macOS/未装字体环境会回退宋体，"手写感"打折
- ⚠️ 协作感弱（无头像、回复树浅），更适合 1–2 人批注而非团队群审
- ⚠️ 正文 `margin-right: 260px` 会让 820px 阅读区实际可用宽度收窄到 ~560px，长段落换行变密，需评估可读性

## 方向 C · 现代极简风

### 一句话定位

大量留白、浮动气泡、底部抽屉式评论区、单色高亮、微动效——最克制最现代，评论系统"在场但不打扰"，把舞台让给原文。

### 视觉语言

**配色策略**：极简去装饰，不引入新色相。高亮与交互全部基于现有 `--ink-primary` 与 `--accent` 的低透明度衍生，类型区分靠**左侧色条**而非彩色底块。

```css
:root {
  /* —— C 方向：评论系统专属变量（前缀 --cmtC-）—— */
  --cmtC-ink:          var(--ink-primary);
  --cmtC-ink-soft:     rgba(44, 44, 44, 0.06);   /* 极淡墨底 */
  --cmtC-ink-soft-2:   rgba(44, 44, 44, 0.10);
  --cmtC-accent:       var(--accent);             /* 仅激活态用 */
  --cmtC-accent-soft:  rgba(139, 90, 43, 0.10);
  --cmtC-panel-bg:     #ffffff;
  --cmtC-panel-border: var(--border);

  /* 5 种类型 → 单色色条（低饱和，仅作区分）*/
  --cmtC-bar-error:    #c0392b;
  --cmtC-bar-praise:   #8b5a2b;   /* 复用 accent，夸奖=站点主色 */
  --cmtC-bar-discuss:  #5a5651;
  --cmtC-bar-supply:   #8a8580;
  --cmtC-bar-feeling:  #a8a29e;

  /* 高亮：极淡墨底 + 左侧色条（无彩色底块）*/
  --cmtC-hl-bg:        rgba(44, 44, 44, 0.055);
  --cmtC-hl-bg-hover:  rgba(44, 44, 44, 0.10);

  --cmtC-radius-drawer: 16px;   /* 抽屉上圆角，比站点 6px 大，更现代 */
  --cmtC-radius-card:   6px;    /* 卡片沿用站点圆角 */
  --cmtC-radius-bubble: 10px;
  --cmtC-shadow-drawer: 0 -4px 24px rgba(0, 0, 0, 0.06);
  --cmtC-shadow-bubble: 0 2px 8px rgba(0, 0, 0, 0.05);

  /* 字体：评论用 sans，统一现代感；原文 serif 不变 */
  --cmtC-font-ui:      var(--font-sans);
}
```

**字体与材质**：评论 UI 全用 `--font-sans`，与原文 serif 形成"内容/工具"分层但不像 A 方向那样引入蓝色，而是用**纯墨色系**保持极简。抽屉与气泡纯白 `#fff`，阴影极轻（`0 2px 8px`），强调"轻存在"。无任何纹理、无装饰线、无类型彩色底块——类型仅靠左侧 2px 色条暗示。

**氛围**：克制、留白、不打扰。圆角在抽屉处放大到 16px（现代感），卡片保持 6px（与站点一致）。阴影最轻，几乎"贴着"而非"浮起"。

### 核心交互模式

| 环节 | 形态 |
|---|---|
| **圈选触发** | 选区松开后，选区右端浮出**极简图标气泡**（仅一个 ✎ 图标，无文字，28×28 圆形），点击展开输入 |
| **写评论** | 点图标 → 图标位原地展开为**内联输入条**（宽 280px，textarea + 类型选择器 + 提交），不弹面板 |
| **高亮样式** | 选区加**极淡墨底**（5.5% 透明度）+ 选区**左侧 2px 色条**（类型色）；hover 墨底加深；无波浪线、无彩色底块 |
| **点击高亮** | 点击文中高亮 → 底部抽屉上滑展开并滚动到对应评论 |
| **评论区位置** | reader 底部**抽屉式面板**，默认收起为一条 40px 窄条贴底（"评论 3 · 未解决 2"），点击上滑展开至 40vh（最大 360px） |
| **评论列表** | 抽屉内横向通栏列表，每条单行高度紧凑（72px），左侧 2px 类型色条 + 引用片段（1 行截断）+ 正文（2 行截断）+ 状态 |
| **回复** | 点条目 → 右侧展开详情面板（抽屉内左右分栏：左列表右详情），详情含完整正文 + 回复输入 |
| **解决** | 条目右侧 ✓ 图标，点击后条目降至 45% 透明 + 色条变灰，移入抽屉底部"已解决"折叠组 |
| **导出/专家团** | 抽屉窄条右侧两个极简文字按钮 [导出] [专家团]，无图标无背景 |

### 适合场景与取舍

**适合**：阅读优先、不希望评论干扰正文、移动端友好、单人或轻量批注、追求现代克制美学。

**取舍**：
- ✅ **正文零挤压**——抽屉在底部、气泡内联，820px 阅读区完全保留，可读性最佳
- ✅ **响应式最友好**——底部抽屉在移动端天然适配，无需降级方案
- ✅ 单色高亮最不打扰，气质中性，与暖纸底协调不冲突
- ✅ 抽屉收起时几乎"隐形"，沉浸阅读与批注切换干净
- ⚠️ 批注与原文**位置对应最弱**（无 margin notes、无右侧面板定位），找对应原文需靠点击跳转
- ⚠️ 协作/回复流程藏在二级详情面板，比 A 方向多一步操作
- ⚠️ 极简风格"个性"最弱，差异化辨识度不如 A、B

## 三方向对比速查

| 维度 | A 飞书云文档风 | B 古籍批注风 | C 现代极简风 |
|---|---|---|---|
| **气质** | 现代协作 SaaS | 古雅沉浸，与站点统一 | 克制中性，阅读优先 |
| **主色** | 飞书蓝 `#3370ff`（冷） | 朱砂 `#c0392b` + 赭石（暖） | 墨色低透明度（中性） |
| **与站点协调度** | 对比（冷工具层 vs 暖内容） | 完全统一 | 中性协调 |
| **高亮样式** | 类型色底块 16% | 朱笔波浪下划线 + 淡朱底 | 极淡墨底 + 左 2px 色条 |
| **评论位置** | 右侧悬浮面板 340px | 右侧 margin notes 240px | 底部抽屉 40vh |
| **正文影响** | 不挤压（面板浮于右侧） | 挤压（margin-right 260px） | 零挤压（底部覆盖） |
| **回复/解决** | 内联树 + 解决折叠 | 缩进再批 + "阅"印 | 二级详情面板 |
| **协作感** | 强 | 弱 | 中 |
| **移动端** | 需降级为抽屉 | 需降级为文末汇总 | 天然友好 |
| **学习成本** | 最低（飞书心智） | 中（需理解朱批隐喻） | 低 |
| **差异化辨识** | 中 | 最强 | 弱 |
| **实现复杂度** | 中 | 高（margin 定位 + 楷体回退） | 中低 |

## 选型建议

- **若核心用户是团队/多人群审** → 选 **A**：协作流程最顺，学习成本最低。
- **若核心是个人沉浸读古籍、要气质统一** → 选 **B**：与站点浑然一体，文化辨识度最强，是这个站点"该有的样子"。
- **若阅读体验优先、要移动端友好、评论是轻量辅助** → 选 **C**：最不打扰，正文可读性最佳。

## 实现约定（三方向共用）

无论选哪个方向，以下接口与数据层保持一致，仅替换视觉层：

1. **数据模型**：每条评论 = `{ id, noteId, quote(原文片段), anchorPath(选区路径), type(5 选 1), author, content, createdAt, replies[], resolved }`
2. **存储**：`localStorage`，key = `comments:<noteId>`；导出 JSON 即序列化该 key
3. **选区定位**：基于 `Range` + 文本节点 offset，序列化为 `{startOffset, endOffset, textSnapshot}`（纯静态无后端，不做 DOM path 强定位，靠文本快照校验）
4. **高亮渲染**：用 `Range.surroundContents` 或拆分文本节点包 `<mark class="cmt-hl" data-cmt-id>`；注意跨节点选区需拆分
5. **专家团入口**：三方向统一为"导出 JSON → 喂给 AI → 回填建议评论"流程，入口按钮位置随方向不同但逻辑一致
6. **CSS 命名空间**：所有评论系统 class 加 `cmt-` 前缀，CSS 变量加 `--cmtA/-B/-C-` 前缀，避免污染现有 `:root`
7. **JS 入口**：单文件 `site/js/comments.js`，监听 `.markdown-body` 的 `mouseup`/`selectionchange`，与现有 `app.js` 解耦
