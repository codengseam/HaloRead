# 核心观点

二分查找是**"利用有序性通过排除法缩小搜索范围"的经典算法**，核心在于理解**循环不变量与边界的精确控制**。从基础的"查找目标值是否存在"到5种变体（第一个/最后一个/大于等于/小于等于），再到IP地址库的实战应用和二分答案技术，掌握二分查找能帮你高效解决搜索问题，是面试必考的算法基础，也是后续学习高级搜索、优化的前提。

## 核心概念解析（20% 重点深度解析）

### 1. 二分查找的本质与思想

- **定义**：在有序数组中，每次排除一半数据，将查找范围缩小的搜索算法
- **为什么重要**：
  - ⭐ **O(log n)的对数级效率**（100万个元素只需约20次比较！）
  - 面试最高频算法之一，几乎每场技术面试都会涉及
  - 是很多高级算法的基础（二分答案、旋转数组、矩阵搜索等）

- **关键机制——二分查找的核心思想**：

  **本质：有序性 + 排除法**
  ```
  前提条件：数组必须有序（升序或降序）

  核心操作：
  1. 取中间位置 mid = (low + high) / 2
  2. 比较 arr[mid] 与 target
  3. 根据"有序性"排除一半数据：
     - arr[mid] > target → 排除右半边（因为右边更大）
     - arr[mid] < target → 排除左半边（因为左边更小）
     - arr[mid] == target → 找到了！

  为什么能排除一半？
  因为有序性保证了：如果中间值比target大，
  那么右边的所有值都比target大（升序情况下）
  ```

  **时间复杂度分析**
  ```
  每次排除一半 → 类似折纸问题

  n=1000: 最多10次 (2^10 = 1024)
  n=100万: 最多20次 (2^20 ≈ 104万)
  n=10亿: 最多30次 (2^30 ≈ 10.7亿)

  时间复杂度：O(log n) ✅
  空间复杂度：
    - 迭代实现：O(1) ✅
    - 递归实现：O(log n)（栈空间）
  ```

- **标准实现（迭代版）⭐必须掌握**
  ```java
  int binarySearch(int[] arr, int target) {
      int left = 0;
      int right = arr.length - 1;  // 注意：闭区间 [left, right]

      while (left <= right) {      // 注意：<= 而不是 <
          int mid = left + (right - left) / 2;  // 防止溢出！

          if (arr[mid] == target) {
              return mid;           // 找到目标
          } else if (arr[mid] < target) {
              left = mid + 1;       // 排除左半边（包括mid）
          } else {
              right = mid - 1;      // 排除右半边（包括mid）
          }
      }
      return -1;                    // 未找到
  }

  // 关键细节解释：
  // 1. while (left <= right)：用闭区间，left==right时还有1个元素要检查
  // 2. mid = left + (right-left)/2：防止(left+right)整数溢出
  // 3. left = mid+1, right = mid-1：已经检查过mid，下次不再包含
  ```

### 2. 二分查找的5大变体 ⭐面试重点

- **定义**：针对不同查找需求的二分查找变形，核心区别在于"找到目标后的处理方式"
- **为什么重要**：
  - ⭐ **面试真题的高发区**（LeetCode 34/35/69/81等题都是变体）
  - 掌握变体的关键是理解"搜索区间"和"终止条件"
  - 实际应用中的需求往往不是简单的"存在性判断"

- **关键机制——5种变体的实现模式**：

  **变体1：查找目标值是否存在（标准版）**
  ```java
  // 返回目标值的索引，不存在返回-1
  int search(int[] nums, int target) {
      int left = 0, right = nums.length - 1;
      while (left <= right) {
          int mid = left + ((right - left) >> 1);
          if (nums[mid] == target) return mid;
          else if (nums[mid] < target) left = mid + 1;
          else right = mid - 1;
      }
      return -1;
  }

  // 适用场景：确认元素是否存在
  // 示例：用户ID是否在白名单中
  ```

  **变体2：查找第一个等于 target 的位置**
  ```java
  int findFirst(int[] nums, int target) {
      int left = 0, right = nums.length - 1;
      while (left <= right) {
          int mid = left + ((right - left) >> 1);

          if (nums[mid] < target) {
              left = mid + 1;            // target在右半边
          } else if (nums[mid] > target) {
              right = mid - 1;           // target在左半边
          } else {
              // nums[mid] == target，但不确定是不是第一个
              if (mid == 0 || nums[mid-1] != target) {
                  return mid;             // ✅ 第一个出现的位置
              } else {
                  right = mid - 1;        // 继续向左找
              }
          }
      }
      return -1;

      // 示例：[1,2,2,2,3], target=2 → 返回1（索引1）
      // 应用场景：查找某个版本的首次发布时间、查找第一次出现的错误日志
  }
  ```

  **变体3：查找最后一个等于 target 的位置**

  ```java
  }
  ```

### 3. 实战案例：如何用二分查找快速定位 IP 对应的省份？

- **定义**：利用二分查找在海量IP地址库中快速定位IP归属地的工程实践
- **为什么重要**：
  - ⭐ **二分查找的真实业务场景**（不只是算法题）
  - IP地址库通常有几百万条记录，线性扫描太慢
  - 体现"预处理+高效查询"的设计模式

- **关键机制——IP地址库的二分查找实现**：

  **Step 1: 理解IP地址库的数据结构**
  ```
  IP地址库示例（简化版）：
  ┌─────────────┬─────────────┬──────────┐
  │ 起始IP范围   │ 结束IP范围   │ 省份     │
  ├─────────────┼─────────────┼──────────┤
  │ 101.23.0.0  │ 101.23.255.255 │ 北京    │
  │ 101.24.0.0  │ 101.24.255.255 │ 上海    │
  │ 101.25.0.0  │ 101.25.255.255 │ 广东    │
  │ ...         │ ...         │ ...      │
  └─────────────┴─────────────┴──────────┘

  特点：
  1. 按起始IP排序（有序！→ 可以用二分查找）
  2. 每条记录是一个连续的IP段
  3. 总记录数可能达到数百万
  ```

  **Step 2: 将IP地址转换为可比较的整数**
  ```java
  // IPv4地址转整数（32位无符号整数）
  long ipToLong(String ipAddress) {
      String[] octets = ipAddress.split("\\.");
      long result = 0;
      for (int i = 0; i < 4; i++) {
          result = result * 256 + Long.parseLong(octets[i]);
      }
      return result;
  }

  // 示例：
  // "101.23.45.67" → 101×256³ + 23×256² + 45×256 + 67 = 1694590947
  // 这样就可以直接比较大小了！
  ```

  **Step 3: 二分查找定位IP段**
  ```java
  String findProvince(long targetIp, List<IpRange> ipDatabase) {
      int left = 0;
      int right = ipDatabase.size() - 1;
      String result = "未知";

      while (left <= right) {
          int mid = left + (right - left) / 2;
          IpRange range = ipDatabase.get(mid);

          if (range.startIp <= targetIp && targetIp <= range.endIp) {
              return range.province;  // ✅ 找到了！
          } else if (targetIp < range.startIp) {
              right = mid - 1;        // 在左半边
          } else {
              left = mid + 1;         // 在右半边
          }
      }
      return result;
  }

  // 性能分析：
  // 假设IP库有100万条记录
  // 线性扫描：最坏100万次比较
  // 二分查找：最多20次比较（log₂(100万)≈20）
  // 速度提升：50000倍！
  ```

  **Step 4: 工程优化考虑**
  ```
  ① 内存优化：
     - IP库可以压缩存储（只存起始IP和长度）
     - 使用内存映射文件（mmap）避免全量加载

  ② 查询优化：
     - 缓存热门IP段的查询结果（LRU Cache）
     - 批量查询时先排序再批量二分

  ③ 更新策略：
     - IP分配信息不常变，可以定期重建索引
     - 或使用增量更新 + 后台合并
  ```

### 4. 二分答案技术：在解空间上二分

- **定义**：当问题的答案具有单调性时，可以对"可能的答案空间"进行二分查找
- **为什么重要**：
  - ⭐ **将优化问题转化为判定问题的通用技巧**
  - LeetCode 中等/困难题的核心解法（如"最小化最大值"、"最大化最小值"）
  - 体现"二分查找不仅用于搜索，更是一种思维方式"

- **关键机制——二分答案的模式**：

  **核心思想**
  ```
  传统二分：在已知的有序数组中查找目标值
  二分答案：在可能的答案范围内，寻找满足条件的最优值

  适用条件：
  1. 答案具有单调性：如果x可行，则所有≥x（或≤x）都可行
  2. 能快速判断某个候选答案是否可行

  模式：
  function binaryAnswer() {
      low = 最小可能的答案
      high = 最大可能的答案

      while (low < high) {
          mid = (low + high) / 2

          if (check(mid)) {        // 判断mid是否可行
              high = mid;           // 尝试更小的答案（求最小化）
              // 或 low = mid + 1;  // 尝试更大的答案（求最大化）
          } else {
              low = mid + 1;        // 需要更大的答案
              // 或 high = mid - 1;
          }
      }
      return low;  // 或high
  }
  ```

  **经典案例①：分割数组的最大值（LeetCode 410）**
  ```
  问题：将数组分成m个子数组，使各子数组之和的最大值最小化

  示例：[7,2,5,10,8], m=2
  可能的分割：
  - [7,2,5] [10,8] → max(14, 18) = 18
  - [7,2,5,10] [8] → max(24, 8) = 24
  - [7] [2,5,10,8] → max(7, 25) = 25
  最优解：18

  二分答案思路：
  答案范围：max(nums) ≤ answer ≤ sum(nums)
  （每个子数组至少包含一个元素，至多包含全部）

  check(mid): 能否分成≤m个子数组且每个子组和≤mid？
    贪心分割：尽可能多放，看需要的子数组数量是否≤m

  时间复杂度：O(n × log(sum-max))
  ```

  **经典案例②：吃香蕉问题（LeetCode 875）**
  ```
  问题：Koko每小时吃k根香蕉，H小时内吃完，求最小的k

  单调性：如果k小时能吃完，那么k+1小时也能吃完 ✓
  → 可以二分k！

  check(k): 以速度k吃，能否在H小时内吃完？
    totalHours = Σ ceil(piles[i] / k)
    return totalHours <= H

  答案范围：1 ≤ k ≤ max(piles)
  ```

  **经典案例③：制造材料问题（POJ 1064）**
  ```
  问题：N根绳子，切出K根等长的绳子，求最大长度

  单调性：如果长度L能切出≥K根，那么L' < L肯定也能切出更多根 ✓
  → 可以二分L！（注意这里是最大化，方向相反）

  check(L): 每根绳子能切出 floor(length[i]/L) 根
    总数 = Σ floor(length[i]/L) ≥ K ?
  ```

### 5. 二分查找的常见陷阱与调试技巧

- **定义**：编写二分查找代码时最容易犯的错误及应对方法
- **为什么重要**：
  - ⭐ **看似简单但极易写错**（据统计60%的二分查找实现有bug）
  - 面试官经常让你现场写代码并指出潜在问题
  - 这些陷阱也是调试其他循环算法时的常见错误来源

- **三大陷阱详解**：

  **陷阱①：死循环（Infinite Loop）⭐最常见**
  ```java
  // 错误代码示例
  while (left < right) {          // ❌ 用<而不是<=
      int mid = left + (right - left) / 2;
      if (arr[mid] < target) {
          left = mid;             // ❌ 应该是mid+1！
      } else {
          right = mid;            // ❌ 应该是mid-1！或者根据情况调整
      }
  }
  // 当left=0, right=1, mid=0时：
  // 如果进入right=mid分支，则right还是0，永远无法退出！

  // 正确写法：
  while (left <= right) {         // ✅ 用<=
      int mid = left + (right - left) / 2;
      if (arr[mid] < target) {
          left = mid + 1;         // ✅ 排除mid
      } else {
          right = mid - 1;        // ✅ 排除mid
      }
  }

  // 或者另一种正确的写法（搜索区间[left, right)）：
  while (left < right) {
      int mid = left + (right - left) / 2;
      if (arr[mid] < target) {
          left = mid + 1;
      } else {
          right = mid;            // ✅ 区间是[left,right)，所以right=mid是对的
      }
  }
  ```

  **陷阱②：整数溢出（Integer Overflow）**
  ```java
  // 错误写法
  int mid = (left + right) / 2;   // ❌ 当left+right > 2^31-1时溢出！

  // 正确写法
  int mid = left + (right - left) / 2;  // ✅ 先减后加，不会溢出

  // 更简洁的位运算写法
  int mid = left + ((right - left) >> 1);  // ✅ 等价于除以2

  // 溢出示例（32位int）：
  // left = 2,000,000,000
  // right = 2,100,000,000
  // left + right = 4,100,000,000 > 2^31-1 (2,147,483,647)
  // → 溢出变成负数！结果完全错误
  ```

  **陷阱③：边界条件错误（Off-by-One Error）**
  ```java
  // 场景1：空数组未处理
  int binarySearch(int[] arr, int target) {
      if (arr == null || arr.length == 0) return -1;  // ✅ 必须检查
      // ...
  }

  // 场景2：搜索范围错误
  // 如果用[left, right]闭区间：
  //   while条件应该是 <=
  //   更新时应该 mid±1
  // 如果用[left, right)左闭右开区间：
  //   while条件应该是 <
  //   更新时左边mid+1，右边mid

  // 场景3：重复元素处理不当
  // [1,2,2,2,3] 查找2，返回哪个？
  // 标准：任意一个即可
  // 变体：第一个或最后一个（需要额外判断）
  ```

- **调试技巧**：
  ```
  技巧1：画图辅助思考
  画出每次迭代后left/right/mid的变化
  特别关注最后几次迭代

  技巧2：小规模测试
  先用n=1,2,3,4的数组测试各种情况
  包括：目标在开头、结尾、中间、不存在

  技巧3：打印循环变量
  System.out.printf("left=%d, right=%d, mid=%d\n", left, right, mid);
  观察是否有异常跳转或死循环

  技巧4：使用循环不变量证明
  明确 invariant：target一定在[left, right]范围内
  每次迭代都要保持这个不变量成立
  ```

## 次要信息速览（80% 快速带过）

- 【二分查找 vs 线性查找 vs 哈希查找】：
  - **二分查找**：O(log n)，需要有序，适合静态数据
  - **线性查找**：O(n)，无需预处理，适合小数据或无序数据
  - **哈希查找**：O(1)，需要额外空间，适合频繁查找
  - 选择依据：数据规模、是否有序、内存限制、查询频率

- 【二分查找的递归实现】：
  - 递归版更直观（符合分治思想），但有栈开销
  - 迭代版性能更好（无函数调用开销）
  - 工程实践中优先使用迭代版

- 【二分查找的变种应用】：
  - **旋转数组查找**（LeetCode 33）：先找旋转点，再分段二分
  - **二维矩阵查找**（LeetCode 240）：从右上角开始逐步排除
  - **查找峰值**（LeetCode 162）：利用导数的正负性二分

# 思考问答

1. **以下二分查找代码有什么问题？会导致什么后果？如何修复？**
   ```java
   int search(int[] nums, int target) {
       int left = 0, right = nums.length - 1;
       while (left < right) {
           int mid = (left + right) / 2;
           if (nums[mid] == target) return mid;
           else if (nums[mid] < target) left = mid;
           else right = mid;
       }
       return -1;
   }
   ```

# 实践要点

- ✅ **统一使用一种区间表示**：要么始终用[left, right]，要么始终用[left, right)，不要混用
- ✅ **用 `left + (right - left) / 2` 计算中点**：养成习惯，防止整数溢出
- ✅ **先处理空数组/单元素等边界情况**：防御性编程
- ✅ **手动模拟小例子验证**：用[1,2,3]测试目标在各个位置的执行过程
- ❌ **在while循环中忘记 `+1` 或 `-1`**：这是死循环的头号原因
- ❌ **混淆不同变体的返回语义**：明确是要找"第一个"、"最后一个"还是"任意一个"
- ❌ **忽略数据预处理成本**：二分查找需要O(n log n)排序，如果只查一次不如线性查找
