# 核心知识清单(20%重点)

## 一、为什么要了解数据库原理?(核心价值)

### 1.1 快速定位问题
**场景案例**:政府系统每天早上重启才能连接数据库
- **问题本质**:数据库默认超时时间8小时,下班后无人使用,第二天早上超时断开
- **解决方法**:修改超时参数,避免连接断开
- **关键认知**:了解参数含义就能快速定位问题,无需深入源码

**思考**:如果你只知道"连接失败"这个表象,会浪费多少时间排查网络、防火墙等问题?

### 1.2 巧妙解决问题
**场景案例**:贴吧权限系统加载慢(正常2分钟,异常重启需30分钟)
- **问题根源**:MySQL空闲时疯狂刷脏页,占用磁盘资源
- **巧妙方案**:每隔0.5秒发送简单SQL查询,告诉数据库"我很忙"
- **原理应用**:理解脏页刷盘机制→调整系统行为→问题解决
- **实践价值**:该脚本离职两年后仍在使用

**关键知识点**:
- 脏页刷盘机制:系统空闲时MySQL会加速刷脏页
- 参数理解:知道参数含义→正确设置
- 原理理解:懂得原理→巧妙解决问题

### 1.3 源码能力带来的优势
**场景案例**:阿里双十一SSD缓存性能问题
- **问题**:Flashcache作为二级缓存,性能反而不如纯机械盘
- **源码发现**:Flashcache分桶管理,单桶80%脏页就强制刷盘
- **顺序写问题**:日志等顺序写数据容易写满单个桶
- **解决方案**:顺序写数据放机械盘,随机写数据放Flashcache

**思考**:如果不看源码,可能永远找不到性能瓶颈的根本原因。

## 二、MySQL学习路径(方法论)

### 2.1 三阶段递进路径
```
会用 → 发现问题 → 实践 → 看官方手册
```

**阶段一:会用**
- 了解每个参数的意义
- 不要盲目套用网上建议
- 理解"为什么这样用"

**阶段二:发现问题**
- 专栏思考题训练(不看答案先思考)
- 在数据库中实际验证
- 建立问题意识

**阶段三:实践**
- 搭建主备架构
- 构造延迟/不一致场景
- 分析问题→解决问题

**阶段四:看官方手册**
- 前提:已有知识脉络和网络
- 目的:查漏补缺
- 禁忌:一开始就啃手册(100多万英文单词,看了后面忘前面)

### 2.2 推荐学习资源
