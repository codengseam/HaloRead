## 题目信息

**题目名称**：求1+2+...+n

**LeetCode链接**：[求1+2+…+n](https://leetcode.cn/problems/qiu-12n-lcof/)

**难度**：Medium

**描述**：求 1+2+...+n ，要求不能使用乘除法、for、while、if、else、switch、case等关键字及条件判断语句（A?B:C）。

## 🛒 题目"人话"翻译

**"带着镣铐跳舞"**。

*   算个加法，但是把你所有的常用工具都没收了。
*   不能循环（for/while），不能判断（if/else）。
*   就像让你用一只手做俯卧撑。

## 🧊 知识点"破冰"讲解

**短路效应 (Short-circuit Evaluation)**。

*   既然不能用 `if`，那就用逻辑运算符 `and` 和 `or`。
*   **原理**：
    *   `A and B`：如果 A 是 False，那 B 根本不会执行（短路）。这不就是一个隐形的 `if (A) { B }` 吗？
    *   `A or B`：如果 A 是 True，那 B 也不会执行。
*   **递归**：既然不能循环，那就递归。
    *   `n > 0 and (sum(n-1) + n)`
    *   当 `n=0` 时，`n > 0` 为 False，后面那个括号里的递归就不会执行了，直接返回 0。完美替代了 `if (n==0) return 0`。

## 💡 思路"推导"过程

这道题的关键在于如何绕过限制条件。

**第一阶段：暴力解法**
*   用循环或者公式，但题目禁止了。

**第二阶段：递归**
*   用递归代替循环。
*   但递归需要终止条件，通常用 if，题目也禁止了。

**第三阶段：短路效应**
*   利用逻辑运算符的短路特性，实现条件判断。
*   `n and (n + sumNums(n-1))`，当 n=0 时，短路返回 0。

## ❓ "为什么"的解释

**Q：为什么 `n and (n + sumNums(n-1))` 能工作？**

**A：** 当 n > 0 时，`n` 为真，会执行后面的递归；当 n = 0 时，`n` 为假，短路返回 0。这样就实现了递归的终止条件，而且没有使用 if 语句。

## 🚗 代码"带逛"

这里展示两种解法。第一种是利用 Python 的内置函数（稍微有点取巧），第二种是真正的逻辑短路法。

```python
# 方法一：利用 sum 和 range（有点作弊嫌疑，但面试官没说不能用库函数）
def sumNums_cheat(n):
    return sum(range(1, n + 1))

# 方法二：短路递归（正宗解法）
def sumNums(n):
    # 利用 and 的短路特性
    # 如果 n > 0 为真，则执行后面的递归
    # 如果 n <= 0 为假，直接返回 0（False在数值运算中等于0）
    return n and (n + sumNums(n - 1))
```
