## 题目信息

**题目信息**：LeetCode 240. Search a 2D Matrix II / 剑指 Offer 04

*   **原题链接**：[Search a 2D Matrix II](https://leetcode.cn/problems/search-a-2d-matrix-ii/)
*   **难度**：Medium
*   **描述**：编写一个高效的算法来搜索 `m x n` 矩阵 `matrix` 中的一个目标值 `target`。该矩阵具有以下特性：
    *   每行的元素从左到右升序排列。
    *   每列的元素从上到下升序排列。

### 🛒 题目 "人话" 翻译

想象一下，你站在一个 **"强迫症晚期" 整理过的图书馆**里。

*   书架是一排排、一列列摆好的。
*   **规则一**：每一排书架，书的编号从左到右越来越大。
*   **规则二**：每一列书架，书的编号从上到下也越来越大。

**你的任务**：图书管理员给了你一个编号（比如 2077），问你："这书在不在咱们这儿？"

你要做的就是用最快的方法找到它，或者确定它根本不在。



### 💡 思路 "推导" 过程

我们来模拟一下小白的思考过程：

**第一阶段：暴力解法（老实人做法）** 🐢
*   **想法**：管它什么规律，我从第一行第一个开始，一个一个看！双重 `for` 循环遍历整个二维数组。
*   **结局**：肯定能做出来。
*   **问题**：太慢了！如果这个图书馆有 1 万行、1 万列，你要找 1 亿次。这就好比你在拥有 1 亿本书的图书馆里一本本翻，管理员会打人的。

**第二阶段：稍微聪明点（每行二分）** 🐇
*   **想法**：既然每一行都是排好序的，那我在每一行里用 **"二分查找"** 也就是折半查找不行吗？
*   **结局**：比暴力快多了。
*   **问题**：虽然快了，但你还是没有利用"每一列也是排好序的"这个条件。感觉还是有点浪费线索。

**第三阶段：灵光一闪（站在右上角）** ✨
*   **想法**：我们试试站在 **右上角**（第一行的最后一列）看看。
    *   假设你要找的数是 **Target**，当前右上角的数是 **Current**。
    *   **情况 A**：`Current > Target`。
        *   哎呀，现在的数比目标大。
        *   那目标肯定不在这一 **列** 了（因为这一列下面的数会更大）。
        *   **行动**：**剔除这一列**，向左移动一步。
    *   **情况 B**：`Current < Target`。
        *   哎呀，现在的数比目标小。
        *   那目标肯定不在这一 **行** 了（因为这一行左边的数会更小）。
        *   **行动**：**剔除这一行**，向下移动一步。
    *   **情况 C**：`Current == Target`。
        *   **行动**：找到了！开心回家！

*   **结论**：用这个方法，每一步都能排除一行或者一列，速度飞快！



### 🚗 代码 "带逛"

```python
def findNumberIn2DArray(matrix, target):
    """
    在二维数组中查找目标值
    :param matrix: List[List[int]] 二维数组
    :param target: int 要找的目标数字
    :return: bool 找到了返回True，没找到返回False
    """
    # 1. 先处理特殊情况：如果数组是空的，或者是个空壳子，直接说没找到
    if not matrix or not matrix[0]:
        return False
    
    # 2. 获取数组的"身高"（行数）和"腰围"（列数）
    rows = len(matrix)
    cols = len(matrix[0])
    
    # 3. 决定我们的"出生点"：右上角
    # 行索引 row 从 0 开始（第一行）
    # 列索引 col 从 最后一列 开始
    row = 0
    col = cols - 1
    
    # 4. 开始寻宝循环！
    # 只要我们还没有走出数组的边界（行不能超过最大行数，列不能小于0），就继续找
    while row < rows and col >= 0:
        
        # 取出当前位置的数字
        current_num = matrix[row][col]
        
        if current_num == target:
            # 哇！运气真好，直接找到了！
            return True
            
        elif current_num > target:
            # 当前数字比目标大
            # 因为这一列下面的数字会更大，所以目标肯定不在这列
            # 咱们往左挪一挪，去更小的数字那边找
            col -= 1
            
        else: # current_num < target
            # 当前数字比目标小
            # 因为这一行左边的数字会更小，所以目标肯定不在这行
            # 咱们往下挪一挪，去更大的数字那边找
            row += 1
            
    # 5. 如果循环结束了还没返回True，说明找遍了所有可能的路都没找到
    return False
```
