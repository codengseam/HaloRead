## 题目信息

**题目名称**：反转链表

**LeetCode链接**：[Reverse Linked List](https://leetcode.cn/problems/reverse-linked-list/)

**难度**：Easy

**描述**：给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。



## 🧊 知识点"破冰"讲解

这道题是链表操作的 **基本功**。

*   **核心痛点**：当你改变 B 的指针指向 A 时，B 原来指向 C 的那条路就断了。如果你没提前抓住 C，C 就丢了。
*   **解决方案**：需要 **三个指针**。
    *   `prev` (前任)：用来存你要指回去的目标（最开始是 None）。
    *   `curr` (现任)：当前正在操作的节点。
    *   `next` (继任)：用来临时保存下一步要去的地方（防止断链）。



## ❓ "为什么"的解释

**Q：为什么需要三个指针？两个不行吗？**

**A：** 不行！
*   如果你只有 `prev` 和 `curr`，当你执行 `curr.next = prev` 后，你就再也找不到原来的下一个节点了。
*   链表就断了，后面的节点都丢失了。
*   所以必须提前用 `next_node` 把下一个节点保存起来。

**Q：为什么最后返回的是 `prev` 而不是 `curr`？**

**A：** 
*   当循环结束时，`curr` 已经是 `None` 了（因为循环条件是 `while curr`）。
*   `prev` 才是最后一个被处理的节点，也就是反转后的新头节点。
