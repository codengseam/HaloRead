## 题目信息

**题目名称**：复杂链表的复制

**LeetCode链接**：[Copy List with Random Pointer](https://leetcode.cn/problems/copy-list-with-random-pointer/)

**难度**：Medium

**描述**：请实现 `copyRandomList` 函数，复制一个复杂链表。在复杂链表中，每个节点除了有一个 `next` 指针指向下一个节点，还有一个 `random` 指针指向链表中的任意节点或者 `null`。



## 🧊 知识点"破冰"讲解

**哈希表 (HashMap)** vs **原地拼接法**。

*   **方法一：哈希表（土豪法）**
    *   准备一个小本本（Map）。
    *   第一遍遍历：只管克隆人，不管关系。见到 A，就造个 A'，并在本子上记下 `{A: A'}`。
    *   第二遍遍历：看着本子连线。A 的 next 是 B？那 A' 的 next 就指向 `Map[B]`；A 的 random 是 C？那 A' 的 random 就指向 `Map[C]`。
    *   **优点**：简单好懂。**缺点**：多用了一个 Map 的空间 O(N)。

*   **方法二：原地拼接（省钱法）**
    *   **第一步（影分身）**：把每个新节点挂在老节点后面。`A -> A' -> B -> B' -> C -> C'`。
    *   **第二步（连线）**：复制 random 指针。如果 A 的 random 指向 C，那么 A' 的 random 就指向 C'（也就是 `C.next`）。
    *   **第三步（分家）**：把长链表拆成两个。A 连 B，A' 连 B'。



## ❓ "为什么"的解释

**Q：为什么原地拼接法可以节省空间？**

**A：** 
*   哈希表法需要额外的 Map 来存储节点对应关系，空间复杂度 O(N)。
*   原地拼接法利用原链表的结构，把新节点插入到原节点后面，不需要额外的数据结构。
*   虽然修改了原链表，但最后会恢复，所以不影响。

**Q：原地拼接法的三个步骤分别做什么？**

**A：** 
1.  **影分身**：复制节点并插入到原节点后面。
2.  **连线**：利用 `node.next.random = node.random.next` 设置 random 指针。
3.  **分家**：拆分链表，恢复原链表，提取新链表。
