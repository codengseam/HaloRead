## 题目信息

**题目信息**：LeetCode 剑指 Offer 06. 从尾到头打印链表

*   **原题链接**：[剑指 Offer 06. 从尾到头打印链表](https://leetcode.cn/problems/cong-wei-dao-tou-da-yin-lian-biao-lcof/)
*   **难度**：Easy
*   **描述**：输入一个链表的头节点，从尾到头反过来返回每个节点的值（用数组返回）。
    *   **示例**：输入 `head = [1,3,2]`，输出 `[2,3,1]`。

### 🛒 题目 "人话" 翻译

这道题就像是一个 **"单向寻宝游戏"**。

*   你手里拿着一张藏宝图，上面只写着第一个线索在哪。
*   你顺着线索找到第一个点，那里只有一张纸条，写着下一个点在哪。
*   你就这样一直找一直找，直到找到终点。

**你的任务**：等你跑到终点后，你需要 **倒着回忆** 这一路经过的所有地点。比如你先去了 A，再去了 B，最后去了 C，现在你要告诉我："C -> B -> A"。



### 💡 思路 "推导" 过程

我们来模拟一下小白的思考过程：

**第一阶段：暴力解法（小本本记下来）** 🐢
*   **想法**：既然链表只能从头走到尾，那我就拿个小本本（列表），每到一个地方就把名字记下来。
*   **行动**：A -> B -> C，本子上记了 [A, B, C]。
*   **最后**：看着本子，从最后一行开始念，或者直接把本子翻个底朝天（反转列表）。
*   **结局**：非常完美！简单粗暴且有效。

**第二阶段：看起来很厉害的解法（递归）** 🐇
*   **想法**：利用 **递归** 的神力。
    *   递归函数说："嘿，你先把后面的节点打印出来，再打印你自己。"
    *   比如对 A 说："你去搞定 B 及其后面的人，完了再打印 A"。
    *   对 B 说："你去搞定 C 及其后面的人，完了再打印 B"。
    *   C 后面没人了，打印 C。任务完成，回过头打印 B，再打印 A。
*   **结局**：代码很短，逼格很高。
*   **问题**：如果链表特别长（比如几万个节点），递归层数太深，电脑的脑子（栈内存）可能会爆掉（Stack Overflow）。所以工程上还是推荐第一种"小本本"法。



### 🚗 代码 "带逛"

这里展示最实用、最稳妥的 **"栈（列表）法"**。

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

def reversePrint(head):
    """
    从尾到头打印链表
    :param head: ListNode 链表的头节点
    :return: List[int] 倒序的节点值列表
    """
    # 1. 准备一个"小本本"（栈/列表），用来记录路过的节点
    stack = []
    
    # 2. 从头开始，一个一个顺藤摸瓜
    current_node = head
    while current_node:
        # 把当前节点的值记在小本本上
        stack.append(current_node.val)
        # 顺着线索找下一个节点
        current_node = current_node.next
        
    # 3. 此时 stack 里存的是 [A, B, C]
    # 我们需要返回倒序的 [C, B, A]
    # Python 的切片操作 [::-1] 就是把列表翻转
    return stack[::-1]
```

**解法二：递归法（Russian Nesting Doll Approach）**

```python
def reversePrint_recursion(head):
    """
    递归解法：利用系统栈（函数调用栈）来实现倒序
    :param head: ListNode 当前处理的节点
    :return: List[int] 倒序的列表
    """
    # 1. 递归的尽头（Base Case）：如果节点是空的，说明走到头了
    # 就像俄罗斯套娃拆到最后一个，发现里面是空的
    if not head:
        return []
    
    # 2. 递（Go deeper）：先去搞定"后面的人"
    # 我们先把任务交给下一个人（head.next），让他把后面的结果给我
    # 比如现在的链表是 [1, 2, 3]，我们在处理 1
    # 我们对 2 说："你把 [2, 3] 倒序后的结果给我！"
    # 2 会对 3 说："你把 [3] 倒序后的结果给我！"
    sub_result = reversePrint_recursion(head.next)
    
    # 3. 归（Return）：把"后面的人"给我的结果，加上"我自己"
    # 当 2 拿到 [3] 后，它把自己加进去，变成了 [3, 2]
    # 当 1 拿到 [3, 2] 后，它把自己加进去，变成了 [3, 2, 1]
    # 注意顺序：后面的结果 + [我自己]
    return sub_result + [head.val]
```
