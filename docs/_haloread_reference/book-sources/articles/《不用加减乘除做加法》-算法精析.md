## 题目信息

*   **难度**：Easy
*   **描述**：写一个函数,求两个整数之和,要求在函数体内不得使用 "+"、"-"、"*"、"/" 四则运算符号。

### 🛒 题目 "人话" 翻译

**"计算机的底层思维"**。

*   虽然我们平时写代码都是 `a + b`,但 CPU 其实只会玩二进制。
*   它通过 **异或 (XOR)** 和 **与 (AND)** 来模拟加法。



### 💡 思路 "推导" 过程

我们来模拟一下小白的思考过程：

**第一阶段：观察二进制加法** 🐢
*   **想法**：加法在二进制里是怎么实现的？
*   **观察**：
    *   `1 + 0 = 1` (异或)
    *   `1 + 1 = 10` (进位)
    *   `0 + 0 = 0` (异或)
*   **推导**：加法可以分解为"不进位加法"和"进位"两部分。

**第二阶段：位运算模拟** 🐇
*   **想法**：用异或模拟不进位加法,用与模拟进位。
*   **公式**：
    *   `sum = a ^ b` (不进位和)
    *   `carry = (a & b) << 1` (进位)
*   **循环**：把 sum 和 carry 继续相加,直到 carry 为 0。

**第三阶段：Python 特殊处理** ✨
*   **问题**：Python 的整数是无限大的,负数处理起来会有点麻烦（不会自动溢出截断）。
*   **解法**：需要手动模拟 32 位整数的溢出。



### 🚗 代码 "带逛"

Python 的整数是无限大的,所以负数处理起来会有点麻烦（不会自动溢出截断）,需要手动模拟 32 位整数的溢出。

```python
def add(a, b):
    # 0xFFFFFFFF 是 32 位全 1 的掩码
    x = 0xFFFFFFFF
    
    # Python 的整数是大整数,负数显示的补码很长
    # 为了模拟 32 位 int,我们需要把它们限制在 32 位内
    a &= x
    b &= x
    
    while b != 0:
        # 1. 算出不进位的和
        # 2. 算出进位
        # 3. 把进位加到和上（这就变成了下一轮的加法任务）
        carry = (a & b) << 1
        a ^= b
        b = carry & x  # 进位也要限制在 32 位
        
    # 如果结果是正数,直接返回
    # 如果结果最高位是 1（负数）,需要还原成 Python 的负数形式
    # ~(a ^ x) 是取反操作的逆运算
    return a if a <= 0x7FFFFFFF else ~(a ^ x)
```
