## 题目信息

**题目信息**：LeetCode 剑指 Offer 51. 数组中的逆序对

*   **原题链接**：[数组中的逆序对](https://leetcode.cn/problems/shu-zu-zhong-de-ni-xu-dui-lcof/)
*   **难度**：Hard
*   **描述**：在数组中的两个数字,如果前面一个数字大于后面的数字,则这两个数字组成一个逆序对。输入一个数组,求出这个数组中的逆序对的总数。

### 🛒 题目 "人话" 翻译

**"插队统计"**。

*   想象大家在排队买票。
*   本来应该按身高从小到大排（升序）。
*   但是如果你发现前面有个高个子挡住了后面的矮个子,这就是一个 **"逆序对"**。
*   你的任务是数一数,队伍里总共有多少这种"不和谐"的组合。



### 💡 思路 "推导" 过程

我们来模拟一下小白的思考过程：

**第一阶段：暴力统计** 🐢
*   **想法**：两层循环,统计所有逆序对。
*   **问题**：时间复杂度 O(N^2),数据量大时会超时。

**第二阶段：归并排序优化** 🐇
*   **想法**：在归并排序的过程中统计逆序对。
*   **关键点**：
    *   当左边数组的元素 > 右边数组的元素时,产生逆序对。
    *   因为左边数组已排序,所以左边当前元素及其后面的所有元素都能和右边当前元素构成逆序对。
*   **公式**：`count += (mid - i + 1)`,其中 `i` 是左边当前元素的索引,`mid` 是左边数组的末尾。

**第三阶段：实现细节** ✨
*   **步骤**：
    1.  递归排序左半部分
    2.  递归排序右半部分
    3.  合并两个有序数组,同时统计逆序对
    4.  返回总逆序对数



### 🚗 代码 "带逛"

```python
def reversePairs(nums):
    """
    计算逆序对的总数
    """
    # 辅助函数：归并排序的同时统计逆序对
    def merge_sort(nums, start, end, temp):
        if start >= end:
            return 0
            
        mid = (start + end) // 2
        
        # 递归统计左半边和右半边
        left_count = merge_sort(nums, start, mid, temp)
        right_count = merge_sort(nums, mid + 1, end, temp)
        
        # 统计跨越左右两边的逆序对
        merge_count = 0
        
        # i 指向左半边的末尾, j 指向右半边的末尾
        i, j = mid, end
        # k 指向临时数组的末尾
        k = end
        
        while i >= start and j >= mid + 1:
            if nums[i] > nums[j]:
                # 核心逻辑：如果左边的当前数字 > 右边的当前数字
                # 说明左边当前数字及其前面的所有数字都能和右边这个数字构成逆序对
                # (因为左边是排好序的)
                merge_count += (j - mid)
                temp[k] = nums[i]
                k -= 1
                i -= 1
            else:
                temp[k] = nums[j]
                k -= 1
                j -= 1
                
        # 处理剩下的
        while i >= start:
            temp[k] = nums[i]
            k -= 1
            i -= 1
        while j >= mid + 1:
            temp[k] = nums[j]
            k -= 1
            j -= 1
            
        # 把排好序的临时数组拷回去
        nums[start:end+1] = temp[start:end+1]
        
        return left_count + right_count + merge_count

    if not nums: return 0
    temp = [0] * len(nums)
    return merge_sort(nums, 0, len(nums) - 1, temp)
```
