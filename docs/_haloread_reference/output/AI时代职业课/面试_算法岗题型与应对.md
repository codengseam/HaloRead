# 八股、手撕、项目、工程：算法岗怎么筛人

算法岗——大模型、算法工程师、机器学习工程师——在 2026 年依然是 AI 求职赛道上最拥挤也最挑剔的一栏。岗位变少、要求变高、面试官越来越像"问题解决者"而非"算法研究者"在挑人。本文按"八股—手撕—项目—关注点—应对"五个维度，把社区面经中真实出现过的高频题型拆给你看，并给出可落地的应对策略。

## 一、高频八股：从"背"到"推导"

以下题目整理自社区面经（来源：牛客/阿里系 nowcoder.com/discuss/848942791164981248），覆盖 Transformer、训练范式、推理工程三大块。

### 1.1 Transformer 与位置编码

**Self-Attention 为什么除以 √d_k？** 这几乎是开场必问题。本质是防止点积 QK^T 数值过大，让 softmax 进入饱和区、梯度趋零。除以 √d_k 让点积方差稳定在 O(1)，这是缩放点积注意力的核心动机。如果你只能背"防止梯度消失"，面试官会追问"为什么除以 √d_k 而不是 d_k 或 d_k²"，正确答案应回溯到方差推导：当 Q、K 各维独立同分布、均值 0 方差 1 时，点积方差为 d_k，标准差 √d_k，除之即归一。

**RoPE 相对位置编码的复数域旋转矩阵推导。** 高阶题，能筛掉一半候选人。RoPE 把位置信息编码为复数域上的旋转：对查询向量 q 的每两个维度视为一个复数，乘以 e^{imθ}（m 为位置、θ 为旋转角度），键向量 k 同理，则 q·k 的内积只依赖相对位置 m−n。面试官喜欢追问"为什么 RoPE 能外推""长度外推时 NTK-aware、YaRN 在改什么"。能讲清"旋转矩阵对相对位置的内积不变性"+"高频分量衰减快、低频分量可外推"才算过关。

**Pre-Norm vs Post-Norm。** Post-Norm（原始 Transformer）训练不稳定但效果略好，Pre-Norm（GPT/Llama 系列）训练稳定、可堆深层。深层大模型几乎清一色 Pre-Norm。

**RMSNorm 为什么去掉均值项？** LayerNorm 减均值再除标准差，RMSNorm 只除均方根、不减均值。原因：在激活值均值近似为 0 的高维残差网络里，减均值带来的收益有限但计算开销不小，去掉后速度更快、效果几乎无损——Llama 全系采用。

### 1.2 训练范式

**RLHF 三阶段：SFT → RM → PPO。** 标准答案要讲清：SFT 监督微调让模型学会指令跟随；RM 训练奖励模型对同一 prompt 的多个回答打分；PPO 用奖励模型作 critic 优化策略。追问常落在"RM 的 pairwise loss 怎么写""PPO 中 KL 散度惩罚项的作用""为什么 PPO 训练不稳定"。

**GRPO vs DPO vs PPO。** 2026 年必问。PPO 在线、需要 critic、显存重；DPO 离线、把 RM 隐式化进策略、两阶段变一阶段，但依赖成对偏好数据；GRPO（DeepSeek 提出）省掉 critic、用组内相对优势估计，显存友好、适合大模型。能讲清三者的"显存—数据—稳定性"权衡，比背定义值钱十倍。

**LoRA 微调为何优先 Q/K/V/O？** 因为注意力投影矩阵是"信息混合"的核心，低秩适配在此收益最大。也有论文实验显示全矩阵微调与 LoRA 在 Q/V 上效果接近，但工程上 LoRA 显存省一个数量级。陷阱题：LoRA 的秩 r 怎么选？太小欠拟合，太大退化成全微调，常用 8/16/32 起步。

### 1.3 推理与工程

**KV Cache 显存计算。** 给公式 `Memory_KV = L × 2 × H × d × Precision × S × B`（L 层数、H 头数、d 头维度、Precision 字节、S 序列长度、B 批大小），让你估 Llama-3 的 KV Cache 大小。算下来约 2GB 量级，再追问"怎么省"——答案指向 GQA、PagedAttention、量化 KV。

**GQA / MHA / MQA 权衡。** MHA 每个头独立 K/V，效果最好显存最大；MQA 所有头共享一组 K/V，显存最省效果损失明显；GQA 分组共享，是二者折中，Llama-3 用 GQA。

**SwiGLU 参数量陷阱题。** 标准 FFN 是两层线性，中间维度通常 4d。SwiGLU 用两个门控分支，中间维度是 2d/3d 而**不是 4d**——很多候选人惯性答 4d 直接挂。Llama 实际把中间维度设为 `2/3 × 4d` 以保持总参数量与标准 FFN 相当。

## 二、手撕代码：从"刷题"到"造轮子"

以下题目整理自社区面经（来源：牛客阿旺 nowcoder.com/feed/main/detail/5a43a5d03e5746e5b743bb488a5c00f9），算法岗面试的手撕分两类：模型组件手撕 + LeetCode。

### 2.1 模型组件手撕

| 组件 | 考点 |
|---|---|
| RoPE | 复数旋转实现，注意 dtype 与广播 |
| MHA / MQA / GQA / MLA | 头分组与 KV 共享的差别 |
| LayerNorm / RMSNorm | 维度方向、减均值与否 |
| SwiGLU | 门控相乘、中间维度陷阱 |
| KL 散度 | 数值稳定（logsumexp）|

面试官看的不只是跑通，而是你能不能 5 分钟内写出可读、向量化、dtype 正确的实现。写 for 循环逐元素算的，基本凉一半。

### 2.2 LeetCode 高频

- 674 最长连续递增子序列（滑动窗口）
- 25 K 个一组翻转链表（指针操作）
- 二分求根号（浮点二分边界）
- 无序数组第 K 大（快速选择 / 堆）
- 滑动窗口最大值（单调队列）
- 判断链表有环（快慢指针）

刷题不是目的，目的是让面试官确认你的工程基本功没退化。

## 三、项目追问：从"做了什么"到"为什么这么做"

项目深挖是算法岗真正的筛人环节。下面这条追问链整理自社区面经（来源：作业帮一面凉经 nowcoder.com feed/ccb934acd8e74b15a736d61375b7ed32）：

> 业务背景 → 数据来源 → 训练方法选型理由 → 评估效果（不止 BLEU，要业务指标）→ 幻觉处理 → 能力通用性

每一步都在逼你回答"为什么"。比如：

- "为什么用 LoRA 而不是全量微调？" —— 不能只说"省显存"，要讲"我们的任务是领域适配而非能力注入，低秩假设成立"。
- "评估效果用什么指标？" —— BLEU/ROUGE 是过程指标，面试官要的是业务指标：客服场景的人工接管率下降多少、文档问答的首答准确率提升多少。
- "幻觉怎么处理？" —— RAG 兜底 + 输出置信度过滤 + 人工抽检，三件套。
- "能力能迁移到其他业务吗？" —— 这是考察你有没有"通用方法论"意识，而不是只会调一个垂域模型。

GUI-Agent 类项目尤其会被拷打极深：怎么定义"操作成功"、怎么处理动态 UI 元素、怎么解决长 horizon 的误差累积、为什么用 Set-of-Mark 而不是纯文本。准备这类项目的同学，建议把"决策点—失败案例—改进—复测"四件套写进简历备注。

## 四、面试官关注点：黄金三角

2026 年面试官的评判标准已经从"算法研究者"转向"问题解决者"。社区调研显示（来源：掘金 2026），AI 工具使用能力在算法岗评估中的占比从 2024 年的 5% 升至 2026 年的 25%，而纯论文复现能力的权重在下降。

面试官真正在找的，是"黄金三角"齐全的人：

1. **技术落地**：能把论文里的方法跑通、调对、上线。
2. **工程化**：会处理 OOM、会做量化（GPTQ/AWQ）、会用 ZeRO-2/ZeRO-3、会 prof 耗时瓶颈。
3. **商业价值**：能说清"这个模型上线后给业务带来什么"——省钱、提效、还是增收。

三角缺一角，就会被卡。只会调包的"调参侠"和只会发 paper 的"研究员"在工业界都不再吃香。

## 五、应对策略：三位一体

把准备做成三件事，每件事都有可量化的产出物。

### 5.1 理论深度：练"为什么"

八股不要背，要练推导。准备一份自己的"推导笔记"，覆盖：注意力缩放的方差推导、RoPE 的复数推导、PPO 的目标函数与 clip、DPO 的 Bradley-Terry 推导、SwiGLU 的参数量计算。能在白板上推一遍，比背十遍定义管用。

### 5.2 项目落地：练"量化"

每个项目至少准备三个数字：基线指标、你的方法、提升幅度。例如"客服首答准确率从 68% → 81%，人工接管率从 23% → 14%"。没有数字的项目在面试官眼里等于没做过。

### 5.3 工程优化：练"瓶颈分析"

准备一个"训练踩坑清单"：OOM 怎么排查（梯度累积、ZeRO、checkpoint）、推理慢怎么 prof（KV Cache、算子融合、量化）、分布式怎么调试（NCCL hang、负载不均）。能讲出"我遇到过 X 瓶颈，定位用 Y 工具，最终用 Z 方案解决"的完整链路，比简历上写"熟悉分布式训练"有说服力一百倍。

算法岗的面试，本质是在筛"能在不确定中把模型送到线上"的人。八股是门票，手撕是底线，项目是分水岭，工程化是加分项，商业判断力才是 offer 的决定因素。把这五层都练扎实，再去投简历。
