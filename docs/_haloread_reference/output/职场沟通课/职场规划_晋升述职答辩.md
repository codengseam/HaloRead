# 一场走神的述职

## 五十页流水账

某公司年度晋升答辩，小王上台讲述职。

他打开一份五十页的演示文稿，从年初第一个项目讲起："一月份我参与了X项目，负责前端开发，用了React，做了三个页面……"

讲到第五分钟，评委开始看手机。讲到第十分钟，评委交头接耳。讲到第十五分钟，小王还在讲三月份的一个小需求。

主讲人打断他："你直接说，你这一年最大的贡献是什么？"

小王愣了一下："我做了很多……"

"具体哪个项目？你负责什么？结果是什么？"

小王支支吾吾："X项目我负责前端……"

"前端是你一个人做的吗？"

"不是，还有两个人……"

"那你具体做了哪部分？"

小王答不上来。

评委心里已经有数了——这人要么没干多少，要么干了说不清。无论哪种，都不该升。

小王冤不冤？也许他干得不少。可述职答辩比的是"讲不讲得清"，"干了多少"反在其次。讲不清，等于没干。

## 述职是证明，表功只是表象

很多人把述职当表功——我把这一年干的事都列出来，评委看着给分。

这是错的。

述职的本质，是证明你已经达到下一级的能力。

评委评的是"你能不能干下一级的活"，"你干了多少活"在其次。你列了一堆执行层的活，评委看完只会想："这人干得挺多，但还是执行者的思路，没到下一级。"

所以述职的核心是证明——证明你的能力已经到了下一级。

怎么证明？靠你选什么项目讲、怎么讲、怎么答。

- 选项目：选能体现下一级能力的项目，不是选最大的项目。
- 讲项目：讲你的思考、判断、决策，不是讲你做了什么操作。
- 答辩：答出深度，证明你是真的懂，不是蹭的。

述职答辩两件事：述职讲"我到了下一级"，答辩验证"你真的到了"。

## 冯谖焚券市义

《战国策·齐策》里有个故事。

孟尝君让门客冯谖去薛地收债。冯谖问："收完债，买点什么回来？"孟尝君说："你看我家缺什么就买什么。"

冯谖到了薛地，把欠债的百姓召集起来，核对完债券，当众一把火烧了。回来跟孟尝君说："我给您买回了'义'。"

孟尝君当时不高兴——他缺的是钱，不是义。

后来孟尝君被齐王罢免，回到薛地，薛地百姓扶老携幼出城迎接。孟尝君这才明白冯谖当年烧债券的用意——提前布局，关键时刻拿出成果。

冯谖的高明在哪？他没等孟尝君被罢免了才去收买人心，而是提前一年就布局。等关键时刻到来，成果自然显现。

述职答辩也是这个道理。

你不能等述职前一周才开始准备"我这一年干了什么"，那时候能拿出来的只有流水账。真正会述职的人，从年初就在布局——选什么项目、攒什么数据、留什么证据，到了年底述职，自然有东西可讲。

述职靠全年布局，年底只是收口。年底讲的每一句话，背后都该有提前攒好的证据。

## 述职三段式

述职怎么讲？用三段式。

**第一段：价值贡献。**

讲你这一年创造了什么价值。讲你做的事带来了什么结果，不是讲你做了多少事。

错误讲法："我今年做了五个项目，修复了二十个bug，写了三万行代码。"
正确讲法："我今年主导了X项目，把转化率从百分之十五提到百分之二十二，给公司带来Y万营收。"

价值贡献的要点：
- 讲结果，不讲过程。评委要的是"你做成了什么"，"你做了什么"反在其次。
- 讲数据，不讲感受。"转化率提升百分之五十"比"效果很好"有说服力一百倍。
- 讲贡献，不讲参与。"我主导""我负责"比"我参与"强。参与了不算贡献，主导了才算。

**第二段：能力提升。**

讲你这一年能力上有什么提升。这一段是证明你"到了下一级"的关键。

错误讲法："我学了很多新技术，成长很大。"
正确讲法："今年我从执行者升级为项目协调者，独立协调了三个部门、十二个人，把项目从延期一个月拉回按时交付。这让我具备了下一级需要的协调能力。"

能力提升的要点：
- 讲升级，不讲学习。学习是过程，升级是结果。"我从X升级到Y"比"我学了X"强。
- 对标下一级。讲的能力，要正好是下一级需要的。下一级要协调能力，你就讲协调能力；下一级要带人能力，你就讲带人能力。
- 有证据。每个能力提升，都要有具体项目支撑。空谈能力，评委不信。

**第三段：未来规划。**

讲你下一年的规划。这一段是证明你"已经在想下一级的事"。

错误讲法："明年我继续努力，把工作做得更好。"
正确讲法："明年我计划主导X项目，目标是Y。同时我打算带一个新人，把Z这块的经验沉淀成文档，让团队能复用。"

未来规划的要点：
- 讲具体，不讲口号。"主导X项目""带一个新人""沉淀Z文档"比"继续努力"强。
- 讲下一级的事。你的规划要体现你已经站在下一级的视角想问题。
- 讲对团队的价值。不只是"我要做什么"，是"我做的事对团队有什么贡献"。

三段式讲完，评委心里就有数了：这人创造了什么价值、具备了什么能力、未来能干什么。三个问题答清，晋升就有戏。

## 答辩四类问题

述职讲完，进入答辩。评委追问，主要四类问题。

**第一类：你具体做了什么？**

评委问这个，是在验证你是不是蹭的。

错误回答："我负责X项目。"——太虚，评委不信。
正确回答："X项目里，我负责方案设计、跨部门协调、上线跟进。方案设计是我独立完成的，跨部门协调我对接了产品和测试。"——具体到模块，评委信。

应对要点：把"我负责"拆成"我具体做了哪几块"，每块都能展开讲。

**第二类：你是怎么做的？**

评委问这个，是在验证你的方法论。

错误回答："就正常做呗。"——没方法论，评委觉得你是蒙的。
正确回答："我先调研了三个同类方案，对比成本和风险，选了A方案。实施时分三步：先做MVP验证，再扩展功能，最后优化性能。"——有方法论，评委信。

应对要点：讲清楚你的思考过程——为什么这么做、有没有别的选择、为什么选这个。

**第三类：遇到什么困难？**

评委问这个，是在验证你的解决问题能力。

错误回答："没什么困难，挺顺利的。"——评委会想：要么你干的活太简单，要么你在撒谎。
正确回答："最大的困难是跨部门协调。产品要快，测试要稳，两边吵起来。我先分别听诉求，找到共同点——都要按时上线。然后提折中方案：核心功能先上，边缘功能延后。两边都接受了。"——有困难、有解决、有结果。

应对要点：讲一个真实的困难，讲清楚你怎么分析的、怎么解决的、结果如何。困难别太大（显得你搞不定），也别太小（显得你没挑战）。

**第四类：你学到了什么？**

评委问这个，是在验证你的复盘能力和成长性。

错误回答："学到了很多。"——太虚。
正确回答："这次最大的教训是前期调研不够，导致中途返工。下次我会把调研做透再动手，宁可前期多花一周，也不要后期返工两周。"——有具体教训、有改进方向。

应对要点：讲一个具体的教训，讲清楚你下次会怎么改。复盘展示成长性，评委最看重这个。

## 名家解析

明托在《金字塔原理》中提出"结论先行"的沟通原则：先讲结论，再讲支撑。述职答辩尤其适用——评委时间有限，第一句话就要让他听到你的核心价值。结论埋在过程里，评委走神了就再也找不回来。

## 避坑

**第一坑：讲成流水账。** 从年初讲到年底，事无巨细。用三段式，每段聚焦核心。

**第二坑：只讲参与不讲主导。** "我参与"不算贡献，"我主导"才算。分不清参与和主导，评委觉得你没担当。

**第三坑：答辩答不出深度。** 评委一追问就支吾，说明你要么没干、要么没想。每个项目都要能展开讲三层：做了什么、怎么做的、为什么这么做。

**第四坑：只报喜不报忧。** 评委一问困难就卡壳。讲一个真实的困难和你怎么解决的，比全讲成功更有说服力。

**第五坑：临时抱佛脚。** 述职前一周才开始准备，只能拿出流水账。从年初就布局，选项目、攒数据、留证据，年底自然有东西讲。

## 实操

用三段式写一份你的晋升述职提纲：

**第一段：价值贡献（占述职时间一半）**
- 列出你今年主导的两个核心项目
- 每个项目写一句话结果：做了什么+带来什么数据结果
- 删掉所有"参与"的项目，只留"主导"的

**第二段：能力提升（占述职时间三成）**
- 对标下一级的能力要求，列出你今年提升的两项能力
- 每项能力配一个具体项目做证据
- 用"从X升级到Y"的句式，不讲"我学了X"

**第三段：未来规划（占述职时间两成）**
- 列出明年你打算主导的一个项目
- 列出你打算带新人或沉淀经验的具体动作
- 每条规划都要体现你站在下一级视角想问题

提纲写完，再准备答辩四类问题的答案：每个核心项目都能答出"做了什么、怎么做、遇到什么困难、学到什么"。四类问题答得出来，答辩就稳了。

## 一句话

述职的本质是证明你已经达到下一级的能力——三段式讲清价值、能力、规划，答辩四类问题答出深度，全年布局，年底自然有东西可讲。
