# 沟通的本质是利益对齐

## 跨部门那堵墙

老陈在产品部，负责一个新功能上线。他需要技术部配合改接口，约了三次会，技术部的张工每次都说"忙，下周再说"。

老陈急了，找技术部总监。总监说："我们排期满了，你这事不急的话往后排。"老陈更急了，回去写了一封长长的邮件，抄送了一堆领导，列了十八条理由说明这个功能多重要。

邮件发出去三天，没人回。

老陈跟同事吐槽："技术部那帮人太难搞了，仗着自己手里有资源，谁都不放在眼里。"

同事问他："你跟张工聊的时候，知道他这个季度背的是什么KPI吗？"

老陈愣住。

他从来没想过这个问题。他满脑子都是"我的项目要上线"，从来没想过"张工为什么要帮我"。

这就是大多数人沟通失败的根本原因：你以为对方难搞，其实是没找到对方的利益点。

## 沟通不是表达自我，是找交集

很多人对沟通有个误解，以为沟通就是把自己的想法说清楚。所以他们在沟通前花大量时间准备"我要说什么"，列提纲、做PPT、反复措辞。

然后沟通失败，他们困惑："我都说这么清楚了，他怎么就不懂呢？"

问题在于：你说得再清楚，跟对方有什么关系？

沟通不是单向广播，是双向校准。你要的和他要的，往往不是一回事。你以为你在讲道理，他在算成本；你以为你在谈情怀，他在看风险；你以为你在推进项目，他在想这事会不会给他添麻烦。

真正的沟通，是先搞清楚双方各自要什么，然后找到那个交集。交集找到了，话就好说；交集找不到，你说破天也没用。

每次沟通前，先问自己三个问题：对方要什么？我要什么？交集在哪？

这三个问题问不出来，就别急着开口。问出来了，沟通就成功了一半。

## 触龙怎么劝动赵太后

《战国策》里有一个教科书级别的沟通案例：触龙说赵太后。

战国时期，秦国打赵国，赵国向齐国求救。齐国说：可以救，但要把赵太后的小儿子长安君送来当人质。太后死活不肯，大臣们轮流劝，太后急了，放话说："谁再敢劝我把长安君送出去，老娘就朝他脸上吐口水。"

满朝文武没人敢说话了。

触龙去见太后。太后知道他来干什么，沉着脸等他。触龙进门，慢慢走，坐下后先不提长安君，而是说："老臣腿脚不好，好久没来拜见太后了，今天特地来看看。太后最近饭量怎么样？每天走几步？"

太后脸色缓和了一点，说："我靠人推车走，饭量也一般。"

触龙又聊起自己的小儿子："老臣有个不情之请，我那个小儿子舒祺，不成器，我想求太后给他个黑衣卫士的差事，趁我还在世，给他安排好。"

太后笑了："原来男人也疼小儿子啊？"

触龙说："比女人还疼。"

太后说："不对，女人更疼。"

触龙这时候才切入正题："太后您疼燕后（太后嫁到燕国的女儿）比疼长安君更甚。送燕后出嫁时您哭得那么伤心，可祭祀时您却祷告'千万别让她回来'，您是想让她子孙在燕国世代为王。这才是真疼孩子——为他的长远打算。"

太后沉默。

触龙接着说："现在长安君地位尊贵，封地肥沃，可他对赵国没什么功劳。一旦您不在了，他凭什么在赵国立足？您不让他趁现在为赵国立功，将来怎么服众？"

太后长叹一声："行吧，你安排吧。"

触龙从头到尾没说一句"国家大事为重"，没说一句"您不能这么自私"。他做的只有一件事：从太后爱儿子这个利益点切入，把"送人质"这件太后抗拒的事，翻译成"为长安君长远打算"这件太后认同的事。

利益对齐了，话就通了。

## 利益对齐三步法

触龙用的那套，可以拆成一个通用的框架：识别利益、翻译需求、设计共赢。

**第一步：识别利益。**

沟通前，先把双方的真实利益列出来。注意，是"真实利益"，不是"嘴上说的"。

张工嘴上说的是"忙"，真实利益可能是"这个季度我的KPI是稳定性，你这个新功能有风险，我帮你做了对我没好处，出了事还背锅"。

领导嘴上说的是"再考虑考虑"，真实利益可能是"这事我没把握，万一搞砸了我得担责"。

客户嘴上说的是"太贵了"，真实利益可能是"我预算就这么多，但我更怕的是买了用不上"。

识别利益的关键是问自己：他如果答应我，能得到什么好处？他如果拒绝我，是在怕什么？这两个问题答上来，对方的真实利益就浮出来了。

**第二步：翻译需求。**

把你要的东西，翻译成对方利益的语种。

老陈要张工改接口，他原来说的是"这个功能很重要，下周必须上线"。翻译成张工的利益语种，应该是："这个接口改动不大，但能帮你们组那个稳定性指标涨一截，因为现在的接口在高并发下会丢数据，改了之后线上事故率会降。我可以把这个功劳写进你们季度的稳定性总结里。"

同一个事，从"我要"翻译成"你得"，对方听的感受完全不一样。

**第三步：设计共赢。**

利益对齐与其各退一步地妥协，不如找到那个让双方都赚的方案。

老陈和张工的共赢方案可能是：老陈把接口改动拆成两个小迭代，第一个迭代只改最关键的丢数据问题，张工组用这个改动冲稳定性KPI；第二个迭代等张工组排期松了再做，老陈这边先上线核心功能。这样老陈的项目能按时上，张工的KPI也有交代。

共赢不是抽象的口号，是具体的方案。你得替对方想清楚"他答应你之后，怎么向他的领导交代"，他才会真的答应。

## 把人和问题分开

费希尔在《谈判力》里讲过一个核心原则：把人和问题分开。

大意据《谈判力》：对人柔和，对事强硬。

这句话听起来简单，做起来极难。我们一沟通就容易把人和事搅在一起。对方拒绝你的方案，你觉得对方针对你；对方态度不好，你觉得对方人品有问题。然后你带着情绪去沟通，事情更难谈。

把人和问题分开，意思是：对方拒绝的是这个方案，不是你这个人；对方态度不好，可能是他今天被领导骂了，跟你的方案好不好没关系。

老陈一开始就是把人和事搅在一起了。他觉得张工"难搞"，是针对他。其实张工只是按自己的KPI在排期，跟老陈这个人没半点关系。

把人和问题分开之后，老陈才能冷静地去想：张工的KPI是什么？我的方案怎么帮他完成KPI？这才是利益对齐的起点。

## 用利益地图分析你正在推的一件事

读完这一章，挑一件你正在推进、但卡住的事，画一张利益地图。

拿一张纸，画三栏。

第一栏写"我要什么"。不是表面的"项目上线"，是底层的"我背的KPI是什么，我担心什么，我希望这件事给我带来什么"。

第二栏写"对方要什么"。如果你不知道，就先去问、去观察、去侧面打听。这一栏写不出来，说明你对对方的了解还不够，沟通肯定卡。

第三栏写"交集在哪"。把前两栏对照着看，找那个双方都想要的点。如果找不到，就想想能不能设计一个让双方都赚的方案。

这张图画完，你大概率会发现：你之前卡住，不是因为对方难搞，是因为你一直在讲自己的利益，没看见对方的利益。

利益对齐不是技巧，是视角。是你愿不愿意暂时放下"我要什么"，先去看看"他要什么"。这个视角一换，沟通的难度会下降一大半。

职场里没有那么多难搞的人，只有那么多没被看见的利益。你看见了他的利益，他自然就看见了你的方案。
