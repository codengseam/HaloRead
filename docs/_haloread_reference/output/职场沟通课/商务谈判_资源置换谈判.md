# 没钱没权也能换到东西

## 求人不如换人

小杨在市场部，要搞一场线下活动，缺技术支持——需要一个H5页面、一套报名系统、现场扫码互动。

他去找技术部，技术部经理老陈一脸为难："排期满了，你这个加进来，得排到下个月。"

小杨急了："活动下周就办，下个月黄花菜都凉了。"

老陈摊手："那我也没办法，技术部就这几个人，需求排着队呢。"

小杨回去愁了一晚上。第二天他换了个思路——他想起老陈前阵子在饭桌上抱怨，说技术部招人难，简历筛不过来，HR那边又催得紧。

小杨找到老陈："你那边招人筛简历的事，我帮你搞。我认识几个技术社区的运营，可以帮你精准推简历，我还能帮你写招聘文案。但你得帮我一个忙——下周那个活动的H5和报名系统，给我插个队。"

老陈眼睛一亮："你真能帮我推简历？"

小杨说："我下午就给你推第一批，你看看质量。"

当天下午，小杨给老陈推了二十份简历，其中三份直接进了面试环节。老陈高兴坏了，第二天就安排人给小杨的活动插了队。

活动办完，老陈还专门发消息谢小杨："下次有事还找我。"

小杨没花一分钱，没动用一丝权力，用自己手上的资源换了技术部的排期。

求人不如换人。

## 置换的本质是做大蛋糕

很多人在职场里要资源，只会两条路：要么找领导要，要么求同事帮忙。

找领导要——领导不一定给，给了也有限制。
求同事帮忙——同事不一定帮，帮了也欠人情。

这两条路都是"单向索取"——你拿别人的资源，不给对方任何东西。对方凭什么给你？

资源置换是第三条路：**找到对方也缺的东西，用你有的换你要的**。

它的本质是**做大蛋糕**。

费希尔在《谈判力》里讲过一个核心思路：谈判不是切分固定大小的蛋糕，而是想办法把蛋糕做大，让双方都拿到更多。

资源置换就是做大蛋糕的思路。

你要技术部的排期，技术部要招人的简历。两样东西在各自部门都是稀缺资源，但两样东西在对方眼里都有价值。你用你手上的（简历渠道）换你想要的（技术排期），双方都拿到了原本拿不到的东西。

蛋糕做大了一倍——你拿到了排期，技术部拿到了简历。双方都比置换前更好。

这就是置换的力量——**不靠权力、不靠金钱、不靠人情，靠的是找到双方都有需求、都能给的东西**。

## 触龙的"为之计深远"

《战国策·赵策》里触龙说赵太后的故事，是资源置换的教科书案例。

赵太后的小儿子长安君要去齐国当人质，齐国才肯出兵救赵。太后舍不得儿子，怎么劝都不行，大臣们急得团团转。

触龙去见太后。他没一上来就劝"您得让长安君去"，他知道这条路走不通——太后爱子心切，硬劝只会让她更抗拒。

触龙换了个思路。

他先聊家常："老臣腿脚不好，好久没来给您请安了。您饭量还行吧？走路还利索吧？"太后放松了警惕，把他当成长辈来聊天。

然后触龙提了个请求："老臣有个小儿子，最不成器，能不能请您给他安排个黑衣卫士的差事？"

太后笑了："男人也疼小儿子？"

触龙说："比女人还疼。"

太后说:"我看是女人更疼。"

触龙顺势切入正题："老臣觉得，您爱燕后（太后的女儿）胜过爱长安君。"

太后说:"不，我爱长安君更多。"

触龙说:"父母爱子，则为之计深远。您送燕后出嫁的时候，哭着舍不得，但祭祀时却祈祷'千万别回来'——您是希望她在那边站稳脚跟。这是为她计深远。"

太后点头。

触龙接着说:"可您对长安君呢？您给他尊贵的地位、丰厚的封地、国家的重器，却不让他为赵国立功。一旦您不在了，长安君凭什么在赵国立足？您这是爱他还是害他？"

太后沉默了。

触龙说:"让长安君去齐国当人质，是给他一个为赵国立功的机会。立了功，将来他在赵国才有地位。这才是为他计深远。"

太后想通了："行，你安排吧。"

长安君去了齐国，齐国出兵，赵国解围。

触龙这一套，是教科书级的资源置换。

第一步：**识别对方痛点**。太后的痛点不是"该不该送人质"，是"舍不得儿子去受苦"。触龙没在人质问题上纠缠，他抓住了痛点——爱子之心。

第二步：**重塑价值**。触龙把"送人质"这件事从"让儿子去受苦"重塑成"给儿子立功的机会"。同一件事，框架变了，太后的接受度就变了。

第三步：**做大蛋糕**。原来太后觉得送人质是"损失"——失去儿子。触龙让她看到送人质也是"收益"——儿子立功、赵国解围、儿子将来有地位。损失加收益，太后算账之后觉得划算。

资源置换的高手，不是会算计，是会重塑——把对方眼里的"付出"重塑成"收益"，让双方都觉得赚了。

## 资源置换三步

**第一步：盘点筹码。**

置换的前提是你有筹码。筹码不是钱、不是权，是你手上**对方也想要的东西**。

盘点筹码从三个方向想：

- **你独有的资源**：人脉、渠道、信息、技能、品牌、流量。这些是你有别人没有的。
- **你部门的资源**：你部门能调动的东西——市场部的内容生产能力、技术部的开发能力、运营部的用户触达能力。你能调动的，就是你的筹码。
- **你个人的能力**：你会写、会讲、会算、会搞关系。这些能力本身也是筹码，可以帮对方解决他的问题。

盘点的时候别小看自己。很多人觉得自己手上没什么筹码，其实是不识货。你认识一个技术社区的运营，对你来说是日常社交，对技术部招人来说就是稀缺渠道。你的写作能力，对你来说是本职工作，对其他部门写文案发愁的人来说就是救急资源。

筹码不分大小，分有没有用。对方需要的，再小也是筹码；对方不需要的，再大也没用。

**第二步：识别对方痛点。**

光有筹码不够，还得知道对方缺什么。

识别对方痛点从三个方向想：

- **对方的KPI**：他这个季度要完成什么？什么卡住了他？什么让他头疼？
- **对方的处境**：他最近在跟谁较劲？他在领导面前要交什么差？他缺什么资源？
- **对方的抱怨**：他平时抱怨什么？抱怨里藏着痛点。

识别痛点靠的是日常积累。平时多跟其他部门的人聊，多听少说，听到他们抱怨什么、愁什么，记下来。这些就是你将来置换的切入点。

痛点要具体。"技术部缺人"是模糊的，"技术部招前端工程师筛不到合适简历"是具体的。具体到这个程度，你才知道你的筹码能不能用上。

**第三步：设计互换。**

筹码有了，痛点找到了，设计互换就水到渠成。

设计互换要回答三个问题：

- 我给他什么？——具体到动作，不是"我帮你"，是"我帮你推二十份简历"。
- 他给我什么？——具体到动作，不是"他帮我"，是"他给我插一个排期"。
- 什么时候交付？——双方都给个时间点，避免拖延。

设计互换的时候，**对等**很重要。你给的价值和他给的价值要大致对等，不能你给一根葱换他一头牛。对等不是数字上的相等，是双方都觉得划算。

互换设计好，开口的时候要直接。"我帮你搞定简历的事，你帮我插个排期，怎么样？"——直接说，别绕。绕了对方觉得你有套路，反而警惕。

## 不置换核心利益

资源置换有底线：**不置换核心利益**。

什么是核心利益？是你或你部门赖以立身的东西。

市场部的核心利益是品牌调性——你不能为了换资源，让别的部门在你的品牌内容里塞乱七八糟的东西。
技术部的核心利益是系统稳定——你不能为了换资源，让技术部上线一个没测过的功能。
销售部的核心利益是客户关系——你不能为了换资源，把客户资源拱手让人。

核心利益一置换，短期得了好处，长期丢了立身之本。

置换的是**冗余资源**——你有富余的、对方缺的。不是核心资源——你赖以生存的、对方也想要的。

这条底线要守住。守不住的话，置换越多，你越没价值。你把核心资源都置换出去了，下次对方再来找你，你拿什么换？

## 别只想着"求人"

很多人在职场里要资源，第一反应是"求人"。

求领导批预算，求同事帮个忙，求别的部门给点支持。求的姿态一摆出来，你就矮半截——对方给不给你都得感恩，对方提什么条件你都得接受。

资源置换换个姿态——**不是求人，是合作**。

你手上有点东西，对方手上有点东西，双方都有需求，互换一下大家都好。这个姿态是平等的，对方给不给你都不欠人情，对方提什么条件你都可以谈。

姿态一变，谈判的主动权就变了。求人是对方说了算，置换是双方说了算。

职场里能调动资源的人，不一定是权力最大的人，往往是会置换的人。权力有边界，置换无边界——只要你能找到对方的需求，你就能换到资源。

## 实操

列出你手上的三个可置换筹码，想清楚能换什么。

按下面三个问题写：

1. 这个筹码是什么？具体到能做什么——不是"我有人脉"，是"我认识XX社区的运营，能精准推技术简历"。
2. 这个筹码对谁有用？哪个部门、哪个岗位、什么场景下会需要？
3. 用这个筹码能换什么？对方手上有什么是你想要的？排期、资源、信息、支持？

三个筹码列完，你心里就有了一张"置换地图"——谁缺什么、你能给什么、能换什么。

下次要资源的时候，别先想"求谁"，先想"换什么"。求人难，换人易。

## 一句话

资源置换的本质是做大蛋糕——盘点筹码、识别痛点、设计互换，不靠权力不靠金钱，靠的是找到双方都缺的东西。
