# 第一性原理拆穿类比

## 一、入戏

你想开一家咖啡店。第一件事干什么？大概率是去看别的咖啡店怎么干的——装修什么风格、菜单多少款、定价多少、选址在哪。你逛了二十家，记了一本笔记，然后照着行业平均的样子把自己的店搭起来。开业三个月，亏到关门。

你不知道的是，那二十家店里，赚钱的可能只有三四家，你抄的是它们的样子，没抄到它们赚钱的原因。你抄了装修，没抄到那家店是因为门口有早高峰人流才赚钱；你抄了菜单，没抄到那家店是因为豆子供应链是自己压下来的才赚钱。样子是结果，原因是另一回事。

这就是类比思维的陷阱——"别人怎么做我就怎么做"。大脑天生爱类比，因为类比省力：别人试过的，我照搬，省得自己想。但省力的代价是，你抄到了壳，抄不到核。第一性原理这把刀，就是专门用来拆这个壳的。它不问你"别人怎么干"，它问你"这件事最底的事实是什么"。

## 二、破题

第一性原理这个词由来已久。两千多年前亚里士多德就提过：大意据亚里士多德，在每一系统的探索中，存在第一本原，它是推理的起点，本身不能再被推出。意思是任何一套道理往下追，追到最底下总有几条不能再拆的基本事实，所有结论都从这几条往上长。

这套东西沉了两千年，被马斯克捡起来用到了造火箭上。他做 SpaceX 时，去买一发现成火箭，报价六千多万美元。他不信这个数，回去查铝、钛、碳纤维、铜这些原料的现货价，发现光是材料成本只占火箭售价的百分之二左右。他判断：火箭贵不是因为材料贵，是因为行业一直在用"买现成"的方式，层层加价、层层低效。他自己造，把成本压到原来的零头。

马斯克用的就是第一性原理：不问"别人卖多少"，问"火箭到底由什么构成、这些构成的市场价是多少、剩下的差价去哪了"。把"火箭一发六千万"这个类比结论拆掉，回到"原料、人工、工艺、利润"这些基本事实，再从基本事实往上重新推一遍。

贝佐斯说过一句互补的话：很多人问未来十年什么会变，他更关心未来十年什么不变。变的你要跟上，不变的才是你能押注的地基。这也是第一性原理的一种用法——不追潮流（潮流是类比），追恒量（恒量是基本事实）。

第一性原理跟类比思维的根本差别在哪？类比思维从"别人怎么干"出发，第一性原理从"这事最底的事实是什么"出发。前者抄结果，后者找原因。抄结果的人永远慢半拍，因为等你抄到，那个结果已经过时；找原因的人才能造出下一个让别人抄的结果。强者思维里这一刀的价值就在这——它把你从"跟随者"的位置上解放出来。

## 三、方法论

第一性原理听着玄，做起来就四步。这四步是把"拆壳找核"这个动作工程化。

**第一步：识别你在用类比。**

大多数决策你都没意识到自己在用类比。判断的办法是问自己一句话："我是怎么得出这个结论的？"如果答案是"大家都这样""行业惯例""我看某某就是这么做的""网上查的平均数"——你就在用类比。类比不一定是错，但你得知道自己在用，才有可能去拆它。

跳槽谈薪，你查"行业平均薪资"按平均数谈，这是类比；开店定价，你按"周边竞品均价"定，这是类比；教育孩子，你按"别人家孩子几岁学钢琴"安排，这也是类比。先把决策里这些"照搬"的地方标出来，才能进入下一步。

**第二步：拆到基本事实。**

标出类比之后，对每一个类比往下追：这个结论背后的基本事实是什么？拆的方法是连问"为什么"，问到不能再问为止。

"行业平均薪资两万"——为什么是两万？因为是市场上多数岗位的成交价。为什么是这些岗位的成交价？因为雇主愿意出这个价。雇主为什么愿意出这个价？因为这个人能为公司创造超过两万的价值。拆到这一层，基本事实浮出来了：薪资的本质在于你能创造的价值，行业平均数只是参照。行业平均两万，是因为市场上多数人创造的价值落在这个区间；你能创造多少价值，才是你的薪资该锚定的基本事实。这一拆，"按平均数谈"就成了笑话——你该按"我能创造多少价值"谈。

**第三步：从事实往上重建。**

拆到基本事实之后，不要急着下结论，先从事实重新推一遍。火箭由原料、人工、工艺、利润构成，原料占百分之二，那剩下的百分之九十八去哪了？去层层加价、去低效工艺、去行业惯性。如果你自己造、自己优化工艺，能把那百分之九十八压下来多少？这一推，"火箭一发六千万"就不再是铁律，是可被改写的中间结论。

重建的关键是只信基本事实，不信中间结论。中间结论是别人在某个特定条件下推出来的，换个条件就不成立；基本事实是物理的、数学的、市场的硬约束，到哪里都成立。从硬约束往上推，你推出来的方案才是真正属于你的方案。

**第四步：验证能不能落地。**

第一性原理推出来的方案，常有"听起来很美但落不了地"的风险。所以最后一步要验证：这个从基本事实推出来的方案，在现实里能不能跑通？卡点在哪？哪些卡点是真卡点（物理限制），哪些是假卡点（只是没人这么干过）？

马斯克拆完火箭成本，还要验证自造能不能解决工艺问题、能不能拿到发射许可、能不能找到客户。基本事实给你方向，验证给你落地。只拆不验，第一性原理就变成空想；只验不拆，你又退回类比。两步要一起跑。

## 四、避坑

第一性原理有三个坑，每一个都会让这把刀砍偏。

**坑一：把第一性原理当成推翻一切常识。**

有人一学第一性原理，看什么都想拆，连"过马路要看红绿灯"都要拆一遍，拆完得出"红绿灯只是惯例我可以闯"的结论。这是把第一性原理用成了抬杠工具。常识里有两类：一类是真正的基本事实（红绿灯保证通行秩序，是物理与社会规律的结合），一类是可拆的中间结论（行业惯例、平均数、约定俗成）。第一性原理只拆后者，前者保留。区分这两者的办法是问：这条"常识"背后有没有物理或数学的硬约束？有，它就是基本事实，别拆；没有，它是惯例，可以拆。把什么都拆，跟什么都不拆，一样懒。

**坑二：凡事都用第一性原理。**

第一性原理是重活，每拆一次要花时间精力。小事不值得拆，大事才值得。买杯咖啡按类比选一家就行，不用拆到咖啡豆的期货价；但选行业、选伴侣、做重大投资、设计产品，这些值得停下来好好拆。把第一性原理用在所有事上，你会累死在拆解里，一天干不成一件正事。判断标准很简单：这个决策的后果你能扛多久、扛多重？后果越重，越值得拆。

**坑三：把"我觉得"当成"物理事实"。**

拆到深处，最容易骗自己的就是把主观判断当成基本事实。"我觉得这个市场有需求""我觉得用户会喜欢""我觉得这条路走不通"——这些"我觉得"都算不上基本事实，只是你未经检验的假设。基本事实要可证伪、可查证：原料的现货价是查得到的，用户的付费行为是测得到的，物理常数是查得到的。曾国藩打仗讲究「结硬寨，打呆仗」——《曾国藩家书》，拙能胜巧，靠的正是这份拙劲，把"我觉得"挡在门外，一仗一仗把可验证的事实攒起来。拆假设时也要有这份拙劲——把每一个"我觉得"逼到"我能证明"为止，逼不下去的，就老实承认它是假设，离事实还差一步。

## 五、践行

这一周，做这三件事，把第一性原理这把刀用上手。

第一件，挑一个你正准备"照搬"的决策，做一次完整拆解。比如你正打算报某个培训班因为"大家都报"，或者正打算按某个模板写简历因为"网上都这么写"。把这个决策写下来，连问五次"为什么"，把每一层的答案都写出来，直到摸到不能再拆的基本事实。然后从基本事实往上重新推一遍：基于这个事实，你该怎么做？推出来的方案跟你原本要照搬的方案，差在哪？把两个方案并排写在纸上，差距一目了然。

第二件，给自己的生活做一次"类比盘点"。拿出一张纸，写下你这周做的十个决策，从大到小。在每个决策后面标注：这个决策是"照搬类比"还是"基于事实推演"？你大概率会发现，十个里有七八个是照搬。这不是坏事，是真相——知道自己在用类比，才有机会在大事上换成第一性原理。盘点的目的不是消灭类比（小事用类比省力是合理的），是让你在大事来临时有意识地切换工具。

第三件，找一件你长期"觉得不行"的事，验证它到底是事实还是假设。比如"我数学不行""我学不会编程""我做不了销售"。把这句话拆开：这个判断的依据是什么？是某次考试没考好（一个数据点），还是别人这么说过（一个观点）？把它当假设，找一个最小验证——花两小时学一个最基础的编程概念，看是不是真的学不会。大多数"我不行"，拆到底都是"我没试过"或"我试过一次没成"。一次没成是数据，不是判决。

三件事做完，你会感觉到一个变化：你做决策时，"别人怎么干"这个声音变弱了，"这事最底的事实是什么"这个声音变强了。这个声音的切换，就是第一性原理开始接管你思维的声音。下一章，我们拿起第三把刀——成长型思维，去改写你对失败的编码。
