# 考公全周期备考 · 写作提示词存档

> 本文件为专栏写作过程使用的提示词与执行方案存档，不进入专栏展示（下划线前缀文件被 check_book_structure.py 豁免、被 build_site.py 排除出站点）。保留用于复盘与迭代。

## 一、专栏定位

- 主题：2025 国考/省考全周期备考（成都+重庆地区重点）
- 读者：在职备考者（时间碎片化）、在校大学生/应届生（时间充裕但易陷学生思维）、屡考不中的公考老油条、零基础小白
- 核心目标：助力考生成功上岸国考省级公务员（成都+重庆地区）
- 核心精髓：「选择大于努力，思维大于刷题」——斩断做题家思维，植入机关思维、行政逻辑、博弈取舍
- 输出：50 篇正文 + 3 个辅助文件（_meta.yaml / _参考资料.md / _提示词.md）

## 二、执行流程

1. **建专栏骨架**：创建 _meta.yaml（category=技 / archetype=knowledge）+ _参考资料.md（权威资料清单+引用规范）+ _提示词.md（本文件）
2. **多专家团评估方案**：用 Task 工具启动 3 个 subagent（公考教研专家 / 机关笔杆子 / 在职在校差异化视角）并行评审 50 章大纲
3. **并行写作**：用 Task 工具启动多个 subagent 按 7 篇章分片并行写作（分批执行），每个 subagent 直接 Write 文件落盘
4. **严格质检**：跑 check_book_structure.py --strict（0 问题）+ 内容质检（六维度评分）
5. **批量修复**：根据质检报告修复 P0/P1/P2 问题与低分章节
6. **沉淀**：更新本文件记录执行流程与教训，必要时追加 loop_log

## 三、写作硬约束（写入每个写作 subagent 的 query）

1. 目录：`/workspace/output/考公全周期备考/`
2. 文件命名：`公考实战XX_章节名.md`（XX 两位数前导零 01-50，章节名 4-15 字中文）
3. frontmatter 八字段：`title` / `book` / `chapter` / `event` / `sort` / `chapter_sort` / `created_at` / `source_agents`
   - title 格式：`第XX章：主标题（点睛式，含核心张力）`
   - book 固定：`考公全周期备考`
   - chapter 格式：`公考实战XX`（XX 与文件名一致）
   - event：章节名（与文件名下划线后部分一致）
   - sort 与 chapter_sort 都用纯整数 1..50（**禁止前导零！YAML 八进制陷阱：`010`→8**）
   - created_at：`2026-07-03`
   - source_agents：`- exam-strategist`（或对应专家角色）
4. archetype: knowledge（category=技 → archetype=knowledge，自动映射）
5. **单章 4 段结构（必须严格对齐）**：
   - `## 概念`：包含「学情把脉与痛点」+「本质透视」
   - `## 原理`：包含「重难点突破与思维可视化」+「典型真题思维链拆解」
   - `## 实践`：包含「易错陷阱与避坑指南」+「最新考情与命题趋势」
   - `## 速查/自测`：包含「在职/在校分类行动指南」+ 速查卡 + 自测题 + 一句话小结
   - 用户 7 步法内容嵌入 4 段内，**禁止用 7 步法的标题作为 ## 二级标题**（避免命中灵魂维度扣分）
   - 4 段标题下用 `### 三级标题` 组织 7 步内容
6. 正文中文，单章 2500-3500 字
7. **思维链硬性要求**：典型真题拆解必须展示完整 CoT（审题→联想→破局→执行→回顾），严禁直接给答案；严禁"易知""显然""我们直接看出答案"等马后炮表述
8. **在职/在校差异化硬性要求**：每章 `## 速查/自测` 段必须包含「在职抓手」「在校抓手」「心态管理」三类行动指南
9. **反套路硬性要求**：申论和面试内容必须体现"反套路"与"政府视角"，严厉批判"万能模板"，强调"真情实感、具体问题具体分析"
10. **政治红线**：涉及政治理论、申论大作文、面试综合分析时，绝对政治正确，引用官方权威表述（党的二十大报告、政府工作报告、党章、人民日报、求是等）
11. 真题题号沉文末「## 参考来源」，正文行内出处 ≤3/千字
12. 2024 机考后真题标「2024 年·行测·XX 相关（考生回忆）」禁精确题号
13. **文末标题必须是「## 参考来源」**（check_sources_section 用 `##?\s*参考来源` 正则匹配，`## 参考` 不匹配会报「缺参考来源章节」）
14. 禁 AI 套话（综上所述 / 值得注意的是 / 我们可以看到 / 这告诉我们 / 不难发现 / 总而言之）
15. 删除跨章跳转（见上文 / 详见下章 / 见第 X 章 / 详见第 X 章）
16. 考情差异标注：国考与省考（副省级/市地级/行政执法类）题量或题型不一致处，用 `> ⚠️ 考情差异提示：...` 标注
17. 排版高亮：行测核心公式、申论核心找点词、面试核心框架用加粗或引用块（`>`）突出

## 七、政治红线写作护栏（写入每个 subagent 的 query）

1. 政治表述只引官方定论，不自创表述（人民日报/求是/新华社/党的二十大报告原文）
2. 敏感话题设"四不碰"红线（政策得失否定评价/历史问题自行定性/涉台港疆藏展开/群体事件对立渲染）
3. 批判套路必须配替代方案（禁止只破不立）
4. 申论大作文禁止三件套（开篇背领导讲话/分论点排比/结尾喊口号）
5. 面试答题禁止套路开头（"辩证地看"/"首先反思自己"/"领导信任我"）
6. 时政引用必须安全转述（转述官方表述+服务论点，禁自行评论政策得失）
7. 体制内内容严禁江湖化（禁站队/讨好/人情世故/灰色地带）
8. 情绪基调全程正向锚定（禁渲染焦虑，失败章以复盘成长收尾）

## 十一、关键教训（参考系统架构师备考专栏 BUG-031 沉淀）

1. **YAML 八进制陷阱**：frontmatter 的 `chapter_sort` 若写 `010` 会被 PyYAML 解析为八进制 = 8，导致站点章节顺序静默错乱。必须用纯整数 1..50。前导零只允许出现在文件名/章节字符串 `公考实战01`（字符串排序安全）。
2. **「参考来源」标题字面匹配**：check_sources_section 用 `##?\s*参考来源` 正则匹配，`## 参考` 不匹配会报「缺参考来源章节」并触发灵魂维度扣分。必须用 `## 参考来源`。
