# 摄影系统课 · 写作提示词与执行方案存档

> 本文件为专栏写作过程使用的提示词、执行方案与验收记录存档，不进入专栏展示（下划线前缀文件被 check_book_structure.py 豁免、被 build_site.py 排除出站点）。保留用于复盘与迭代。

## 一、专栏定位

- 主题：佳能 90D（APS-C 画幅/EF 与 EF-S 卡口）摄影系统课
- 读者：从新手小白到可收费水平的摄影学习者
- 核心目标：教会新手学会摄影，非新手向大神迈进，达到可收费水平；任何环境下知道如何引导人物；结合光影构图拍出好看的人像照片，同时也会拍风景
- archetype：modern（现代方法论桶）
- 输出目录：`/workspace/output/摄影系统课/`
- 输出：60 篇正文 + 4 个辅助文件（_meta.yaml / _器材参数表.md / _参考资料.md / _提示词.md）

## 二、执行流程（已落地）

1. **多专家团评估方案**：用 Task 工具启动 3 个 subagent（架构师/内容专家/规则）并行评审 14 模块 60 篇大纲，整合意见后定稿
2. **并行调研补全资料**：用 Task 工具启动 2 个 subagent 分别调研器材技术参数（30 个权威 URL）+ 内容资料（50 URL + 71 书籍引用），输出 `docs/reviews/photography-research-gear.md` 与 `photography-research-content.md`
3. **写资产文件**：创建 `_器材参数表.md`（参数基石）+ `_参考资料.md`（资料存档）+ `_meta.yaml`（14 chapters 映射）
4. **并行写作**：用 Task 工具启动 8 个 subagent 分两波并行写作（第一波 4 个写 30 篇基础模块 + 第二波 4 个写 30 篇应用模块）
5. **严格质检**：跑 `check_book_structure.py --strict`（结构 P0/P1/P2 全 0）+ `run_content_quality_checks(modern)`（60/60 全≥85）
6. **三方评审 + 修复**：3 个评审 subagent 五维度打分，修复 5 处参数硬伤 + 78 处内联跳转 + 6 处本地路径 + 15 处占位

## 三、写作硬约束（写入每个写作 subagent 的 query）

1. 目录：`/workspace/output/摄影系统课/`
2. 文件命名：两级路径 `{chapter}_{event}.md`，中文，下划线 `_` 分隔，禁止用 `-`，禁止子目录
3. 每篇必须有完整 frontmatter（title/book/chapter/event/sort/chapter_sort/created_at/source_agents 八字段）
4. archetype: modern（现代方法论桶）
5. 正文中文，每篇 2500-4000 字
6. 5 段结构固定段名：入戏 / 破题 / 方法论 / 避坑 / 践行
7. 文末必须有 `## 参考来源` 标题（不是"参考资料""参考文献"）
8. 禁内联跳转（"（见XX篇）""（详见下章）"等带括号跳转提示）；提及他篇用自然语句"（XX 篇展开）"
9. 禁用"底层逻辑""底层操作系统"
10. 术语词序铁律："英文+中文"词序（如"AV 模式"而非"光圈Av""ISO 感光度"而非"感光度ISO"）
11. 子标题"名词：洞察短语"
12. 每篇开头一句边界声明"本篇覆盖 X，不覆盖 Y（XX 篇展开）"
13. 文末加 `> 信息截止：2026-06`

## 七、关键纠正（调研发现）

1. **电池型号**：用户清单标"BP-A12"为错误，90D 实际电池为 **LP-E6N**（7.2V，1865mAh，约 79g，CIPA 1300 张）
2. **适马 30mm f/1.4 DC DN Contemporary 不可用**：该镜头仅 L-Mount/EF-M/RF/Fuji X/M43/Nikon Z/Sony E 七种无反卡口，90D 为 EF 卡口单反不可用。替代方案：适马老款 30mm f/1.4 DC HSM Art EF 卡口，或 Canon EF 35mm f/2 IS USM
3. **500 法则在 90D 上失效**：32.5MP APS-C 像素密度高，改用 400 法则或 NPF 法则
4. **90D 基准 ISO 100**：无 ISO 50 扩展档，扩展档只有 H:51200；不存在"ISO 160 比 ISO 100 更干净"的现象（该现象属其他机型误植）
