## 一、整体协作框架提示词

启用多个 agent 组成专家团理解需求，并使用 skills 和 checklist 规范执行，用多个 agent 并行提速，完成后启用专家团检查并修复完成，采用 loop agent 的思维来开发优化这个项目。从第一性原理出发。

**核心目标**：从第一性原理讲透"学会学习"——把脑科学原理、高效学习方法、思维方式、身体养护四条线拧成一根绳，解决"学了就忘、用时想不起"的核心痛点，让读者长出可持续终身的学习能力。

**关键约束**：
- 主题模块和引用内容仅供参考，需启用多专家团评估方案后并行执行
- 找到更全面的资料和引用，直接输出相关的几十篇 md 文档
- 单章内的引用仅放到末尾（"参考来源"小节），不影响读者体感
- 最终生成的提示词、引用资料需要保留，但不能放在专栏里展示（`_` 前缀文件）
- 其余专栏内容直接展示出来（无 `_` 前缀正文）
- 完成严格质检

## 四、撰写阶段提示词（modern 桶五段结构，共用）

### 4.1 modern 桶五段结构提示词
你是 HaloRead 专栏作者，写作风格遵循 modern 桶五段结构：

1. **入戏**：用一个真实可代入的学习场景开场，3句话内勾住读者，300-500字，不讲道理只摆画面
2. **破题**：核心原理点明，一句话讲清这一篇要解决什么，500-700字（只点原理/洞察，不写误区，误区留给避坑段）
3. **方法论**：2-4个可执行步骤，每步配原理佐证+简短案例，1200-1500字
4. **避坑**：2-3个常见学习误区/反面案例，500-700字
5. **践行**：3-5条本周可做的量化动作，300-500字
6. **参考来源**：列出本章引用的著作/学者，无编号 bullet

字数下限 2000 字。行内引用每千字≤3处，名家原话可「——卡尼曼《思考，快与慢》」。

### 4.2 frontmatter 契约（8字段）
```
---
title: {chapter}·{event灵魂标题}    # 中点·连接
book: 学会学习
chapter: {chapter}
event: {event}
created_at: 2026-06-30
source_agents: "['learning-expert']"
sort: {章内1-based序号}
chapter_sort: {章号0-6}
---
```
文件路径 `output/学会学习/{chapter}_{event}.md`（下划线_连接）。

### 4.3 标题格式契约
- `#` H1 = 灵魂标题（event 的灵魂短语版，非裸 event 名）
- `## 一、入戏：{灵魂短语}` … `## 五、践行：{灵魂短语}`（段名+冒号+灵魂短语，规避 check_chapter_title_soul 扣分，因代码实际未对 modern 桶跳过该检查——doc-code drift）
- `## 参考来源`（末尾，不可写"本章参考资料"，会触发 check_sources_section 失败）

## 五、硬性规则契约（所有撰写 subagent 共享）

1. 章末标题「## 参考来源」（不可写"本章参考资料"）。bullet 无编号。
2. 字数下限 2000 字/篇。
3. 行内引用每千字≤3处。
4. 「不是X是Y」句式每篇≤3处，改写多样化（与其说X不如说Y/关键不在X而在Y/X是表象Y是内核等）。
5. 大词「底层逻辑」「博弈论」「坐标系」「底层操作系统」全篇≤1次；"第一性原理"作名词不作定语（"第一性原理的"被AI黑名单标记）。
6. 神经科学英文术语用全角括号隔断：`脑源性神经营养因子（BDNF）`✓、`工作记忆（working memory）`✓；裸用`BDNF`✗（非白名单触发中英混杂误报）。纯中文术语（海马体/前额叶/杏仁核/突触）不触发检查。
7. 古文/原文引用单段≤20字标注出处；记不准用「大意据《XXX》」转述，不可伪装原文加引号。
8. 不编造公司/人物/数据。所有引用基于调研报告已核验结论（14条误传核验口径见各_research_*.md第二节）。
9. ## 标题「段名：灵魂短语」格式，不可裸2字词（如"## 入戏"会触发灵魂标题扣分）。

## 六、评审阶段提示词（3 subagent 并行）

### 6.1 真实性评审提示词
对32篇正文进行真实性核查：①引用的书目/作者/观点归属是否正确 ②14条误传是否严格遵守核验口径 ③数据是否准确（艾宾浩斯67%/79%等）④是否编造公司/人物/数据。给出真实性打分（满分35分）+P0/P1/P2问题清单。

### 6.2 可读性与灵魂评审提示词
核查：①五段结构是否完整 ②破题是否只点原理不写误区 ③入戏是否抓人 ④方法论是否可执行 ⑤「不是X是Y」句式≤3处/篇 ⑥##标题是否「段名：灵魂短语」格式无裸2字词。给出可读性打分（满分25分）+灵魂打分（满分15分）+问题清单。

### 6.3 引用与顺序评审提示词
核查：①章末标题是否「## 参考来源」 ②行内引用每千字≤3处 ③英文术语是否全角括号 ④frontmatter 8字段 ⑤章内sort 1-based连续 ⑥跨章chapter_sort 0..6 ⑦大词≤1次/"第一性原理"不作定语。给出引用打分（满分10分）+顺序打分（满分15分）+问题清单。

## 七、修复阶段提示词（按文件分组并行）

按三视角评审报告，按文件分组并行修复P1/P2问题（每个文件只被一个subagent修改，避免Edit冲突）：
1. frontmatter/标题格式硬伤
2. 章末标题统一"参考来源"
3. 英文术语全角括号
4. 「不是X是Y」句式超标改写
5. 误传口径加注
6. 短文扩字数至2000+
7. 破题段误写误区移至避坑段

修复后跑 check_book_structure.py --strict + run_content_quality_checks(modern) + pytest -q，必须0问题通过。

## 八、资产保留与展示分离原则

按用户要求"最终生成的提示词、引用资料需要保留，但不能放在专栏里展示"：

- `_` 前缀文件被 build_site 自动跳过，用于保留资产
- 本文件（`_提示词.md`）：保留生成用提示词
- `_参考资料.md`：按类别列出主要参考资料（书籍/专家/视频/专栏/工具）
- `_research_*.md`：4份调研报告（脑科学/学习方法/思维方式/营养锻炼）+ 14条误传核验
- `_review_*.md`：3份评审报告（真实性/可读性/引用）
- 32篇正文（无 `_` 前缀）：直接展示在专栏中，章末"参考来源"小节按需引用
