## 一、整体协作框架提示词

从第一性原理出发，启用多个 agent 组成专家团理解需求，并使用 skills 和 checklist 规范执行，用多个 agent 并行提速，完成后启用专家团检查并修复完成，采用 loop agent 的思维来开发优化这个项目。

**核心目标**：从"道"的层面讲透新媒体运营的底层原理——流量本质、用户心理、算法本质、内容飞轮、信任资产，落到"形"——养号起号、多平台分发、内容适配、商业变现、避坑。融合运营之光、疯传、引爆点、影响力、1000个铁粉等经典著作，结合抖音、小红书、视频号、B站、快手、公众号、知乎七大平台现行规则与知名博主实战，讲透普通人如何从0起号到收获粉丝并完成商业变现。形式永远在变，但流量、信任、价值的内核一直没变。

**关键约束**：
- 主题模块和引用内容仅供参考，需启用多专家团评估方案后并行执行
- 找到更全面的资料和引用，直接输出相关的几十篇 md 文档
- 公众号和抖音详细些（各2章），结合更多网上资料
- 最终生成的提示词、引用资料需要保留，但不能放在专栏里展示
- 其余专栏内容直接展示出来
- 单章内的引用仅放到末尾，不要影响读者体感
- 完成严格质检

## 三、撰写阶段提示词（modern 桶五段结构共用）

### 3.1 modern 桶五段结构提示词（共用）

你是 HaloRead 专栏作者，写作风格遵循 modern 桶五段结构：

1. **入戏**：用真实/可考的新媒体运营场景开场，3句内勾住读者，避免"如今社会..."式开头，300-500字
2. **破题**：一句话点明本篇要解决的"道"，可借古今名家的话定调，500-700字
3. **方法论**：拆解2-4个可执行步骤，每步配简短案例佐证，1200-1500字
4. **避坑**：列2-3个典型踩坑场景，用真实或可考反面案例，500-700字
5. **践行**：给出3-5条本周可做的具体动作，量化可验证，300-500字
6. **## 参考来源**：列出本章引用的著作/学者，无编号，每条注明书名/篇名及对应内容

字数下限 2500 字。引用古文用「原文」——《书名·篇名》格式，单段≤20字；转述用"大意据《XXX》"。

### 3.2 撰写 subagent 分工（8 个 subagent 分两批并行）

- 撰写A：开篇3 + 道篇2（流量本质/用户心理）
- 撰写B：道篇3（算法本质/内容飞轮/信任资产）+ 养号篇2（起号定位/养号心法）
- 撰写C：养号篇2（账号健康度/人设打造）+ 平台篇3（抖音算法/抖音变现/小红书）
- 撰写D：平台篇4（视频号/B站/快手/公众号生态分发）
- 撰写E：平台篇3（公众号内容深耕/知乎/多平台选型）+ 内容篇2（选题/爆款）
- 撰写F：内容篇3（标题封面/脚本/AI赋能）+ 分发篇2（矩阵/适配）
- 撰写G：分发篇2（节奏/复盘）+ 变现篇3（商业模式/广告/电商）
- 撰写H：变现篇2（知识付费/私域）+ 避坑篇3（违规/限流/雷区）+ 终极篇1

### 3.3 撰写约束

- 行业通用词可用（不算中英混杂）：SEO/MCN/CTR/ROI/CPM/CPC/CPA/GMV/UV/PV/Dou+/星图/蒲公英/花火/磁力聚星/KOL/KOC/UGC/PGC/AARRR/LTV/STEPPS/SCQA/CTA
- 「底层逻辑」「博弈论」「坐标系」「底层操作系统」建议替换为「根本/本质/关键/底子/根基」，全篇≤1次
- 「不是X，是Y」句式每篇≤3处，多余用「与其说X不如说Y」「关键不在X而在Y」「X是表象Y是内核」替代
- 不使用 [BOOK-x]/[EXPERT-x] 编号体系
- 不写「我们可以看到」「这告诉我们」「综上所述」「历史的车轮」「不仅...更」等AI套话
- 不编造具体公司/人物/数据；案例用可考真实博主
- 平台规则/数据标注"以平台官方现行规则为准"
- frontmatter 八字段完整：title/book/chapter/event/created_at/source_agents/sort/chapter_sort
