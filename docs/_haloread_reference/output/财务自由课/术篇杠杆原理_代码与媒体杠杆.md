## 入戏

有个程序员在 GitHub 上开源了一个小工具，处理 Excel 表格用的，自己周末写的。三年后他偶然发现，这个仓库每月还在被下载十几万次，有人替他提 bug、有人替他写文档、有人基于它做了衍生产品。他没花一分钱推广，没雇一个人，这段代码自己长成了一个生命体。纳瓦尔把这叫"代码杠杆"——一次写，无数次跑，复制免费。

## 破题

代码和媒体，是纳瓦尔四类杠杆里最年轻的两种，也是他最推崇的两种。原因就两条：不需要许可、边际成本接近零。

"不需要许可"意味着你不需要说服老板、说服投资人、说服任何看门人。一台电脑、一根网线，你就可以开始。"边际成本为零"意味着你写一本书、录一段视频、写一段代码，第二份和第一百万份的成本几乎一样。这在人类历史上是头一次。在工业时代，你卖一百万辆车就要造一百万辆；在互联网时代，你写一段代码，一百万个人用，你不需要再造九十九万九千九百九十九份。

## 方法论

这两类杠杆的共同本质，是"复制即免费"。理解这一点，就能理解为什么它们是普通人的红利。

**代码杠杆**有两种典型形态。一种是产品型——一个软件、一个网站、一个 App，写完之后用户自己来用，你睡觉时它也在工作。另一种是工具型——一段脚本、一个开源库、一个插件，解决某一类具体问题，被集成到无数其他人的工作流里。前者直接变现，后者建立影响力与信任，间接变现。

**媒体杠杆**的形态更丰富：书、文章、播客、视频、课程、社交媒体内容。它们的共同点是一次创作、长期留存、持续被发现。纳瓦尔自己就是媒体杠杆的活样本——他的播客和推特风暴被反复传播，让一个不主动做营销的投资人，成了全球最知名的财富思想家之一。

代码与媒体杠杆还有一个被低估的特点：它们是累积资产。雇错一个员工是当期亏损，写废一篇文章几乎不亏——它会被搜索引擎慢慢收录，可能在三年后某个时刻被一个人看到，然后改变他的命运，也改变你的回报曲线。这种"延迟兑现"的特性，让这两类杠杆特别适合愿意长期坚持的人。

纳瓦尔在播客里反复说一句话：代码和媒体是"睡后收入"的来源——你睡着了，你的代码还在跑，你的内容还在被推荐。这两类杠杆让普通人第一次拥有了"不靠出卖时间"的可能性。

## 避坑

第一个坑，是以为"发了就有人看"。代码和媒体杠杆的好处是分发门槛低，坏处也是分发门槛低——所有人都能发，所以所有人都在发。生产容易，被看见难。只生产不迭代、不研究分发，你的作品会被淹没在噪音里。媒体杠杆的另一半其实是注意力与关系问题，这部分会在后续单独展开。

第二个坑，是只做不沉淀。今天写一篇文章、明天录一个视频，彼此没有关系，等于在做一次性内容。真正的杠杆资产是有体系的——围绕一个主题持续输出，让读者因为信任你而订阅你，让搜索引擎因为相关性而持续推荐你。零散创作产生流量，体系创作产生资产。

第三个坑，是过早追求变现。粉丝一千就接广告、文章刚有起色就开付费，会破坏信任。代码与媒体杠杆的红利期在第二、第三年，前期是积累影响力、积累搜索权重、积累信任。太早变现，等于杀鸡取卵。

## 践行

如果你不会写代码，从媒体杠杆开始更友好。选一个你能持续输出的领域——你的职业、你的爱好、你的钻研——定一个最低频率，比如每周一篇，先写一年。不要追求爆款，追求持续。一年后你会发现，最值钱的不是某篇文章，是这一年累积下来的"有人在写这个主题"的信号。

如果你会写代码，挑一个你自己日常遇到的小痛点，写一个工具解决它，然后开源。不要一开始就想做平台、做产品，先做一个有人愿意用的小工具。开源是建立技术信誉最快的方式，也是被招聘、被合作、被投资的机会入口。

无论哪种，记住一点：作品是你的资产，平台不是。你在一个平台上积累的粉丝随时可能因为规则变化而归零，但你的内容、你的技能、你的口碑是跟着你走的。早期可以借平台流量，中期一定要建立自己的渠道——邮件列表、自有网站、独立社区。

最后，给自己一个三年期。代码和媒体杠杆都是慢起快涨，前一年几乎看不到回报，第二年有零星反馈，第三年开始指数级加速。多数人放弃在第一年。能熬过的人，才配享受"复制即免费"的红利。
