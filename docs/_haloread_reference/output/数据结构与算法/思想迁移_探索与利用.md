# 算法课·35｜探索 vs 利用：从多臂老虎机到生活选择

> 前置知识：本专栏前面讲过的算法都建在"信息完整"假设上——排序前数据已知、二分前数组已排序。本章处理完全相反的设定：信息只能边做边学。

## 一、问题与思路：老餐厅还是新餐厅，这其实是个算法问题

### 1. 一个生活场景：周五晚饭的两难

周五晚上下班，你站在公司楼下。左边是你去过 50 次的那家川菜馆，知道招牌水煮鱼稳定 8 分好吃；右边是上周新开的泰国菜，要么惊艳、要么踩雷。

去川菜馆，叫**利用（exploitation）**——用你已知的最好选择，稳拿 8 分。去泰国菜，叫**探索（exploration）**——放弃眼前的 8 分，赌一个未知分布，可能拿到 10 分、也可能只有 3 分。直觉告诉你："一直去老餐厅太亏，一直试新餐厅太冒险。"——这就是探索-利用困境（exploration-exploitation dilemma）的核心张力。它不是一个"该吃哪家"的小问题，是所有"在不完全信息下做序列决策"问题都要回答的根本困境：**当信息只能靠尝试获得时，你永远在"用已知的最优"和"赌一个未知"之间分配资源**。

### 2. 这个困境为什么无解

很多人第一反应："试一次新餐厅不就知道了？"——这话有两个隐藏陷阱。

**陷阱一：探索本身有成本。** 你去泰国菜那一次，如果踩雷，损失了一顿饭的体验。这顿饭回不来了——你不能"先去泰国菜再回到周五那一刻重新选川菜"。在序列决策里，每一次尝试都消耗一次"机会"，机会不可逆。

**陷阱二：探索的结论也不完全确定。** 泰国菜吃了一次觉得一般，但你不知道是"这家店真的不行"还是"今天厨师状态差"。要确认"这家店平均几分"，你得去好几次——每次去都是用掉一次"本来可以去川菜馆"的机会。

把这两个陷阱叠加，困境就浮现了：**全利用，永远卡在已知的次优；全探索，每次都赌，长期平均收益惨淡**。任何比例的混合都不可能让"短期收益"和"长期信息收益"同时最大化——这是个根本性的张力，不是"找个聪明策略"就能消解的。后面会讲，信息论给出了悔恨下界 $O(\log T)$，任何策略都做不到更好——困境是数学上的，不是工程上的。

**悔恨下界为什么是 $O(\log T)$？** 直觉这样想：要确认"臂 A 比臂 B 好"，至少要把两个臂各拉一定次数才能让样本均值的置信区间不重叠——这个"最低必要拉杆数"是 $\log T$ 量级的（和置信区间收缩速度相关）。如果你拉得比这少，就无法区分最优臂和次优臂，必然拉错；如果你拉得比这多，就是在已经能区分的臂上浪费拉杆。所以悔恨的下界由"区分两个臂所需的最少观测数"决定，这个数随 $T$ 对数增长。任何策略都无法突破这个下界——这是信息论给出的硬约束，不是工程实现的不完美。

**一个反直觉推论**：悔恨 $O(\log T)$ 意味着"平均悔恨"（Regret/T）随 $T$ 趋近于 0。换句话说，**只要策略对，长期看几乎不亏**——前 1000 次可能亏不少，但拉到 100 万次时，平均下来每次只亏一点点。这条结论给了工程上的底气：bandit 策略在长期跑的实验里值得信任，前提是策略对、假设没被违反（后面反例段会讲哪些违反会让"长期不亏"承诺作废）。

### 3. 历史地位：从二战统计到 RLHF

探索-利用困境最早被严肃讨论，是在二战期间的统计序贯分析里（Abraham Wald, 1945）。真正把它做成"教科书问题"的是 Herbert Robbins 在 1952 年提出的**多臂老虎机（multi-armed bandit, MAB）**模型——名字来自赌场老虎机，每台机器的回报分布不同，你只能靠拉杆来学，目标是在有限次拉杆内最大化总收益。

之后 70 年，这个问题从赌场扩展到了所有"在线学习"场景：

- 1990 年代：临床试验——把病人分到几个候选疗法，边治边学哪个最有效，少死一个人是少一个。
- 2000 年代：在线广告——千次曝光分给几个候选广告，边投边学点击率最高的那个，少浪费一次曝光。
- 2010 年代：推荐系统——Netflix、字节跳动都在用 bandit 算法给新内容分配"探索流量"。
- 2020 年代：大模型 RLHF——给同一个 prompt 的多个回答打分时，也是在探索（试新回答风格）和利用（用已知高分模板）之间做权衡。

**一个反直觉的史实**：本章后面会讲 Thompson 采样（1933 年提出）其实是历史最久的 bandit 算法，比 UCB1（2002 年）早 70 年。但 Thompson 采样在提出后的 70 年里几乎被遗忘——为什么？因为它是贝叶斯方法，每次拉杆前要从后验采样，1933 年没有计算机，靠手算 Beta 分布采样根本不现实。直到 2010 年前后，Google、Microsoft 这些公司在广告系统里大规模验证了 Thompson 的工程优势（[Scott 2010]），它才从冷宫里被翻出来，成为工业界最常用的 bandit 策略之一。这段史实提示一个工程经验：**算法的兴衰不只看理论优雅，还看实现成本——计算能力跟上时，被遗忘的好算法会复活**。这也是为什么今天看 RLHF、扩散模型这些"老想法新做法"，不要轻易说"这个不是早就有人提过吗"——能落地才是真·新。

可以说，**只要场景满足"信息只能通过行动获得、行动有成本、目标是在有限步内最大化累积收益"，就是探索-利用问题**。这个抽象让它在 70 年里没过时——从赌场到 RLHF，问题形态一直在变，困境结构没变。本章作为思想迁移篇的起手章，正是要让你看到：掌握了这个抽象，就能在看似无关的场景里看到同一个困境。

### 4. 怎么识别一个问题是不是探索-利用问题

迁移章的价值不在"教会你写 ε-greedy"，在"让你能把陌生问题映射到这个框架"。识别一个问题是探索-利用问题，看四个特征——四个都满足才是，缺一个就要换框架。

**特征一：信息只能通过行动获得。** 你不能"先观察别人的拉杆再决定自己拉哪个"——别人的拉杆不告诉你"如果你自己拉会怎样"（不同人的回报分布不同）。临床试药的疗效只能通过给当前病人用药才能观测，广告点击率只能通过曝光才能观测，新餐厅合不合口味只能通过自己去吃才能知道。

**特征二：行动有不可逆成本。** 每次拉杆消耗一次"机会窗口"，机会不能复用。一次广告曝光给了 A 创意，这次曝光就没法再给 B——损失的潜在收益是悔恨。

**特征三：目标是有限窗口内的累积收益。** 不是"最终学会最优臂"，是"在 T 次拉杆内拿到的总回报最大"。这两者不一样——前者可以无限探索把所有臂学透，后者必须把每次拉杆都"既学又赚"。

**特征四：回报有随机性，但分布平稳。** 每次拉杆的回报是分布里的一个样本，不是确定值。但分布本身是固定的（不随时间漂移）。第 4 个特征是隐含假设——后面反例段会讲违反它会发生什么。

**反例：哪些问题看起来像 bandit 但不是。**

- **下棋**：行动后能完全预测对方的可能回应（信息虽然不完全但可以通过搜索推演），不满足特征一——你不需要"实际下一步棋"才知道某步的优劣，可以靠搜索估计。下棋是搜索问题，不是 bandit。
- **股票短期交易**：回报分布非平稳（市场结构会变），违反特征四。强行套 bandit 会用历史数据估未来分布，严重滞后——这就是为什么 2008 年金融危机里很多量化策略崩盘。
- **一次性决策**（比如选学校、选配偶）：行动次数太少（一两步），不满足特征三的"序列"。一次性决策更适合用贝叶斯决策框架，bandit 的"分配探索预算"思路不直接适用。
- **完整信息博弈**（如国际象棋的残局求解）：不存在"信息只能通过行动获得"——所有信息都在棋盘上。这是搜索问题。

记住这四个特征 + 四个反例，你就能在面对一个新场景时快速判断"是不是 bandit"——是的话用 bandit 框架，不是的话换框架。这种"先识别问题类型再选算法"的习惯，比"背一堆算法模板"更值钱。

## 二、原理与实现：三种策略的演化

### 1. 把问题说清楚：多臂老虎机的形式化

多臂老虎机（MAB）的标准设定：有 $K$ 个选项（"臂"），每个臂 $i$ 有一个未知的回报分布 $D_i$，期望是 $\mu_i$。你拉一次臂 $i$，得到一个样本 $r \sim D_i$，目标是在 $T$ 次拉杆内最大化累积回报 $\sum_t r_t$。

衡量策略好坏用**悔恨（regret）**：你实际拿到的回报，和"如果你一开始就知道哪个臂最好、每次都拉它"的差距。

$$\text{Regret}(T) = T \cdot \mu^* - \mathbb{E}\left[\sum_{t=1}^T r_t\right]$$

其中 $\mu^* = \max_i \mu_i$ 是最优臂的期望。**好策略的目标是让 Regret 增长尽量慢**——理想是 $O(\log T)$，意味着每多拉一次，平均悔恨趋近于 0。如果 Regret 是 $O(T)$（线性增长），说明策略没在学，永远在原地踏步。

这条"线性 vs 对数"的分界线是评价所有 bandit 策略的标尺。下面三种策略就是按"如何逼近 $O(\log T)$"演化的——从最朴素的固定比例混合，到基于不确定度的精准探索，再到基于后验采样的概率匹配。

### 2. ε-greedy：最朴素的混合

策略极简：以概率 $\epsilon$ 随机探索（随便挑一个臂拉），以概率 $1-\epsilon$ 利用（拉当前样本均值最高的臂）。

```python
import random

def epsilon_greedy(arms, T, eps):
    # arms: 各臂真实均值（隐含分布），T: 总轮数
    K = len(arms)
    counts = [0] * K          # 每个臂被拉过的次数
    rewards = [0.0] * K       # 每个臂的累计回报
    total = 0
    for t in range(T):
        if random.random() < eps or t < K:   # 探索（前 K 轮强制各拉一次）
            a = random.randint(0, K - 1)
        else:                                # 利用
            avg = [rewards[i]/counts[i] for i in range(K)]
            a = max(range(K), key=lambda i: avg[i])
        r = 1 if random.random() < arms[a] else 0   # 伯努利臂
        counts[a] += 1
        rewards[a] += r
        total += r
    return total
```

机制三句话：①前 $K$ 轮强制每个臂拉一次（冷启动，避免没观测过就谈均值）②之后每次以 $\epsilon$ 概率随机选一个臂（保证持续探索）③其余时候拉当前样本均值最大的臂（利用已知信息更新最优估计）。

**它的悔恨是 $O(\log T)$**，但常数项不漂亮——因为 $\epsilon$ 是固定的，即使某个臂已经明显是最优的，策略仍然会以 $\epsilon$ 概率去拉那些明显差的臂，这部分"无谓探索"的悔恨是 $\epsilon \cdot T \cdot (\mu^* - \bar{\mu}_{\text{others}})$，长不出来。这就是它的根本短板：探索是"盲目"的，不区分"哪个臂更值得探索"。

**ε-greedy 的第一个改良：衰减 $\epsilon$。** 既然固定 $\epsilon$ 会让"无谓探索"线性增长，自然想到让 $\epsilon$ 随时间衰减——早期 $\epsilon$ 大（多探索），后期 $\epsilon$ 小（多利用）。典型衰减方案 $\epsilon_t = \min(1, cK/t)$，其中 $c$ 是常数。这个改良能把悔恨常数项压下来，但工程上有个新问题：衰减速度怎么定？衰减太快，没探索够就转利用，可能锁定错的臂；衰减太慢，等于没改良。本质上衰减 $\epsilon$ 是用"经验法则"代替"自适应"，没有 UCB 那种"基于不确定度自动调整"的优雅。这条改良路线走到头，自然就演化出 UCB——与其手动调 $\epsilon$ 衰减，不如让不确定度本身来决定探索预算。

### 3. UCB1：让"不确定"的臂优先

ε-greedy 的短板正是 UCB（Upper Confidence Bound，置信区间上界）的破局点。思路是：**给每个臂一个"乐观估计"——样本均值加上一个"不确定度奖励"，挑这个值最大的拉**。

```python
import math

def ucb1(arms, T):
    K = len(arms)
    counts = [0] * K
    rewards = [0.0] * K
    total = 0
    for t in range(T):
        if t < K:                      # 前 K 轮各拉一次
            a = t
        else:
            # 每个臂的 UCB 值 = 样本均值 + √(2 ln t / n_i)
            ucb = [rewards[i]/counts[i] +
                   math.sqrt(2 * math.log(t) / counts[i])
                   for i in range(K)]
            a = max(range(K), key=lambda i: ucb[i])
        r = 1 if random.random() < arms[a] else 0
        counts[a] += 1
        rewards[a] += r
        total += r
    return total
```

公式里那项 $\sqrt{2 \ln t / n_i}$ 是关键——它衡量"对这个臂的估计有多不确定"。$n_i$ 越小（拉得越少），不确定度越大，奖励越高。这给了"还没充分探索的臂"一个补偿分，把探索预算精准投到"信息最稀疏"的地方，而不是像 ε-greedy 那样均匀撒到所有臂上。

**UCB1 的悔恨是严格的 $O(\log T)$**，且常数比 ε-greedy 好。它还有一个工程上极重要的性质——**确定性**：给定相同历史，它选的臂是确定的（不像 ε-greedy 要掷骰子）。这让实验可复现、A/B 测试可追踪、调试可重放——这些在生产系统里都是硬要求。

### 4. Thompson 采样：用贝叶斯做"概率匹配"

Thompson 采样（Thompson Sampling, TS）走的是完全不同的路子：**给每个臂的回报分布一个先验，每次从后验里采样一个值，挑采样值最大的臂拉**。

```python
def thompson_sampling(arms, T):
    # 假设伯努利臂，用 Beta(α, β) 共轭先验
    K = len(arms)
    alpha = [1] * K      # 成功次数 + 1
    beta  = [1] * K      # 失败次数 + 1
    total = 0
    for t in range(T):
        samples = [random.betavariate(alpha[i], beta[i]) for i in range(K)]
        a = max(range(K), key=lambda i: samples[i])
        r = 1 if random.random() < arms[a] else 0
        if r == 1:
            alpha[a] += 1
        else:
            beta[a] += 1
        total += r
    return total
```

机制三句话：①每个臂维护一个 Beta 分布（伯努利臂的共轭先验），$\alpha$ 记成功次数，$\beta$ 记失败次数②每次从每个臂的 Beta 分布各采一个样本，挑最大的拉③根据结果更新对应臂的 $\alpha$ 或 $\beta$。

直觉解释：Beta 分布的"宽度"代表不确定度。一个臂只拉过两次（一次成一次败），它的 Beta(2,2) 是个宽峰，采出来的值有相当概率很高——这会让它有机会被选中再探索。一个臂拉过 1000 次、成功率稳定 0.7，它的 Beta(701,301) 是个很窄的峰在 0.7 附近——采出来的值稳定在 0.7，胜负主要看真值。这就是"概率匹配"——按"这个臂是最优臂的后验概率"成比例地分配流量。

**Thompson 采样的悔恨也是 $O(\log T)$**，但工程上有两个 UCB 没有的优势：①对回报分布的形状更鲁棒（不只看均值，整个分布都用上了）②稀疏奖励场景下收敛更快（UCB 在小样本时 $\sqrt{2\ln t/n_i}$ 项会过大，导致过度探索）。Google 2012 年的一篇大规模实验论文（[Scott 2010]）显示，在广告选型的真实流量上，Thompson 比 UCB 平均多拿 10-30% 收益——这种差距在生产里直接是钱。

### 5. 三种策略对比：什么场景选哪个

| 策略 | 悔恨 | 探索方式 | 是否确定性 | 适用场景 |
|---|---|---|---|---|
| ε-greedy | $O(\log T)$（常数差） | 均匀随机 | 否 | 工程起步、调试简单 |
| UCB1 | $O(\log T)$ | 按不确定度 | 是 | 可复现实验、A/B 测试 |
| Thompson | $O(\log T)$ | 按后验采样 | 否 | 稀疏奖励、分布敏感 |

为什么三者悔恨都是 $O(\log T)$ 还要演化？因为**渐近界只看无穷远，工程看的是 $T$ 在 100-10000 这个范围谁先收敛**。Thompson 在稀疏奖励（比如广告点击率 0.1%）场景下能比 UCB 早 5-10 倍轮数找到最优臂——这种差距在生产里直接是营收。理解三者的差别不在"理论界"，在"工程常数"——这也是为什么本章作为迁移章不深挖证明，把篇幅留给反例和迁移。

### 6. 走一遍例子：三种策略在同一个问题上的行为差异

设一个 3 臂 bandit，真实均值 $\mu = [0.3, 0.5, 0.7]$，臂 3 最优。跑 $T=1000$ 轮，看三种策略怎么分配拉杆次数。

**ε-greedy（$\epsilon=0.1$）的典型行为**：

- 前 3 轮各拉一次，估计分别是 0/1, 1/1, 1/1（伯努利样本，结果受随机性影响）
- 第 4-50 轮：因为臂 3 第一次拉成功了，初期它会利用臂 3，但每次仍有 10% 概率随机拉其他臂
- 第 50-1000 轮：臂 3 的样本均值逐渐稳定在 0.7 附近，策略主要拉臂 3
- **最终分布**：臂 1 ≈ 100 次、臂 2 ≈ 100 次、臂 3 ≈ 800 次。100 次臂 1、100 次臂 2 几乎都是 $\epsilon$ 触发的"无谓探索"——明明知道它们差，还是拉了

**UCB1 的典型行为**：

- 前 3 轮各拉一次
- 第 4 轮：每个臂 $n_i=1$，UCB 值 = 样本均值 + $\sqrt{2\ln 3/1} \approx$ 样本均值 + 1.48。这个不确定度项极大，胜负主要由"哪个臂还没充分试"决定
- 第 4-30 轮：三个臂轮流拉（因为不确定度项主导），但每次拉了之后该臂的不确定度下降，下次轮到别人
- 第 30-200 轮：臂 3 的样本均值开始稳定在 0.7 附近，即使不确定度项下降，总 UCB 仍最高，逐渐主导
- **最终分布**：臂 1 ≈ 30 次、臂 2 ≈ 50 次、臂 3 ≈ 920 次。探索预算精准投到"还需要确认的臂"上，几乎没有无谓探索

**Thompson 采样的典型行为**：

- 前 3 轮各拉一次（Beta(1,1) 采样相当于均匀分布，第一次拉哪个臂随机性大）
- 第 4-50 轮：三个臂的 Beta 分布都很宽，胜负主要由"哪个臂的后验采样值最高"决定——这个采样过程自然把流量按"该臂是最优臂的后验概率"分配
- 第 50-1000 轮：臂 3 的 Beta(α, β) 逐渐变成窄峰在 0.7 附近，臂 1、臂 2 的 Beta 也在 0.3、0.5 附近成窄峰。每次采样，臂 3 的采样值大概率最高，但偶尔臂 2 采样到 0.6+ 也能赢一次
- **最终分布**：臂 1 ≈ 20 次、臂 2 ≈ 60 次、臂 3 ≈ 920 次。和 UCB 接近，但稀疏奖励场景下收敛更快

**对比观察**：

| 指标 | ε-greedy | UCB1 | Thompson |
|---|---|---|---|
| 臂 3 拉杆次数（1000 轮） | ~800 | ~920 | ~920 |
| "无谓探索"次数 | ~200 | ~30 | ~30 |
| 1000 轮总收益（期望） | ~660 | ~680 | ~680 |
| 100 轮时是否找到最优臂 | 不一定 | 较可能 | 最可能 |

最后一行是关键——**短期（100 轮）Thompson 比 UCB 更早收敛到最优臂**，这是它在工程上更受欢迎的核心原因。UCB 在小样本时的不确定度项 $\sqrt{2\ln t/n_i}$ 容易过大，导致过度探索；Thompson 的后验采样在样本少时确实给"未充分试的臂"流量，但流量分配按后验概率走，比 UCB 的"硬性不确定度奖励"更平滑。

## 三、实践与面试：A/B 测试、反例与边界

### 1. A/B 测试的本质：两臂 bandit 的"纯探索"特例

很多人以为 A/B 测试和 bandit 是两个东西。其实 A/B 测试是 bandit 的退化版——**只两个臂、纯探索期、固定比例分配**。

标准 A/B 测试流程：①流量五五分到 A、B 两组②跑固定时间（比如 14 天）③统计显著性检验（p < 0.05）④赢家通吃，全量切到胜组。

把它对应到 bandit 框架：①两臂（A、B）②探索期固定 14 天、固定 50/50 分配③用统计检验判定"哪个臂更好"④探索结束后 100% 利用胜者。

**A/B 测试的代价**：14 天内，五五分流量意味着"如果 B 明显更好，你前 14 天少赚了一半的潜在收益"。这是"纯探索期"的悔恨——和固定 $\epsilon$ 的 ε-greedy 一样，固定探索比例不能自适应。

**bandit 相对 A/B 测试的优势**：边探索边利用。早期流量仍五五分，但一旦数据显示 B 更好，bandit 会自动把流量倾斜给 B——一边继续探索确认，一边把大部分流量给当前最优。这就是为什么 Google、Netflix 大量用 bandit 替代固定 A/B——同样的实验周期，总收益高 10-30%。

**反过来 A/B 测试相对 bandit 的优势**：①统计显著性检验能给出"我们有多确信 B 比 A 好"的明确概率，bandit 的输出是"流量的倾斜比例"，没有"显著"这个概念②A/B 测试的事后分析（用对照组数据估计因果效应）更标准、更可解释③监管严格的领域（医疗、金融）必须用 A/B 测试，bandit 的"动态调整"在合规上更难解释。

**总结一句**：A/B 测试适合"需要统计严谨性、能接受探索期成本"的场景；bandit 适合"动态调整可接受、追求总收益最大化"的场景。两者不是替代，是互补——很多公司在"探索早期用 A/B 找方向、确认方向后用 bandit 持续优化"的混合模式。

### 2. 反例与边界：三种策略各自会怎么翻车

迁移章的重点不是"教你怎么用"，是"教你怎么识别什么时候不能用"。下面是三种策略各自的反例——遇到这些边界条件，再好的策略也得让位。

**反例一：ε-greedy 在非平稳分布上完全失效。**

ε-greedy 的"样本均值"是过去所有观测的平均。但很多真实场景的回报分布是**非平稳的**——比如推荐系统的用户兴趣会漂移，今天爱看的明天不爱看。ε-greedy 把三年前的点击和今天的点击一视所见，估计严重滞后。

边界：只要回报分布会随时间漂移，固定 $\epsilon$ 的 ε-greedy 就不能用。变种方案是"滑动窗口 ε-greedy"或"指数衰减 ε-greedy"——只看最近 $W$ 次或对历史观测指数衰减加权。生产里几乎所有推荐场景都违反"平稳"假设，纯 ε-greedy 直接出局。

**反例二：UCB 在稀疏奖励 + 大量臂时严重过探索。**

UCB 的 $\sqrt{2\ln t / n_i}$ 项在 $n_i$ 很小时极大。如果场景是"1000 个臂、每个臂点击率 0.1%"（典型推荐系统长尾），UCB 会让每个臂至少被拉几十次才收敛——总流量根本不够分。

边界：臂数量大、单臂回报稀疏时，UCB 的"逐臂探索"成本爆炸。变种方案是"线性 bandit"或"上下文 bandit"——不再独立估计每个臂，而是把臂的特征向量化，让相似的臂共享观测。Thompson 在这个场景也明显优于 UCB，因为后验采样天然给"还没充分试过的臂"按概率分配流量，而不是强制每个都试够次数。

**反例三：Thompson 采样在先验选错时收敛极慢。**

Thompson 的"贝叶斯"前提是先验合理。如果你给一个伯努利臂配了 Beta(1,1) 先验，但真实点击率是 0.01%，前几千轮 Thompson 会反复"乐观"地探索这个臂——因为 Beta(1,1) 在 0.999 附近采样概率不低。这种"先验错配"在工程里很常见——你以为的"无信息先验"其实带了强烈偏见。

边界：先验分布的选择对 Thompson 影响极大。如果对回报量级没把握，要么用更保守的先验（比如 Beta(0.5, 0.5)），要么先用一小段数据估先验、再切到 Thompson。这条边界是 Thompson 相对 UCB 的代价——UCB 是"无参的"，不依赖先验假设。

**反例四：所有 bandit 策略在"延迟反馈"下都失效。**

bandit 的理论假设是"拉一次立刻看到回报"。但很多场景反馈延迟——信用卡转化要等用户月底还款才知道、留存率要等 7 天后才知道、订阅续费要等一年。

边界：反馈延迟超过"决策窗口"的 10 倍以上，所有标准 bandit 策略都不适用。变种方案是"延迟反馈 bandit"——把延迟建模进更新规则，但收敛速度大幅下降，工程上常常不得不回退到固定 A/B 测试。这是为什么广告投放（秒级反馈）大量用 bandit，而订阅产品（年级反馈）很少用——延迟差距决定策略选型。

**反例五：违反"独立同分布"假设的场景。**

bandit 假设每个臂的回报分布是固定的、独立的。但很多场景不满足——①臂之间相关（推荐电影 A 喜欢的人大概率也喜欢 B，独立估计浪费信息）②回报有上下文依赖（同一个广告对不同用户点击率不同，不能忽略用户特征）③回报有位置偏差（排在前面的总比排在后面的点击率高，但不是因为更好）。

边界：违反 IID 时，标准 bandit 退化为"上下文 bandit"（contextual bandit）——把用户、位置等上下文作为特征，估计 $\mu_i(\text{context})$ 而不是 $\mu_i$。这是工业推荐系统的主流形态，但已经超出本章范围。本章作为迁移章只画边界——真要落地，去看 contextual bandit 和 LinUCB 的专门材料。

**反例六：用具体数字看 UCB 在稀疏奖励下怎么过探索。**

把"反例二"用具体数字算一遍，看 UCB 到底有多糟。设 100 个臂、每个臂真实均值 0.001（0.1% 点击率），$T=10^6$ 次总流量。

UCB1 的逻辑：每个臂至少要拉到 $n_i$ 满足 $\sqrt{2\ln T / n_i} \le \mu^* - \mu_i \approx 0.001$（即不确定度项小于臂间差距）才"基本确认"。解出 $n_i \ge 2\ln T / 0.001^2 \approx 2 \times 13.8 / 10^{-6} \approx 2.76 \times 10^7$。

这意味着 UCB1 想要"基本确认"每个臂的均值，要把每个臂拉 2700 万次——但总流量才 100 万次！换句话说，UCB1 在这个场景下根本走不到"利用"阶段，永远在"探索"——100 万次流量被均匀分给 100 个臂，每个臂拉 1 万次，连"基本确认"的零头都不够。

Thompson 在这个场景下行为完全不同——后验采样自然按"该臂是最优臂的概率"分配流量。即使所有臂的后验都很宽，采样值之间的差距也能让流量倾斜——100 万次流量可能 90% 给了某个臂。这就是为什么稀疏奖励 + 大量臂场景下 Thompson 几乎总是赢 UCB——UCB 的"硬性不确定度项"在小样本时太大，Thompson 的"软性后验采样"自适应。

这个具体数字也提示了第二条工程经验：**选策略不只看渐近界，要看在你的场景参数下（臂数、稀疏度、T 量级）谁的实际行为合理**。理论界都是 $O(\log T)$，工程常数差几个数量级。

### 3. 教科书做法 vs 生产做法

| 场景 | 教科书做法 | 生产做法 | 原因 |
|---|---|---|---|
| 在线广告创意选 | ε-greedy | 上下文 bandit + Thompson | 用户特征强相关，需建模 |
| 推荐新内容冷启动 | UCB1 | 多目标 bandit + 探索预算 | 单纯 CTR 优化会推标题党 |
| 临床试验疗法对比 | A/B 测试 | 自适应随机化试验 | 合规要求 + 伦理约束 |
| 大模型 RLHF 偏好学习 | ε-greedy | KL-正则化 bandit | 防止策略漂移过远 |

**生产里几乎不用纯 ε-greedy / UCB / Thompson**——它们是"教科书基线"，真实系统都在基线上加约束：探索预算上限、多目标加权、安全带（不许推明显违规的臂）、反馈延迟处理。理解纯策略是为了知道生产变体在改什么——而不是真去写一个 ε-greedy 上线。这一节的"教科书 vs 生产"对照，就是为了让你看到从"原理正确"到"工程可用"中间还隔着多少层。

**纯 ε-greedy 在哪种生产场景里反而合适？** 不是没有。两种场景：①小流量实验（流量 < 1 万次/天）—— Thompson 和 UCB 的渐近优势在小样本下不明显，ε-greedy 代码简单、可调试性强，反而更合适②多目标难定义时——如果你说不清"回报"具体是什么（既要 CTR 又要 GMV 又要留存），Thompson 的"分布建模"反而难落，ε-greedy 的"利用当前最优"可以直接套业务规则。换句话说，**策略选型不只看算法性能，还看业务能不能给出干净的回报定义**——能给就用 Thompson，给不出就退回 ε-greedy 加业务规则。

**从标准 bandit 到 contextual bandit 的演进预告。** 本章讲的 ε-greedy / UCB / Thompson 都是"无上下文"的——每个臂的回报分布对所有用户一样。但工业推荐系统的真实形态是"上下文 bandit"——同一个广告对不同用户点击率差很多，必须把用户特征加进模型。从标准 bandit 到 contextual bandit 的关键跳跃是：把"估计每个臂的标量均值 $\mu_i$"升级为"估计每个臂的条件均值 $\mu_i(x)$"，其中 $x$ 是用户/位置/时间等上下文特征。具体算法有 LinUCB（线性模型 + UCB 思路）、Thompson + 线性模型、Neural Bandit（用神经网络学 $\mu_i(x)$）。这是工业推荐系统的主流形态，但已经超出本章作为迁移章的范围——本章只画边界，告诉你"再往下走是另一章"。

### 速查表：三种策略的关键参数

| 策略 | 核心参数 | 悔恨界 | 关键假设 | 失效边界 |
|---|---|---|---|---|
| ε-greedy | $\epsilon \in [0,1]$ | $O(\log T)$ | 平稳分布 | 非平稳分布、稀疏奖励 |
| UCB1 | 探索系数 $c=\sqrt{2}$ | $O(\log T)$ | 平稳、IID | 大量臂、稀疏奖励 |
| Thompson | 先验分布 | $O(\log T)$ | 先验合理 | 先验错配、延迟反馈 |

**A/B 测试 vs Bandit 速查**：

| 维度 | A/B 测试 | Bandit |
|---|---|---|
| 臂数 | 通常 2 | 任意 K |
| 探索期 | 固定 | 自适应 |
| 流量分配 | 固定 50/50 | 动态倾斜 |
| 显著性 | 有 p-value | 无 |
| 总收益 | 低（探索期浪费） | 高 |
| 合规可解释 | 强 | 弱 |
| 适用 | 医疗、金融、监管场景 | 广告、推荐、内容分发 |

**Bandit 算法族谱速查**（按演化脉络排列，本章覆盖前三个家族）：

| 家族 | 代表算法 | 探索思路 | 提出年代 | 适用场景 |
|---|---|---|---|---|
| 随机探索 | ε-greedy、Softmax | 概率随机 | 1970s | 工程起步、可调试 |
| 不确定度优先 | UCB1、KL-UCB、MOSS | 置信区间上界 | 2002 | 可复现、平稳分布 |
| 后验采样 | Thompson Sampling | 贝叶斯后验采样 | 1933（复兴于 2010） | 稀疏奖励、分布敏感 |
| 上下文 bandit | LinUCB、Neural Bandit | 把用户特征加进模型 | 2010 | 工业推荐系统主流 |
| 强化学习 | RLHF、PPO + 探索奖励 | 状态 + 动作 + 长期回报 | 2015+ | 大模型对齐、游戏 AI |

族谱的演化方向是"从简单到复杂"——每个新家族都解决前一个家族的某个根本短板。但**演化不等于替代**——简单家族在合适场景里仍然好用，族谱是为了让你知道"我现在的场景该用哪个家族"，不是"用最新的一定最好"。

### 自测三问

**问题一：** 为什么"全利用"和"全探索"都不是好策略？请用悔恨（regret）的视角解释这个困境为什么是根本性的。

**参考答案：** 全利用意味着永远拉当前样本均值最高的臂——但你最初的样本均值是噪声，可能拉错臂一辈子都不回头。这种策略的悔恨是 $O(T)$（线性增长），因为它不学。全探索意味着每次随机拉一个臂——学到了每个臂的分布，但学到的信息从不用，悔恨也是 $O(T)$。困境之所以根本，是因为"信息获取"和"信息使用"在每一步都要竞争同一次机会——同一次拉杆不能既用来探索（学到分布）又用来利用（拿到最高回报）。任何策略都在"用这次拉杆拿短期收益"和"用这次拉杆换长期信息"之间分配资源，最优分配对应 $O(\log T)$ 悔恨——这是信息论给出的下界，没有策略能更好。

**问题二：** UCB1 公式里那项 $\sqrt{2 \ln t / n_i}$ 在解决什么问题？ε-greedy 缺这个项，会怎么翻车？

**参考答案：** 那项是"不确定度奖励"——$n_i$ 越小（臂被拉得越少），不确定度越大，奖励越高。它解决的是"探索应该投到哪"的精准化问题：与其像 ε-greedy 那样均匀随机撒探索预算，不如把探索预算优先给"还没充分试过的臂"。ε-greedy 缺这个项的后果是：即使某个臂已经被拉过 10000 次、明显是最优的，ε-greedy 仍然会以 $\epsilon$ 概率去拉那些只拉过两次、明显很差的臂——这是"无谓探索"，悔恨常数项大。UCB1 通过"不确定度奖励"自动把探索预算集中到真正需要更多信息的臂上，常数项更小、收敛更快。两者渐近界都是 $O(\log T)$，但工程常数差好几个数量级。

**问题三：** 假设你要给一个内容平台设计"新作者冷启动"机制——新作者发布的前 10 篇内容该有多少曝光？请用 bandit 框架分析，并指出至少两个生产里会破坏纯 bandit 假设的因素。

**参考答案：** 用 bandit 框架看：每个新作者是一个"臂"，前 10 篇内容的曝光就是"探索预算"。如果探索预算给太少（比如每篇只给 100 曝光），样本太少、估计噪声大，可能误杀有潜力的作者；如果给太多（每篇给 10 万曝光），挤压了成熟作者的流量，整体收益下降。最优策略类似 UCB 或 Thompson——给每个新作者一个"不确定度奖励"的初始曝光，根据早期反馈动态调整后续曝光。生产里破坏纯 bandit 假设的因素至少有四个：①回报延迟——一篇文章可能几天后才被搜索流量带火，反馈不在当次曝光窗口内②上下文依赖——同一篇文章对不同用户群点击率差很多，不能忽略用户特征③非平稳——算法改版、热点事件会让历史数据失效④多目标——纯 CTR 优化会推标题党，需要把"质量分""举报率"等指标加进回报函数。这些因素让生产里的"新作者冷启动"从纯 bandit 退化为"上下文 bandit + 多目标 + 延迟反馈"的复杂变种，但底层的"探索-利用权衡"框架仍然有效。

### 算法思想 × 生活迁移

这一章的核心思想是「**信息有成本，决策有窗口——在两者之间分配资源是所有学习型决策的根本问题**」。这是迁移章要重点消化的一段——下面三个迁移比算法本身更值得带走。

**迁移一：职业选择的探索期 vs 利用期。** 职业生涯早期是高探索期——你不知道自己适合什么，应该多试不同行业、不同岗位、不同公司，每次"试"都是一次拉杆，损失的是时间但换来的是对自己的信息更新。职业生涯中后期是高利用期——你已经知道自己擅长什么，应该深耕一个领域、复利积累。**反例**：很多人 35 岁还在"全探索"——每年换行业，没有复利；很多人 25 岁就"全利用"——毕业进一家公司再不挪窝，从未探索过自己的可能性。两者都是 $O(T)$ 悔恨的人生版本。最优策略类似 UCB——年轻时给"不确定度奖励"高，多探索；随年龄增长、对自己的认知不确定度下降，逐步转向利用。这个框架也解释了为什么"30 岁焦虑"是个算法问题——它是探索期结束、利用期开始的切换点，自然伴随身份认同的重新校准。

**迁移二：读书的"老书 vs 新书"分配。** 你有 10 本爱不释手、读过多次的老书（利用），和书架上 100 本没翻开过的新书（探索）。全读老书，舒适但没新信息；全读新书，信息量大但踩雷率高。bandit 框架给出的策略：①大部分时间读老书（保底收益、稳定吸收）②按"不确定度"挑要读的新书——封面、目录、推荐语都让你"不确定它好不好"的新书优先读③读到一本好的新书，把它升级成"老书"反复读。这套策略比"随机抓一本新书读"或"只读老书"都高效。它也解释了为什么"主题阅读"比"散漫阅读"收益高——主题阅读是把"探索"集中在相邻的臂上，相似臂的观测可以互相强化，单位探索预算换的信息更多。

**迁移三：交友的"老朋友 vs 新朋友"。** 老朋友是高利用——关系深、互动成本低、情感稳定；新朋友是高探索——可能带来新机会、新视角，也可能浪费时间。bandit 框架同样适用：①不要 100% 时间给老朋友（信息源单一、视野窄化）②不要 100% 时间给新朋友（关系浅、情感支持弱）③按"不确定度"分配——某些"看起来有意思但你不太了解"的人优先投入时间，确认了就升级为老朋友。这个框架解释了为什么"工作后交新朋友越来越难"——不是没时间，是探索成本（情感投入、信任建立）随年龄上升，自然倾向利用。要打破这个倾向，得主动给自己设"探索预算"——比如每周强制见一个不熟的人。

**迁移四：技术学习的"深耕 vs 拓荒"。** 你已经精通 Python（利用），但要不要花时间学 Rust（探索）？按 bandit 框架分析：①深耕 Python 边际收益递减——从 80 分到 90 分要花的精力，远大于从 0 分到 60 分学一门新语言②拓荒 Rust 不一定有用——可能学了两年也没项目能落地，回报分布方差大③但拓荒的"信息价值"无法替代——不学 Rust 你永远不知道"内存安全"对自己代码质量的影响有多大，这种认知升级只能通过实际写一遍才能获得。最优策略仍是 UCB 式——给"自己不太了解但看起来潜力大"的技术分配探索时间，根据早期反馈调整后续投入。**反例**：追新技术博客每周换一个框架的"全探索"是 $O(T)$ 悔恨——没复利；只守着第一份工作用的技术栈十年不挪的"全利用"也是 $O(T)$ 悔恨——没视野。技术学习的关键是"对探索预算设置上限"——比如 20% 时间留给拓荒，80% 时间深耕，既保证复利又留出视野。

**迁移五：投资组合的"老仓位 vs 新机会"。** 你已经重仓的股票是高利用——你研究透、有信心、仓位重；新机会是高探索——可能涨更多、也可能踩雷。bandit 框架的提示：①全仓老股票等于"全利用"——错过新机会，长期跑输市场②全仓新机会等于"全探索"——风险高、信息少，长期波动大③最优策略是"核心 + 卫星"——80% 仓位给已研究透的核心标的（利用），20% 给新机会（探索），新机会确认后升级为核心。这个框架其实和"巴菲特能力圈"一脉相承——能力圈内是利用，能力圈边缘是探索，能力圈外是赌博。**反例**：违反平稳分布假设的情况——股市不是平稳 bandit，市场结构会变。直接套 bandit 会用历史数据估未来，严重滞后。这就是为什么量化策略要频繁 retraining——本质是给 bandit 加"非平稳处理"。

**为什么这些迁移成立：** 探索-利用困境的成立前提是"①信息只能通过行动获得 ②行动有成本 ③目标是在有限窗口内最大化累积收益"。这三个前提在职业选择、读书、交友、技术学习、投资里都成立——你只能通过"做"来知道自己适不适合、喜不喜欢、值不值得；每次尝试都消耗时间和精力；生命/职业/精力的窗口是有限的。迁移成立的关键是"信息只能边做边学"——如果你能不付出行动就获得信息（比如读传记能替代亲身经历），困境就消解了。这就是为什么"听别人讲人生道理"无法替代"自己经历一遍"——前者是免费的，但只能给你结论，给不了你自己分布的形状。

**迁移的边界：什么时候不该套 bandit 框架。** 不是所有生活选择都能套 bandit。最明显的反例是"重大一次性决策"——结婚、生子、转行这种"几乎不可逆且机会窗口极少"的选择。bandit 的"探索预算"思路假设你可以多次尝试，但结婚不是"试几次找最优"。这类决策更适合用"贝叶斯决策 + 后悔最小化"框架——尽量在决策前用最低成本获取信息（试婚、实习、轮岗），把不可逆决策的"信息缺口"在决策窗口关闭前尽量补上。强行套 bandit 会让你对"不可逆决策"轻率对待——以为还有下次探索，其实没有。这是迁移章必须画的一条边界：**框架是脚手架，不是万能解药——碰到框架的边界要主动放弃框架，回到具体问题**。

这也是本章作为思想迁移篇起手章要传递的最深一层——**算法不是只用来排序和查找的，它是一套用来理解"如何在不确定下做决策"的语言**。掌握这套语言，你能看到职业选择、读书、交友、技术学习、投资里的同一个困境结构；碰到框架的边界，你能主动承认并切换到合适的工具。这种"看到结构 + 知道边界"的能力，比任何具体的算法模板都值钱。

**本章在迁移篇中的定位**。本章是思想迁移篇的起手章，定位是"教你怎么用算法框架看生活问题"。后续迁移章会沿用本章建立的"识别问题类型 → 套框架 → 知道边界"三步法，处理不同的算法思想向生活的迁移——比如分治思想怎么用到团队管理、贪心思想怎么用到时间管理、动态规划怎么用到长期决策。每一章迁移章都不会比本章更"工程"，只会更"思想"——本章的代码和反例已经够多，后续章会更聚焦"思想如何映射"。

读完本章你应该带走三样东西：①一个具体的困境结构（探索-利用）②一套识别问题的特征清单（四特征 + 四反例）③一种"框架有边界"的自觉。前两样让你能"套"框架，第三样让你能"放"框架——三者合一才是迁移章的真目的。
