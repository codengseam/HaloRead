## 一、问题与思路：贪心的直觉与"看起来对"的陷阱

### 1. 一个生活场景：超市收银台找零钱

你帮妈妈在超市买完东西，收银员要找给你 6 元零钱。手边有 1 元、3 元、4 元三种硬币。收银员想用最少的硬币找给你。

**做法一（贪心）**：每次拿最大的、不超过剩余金额的硬币。

- 第一枚：拿 4 元，剩 2 元
- 第二枚：拿 1 元，剩 1 元
- 第三枚：拿 1 元，剩 0 元
- 共 3 枚：4+1+1

**做法二（穷举）**：试试看，3 元 + 3 元 = 6 元，只需 2 枚！

收银员的"贪心"做法多拿了一枚。这就是贪心算法最经典的陷阱——**每一步看起来都是当前最好的选择，最后却不是全局最优**。问题不在"贪心"这个思路本身，问题在于"找零钱"这个问题上，"每步拿最大"这个贪心策略不成立。

但有些问题贪心就是成立的。比如美式硬币 `[25, 10, 5, 1]`，找零 30 分：贪心拿 25+5 = 2 枚，确实最优。为什么美式硬币贪心成立、`[1,3,4]` 不成立？这就是本章要回答的核心问题：**贪心什么时候成立？什么时候不成立？怎么判断？**

### 2. 贪心的直觉与本质

贪心算法的直觉用一句话说清：**每一步都选"当前看起来最好"的，期望这样能得到全局最优解。**

它的关键词是"不回头"——做完一个选择就锁定，绝不撤销、绝不重新考虑。这和动态规划（DP）形成鲜明对比：DP 会把所有子问题都算一遍，最后比较谁最优；贪心只走一条路，赌它就是最优。

为什么人类直觉上喜欢贪心？因为它快。DP 要存所有子问题的解，时间和空间都贵；贪心一杆子插到底，通常 $O(n \log n)$ 甚至 $O(n)$ 就能跑完。所以工程上有一句话：**能用贪心就别用 DP**。

但"能用贪心"这四个字背后有严格的数学门槛。本章的核心就是：**贪心成立的充要条件，是问题具备两个性质——贪心选择性质 + 最优子结构**。这两个性质后面会用严格定义讲清楚，并配反例说明"缺了任一个会怎样"。

### 3. 历史地位：从 Kruskal 到拟阵

贪心思想最早被严肃地用在图论。1956 年，Joseph Kruskal 在 *Proceedings of the AMS* 发表最小生成树算法——按边权从小到大排序，能加就加（不形成环）。**这是贪心思想最早的纯粹体现**：每步选"当前最小且不冲突"的边，最后得到全图最小生成树。

1959 年，Dijkstra 提出最短路径算法——每步选"当前距离最近的未确定点"松弛。同样是贪心：每步锁定一个最近点，永不回头。

1952 年，David Huffman 提出 Huffman 编码——每步合并两个频率最低的节点。还是贪心：每步选"最便宜的两个"合并。

这三组经典算法证明贪心在图论和编码领域威力巨大。但贪心的"成立判据"长期依赖具体问题逐一证明。直到 1971 年，Jack Edmonds 把"什么样的优化问题能用贪心"这个问题抽象成**拟阵（Matroid）**——一组抽象公理（独立集、遗传性、交换性），证明"问题对应一个拟阵"是贪心成立的充要条件。这是贪心算法从"经验技巧"升级为"数学理论"的转折点。

## 二、原理与实现：从代码到成立判据

### 1. 经典成立：活动选择问题（逐行讲透）

**问题**：有 $n$ 个活动，每个活动有开始时间 $s_i$ 和结束时间 $f_i$。一个会议室同一时间只能进行一个活动，求最多能安排多少个不冲突的活动。

**贪心策略**：按结束时间从小到大排序，每步选"结束最早且不与已选活动冲突"的活动。

```python
def activity_selection(activities):
    # activities: [(s1, f1), (s2, f2), ...]
    if not activities:
        return []
    activities.sort(key=lambda x: x[1])   # 按结束时间排序
    selected = [activities[0]]            # 第一个必选（结束最早）
    last_end = activities[0][1]
    for s, f in activities[1:]:
        if s >= last_end:                 # 与上一个不冲突
            selected.append((s, f))
            last_end = f                  # 更新最后结束时间
    return selected
```

**逐行讲透**（左神风格，每一行都不能含糊）：

- **`activities.sort(key=lambda x: x[1])`**：按结束时间排序。这是关键。为什么不按开始时间排？为什么不按长度排？后面的反例会证明这两种都失败。按结束时间排能让每一步"占用会议室的时间最少"，给后面的活动留最多的空间。
- **`selected = [activities[0]]`**：第一个活动（结束最早的）必选。为什么不试试不选它？因为后面的"贪心选择性质"会证明：存在一个最优解，它一定包含结束最早的活动。
- **`last_end = activities[0][1]`**：记录最后一个被选活动的结束时间。后面判断冲突时只需要这一个变量。
- **`for s, f in activities[1:]`**：从第二个活动开始遍历。每个活动只看一次，所以是 $O(n)$。
- **`if s >= last_end`**：当前活动开始时间 ≥ 上一个活动结束时间 → 不冲突。注意是 `>=` 不是 `>`——上一个活动 5 点结束、这个 5 点开始，算不冲突。
- **`selected.append((s, f))`**：选入。锁定，不回头。
- **`last_end = f`**：更新。注意只更新被选活动的 f，没选的活动不影响 `last_end`。

**复杂度**：排序 $O(n \log n)$ + 遍历 $O(n)$ = $O(n \log n)$。空间 $O(1)$（不计输出）。

**为什么这个贪心成立？** 后面"成立判据"一节会用反证法严格证明。这里先给个直觉：每个活动的"成本"是占用会议室一段时间，"收益"是排进了一个活动。按结束时间排序意味着——你每次选的是"成本最小的活动"（占用时间段最早结束），自然能给后面留最多空间。

### 2. 另一个经典：跳跃游戏（逐行讲透）

**问题**（LeetCode 55/45）：数组 `nums`，每个元素表示从该位置最多能跳几步。从位置 0 出发，求最少跳几次到末尾。

**贪心策略**：维护"当前这一跳能到达的最远位置 `cur_end`"和"下一跳能到达的最远位置 `farthest`"。每走到 `cur_end` 时，必须再跳一次。

```python
def jump(nums):
    n = len(nums)
    if n <= 1:
        return 0
    jumps = 0               # 跳了几次
    cur_end = 0             # 当前这一跳能到达的最远右边界
    farthest = 0            # 遍历过程中，下一跳能到的最远位置
    for i in range(n - 1):  # 注意不遍历最后一个位置
        farthest = max(farthest, i + nums[i])
        if i == cur_end:    # 走到当前跳的边界，必须再跳
            jumps += 1
            cur_end = farthest
            if cur_end >= n - 1:
                break
    return jumps
```

**逐行讲透**：

- **`jumps = 0`**：跳跃次数计数。
- **`cur_end = 0`**：当前这一跳最远能到哪。初始在位置 0，还没跳。
- **`farthest = 0`**：在遍历 `[0, cur_end]` 这一段时，记录每一点能跳到的最远位置。这就是"贪心"——在当前这一跳覆盖的范围里，找下一跳能到最远的位置。
- **`for i in range(n - 1)`**：遍历到倒数第二个位置。为什么不遍历最后一个？因为到了最后一个就不用再跳了。
- **`farthest = max(farthest, i + nums[i])`**：从位置 i 能跳到 `i + nums[i]`，取最远。这是贪心的核心：在当前覆盖范围内，不断更新"下一跳最远能到哪"。
- **`if i == cur_end`**：走到当前跳的边界了，必须做决定——再跳一次。
- **`jumps += 1`**：跳一次。
- **`cur_end = farthest`**：新的边界变成"刚才记录的下一跳最远位置"。注意这一步锁定了下一跳的范围。
- **`if cur_end >= n - 1: break`**：已经能到末尾了，提前退出。

**复杂度**：$O(n)$ 一次遍历，$O(1)$ 空间。比 BFS 或 DP 的 $O(n^2)$ 都快。

**为什么这个贪心成立？** 直觉：每一跳覆盖一段区间，下一段的起点和终点只取决于"上一段里能跳到的最远位置"。不存在"少跳一段更优"的情况——因为少跳一段意味着在中间停下来，而中间停不比直接跳到最远更优（你只是把决策推迟了一次，不会因此减少总跳数）。

### 3. 贪心成立的两个判据（严格定义）

这是本章的核心。**贪心成立的充要条件是问题同时满足下面两个性质**：

**判据一：贪心选择性质（Greedy Choice Property）**

> 若一个问题的整体最优解可以通过一系列局部最优选择得到，则称该问题具有贪心选择性质。

严格表述：设 $S$ 是原问题，$x$ 是贪心策略第一步选出的元素。则存在 $S$ 的一个最优解 $A^*$，使得 $x \in A^*$。

**判据二：最优子结构（Optimal Substructure）**

> 若一个问题的最优解包含其子问题的最优解，则称该问题具有最优子结构性质。

严格表述：设 $A^*$ 是 $S$ 的最优解，$x$ 是 $A^*$ 中贪心选出的元素。则 $A^* \setminus \{x\}$ 是子问题 $S' = S \setminus \{x\}$（即去掉 $x$ 后剩下的部分）的最优解。

**这两个判据缺一不可**。后面两个反例分别说明"缺了哪个判据会怎样"。

**活动选择为什么满足这两个判据？**

- 贪心选择性质：设活动按结束时间排好序，$a_1$ 是结束最早的。设 $A^*$ 是某个最优解，$a_k$ 是 $A^*$ 中结束最早的活动。若 $a_1 \ne a_k$，则 $f_1 \le f_k$（$a_1$ 结束最早），可以把 $a_k$ 换成 $a_1$，得到的 $A^* \setminus \{a_k\} \cup \{a_1\}$ 仍是可行解且活动数相同——所以存在包含 $a_1$ 的最优解。
- 最优子结构：选了 $a_1$ 后，剩下的子问题是"在所有 $s_i \ge f_1$ 的活动里选最多不冲突的"。$A^* \setminus \{a_1\}$ 必是这个子问题的最优解——否则能换成更优的，加回 $a_1$ 得到比 $A^*$ 更优的解，矛盾。

### 4. 反例一：0-1 背包贪心不成立 vs 分数背包成立

**0-1 背包问题**：背包容量 $W=50$。三件物品：

| 物品 | 重量 | 价值 | 单位价值 |
|---|---|---|---|
| A | 10 | 60 | 6 |
| B | 20 | 100 | 5 |
| C | 30 | 120 | 4 |

**贪心策略**：按单位价值（价值/重量）从大到小选。

- 选 A（密度 6），剩 40
- 选 B（密度 5），剩 20
- 选 C？C 重 30，装不下，跳过
- 总价值 = 60 + 100 = **160**

**最优解**：选 B + C，总价值 = 100 + 120 = **220**。

贪心选了 A 反而错过了 B+C 这个更优组合。**0-1 背包的贪心选择性质不成立**——你不能保证"单位价值最高的物品一定在某个最优解里"。因为物品不可分割，选了高密度小件 A 就锁死了 10 的容量，错过了更值钱的大件组合。

**分数背包问题**：同样的物品和容量，但物品可以切割（装一部分）。

**同样的贪心策略**：按密度从大到小，装满为止。

- 选 A（密度 6），装 10，剩 40
- 选 B（密度 5），装 20，剩 20
- 选 C（密度 4），装 20/30 = 2/3 件，正好填满
- 总价值 = 60 + 100 + 120×(2/3) = **240**

**这就是分数背包的最优解。** 因为物品可切割，"密度高的先装"不会"锁死容量"——多余的部分可以用低密度物品填。分数背包满足贪心选择性质。

**0-1 vs 分数的关键区别**：

| | 0-1 背包 | 分数背包 |
|---|---|---|
| 物品 | 不可分割 | 可切割 |
| 贪心成立 | **不成立** | 成立 |
| 通用解法 | DP，$O(nW)$ | 贪心，$O(n \log n)$ |
| 失败原因 | 选小件锁死容量，错过最优组合 | 可切割，不存在"锁死" |

这个对比是贪心章节最重要的反例——**"几乎一样的问题"贪心可能成立也可能不成立，差别只在"能不能切割"这个微小细节**。这也说明判断贪心是否成立必须严格证明，不能"看起来一样就用"。

### 5. 反例二：找零钱问题——硬币面额决定贪心是否成立

**问题**：用最少硬币凑出金额 $amount$，硬币面额给定为 `coins`。

**贪心策略**：每次拿不超过剩余金额的最大面额。

**情况一：`coins = [25, 10, 5, 1]`，amount = 30**

- 拿 25，剩 5
- 拿 5，剩 0
- 共 2 枚，确实最优。**贪心成立。**

**情况二：`coins = [1, 3, 4]`，amount = 6**

- 拿 4，剩 2
- 拿 1，剩 1
- 拿 1，剩 0
- 共 3 枚（4+1+1）
- 但最优是 3+3 = 2 枚。**贪心不成立。**

为什么美式硬币成立、`[1,3,4]` 不成立？因为美式硬币面额之间有"整除关系"（25 = 5×5，10 = 2×5），每次拿大的不会"卡住"后面；而 `[1,3,4]` 中 4 不能被 3 整除，拿 4 之后剩下 2 只能用 1 凑，错过了用 3+3 的可能。

**判据诊断**：

- 美式硬币：贪心选择性质成立——可以证明"最大面额硬币一定在某个最优解里"。
- `[1,3,4]`：贪心选择性质**不成立**——拿最大面额 4 反而偏离了最优解（3+3）。

**这种"硬币面额决定贪心是否成立"的现象**告诉我们：贪心的成立与否，不取决于"算法长什么样"，而取决于"问题的数据是否满足某种数学性质"。通用找零问题只能用 DP（$O(n \cdot amount)$），特殊面额才能用贪心。

### 6. 高手向：拟阵直觉——贪心成立的"数学骨架"

前面两个判据是"必要条件层面"的描述。**真正的充要条件是 1971 年 Jack Edmonds 提出的拟阵理论**。这一节给拟阵直觉，不讲全部证明。

**拟阵的定义**：拟阵是一个序对 $M = (E, I)$，其中 $E$ 是有限集合（如所有边），$I$ 是 $E$ 的子集族（称为独立集），满足三条公理：

1. **非空性**：空集是独立集，$\emptyset \in I$。
2. **遗传性**：若 $A \in I$ 且 $B \subseteq A$，则 $B \in I$。即独立集的子集也是独立集。
3. **交换性**：若 $A, B \in I$ 且 $|A| < |B|$，则存在 $e \in B \setminus A$，使得 $A \cup \{e\} \in I$。即较小的独立集总能"扩展"成较大的独立集。

**关键定理（Edmonds 1971）**：设 $M = (E, I)$ 是一个拟阵，给 $E$ 中每个元素赋一个权重 $w(e)$。则"按权重从小到大，能加就加（不破坏独立性）"这个贪心策略，能得到权重和最小（或最大）的独立集。**而且，反过来也对：如果一个加权问题能用贪心求解，那么它必然对应一个拟阵。**

**把拟阵套到 Kruskal 上**（最清晰的例子）：

- $E$ = 图的所有边
- $I$ = "不形成环的边集"（即森林）
- 非空性：空集是森林 ✓
- 遗传性：森林的子集还是森林 ✓
- 交换性：两个森林 $A$（小）、$B$（大），$|A| < |B|$，$B$ 中必有边 $e$ 加到 $A$ 不形成环（这是图论的一个经典引理）✓

所以 Kruskal 是拟阵上的贪心，**必然得到最小生成树**。这就是 Kruskal 正确性的"骨架"。

**拟阵的威力**：它把"贪心成立"从"每个问题单独证明"提升到"检查三条公理"。但注意——拟阵只覆盖一部分贪心问题（Kruskal、任务调度、最小生成树变种等）。Huffman 编码、Dijkstra、活动选择虽然贪心也成立，但它们不直接对应拟阵（结构更复杂），需要单独证明。**拟阵是贪心成立的充分骨架——满足拟阵一定能贪心，能贪心不一定对应拟阵**。

**对高手的启发**：拟阵是"问题归类"思维的极致——把"哪些问题能用贪心"这个问题归约到一组抽象公理。这种思维在算法设计中是通用的：识别问题的代数结构（拟阵、格、半环），用结构性质保证算法正确性。现代图算法、机器学习中的子模函数优化，都是这种"结构驱动算法设计"思路的延伸。

### 7. 贪心 vs DP 的边界

| | 贪心 | 动态规划（DP） |
|---|---|---|
| 决策方式 | 不回头，一次锁定 | 穷举所有子问题，比较择优 |
| 时间复杂度 | 通常 $O(n \log n)$ 或 $O(n)$ | $O(n^2)$、$O(nW)$ 等 |
| 空间复杂度 | $O(1)$ 或 $O(n)$ 输出 | $O(n)$ 或 $O(n^2)$ 状态表 |
| 成立条件 | 贪心选择性质 + 最优子结构 | 最优子结构 + 重叠子问题 |
| 正确性 | 必须严格证明，否则可能错 | 天然正确（穷举） |
| 风险 | 用错就是错答案 | 只是慢，不会错 |

**判断口诀**：

1. **先想能不能贪心**：如果直觉上"每步选最大的/最小的/最早的/最晚的"就能解，先尝试贪心。
2. **再验证贪心是否成立**：用反证法——假设存在一个不包含贪心选择的最优解，能否构造出矛盾。能矛盾则成立。
3. **证不出来就用 DP**：贪心证不出来意味着可能不成立，直接上 DP，至少保证正确。
4. **能用贪心就别用 DP**：贪心通常快一个数量级。生产里"贪心 + 对数器"比"DP"更受欢迎——只要对数器在大量随机数据上对比通过，工程上就能接受。

**典型对应**：

- 0-1 背包 → DP（贪心不成立）
- 分数背包 → 贪心（成立）
- 找零（通用面额）→ DP（贪心不成立）
- 找零（美式硬币）→ 贪心（成立）
- 活动选择 → 贪心（成立）
- 最长递增子序列 → DP（贪心不成立，需要二分优化版）
- 区间调度 → 贪心（按结束时间排，成立）

## 三、实践与面试：手撕模板、对数器、面试题

### 1. 面试手撕模板（活动选择 + 跳跃游戏）

面试中贪心题的高频模板就是"按某种维度排序后一杆子插到底"。两个模板如下：

```python
def activity_selection(activities):
    """活动选择 - 按结束时间贪心。"""
    if not activities:
        return []
    activities.sort(key=lambda x: x[1])    # 关键：按结束时间排
    selected = [activities[0]]
    last_end = activities[0][1]
    for s, f in activities[1:]:
        if s >= last_end:
            selected.append((s, f))
            last_end = f
    return selected

def jump(nums):
    """跳跃游戏 II - 最少跳跃次数。"""
    n = len(nums)
    if n <= 1:
        return 0
    jumps = cur_end = farthest = 0
    for i in range(n - 1):
        farthest = max(farthest, i + nums[i])
        if i == cur_end:
            jumps += 1
            cur_end = farthest
            if cur_end >= n - 1:
                break
    return jumps
```

**面试时怎么讲**（按"技术的水有多深"八层面，挑重点讲）：

1. **问题层**（本质）：贪心每步选当前最优、不回头，期望得到全局最优。成立需满足贪心选择性质 + 最优子结构。
2. **原理层**：活动选择按结束时间排序——结束早的给后面留空间多；跳跃游戏维护"当前跳覆盖范围"和"下一跳最远位置"，到边界就跳。
3. **优劣层**：$O(n \log n)$ 排序 + $O(n)$ 遍历，空间 $O(1)$；但贪心正确性必须严格证明，证不出来就不能用。
4. **演进层**：从 Kruskal 1956、Dijkstra 1959、Huffman 1952 的具体算法，到 Edmonds 1971 的拟阵统一理论。拟阵给出了"贪心成立的充要骨架"（部分覆盖）。
5. **反例层**：0-1 背包按密度贪心失败（小件锁死容量）；找零 `[1,3,4]` amount=6 贪心失败（拿 4 错过 3+3）。
6. **对比层**：贪心 vs DP——贪心快但需证明，DP 慢但天然正确。能用贪心就别用 DP。
7. **工程层**：生产里"贪心 + 对数器"是常见组合——只要大量随机数据对比通过，工程上即可接受。
8. **陷阱层**：不要"看起来像贪心就用贪心"。0-1 背包和分数背包长得几乎一样，但前者贪心不成立、后者成立，差别只在"能否切割"。

**面试加分点**：主动提"贪心成立的两个判据 + 拟阵"，并说出 0-1 vs 分数背包的反例——这能立刻区分你和"只会背模板"的候选人。

### 2. 对数器：左神风格的验证方法

左神反复强调：**贪心算法最容易写错，因为"看起来对"的策略可能根本不成立。对数器是验证贪心正确性的最后一道防线**。思路是：写一个绝对正确的暴力解（穷举所有可能），和你的贪心解对比，跑足够多次随机数据，只要有不一致就说明贪心策略错了。

下面是活动选择问题的对数器——贪心解 vs 暴力穷举解，跑 5000 次：

```python
import random
from itertools import combinations

def activity_selection_greedy(activities):
    """贪心解：按结束时间排序。"""
    if not activities:
        return 0
    activities = sorted(activities, key=lambda x: x[1])
    count = 1
    last_end = activities[0][1]
    for s, f in activities[1:]:
        if s >= last_end:
            count += 1
            last_end = f
    return count

def activity_selection_brute(activities):
    """暴力解：枚举所有子集，找最大不冲突子集。O(2^n) 绝对正确。"""
    n = len(activities)
    best = 0
    for k in range(n + 1):
        for subset in combinations(range(n), k):
            picked = sorted([activities[i] for i in subset], key=lambda x: x[0])
            ok = True
            for i in range(1, len(picked)):
                if picked[i][0] < picked[i-1][1]:
                    ok = False
                    break
            if ok:
                best = max(best, k)
    return best

def checker(test_times=5000, max_n=12, max_time=20):
    """对数器：5000 次随机对比。max_n 控制小（暴力是 O(2^n)）。"""
    for _ in range(test_times):
        n = random.randint(0, max_n)
        activities = []
        for _ in range(n):
            s = random.randint(0, max_time - 1)
            f = random.randint(s + 1, max_time)
            activities.append((s, f))
        g = activity_selection_greedy(activities)
        b = activity_selection_brute(activities)
        if g != b:
            print(f"出错！输入: {activities}, 贪心: {g}, 暴力: {b}")
            return False
    print("对数器验证通过！贪心解 == 暴力解，5000 次随机数据全对。")
    return True

checker()
```

**对数器关键设计点**：

- **暴力解必须绝对正确**：这里用 `itertools.combinations` 枚举所有子集，$O(2^n)$，n 控制在 12 以内（$2^{12} = 4096$），5000 次跑几秒。
- **随机数据要覆盖边界**：n=0（空集）、n=1（单活动）、所有活动同时开始（退化情况）、活动完全嵌套（区间包含）。
- **不一致就立即停**：找到第一个反例就输出，方便 debug。

**对数器的真正价值——验证你"自创"的贪心策略**：

如果你想试试"按开始时间排序"或"按长度排序"是否成立，把 `activity_selection_greedy` 换成你的策略，对数器一跑就知道——大概率 5000 次里会出错。这种"自我证伪"是贪心学习的核心训练。

```python
def activity_selection_bad_by_start(activities):
    """错误策略：按开始时间排序。对数器一跑就发现错。"""
    if not activities:
        return 0
    activities = sorted(activities, key=lambda x: x[0])
    count = 1
    last_end = activities[0][1]
    for s, f in activities[1:]:
        if s >= last_end:
            count += 1
            last_end = f
    return count
# 反例：[(1, 10), (2, 3), (4, 5)]
# 按开始排：选 (1,10)，后面都被它覆盖，结果 1
# 最优：选 (2,3) 和 (4,5)，结果 2
```

**对高手的启发**：对数器不止用于排序。任何"有标准答案可对照"的算法都能用——贪心、DP、图算法都能对比暴力解。这是"工程化验证"的思维，比"理论推导"更可靠，尤其在面试中遇到"自创贪心策略"时，对数器是判断对错的利器。

### 3. 教科书做法 vs 生产做法

| 场景 | 教科书做法 | 生产做法 | 原因 |
|---|---|---|---|
| 任务调度（活动选择类） | 手写贪心 | 调度框架（如 K8s scheduler） | 生产要考虑优先级、抢占、亲和性，纯贪心不够 |
| 区间合并 / 会议室 | 手写贪心 | 排序 + 标准库 | 模板化，标准库 `sorted` + 一次遍历 |
| 最短路径 | 手写 Dijkstra | NetworkX / `scipy.sparse.csgraph` | 工程实现有堆优化、斐波那契堆优化 |
| 最小生成树 | 手写 Kruskal/Prim | NetworkX / Union-Find 库 | 并查集实现成熟，无需重造 |
| 数据压缩 | 手写 Huffman | `zlib` / `gzip` / LZ77+Huffman | 工业压缩是 LZ77 + Huffman 混合，不是纯 Huffman |
| 背包问题 | 手写 DP 或贪心 | 整数规划求解器（Gurobi/CPLEX） | 通用背包是 NP-hard，工业用 ILP 求解器 |

**生产里贪心几乎不裸用**，原因有二：

1. **生产问题的目标函数往往多目标**（成本 + 时延 + 公平性），单一贪心策略无法平衡。
2. **生产数据有"坏例子"风险**——贪心一旦在某种数据分布上不成立，线上就出故障。所以生产里"贪心 + DP 验证 + 兜底"是常见组合。

但贪心在两个生产场景仍是主力：

- **面试和笔试**：高频，必须手撕。
- **启发式优化**：作为复杂优化的"快速初值生成器"——先用贪心给一个解，再用模拟退火 / 遗传算法 / ILP 优化。

### 4. 三个真实工程坑

**坑一：把 0-1 背包当分数背包用贪心。**

```python
def knapsack_greedy_wrong(weights, values, capacity):
    """错误：按密度贪心，0-1 背包不成立。"""
    items = sorted(zip(weights, values), key=lambda x: x[1]/x[0], reverse=True)
    total = 0
    for w, v in items:
        if w <= capacity:
            capacity -= w
            total += v
    return total
# 反例：weights=[10,20,30], values=[60,100,120], capacity=50
# 贪心选 A+B = 160，最优是 B+C = 220
```

症状：在 0-1 背包上贪心解小于最优解。根因：0-1 背包物品不可分割，选小件锁死容量。修复：0-1 背包必须用 DP，分数背包才能用贪心。

**坑二：找零钱问题用了"美式硬币"假设。**

```python
def coin_change_greedy_wrong(coins, amount):
    """错误：假设贪心成立。"""
    coins.sort(reverse=True)
    count = 0
    for c in coins:
        count += amount // c
        amount %= c
    return count if amount == 0 else -1
# 反例：coins=[1,3,4], amount=6
# 贪心：4+1+1 = 3 枚，最优：3+3 = 2 枚
```

症状：某些面额组合下贪心解大于最优解。根因：面额之间没有整除关系时，拿大额可能"卡住"后续。修复：通用找零用 DP；只有"规范硬币系统"（canonical coin system）才能用贪心，需事先验证。

**坑三：区间调度用了"按开始时间排序"。**

```python
def interval_bad_by_start(intervals):
    """错误：按开始时间排序。"""
    if not intervals:
        return 0
    intervals = sorted(intervals, key=lambda x: x[0])
    count = 1
    last_end = intervals[0][1]
    for s, f in intervals[1:]:
        if s >= last_end:
            count += 1
            last_end = f
    return count
# 反例：[(1, 10), (2, 3), (4, 5)]
# 按开始排：选 (1,10) 后无法再选，结果 1
# 最优：选 (2,3) + (4,5)，结果 2
```

症状：贪心解小于最优解。根因：按开始时间排序会先选到"开始早但很长"的区间，把后面一堆短区间全挡掉。修复：必须按结束时间排序——给后面留最多空间。

### 5. 面试高频题清单

**高频题一：活动选择 / 会议室 II**（区间调度类）

- 模板：按结束时间排序 + 一次遍历
- 变种：会议室 II（求最少会议室数）→ 按开始时间排序 + 最小堆维护结束时间

**高频题二：跳跃游戏 / 跳跃游戏 II**

- 模板：维护 `cur_end` 和 `farthest`，到边界就跳
- 变种：跳跃游戏 I（能否到达）→ 维护 `farthest`，若 `i > farthest` 则 False

**高频题三：分发糖果**（LeetCode 135）

- 模板：两次遍历。先从左到右（右边比左边大就 +1），再从右到左取 max
- 关键：不是"一杆子"贪心，是"双向贪心"——分别满足两个方向的约束

```python
def candy(ratings):
    n = len(ratings)
    candies = [1] * n
    for i in range(1, n):
        if ratings[i] > ratings[i-1]:
            candies[i] = candies[i-1] + 1
    for i in range(n-2, -1, -1):
        if ratings[i] > ratings[i+1]:
            candies[i] = max(candies[i], candies[i+1] + 1)
    return sum(candies)
```

**高频题四：区间合并 / 插入区间 / 用最少数箭引爆气球**

- 模板：按起点排序 + 一次遍历合并 / 计数

**面试题**："手写活动选择，说明贪心为什么成立，并说明哪些类似问题贪心不成立。"

**满分回答要点**：

1. 写出按结束时间排序的模板，跑通 $O(n \log n)$。
2. 用反证法证明贪心选择性质：假设最优解不含结束最早的活动，可以替换出包含它的最优解。
3. 主动给反例：0-1 背包（按密度贪心失败）、找零 `[1,3,4]` amount=6（拿 4 错过 3+3）。
4. 升级到拟阵：贪心成立的充要骨架是问题对应一个拟阵（独立集 + 遗传性 + 交换性），Kruskal 就是拟阵上的贪心。

## 四、速查与自测

### 速查表：贪心算法核心要素

| 维度 | 内容 |
|---|---|
| 直觉 | 每步选当前最优，不回头，期望全局最优 |
| 成立判据 | ①贪心选择性质 ②最优子结构（缺一不可） |
| 时间复杂度 | 通常 $O(n \log n)$（排序主导）或 $O(n)$ |
| 空间复杂度 | $O(1)$ 或 $O(n)$ 输出 |
| 正确性保证 | 必须严格证明；证不出来用 DP |
| 验证手段 | 对数器（贪心解 vs 暴力穷举解） |

**经典成立问题与策略**：

| 问题 | 贪心策略 | 复杂度 |
|---|---|---|
| 活动选择 | 按结束时间排序 | $O(n \log n)$ |
| 跳跃游戏 II | 维护覆盖范围，到边界跳 | $O(n)$ |
| 分数背包 | 按密度从大到小装 | $O(n \log n)$ |
| Huffman 编码 | 每次合并频率最低两个 | $O(n \log n)$ |
| Kruskal MST | 按边权从小到大，不形成环则加 | $O(E \log E)$ |
| Prim MST | 从任一点出发，每次加最近邻 | $O(V^2)$ 或 $O(E \log V)$ |
| Dijkstra | 每次锁定最近未确定点 | $O((V+E) \log V)$ |
| 区间调度 | 按结束时间排序 | $O(n \log n)$ |

**经典反例（贪心不成立）**：

| 问题 | 错误策略 | 正确做法 |
|---|---|---|
| 0-1 背包 | 按密度选 | DP，$O(nW)$ |
| 通用找零 | 按面额从大到小 | DP，$O(n \cdot amount)$ |
| 最长递增子序列 | 每步选最小可接 | DP 或二分优化版 |
| 旅行商 TSP | 每次去最近未访问城市 | NP-hard，需精确算法 |

### 自测三问

**问题一：** 贪心成立的两个判据是什么？用活动选择问题说明它满足这两个判据。

**参考答案：** 两个判据是 ①贪心选择性质：存在一个最优解包含贪心策略的第一步选择；②最优子结构：最优解去掉贪心选择后，是子问题的最优解。活动选择满足贪心选择性质——设 $a_1$ 是结束最早的活动，$A^*$ 是任一最优解，$a_k$ 是 $A^*$ 中结束最早的活动。若 $a_1 \ne a_k$，由于 $f_1 \le f_k$，可以把 $a_k$ 换成 $a_1$，仍是可行解且活动数不变，所以存在包含 $a_1$ 的最优解。活动选择也满足最优子结构——选了 $a_1$ 后，$A^* \setminus \{a_1\}$ 必是子问题（所有 $s_i \ge f_1$ 的活动）的最优解，否则能换成更优的，加回 $a_1$ 得到比 $A^*$ 更优的解，矛盾。

**问题二：** 0-1 背包和分数背包看起来几乎一样，为什么前者贪心不成立、后者成立？请用"贪心选择性质"解释。

**参考答案：** 0-1 背包的贪心选择性质不成立——"单位价值最高的物品一定在某个最优解里"这句话不成立。反例：容量 50，物品 A(10,60,密度6)、B(20,100,密度5)、C(30,120,密度4)。按密度贪心选 A+B=160，但最优是 B+C=220。原因：物品不可分割，选了 A 就锁死了 10 的容量，错过了 B+C 的更优组合——A 不在任何最优解里。分数背包满足贪心选择性质——物品可切割，"密度最高的物品一定在最优解里"成立（因为可以只装一部分，多余容量用低密度物品填，不会"锁死"）。**两者的差别只在"能否切割"这个微小细节，却导致贪心成立性完全相反**——这是判断贪心必须严格证明、不能"看起来一样就用"的最佳例子。

**问题三：** 拟阵是什么？它和贪心算法有什么关系？为什么说拟阵是"贪心成立的充要骨架"但只覆盖部分贪心问题？

**参考答案：** 拟阵是序对 $M = (E, I)$，$E$ 是有限集，$I$ 是独立集族，满足三条公理：①非空性（空集独立）②遗传性（独立集的子集独立）③交换性（小独立集能扩展成大独立集）。Edmonds 1971 证明：在加权拟阵上，"按权重从小到大能加就加"的贪心能得到最优独立集；反过来，能用这种贪心求解的问题必然对应一个拟阵。Kruskal 就是最清晰的例子——$E$=边集，$I$=森林，三公理都成立，所以 Kruskal 必然正确。但拟阵只覆盖部分贪心问题：Huffman、Dijkstra、活动选择虽然贪心也成立，但它们不直接对应拟阵结构，需要单独证明。所以拟阵是"贪心成立的充分骨架"——满足拟阵必能贪心，但能贪心不一定对应拟阵。拟阵的真正价值是把"每个贪心算法单独证明"提升到"检查三公理"的统一框架。

### 算法思想 × 生活迁移

贪心思想的核心是"**局部最优推全局最优**"——每一步选当前最好的，期望最后整体也最好。但本章反复强调：这个推论只在"问题具备特定结构"时成立，否则会陷入"每步都对、整体错了"的陷阱。

**迁移一：投资决策里的"贪心陷阱"。** 看到一只股票连涨三天就追入（局部最优——短期涨幅最大），结果追在山顶。这是典型的贪心不成立——短期涨幅最优不等于长期收益最优。聪明的投资者用"再平衡"策略（定期把组合拉回目标比例），本质是放弃"每步贪心"，改用"全局规划"。**迁移启示**：投资是 0-1 背包式问题——选了这只就锁死了仓位，不能用贪心。

**迁移二：职场选择的"贪心 vs DP"。** 每次跳槽都选"工资最高的 offer"是贪心策略。它成立的前提是"工资最高的 offer 一定在最优职业路径里"——但这往往不成立（高薪但方向不对，错过长期成长）。聪明的职场人用 DP 思维——穷举几条可能的职业路径，比较 5-10 年后的总收益。**迁移启示**：职场是 0-1 背包式（每次选择锁死后续），贪心不成立；但日常小事（今天午饭吃哪家）可以用贪心——单次决策不锁死长期。

**迁移三：项目排期的"活动选择"。** 多个项目抢同一批人，怎么排能做最多？这正是活动选择问题——按"结束时间"（项目预期交付日）排序，先做早结束的，给后面留人。但要注意：这里的"结束时间"必须真实可控，否则贪心会失效（项目延期把后面全挡掉）。**迁移启示**：活动选择贪心成立的前提是"时间预估准确"，工程上要加 buffer。

**为什么这些迁移成立 / 不成立：** 贪心迁移到生活成立的关键，是判断"这个生活问题具备贪心选择性质吗"。能切割、能反悔、单次不锁死后续的（分数背包式），贪心成立；一旦选了就锁死后续、不可切割的（0-1 背包式），贪心不成立，要用"全局规划"思维。这种"先判问题结构、再选策略"的思路，正是贪心章节最有价值的方法论——**不是"什么场景都用贪心"，而是"先判断这个场景贪心成不成立，再决定策略"**。
