# 算法课·32｜线段树与树状数组：区间问题的两把利器

> 前置知识：学完第 31 章（堆与优先队列），理解完全二叉树的数组表示法；学完第 03 章（前缀和），知道前缀和查询 O(1) 的代价
> 学完你能：①说清树状数组靠 lowbit 维护前缀和的原理 ②默写线段树 build/update/query 三件套 ③手撕线段树结构应对面试 ④看懂懒标记 lazy propagation 为何把区间更新从 O(n) 压到 O(log n) ⑤用对数器对比朴素前缀和与线段树

## 一、问题与思路：朴素 O(n) 不够快，区间问题怎么破

### 1. 一个生活场景：班主任的月考成绩单

班主任手上有 100 个学生的月考成绩。她要做两件事：①随时查询某段学号区间（比如第 30-50 号）的总分；②某个学生补考后改成绩，要立即反映到所有相关区间和里。

朴素做法：每次查询就 `for i in [l, r]: s += a[i]`，O(r-l+1) 即 O(n)。改一个成绩 O(1)。1000 次查询就是 O(1000n)。

**前缀和能不能救？** 前缀和 `prefix[i] = a[0]+...+a[i]`，区间和 = `prefix[r] - prefix[l-1]`，查询 O(1)。但改一个成绩 `a[k]`，要把 `prefix[k..n]` 全部重算，更新 O(n)。1000 次更新又是 O(1000n)。

**反直觉点**：前缀和把查询做到 O(1)，代价是更新退化到 O(n)。它把"重算成本"从查询侧搬到更新侧，总成本没变。**频繁更新+频繁查询的场景下，前缀和与朴素做法一样慢**。这就是区间问题的第一道坎——当更新和查询都频繁时，O(n) 的任意一侧都会拖垮整体。

### 2. 区间问题的两类操作

区间问题归结为两类操作：**点操作**（单点更新/单点查询）和**区间操作**（区间更新/区间查询）。组合起来四种典型场景，每种场景下不同数据结构的复杂度差异巨大：

| 操作组合 | 朴素 | 前缀和 | 树状数组 | 线段树 |
|---|---|---|---|---|
| 单点更新 + 区间查询 | 更新 O(1) / 查询 O(n) | 更新 O(n) / 查询 O(1) | 都是 O(log n) | 都是 O(log n) |
| 区间更新 + 区间查询 | 都是 O(n) | 不支持 | 部分支持（差分） | 都是 O(log n) |
| 单点更新 + 区间最值 | 都是 O(n) | 不支持 | 不支持 | 都是 O(log n) |
| 区间更新 + 区间最值 | 都是 O(n) | 不支持 | 不支持 | 都是 O(log n) |

**核心论点**：线段树通用，四种场景全覆盖，但常数大、代码长；树状数组轻量，常数小、代码短，但只能处理"前缀可合并"的操作（和、积、异或这种满足可减性的），最值做不到，区间更新也得靠差分技巧且仅限和。

### 3. 两把利器的分工思路

**树状数组（Binary Indexed Tree，BIT）** 的思路：用部分前缀和拼出任意区间和。每个节点 `tree[i]` 维护一段长度为 `lowbit(i)` 的区间和，查询时把若干段拼起来，更新时把若干段一起改。代码极短，常数极小，但**只能处理满足"可减性"的操作**——和、异或这种 `op(a, b) - op(b) = a` 的，最值不满足可减性，树状数组做不到。

**线段树（Segment Tree）** 的思路：把数组建成一棵二叉树，每个节点维护一段区间。查询/更新时把目标区间递归拆成若干节点，合并即可。**只要操作满足"可合并性"**——区间信息能从两个子区间合并出来，线段树就能处理：和、最值、最大子段和、染色计数、历史最值，全包。代价是常数大、代码长，且区间更新必须靠懒标记 lazy propagation 才能不退化。

**选型口诀**：能用前缀和就别上树状数组，能用树状数组就别上线段树。功能够用即可，不要为通用性付不必要的常数代价。

## 二、原理与实现：树状数组的 lowbit 与线段树的分治

### 1. 树状数组：lowbit 定义了节点管辖的区间长度

#### lowbit 是什么

`lowbit(x) = x & (-x)`，取 x 二进制最低位的 1。例如：

- `lowbit(6)` = `lowbit(0b110)` = `0b10` = 2
- `lowbit(8)` = `lowbit(0b1000)` = `0b1000` = 8
- `lowbit(7)` = `lowbit(0b111)` = `0b1` = 1

`x & (-x)` 的原理：负数在补码表示下是"按位取反加一"，所以 `-x` 的最低位 1 和 `x` 的最低位 1 在同一位置，两者相与就只保留这一位。

#### 节点管辖的区间

`tree[i]` 维护原数组下标区间 `[i - lowbit(i) + 1, i]` 的和。比如：

- `tree[6]` 维护 `[6-2+1, 6] = [5, 6]`，长度 2
- `tree[8]` 维护 `[8-8+1, 8] = [1, 8]`，长度 8
- `tree[7]` 维护 `[7-1+1, 7] = [7, 7]`，长度 1

这张分布藏着一个分形结构：`tree[i]` 的区间长度等于 `lowbit(i)`，而 `lowbit(i)` 恰好是 i 在二叉树结构里"管多大一段"的几何量。lowbit 大的节点管得宽，lowbit 小的节点管得窄，正好对应二叉树里"高层节点管大段、底层节点管小段"的层级关系。

#### 查询前缀和

查询 `a[1] + a[2] + ... + a[i]`：从 i 开始，累加 `tree[i]`，然后跳到 `i - lowbit(i)`，再累加，直到跳到 0。

```python
def query(tree, i):
    s = 0
    while i > 0:
        s += tree[i]
        i -= i & (-i)         # 跳到上一个管辖区间
    return s
```

**逐行讲透**：

- **`while i > 0`**：i 跳到 0 就停。下标从 1 开始，0 是哨兵。
- **`s += tree[i]`**：累加当前节点管的区间和。
- **`i -= i & (-i)`**：跳到 `i - lowbit(i)`，即当前管辖区间的左端点再往左一格——下一个还没被覆盖的下标。

**为什么对**：每次累加 `tree[i]` 覆盖 `[i-lowbit(i)+1, i]`，跳到 `i-lowbit(i)` 就是跳到这段区间的左边一格，也就是下一个还没被覆盖的下标。累加的区间不重不漏地拼成 `[1, i]`。

**走一遍**：i=7。`tree[7]` 管 [7,7]，跳到 6；`tree[6]` 管 [5,6]，跳到 4；`tree[4]` 管 [1,4]，跳到 0。`tree[7]+tree[6]+tree[4]` = `a[7] + a[5..6] + a[1..4]` = `a[1..7]`。

#### 单点更新

把 `a[i]` 加上 delta：从 i 开始，把所有"管到 i 这个下标"的 `tree[j]` 都加上 delta。j 怎么找？`j += lowbit(j)` 一直跳到 n。

```python
def update(tree, n, i, delta):
    while i <= n:
        tree[i] += delta
        i += i & (-i)         # 跳到下一个能管到 i 的节点
```

**逐行讲透**：

- **`while i <= n`**：i 跳出 n 就停。
- **`tree[i] += delta`**：当前节点管到 i，所以它的和也要加 delta。
- **`i += i & (-i)`**：跳到 `i + lowbit(i)`，即下一个能管到 i 的节点。

**为什么对**：`tree[j]` 管 `[j-lowbit(j)+1, j]`，要让它管到 i 必须 `j-lowbit(j)+1 <= i <= j`。从 i 出发 `j = i + lowbit(i)` 是满足这个条件的最小 j——因为 `lowbit(i + lowbit(i))` 至少是 `2 * lowbit(i)`，所以新 j 管的区间一定能覆盖原 i 的位置。

**走一遍**：i=3，n=8。`tree[3]` += delta，跳到 4；`tree[4]` += delta，跳到 8；`tree[8]` += delta，跳出。这三个节点 `tree[3]`、`tree[4]`、`tree[8]` 都管到下标 3。验证：`tree[3]` 管 [3,3]✓；`tree[4]` 管 [1,4]✓（含 3）；`tree[8]` 管 [1,8]✓（含 3）。

#### 复杂度

每次查询/更新，i 每次至少减半/加倍（`lowbit(i) <= i`），所以跳的次数不超过 `log n`。查询和更新都是 O(log n)。

### 2. 线段树：把数组建成二叉树

树状数组靠 lowbit 的分形压榨空间，但牺牲了通用性——只能处理"前缀可合并"的操作。线段树换思路：**直接把数组区间建成一棵二叉树**，每个节点管一段区间，左右子树各管一半。这样任何"能从子区间合并"的操作都能上。

#### 建树 build

数组 `a[0..n-1]`，建一棵线段树（用数组存，根在 1）：

- 节点 `tree[node]` 管区间 `[l, r]`
- 中点 `mid = (l + r) // 2`，左子树管 `[l, mid]`，右子树管 `[mid+1, r]`
- 叶子（`l == r`）：`tree[node] = a[l]`
- 内部节点：`tree[node] = tree[left] + tree[right]`（合并）

```python
def build(tree, a, node, l, r):
    if l == r:
        tree[node] = a[l]
        return
    mid = (l + r) // 2
    build(tree, a, node*2, l, mid)
    build(tree, a, node*2+1, mid+1, r)
    tree[node] = tree[node*2] + tree[node*2+1]    # 合并子区间
```

**逐行讲透**：

- **`if l == r`**：到叶子了，叶子管单个元素，直接赋值。
- **`node*2` 和 `node*2+1`**：根在 1 的二叉树，左子是 `2*node`，右子是 `2*node+1`。这是堆的数组表示法，第 31 章讲过。
- **`mid = (l + r) // 2`**：区间对半分，保证树高 O(log n)。
- **合并 `tree[node] = tree[node*2] + tree[node*2+1]`**：这是线段树的"可合并性"——只要操作能从两段子区间合并出整段信息，就能用线段树。和、最值、最大子段和、染色计数都满足。

**空间**：n 个叶子，内部节点约 n-1 个，总节点不超过 4n。所以 `tree` 数组开 `4*n` 大小（不是 2n，n 不是 2 的幂时会超过 2n）。

#### 区间查询 query

查询 `a[l..r]` 的和：递归到当前节点，如果当前节点区间被 `[l, r]` 完全覆盖，直接返回 `tree[node]`；否则递归左右子树，合并结果。

```python
def query(tree, node, nl, nr, l, r):
    if l <= nl and nr <= r:                  # 当前区间被完全覆盖
        return tree[node]
    mid = (nl + nr) // 2
    s = 0
    if l <= mid:                             # 左子树有交集
        s += query(tree, node*2, nl, mid, l, r)
    if r > mid:                              # 右子树有交集
        s += query(tree, node*2+1, mid+1, nr, l, r)
    return s
```

**逐行讲透**：

- **`if l <= nl and nr <= r`**：当前节点管 `[nl, nr]`，被查询区间 `[l, r]` 完全包含，不用再往下，直接返回这个节点预算好的和。这是线段树快的关键——把查询区间拆成 O(log n) 个"完全覆盖"的节点。
- **`if l <= mid`**：查询区间和左子树 `[nl, mid]` 有交集。注意是 `l <= mid` 不是 `l <= nl`——只要查询区间的左端点没超过 mid，左子树就可能有贡献。
- **`if r > mid`**：查询区间和右子树 `[mid+1, nr]` 有交集。`r > mid` 等价于 `r >= mid+1`。

**复杂度**：每次查询把区间拆成最多 O(log n) 个完全覆盖节点。证明思路：每一层最多选 2 个"部分覆盖"的节点继续往下，所以总节点数不超过 `2 log n`。

#### 单点更新 update

把 `a[idx]` 改成 val：递归到叶子，改完后回溯更新所有经过的节点。

```python
def update(tree, node, nl, nr, idx, val):
    if nl == nr:
        tree[node] = val
        return
    mid = (nl + nr) // 2
    if idx <= mid:
        update(tree, node*2, nl, mid, idx, val)
    else:
        update(tree, node*2+1, mid+1, nr, idx, val)
    tree[node] = tree[node*2] + tree[node*2+1]    # 回溯更新
```

**逐行讲透**：

- **`if nl == nr`**：到叶子了，直接改。
- **`idx <= mid`**：要改的下标在左子树，往左走；否则往右走。
- **回溯 `tree[node] = tree[node*2] + tree[node*2+1]`**：从叶子回溯到根，每经过一个节点都要重新合并——因为子树改了，父节点的合并结果也变了。**这一行是线段树正确性的关键，漏了就全错**。

#### 复杂度对比

| 操作 | 树状数组 | 线段树 | 朴素 |
|---|---|---|---|
| 单点更新 | O(log n) | O(log n) | O(1) |
| 区间查询（和） | O(log n) | O(log n) | O(n) |
| 区间更新 | 不直接支持 | O(log n)（需懒标记） | O(n) |
| 区间最值 | 不支持 | O(log n) | O(n) |
| 常数 | 极小（位运算） | 大（递归） | - |
| 代码量 | ~20 行 | ~40 行 | ~5 行 |

### 3. 懒标记 lazy propagation：区间更新的关键

#### 没有懒标记的区间更新有多慢

要把 `a[l..r]` 全部加上 delta，朴素做法是递归到所有叶子逐个改，O(r-l+1) 即 O(n)。1000 次区间更新就是 O(1000n)。

#### 懒标记的思路

懒标记的思路是**先记账，不真做**。当一次区间更新完全覆盖某个节点 `[nl, nr]` 时，不递归到叶子，而是把这个节点的 `tree[node]` 直接加上 `delta * (nr - nl + 1)`（这段区间的总和变化量），并在 `lazy[node]` 上记一笔"我的所有子节点都欠 delta"。

之后什么时候"真做"？**当下次有查询/更新需要穿过这个节点往下走时，把 lazy 下传给两个子节点**——这叫 `push_down`。下传后这个节点的 lazy 清零。

```python
def push_down(tree, lazy, node, nl, nr):
    if lazy[node] != 0:
        mid = (nl + nr) // 2
        # 左子节点
        tree[node*2] += lazy[node] * (mid - nl + 1)
        lazy[node*2] += lazy[node]
        # 右子节点
        tree[node*2+1] += lazy[node] * (nr - mid)
        lazy[node*2+1] += lazy[node]
        # 自己清零
        lazy[node] = 0

def update_range(tree, lazy, node, nl, nr, l, r, delta):
    if l <= nl and nr <= r:                  # 完全覆盖，记账
        tree[node] += delta * (nr - nl + 1)
        lazy[node] += delta
        return
    push_down(tree, lazy, node, nl, nr)      # 要往下走了，先下传 lazy
    mid = (nl + nr) // 2
    if l <= mid:
        update_range(tree, lazy, node*2, nl, mid, l, r, delta)
    if r > mid:
        update_range(tree, lazy, node*2+1, mid+1, nr, l, r, delta)
    tree[node] = tree[node*2] + tree[node*2+1]    # 回溯
```

**逐行讲透**：

- **`tree[node] += delta * (nr - nl + 1)`**：完全覆盖时，这个节点的和直接加上 delta 乘以区间长度——所有元素都加 delta，总和就是加 `delta * len`。
- **`lazy[node] += delta`**：记账。注意是 `+=` 不是 `=`——多次区间更新可能叠加多个 delta，子节点欠的是总和。
- **`push_down`**：要往子树走了，把欠账下传。左子节点 `tree[node*2] += lazy[node] * (mid-nl+1)`，左子的 lazy 也加上；右子节点同理。最后 `lazy[node] = 0`——账下传完了，自己清零。
- **`push_down` 的时机**：在 `update_range` 和 `query_range` 里，**只要不是完全覆盖、需要往子树走之前，必须先 `push_down`**。否则子节点的 `tree` 值是过期的，往下算就是错的。

#### 查询也要 push_down

```python
def query_range(tree, lazy, node, nl, nr, l, r):
    if l <= nl and nr <= r:
        return tree[node]
    push_down(tree, lazy, node, nl, nr)      # 查询也要先下传
    mid = (nl + nr) // 2
    s = 0
    if l <= mid:
        s += query_range(tree, lazy, node*2, nl, mid, l, r)
    if r > mid:
        s += query_range(tree, lazy, node*2+1, mid+1, nr, l, r)
    return s
```

**关键认知**：lazy 不是"懒"在性能上，是"懒"在执行时机——把 N 次单点更新合并成 1 次区间标记，等真正需要细节时才摊销。这是把区间更新从 O(n) 压到 O(log n) 的关键。

## 三、实践与面试：手撕线段树与对数器验证

### 1. 手撕结构：线段树 build/update/query

面试中"手撕线段树"通常要求：建树 + 单点更新 + 区间查询。这是面试官考察"你能不能把抽象数据结构落到代码上"的硬核题。完整模板（区间和版，封装成类）：

```python
class SegTree:
    def __init__(self, a):
        self.n = len(a)
        self.tree = [0] * (4 * self.n)
        self._build(a, 1, 0, self.n - 1)

    def _build(self, a, node, l, r):
        if l == r:
            self.tree[node] = a[l]; return
        mid = (l + r) // 2
        self._build(a, node*2, l, mid)
        self._build(a, node*2+1, mid+1, r)
        self.tree[node] = self.tree[node*2] + self.tree[node*2+1]

    def update(self, idx, val):
        self._update(1, 0, self.n - 1, idx, val)

    def _update(self, node, nl, nr, idx, val):
        if nl == nr:
            self.tree[node] = val; return
        mid = (nl + nr) // 2
        if idx <= mid: self._update(node*2, nl, mid, idx, val)
        else: self._update(node*2+1, mid+1, nr, idx, val)
        self.tree[node] = self.tree[node*2] + self.tree[node*2+1]

    def query(self, l, r):
        return self._query(1, 0, self.n - 1, l, r)

    def _query(self, node, nl, nr, l, r):
        if l <= nl and nr <= r: return self.tree[node]
        mid = (nl + nr) // 2; s = 0
        if l <= mid: s += self._query(node*2, nl, mid, l, r)
        if r > mid: s += self._query(node*2+1, mid+1, nr, l, r)
        return s
```

**面试时怎么讲**（按"技术的水有多深"四层面）：

1. **问题层**：朴素区间查询 O(n)，前缀和查询 O(1) 但更新 O(n)。线段树把两者都做到 O(log n)。
2. **原理层**：把数组建成二叉树，每个节点管一段区间，查询/更新把目标区间拆成 O(log n) 个完全覆盖节点。
3. **优劣层**：通用（任何可合并操作）、平衡（更新查询同复杂度）；但常数大（递归）、代码长、空间 4n。
4. **演进层**：树状数组适合"前缀和型"操作，线段树适合"任意可合并操作"。生产里 Java 用 FenwickTree，C++ 通常自己封装或用 `__gnu_pbds::tree`。

**面试加分点**：主动说"线段树数组开 4n 不是 2n"——能立刻区分你和"只会背模板"的候选人。n 不是 2 的幂时节点数会超过 2n，开 2n 会越界。

### 2. 对数器：对比朴素前缀和与线段树

左神的对数器思路同样适用：用朴素做法（绝对正确但慢）对照线段树（快但可能写错），跑随机数据，发现不一致立刻定位 bug。

```python
import random

class SegTree:
    """同上的线段树实现，省略。"""
    pass

def naive_update(a, idx, val):
    a[idx] = val

def naive_query(a, l, r):
    return sum(a[l:r+1])

def checker(test_times=500, max_n=50, max_val=100):
    for _ in range(test_times):
        n = random.randint(1, max_n)
        a = [random.randint(0, max_val) for _ in range(n)]
        st = SegTree(a)
        # 每次跑 100 个随机操作
        for _ in range(100):
            op = random.randint(0, 1)
            if op == 0:
                # 单点更新
                idx = random.randint(0, n-1)
                val = random.randint(0, max_val)
                naive_update(a, idx, val)
                st.update(1, 0, n-1, idx, val)
            else:
                # 区间查询
                l = random.randint(0, n-1)
                r = random.randint(l, n-1)
                expected = naive_query(a, l, r)
                actual = st.query(1, 0, n-1, l, r)
                if expected != actual:
                    print(f"出错！a={a}, l={l}, r={r}, 期望={expected}, 实际={actual}")
                    return False
    print("对数器验证通过！")
    return True

checker()
```

**对数器的价值**：

- **覆盖边界**：随机操作序列能触发各种边界——更新刚查询过的下标、查询单点区间（`l == r`）、查询全区间（`l=0, r=n-1`）、连续更新同一位置。
- **不依赖人工**：500 次测试 × 每次 100 操作 = 5 万次操作，几秒内跑完，覆盖度远超手动构造。
- **左神原话**："线段树最容易写错的地方是边界和合并。对数器跑一遍，所有边界 bug 全暴露。"

**对高手的启发**：对数器思想不止用于排序——任何"有标准答案可对照"的算法都能用对数器。写完线段树？对数器对比朴素循环。写完树状数组？对数器对比线段树。这是"工程化验证"的思维，比"理论上推导正确"更可靠。

### 3. 教科书做法 vs 生产做法

| 场景 | 教科书 | 生产 | 原因 |
|---|---|---|---|
| 单点更新 + 区间和 | 手写线段树 | 树状数组 / 前缀和（无更新时） | 树状数组常数小、代码短 |
| 区间更新 + 区间和 | 手写线段树 + 懒标记 | 线段树 / 差分树状数组 | 都行，看团队习惯 |
| 区间最值 | 手写线段树 | 线段树 / ST 表（静态） | 树状数组做不到 |
| 复杂区间操作（染色、历史最值） | 手写线段树 | 线段树 / 平衡树 | 必须线段树 |
| 海量数据（n > 10^9） | 线段树不行 | 动态开点线段树 / 离散化 | 内存不够 |

**生产里的选择优先级**：

1. **能用前缀和就别上树状数组**——无更新时前缀和 O(1) 查询最快，加树状数组纯属多余。
2. **能用树状数组就别上线段树**——单点更新+区间和场景，树状数组代码短、常数小，是首选。
3. **必须线段树时别犹豫**——区间更新、区间最值、复杂合并操作，没有替代品。

### 4. 高手向：懒标记 lazy propagation 的进阶

懒标记不只是"区间加"这么简单，它能处理一大类"区间赋值型"操作。下面看两个进阶变体。

#### 区间赋值 + 区间和

把 `a[l..r]` 全部赋成 val（不是加，是覆盖）。懒标记记的是"这个区间应该全等于多少"，push_down 时直接覆盖子节点的 tree 和 lazy。

```python
def push_down_assign(tree, lazy, has_lazy, node, nl, nr):
    if has_lazy[node]:
        mid = (nl + nr) // 2
        val = lazy[node]
        # 左子
        tree[node*2] = val * (mid - nl + 1)
        lazy[node*2] = val
        has_lazy[node*2] = True
        # 右子
        tree[node*2+1] = val * (nr - mid)
        lazy[node*2+1] = val
        has_lazy[node*2+1] = True
        # 自己清
        has_lazy[node] = False
```

**和区间加的区别**：区间加的 lazy 是 `+=`（叠加），区间赋值的 lazy 是 `=`（覆盖，后到的赋值覆盖前到的）。需要额外的 `has_lazy` 数组标记"这个节点有没有未下传的赋值"，因为赋值 0 也算有效的 lazy，不能靠 `lazy[node] != 0` 判断。

#### 多重懒标记：区间加 + 区间乘

更复杂的场景：同时支持 `a[l..r] += v` 和 `a[l..r] *= v`。需要两个 lazy：`add` 和 `mul`。push_down 时按数学优先级处理——先乘后加：

```python
# 子节点 = 子节点 * mul + add
def push_down_mul_add(tree, add, mul, node, nl, nr):
    mid = (nl + nr) // 2
    # 左子: tree = tree * mul + add * len
    tree[node*2] = tree[node*2] * mul[node] + add[node] * (mid - nl + 1)
    mul[node*2] *= mul[node]
    add[node*2] = add[node*2] * mul[node] + add[node]
    # 右子同理
    tree[node*2+1] = tree[node*2+1] * mul[node] + add[node] * (nr - mid)
    mul[node*2+1] *= mul[node]
    add[node*2+1] = add[node*2+1] * mul[node] + add[node]
    # 清
    mul[node] = 1
    add[node] = 0
```

**关键认知**：多重懒标记必须按数学优先级下传——先乘后加，否则会算错。子节点原本是 `tree * mul_child + add_child`，父节点下传 `(mul, add)` 后变成 `(tree * mul_child + add_child) * mul + add` = `tree * (mul_child * mul) + (add_child * mul + add)`，所以新的 `mul_child = mul_child * mul`，新的 `add_child = add_child * mul + add`。这是线段树最考察"对操作语义理解"的地方。洛谷 P3373 区间加乘混合就是这种类型。

### 5. 面试高频题

#### 题 1：区域和检索（LeetCode 307）

> 给定数组 `nums`，支持 `update(index, val)` 修改单点，`sumRange(l, r)` 查询区间和。要求两种操作都高效。

**朴素做法**：每次 `sumRange` 都遍历，O(n)。会超时。

**前缀和不行**：update 是 O(n)，频繁 update 也会超时。

**线段树解法**（参考前面 SegTree 类，调用方式）：

```python
class NumArray:
    def __init__(self, nums):
        self.n = len(nums)
        self.tree = [0] * (4 * self.n)
        self._build(nums, 1, 0, self.n - 1)
    
    def _build(self, a, node, l, r):
        if l == r:
            self.tree[node] = a[l]; return
        mid = (l + r) // 2
        self._build(a, node*2, l, mid)
        self._build(a, node*2+1, mid+1, r)
        self.tree[node] = self.tree[node*2] + self.tree[node*2+1]
    
    def update(self, index, val):
        self._update(1, 0, self.n - 1, index, val)
    
    def _update(self, node, nl, nr, idx, val):
        if nl == nr:
            self.tree[node] = val; return
        mid = (nl + nr) // 2
        if idx <= mid: self._update(node*2, nl, mid, idx, val)
        else: self._update(node*2+1, mid+1, nr, idx, val)
        self.tree[node] = self.tree[node*2] + self.tree[node*2+1]
    
    def sumRange(self, l, r):
        return self._query(1, 0, self.n - 1, l, r)
    
    def _query(self, node, nl, nr, l, r):
        if l <= nl and nr <= r: return self.tree[node]
        mid = (nl + nr) // 2; s = 0
        if l <= mid: s += self._query(node*2, nl, mid, l, r)
        if r > mid: s += self._query(node*2+1, mid+1, nr, l, r)
        return s
```

**树状数组也能做**（更短，复用 Section 2.1 的 `_add` / `_prefix`）：

```python
class NumArray:
    def __init__(self, nums):
        self.n = len(nums)
        self.nums = nums[:]                 # 留一份原值，更新时算 delta
        self.tree = [0] * (self.n + 1)
        for i in range(self.n):
            self._add(i + 1, nums[i])       # 下标 +1：原数组 0-based → BIT 1-based

    def update(self, index, val):
        self._add(index + 1, val - self.nums[index])    # delta = 新值 - 旧值
        self.nums[index] = val

    def sumRange(self, l, r):
        return self._prefix(r + 1) - self._prefix(l)    # 区间和 = 前缀差
```

**面试时怎么选**：题目说"update 和 sumRange 都很多次"，两者都 O(log n)。树状数组代码短常数小，优先选；如果面试官追问"换成区间最值怎么办"，那只能线段树。

#### 题 2：翻转对（LeetCode 493）

> 给定数组 `nums`，统计满足 `i < j` 且 `nums[i] > 2 * nums[j]` 的对数。

**朴素**：双重循环 O(n²)，会超时。

**树状数组解法**：把"统计对数"转化为"对每个 j，统计左边有多少个 `nums[i] > 2*nums[j]`"。值域很大（`-2^31 ~ 2^31-1`），需要离散化。

```python
import bisect

def reversePairs(nums):
    # 离散化：把 nums[i] 和 2*nums[j] 一起排序去重
    vals = sorted(set(nums) | set(2 * x for x in nums))
    rank = {v: i + 1 for i, v in enumerate(vals)}    # 1-based
    n = len(vals)
    tree = [0] * (n + 1)
    
    def add(i, delta):
        while i <= n:
            tree[i] += delta
            i += i & (-i)
    
    def query(i):    # 查 [1, i] 的和
        s = 0
        while i > 0:
            s += tree[i]
            i -= i & (-i)
        return s
    
    ans = 0
    for x in nums:                          # x 当作 nums[j]
        # 已入树的都是 i < j 的元素，查 nums[i] > 2*x 的个数
        # = total - count(value <= 2*x)
        idx = bisect.bisect_right(vals, 2 * x)    # vals 中 <= 2x 的个数
        ans += query(n) - query(idx)
        add(rank[x], 1)                     # 把 x 入树
    return ans
```

**讲解要点**：

1. **离散化**：值域 `2^31` 太大，但只有 `2n` 个不同值（`nums[i]` 和 `2*nums[j]`），离散化到 `[1, 2n]`，树状数组才开得起。
2. **从"统计对数"到"前缀查询"的转化**：把双重循环的对数统计，转化为"对每个元素，查询已入树的前缀和"——这是树状数组最经典的应用模式。LeetCode 315 计算右侧小于当前元素的个数、LeetCode 327 区间和计数都是这个套路。
3. **树状数组 vs 归并排序**：这道题也能用归并排序做，O(n log n)。树状数组的好处是"在线"——可以支持动态修改；归并是离线的，数组定了才能算。

**为什么 `bisect_right` 而不是 `bisect_left`**：要统计 `nums[i] > 2*x`（严格大于），等价于 `total - count(nums[i] <= 2*x)`。`bisect_right` 返回 `<= 2*x` 的元素个数，正好是要减去的部分。如果用 `bisect_left`，会把等于 `2*x` 的元素算进"大于"那侧，错。

#### 题 3：计数数组（流式数据 + 前缀查询）

> 给定一个流式数据，支持两种操作：①加入一个数 x；②查询当前 ≤ k 的数有多少个。

这是树状数组的"原生场景"——单点更新（加一个数）+ 前缀查询。

```python
class Counter:
    def __init__(self, max_val):
        self.n = max_val
        self.tree = [0] * (self.n + 1)
    
    def add(self, x):                       # 值 x 对应下标 x
        i = x
        while i <= self.n:
            self.tree[i] += 1
            i += i & (-i)
    
    def count_le(self, k):                  # 查询 <= k 的个数
        s = 0
        while k > 0:
            s += self.tree[k]
            k -= k & (-k)
        return s
```

**这道题考察什么**：

- **树状数组的"原型"**：这是树状数组最初被设计出来的场景——统计频率 + 前缀查询。Peter Fenwick 1994 年的原始论文就是为了这种"累积频率表"查询。
- **离散化大值域**：如果数据范围 `10^9`，必须先离散化——把所有出现过的值排序去重，映射到 `[1, m]`。
- **在线 vs 离线**：树状数组支持在线（数据流式到来也能查），前缀和数组必须离线（先收集所有数据再建表）。

### 6. 三个真实工程坑

**坑一：线段树数组开小了，越界段错误。**

```python
# 错误：开 2n 不够
tree = [0] * (2 * n)   # n 不是 2 的幂时越界！

# 正确：开 4n
tree = [0] * (4 * n)
```

症状：跑大数据时 segment fault 或结果错乱。根因：n 不是 2 的幂时，线段树的节点数会超过 2n，最多到 4n。修复：永远开 4n。

**坑二：树状数组下标从 0 开始，`lowbit(0) = 0` 死循环。**

```python
# 错误：i 从 0 开始时，i += i & (-i) 在 i=0 处死循环（lowbit(0)=0）
# 正确：下标从 1 开始，原数组 0-based 下标 i 对应 BIT 下标 i+1
```

症状：调用 `update(tree, n, 0, delta)` 程序卡死。根因：`lowbit(0) = 0`，`i += 0` 死循环。修复：树状数组下标必须从 1 开始，原数组下标 i 对应树状数组下标 `i + 1`。

**坑三：懒标记忘了 push_down，区间更新后查询结果错。**

```python
# 错误：query_range 在不完全覆盖时直接递归子树，没先 push_down
# 正确：不完全覆盖、要往子树走之前，必须先 push_down(tree, lazy, node, nl, nr)
```

症状：先区间更新 `a[1..5] += 3`，再查 `a[2..3]` 的和，结果不对。根因：区间更新时只在完全覆盖的节点上记了 lazy，没下传到子节点；查询时直接读子节点的 tree 值，是更新前的旧值。修复：query_range 在不完全覆盖、要往子树走之前，必须先 `push_down`。

## 四、速查与自测

### 速查表：树状数组 vs 线段树

| 指标 | 树状数组（BIT） | 线段树 |
|---|---|---|
| 单点更新 | O(log n) | O(log n) |
| 区间查询（和） | O(log n) | O(log n) |
| 区间查询（最值） | 不支持 | O(log n) |
| 区间更新（和） | 差分版 O(log n) | O(log n)（需懒标记） |
| 区间更新（最值） | 不支持 | O(log n)（需懒标记） |
| 复杂区间操作 | 不支持 | 支持（染色、历史最值等） |
| 空间 | n+1 | 4n |
| 常数 | 极小（位运算） | 大（递归） |
| 代码量 | ~20 行 | ~40 行（带懒标记 ~60 行） |
| 适用场景 | 前缀和型操作 | 任意可合并操作 |

**线段树节点合并操作的常见类型**：

| 操作 | 合并方式 | lazy 含义 |
|---|---|---|
| 区间和 | `tree[l] + tree[r]` | `+= delta` |
| 区间最值 | `max(tree[l], tree[r])` | 不支持普通 lazy |
| 区间最大子段和 | 三段合并（左/右/跨中） | 复杂 |
| 区间染色 | `tree[l] == tree[r] ? tree[l] : -1` | `= color` |
| 区间加 + 区间乘 | `tree * mul + add` | `(add, mul)` 双标记 |

### 自测三问

**问题一：** 树状数组 `tree[i]` 维护的区间长度为什么是 `lowbit(i)`？请用 `i=6` 和 `i=8` 举例说明，并解释为什么这种设计能让查询在 O(log n) 内完成。

**参考答案：** `lowbit(i)` 是 i 二进制最低位的 1。`tree[i]` 维护 `[i - lowbit(i) + 1, i]` 的和，区间长度恰好是 `lowbit(i)`。`i=6`（二进制 110），`lowbit(6)=2`，`tree[6]` 维护 `[5,6]` 长度 2；`i=8`（二进制 1000），`lowbit(8)=8`，`tree[8]` 维护 `[1,8]` 长度 8。这种"lowbit 大的节点管得宽、lowbit 小的节点管得窄"的设计，让查询时 `i -= lowbit(i)` 跳跃的步长正好等于跳过的区间长度——跳一次就跳过一段已经累加过的区间，不重不漏。i 的二进制位数为 `log n`，每次跳跃至少消去一个 1，所以最多跳 `log n` 次就到 0。

**问题二：** 线段树的懒标记 lazy propagation 为什么能把区间更新从 O(n) 压到 O(log n)？关键操作是什么？查询时为什么也要 push_down？

**参考答案：** 关键在于"先记账不真做"。区间更新完全覆盖某个节点时，不递归到叶子，而是直接 `tree[node] += delta * len` 并在 `lazy[node]` 上记一笔"子节点欠 delta"。这样一次区间更新只触及 O(log n) 个完全覆盖节点，不需要扫到所有叶子。关键操作是 `push_down`——当后续操作需要穿过这个节点往下走时，把 lazy 下传给两个子节点并清零自己的 lazy。查询时也要 push_down 的原因：查询可能需要往子树走，如果不先下传 lazy，子节点的 tree 值是更新前的旧值，查询结果就错了。**懒标记不是性能上的偷懒，是执行时机上的延迟——把 N 次单点更新合并成 1 次区间标记，等真正需要细节时才摊销**。

**问题三：** 同一道题（如 LeetCode 307 区域和检索）既能用树状数组也能用线段树，怎么选？请从功能、常数、代码量、可扩展性四方面对比，并给出选型策略。

**参考答案：**

- **功能**：307 是单点更新 + 区间和，两者都支持，平手。
- **常数**：树状数组用位运算、非递归，常数远小于线段树（递归）。大数据下树状数组快 2-3 倍。
- **代码量**：树状数组 ~20 行，线段树 ~40 行，树状数组胜。
- **可扩展性**：如果面试官追问"改成区间最值/区间更新/染色"，树状数组做不到（或需要复杂变形），线段树直接换合并函数和 lazy 即可。线段树胜。

**选型策略**：307 这种单点更新 + 区间和，先用树状数组；如果题目可能扩展（面试官有后续追问），或本身就是区间更新/区间最值，直接上线段树。生产里同样如此——能用树状数组就别上线段树，省常数省代码。

### 算法思想 × 生活迁移

区间数据结构的核心思想是"**分治 + 缓存合并结果**"——把大区间拆成小区间，每个节点缓存自己区间的合并结果，查询时拼装若干缓存，更新时回溯刷新缓存。

**迁移一：缓存系统设计。** 你维护一个电商网站的销售额统计。朴素做法是每次查"过去 7 天总销售额"都查数据库累加——O(n)。线段树式做法是按天建一棵"统计树"，每天销售额变动只更新一个叶子和它的祖先（O(log n)），查询任意时间段总和拼装 O(log n) 个节点。这就是为什么实时大屏的统计接口能做到毫秒级响应——本质是线段树的"分治 + 缓存"。

**迁移二：组织架构管理。** 公司有 1 万人，要频繁查"某部门总人数""某条业务线总产出"。朴素做法每次遍历部门成员——O(n)。线段树式做法：把组织架构建成一棵树（部门-小组-个人），每个节点缓存自己子树的人数/产出总和。人员变动只更新从叶子到根的 O(log n) 个节点，查询任意部门 O(log n)。**只要"区间信息能从子区间合并出来"，就能套这种结构**。

**迁移三：懒标记思想——延迟计算。** 团队 leader 给 10 个下属都布置了任务（"区间更新"），不需要立即逐个交代细节，只需要在团队看板记一笔"全员欠 XX 任务"（lazy 标记）。等某个人来问"我具体要做什么"时（需要细节），才把任务下传给他（push_down）。这种"先记账后摊销"的思路在工程里很常见——React 的批量更新、数据库的写前日志、操作系统的写时复制，都是"先标记延迟执行"的变体。

**为什么这些迁移成立：** 线段树成立的前提是"区间信息可合并"——整体信息能从部分信息计算出来。这个前提在销售额统计（和可合并）、人数统计（和可合并）、任务派发（标记可叠加）里都成立。迁移成立的关键是"信息可合并"和"操作可延迟"——如果操作必须立即生效（比如实时风控不能延迟），懒标记就不适用，只能用单点更新版本。
