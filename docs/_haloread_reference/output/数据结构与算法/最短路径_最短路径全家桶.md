## 一、问题与思路：最短路径不是一个算法，是一个家族

### 1. 一个反直觉问题：导航软件到底用了几个算法

打开高德地图，输入"国贸到首都机场"，0.3 秒返回最短路线——你大概率以为它跑的就是 Dijkstra。错了。真实导航引擎里 Dijkstra、A\*、Floyd 各占一席：城际干线用 A\* 启发式搜索，最后一公里用 Dijkstra，跨城多模式换乘查询里 Floyd 仍藏在某些预处理模块中。最短路径从来不是一个算法，是一个家族。

这三者不是"谁替代谁"的关系，而是各管一摊：Dijkstra 管非负权单源最短路径，Bellman-Ford 管带负权的单源，Floyd 管所有点对的全源最短路径。把三者搞混，就像用螺丝刀拧螺母——不是工具不好，是用错地方。

### 2. 为什么"一个算法打天下"走不通：三类边权三套打法

最短路径问题的输入是一个带权图 $G=(V,E)$ 和一个源点 $s$，输出是 $s$ 到其他所有点的最短距离。看似一个问题，实际分裂出三类：

**第一类：边权全部非负。** 这是最常见的场景——路程、时间、费用都不会是负数。此时贪心策略成立：每次从未确定点中选距离最小的，确定下来就不会再被推翻。Dijkstra 就是这个思路。

**第二类：存在负权边但没有负环。** 比如"中转有补贴"——经过某条边反而赚钱，距离变负。此时贪心失效：你确定了一个点的最短距离，后面可能通过负权边绕回来更短。Bellman-Ford 通过反复"松弛"所有边来解决。

**第三类：需要所有点对的最短路径。** 不是问"从 A 到 B"，而是问"任意两点之间的最短路径"。可以对每个点跑一次 Dijkstra，但 Floyd 给了更直接的动态规划解法，$O(V^3)$ 一次性算完。

**关键反直觉点**：Dijkstra 遇到负权边会出错。很多人记忆里"Dijkstra 就是最短路径算法"就以为它能处理所有情况，实际它只对非负权正确。下面用一个反例证明。

### 3. 三种算法的历史脉络与分工

- **Dijkstra（1959）**：Edsger Dijkstra 在阿姆斯特丹数学中心喝咖啡时，二十分钟构思出这个算法 [8]。它解决非负权单源最短路径，是工程上用得最多的最短路径算法。
- **Bellman-Ford（1958-1962）**：Richard Bellman 和 Lester Ford 分别独立提出，能处理负权边并检测负环。动态规划宗师 Bellman 把它看作 DP 的早期应用。
- **Floyd（1962）**：Robert Floyd 提出的全源最短路，本质是动态规划。同年 Warshall 用类似思路算传递闭包，合称 Floyd-Warshall。
- **SPFA（1994）**：段家发（Duan Fanding）在 Bellman-Ford 基础上做队列优化，国内 OI 圈广泛使用，但最坏仍 $O(VE)$。
- **A\*（1968）**：Hart、Nilsson、Raphael 提出，引入启发式函数引导搜索方向，是 Dijkstra 的"带方向版"，至今仍是游戏寻路和地图导航的主流。

三者的分工可以这样记：**非负权找 Dijkstra，带负权找 Bellman-Ford，全源找 Floyd。**

## 二、原理与实现：从单源到全源，从非负到负权

### 1. Dijkstra：贪心 + 堆，非负权上的单源最优解

#### 1.1 核心思想：确定一个就锁死一个

Dijkstra 的思路是一句话：**每次从未确定点中选距离源点最近的，把它"确定"下来，然后用它去松弛邻居。**

为什么能"确定就锁死"？前提是边权非负。假设当前最近的未确定点是 $u$，距离 $d(u)$。如果未来存在某条更短的路径 $s \to \dots \to v \to u$，因为边权非负，$d(v) \ge d(u)$（$v$ 在 $u$ 之后被确定，距离不会比 $u$ 小），所以 $d(v) + w(v,u) \ge d(u)$，不可能更短。这就锁死了 $u$。

**反过来，如果存在负权边**，这个论证就崩了：$v$ 即使后来才被确定，$w(v,u)$ 可能是负的，绕一圈回来反而更短。这就是 Dijkstra 不能处理负权的根因。

#### 1.2 朴素版 $O(V^2)$

先看朴素版本，理解机制：

```python
import sys

def dijkstra_v1(graph, n, src):
    """朴素 Dijkstra，O(V^2)。graph 是邻接矩阵，graph[u][v] 表示边权，无边用 INF。"""
    INF = sys.maxsize
    dist = [INF] * n
    visited = [False] * n
    dist[src] = 0

    for _ in range(n):
        # 1. 在未确定点中选 dist 最小的
        u = -1
        min_d = INF
        for i in range(n):
            if not visited[i] and dist[i] < min_d:
                min_d = dist[i]
                u = i
        if u == -1:               # 剩下的点都不可达
            break
        visited[u] = True

        # 2. 用 u 松弛邻居
        for v in range(n):
            if not visited[v] and graph[u][v] != INF:
                if dist[u] + graph[u][v] < dist[v]:
                    dist[v] = dist[u] + graph[u][v]
    return dist
```

**逐行讲透**（左神风格，每一行都不能含糊）：

- **`dist[src] = 0`**：源点到自己距离 0，其他点初始无穷大（不可达）。
- **外层 `for _ in range(n)`**：每轮确定一个点，最多 n 轮。
- **选最小 `u`**：在未确定点里挑 `dist` 最小的，这一步朴素做法是 $O(V)$ 扫一遍，是性能瓶颈。
- **`visited[u] = True`**：锁死 $u$，后续不再动它。
- **松弛邻居**：对 $u$ 的每条出边 $(u,v,w)$，若 `dist[u] + w < dist[v]`，更新 `dist[v]`。这一步叫"松弛"（relaxation），是所有最短路径算法的核心操作。

#### 1.3 堆优化版 $O(E \log V)$

瓶颈在"选最小 u"。用最小堆（Min-Heap）维护未确定点，每次 pop 出最小，复杂度降到 $O(E \log V)$：

```python
import heapq
import sys

def dijkstra(graph, n, src):
    """堆优化 Dijkstra，O(E log V)。graph 是邻接表，graph[u] = [(v, w), ...]。"""
    INF = sys.maxsize
    dist = [INF] * n
    dist[src] = 0
    heap = [(0, src)]              # (距离, 点)，堆按距离排序

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]:            # 堆里可能有同一点的过期记录，跳过
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(heap, (nd, v))
    return dist
```

**逐行讲透**：

- **`heap = [(0, src)]`**：堆初始放源点，距离 0。堆里存 `(距离, 点)`，元组比较先比距离。
- **`d, u = heapq.heappop(heap)`**：每次弹出当前距离最小的点。
- **`if d > dist[u]: continue`**：这是关键一行——堆里可能有同一个点的多条记录（每次松弛都 push 一条），如果弹出的距离已经大于 `dist[u]`，说明是过期记录，跳过。这种"惰性删除"比从堆里精确删除简单得多。
- **松弛并 push**：每次松弛成功就把新距离 push 进堆，不修改旧记录。

**复杂度**：每条边松弛一次会触发一次 push，最多 $O(E)$ 次入堆，每次 push/pop $O(\log V)$，总计 $O(E \log V)$。这就是 Dijkstra 用堆优化后能处理百万级边图的根因。

#### 1.4 Dijkstra 遇到负权边会出错的反例

图：$s \to a$ 权 5，$a \to b$ 权 -10，$s \to b$ 权 2。

```
正确答案：dist[b] = 5 + (-10) = -5（s→a→b）
Dijkstra 执行顺序：
  1. 确定 s，松弛 a=5, b=2
  2. 选最小 b（d=2），确定 b ← 错了！b 一旦确定就不再更新
  3. 选 a（d=5），松弛 b = 5 + (-10) = -5，但 b 已 visited，跳过
  → dist[b] 算成 2，实际是 -5
```

**根因**：b 比 a 早被确定（$d=2 < d=5$），但通过 a 绕回来的负权边能让 b 更短。Dijkstra 的"确定就锁死"在负权边面前不成立。所以 Dijkstra **严禁**用在有负权边的图上。

### 2. Bellman-Ford：松弛建立在最朴素的事实上，能处理负权

#### 2.1 思路：暴力松弛所有边 V-1 轮

Dijkstra 失败在"过早确定"。Bellman-Ford 干脆不"确定"，把所有边松弛 V-1 轮，每一轮都可能更新距离，最后一定能收敛到正确答案。

为什么是 V-1 轮？因为最短路径（无负环时）最多 V-1 条边——再长就重复走点形成环，非负环只会让路径更长（去掉环更短）。所以 V-1 轮松弛足以让最短路径的"信息"从源点传到最远的点。

```python
import sys

def bellman_ford(edges, n, src):
    """Bellman-Ford。edges 是边列表 [(u, v, w), ...]。能处理负权，能检测负环。"""
    INF = sys.maxsize
    dist = [INF] * n
    dist[src] = 0

    # 1. 松弛 V-1 轮
    for _ in range(n - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != INF and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        if not updated:            # 本轮没更新，提前结束
            break

    # 2. 检测负环：再松弛一轮，还能更新就说明有负环
    for u, v, w in edges:
        if dist[u] != INF and dist[u] + w < dist[v]:
            return None            # 存在负环

    return dist
```

**逐行讲透**：

- **松弛 V-1 轮**：每轮遍历所有边，尝试松弛。注意 `dist[u] != INF` 的判断——`INF + 负数` 在 Python 里仍是合法大数，会错误更新，所以要先排除不可达点。
- **`updated` 提前结束**：如果某一轮没有任何更新，说明已收敛，提前 break。这是工程优化，最坏仍是 $O(VE)$。
- **负环检测**：V-1 轮后如果还能松弛，说明存在负环——绕一圈距离变负，可以无限绕下去，最短路径无定义。

**复杂度**：$O(VE)$，比 Dijkstra 慢一个量级。但能处理负权和检测负环，是它的不可替代性所在。

#### 2.2 应用场景：套汇交易

Bellman-Ford 最经典的应用是货币套汇。若 1 美元换 0.9 欧元，1 欧元换 1.2 瑞郎，1 瑞郎换 0.95 美元，绕一圈 1 美元变 $0.9 \times 1.2 \times 0.95 = 1.026$ 美元——赚了 2.6%。

把汇率取负对数作为边权（$-\ln$ 汇率），"绕一圈赚钱"等价于"绕一圈距离变负"，即负环。Bellman-Ford 检测负环，就是检测套汇机会。这是金融工程里 Bellman-Ford 的真实落地。

### 3. SPFA：Bellman-Ford 的队列优化，以及它的"名声争议"

#### 3.1 思路：只松弛"上轮被更新过的点"的出边

Bellman-Ford 每轮松弛所有边，但只有"上轮被更新的点"的出边才可能引起新的松弛。SPFA（Shortest Path Faster Algorithm）用一个队列维护"待松弛的点"，只处理这些点的出边：

```python
import sys
from collections import deque

def spfa(graph, n, src):
    """SPFA：Bellman-Ford 的队列优化。graph 是邻接表。"""
    INF = sys.maxsize
    dist = [INF] * n
    dist[src] = 0
    in_queue = [False] * n
    cnt = [0] * n                  # 记录每个点入队次数，用于检测负环

    q = deque([src])
    in_queue[src] = True

    while q:
        u = q.popleft()
        in_queue[u] = False
        for v, w in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                cnt[v] = cnt[u] + 1
                if cnt[v] >= n:    # 入队次数 >= n，存在负环
                    return None
                if not in_queue[v]:
                    q.append(v)
                    in_queue[v] = True
    return dist
```

**复杂度争议**：SPFA 平均 $O(kE)$（$k$ 是常数约 2），最坏 $O(VE)$，和 Bellman-Ford 一样。问题在于"最坏"很容易被构造——NOI 2008 出题人专门卡 SPFA 的数据让大量选手爆零，从此国内 OI 圈有"SPFA 已死"的调侃。但工程上随机图里 SPFA 比 Bellman-Ford 快几倍，仍有用武之地。

**面试建议**：写 SPFA 要主动说"最坏 $O(VE)$，可用 Bellman-Ford 或 Dijkstra 替代"，体现你知道它的边界。

### 4. Floyd：动态规划，$O(V^3)$ 算出所有点对

#### 4.1 思路：枚举中转点

Floyd 的思路非常直接：对每一对点 $(i,j)$，看能否通过某个中转点 $k$ 让路径更短。三重循环：

```python
import sys

def floyd(graph, n):
    """Floyd：全源最短路径，O(V^3)。graph 是邻接矩阵。"""
    INF = sys.maxsize
    dist = [row[:] for row in graph]   # 深拷贝，不破坏原矩阵

    for k in range(n):                 # 中转点
        for i in range(n):             # 起点
            for j in range(n):         # 终点
                if dist[i][k] != INF and dist[k][j] != INF:
                    if dist[i][k] + dist[k][j] < dist[i][j]:
                        dist[i][j] = dist[i][k] + dist[k][j]
    return dist
```

**逐行讲透**：

- **`k` 在最外层**：这是 Floyd 的关键。如果 `k` 在最内层，会用到本轮还没算完的中间结果，结果错。`k` 在外层表示"依次允许经过中转点 $0, 1, \dots, k$"，是动态规划的状态递进。
- **`dist[i][k] != INF and dist[k][j] != INF`**：防溢出——`INF + INF` 在 Python 是合法大数但语义错误。
- **状态转移**：`dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`，等价于"用 $k$ 做中转点能否让 $i \to j$ 更短"。

#### 4.2 为什么 $O(V^3)$ 还有人用

Floyd 三重循环看着吓人，但有几个独特优势：

1. **代码极短**：三重循环 + 一行状态转移，比写 N 遍 Dijkstra 简单得多。
2. **能处理负权**：只要没有负环，Floyd 都能算对（负环检测：跑完后看 `dist[i][i] < 0` 即可）。
3. **小图上很快**：$V=200$ 时 $V^3 = 8 \times 10^6$，毫秒级。比起跑 200 次 Dijkstra（$200 \times E \log V$）反而快。

**适用边界**：$V \le 500$ 时 Floyd 是首选；$V$ 上千就得上 N 次 Dijkstra（非负权）或 N 次 Bellman-Ford（带负权）。

### 5. A\*：启发式搜索的代表

A\*（A-star）是 Dijkstra 的"带方向版"。Dijkstra 像往水里扔石子，波纹向四面八方均匀扩散；A\* 加一个启发式函数 $h(v)$（$v$ 到终点的估计距离），优先朝终点方向扩散，能少探索大量无关点。

公式：$f(v) = g(v) + h(v)$。$g(v)$ 是源点到 $v$ 的实际距离（Dijkstra 的 `dist`），$h(v)$ 是 $v$ 到终点的估计。堆里按 $f(v)$ 排序。

**关键约束**：$h(v)$ 必须是"可采纳的"（admissible），即 $h(v) \le$ 实际最短距离。欧氏距离、曼哈顿距离都满足。若 $h \equiv 0$，A\* 退化成 Dijkstra。

**典型场景**：地图导航（$h$ 用两地直线距离）、游戏寻路（$h$ 用曼哈顿距离）。A\* 不保证对所有图都比 Dijkstra 快，但在"有明确终点且能估计距离"的图上效果显著。本节不展开实现，理解它的"加方向引导"思想即可。

### 6. 三种算法适用场景对比

| 算法 | 时间复杂度 | 适用边权 | 单源/全源 | 典型场景 |
|---|---|---|---|---|
| Dijkstra（堆优化） | $O(E \log V)$ | 非负权 | 单源 | 路网导航、网络路由 |
| Bellman-Ford | $O(VE)$ | 含负权 | 单源 | 套汇检测、负环检测 |
| SPFA | 平均 $O(kE)$，最坏 $O(VE)$ | 含负权 | 单源 | 随机图上的 BF 替代 |
| Floyd | $O(V^3)$ | 含负权（无负环） | 全源 | 小图全源、传递闭包 |
| A\* | 依赖 $h$ | 非负权 | 单源单终点 | 地图导航、游戏寻路 |

**选型口诀**：单源非负用 Dijkstra，单源负权用 Bellman-Ford（或 SPFA），全源小图用 Floyd，有明确终点且能估计距离用 A\*。

## 三、实践与面试：手撕模板、对数器、面试题

### 1. 三个面试手撕模板

Dijkstra 和 Bellman-Ford 模板前面已给，这里补 Floyd 的讲解要点和三者的"边写边讲"提示。

**Dijkstra 讲解要点**（边写边讲）：

1. "用最小堆维护未确定点，每次弹出距离最小的。"
2. "弹出来如果 `d > dist[u]` 跳过——这是惰性删除，处理堆里同一点的过期记录。"
3. "松弛邻居：`dist[u] + w < dist[v]` 就更新并 push。"
4. "复杂度 $O(E \log V)$，前提是边权非负。"

**Bellman-Ford 讲解要点**：

1. "松弛 V-1 轮，每轮遍历所有边。"
2. "V-1 是因为最短路径最多 V-1 条边。"
3. "再松弛一轮还能更新就说明有负环。"
4. "复杂度 $O(VE)$，能处理负权。"

**Floyd 讲解要点**：

1. "三重循环，k 必须在最外层——这是动态规划的状态递进。"
2. "状态转移：`dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`。"
3. "复杂度 $O(V^3)$，但代码极短，能处理负权。"
4. "V ≤ 500 首选，V 上千就换 N 次 Dijkstra。"

**面试加分点**：主动说"Dijkstra 不能处理负权，BF 能但有 $O(VE)$，Floyd 全源但 $O(V^3)$"——能立刻区分你和"只会背模板"的候选人。

### 2. 对数器：随机图交叉验证三种算法

写完三个算法，用对数器在随机图上交叉验证：在**非负权**随机图上，Dijkstra、Bellman-Ford、Floyd 的结果必须完全一致。

```python
import random
import sys
import heapq

# dijkstra / bellman_ford / floyd 三个函数实现见前文，此处略

def gen_random_graph(n, edge_prob=0.3, max_w=20, allow_neg=False):
    """生成随机图，返回邻接表 + 邻接矩阵 + 边列表。"""
    INF = sys.maxsize
    adj = [[] for _ in range(n)]
    mat = [[INF] * n for _ in range(n)]
    edges = []
    for i in range(n):
        mat[i][i] = 0
        for j in range(n):
            if i != j and random.random() < edge_prob:
                w = random.randint(1, max_w) if not allow_neg else random.randint(-5, max_w)
                adj[i].append((j, w))
                mat[i][j] = w
                edges.append((i, j, w))
    return adj, mat, edges

def checker(test_times=500, max_n=20):
    """非负权随机图上，三种算法结果必须一致。"""
    for _ in range(test_times):
        n = random.randint(2, max_n)
        adj, mat, edges = gen_random_graph(n, allow_neg=False)
        src = random.randint(0, n - 1)

        d1 = dijkstra(adj, n, src)
        d2 = bellman_ford(edges, n, src)
        d3 = floyd(mat, n)[src]

        if d1 != d2 or d2 != d3:
            print(f"不一致！n={n}, src={src}")
            print(f"Dijkstra:     {d1}")
            print(f"Bellman-Ford: {d2}")
            print(f"Floyd:        {d3}")
            return False
    print("对数器验证通过！三种算法在非负权图上结果一致。")
    return True

checker()
```

**对数器的价值**：

- **跨算法交叉验证**：三个独立实现的算法在同一组数据上结果一致，比单算法对标准库更有说服力。
- **覆盖负权边界**：把 `allow_neg=True`，可以单独验证 Bellman-Ford 和 Floyd 一致（Dijkstra 不参与，因为它在负权图上不正确）。
- **暴露隐蔽 bug**：堆优化 Dijkstra 最常见的 bug 是漏写 `if d > dist[u]: continue`，导致用过期距离松弛邻居——对数器能在几秒内抓出。

**对高手的启发**：对数器思想不止用于排序——任何"有多个独立实现可对照"的算法都能用。写完三种最短路径？互相对照。写完红黑树和跳表？各自对照标准库。这是"工程化验证"的思维，比"理论上推导正确"更可靠。

### 3. 教科书做法 vs 生产做法

| 场景 | 教科书做法 | 生产做法 | 原因 |
|---|---|---|---|
| 路网导航 | Dijkstra | A\* + 分层图（Highway Hierarchies） | A\* 有方向性，分层图预处理加速 |
| 网络路由（OSPF） | Dijkstra | Dijkstra（直接用） | 协议规定，且链路权值非负 |
| 套汇检测 | Bellman-Ford | Bellman-Ford（直接用） | 负环检测就是它的本职 |
| 小图全源（V≤500） | Floyd | Floyd（直接用） | 代码简短，速度够快 |
| 大图全源（V上千） | Floyd | N 次 Dijkstra（非负权） | $O(V^3)$ 不可接受 |

**生产里 Dijkstra 的常见加速**：

1. **双向 Dijkstra**：从源点和终点同时跑 Dijkstra，中间相遇，能少探索一半点。
2. **分层图**：把路网按"高速/省道/县道"分层，远距离只在高速层跑，最后一公里切到县道层。
3. **Contraction Hierarchies**：预处理把重要点提升到上层，查询时只走上层。这些加速让 Dijkstra 在大陆级路网上的查询降到毫秒级。

### 4. 真实工程坑

**坑一：Dijkstra 用在有负权边的图上，结果错。**

```python
# 图：s→a 权 5, a→b 权 -10, s→b 权 2
# 正确 dist[b] = -5（s→a→b）
# Dijkstra 算成 dist[b] = 2（b 比 a 早确定，错过负权绕回）
```

症状：在带负权的图上跑 Dijkstra，结果偏大。根因：Dijkstra 的"确定就锁死"前提依赖非负权。修复：换 Bellman-Ford 或 SPFA。

**坑二：Bellman-Ford 漏写 `dist[u] != INF` 检查，松弛了不可达点。**

```python
# 错误写法
for u, v, w in edges:
    if dist[u] + w < dist[v]:    # dist[u] 可能是 INF
        dist[v] = dist[u] + w
# dist[u] = INF 时，INF + 负数仍是合法大数，会错误更新 dist[v]
```

症状：不可达点被算成"可达但距离很大"。根因：`INF + 负数` 不再是 INF。修复：加 `dist[u] != INF` 判断，或用 `float('inf')`（Python 里 `inf + 负数 = inf`，但仍建议显式判断以防 C++/Java 溢出）。

**坑三：Floyd 把 k 写在最内层，结果错。**

```python
# 错误写法
for i in range(n):
    for j in range(n):
        for k in range(n):    # k 在最内层，错！
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

症状：算出来的距离偏大。根因：`k` 在内层会用到本轮还没更新完的 `dist[i][k]` 和 `dist[k][j]`，破坏了动态规划的状态递进。修复：`k` 必须在最外层。

**坑四：SPFA 在被故意构造的数据上退化到 $O(VE)$ 超时。**

症状：随机图上 SPFA 飞快，但在 NOI 等竞赛数据上 TLE。根因：SPFA 最坏 $O(VE)$，存在专门卡 SPFA 的数据。修复：除非确认数据随机，否则用 Dijkstra（非负权）或 Bellman-Ford（负权）。

### 5. 面试高频题：743 网络延迟时间 + 787 K 站中转

#### 题一：LeetCode 743 网络延迟时间

**题意**：给 $n$ 个节点、有向边 `times[i] = [u, v, w]`，求从节点 $k$ 发出信号到所有节点都收到的最短时间。若有节点不可达返回 -1。

**分析**：典型的单源最短路径，边权（时间）非负，用 Dijkstra。结果是 `max(dist)`（最远的节点收到的时间就是总延迟），有 `dist == INF` 则返回 -1。

```python
import heapq
import sys

def networkDelayTime(times, n, k):
    graph = [[] for _ in range(n + 1)]
    for u, v, w in times:
        graph[u].append((v, w))

    INF = sys.maxsize
    dist = [INF] * (n + 1)
    dist[k] = 0
    heap = [(0, k)]
    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]:
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(heap, (nd, v))

    ans = max(dist[1:])              # 1-indexed，跳过 dist[0]
    return ans if ans != INF else -1
```

**面试追问"为什么不用 BFS"**：BFS 只能求无权图最短路径（每条边权当作 1）。本题边权不同，必须用 Dijkstra。如果面试官追问"边权都是 1 怎么办"，回答"退化为 BFS，更简单"。

#### 题二：LeetCode 787 K 站中转内最便宜的航班

**题意**：$n$ 个城市，`flights[i] = [u, v, w]` 是从 $u$ 到 $v$ 价格 $w$。从 `src` 出发到 `dst`，最多经过 $k$ 个中转（即最多 $k+1$ 条边），求最便宜价格，不可达返回 -1。

**分析**：这是带"边数限制"的最短路径。Dijkstra 的"确定就锁死"会出问题——可能少边数路径价格高，多边数路径价格低，先确定的不是最优。用 Bellman-Ford 改造：松弛 $k+1$ 轮，每轮记录"恰好用 $i$ 条边"的最短距离。

```python
import sys

def findCheapestPrice(n, flights, src, dst, k):
    INF = sys.maxsize
    dist = [INF] * n
    dist[src] = 0

    for _ in range(k + 1):                # 最多 k+1 条边
        backup = dist[:]                  # 关键：用上一轮结果松弛，避免本轮连锁更新
        updated = False
        for u, v, w in flights:
            if backup[u] != INF and backup[u] + w < dist[v]:
                dist[v] = backup[u] + w
                updated = True
        if not updated:
            break

    return dist[dst] if dist[dst] != INF else -1
```

**关键一行 `backup = dist[:]`**：必须用上一轮的 `dist` 松弛本轮，否则本轮前面的更新会影响后面的松弛，相当于一轮里走了多条边，破坏了"恰好 $i$ 条边"的语义。这是这道题最容易踩的坑。

**面试官追问"为什么不能用 Dijkstra"**：Dijkstra 不限制边数，可能算出"价格最低但中转超 $k$"的路径。带边数限制的最短路径是 Bellman-Ford 的主场。可以提到"分层图最短路"——把每个点拆成 $k+2$ 层（按已用边数），层内跑 Dijkstra，但工程上 Bellman-Ford 改造更直接。

## 四、速查与自测

### 速查表：最短路径算法五指标对比

| 算法 | 时间 | 空间 | 适用边权 | 单源/全源 | 负环检测 |
|---|---|---|---|---|---|
| Dijkstra（堆优化） | $O(E \log V)$ | $O(V)$ | 非负权 | 单源 | 不能 |
| Bellman-Ford | $O(VE)$ | $O(V)$ | 含负权 | 单源 | 能 |
| SPFA | 平均 $O(kE)$，最坏 $O(VE)$ | $O(V)$ | 含负权 | 单源 | 能 |
| Floyd | $O(V^3)$ | $O(V^2)$ | 含负权（无负环） | 全源 | 能 |
| A\* | 依赖 $h$ | $O(V)$ | 非负权 | 单源单终点 | 不能 |

**选型决策树**：

```
有负权边吗？
├─ 没有 → 单源还是全源？
│        ├─ 单源 → Dijkstra（堆优化）
│        └─ 全源 → V≤500 用 Floyd，否则 N 次 Dijkstra
└─ 有 → 有负环吗？
         ├─ 要检测负环 → Bellman-Ford
         ├─ 数据随机 → SPFA
         └─ 数据可能被卡 → Bellman-Ford
```

### 自测三问

**问题一：** 为什么 Dijkstra 不能处理负权边？请用一个具体反例说明，并解释"确定就锁死"在负权边面前为什么失效。

**参考答案：** Dijkstra的核心是"每次从未确定点中选距离最小的，确定下来就不再动它"。这个"确定就锁死"依赖非负权：未来任何更短路径 $s \to \dots \to v \to u$，因为边权非负，$d(v) \ge d(u)$，所以 $d(v) + w(v,u) \ge d(u)$，不可能更短。反例：$s \to a$ 权 5，$a \to b$ 权 -10，$s \to b$ 权 2。Dijkstra 先确定 $b$（d=2），再确定 $a$（d=5），但 $a \to b$ 让 $b$ 实际应为 -5。$b$ 已被锁死，无法更新。根因是负权边让"晚确定的点绕回早确定的点"可能更短，破坏了贪心的最优性。

**问题二：** Bellman-Ford 为什么松弛 V-1 轮就够？再多松弛一轮还能更新说明什么？

**参考答案：** 无负环时，最短路径最多 V-1 条边——再长就重复经过点形成环，去掉非负环后路径只会更短。所以 V-1 轮松弛足以让最短路径的"信息"从源点传到最远的点。如果第 V 轮还能更新，说明存在一条长 V 条边的"更短路径"，这只可能是负环——绕一圈距离变负，可以无限绕。这就是 Bellman-Ford 检测负环的原理：再松弛一轮，能更新即有负环。

**问题三：** Floyd 为什么 `k` 必须在最外层？把 `k` 放最内层会怎样？

**参考答案：** Floyd 的状态定义是 `dist[i][j]` 表示"只允许经过中转点 $0, 1, \dots, k$ 时 $i \to j$ 的最短距离"。`k` 在最外层表示状态递进——每一轮允许经过的中转点多一个。如果 `k` 在最内层，对同一对 $(i,j)$ 会用到本轮还没更新完的 `dist[i][k]` 和 `dist[k][j]`，相当于"用本轮还在变化的中转结果再中转"，破坏了 DP 的状态依赖。结果会偏大（少算了一些路径），甚至出错。**`k` 在最外层是 Floyd 正确性的前提，不是性能优化**。

### 算法思想 × 生活迁移

最短路径家族的核心思想是**"问题特性决定算法选择"**——边权是否非负、要单源还是全源、有无边数限制，每多一个约束就换一个算法。

**迁移一：出差选路线 = 带约束的最短路径。** 从北京到上海，"最快"是 Dijkstra（边权是时间，非负）；"最便宜且最多中转一次"是 LeetCode 787 的带边数限制版（Bellman-Ford 改造）；"任意两城市互飞最便宜"是 Floyd（全源）。生活中的"选路线"看似一个问题，加不同约束就分裂成不同算法——和工程里"加一个约束换一个解法"的逻辑一致。

**迁移二：职业规划 = 启发式搜索。** A\* 比 Dijkstra 快，因为加了"到终点的估计距离"作为引导。职业规划也一样——纯"广撒网投简历"是 Dijkstra（每个机会平等评估），"先选大方向再投"是 A\*（加 $h(v)$ 引导）。$h(v)$ 越准（对行业的判断越准），探索的无效路径越少。但 $h(v)$ 必须可采纳（不能高估实际距离）——类比职业规划里"目标不能定得离实际太远"，否则会误导决策。

**迁移三：套汇检测 = 负环检测。** Bellman-Ford 检测负环的思路——"V-1 轮后还能松弛就说明有负环"——可以迁移到"系统里是否存在无风险套利"。汇率、利息、补贴，任何"绕一圈净赚"的闭环都可以建模成负环检测。这种"把业务问题翻译成图论问题"的能力，比记住某个算法的实现更重要。

**为什么这些迁移成立：** 最短路径家族的精髓不在某个算法，而在"先看问题特性（边权/源点数/约束），再选算法"的思路。这个思路在工程里处处适用——选数据库（事务/非事务）、选消息队列（顺序/吞吐）、选缓存策略（强一致/最终一致）都是"先看特性再选工具"。算法是死的，"特性驱动选型"的思维方式是活的。
