# 第63章：攻防技术与设备防护

攻防是中级信安工程师的案例题主战场，10-15 分几乎必考。渗透测试流程、Web 攻击手段、防火墙/IDS/IPS/WAF 选型，都是案例题里"给你一段攻击场景让你分析"的素材。对架构师来说，这章的价值在于**纵深防御**——知道攻击者怎么打，才知道每一层防护该放什么设备。这章从攻击侧（渗透+Web 攻击）讲到防御侧（防火墙/IDS/IPS/WAF/蜜罐）再到设备加固，给你一套能用在架构设计里的攻防知识体系。

## 概念

### 这章考什么

- **科目**：中级信安工程师综合知识+案例分析高频
- **考频**：中级信安约 10-15 分；攻防是案例题重区
- **分值**：综合知识 6-8 分+案例分析 4-8 分

### 学习策略

攻击侧记"流程+手段"：渗透测试 PTES 七阶段、Web 攻击 OWASP Top 10 的原理和防御。防御侧记"设备能力边界"：包过滤/状态检测/NGFW 三代防火墙、IDS 旁路检测 vs IPS 在线阻断、WAF 三种引擎。设备加固记"基线配置"清单。案例题常用套路：给一段攻击日志，问你是什么攻击、怎么防、加固建议。

## 原理

### 【中级重点】渗透测试流程（PTES 七阶段）

渗透测试（Penetration Testing）标准流程 PTES 七阶段：

1. **前期交互**：明确范围、目标、授权
2. **情报收集**：被动收集（搜索引擎/WHOIS）+ 主动收集（端口扫描）
3. **威胁建模**：分析攻击面和潜在路径
4. **漏洞分析**：扫描+验证漏洞
5. **漏洞利用**：实施攻击获取权限
6. **后渗透**：提权、横向移动、数据窃取
7. **报告**：漏洞详情+修复建议

简化记忆版：**信息收集→扫描→枚举→漏洞利用→后门→清痕**。注意渗透测试必须**获得书面授权**，未授权即违法。黑白盒测试：白盒（已知全部信息）、黑盒（仅知目标）、灰盒（部分信息）。

### 【中级重点】漏洞分类与评分

| 编号体系 | 维护方 | 说明 |
|---|---|---|
| **CVE** | MITRE | 通用漏洞披露，国际通用编号 |
| **CNNVD** | 国家信息安全漏洞库 | 中国国家漏洞库 |
| **CNVD** | 国家信息安全漏洞共享平台 | 中国共享平台 |

**CVSS 评分**（Common Vulnerability Scoring System）三维度：

- **基础评分**：漏洞本身属性（攻击向量/复杂度/认证/影响），固定值
- **时间评分**：随时间变化（利用代码成熟度/补丁 availability）
- **环境评分**：特定环境影响（资产价值/业务场景）

CVSS 0-10 分：0 无风险、0.1-3.9 低、4.0-6.9 中、7.0-8.9 高、9.0-10.0 严重。

**OWASP Top 10**：开放 Web 应用安全项目发布的 Web 应用十大风险，是 Web 安全风向标（2021 版包含注入、失效访问控制、加密失败、不安全设计、安全配置错误、易受攻击组件、认证失败、软件数据完整性失败、日志监控不足、SSRF）。

### 【中级重点】常见 Web 攻击

| 攻击 | 原理 | 防御 |
|---|---|---|
| **SQL 注入** | 拼接恶意 SQL 篡改查询（联合/盲注/时间盲注） | 参数化查询、预编译、最小权限 |
| **XSS** | 注入脚本到页面执行（反射/存储/DOM） | 输出编码、CSP、HttpOnly Cookie |
| **CSRF** | 诱导已登录用户发请求 | CSRF Token、SameSite Cookie |
| **SSRF** | 服务端发起伪造请求访问内网 | URL 白名单、禁止内网访问 |
| **文件上传** | 上传可执行文件获取 webshell | 类型校验、重命名、存非 web 目录 |
| **命令注入** | 拼接系统命令执行 | 不用 shell 调用、参数校验 |
| **反序列化** | 反序列化恶意对象触发代码执行 | 校验类型、白名单、禁用危险类 |
| **XXE** | XML 外部实体读取文件/SSRF | 禁用 DTD、禁用外部实体 |
| **目录遍历** | `../` 越权访问文件 | 路径规范化、限制根目录 |

SQL 注入三种类型：联合查询型（UNION）、盲注（布尔/时间盲注，无回显靠条件判断）、报错型。XSS 三种：反射型（URL 参数触发）、存储型（存数据库，危害最大）、DOM 型（前端 JS 触发）。

### 【中级重点】防火墙三代演进

| 类型 | 检测层 | 原理 | 局限 |
|---|---|---|---|
| **包过滤** | 网络层 | ACL 基于 IP/端口/协议过滤 | 不看状态、不看内容 |
| **状态检测** | 传输层 | 维护会话表，跟踪连接状态 | 不识别应用层 |
| **NGFW 下一代** | 应用层 | DPI 深度包检测+应用识别+IPS | 性能开销大 |

NGFW（Next-Generation Firewall）在状态检测基础上集成应用识别、入侵防御、用户身份识别、SSL 解密、沙箱等。下一代防火墙功能=传统防火墙+IPS+应用控制+URL 过滤+反病毒。

### 【中级重点】IDS 与 IPS

| 维度 | IDS 入侵检测 | IPS 入侵防御 |
|---|---|---|
| 部署 | 旁路（镜像流量） | 在线（串联） |
| 动作 | 检测告警，不阻断 | 检测并阻断 |
| 检测方法 | 特征/异常 | 特征/异常 |
| 影响 | 不影响业务 | 误判会断业务 |

IDS 检测方法：**特征检测**（匹配已知攻击签名，误报低漏报高）、**异常检测**（建立正常基线，偏离即告警，误报高漏报低）。

**IDS 逃逸技术**：分片（拆包绕过）、编码（URL/Unicode 编码绕过特征）、会话拼接（分多次发送绕过单包检测）。这些是高级考点，IPS 需要会话重组来对抗。

### 【中级重点】WAF 与蜜罐

**WAF（Web Application Firewall）** 三种引擎：

- **规则引擎**：正则匹配已知攻击，误报漏报靠规则质量
- **语义引擎**：理解 HTTP 语义，准确率高
- **学习引擎**：基于机器学习正常流量基线，异常即拦

WAF 与 IPS 区别：WAF 专注 Web 应用层（HTTP/HTTPS），IPS 通用网络层。

**蜜罐（Honeypot）**：伪装成脆弱系统诱捕攻击者，收集攻击情报。

| 类型 | 交互度 | 风险 |
|---|---|---|
| **低交互** | 模拟服务，不开放真实系统 | 风险低，但易被识破 |
| **高交互** | 真实系统，可被完全攻破 | 情报多，但可能被当跳板 |

蜜网（Honeynet）是多个蜜罐组成的网络，提供更全面的攻击行为捕获。

### 【中级重点】网络设备加固基线

| 设备 | 加固要点 |
|---|---|
| **路由器/交换机** | 关闭无用服务（Telnet/HTTP/CDP）、改默认口令、SNMPv3 替代 v2c、限制管理 IP、启用日志审计 |
| **防火墙** | 最小开放端口、默认拒绝策略、定期审计规则、管理通道加密 |
| **通用** | 账号最小权限、密码复杂度策略、双因素、补丁及时更新、配置备份 |

SNMP 三个版本：v1/v2c 明文 community、不安全；v3 支持认证加密，推荐。管理协议优先 SSH 替代 Telnet、HTTPS 替代 HTTP。

## 实践

### 【高级衔接】纵深防御架构设计

架构师设计安全架构的核心思路是**纵深防御（Defense in Depth）**——不指望单点挡住所有攻击，而是多层防护层层过滤，攻击者突破一层还有下一层。

**四层防护架构**：

| 层次 | 防护手段 | 应对威胁 |
|---|---|---|
| **网络层** | 防火墙 ACL、IPS、抗 DDoS、网络分段 | 网络攻击、流量型攻击 |
| **主机层** | 主机加固、HIDS、EDR、最小权限 | 主机入侵、提权 |
| **应用层** | WAF、安全编码、身份认证、授权 | Web 攻击、越权 |
| **数据层** | 加密、脱敏、访问审计、备份 | 数据泄露、篡改 |

设计要点：

1. **网络分段**：DMZ 区放对外服务、内网区放业务系统、管理区单独隔离，区间靠防火墙控策略。
2. **南北向+东西向都要防**：传统架构只防南北向（外部进内网），云原生/微服务还要防东西向（服务间横向），用 Service Mesh + mTLS。
3. **检测与响应闭环**：IDS/WAF/EDR/日志分析平台联动，告警进 SOC，形成"检测→分析→响应→加固"闭环。
4. **冗余与容灾**：抗 DDoS 靠流量清洗+CDN 分流，关键服务多活。

纵深防御的反面是"单点依赖"——只靠一道防火墙，一旦绕过（如 0day+钓鱼拿到内网账号）整个系统沦陷，这正是 2021 年 Colonial Pipeline 被勒索的剧本。

### 真题示例

**2023 年·案例分析·攻防(考生回忆)**：某公司 Web 系统被攻击，日志显示攻击者提交了 `id=1 UNION SELECT username,password FROM users--`，导致用户表泄露。

问题：①这是什么攻击？②给出三条防御措施。③若在数据库前部署 WAF，WAF 应具备哪种能力？

**参考答案**：①SQL 注入（联合查询型 UNION）。②防御：参数化查询/预编译语句；数据库账号最小权限（不赋予 DROP/FILE）；输入校验过滤。③WAF 应能识别 SQL 关键字组合（UNION SELECT）和注释符（--），支持语义分析而非仅正则，并支持会话重组防绕过。

## 速查/自测

### 30 秒速查卡

- **PTES 七阶段**：前期交互→情报收集→威胁建模→漏洞分析→漏洞利用→后渗透→报告
- **CVSS 三维度**：基础/时间/环境；≥7.0 高危，≥9.0 严重
- **SQL 注入三类**：联合/盲注(布尔+时间)/报错
- **XSS 三类**：反射/存储/DOM
- **防火墙三代**：包过滤(网络层)→状态检测(传输层)→NGFW(应用层+DPI)
- **IDS vs IPS**：IDS 旁路告警/IPS 在线阻断
- **IDS 检测法**：特征(误报低漏报高)/异常(误报高漏报低)
- **WAF 三引擎**：规则/语义/学习
- **蜜罐两类**：低交互(安全易识破)/高交互(情报多风险高)
- **设备加固**：关 Telnet 用 SSH、关 SNMPv2c 用 v3、最小权限、日志审计

### 防火墙三类型对比

| 类型 | 工作层 | 看状态 | 看内容 | 典型场景 |
|---|---|---|---|---|
| 包过滤 | 网络层 | 否 | 否 | 简单 ACL |
| 状态检测 | 传输层 | 是 | 否 | 传统企业边界 |
| NGFW | 应用层 | 是 | 是(DPI) | 现代企业+应用识别+IPS |

### 自测题 3 道

1. 某系统部署 IDS 后仍被攻击者用 URL 编码绕过特征库，这属于哪种 IDS 逃逸技术？IPS 如何对抗？
2. 简述 XSS 三种类型的区别，存储型 XSS 为什么危害最大？
3. 设计一个电商网站的纵深防御架构，列出网络层/主机层/应用层/数据层各部署哪些设备？
