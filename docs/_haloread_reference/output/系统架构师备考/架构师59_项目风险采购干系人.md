# 第59章：风险采购干系人三件套，八策略合同类型矩阵是命门

风险、采购、干系人三大管理在中级《系统集成项目管理工程师》里每年 8-12 分，案例分析里风险应对策略是必考项。消极四策略+积极四策略、合同类型风险分担、权力-利益矩阵四象限——这三块背熟，案例题有抓手。

## 概念

### 这章考什么
- **科目**：中级综合知识（每年 8-12 题）+ 案例分析
- **分值**：中级约 8-12 分；案例 5-15 分
- **考频**：高频，风险应对策略案例必考
- **难度**：中等，记忆为主，决策树计算是难点

核心类比：**风险管理**是"未雨绸缪"——先识别可能下雨、迷路、断粮的风险，定性看概率影响、定量算损失期望，再决定规避、转移、减轻还是接受；**采购管理**是"买还是造"的抉择——自制vs外购、选什么合同类型，固定总价把风险甩给卖方，成本补偿风险留在买方；**干系人管理**是"理顺人"——先识别谁影响项目、受项目影响，再用权力-利益矩阵分四象限，重点盯住"高权力高利益"的关键人物。

## 原理

### 【中级重点】风险管理：六过程与八策略

风险管理六过程：**规划风险管理 → 识别风险 → 实施定性风险分析 → 实施定量风险分析 → 规划风险应对 → 控制风险**。

**规划风险管理**：定义如何开展风险管理活动。产出风险管理计划，定义风险类别、概率影响矩阵、风险阈值、报告格式、角色职责。

**识别风险**：找出可能影响项目的风险。工具：头脑风暴、Delphi、检查表、SWOT、假设分析、文档审查。产出**风险登记册**。

**风险登记册内容**（高频）：风险列表、风险描述、风险类别、概率、影响、优先级、应对策略、责任人、状态。

**定性风险分析**：按概率和影响排序风险。核心工具是**概率-影响矩阵 P-I Matrix**：

```
              影响
        低    中    高
概 高 │  中    高    极高
率 中 │  低    中    高
   低 │  极低  低    中
```

- 概率和影响各分多档（如 5 档：很低/低/中/高/很高），相乘得风险值。
- 高风险值优先应对，低风险值观察。

**定量风险分析**：用数值量化风险影响。三大工具：

| 工具 | 含义 | 适用 |
|---|---|---|
| **决策树** | 画树形算各分支期望值，选最优 | 多方案决策 |
| **EMV 期望货币值** | EMV = 概率 × 影响（金额），加总各分支 | 量化风险损失 |
| **蒙特卡洛模拟** | 大量随机抽样模拟结果分布 | 复杂项目不确定性 |

**EMV 计算**（高频命门）：每个风险分支的 EMV = 该分支发生概率 × 该分支影响金额，决策点的 EMV = 各分支 EMV 之和（按发生概率加权）。

**例**：决策点 A 方案有 70% 概率赚 100 万、30% 概率亏 20 万；B 方案稳赚 50 万。
- EMV(A) = 0.7×100 + 0.3×(-20) = 70-6 = **64 万**
- EMV(B) = 50 万
- 选 A（64 万 > 50 万）。

**规划风险应对——八策略**（绝对命门）：

**消极风险（威胁）四策略**：

| 策略 | 含义 | 例子 |
|---|---|---|
| **规避 Avoid** | 改计划绕开风险 | 换技术方案、删高风险功能 |
| **转移 Transfer** | 风险转给第三方 | 买保险、外包、固定总价合同 |
| **减轻 Mitigate** | 降概率或降影响 | 加测试、做冗余、备预案 |
| **接受 Accept** | 不主动处理，准备应急 | 被动接受（不作为）或主动接受（设应急储备） |

**积极风险（机会）四策略**：

| 策略 | 含义 | 例子 |
|---|---|---|
| **开拓 Exploit** | 确保机会发生 | 派最强团队、加资源保机会 |
| **分享 Share** | 与第三方共享机会 | 合作开发、合资 |
| **提高 Enhance** | 提概率或提影响 | 增加促成因素 |
| **接受 Accept** | 不主动追求，有机会就抓住 | 顺其自然 |

口诀：**消极规转减轻接受，积极开分提接受**。注意消极和积极都有"接受"——消极是"认了"，积极是"顺手抓住"。

**控制风险**：监控风险状态、评估应对效果、识别新风险、更新风险登记册。

**风险登记册 vs 风险管理计划易错点**：
- 风险管理计划：方法论、角色职责、概率影响矩阵定义——**怎么管风险**。
- 风险登记册：具体风险列表、优先级、应对策略——**具体风险是什么**。

### 【中级重点】采购管理：合同类型与风险分担

采购管理四过程：**规划采购管理 → 实施采购 → 控制采购 → 结束采购**。

**规划采购**：决定**自制还是外购**（Make-or-Buy Analysis）。考虑成本、能力、时间、风险、战略。

**实施采购**：发招标文件、收投标、评标、选卖方、签合同。

**控制采购**：监控合同执行、管理变更、付款、绩效审查。

**结束采购**：合同正式关闭、最终结算、经验教训归档。

**合同类型——三大类与风险分担**（高频命门）：

| 合同类型 | 全称 | 特点 | 买方风险 | 卖方风险 |
|---|---|---|---|---|
| **FFP 固定总价** | Firm Fixed Price | 总价固定，超支卖方担 | **低** | **高** |
| **FPIF 总价+激励** | Fixed Price Incentive Fee | 固定价+绩效激励 | 中 | 中 |
| **CPFF 成本+固定费用** | Cost Plus Fixed Fee | 报销成本+固定酬金 | **高** | **低** |
| **CPIF 成本+激励费用** | Cost Plus Incentive Fee | 报销成本+绩效激励 | 高 | 中 |
| **CPAF 成本+奖励费用** | Cost Plus Award Fee | 报销成本+主观奖励 | 高 | 中 |
| **T&M 工料** | Time and Material | 按工时单价+材料费 | 中 | 中 |

**风险分担规律**：
- **固定总价类**：买方风险低（总价锁死），卖方风险高（成本超支自己担）。买方省心，卖方需精确估算。
- **成本补偿类**：买方风险高（成本无上限），卖方风险低（报销成本）。买方需严密监督，卖方少操心。
- **工料合同 T&M**：介于两者之间，小型、紧急、范围不明的工作常用。

**易错点**：
- "固定总价"对**买方**最有利（风险低），常考。
- "成本补偿"对**卖方**最有利（风险低），但买方监督成本高。
- T&M 兼具按工时和按单价特点，常用于顾问、临时资源。
- 风险高、范围明 → 固定总价；风险低、范围不明 → 成本补偿或 T&M。

### 【中级重点】干系人管理：权力-利益矩阵

干系人管理四过程：**识别干系人 → 规划干系人管理 → 管理干系人参与 → 控制干系人参与**。

**识别干系人**：找出受项目影响或能影响项目的人/组织。工具：干系人分析、专家判断。产出**干系人登记册**（姓名、角色、利益、期望、影响、分类）。

**规划干系人管理——权力-利益矩阵**（高频命门）：

按"权力（影响项目能力）"和"利益（受项目影响程度）"分四象限：

```
            利益
        低          高
权 高 │ 令其满意    重点管理
力 低 │ 监督        随时告知
```

| 象限 | 策略 | 例子 |
|---|---|---|
| **高权力高利益** | **重点管理** | 发起人、关键客户、PM |
| **高权力低利益** | **令其满意** | 高层领导、监管机构 |
| **低权力高利益** | **随时告知** | 最终用户、运维团队 |
| **低权力低利益** | **监督** | 一般外围干系人 |

口诀：**高权高利重点管，高权低利令满意，低权高利告知晓，低权低利只监督**。

**管理干系人参与**：与干系人沟通协作，满足其期望、解决分歧。

**控制干系人参与**：监控干系人关系，调整策略。

## 实践

### 【高级衔接】架构选型风险评估

架构师在风险管理中的独特价值是识别**技术选型风险**。常见架构选型风险：

- **技术成熟度风险**：选了未经验证的新技术，可能踩坑。
- **供应商锁定风险**：选了某厂商专有技术，迁移成本高。
- **性能风险**：选了不适合高并发场景的框架。
- **人才风险**：选了团队不熟悉的技术栈，学习曲线长。
- **合规风险**：选了不符合行业监管的开源协议。

架构师评估选型风险的清单：
1. 该技术在生产环境有多少大规模案例？
2. 社区活跃度和长期维护承诺？
3. 与现有技术栈的兼容性？
4. 出问题时的备选方案（Plan B）？
5. 团队能否在 3 个月内掌握？
6. 许可证是否与商业目标冲突？

**应对策略**：成熟度低 → 减轻（做 POC 验证）；锁定严重 → 规避（选开放标准）；性能不确定 → 减轻（做压测）；人才缺 → 转移（请外部顾问）+ 减轻（培训）。

### 【高级衔接】供应商技术评估清单

架构师评估供应商技术方案的清单：

- 方案是否满足关键质量属性（性能、可用性、安全）？
- 是否符合行业标准和开放接口？
- 总体拥有成本 TCO（含许可、运维、迁移）是否透明？
- 厂商的 SLA 承诺和违约责任？
- 数据可携带性（能否导出、迁移）？
- 是否有同行大规模成功案例？
- 二次开发能力和 API 完整度？

供应商选型走采购流程时，架构师应在评标委员会中代表技术视角，把上述清单作为评标打分维度之一。

### 真题示例

**考法一：风险应对策略**

> 例：（2024年·中级综合知识·风险管理相关，考生回忆）项目团队为应对关键模块可能延期，决定增加一名开发人员并预留 5 天缓冲，这种策略属于（　）。
> A. 规避　B. 转移　C. 减轻　D. 接受

**答题思路**：加人和缓冲是降低风险影响（不消除也不转移），属减轻。答案 **C**。

**考法二：合同类型风险分担**

> 例：（2023年·中级综合知识·采购管理相关，考生回忆）以下合同类型中，对买方风险最低的是（　）。
> A. CPFF 成本加固定费用　B. CPIF 成本加激励费用　C. FFP 固定总价　D. T&M 工料合同

**答题思路**：固定总价买方风险最低（总价锁死）。答案 **C**。

**考法三：EMV 决策树**

> 例：（2024年·中级案例分析·风险管理相关，考生回忆）某方案有 60% 概率成功获利 200 万，40% 概率失败亏 50 万。请用 EMV 判断是否采纳该方案。

**答题思路**：EMV = 0.6×200 + 0.4×(-50) = 120 - 20 = **100 万**。EMV > 0，应采纳。

**考法四：权力-利益矩阵**

> 例：（2023年·中级综合知识·干系人管理相关，考生回忆）项目发起人对项目高度关注且拥有决定项目是否继续的权力，在权力-利益矩阵中应归入（　）象限。
> A. 重点管理　B. 令其满意　C. 随时告知　D. 监督

**答题思路**：高权力高利益 → 重点管理。答案 **A**。

### 案例分析答题公式

风险案例题套路："**识别风险 + 定性定量分析 + 应对策略 + 控制措施**"四段式。

> 例：项目团队识别到"核心开发人员可能离职"的风险，请给出完整的应对方案。

**答题模板**：
1. **风险识别**：核心人员离职属资源风险，已记入风险登记册，描述具体场景。
2. **定性分析**：概率中（核心人员流失概率非零）、影响高（核心知识断层）。在 P-I 矩阵中属高风险，需重点应对。
3. **应对策略**：① **减轻**——做知识文档化、双人备份关键模块、交叉培训；② **转移**——关键岗位签竞业协议、外备咨询资源；③ **接受**——设应急储备，离职后启动紧急招聘。
4. **控制措施**：每月审查风险状态，关注核心人员满意度和离职信号，更新风险登记册。

### 风险应对八策略速查

```
消极（威胁）：规避（绕开）  转移（甩锅）  减轻（降压）  接受（认了）
积极（机会）：开拓（确保）  分享（共享）  提高（促发）  接受（顺手）
```

| FPIF 总价激励 | 中 | 中 | 鼓励卖方超目标 |

### 自测题

**自测 1**（EMV 计算）：某项目方案 A 有 50% 概率赚 300 万、30% 概率赚 100 万、20% 概率亏 80 万；方案 B 稳赚 130 万。请用 EMV 判断选哪个方案。

> 参考答案：
> - EMV(A) = 0.5×300 + 0.3×100 + 0.2×(-80) = 150 + 30 - 16 = **164 万**
> - EMV(B) = 130 万
> - EMV(A) > EMV(B)，选 **方案 A**。

**自测 2**：分别说明消极风险"减轻"和"接受"的含义，并各举一例。为什么小风险常选"接受"？

> 参考答案：**减轻**是降低风险发生概率或影响，如为关键模块加冗余备份、增加测试用例。**接受**是不主动处理风险，被动接受是不作为、设应急储备应对，主动接受是承认风险并准备预案。小风险常选接受，因为应对成本可能高于风险本身损失，且小风险数量多，逐项减轻不现实，不如设一笔应急储备统一应对。

**自测 3**：某项目干系人包括：发起人（高权高利）、运维部门（低权高利）、政府监管机构（高权低利）、外围供应商（低权低利）。请按权力-利益矩阵分别给出管理策略。

> 参考答案：
> - 发起人（高权高利）→ **重点管理**，密切沟通满足期望
> - 运维部门（低权高利）→ **随时告知**，让其了解项目进展和运维需求
> - 政府监管机构（高权低利）→ **令其满意**，确保合规、不触发监管干预
> - 外围供应商（低权低利）→ **监督**，定期关注状态即可

### 一句话小结

风险六过程（**规划-识别-定性-定量-应对-控制**）+ 八策略（**消极规转减接、积极开分提接**）+ EMV 决策树，合同三类型（**FFP 买方低险、CPFF 卖方低险、T&M 中间**），干系人四象限（**高高重点、高低满意、低高告知、低低监督**）——三块背熟，中级 8-12 分稳拿。
