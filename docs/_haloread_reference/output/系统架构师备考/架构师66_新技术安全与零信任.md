# 第66章：新技术安全与零信任

云、大数据、AI、IoT、区块链、5G、零信任——这七块是中级信安工程师 2026 新增考点，6-10 分；对高级架构师更是高频方向。传统边界安全已被云原生打散，零信任成为新主线，这一章几乎是必考。这章把新技术安全风险和防护一次讲透，重点钉死零信任架构（NIST SP 800-207）和云原生零信任实战，配云责任共担、AI 攻击四类型、5G 三场景等速查表，背完能答中级新题，也能撑起高级架构师论文的安全新方向。

## 原理

### 【中级重点】云安全与责任共担

**责任共担模型**（云服务商与租户责任划分）：

| 模式 | 云服务商负责 | 租户负责 |
|---|---|---|
| **IaaS** | 基础设施（物理/网络/虚拟化） | 操作系统及以上（应用/数据/身份） |
| **PaaS** | 基础设施+操作系统+运行时 | 应用代码+数据+身份 |
| **SaaS** | 基础设施+应用+运行时 | 数据+身份+访问配置 |

责任划分原则：越往上层（SaaS），云服务商负责越多；但**数据和身份管理始终是租户责任**。一个常见坑：租户以为用了 SaaS 就不用管安全，结果因配置错误（如 S3 桶公开）泄露数据。

**容器与 K8s 安全**：

- **镜像安全**：基础镜像扫描（CVE）、签名验证、最小化镜像（distroless）
- **运行时隔离**：namespace+cgroup 隔离，避免特权容器（privileged）
- **K8s 安全机制**：
  - **RBAC**（基于角色的访问控制）：集群操作授权
  - **NetworkPolicy**：Pod 间网络流量控制（东西向）
  - **PodSecurityPolicy**（新版 Pod Security Admission）：限制 Pod 特权、挂载
  - **Secret 加密**：etcd 中 Secret 静态加密+KMS 管理

**Serverless 安全**：事件注入（恶意输入触发非预期函数执行）、权限过大（函数 IAM 权限过宽）、依赖漏洞。防护：输入校验、最小 IAM 权限、依赖扫描。

### 【中级重点】大数据安全

| 措施 | 说明 |
|---|---|
| **数据脱敏** | 静态脱敏（落地前替换）/动态脱敏（查询时实时替换）；可逆（保留可逆映射）/不可逆（哈希/截断） |
| **加密** | 传输加密（TLS）/存储加密（TDE）/计算中加密（同态加密 Homomorphic Encryption） |
| **访问控制** | RBAC（基于角色）/ABAC（基于属性，更细粒度） |
| **差分隐私** | 加噪声使个体数据不可识别，保护统计结果可用 |

差分隐私（Differential Privacy）核心：在查询结果加噪声，使攻击者无法判断某个体是否在数据集中，兼顾数据可用性和隐私。同态加密允许在密文上直接计算，无需解密，适合云端外包计算。

### 【中级重点】AI 安全四类攻击

| 攻击类型 | 阶段 | 原理 |
|---|---|---|
| **对抗样本** | 推理阶段 | 构造细微扰动输入使模型误判（白盒有梯度/黑盒无梯度） |
| **数据投毒** | 训练阶段 | 注入恶意样本污染训练集，植入后门 |
| **模型窃取** | 推理阶段 | 反复查询 API 反推模型参数或复制能力 |
| **模型可解释性** | 推理阶段 | 模型决策不透明，难追溯（需 SHAP/LIME 解释） |

**AI 生成内容检测**：AIGC 兴起后，深度伪造（Deepfake）检测、AI 生成文本检测成新方向。防护思路：对抗训练（增强鲁棒性）、模型水印（标识来源）、输入净化、输出审核。

### 【中级重点】IoT 与区块链安全

**IoT 安全**：设备身份认证（证书/PSK）、固件安全（签名校验/防回滚）、轻量级加密（受限于算力，用 ECC 而非 RSA）、边缘节点防护（网关隔离）。

**区块链安全**：

| 风险 | 原理 |
|---|---|
| **51% 攻击** | 掌握超半数算力可双花/篡改 |
| **重入攻击** | 合约调用外部合约时被递归调用，重复提币（DAO 事件） |
| **整数溢出** | 数值运算溢出绕过校验 |
| **时间戳依赖** | 用块时间戳做随机数可被矿工操纵 |
| **私钥管理** | 私钥泄露即资产失窃 |

智能合约审计要点：检查重入（用 transfer 替代 call）、溢出（用 SafeMath）、权限控制、外部调用顺序。

### 【中级重点】5G 安全

**网络切片隔离**：5G 把网络切分成多个虚拟端到端网络，切片间逻辑隔离，安全策略按切片定制。

**三场景安全需求**：

| 场景 | 全称 | 特点 | 安全重点 |
|---|---|---|---|
| **eMBB** | 增强移动宽带 | 大带宽、高吞吐 | 数据加密、隐私 |
| **uRLLC** | 超可靠低时延 | 毫秒时延、99.999% 可靠 | 轻量认证、低开销 |
| **mMTC** | 大规模机器通信 | 海量连接、低功耗 | 设备认证、组认证 |

**SBA 服务化架构安全**：5G 核心网基于服务化架构，网元间用 HTTP/2 通信，安全靠双向 TLS+API 网关授权+网络切片隔离。

### 【中级重点】零信任架构（NIST SP 800-207）

零信任（Zero Trust）核心原则：**永不信任，始终验证**——不区分内外网，每次访问都基于身份+设备+上下文动态授权，最小权限。

**NIST SP 800-207 架构两大平面**：

| 平面 | 组件 | 作用 |
|---|---|---|
| **控制平面** | **PDP 策略决策点** | 评估访问请求，决策是否放行 |
| **数据平面** | **PEP 策略执行点** | 执行决策，放行或拒绝流量 |

PDP（Policy Decision Point）做决策，PEP（Policy Enforcement Point）做执行，两者分离是零信任关键设计。PDP 基于身份+设备状态+行为上下文动态决策，PEP 在资源前拦截每次访问。

**SDP 软件定义边界**：零信任的一种实现，控制器+发起方+接收方三组件，连接前先认证授权，端口默认关闭（黑名单变白名单）。

**持续认证与自适应访问**：不一次性信任，会话期间持续评估风险，异常即降权或断开（如设备状态变化、地理位置突变）。

**零信任 vs VPN 对比**：

| 维度 | VPN | 零信任 |
|---|---|---|
| 信任模型 | 接入即信任 | 持续验证 |
| 访问粒度 | 网络层（整网可达） | 应用层（单应用授权） |
| 横向移动 | 高风险 | 最小权限收敛 |
| 终端要求 | 弱 | 强（合规检查） |
| 部署 | 集中网关 | 分布式 PEP |

## 实践

### 【高级衔接】云原生零信任架构实战

云原生+微服务架构下，零信任落地有标准范式：**Service Mesh + 零信任 + 微服务 mTLS**。

**架构分层**：

1. **身份层**：工作负载身份（SPIFFE/SPIRE）替代 IP，每个服务有可验证身份证书。
2. **传输层**：Service Mesh（Istio/Linkerd）自动给服务间通信加 mTLS（双向 TLS），无需改业务代码。这是云原生零信任的基础——服务间默认加密+双向认证。
3. **授权层**：网格内 AuthorizationPolicy 按服务身份做细粒度访问控制（A 能调 B，C 不能调 B），实现东西向流量最小权限。
4. **入口层**：API 网关做用户身份认证（OIDC/OAuth2）+ 设备合规检查，外部请求先过网关认证，内部服务间靠 mTLS。
5. **观测层**：全链路日志+审计+异常检测，支撑持续认证和自适应访问。

**对比传统方案**：

| 维度 | 传统微服务 | 云原生零信任 |
|---|---|---|
| 服务间认证 | 无或 IP 白名单 | mTLS 双向证书 |
| 访问控制 | 网络层 | 服务身份层（细粒度） |
| 加密 | 可选 | 默认强制 |
| 横向移动 | 风险高 | 最小权限收敛 |

落地建议：渐进式部署——先在非核心服务试点 Service Mesh 开 mTLS，验证无性能问题后全量推广；AuthorizationPolicy 从宽到严逐步收紧；旧系统用 Sidecar 模式接入，避免大改。常见坑：mTLS 性能开销（CPU 加解密），需评估容量；证书轮换自动化（SPIRE 自动签发短时证书）避免运维噩梦。

### 真题示例

**2024 年·综合知识·零信任(考生回忆)**：关于零信任架构，下列说法错误的是？
A. 零信任不区分内外网，所有访问都需验证
B. PDP 负责策略决策，PEP 负责策略执行
C. 零信任架构下，用户接入 VPN 后即可访问所有内网资源
D. 零信任支持持续认证和自适应访问

**答案**：C。"接入 VPN 后访问所有内网资源"是传统 VPN 的信任模型，正是零信任要打破的。零信任按应用粒度授权，最小权限，且持续验证。A/B/D 均正确。

## 速查/自测

### 30 秒速查卡

- **云责任共担**：IaaS/PaaS/SaaS 服务商负责递增，数据+身份始终租户负责
- **K8s 四机制**：RBAC/NetworkPolicy/PodSecurityPolicy/Secret 加密
- **大数据**：脱敏(静态/动态/可逆/不可逆)+加密(传输/存储/同态)+访问控制(RBAC/ABAC)+差分隐私
- **AI 攻击四类**：对抗样本(推理)/数据投毒(训练)/模型窃取(API 反推)/可解释性
- **区块链**：51%攻击/重入/整数溢出/时间戳依赖/私钥管理
- **5G 三场景**：eMBB(宽带)/uRLLC(低时延)/mMTC(海量)
- **零信任核心**：永不信任始终验证，控制平面 PDP+数据平面 PEP
- **零信任 vs VPN**：持续验证/应用粒度/最小权限/强终端
