# 第61章：信安基础与法规标准

中级「信息安全工程师」的第一关，是先把基础概念和法规标准钉死。这章在信安工程师综合知识里属于"必拿分"——CIA 三要素、等保 2.0、六部信安法律几乎年年考，背熟就是 8-12 分到手。对考高级架构师的你来说，这些知识是设计任何系统时绕不开的合规底线：系统架构不满足等保要求、不对应数据分类分级，方案写得再漂亮也过不了评审。这章把基础概念、法规、标准一次串清，给你一份能直接背、能直接用的清单。

## 概念

### 这章考什么

- **科目**：中级信安工程师综合知识必考；高级架构师综合知识偶现
- **考频**：中级信安基础+法规约 8-12 分；高级架构师 1-2 分
- **分值**：综合知识单选 8-12 分（基础概念 4-6 分+法规标准 4-6 分）
- **难度**：★★（CIA/等保背熟即拿分；法规部分条文多而细，需对比记忆）

### 学习路线建议

先把 CIA 三要素和五类扩展属性背成肌肉记忆，再记 OSI 安全体系的五服务八机制（这张表是中级高频考点）。法规部分用"六部法律一张表"对比记忆，重点抓每部法的"特色条款"（如网安法的等保制度、数安法的数据分类分级、个保法的同意规则、密码法的三类密码）。标准部分把等保 2.0 五级和 ISO 27001/27002 区分清楚即可。

## 原理

### 【中级重点】信息安全 CIA 三要素与扩展属性

信息安全的目标用一组属性概括，前三个是国际通用的核心三件套（CIA Triad），后几个是国内外等保体系强调的延伸：

| 属性 | 含义 | 对应威胁 | 对应手段 |
|---|---|---|---|
| **机密性（Confidentiality）** | 信息不被未授权获取 | 窃听、泄露 | 加密、访问控制 |
| **完整性（Integrity）** | 信息不被未授权篡改 | 篡改、伪造 | 哈希、数字签名 |
| **可用性（Availability）** | 合法用户能及时访问 | DoS 攻击、破坏 | 冗余、备份、抗 DDoS |
| **可认证性（Authenticity）** | 确认实体身份真实 | 仿冒、伪造 | 身份认证、数字证书 |
| **抗抵赖性（Non-repudiation）** | 行为事后不能否认 | 否认操作 | 数字签名、审计日志 |
| **可控性（Controllability）** | 对信息流向可控制、可审计 | 越权传播 | 审计、内容过滤 |

记忆口诀：**"机完可、可抗可"**——机密性、完整性、可用性、可认证性、抗抵赖性、可控性。前三项 CIA 是国际通用的三要素，后三项是国内等保体系强调的扩展。考题常把"可控性"和"不可抵赖性"混在一起考，注意"抗抵赖"靠数字签名，"可控"靠审计过滤。

### 【中级重点】安全威胁分类

安全威胁可以从多个维度切分，中级常考分类口径：

- **按来源**：自然威胁（地震、雷击、电磁辐射）/ 人为威胁（黑客、内鬼、误操作）
- **按主观意图**：故意攻击 / 过失行为
- **按攻击方式**：主动攻击（篡改、伪造、重放、拒绝服务）/ 被动攻击（窃听、流量分析）

主动攻击会改变系统状态或数据，可检测但难防范；被动攻击不改数据，难检测但可防范（靠加密）。这是高频考点：被动攻击对应机密性威胁，主动攻击对应完整性/可用性威胁。

### 【中级重点】OSI 安全体系五类安全服务与八类安全机制

OSI 安全体系结构（ITU-T X.800 / ISO 7498-2）规定了五类安全服务和八类安全机制，是中级信安经典考点：

**五类安全服务**：

| 服务 | 作用 |
|---|---|
| **认证（鉴别）服务** | 实体身份确认（对等实体认证、数据源认证） |
| **访问控制服务** | 防止未授权使用资源 |
| **数据机密性服务** | 防止数据被未授权获取（连接/无连接/选择字段） |
| **数据完整性服务** | 防止数据被篡改（带恢复/不带恢复/选择字段） |
| **抗抵赖服务** | 防止发送方/接收方否认（带交付证明） |

**八类安全机制**：加密机制、数字签名机制、访问控制机制、数据完整性机制、认证交换机制、流量填充机制、路由控制机制、公证机制。

口诀：**"加签访完、认流路公"**——加密、数字签名、访问控制、数据完整性、认证交换、流量填充、路由控制、公证。前四个是"特定安全机制"，后四个加上前四个共同构成"普遍安全机制"。考题常考"哪种服务对应哪种机制"，如抗抵赖服务主要靠数字签名+公证机制。

### 【中级重点】六部信安相关法律法规

| 法律法规 | 施行时间 | 核心要点 |
|---|---|---|
| **网络安全法** | 2017.06.01 | 等级保护制度、网络运营者义务、关键信息基础设施保护、个人信息保护原则 |
| **密码法** | 2020.01.01 | 密码分核心密码、普通密码、商用密码三类；商用密码实行市场化管理 |
| **数据安全法** | 2021.09.01 | 数据分类分级保护、重要数据保护、跨境数据传输安全评估 |
| **个人信息保护法** | 2021.11.01 | 处理原则（合法正当诚信）、同意规则、敏感个人信息、跨境传输 |
| **电子签名法** | 2005.04.01（2019 修正） | 可靠电子签名条件、与手写签名同等法律效力 |
| **关键信息基础设施安全保护条例** | 2021.09.01 | 关基认定、保护要求、年度演练、供应链安全 |

六部法律的特色条款要记牢：

- **网络安全法**：确立等级保护制度为基本制度，网络运营者承担安全保护义务，关基运营者实行重点保护。
- **数据安全法**：建立数据分类分级保护制度，"重要数据"实行目录管理，跨境传输需安全评估。
- **个人信息保护法**：处理个人信息须告知并取得同意（敏感信息须单独同意），遵循最小必要原则，跨境传输需安全评估/认证/标准合同。
- **密码法**：核心密码保护国家机密级信息，普通密码保护秘密级信息，商用密码保护非国家秘密信息。

### 【中级重点】等级保护 2.0 与标准体系

**等保 2.0 五级**（GB/T 22239-2019）：

| 级别 | 保护对象 | 监管方式 |
|---|---|---|
| **第一级** | 一般系统（自主保护级） | 自主保护 |
| **第二级** | 受到破坏会损害公民法人权益（指导保护级） | 上级指导 |
| **第三级** | 受到破坏会损害社会秩序公共利益（监督保护级） | 监督检查 |
| **第四级** | 受到破坏会损害国家安全（强制保护级） | 强制监督检查 |
| **第五级** | 受到破坏会严重损害国家安全（专控保护级） | 专门管控 |

口诀：**"自指监强专"**——自主、指导、监督、强制、专控。三级是绝大多数政企系统的主流级别，四级对应金融/能源等关基，五级极少见。等保 2.0 相比 1.0 的扩展：覆盖云计算、物联网、移动互联、工业控制、大数据等新技术场景，从"被动防御"转向"主动防御+综合防控"。

**等保 2.0 安全框架**：一个核心（安全计算环境）+ 三重防护（安全通信网络、安全区域边界、安全计算环境）+ 安全管理中心。考题常考"三重防护"指哪三个。

**主要标准**：

| 标准 | 内容 |
|---|---|
| **GB/T 22239-2019** | 等级保护基本要求 |
| **GB/T 25070-2019** | 等级保护设计技术要求 |
| **GB/T 28448-2019** | 等级保护测评要求 |
| **GB/T 20984** | 信息安全风险评估方法 |
| **ISO/IEC 27001** | 信息安全管理体系（ISMS）要求 |
| **ISO/IEC 27002** | 信息安全控制措施实用规则 |

ISO 27001 是"建体系"（管理框架），27002 是"选措施"（控制项清单），二者配套使用。27001 认证是国内企业常做的国际体系认证。

## 实践

### 【高级衔接】架构师视角的安全合规清单

中级考生记住法条即可，架构师要在系统设计阶段把合规要求落到架构上。给你一份设计自检清单：

1. **定级备案先行**：新系统立项即定级，三级以上系统需公安机关备案，定级不准后期整改代价极大。
2. **数据分类分级落地**：架构设计前先做数据资产盘点，按重要程度分级（一般数据/重要数据/核心数据），重要数据存储加密+传输加密+访问审计三件套。
3. **三重防护映射架构**：安全通信网络对应网络分段+VPN+链路加密；安全区域边界对应防火墙+入侵检测+准入控制；安全计算环境对应主机加固+身份认证+访问控制。
4. **关基识别**：能源、金融、交通、政务等行业的核心系统按关基保护，需做供应链安全评估、年度攻防演练、容灾备份。
5. **个人信息合规**：用户数据采集前端明示同意、最小必要、存储脱敏、跨境走安全评估或标准合同。
6. **商用密码合规**：关基、三级以上系统鼓励使用商用密码，金融/政务优先选国密算法（SM2/SM3/SM4）。

架构评审时把这份清单作为检查项，能挡掉大部分合规风险。一个常见坑：只做技术方案不备案，结果系统上线后被监管通报"未履行等保义务"。

### 真题示例

**2024 年·综合知识·信息安全基础(考生回忆)**：以下不属于 OSI 安全体系五类安全服务的是？
A. 认证服务  B. 访问控制服务  C. 路由控制服务  D. 抗抵赖服务

**答案**：C。路由控制是八类安全机制之一，不是安全服务。五类服务是认证、访问控制、数据机密性、数据完整性、抗抵赖。这道题考查服务和机制的区分，是高频易错点。

## 速查/自测

### 30 秒速查卡

- **CIA 三要素**：机密性、完整性、可用性（加密/哈希/冗余）
- **扩展三属性**：可认证性、抗抵赖性、可控性
- **被动攻击**：窃听、流量分析（防靠加密）；**主动攻击**：篡改、伪造、重放、DoS（防靠完整性+访问控制）
- **五服务**：认证、访问控制、机密性、完整性、抗抵赖
- **八机制**：加签访完、认流路公
- **等保五级**：自指监强专（自主/指导/监督/强制/专控）
- **三重防护**：安全通信网络、安全区域边界、安全计算环境
- **六部法律**：网安法(2017)、密码法(2020)、数安法(2021.9)、个保法(2021.11)、电子签名法、关基条例
- **密码三类**：核心密码(国家机密)、普通密码(秘密)、商用密码(非国家秘密)

### 口诀

- CIA：**"机完可"**，加签访完、认流路公（八机制）
- 等保：**"自指监强专"**，三级主流、四级关基
- 法律顺序：**"网密数个（人）电关"**（按施行时间）

### 自测题 3 道

1. 某电商系统存储用户手机号和订单数据，按数据安全法应如何分类分级？涉及哪几部法律？
2. OSI 五类安全服务中，"防止发送方否认发送过消息"属于哪类服务？主要靠哪两类机制实现？
3. 等保三级系统被攻击导致 10 万用户数据泄露，依据网络安全法，运营者需承担哪些义务？该级别属于哪种保护级？
