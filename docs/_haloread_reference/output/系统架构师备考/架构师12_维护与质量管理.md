# 维护占软件生命周期 60%，质量管理是护城河

维护和质量管理常被新手忽视——但综合知识每年出 2-3 题，COCOMO 估算、CMMI 5 级、维护四类型都是高频考点，性价比极高。

## 概念

这章考什么：
- **科目**：综合知识（2-3 题）
- **分值**：综合约 3-4 分
- **考频**：中高频

核心类比：软件维护像汽车保养——改正性维护是"坏了修"（修 bug），适应性维护是"换新机油适应新路况"（适应环境变化），完善性维护是"加装导航"（增加功能），预防性维护是"定期检查防患未然"。其中"加装导航"占大头。

## 原理

### 一、软件维护四类型（必背）

| 类型 | 含义 | 占比 |
|---|---|---|
| 改正性维护（Corrective） | 修复交付后发现的 bug | **约 20%** |
| 适应性维护（Adaptive） | 适应环境变化（OS 升级、硬件迁移） | **约 25%** |
| **完善性维护（Perfective）** | 增加新功能、优化性能、改善体验 | **约 50%+**（最大头） |
| 预防性维护（Preventive） | 提前重构防未来问题 | **约 5%** |

记忆：完善性占 50%+ 是高频考点，务必记住"用户用着用着提出新需求"归完善性。

### 二、软件度量与估算

**1. 代码行 LOC（Lines of Code）**
最简单但最不靠谱。受语言、风格、复用影响大。

**2. 功能点分析 FP（Function Point）**
不依赖语言，从用户视角度量。五个信息域：外部输入 EI、外部输出 EO、外部查询 EQ、内部逻辑文件 ILF、外部接口文件 EIF。计算公式：

FP = UFC × TCF

UFC = 未调整功能点计数（5 个信息域加权求和），TCF = 技术复杂度因子（0.65 ~ 1.35）。

**3. COCOMO 模型（Constructive Cost Model）**
Boehm 提出，按代码行估算人月（PM）。三级精度：

| 级别 | 精度 | 适用 |
|---|---|---|
| 基本 COCOMO | 粗略 | 初期估算 |
| 中级 COCOMO | 中等 | 详细设计 |
| 详细 COCOMO | 精细 | 各阶段细化 |

公式（基本）：E = a × (KLOC)^b，E 为人月，a、b 是模型参数。有机型、半分离型、嵌入型三种项目类型，参数不同。

**COCOMO II** 是升级版，考虑复用、开发团队、产品复杂度等更多因子。

### 三、CMMI 能力成熟度模型集成（必背 5 级）

| 级别 | 名称 | 特征 |
|---|---|---|
| 1 | 初始级（Initial） | 混乱，依赖个人英雄 |
| 2 | 已管理级（Managed） | 项目级有过程，可重复 |
| 3 | 已定义级（Defined） | 组织级标准过程 |
| 4 | 量化管理级（Quantitatively Managed） | 用统计与量化技术管理 |
| 5 | 优化级（Optimizing） | 持续改进，关注过程改进 |

记忆口诀：初、已、已、量、优。1→2 项目可重复，2→3 组织标准化，3→4 量化数据驱动，4→5 持续优化。

CMMI 两种表示：阶段式（5 级，如上）+ 连续式（能力等级 0-3，按过程域评估）。

### 四、ISO 9000 质量管理体系

ISO 9000 族核心是"八项质量管理原则"：以顾客为关注焦点、领导作用、全员参与、过程方法、管理的系统方法、持续改进、基于事实的决策、互利的供方关系。

ISO 9001 是质量管理体系要求（可认证），ISO 9000 是基础和术语，ISO 9004 是业绩改进指南。

### 五、配置管理（Configuration Management）

配置管理是质量管理的"骨架"。核心活动：

- **标识配置项 CI**：源代码、文档、数据、环境都标识版本
- **版本控制**：CVS / SVN / Git，记录每次变更
- **变更控制**：变更走 CCB 流程
- **配置审计**：物理审计（CI 是否齐全）+ 功能审计（CI 是否满足需求）
- **状态报告**：报告配置项状态

**基线（Baseline）**：经正式评审冻结的配置项集合，是后续变更的基准。

易混点：
- **配置项 vs 变更**：配置项是受控对象，变更是改配置项的流程。
- **物理审计 vs 功能审计**：物理审计查"在不在、对不对版本"；功能审计查"满不满足需求、测试过没"。
- **CMMI 阶段式 vs 连续式**：阶段式评组织整体成熟度（1-5）；连续式评单个过程域能力（0-3）。

## 实践

**真题示例（2024 年·综合知识·维护类型相关·考生回忆）**：题干描述"用户使用半年后提出增加导出 Excel 功能"，问属于什么维护。答案：完善性维护。踩分词"增加新功能 → 完善性"。

**真题示例（2024 年·综合知识·CMMI 相关·考生回忆）**：问"组织用统计过程控制（SPC）管理项目"对应 CMMI 几级。答案：4 级量化管理级。

**估算题答题公式**：
1. **FP 估算**：识别 5 类信息域计数 → 加权求 UFC → 乘 TCF 得 FP → 用生产率换算人月
2. **COCOMO**：估 KLOC → 选项目类型 → 套公式 E = a×(KLOC)^b → 加调整因子

**论文应用**：论文"项目管理"段写"本项目采用 CMMI 3 级过程，配置管理用 Git + Jenkins CI/CD，所有变更走 CCB 审批，基线每周构建，缺陷密度控制在 0.5/KLOC"，体现过程规范。

**案例分析踩分词**：
- 维护类型题：改正=修 bug、适应=换环境、完善=加功能优化、预防=提前重构
- CMMI 题：1 混乱 / 2 项目级 / 3 组织级 / 4 量化 / 5 优化

## 速查/自测

**30 秒速查卡**：

| 维护类型 | 占比 | 触发 |
|---|---|---|
| 改正性 | 20% | bug |
| 适应性 | 25% | 环境变化 |
| 完善性 | **50%+** | 加功能 |
| 预防性 | 5% | 防患 |

| CMMI 级别 | 关键词 |
|---|---|
| 1 初始 | 混乱 |
| 2 已管理 | 项目级可重复 |
| 3 已定义 | 组织级标准 |
| 4 量化管理 | 统计量化 |
| 5 优化 | 持续改进 |

| 度量 | 用途 |
|---|---|
| LOC | 简单粗估 |
| FP | 用户视角不依赖语言 |
| COCOMO | 代码行估人月 |

**口诀**：
- 维护：改适完预，五成在完善。
- CMMI：初已已量优，五级往上走。
- 配置管理：标版变审报，基线是基准。

**自测题**：
1. 用户用 3 个月提出"系统响应太慢要优化"属于哪种维护？→ 完善性（性能优化）
2. CMMI 哪一级开始用统计量化技术？→ 4 级量化管理级
3. FP 估算五个信息域是什么？→ EI/EO/EQ/ILF/EIF
4. 配置审计的两种类型？→ 物理审计 + 功能审计

**一句话小结**：维护四类型 + CMMI 5 级 + FP/COCOMO 估算——三组数据背死，综合知识稳拿 4 分。
