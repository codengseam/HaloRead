# 孔子人物卡片 Demo 方案

## 一、核心目标

借助「三国杀式英雄卡牌」的设计思路，为 HaloRead 项目新增「历史人物卡片」形态，让中学生/大学生/孩子能快速认识历史人物。本阶段产出**圣贤堂第一张卡片——孔子**的 3 种风格 Demo，验证设计思路，为后续文臣/武官/君臣等阵营扩展打样。

**明确不做**：不做游戏、不集成主站点、不做展厅入口、不做其他人物、不引入前端框架。

## 二、现状分析（Phase 1 探索结论）

### 可复用资产
- **孔子内容已齐全**：`output/孔子传/`（26 篇）+ `output/论语/`（18 篇），含完整生平、事迹、名言及出处（史记·孔子世家、论语各篇、孟子、左传等）
- **Demo 模式有先例**：`demos/` 下已有 4 个自包含 HTML（demo_classic/ink/modern/traditional），是项目认可的 Demo 形态——独立文件、内联 CSS、不进主站点
- **设计语言统一**：Noto Serif SC 字体栈、米白纸 `#faf8f5`、墨色 `#2c2c2c`、棕褐 accent `#8b5a2b`、CSS 变量主题
- **现成卡片组件**：`src/web/static-site/css/style.css` L577-675 的 `.book-card` 可作风格参考（hover 上浮、阴影、圆角），但本阶段不直接复用，因 3 种风格各有差异

### 约束
- 技术栈：原生 HTML/CSS/JS，零依赖（唯一外部库 marked.js，本 Demo 不需要）
- 图片来源：必须用项目规定的文生图 API `https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt={prompt}&image_size={image_size}`
- 移动优先：每个 Demo 必须适配手机（375px 宽不破版）

## 五、3 个 Demo 设计要点

### Demo 1：古典卷轴风 `demos/demo_card_scroll.html`

**视觉**：米白宣纸 + 淡墨晕染 + 朱红印章 + 棕褐边框，最有"史书"气质，与项目 `demo_ink.html` 风格一致。

**布局**（移动优先单列）：
1. 顶部：朱红印章「圣贤堂」+ 大字「孔子」+ 副标题「孔丘 · 字仲尼」
2. 中部：人物画像（卷轴框，圆角矩形，米白渐变底，下方落款"先师孔子行教像"）
3. 画像下：朝代/国家/生卒 三段标签（朱红边框小标签）
4. 评分区：六维用**竖向水墨条**（每条左侧维度名+右侧分值，中间墨色填充条），配维度注解
5. 事迹区：竖排小字，每条标题朱红、正文墨色
6. 名言区：朱红印章点缀，每句名言独立成块，附出处
7. 底部：简介一段 + 朱红落款印章

**配色**：`--paper:#faf8f5` `--ink:#2c2c2c` `--vermilion:#a13c2c`（朱红）`--bronze:#8b5a2b` `--muted:#7a7a7a`

**字体**：Noto Serif SC，标题用大字号 + letter-spacing

**手机适配**：375px 单列，画像宽度 80vw，评分条全宽。

### Demo 2：三国杀卡牌风 `demos/demo_card_sanguosha.html`

**视觉**：最接近用户提到的"三国杀网页游戏"原型。深棕底 + 金色边框 + 阵营色徽章 + 六边形雷达图。

**布局**：
- 桌面端（≥768px）：横向卡牌，左 40% 人物立绘（满高），右 60% 信息区
- 移动端（<768px）：竖向堆叠，画像在上（高度 40vh），信息区在下

**信息区结构**：
1. 顶部：姓名「孔子」+ 字号「字仲尼」+ 阵营徽章「圣贤堂」（金色徽章）+ 朝代/国家标签
2. 中部：**六边形雷达图**（SVG 绘制，6 个轴对应六维，金色填充半透明）—— 三国杀风的标志元素
3. 下部：台词区（1 句最经典台词，大字引用，金色引号）+ 事迹区（2-3 条短事迹，带圆点）
4. 底部：生卒 + 简介

**配色**：`--bg:#3d2817`（深棕）`--gold:#c9a961`（金）`--paper:#f5ecd9`（米白文字）`--accent:#a13c2c`（朱红点缀）

**雷达图**：纯 SVG，6 边形网格 + 数据多边形，金色描边 + 半透明填充，每轴标注维度名与分值。

**手机适配**：竖向堆叠，雷达图缩到 280px 宽居中，画像改用 `object-fit: cover` 高度 280px。

### Demo 3：现代极简风 `demos/demo_card_modern.html`

**视觉**：大幅留白、扁平、条形评分、移动端原生友好。最易读，但"历史感"较弱。

**布局**（单列居中，max-width 480px）：
1. 顶部：圆形头像（80px）+ 姓名 + 阵营/朝代/国家标签（灰阶胶囊标签）
2. 简介段：一句 bio
3. 评分区：六维**横向条形图**（每条左侧维度名 80px + 中间灰色轨道 + 蓝色填充 + 右侧分值），下方小字注解
4. 名言区：大字引用（serif 大号），3 句竖排，每句下小字出处
5. 事迹区：时间线样式（左侧圆点 + 竖线 + 右侧标题+正文）

**配色**：`--bg:#fff` `--ink:#2c2c2c` `--accent:#8b5a2b` `--bar:#e5e0d8`（轨道）`--fill:#8b5a2b`（填充）`--muted:#7a7a7a`

**字体**：标题 Noto Serif SC，正文 Noto Sans SC（无衬线，现代感）

**手机适配**：本就是移动优先设计，375px 完美适配。

## 六、图片生成方案

3 个 Demo 共用同一张孔子画像 URL（避免重复生成）：

```
https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt=中国古代圣贤孔子画像，先师孔子行教像风格，长髯老者，身着深衣大袖，手持笏板，水墨工笔人物画，米白宣纸背景，端庄肃穆，历史人物肖像，传统国画&image_size=portrait_4_3
```

- `image_size=portrait_4_3`：竖向 4:3，适合人物半身像
- prompt 已 URL 编码空格（实际写入时需 `encodeURIComponent`）
- 3 个 Demo 各自硬编码该 URL 到 `FIGURE.portrait`

## 九、假设与决策

| 项 | 决策 | 理由 |
|---|---|---|
| 图片来源 | 文生图 API 生成"先师孔子行教像"风格画像 | 项目图片指南强制要求；真实历史画像有版权/获取问题；AI 生成可控制风格统一 |
| 数据组织 | 每个 Demo 硬编码同一份 FIGURE 对象 | 保持自包含（沿用 demos/ 模式），3 个文件独立可打开；后续若集成主站点再抽共享 JSON |
| 评分图形式 | 卷轴风用竖向水墨条 / 三国杀用六边形雷达图 / 极简用横向条形图 | 三种图各匹配风格气质，且能展示同一份数据的不同视觉表达 |
| 评分上限 | 100 分制 | 三国杀惯例，且便于横向对比 |
| 事迹条数 | 4 条 | 卡片信息密度适中，多了挤、少了空 |
| 名言条数 | 3 句 | 同上 |
| 是否做"圣贤堂"入口 | 不做 | 用户明确"先制作圣贤堂的第一个卡片"，入口待后续 |
| 是否集成主站点 | 不做 | 本阶段是 Demo 选型，用户选定风格后再集成 |

## 十、验证步骤

每个 Demo 完成后逐一验证：

1. **浏览器渲染**：用 RunCommand 起 `python -m http.server` 在 demos/ 目录，浏览器打开 3 个 HTML，确认渲染正常
2. **手机适配**：Chrome DevTools 切 iPhone SE (375px) / iPhone 14 (390px) / iPad (768px)，确认不破版、无横向滚动
3. **图片加载**：确认文生图 API URL 能返回图片（非 404/非空）
4. **评分图正确**：
   - Demo 1：6 条水墨条长度与分值成比例（98% → 98% 宽）
   - Demo 2：雷达图 6 个顶点位置与分值对应（影响=100 应触达外圈）
   - Demo 3：6 条横向条形图填充比例正确
5. **内容核对**：4 条事迹 + 3 句名言 + 6 维分值与方案一致，出处可溯源到 `output/论语/` 和 `output/孔子传/`
6. **自包含**：断网打开 HTML（除图片外）应正常渲染，无外部 JS/CSS 依赖
