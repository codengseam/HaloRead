## 洛克菲勒小说专栏生成（2026-07-01，fiction 桶首次落地 + 三专家评审 + BUG-040）

### 触发问题
用户要求从第一性原理出发，创作一部关于约翰·D·洛克菲勒白手起家的财富故事小说——**不是历史传记，而是写给 AI 时代 OPC（一人公司/超级个体）创业者看的"硬核商战启示录"**。核心诉求：传递"系统思维/效率极致化/逆周期操作"三大降维启发；用网文/微短剧的"情绪工程学+注意力控制术"手法（黄金三章/钩子/欲扬先抑/扮猪吃虎/金手指/降维打击/掉马甲/麦高芬/伏笔/信息差/定时炸弹/两难/极限拉扯）；引用资料包括罗恩·切尔诺《Titan》、艾达·塔贝尔《标准石油公司史》、艾伦·内文斯、丹尼尔·耶金《石油大博弈》等；单章引用仅放文末；最终生成的提示词、引用资料保留（_前缀）但不展示；严格质检，允许部分夸张和虚构但整体写实，参考《三国演义》"七实三虚"。

### 执行流程
1. **三件套底座**：`_meta.yaml`（archetype=fiction, sort=200）+ `_参考资料.md`（一手史料/纪录片/播客专栏三类）+ `_提示词.md`（含三专家评审汇总与 32 章大纲）。
2. **32 章分卷并行生成**：卷一至卷七 + 附录，用 Task 工具启动 subagent，每个 subagent 指令自包含（占位符填充 + 五段结构 + OPC 翻译桥 + BUG 教训引用）。
3. **文件命名/frontmatter 对齐项目约定**（参考理财课/三国/孔子传）：去掉 event 数字前缀、`book=目录名`、`event` 与文件名严格一致、`sort` 章内 1-based 重置。一次性迁移脚本 `_migrate_rockefeller_filename.py` 处理 32 个文件。
4. **check_book_structure.py --strict 全清**：迁移后 0 P0/P1/P2（迁移前曾报 78 个问题：32 个 book 字段不一致 + 31 个 event 字段不一致 + 15 个 sort 不连续）。
5. **三专家并行评审**（史实核验/小说技法/OPC 财商教育）：三份报告结论均为"通过"（0 P0），共提出 11 项 P1 修订建议，去重后归为三类反复痛点（详见 BUG-040）。
6. **批量修复 11 项 P1**：史实 3 处（汉迪/盖茨/塔贝尔父女）+ OPC 5 处（卷六两章差异化、卷三/卷七补动作、附录换证据）+ 技法 2 类（卷七3章+附录标题加序号、OPC 桥去引用块前缀）。
7. **re-run check_book_structure.py --strict**：仍 0 问题，无回潮。

### 关键教训
1. **fiction 桶"七实三虚"边界要前置约束**（详见 BUG-040）——"实"的部分（人物姓名/身份/年份/篇数/数字事实）必须核对一手史料，"虚"的部分（对话/心理/细节场景）才允许艺术加工。本次 3 处史实错误都是"实"的部分写错：弗兰克·塔贝尔身份（油桶制造商→油井主）、塔贝尔连载篇数（23→19）、递提案人（布朗→盖茨）、银行家姓名（詹姆斯·汉迪→T. P. Handy）。
2. **多 Agent 并行写作必须强制格式约定**——在子 Agent prompt 里写明"二级标题用 `## 一、入戏` 带中文序号、OPC 桥用 `**OPC 翻译桥**：` 加粗段落"，否则多 Agent 各自遵循本地 convention 必然漂移。卷一至卷六遵循一种格式，卷七3章+附录遵循另一种格式——这是 BUG-019（"模块N"前缀）的同类问题，格式约定不能靠默契。
3. **相邻章节 OPC 桥必须差异化**——OPC 翻译桥是 fiction 桶核心创新（把帝国级启示下沉到一人公司量级），但同卷相邻章节 OPC 桥主题相近时，子 Agent 容易逐字复用而非差异化。卷六 1911 拆分令章与 1916 拆分翻倍章 OPC 桥近乎逐字重复（都以"分项之和大于整体"开头，"一个产品值一百，打包卖八十"完全一致）。修复时按专家建议从不同切口切入：1911 章聚焦"被强制拆分时的应对心态与法律策略"，1916 章聚焦"拆分后市场重新定价的财务机制"。
4. **专家团评审是 fiction 桶内容质量的最后防线**——史实核验和 OPC 桥差异化这两类问题无法靠 Python 断言自动检测，必须靠"史实核验+小说技法+OPC 财商教育"三视角并行评审来兜底。这与 narrative 桶的"质检脚本前置+LLM 主观维度后置"分层不同——fiction 桶的"实"部分必须靠史实专家核验，不能交给 LLM 自检。
