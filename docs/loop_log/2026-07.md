## 洛克菲勒小说专栏生成（2026-07-01，fiction 桶首次落地 + 三专家评审 + BUG-040）

用户要求从第一性原理出发，创作一部关于约翰·D·洛克菲勒白手起家的财富故事小说——**不是历史传记，而是写给 AI 时代 OPC（一人公司/超级个体）创业者看的"硬核商战启示录"**。核心诉求：传递"系统思维/效率极致化/逆周期操作"三大降维启发；用网文/微短剧的"情绪工程学+注意力控制术"手法（黄金三章/钩子/欲扬先抑/扮猪吃虎/金手指/降维打击/掉马甲/麦高芬/伏笔/信息差/定时炸弹/两难/极限拉扯）；引用资料包括罗恩·切尔诺《Titan》、艾达·塔贝尔《标准石油公司史》、艾伦·内文斯、丹尼尔·耶金《石油大博弈》等；单章引用仅放文末；最终生成的提示词、引用资料保留（_前缀）但不展示；严格质检，允许部分夸张和虚构但整体写实，参考《三国演义》"七实三虚"。

4. **大规模并行写作批量修复模式（content_quality）**：55 章并行落盘后，4 类问题（参考来源标题/跨章跳转/AI 套话/错别字）都是机械性、跨文件、可正则化的，**写一次性 Python 批量脚本一键修复比逐文件 Edit 高效得多**。本次脚本对 55 文件统一：`## 参考`→`## 参考来源`、正则删跳转、替套话、修错别字。**这是大规模并行生成专栏的标配收尾动作——质检发现批量问题优先写脚本，不要逐文件 Edit。**

## 软考架构师专栏模块 13 扩展（2026-07-01，中级衔接三专项 17 章扩写 + AskUserQuestion 三决策前置 + 高级速览/中级详讲双轨设计）

### 触发问题

用户在已完成的 55 章架构师备考专栏基础上追加需求：「系统集成项目管理、信息安全、网络工程这三章你写的更详细一些吧，可以拆分出多个章节来写，我想先考相关的中级证书，然后再考高级」。即用户策略调整为「先中级后高级」——先考中级三大证书（系统集成项目管理工程师 / 信息安全工程师 / 网络工程师），再考高级架构师。原专栏这三块只占 4 章（架构师07 网络 / 架构师30-31 信安 / 架构师36 项目管理）是高级速览级，远不足以应对中级考试。

### 执行流程

1. **AskUserQuestion 三决策前置**：拆分粒度（5+6+6=17 章 vs 4+5+5=14 章 vs 6+7+7=20 章）/ 原 4 章处理（保留为高级速览+指引 vs 删除 vs 降级附录）/ 分层标注（【中级重点】+【高级衔接】两块分写 vs 行内 [中]/[高] 标签 vs 不标注）。三项均选推荐方案，避免返工。
2. **3 subagent 一次并行**：按三大中级证书分片（A 系统集成项目管理 5 章 / B 信安 6 章 / C 网络 6 章），每个 subagent 指令自包含（含 frontmatter 模板 / 4 段结构 / chapter_sort 纯整数 / 文末参考来源 / 分层标注 / 每章详细内容要点清单）。subagent 直接 Write 落盘，主 agent 只汇总状态。
3. **原 4 章加中级衔接指引**：架构师07/30/31/36 章末「一句话小结」后、「## 参考来源」前插入「### 中级衔接指引」三级标题，指向对应的 56-72 章详讲版，明确「本章是高级架构师视角的速览，中级详讲见第 XX 章」。

### 关键教训

1. **AskUserQuestion 多决策点前置是大规模扩写的返工保险（content_quality）**：本次扩写涉及三个独立决策（拆分粒度/原章处理/分层标注），每个决策都有 2-3 个选项，若直接开干会大概率返工。用 AskUserQuestion 一次性收集三个决策（每题 2-4 选项含推荐项），用户秒选三项推荐方案，零返工。**这是「先中级后高级」「先 X 后 Y」类多层级备考专栏扩写的必备前置——决策点不止一个时必须用 AskUserQuestion 批量收集，不要逐个问。**

2. **高级速览 + 中级详讲双轨设计是层级备考专栏的稳定模式（book_structure）**：不删除原 4 章（保留 1-55 编号体系不破坏），新增 17 章作为中级详讲（56-72），通过章末「中级衔接指引」三级标题双向互通。这种「不破坏原编号 + 新增详讲 + 双向指引」模式可复用于其他多层级备考专栏（如先初级后中级、先中级后高级的注会/CFA 司法考试专栏）。**比「删除原章重写」风险低得多——原章已通过质检且编号体系已固化，删除会引发 chapter_sort 重排连锁反应。**

3. **【中级重点】+【高级衔接】分层标注在 knowledge 4 段结构内的落地（content_quality）**：中级考点用 `### 【中级重点】XXX` 三级标题嵌入 `## 原理` 段（直接可答题的考点详解），高级考点用 `### 【高级衔接】XXX` 三级标题嵌入 `## 实践` 段（架构师视角的论述题论证）。这种分层不破坏 knowledge 4 段二级标题骨架（概念/原理/实践/速查自测），又能清晰区分两个层级的考点。**比行内 [中]/[高] 标签视觉更清晰，比不标注分层更明确。可复用于其他双层级备考专栏。**

4. **17 章一次并行比 55 章分批更成熟（ai_course）**：本次 3 subagent 各负责 5-6 章，一次并行全部完成，无丢失、无返工。相比上次 55 章分 3 批（5+5+3）还遇到 toolcall result missing，本次规模更小、subagent 任务边界更清晰（按证书分片 vs 按模块分片），一次并行更高效。**经验：subagent 任务边界越清晰（一个证书 = 一个 subagent），并行越稳定。**
