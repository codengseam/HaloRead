# 项目写作规则

本规则用于指导 Agent 生成 Markdown 讲书笔记。详细项目背景见 [README.md](./README.md)。

## 一、输出结构

每篇笔记固定五段正文加一段结语，顺序不可调换：

1. **讲事情**：原文中的几个小故事
2. **讲人物**：主角团的人性画像
3. **讲背景**：古人为什么这么做
4. **讲道理**：名家怎么看
5. **问道悟道**：本质与开智

文末加一段**超级简单且深刻的总结**。

## 二、模块规则

### 1. 讲事情

**要做：**

- 用白话小说笔法讲故事，依照原文但不照搬原文。
- 有场景、有对话、有戏剧性转折。
- 史料丰富，但叙述要白话。

**不做：**

- 不先堆大段古文再翻译。
- 不写成干巴巴的流水账。
- 文言文只用于名言、成语典故出处。

### 2. 讲人物

**要做：**

- 聚焦主角团，写人性矛盾和复杂面。
- 用本书之外的其他史料和事迹佐证人物性格。
- 重点写识人、用人、人性、人才。

**不做：**

- 不贴标签。
- 不只用本事件中的表现来定义人物。

### 3. 讲背景

**要做：**

- 补齐前因后果、环境压力、历史机会。
- 解释"古人为什么这么做"。
- 涉及制度、地理、政治结构、社会风气等。

**不做：**

- 不让古人看起来傻。
- 不跳过关键背景直接讲道理。

### 4. 讲道理

**要做：**

- 只引用真正对该典故/人物有明确点评的名家之言。
- 优先引用古代名家对该事件的直接评论，如司马光"臣光曰"、司马迁、贾谊、王安石、苏轼、王夫之、顾炎武等。
- 说明名家为什么这么说、依据是什么、价值在哪里。

**不做：**

- 不引用沾边但未明确评价该事件的名家之言。
- 不编造名家点评。
- 没有明确评语的名家直接跳过，不写"未找到"等占位段落。
- 现代学者引用要谨慎，避免 AI 编造感。

### 5. 问道悟道

**要做：**

- 从具体事件向上提炼本质规律。
- 用古今中外经典和事例佐证。
- 每个洞察都要有史料支撑。
- 目标是让人有"原来如此""大开眼界"之感。

**不做：**

- 不写鸡汤。
- 不为了深刻而深刻。

### 6. 结语

**要做：**

- 用一句话总结全文最核心的本质。
- 简单、深刻、 memorable。

**不做：**

- 不重复前文内容。
- 不写冗长的升华。

## 三、语言风格

**要做：**

- 中文为主，自然口语化。
- 像王立群、易中天、樊登讲书，有节奏、有温度。
- 语气随内容变化：叙事生动、分析冷静、悟道深刻。

**不做：**

- 不中英文混杂（专有名词除外）。
- 不从头到尾一种"端末"语气。
- 不堆砌"我们可以看到""这告诉我们"等 AI 套路句式。
- 不每段结尾都来一句总结升华。

## 四、引用

**要做：**

- 每个关键事实标注来源。
- 格式：简短原文上下文 + 书名/篇名。
- 来源放文末或脚注。
- 优先正史和经典古籍。

**不做：**

- 不编造来源。
- 不堆砌无关引用。

## 五、禁止事项

- 不做 RAG，不向量化，不存储整书。
- 不编造名家点评和名人名言。
- 不过度引用文言文。
- 不出现中英文混杂。
- 不输出 AI 味浓的重复句式。
- 不硬凑"名家点评"占位段落。
