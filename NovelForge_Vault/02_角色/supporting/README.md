# 配角档案目录（supporting）

> 本目录存放每个配角的独立 md 档案，文件名格式 `配角代号_姓名.md`，例如 `sup_01_林师兄.md`。
> 配角与反派区别：不一定是敌人，但必须有「自身的小弧光」，避免工具人。

---

## 配角档案模板（新建文件时复制本节）

```markdown
# 配角档案：____（姓名）

## 一、基本信息
- 姓名：____
- 别称：____
- 身份：____
- 与主角关系：____
- 出场卷/章：____

## 二、性格与功能
- 性格关键词：____
- 在剧情中的功能：____（导师/情报源/情感线/工具人/对比镜像）
- 自身小弧光：____（一条独立于主线的小目标，最终是否达成）

## 三、关系网
- 与主角：____
- 与其他配角：____
- 与反派：____

## 四、语言指纹
（与主角档案同结构，见 style_guide.md 附录 B）

## 五、出场规划
- 出场卷章：____
- 退场/沉寂章：____（如有）

## 六、状态锚点
> 配角的动态状态是否需要单独 JSON？
> - 重要配角（持续出场 10 章以上）：建 `.state/characters/xxx.json`
> - 次要配角：状态写在本 md 末尾即可
```
