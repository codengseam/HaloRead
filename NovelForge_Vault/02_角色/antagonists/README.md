# 反派档案目录（antagonists）

> 本目录存放每个反派的独立 md 档案，文件名格式 `反派代号_姓名.md`，例如 `ant_01_血煞老祖.md`。
> 反派与配角最大区别：必须有**战力曲线**与**被打脸节点**，方便 architect 设计冲突节奏。

---

## 反派档案模板（新建文件时复制本节）

```markdown
# 反派档案：____（姓名）

## 一、基本信息
- 姓名：____
- 别称：____
- 身份：____
- 出场卷/章：____

## 二、战力曲线
| 阶段 | 境界 | 与主角差距 | 主角能否战胜 |
|---|---|---|---|
| 初遇 | ____ | 高 1 大境界 | 否（只能逃） |
| 中段 | ____ | 同境界 | 苦战可胜 |
| 终战 | ____ | 低 1 境界 | 必胜 |

## 三、出场节奏
- 第一次出场：____章（建立威胁）
- 第二次出场：____章（小胜主角，加深仇恨）
- 第三次出场：____章（被主角反杀或被打脸）

## 四、被打脸节点
> 反派必须给主角「打脸」的机会，列出预定的打脸场景。
- 打脸 1：____章，____（事件）
- 打脸 2：____章，____（事件）

## 五、动机与弱点
- 表层动机：____
- 深层动机：____
- 致命弱点：____（主角最终利用的点）

## 六、语言指纹
（与主角档案同结构，见 style_guide.md 附录 B）

## 七、关系网
- 上级：____
- 党羽：____
- 与主角的私怨起点：____
```
