# 文风指南（style_guide）

> 本文件是 NovelForge 的「语言宪法」，polisher Skill 必须逐条比对，writer Skill 必须在生成时内化。
> 双模式（novel / shortform）共用本文件，但执行不同条目。
> 所有禁用词/句式均给出**违规等级**与**修复策略**，禁止只贴黑名单不给方案。

---

## 一、novel 模式 · 文风规范

### 1.1 禁用词表（分级管控）

> 重要原则：**不搞一刀切禁用，区分旁白与对话**。对话中可保留人物语言风格的自然俗语，旁白须克制。

| 词/短语 | 违规等级 | 适用范围 | 修复策略 |
|---|---|---|---|
| 宛如 / 仿佛 / 交织 | 控量 ≤ 2 次/千字 | 全文 | 改为具体动作或白描；超量视为 P3 |
| 首先 / 其次 / 总之 | P2（仅禁旁白） | 旁白 | 旁白禁用，对话中按身份放行 |
| 不可否认 | P2 | 全文 | 删掉，让事实说话 |
| 具有重要意义 | P2 | 全文 | 改为「决定了 ___」或具体影响 |
| 谱写 | P2（保留） | 全文 | 仅限史诗级场景，每卷 ≤ 1 次 |

> P0/P1（绝对禁用，无例外）：
> - 现代网络用语混入古风（如「yyds」「破防」「绝绝子」出现在修仙背景）
> - AI 翻译腔（如「这是 ___ 的存在」「一种 ___ 的感觉」）
> - 视角混乱（同一章内多次切换 POV 且无明确分隔）

### 1.2 禁用句式

- **过度排比**：连续三个以上结构相同的句子堆砌，特别是景物描写。
- **空洞景物**：景物描写不承载情绪/伏笔，纯凑字数。修复：景物必须与角色心境或后续事件挂钩。
- **上帝视角说教**：旁白跳出剧情去「教育读者」。修复：把观点转化为角色台词或事件结果。
- **总结式收尾**：章末用「这一夜，他终于明白了 ___」式总结。修复：留动作或台词作钩子，让读者自己悟。

### 1.3 提倡

- **多动词少形容词**：每段动词占比 ≥ 30%，形容词不超过动词的一半。
- **对话符合身份**：长老说话不能像少年，商人不能像书生。参见 §三 角色语言指纹规范。
- **心理结合生理**：写心理活动必须有生理映射支撑，禁止纯心理独白超过 3 句。参见 §二 心理-生理映射表。
- **场景可视**：每场戏至少有一个「五感锚点」（视觉/听觉/嗅觉/触觉/味觉任一）。

### 1.4 节奏控制

- 章末必须有钩子（悬念/反转/伏笔提示），禁止平推收尾。
- 单章字数控制在 ± 10% 均值（参考 `.state/chapter_length_history.json`）。
- 爽点间隔：每 3 章至少 1 个明确爽点（打脸/突破/获得/逆袭）。

## 二、shortform 模式 · 文风规范

> shortform = 公众号文章，3-6k 字，半历史商业人物，复用 HaloRead fiction 桶「七实三虚」。
> 与 novel 最大区别：**重情绪密度与转发钩子，轻长线伏笔**。

### 2.1 金句密度

- **金句密度 ≥ 1/500 字**：每 500 字至少一句可被单独截图转发的金句。
- 金句标准：①观点反常识 ②句式短促有力 ③有画面感或冲突感。
- 示例：「不是时代选了他，是他把时代的乱流攥成了自己的命。」

### 2.2 转发点

- **每篇 ≥ 2 个情绪高点**，分别位于前 1/3 与后 1/3。
- 高点类型：①命运反转 ②人情冷暖 ③认知冲击 ④画面奇观。
- 高点前后必须有铺垫句，禁止情绪突兀起飞。

### 2.3 情绪曲线闭环

- 开篇情绪 → 中段冲突 → 高潮反转 → 收束回扣开篇。
- 禁止「虎头蛇尾」或「平推到底」。
- 收尾必须回扣开篇意象或金句，形成闭环。

### 2.4 标题钩子（三维度）

| 维度 | 说明 | 示例 |
|---|---|---|
| 信息密度 | 标题要带具体数字/人物/事件，拒绝空泛 | ❌《一个商人的故事》 ✅《32 岁亏光 200 万，三年后他把茶叶卖到欧洲》 |
| 灵魂指向 | 标题要戳中读者某类焦虑或渴望 | 「亏光」「逆袭」「卖到欧洲」戳财务焦虑 |
| 呼应节奏 | 标题与正文结尾要能对得上 | 结尾必须落回「卖到欧洲」这个意象 |

## 三、附录 A · 心理-生理映射表

> 写心理时必须配生理反应。下表是常用映射，可扩展。来源：写作技巧素材库 `03_素材库/writing_techniques.md` 同步维护。

| 心理状态 | 生理映射（任选 1-2 个） |
|---|---|
| 紧张 | 心跳加速 / 手心出汗 / 呼吸变浅 / 喉咙发紧 |
| 愤怒 | 太阳穴跳动 / 拳头攥紧 / 牙关咬合 / 太阳穴血管凸起 |
| 恐惧 | 后背发凉 / 腿软 / 汗毛倒竖 / 胃部抽搐 |
| 得意 | 嘴角上扬 / 眼神发亮 / 步伐变轻 / 手指轻叩 |
| 犹豫 | 手指敲击 / 目光游移 / 咬嘴唇 / 脚尖转方向 |
| 悲伤 | 鼻腔发酸 / 视线模糊 / 喉咙堵 / 肩膀塌 |
| 羞愧 | 耳根发烫 / 脸颊发热 / 不敢对视 / 手指抠桌沿 |
| 决绝 | 呼吸一沉 / 牙关一咬 / 指节发白 / 脊背挺直 |

> 禁止：同一心理连续用 3 个以上生理映射，会变堆砌。1-2 个足够，配合动作即可。

## 四、附录 B · 角色语言指纹规范

> 每个主要角色必须有「语言指纹」，写在 `02_角色/protagonist.md` 与 `.state/characters/*.json` 中。
> polisher 必须按指纹校验角色台词，防止所有角色说话一个味。

### 4.1 指纹字段

| 字段 | 类型 | 说明 | 示例（主角） |
|---|---|---|---|
| `avg_sentence_length` | int | 角色台词平均句长（字） | 12 |
| `preferred_words` | string[] | 角色高频使用的词 | ["罢了","何须","且看"] |
| `catchphrases` | string[] | 口头禅 | ["且慢"] |
| `forbidden_words` | string[] | 角色绝不会说的词 | ["牛逼","卧槽"] |
| `address_habits` | object | 称呼习惯：对不同人的称呼 | {"师傅":"师尊","敌人":"阁下"} |

### 4.2 校验规则

- 主角 `avg_sentence_length` 偏差超过 ±30% 视为跑味。
- 台词中出现 `forbidden_words` 中任意一个，直接打回重写。
- 称呼不符合 `address_habits`，打回。

### 4.3 示例指纹（主角模板）

```json
{
  "avg_sentence_length": 12,
  "preferred_words": ["罢了", "何须", "且看"],
  "catchphrases": ["且慢"],
  "forbidden_words": ["牛逼", "卧槽", "yyds"],
  "address_habits": {
    "师傅": "师尊",
    "同门": "师兄/师弟",
    "敌人": "阁下",
    "陌生人": "这位"
  }
}
```
